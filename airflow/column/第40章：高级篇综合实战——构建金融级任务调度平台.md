# 第40章：高级篇综合实战——构建金融级任务调度平台

## 项目背景

金鹏金融科技公司是一家持有央行颁发的企业征信牌照的创新型金融科技企业，核心业务涵盖信贷风控建模、合规审计报表生成、交易反欺诈实时检测等。随着业务从单产品线扩展到12条产品线，原有的"Jenkins + Cron + Shell脚本"调度体系彻底崩溃——

- **合规风险**：审计员要求每一笔ETL任务都必须有完整的审计轨迹——谁在何时跑了什么任务、输入数据来自哪里（数据源版本）、输出数据写到了哪里（表名/文件路径/分区）、有没有异常中断。现有系统只是把Shell脚本跑完了事，没有任何结构化日志，审计员每次检查都在"手工考古"。去年一次审计花了整整三周时间整理材料，还是漏了三项关键操作的痕迹，差点被监管机构下发整改通知。

- **延迟失控**：3000+任务共享一台物理机，高峰期任务排队超过2小时。信贷部门的T+1风控报表必须在交易所开盘前（9:30）生成并发送给交易台，但经常拖到上午10点才跑完——等于是拿着昨天的风控数据在做今天的交易决策，合规风险巨大。原因很简单：凌晨1点有大批ETL同时触发，而机器只有48核，排队是必然的。

- **审批缺口**：涉及个人征信原始数据的处理任务，按照《个人信息保护法》要求必须有数据保护官（DPO）的审批。但现有的流程是"企业微信里吼一声 → DPO回复'同意' → 开发手动执行"，没有电子化留痕、没有审批时效追踪、没有驳回原因记录。一旦出问题，无法还原完整的决策链。

- **SLA无保障**：反欺诈模型每天必须在凌晨4:30前完成训练并上线，否则当日交易的反欺诈检测就会使用过期的模型——这意味着欺诈交易的拦截率可能从98%降到85%。但没有任何自动化机制在4:30到来时主动检查——全靠值班工程师手动盯着监控大盘。有一天值班工程师睡着了，模型直到早上6点才上线，那天的欺诈拦截率暴跌。

CTO拍板：自研一套金融级任务调度平台，基于Apache Airflow为核心引擎，但需要大量的定制化扩展来满足金融行业的特殊需求——合规审计、多级审批、SLA强制熔断、数据血缘追踪。小胖被任命为平台架构师，小白负责核心开发，大师担任技术顾问。

目标指标：支撑10000+ Dag、日均百万级TaskInstance、P99调度延迟会<5秒、年可用性>99.99%、审计报告覆盖率100%、审批流程100%电子化。

## 项目设计

周一上午9点，架构评审会在会议室召开。墙上贴满了12条产品线的业务流程拓扑图。大师在白板前放下马克笔，打开了投影仪。

**大师**："金融级调度平台和普通数据调度平台有本质区别——它不只是'把任务跑起来'，而是要同时满足**可靠、合规、可观测、可追溯**四个维度。我们需要在Airflow的基础上做四个核心扩展：

**第一，自定义Executor**：现有Celery Executor基于Redis/RabbitMQ，消息丢失风险不可接受。金鹏有自己的内部消息中间件——'鹏讯'（基于Raft协议自研的可靠消息队列），我们要写一个`PengxinExecutor`，支持任务优先级抢占和消息的高可靠投递。

**第二，合规审计Provider**：每次DagRun结束后自动生成一份审计报告，记录：执行人、执行时间、输入数据源（含数据指纹MD5）、输出表/文件路径、数据行数变化、是否有异常中断、审批人信息。报告导出为PDF并存入对象存储，审计系统可以随时拉取。

**第三，多级审批工作流**：将人工审批节点嵌入Dag执行流程中。利用`@task.branch`实现审批分支，配合`ExternalTaskSensor`实现审批状态的外部监听。审批操作通过企业微信机器人回调触发。

**第四，SLA强制熔断器**：每个Dag都有SLA时间窗口。如果超时——自动终止所有运行中的Task → 触发PagerDuty通知 → 自动切换备份执行链路（如从Spark切换到Presto）。这个熔断逻辑必须内嵌在Executor层，不能依赖外部监控。

另外，我们还需要一个**数据血缘追踪**系统——利用OpenLineage协议自动捕获任务间的数据流转关系，配合自定义Asset Backend实现数据集的版本化管理。"

**小胖**（在白板上快速做加法）："四个核心扩展加上血缘追踪……再加上10000+Dag的调度能力需求，这个工作量不小。我们分三期来做？"

**大师**点头："分三期合理。第一期攻坚Executor和审批流，第二期做审计Provider和SLA熔断，第三期上线血缘追踪和性能极致优化。每一期都要做完整的压测。"

**小白**："那我们从自定义Executor开始吧——我早就想写一个试试了！"

## 项目实战

### Part 1：自定义Executor——PengxinExecutor

"鹏讯"是金鹏自研的基于Raft协议的消息队列，保证消息的高可靠投递（at-least-once）。我们需要实现Airflow的BaseExecutor接口：

```python
# airflow_custom/executors/pengxin_executor.py
from __future__ import annotations

import json
import logging
import time
from queue import PriorityQueue
from typing import Any, Dict, Tuple

from airflow.executors.base_executor import BaseExecutor
from airflow.utils.state import TaskInstanceState

# 鹏讯消息队列SDK（金鹏内部库）
from pengxin import PengxinProducer, PengxinConsumer, Message

log = logging.getLogger(__name__)

PRIORITY_MAP = {
    "critical": 1,   # 最高优先级：反欺诈、风控
    "high": 2,       # 高优先级：T+1报表
    "medium": 3,     # 中优先级：常规ETL
    "low": 4,        # 低优先级：归档、清理
}


class PengxinExecutor(BaseExecutor):
    """
    基于鹏讯消息队列的自定义Executor
    特性：任务优先级抢占、消息高可靠投递、Worker心跳超时自动重入队
    """

    def __init__(self, parallelism: int = 32):
        super().__init__(parallelism=parallelism)
        self.producer: PengxinProducer | None = None
        self.consumer: PengxinConsumer | None = None
        self.topic = "airflow.task.command"
        self.result_topic = "airflow.task.result"
        # 存储已下发但未确认的任务，用于心跳超时检测
        self.outstanding_commands: Dict[str, float] = {}

    def start(self):
        """启动Executor——初始化鹏讯生产者和消费者"""
        self.producer = PengxinProducer(
            brokers=self._get_broker_list(),
            ack_mode="raft_quorum",  # Raft多数派确认，保证不丢消息
            compression="lz4",
        )
        self.consumer = PengxinConsumer(
            brokers=self._get_broker_list(),
            group_id="airflow-pengxin-executor",
            auto_offset_reset="earliest",
        )
        self.consumer.subscribe([self.result_topic])

    def _get_broker_list(self) -> list[str]:
        """获取鹏讯 Broker 列表（从Airflow配置读）"""
        from airflow.configuration import conf
        return conf.get("pengxin", "broker_list").split(",")

    def execute_async(
        self,
        key: str,
        command: Any,
        queue: str | None = None,
        executor_config: Any | None = None,
    ) -> None:
        """
        异步执行任务命令——发送到鹏讯消息队列

        参数：
            key: TaskInstance的唯一标识（dag_id + task_id + run_id + try_number）
            command: 序列化后的任务执行命令
            queue: 队列名（用于Worker组隔离）
            executor_config: 执行器特定配置（如优先级、资源需求）
        """
        # 解析优先级
        priority = PRIORITY_MAP.get(
            executor_config.get("priority", "medium") if executor_config else "medium",
            3,
        )

        # 构建消息
        message = Message(
            topic=self.topic,
            key=key,
            value=json.dumps({
                "command": command,
                "queue": queue or "default",
                "priority": priority,
                "timestamp": time.monotonic(),
            }),
            headers={
                "priority": str(priority),
                "source": "airflow-scheduler",
                "retry_count": "0",
                "max_retries": str(executor_config.get("max_retries", 3) if executor_config else 3),
            },
        )

        # 发送消息（按priority分区，高优先级进独立分区优先消费）
        self.producer.send(message, partition_key=str(priority))
        self.outstanding_commands[key] = time.monotonic()
        log.info("Task dispatched: key=%s, priority=%s, queue=%s", key, priority, queue)

    def sync(self) -> None:
        """
        同步任务状态——从鹏讯消费Worker上报的执行结果

        这是Executor的主循环钩子，由Scheduler在每个调度周期末尾调用。
        """
        if not self.consumer:
            return

        # 拉取一批结果消息
        messages = self.consumer.poll(timeout_ms=1000, max_records=500)

        for msg in messages:
            result = json.loads(msg.value)
            key = result["key"]
            state = result["state"]

            if key in self.outstanding_commands:
                del self.outstanding_commands[key]

            if state == "success":
                self.success(key)
            elif state == "failed":
                self.fail(key)
            elif state == "up_for_retry":
                self.queue_command(
                    key,
                    result["command"],
                    priority=result.get("priority", 3),
                    queue=result.get("queue", "default"),
                )

            self.consumer.commit(msg)

        # 心跳超时检测：超过300秒未收到结果的，标记为失败并触发重试
        now = time.monotonic()
        timeout_keys = [
            k for k, t in self.outstanding_commands.items()
            if now - t > 300
        ]
        for key in timeout_keys:
            log.warning("Task heartbeat timeout, marking as failed: %s", key)
            self.fail(key)
            del self.outstanding_commands[key]

        # 触发新任务入队
        self.trigger_tasks(open_slots=self.parallelism)

    def end(self) -> None:
        """优雅关闭——等待所有未确认消息超时"""
        self.producer.flush(timeout=30)
        self.producer.close()
        self.consumer.close()

    def terminate(self):
        """强制终止"""
        super().terminate()
```

### Part 2：合规审计Provider

自定义Airflow Provider，每次DagRun完成时自动生成审计报告：

```python
# airflow_custom/providers/compliance/audit.py
from __future__ import annotations

import hashlib
import json
import logging
from datetime import datetime, timezone
from pathlib import Path
from typing import Any, Dict, List, Optional

from airflow.providers_manager import ProvidersManager
from airflow.sdk import DAG
from airflow.utils.state import DagRunState

log = logging.getLogger(__name__)


class AuditRecord:
    """单条审计记录的数据模型"""

    def __init__(
        self,
        dag_id: str,
        run_id: str,
        logical_date: datetime,
        executed_by: str,
        approved_by: Optional[str],
        input_sources: List[Dict[str, str]],
        output_targets: List[Dict[str, str]],
        start_time: datetime,
        end_time: datetime,
        state: str,
        task_records: List[Dict[str, Any]],
    ):
        self.dag_id = dag_id
        self.run_id = run_id
        self.logical_date = logical_date
        self.executed_by = executed_by
        self.approved_by = approved_by
        self.input_sources = input_sources
        self.output_targets = output_targets
        self.start_time = start_time
        self.end_time = end_time
        self.state = state
        self.task_records = task_records
        self.record_id = hashlib.sha256(
            f"{dag_id}:{run_id}:{logical_date.isoformat()}".encode()
        ).hexdigest()[:16]
        self.generated_at = datetime.now(timezone.utc)


class ComplianceAuditHook:
    """
    合规审计Hook——负责收集DagRun执行数据并生成审计报告。
    作为Airflow Provider Hook实现，与DagRun生命周期紧密集成。
    """

    def __init__(self, cos_conn_id: str = "compliance_cos"):
        self.cos_conn_id = cos_conn_id
        self.template_path = Path(__file__).parent / "templates" / "audit_report.html"

    def collect_dagrun_audit_data(
        self, dag_id: str, run_id: str
    ) -> AuditRecord:
        """
        从Metadata DB中收集DagRun的完整执行信息
        包括：输入输出源、任务状态、耗时、重试记录等
        """
        from airflow.models.dagrun import DagRun
        from airflow.models.taskinstance import TaskInstance
        from airflow.utils.session import create_session

        with create_session() as session:
            dag_run = session.query(DagRun).filter_by(
                dag_id=dag_id, run_id=run_id
            ).one_or_none()

            if not dag_run:
                raise ValueError(f"DagRun not found: {dag_id}/{run_id}")

            task_instances = session.query(TaskInstance).filter_by(
                dag_id=dag_id, run_id=run_id
            ).all()

            # 解析input/output（从Dag文件的docstring或自定义标注中提取）
            dag = dag_run.get_dag()
            input_sources = self._extract_sources(dag, "input")
            output_targets = self._extract_sources(dag, "output")

            task_records = []
            for ti in task_instances:
                task_records.append({
                    "task_id": ti.task_id,
                    "state": ti.state,
                    "start_date": ti.start_date.isoformat() if ti.start_date else None,
                    "end_date": ti.end_date.isoformat() if ti.end_date else None,
                    "duration": ti.duration,
                    "try_number": ti.try_number,
                    "max_tries": ti.max_tries,
                    "hostname": ti.hostname,
                    "pool": ti.pool,
                    "priority_weight": ti.priority_weight,
                })

            # 提取审批人信息（从DagRun的conf中）
            conf = dag_run.conf or {}
            approved_by = conf.get("approved_by")

            return AuditRecord(
                dag_id=dag_id,
                run_id=run_id,
                logical_date=dag_run.logical_date,
                executed_by=dag_run.triggered_by or conf.get("executed_by", "system"),
                approved_by=approved_by,
                input_sources=input_sources,
                output_targets=output_targets,
                start_time=dag_run.start_date,
                end_time=dag_run.end_date,
                state=dag_run.state,
                task_records=task_records,
            )

    def _extract_sources(self, dag: DAG, source_type: str) -> List[Dict[str, str]]:
        """从Dag对象的自定义属性中提取输入/输出源"""
        if source_type == "input":
            raw = getattr(dag, "compliance_inputs", [])
        else:
            raw = getattr(dag, "compliance_outputs", [])

        results = []
        for src in raw:
            # 计算数据指纹（如果是文件/表的话）
            fingerprint = hashlib.md5(
                json.dumps(src, sort_keys=True).encode()
            ).hexdigest()
            results.append({
                "type": src.get("type", "unknown"),
                "path": src.get("path", ""),
                "table": src.get("table", ""),
                "fingerprint": fingerprint,
                "timestamp": datetime.now(timezone.utc).isoformat(),
            })
        return results

    def generate_pdf_report(self, record: AuditRecord) -> bytes:
        """生成审计报告PDF"""
        from weasyprint import HTML

        html_content = self._render_template(record)
        pdf_bytes = HTML(string=html_content).write_pdf()
        return pdf_bytes

    def _render_template(self, record: AuditRecord) -> str:
        """渲染审计报告HTML模板"""
        with open(self.template_path, "r", encoding="utf-8") as f:
            template = f.read()

        # 计算执行时长
        duration = ""
        if record.start_time and record.end_time:
            secs = (record.end_time - record.start_time).total_seconds()
            duration = f"{int(secs // 60)}分{int(secs % 60)}秒"

        return template.format(
            record_id=record.record_id,
            dag_id=record.dag_id,
            run_id=record.run_id,
            logical_date=record.logical_date.strftime("%Y-%m-%d %H:%M:%S"),
            executed_by=record.executed_by,
            approved_by=record.approved_by or "N/A（自动执行）",
            state=record.state,
            state_class=("success" if record.state == "success" else "failure"),
            duration=duration,
            generated_at=record.generated_at.strftime("%Y-%m-%d %H:%M:%S UTC"),
            input_sources=self._format_sources(record.input_sources),
            output_targets=self._format_sources(record.output_targets),
            task_summary=self._format_task_summary(record.task_records),
            total_tasks=len(record.task_records),
            success_tasks=sum(1 for t in record.task_records if t["state"] == "success"),
            failed_tasks=sum(1 for t in record.task_records if t["state"] == "failed"),
        )

    def _format_sources(self, sources: List[Dict]) -> str:
        rows = []
        for s in sources:
            rows.append(
                f'<tr><td>{s["type"]}</td><td>{s.get("path") or s.get("table")}</td>'
                f'<td>{s["fingerprint"][:8]}</td></tr>'
            )
        return "\n".join(rows)

    def _format_task_summary(self, tasks: List[Dict]) -> str:
        rows = []
        for t in tasks:
            state_color = "#28a745" if t["state"] == "success" else "#dc3545" if t["state"] == "failed" else "#ffc107"
            rows.append(
                f'<tr>'
                f'<td>{t["task_id"]}</td>'
                f'<td style="color:{state_color}">{t["state"]}</td>'
                f'<td>{t["start_date"] or "N/A"}</td>'
                f'<td>{t["end_date"] or "N/A"}</td>'
                f'<td>{t["duration"]:.1f}s</td>'
                f'<td>{t["try_number"]}/{t["max_tries"]}</td>'
                f'</tr>'
            )
        return "\n".join(rows)

    def upload_to_cos(self, record: AuditRecord, pdf_bytes: bytes) -> str:
        """上传审计报告PDF到对象存储"""
        from airflow.providers.amazon.aws.hooks.s3 import S3Hook

        s3_hook = S3Hook(aws_conn_id=self.cos_conn_id)
        key = (
            f"compliance/audit-reports/"
            f"{record.logical_date.strftime('%Y/%m/%d')}/"
```

### Part 4：SLA强制熔断器

熔断器内嵌在执行层，每个TaskInstance启动时自动注册SLA倒计时：

```python
# airflow_custom/sla/circuit_breaker.py
from __future__ import annotations

import logging
import signal
import threading
from datetime import datetime, timedelta, timezone
from typing import Optional

import requests

log = logging.getLogger(__name__)

# SLA熔断配置
SLA_CONFIG = {
    "risk_model_training": timedelta(hours=2),
    "daily_report_t0": timedelta(minutes=30),
    "fraud_detection": timedelta(minutes=15),
    "regulatory_report": timedelta(hours=1),
}


class SLACircuitBreaker:
    """
    SLA强制熔断器——超时自动终止 + 通知 + 切换备份链路

    设计理念：
    1. 在Task执行线程中启动一个独立的倒计时Timer
    2. 超时时自动发送SIGTERM给Worker进程组
    3. 同时触发PagerDuty告警
    4. 通知备份Executor接管后续任务
    """

    def __init__(self, dag_id: str, run_id: str, task_id: str):
        self.dag_id = dag_id
        self.run_id = run_id
        self.task_id = task_id
        self.deadline: Optional[datetime] = None
        self._timer: Optional[threading.Timer] = None
        self._breaked = False

    @classmethod
    def from_dag_id(cls, dag_id: str, run_id: str, task_id: str, start_time: datetime):
        """工厂方法——根据DagId从SLA配置表中获取时间窗口"""
        instance = cls(dag_id, run_id, task_id)
        sla_window = SLA_CONFIG.get(dag_id.lower(), timedelta(hours=2))
        instance.deadline = start_time + sla_window
        return instance

    def arm(self) -> None:
        """启动SLA倒计时"""
        if not self.deadline:
            log.warning("SLA deadline not set, skipping circuit breaker for %s", self.dag_id)
            return

        remaining = (self.deadline - datetime.now(timezone.utc)).total_seconds()
        if remaining <= 0:
            self._trigger_break()
            return

        self._timer = threading.Timer(remaining, self._trigger_break)
        self._timer.daemon = True
        self._timer.start()
        log.info(
            "SLA Circuit Breaker armed: dag=%s run=%s task=%s deadline=%s remaining=%ds",
            self.dag_id, self.run_id, self.task_id,
            self.deadline.isoformat(), int(remaining),
        )

    def _trigger_break(self) -> None:
        """熔断触发——执行多层兜底逻辑"""
        if self._breaked:
            return
        self._breaked = True

        log.critical(
            "SLA BREACHED! Dag=%s Run=%s Task=%s, triggering circuit breaker...",
            self.dag_id, self.run_id, self.task_id,
        )

        # 第一步：发送SIGTERM终止当前Worker进程
        self._terminate_worker()

        # 第二步：PagerDuty告警
        self._notify_pagerduty()

        # 第三步：触发备份链路
        self._activate_backup_link()

    def _terminate_worker(self) -> None:
        """终止当前Worker进程"""
        log.warning("Terminating Worker process due to SLA breach...")
        import os
        os.kill(os.getpid(), signal.SIGTERM)

    def _notify_pagerduty(self) -> None:
        """通过PagerDuty发送紧急告警"""
        try:
            payload = {
                "routing_key": "PD_ROUTING_KEY_AIRFLOW_SLA",
                "event_action": "trigger",
                "payload": {
                    "summary": f"SLA-BREACH: {self.dag_id}/{self.task_id}",
                    "severity": "critical",
                    "source": "airflow-sla-circuit-breaker",
                    "component": self.dag_id,
                    "group": self.run_id,
                    "custom_details": {
                        "dag_id": self.dag_id,
                        "run_id": self.run_id,
                        "task_id": self.task_id,
                        "deadline": self.deadline.isoformat(),
                        "current_time": datetime.now(timezone.utc).isoformat(),
                    },
                },
            }
            requests.post(
                "https://events.pagerduty.com/v2/enqueue",
                json=payload,
                timeout=5,
            )
        except Exception:
            log.exception("Failed to notify PagerDuty")

    def _activate_backup_link(self) -> None:
        """激活备份执行链路——例如从Spark切换到Presto"""
        try:
            requests.post(
                "http://airflow-web:8080/api/v1/dags/backup_link_activator/dagRuns",
                json={
                    "conf": {
                        "failed_dag_id": self.dag_id,
                        "failed_run_id": self.run_id,
                        "backup_dag_id": f"{self.dag_id}_presto_fallback",
                        "logical_date": datetime.now(timezone.utc).isoformat(),
                    }
                },
                headers={"Authorization": f"Bearer {AIRFLOW_API_TOKEN}"},
                timeout=5,
            )
        except Exception:
            log.exception("Failed to activate backup link")

    def disarm(self) -> None:
        """任务正常完成——取消SLA倒计时"""
        if self._timer and not self._breaked:
            self._timer.cancel()
            log.info("SLA Circuit Breaker disarmed: dag=%s task=%s", self.dag_id, self.task_id)
```

### Part 5：OpenLineage数据血缘追踪

集成OpenLineage协议实现自动血缘采集：

```python
# airflow_custom/lineage/openlineage_adapter.py
from __future__ import annotations

from typing import Any, Dict, List, Optional

from openlineage.client import OpenLineageClient
from openlineage.client.facet import (
    JobFacet, RunFacet, InputDatasetFacet, OutputDatasetFacet,
    SchemaField, DocumentationJobFacet,
)
from openlineage.client.run import RunEvent, RunState, Job, Run, Dataset


class OpenLineageAdapter:
    """
    OpenLineage适配器——将Airflow的任务执行事件转换为血缘事件
    """

    def __init__(self, lineage_url: str = "http://marquez:5000"):
        self.client = OpenLineageClient(url=lineage_url)

    def emit_task_start(
        self,
        dag_id: str,
        run_id: str,
        task_id: str,
        inputs: List[Dict[str, str]],
        outputs: List[Dict[str, str]],
    ) -> None:
        """任务开始时发送START事件，声明输入/输出数据集"""
        run = Run(runId=f"{dag_id}.{run_id}.{task_id}")
        job = Job(
            namespace="fintech-airflow",
            name=f"{dag_id}.{task_id}",
            facets={
                "documentation": DocumentationJobFacet(
                    description=f"Finance data pipeline task: {dag_id}/{task_id}"
                )
            },
        )

        input_datasets = [self._build_dataset(ds) for ds in inputs]
        output_datasets = [self._build_dataset(ds) for ds in outputs]

        event = RunEvent(
            eventType=RunState.START,
            eventTime=self._now(),
            run=run,
            job=job,
            inputs=input_datasets,
            outputs=output_datasets,
        )
        self.client.emit(event)

    def emit_task_complete(
        self,
        dag_id: str,
        run_id: str,
        task_id: str,
        row_count: Optional[int] = None,
        bytes_written: Optional[int] = None,
    ) -> None:
        """任务完成时发送COMPLETE事件，携带数据量统计"""
        run = Run(runId=f"{dag_id}.{run_id}.{task_id}")
        job = Job(namespace="fintech-airflow", name=f"{dag_id}.{task_id}")

        facets = {}
        if row_count is not None:
            facets["outputStatistics"] = OutputDatasetFacet(
                rowCount=row_count,
                size=bytes_written or 0,
            )

        output_datasets = [Dataset(
            namespace="fintech-airflow",
            name=f"airflow.{dag_id}.{task_id}",
            facets=facets,
        )]

        event = RunEvent(
            eventType=RunState.COMPLETE,
            eventTime=self._now(),
            run=run,
            job=job,
            outputs=output_datasets,
        )
        self.client.emit(event)

    def _build_dataset(self, source: Dict[str, str]) -> Dataset:
        """将Airflow的输入/输出描述转换为OpenLineage Dataset"""
        namespace = source.get("type", "unknown")
        name = source.get("path") or source.get("table") or source.get("name", "unknown")
        return Dataset(namespace=namespace, name=name)

    def _now(self) -> str:
        return datetime.now(timezone.utc).isoformat()
```

## 项目总结

从零构建一个金融级任务调度平台的旅程，让我们深刻认识到：**Airflow是一个优秀的调度引擎，但成为一个企业级平台，需要围绕它构建一整套生态系统**——自定义Executor解决消息可靠性、合规审计Provider满足监管要求、审批工作流填补人机协作的空白、SLA熔断器守护业务底线、OpenLineage实现数据的可追溯性。

平台上线三个月后，金鹏的年度外部审计如期而至。这一次，审计员不再抱着一堆纸质材料满办公室追问"这个任务是谁执行的"，而是直接打开审计系统的查询界面——输入Dag名称和时间范围，PDF审计报告即时生成。每一份报告都清晰地记录了：执行人、审批人、输入数据指纹、输出数据路径、任务耗时、异常记录。审计员在结束检查时特意给小胖团队发了一封邮件："过去三年，我们是第一家能在48小时内完成数据处理合规审计的金融机构。你们的平台让合规不再是负担。"

这就是工程的价值——用技术让规则自动执行，让人类专注于创造。从技术视角回望，这个平台的成功交付验证了几个核心理念：

1. **开放架构胜过单一产品**：我们不是"自研了一个取代Airflow的系统"，而是在Airflow的开放接口（BaseExecutor、Provider Framework、Callback机制）之上构建了金融专属的扩展层。这种"核心开源+外围定制"的策略，既借力了社区的持续迭代，又保留了业务的灵活性。

2. **合规是能力，不是负担**：很多团队把合规审计视为"额外工作"，但我们将其内嵌为平台的一等公民——不是等Dag跑完了再手工整理材料，而是Dag运行过程中自动采集、自动生成、自动归档。合规从来不是"要不要做"的问题，而是"能不能自动化"的问题。

3. **SLA是契约，不是建议**：金融场景中，"尽量在4:30前完成"和"必须在4:30前完成，否则自动切换到备份链路"有着本质区别。前者是工程团队的主观承诺，后者是系统的客观保障。熔断器的价值不在于"它经常触发"，而在于"当它触发时，你知道有兜底方案"。

4. **可观测性是安全网**：审批工作流、血缘追踪、审计报告——这些看似独立的功能，在架构上其实是同一个"可观测性"目标的不同侧面。知道谁执行了什么、数据从哪里来到哪里去、在什么时间点被谁审批——这是所有金融级系统的共同需求。

当平台稳定运行六个月后，小胖在季度技术分享会上说了这样一句话："很多人问我，为什么要花三个月时间自研这么多组件，而不是直接用开源方案的默认配置？我的回答是——开源的Airflow给了一幅地图，但要到达'金融级任务调度平台'这个目的地，你得自己铺路、搭桥、设护栏。而那次凌晨三点的PagerDuty告警告诉我——这条路，值得走。"

### 思考题

1. PengxinExecutor的消息投递使用Raft Quorum模式保证可靠性，但在极端情况下（如网络分区导致Raft无法选出Leader），Executor应该如何优雅降级？是阻塞等待直到Raft恢复，还是切换到本地磁盘队列做临时缓冲？请设计一种"滑动降级"策略——根据消息优先级级别和网络分区持续时间自动选择降级模式。

2. 在SLA熔断器的设计中，熔断后的备份链路是一个预设的替代Dag。但在复杂的金融场景中，备份Dag往往只能覆盖主链路70%的功能（剩下的30%可能是非核心的监控、日志、审计等）。如何设计一个"部分熔断"机制——仅终止超时的核心任务，而允许非核心的附属任务继续完成？请结合TaskGroup和动态依赖注入，给出具体方案。
