## 1 项目背景（约 500 字）

BI 团队对 **同一聚合查询**（按渠道、日）每天跑上百次，底层扫描 **TB 事实表**。DBA 评估 **Hive 物化视图（Materialized View）** 与 **自动查询重写**：基表刷新后 **重写是否仍安全**？**增量维护** 成本如何？本章界定 **适用 SQL 子集、刷新策略、与报表 SLA** 的匹配度，避免 **「以为自动加速」实际未命中**。

补充：**物化视图命中率的监控** 和 **报表 SLA** 要绑在同一张看板上。否则很容易出现「报表快了但结果旧了三天没人发现」——这在财务月结场景是不可接受的。建议对关键 MV 配置 **staleness 上限 + 自动禁用重写** 策略（参数名依版本）。

## 2 项目设计（约 1200 字）


> **角色（对齐 [template.md](../template.md)）**：**小胖**（生活化比喻、抛问题）· **小白**（原理、边界、风险与备选）· **大师**（选型与「**技术映射**」承接）。  
> **对话结构**：小胖开球 1～2 轮 → 小白追问 2～3 轮 → 大师解答并引出下一子话题；全文循环 **2～3 次**，覆盖本章核心概念。

**小胖**：物化视图是不是缓存？

**小白**：是 **持久化预计算结果** + **优化器重写规则**；缓存通常更短命且不改变语义路径。

**大师**：技术映射：**MV = 以空间/刷新成本换读加速**。

**小胖**：啥时会失效？

**小白**：基表 **INSERT/UPDATE** 后 **staleness** 超阈值、或 **不支持的重写模式**。

**大师**：要对 **可重复读口径** 做 **版本水位线** 管理。


> **§2·第三轮**（对齐 [template.md](../template.md) 的第三循环）

**小胖**：物化视图自动重写像「导航自动换近路」——会不会把我导进单行道？

**小白**：会：**语义不等价的重写**、**过时物化**、**hint 与重写冲突** 都是真实坑；需要 **白名单 SQL + 回归集**。

**大师**：上线策略用 **影子查询对比结果集 hash**（采样全量），再灰度打开重写；别默认全集群开启。

**技术映射**：**自动重写可信度 = 等价证明（或严格模式）× 物化新鲜度 × 回滚开关**。

## 3 项目实战（约 1500～2000 字）

> 语法以 Hive 版本为准；以下为学习路径。

### 步骤 1：阅读官方 MV 文档与 `hive高级话题.md` 相关节

列出 **支持的 join/aggregation 形态**。

### 步骤 2：创建 MV 骨架

```sql
CREATE MATERIALIZED VIEW IF NOT EXISTS mv_channel_daily
DISABLE REWRITE  -- 初学可先禁用，理解存储
AS
SELECT dt, channel, COUNT(*) cnt
FROM dwd_impression
GROUP BY dt, channel;
```

### 步骤 3：`EXPLAIN` 观察 **是否被重写**

对比 **enable/disable REWRITE** 计划差异。

**坑**：**非确定性函数** 导致 MV 不可用。  
**坑**：**分区裁剪与 MV 分区对齐** 失败。

**验证**：刷新作业后 **报表结果与基线 SQL diff** 为零。

### 步骤 4：刷新作业 SLA 设计（示例）

| MV 名称 | 允许最大延迟 | 刷新方式 | 失败降级 |
|---------|--------------|----------|----------|
| mv_channel_daily | 2h | 依赖分区就绪事件 | 关闭 rewrite + 告警 |

### 步骤 5：「是否命中重写」的快速验证

1. `EXPLAIN` 看是否引用 MV 物理路径（输出关键字因版本而异）。  
2. 在测试库 **rename MV** 使重写失败，确认报表 **报错而非静默错数**（取决于客户端配置）。

## 4 项目总结（约 500～800 字）

### 优点与缺点

| 优点 | 缺点 |
|------|------|
| 降重复扫描 | 刷新与存储成本 |
| 对 BI 透明（若重写成功） | 能力边界复杂 |
| 可与优化器统计联动 | 误用导致 **错误加速** 风险需治理 |

### 适用与不适用

- **适用**：高重复聚合、基表 append-only。  
- **不适用**：极高频变更基表且要求实时一致。  

### 注意事项

- **监控**：rewrite 命中率。  
- **权限**：MV 与基表权限继承。  

### 常见生产踩坑

1. **未刷新** 被重写 → **旧数据**。  
2. **隐式类型** 导致 MV 与查询 **模式不匹配**。  
3. **多引擎** 仅 Hive 识别 MV。

### 思考题

1. MV 与 **Iceberg 快照** 的一致性模型差异？  
2. 如何用 **定时校验任务** 防重写错误？  
3. 若 BI 工具生成 SQL **带大量隐式 cast**，MV 重写常失败——你如何推动「SQL 生成器」改造？
