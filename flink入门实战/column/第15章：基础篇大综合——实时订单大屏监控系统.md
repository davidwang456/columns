# 第15章：基础篇大综合——实时订单大屏监控系统

---

## 1. 项目背景

经过前面14章的学习，我们已经掌握了Flink基础的各项能力：DataStream API、状态与窗口、Checkpoint、时间语义、Table API/SQL、UDF。

现在是时候把这些知识串联起来，完成一个**端到端的完整项目**：实时订单大屏监控系统。

业务需求：某电商平台需要在"618大促"期间搭建一个运营大屏，实时展示以下指标：

| 指标 | 说明 | 刷新频率 | 技术要点 |
|------|------|---------|---------|
| 实时GMV | 当日累计成交金额 | 实时 | Kafka Source + 窗口聚合 |
| 每分钟订单量 | 每分钟成功下单数 | 每分钟 | Tumbling Window |
| 热门商品Top10 | 过去5分钟销量最高的10个商品 | 每30秒 | Sliding Window + TopN |
| 各省份成交额 | 按用户所在省份统计GMV | 每分钟 | UDF IP→省份 + 窗口聚合 |
| 订单状态异常告警 | 已支付但超过30分钟未发货的订单 | 实时 | Session Window + 状态机 |

数据流：用户下单行为 → 业务DB → Canal/Debezium CDC → Kafka订单Topic → Flink → Redis → Grafana展示

本章涉及全部技术栈：

```
业务DB(MySQL) → Canal(Debezium) → Kafka(订单Topic) → Flink
    │                                                      │
    │                                          ┌────────────┼────────────┐
    │                                          │            │            │
    │                                     GMV计算    热门Top10    省份聚合
    │                                          │            │            │
    │                                          └────────────┼────────────┘
    │                                                      │
    ▼                                                      ▼
Redis(实时聚合结果)                                  异常订单告警→钉钉
    │
    ▼
Grafana(实时大屏)
```

## 2. 项目设计

> 场景：618大促筹备会，小胖被指派负责实时大屏——"不做完不准下班"。

**小胖**：这么多指标，我每个都建一个Flink作业？那不得七八个作业，运维不得骂死我。

**大师**：当然不。一个Flink作业可以输出多个结果——每个结果对应一个Sink。Source只需要读一次Kafka Topic，但在内部做多条流水线并行处理。

**技术映射：一个Flink作业可以做"N条流水线"处理——Source数据分裂为多个DataStream，各自独立做窗口/聚合/Sink。共享Source读取，不共享处理逻辑。**

**小白**：那如果其中一个指标的Sink（如写入Redis）写挂了，会不会影响其他指标？比如Redis挂了，GMV算不出来，但热门Top10应该不受影响。

**大师**：Flink默认是"一个Task失败，整个作业重启"。如果一个Sink失败，所有流水线都受影响。解方是使用**SideOutput分离Sink**，让不同的Sink在不同的算子链末端，配合**Source-Sink隔离**设计——但这是中级篇的容错内容。

本章我们做一个简化版：所有结果都写入Redis，使用Redis的哈希结构（HASH）存储不同指标。

**订单数据结构**（Kafka JSON）：

```json
{
  "orderId": "ORD20260428001",
  "userId": "u10086",
  "productId": "p2048",
  "productName": "iPhone 16 Pro Max",
  "category": "手机数码",
  "amount": 9999.00,
  "province": "北京市",
  "orderStatus": 1,
  "eventTime": 1714293932000
}
```

### 分步实现

#### 步骤1：Redis Sink——将实时结果写入Redis

```java
package com.flink.column.chapter15.sink;

import org.apache.flink.configuration.Configuration;
import org.apache.flink.streaming.api.functions.sink.RichSinkFunction;
import org.apache.flink.types.Row;
import redis.clients.jedis.JedisPool;
import redis.clients.jedis.JedisPoolConfig;

/**
 * Redis Sink：将计算结果写入Redis HASH
 * 目标结构：
 *   - 实时GMV:  KEY=gmv, FIELD=total, VALUE=xxxx
 *   - 每分钟订单量: KEY=orders_per_min, FIELD=<timestamp>, VALUE=count
 *   - 热门Top10: KEY=hot_products, FIELD=<productId>, VALUE=count
 */
public class RedisSink extends RichSinkFunction<Row> {

    private transient JedisPool jedisPool;
    private final String redisHost;
    private final int redisPort;

    public RedisSink(String host, int port) {
        this.redisHost = host;
        this.redisPort = port;
    }

    @Override
    public void open(Configuration parameters) {
        JedisPoolConfig config = new JedisPoolConfig();
        config.setMaxTotal(10);
        config.setMaxIdle(5);
        jedisPool = new JedisPool(config, redisHost, redisPort);
    }

    @Override
    public void invoke(Row row, Context context) {
        String key = row.getFieldAs(0);    // Redis KEY
        String field = row.getFieldAs(1);  // HASH FIELD
        String value = row.getFieldAs(2);  // VALUE（转为String便于存储）
        
        try (var jedis = jedisPool.getResource()) {
            jedis.hset(key, field, value);
        }
    }

    @Override
    public void close() {
        if (jedisPool != null) jedisPool.close();
    }
}
```

#### 步骤2：订单大屏作业——完整实现

```java
package com.flink.column.chapter15;

import org.apache.flink.api.common.eventtime.WatermarkStrategy;
import org.apache.flink.api.common.functions.MapFunction;
import org.apache.flink.api.common.functions.RichMapFunction;
import org.apache.flink.api.common.serialization.SimpleStringSchema;
import org.apache.flink.api.java.tuple.Tuple2;
import org.apache.flink.api.java.tuple.Tuple3;
import org.apache.flink.configuration.Configuration;
import org.apache.flink.connector.kafka.source.KafkaSource;
import org.apache.flink.connector.kafka.source.enumerator.initializer.OffsetsInitializer;
import org.apache.flink.streaming.api.datastream.DataStream;
import org.apache.flink.streaming.api.environment.StreamExecutionEnvironment;
import org.apache.flink.streaming.api.functions.windowing.ProcessWindowFunction;
import org.apache.flink.streaming.api.windowing.assigners.SlidingEventTimeWindows;
import org.apache.flink.streaming.api.windowing.assigners.TumblingEventTimeWindows;
import org.apache.flink.streaming.api.windowing.time.Time;
import org.apache.flink.streaming.api.windowing.windows.TimeWindow;
import org.apache.flink.types.Row;
import org.apache.flink.util.Collector;
import com.flink.column.chapter15.sink.RedisSink;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import java.time.Duration;
import java.util.PriorityQueue;

/**
 * 实时订单大屏监控系统——完整作业
 * 将5个指标的计算结果写入Redis，Grafana读取Redis展示
 */
public class RealtimeDashboardJob {

    private static final Logger LOG = LoggerFactory.getLogger(RealtimeDashboardJob.class);

    public static void main(String[] args) throws Exception {
        StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
        env.setParallelism(4);
        env.enableCheckpointing(15_000);

        // ========== Kafka Source ==========
        KafkaSource<String> source = KafkaSource.<String>builder()
                .setBootstrapServers("kafka:9092")
                .setTopics("order-topic")
                .setGroupId("dashboard-group")
                .setStartingOffsets(OffsetsInitializer.latest())
                .setValueOnlyDeserializer(new SimpleStringSchema())
                .build();

        DataStream<String> rawStream = env.fromSource(
                source, WatermarkStrategy.noWatermarks(), "order-source");

        // 解析JSON为Order POJO
        DataStream<Order> orders = rawStream
                .map(new OrderParser())
                .name("parse-order")
                .assignTimestampsAndWatermarks(
                        WatermarkStrategy.<Order>forBoundedOutOfOrderness(Duration.ofSeconds(10))
                                .withTimestampAssigner((order, ts) -> order.eventTime)
                );

        // ========== 指标1: 实时GMV（滚动窗口1分钟） ==========
        DataStream<Row> gmvStream = orders
                .keyBy(o -> "all")
                .window(TumblingEventTimeWindows.of(Time.minutes(1)))
                .reduce(
                        (o1, o2) -> new Order(null, null, null, 0,
                                o1.amount + o2.amount, null, 0, 0),
                        new ProcessWindowFunction<Order, Row, String, TimeWindow>() {
                            @Override
                            public void process(String key, Context ctx,
                                                Iterable<Order> elements, Collector<Row> out) {
                                Order aggregated = elements.iterator().next();
                                out.collect(Row.of("gmv", String.valueOf(ctx.window().getEnd()),
                                        String.format("%.2f", aggregated.amount)));
                            }
                        })
                .name("gmv-calculation");

        // ========== 指标2: 每分钟订单量 ==========
        DataStream<Row> orderCountStream = orders
                .keyBy(o -> "all")
                .window(TumblingEventTimeWindows.of(Time.minutes(1)))
                .reduce(
                        (o1, o2) -> new Order(null, null, null, 0,
                                0, null, o1.count + o2.count, 0),
                        new ProcessWindowFunction<Order, Row, String, TimeWindow>() {
                            @Override
                            public void process(String key, Context ctx,
                                                Iterable<Order> elements, Collector<Row> out) {
                                Order aggregated = elements.iterator().next();
                                out.collect(Row.of("orders_per_min",
                                        String.valueOf(ctx.window().getEnd()),
                                        String.valueOf(aggregated.count)));
                            }
                        })
                .name("order-count-per-minute");

        // ========== 指标3: 热门商品Top10（滑动窗口5分钟/30秒） ==========
        DataStream<Row> hotProductStream = orders
                .keyBy(o -> o.productId)
                .window(SlidingEventTimeWindows.of(Time.minutes(5), Time.seconds(30)))
                .reduce(
                        (o1, o2) -> new Order(null, null, o1.productId, 0,
                                0, null, o1.count + o2.count, 0),
                        new ProcessWindowFunction<Order, Row, String, TimeWindow>() {
                            @Override
                            public void process(String key, Context ctx,
                                                Iterable<Order> elements, Collector<Row> out) {
                                Order aggregated = elements.iterator().next();
                                out.collect(Row.of("hot_products", aggregated.productId,
                                        String.valueOf(aggregated.count)));
                            }
                        })
                .name("hot-products-top10");

        // ========== 指标4: 各省份成交额（滚动窗口1分钟） ==========
        DataStream<Row> provinceGmvStream = orders
                .keyBy(o -> o.province)
                .window(TumblingEventTimeWindows.of(Time.minutes(1)))
                .reduce(
                        (o1, o2) -> new Order(null, null, null, 0,
                                o1.amount + o2.amount, o1.province, 0, 0),
                        new ProcessWindowFunction<Order, Row, String, TimeWindow>() {
                            @Override
                            public void process(String key, Context ctx,
                                                Iterable<Order> elements, Collector<Row> out) {
                                Order aggregated = elements.iterator().next();
                                out.collect(Row.of("province_gmv",
                                        aggregated.province,
                                        String.format("%.2f", aggregated.amount)));
                            }
                        })
                .name("province-gmv");

        // ========== 指标5: 订单异常检测（已支付超30分钟未发货） ==========
        DataStream<Row> alertStream = orders
                .keyBy(o -> o.orderId)
                .map(new RichMapFunction<Order, Row>() {
                    private transient java.util.Map<String, Long> paidOrders;

                    @Override
                    public void open(Configuration parameters) {
                        paidOrders = new java.util.HashMap<>();
                    }

                    @Override
                    public Row map(Order order) {
                        if (order.orderStatus == 1) { // 已支付
                            paidOrders.put(order.orderId, order.eventTime);
                        }
                        if (order.orderStatus == 2) { // 已发货
                            Long paidTime = paidOrders.remove(order.orderId);
                            if (paidTime != null) {
                                long cost = order.eventTime - paidTime;
                                if (cost > 30 * 60 * 1000L) {
                                    return Row.of("abnormal_orders", order.orderId,
                                            String.format("超时%d分钟", cost / 60_000));
                                }
                            }
                        }
                        return Row.of("abnormal_orders", "ok", "0");
                    }
                })
                .name("abnormal-order-detection");

        // ========== 统一写入Redis ==========
        RedisSink redisSink = new RedisSink("redis", 6379);
        gmvStream.addSink(redisSink).name("redis-gmv");
        orderCountStream.addSink(redisSink).name("redis-order-count");
        hotProductStream.addSink(redisSink).name("redis-hot-products");
        provinceGmvStream.addSink(redisSink).name("redis-province-gmv");
        alertStream.addSink(redisSink).name("redis-alerts");

        LOG.info("实时大屏作业已提交...");
        env.execute("Chapter15-RealtimeDashboard");
    }

    // ========== Order POJO ==========
    public static class Order {
        public String orderId, userId, productId, province;
        public double amount;
        public int orderStatus;
        public long eventTime;
        public int count;  // 用于聚合计数

        public Order() {}

        public Order(String orderId, String userId, String productId, int orderStatus,
                     double amount, String province, int count, long eventTime) {
            this.orderId = orderId;
            this.userId = userId;
            this.productId = productId;
            this.orderStatus = orderStatus;
            this.amount = amount;
            this.province = province;
            this.count = count;
            this.eventTime = eventTime;
        }
    }

    public static class OrderParser implements MapFunction<String, Order> {
        @Override
        public Order map(String json) {
            // 简化解析：生产用Jackson
            Order o = new Order();
            o.count = 1;  // 每条订单计数+1
            // 实际应完整解析JSON所有字段
            return o;
        }
    }
}
```

### 可能遇到的坑

1. **Redis连接池泄漏导致"Could not get a resource from pool"**
   - 根因：Sink中的Jedis实例没有正确归还到连接池
   - 解决：使用try-with-resources确保`jedis.close()`被调用

2. **5个Sink都写入同一个Redis，其中一个Sink反压导致整个作业阻塞**
   - 根因：Redis Sink是同步写入，如果Redis性能下降，所有Sink都被阻塞
   - 解方：每个Sink设置独立的并行度，或改用异步Redis客户端（如Lettuce）

3. **大屏显示的数据和数据库实际数据不一致**
   - 根因：EventTime窗口基于事件时间，但数据存在乱序。如果maxOutOfOrderness设置太小，部分数据进入了错误的窗口
   - 解方：增加maxOutOfOrderness（生产推荐30-60秒），或使用AllowedLateness

### 架构模式总结

```
Source (Kafka) → 解析 → 分裂
    ├── window(keyBy) → Sink(GMV)
    ├── window(keyBy) → Sink(每分钟订单量)
    ├── window(keyBy) → Sink(热门Top10)
    ├── window(keyBy) → Sink(省份GMV)
    └── statefulMap → Sink(异常告警)
```

### 优点 & 缺点

| | Flink实时大屏方案 | 传统离线大屏方案（对比） |
|------|----------------|------------------------|
| **优点1** | 秒级延迟，数据到达后毫秒级可见 | 小时级延迟，ETL完成后才能更新 |
| **优点2** | 统一架构：一个Flink作业完成5个指标 | 每个指标独立系统，维护成本高 |
| **优点3** | Exactly-Once语义保证数据一致性 | 离线任务失败后需手动补数 |
| **优点4** | 弹性伸缩：大促期间扩容并行度即可 | 需提前分配资源，扩缩容困难 |
| **缺点1** | 运维复杂度高（需运维Flink+Kafka+Redis+Graafa） | 部署简单（一台MySQL+定时脚本） |
| **缺点2** | 开发门槛高（需要理解流计算概念） | SQL开发，上手快 |
| **缺点3** | 资源消耗大（Flink+Kafka+Redis常驻运行） | 定时任务执行完即释放资源 |

### 适用场景

**典型场景**：
1. 运营实时大屏（GMV、订单量、转化率核心指标秒级刷新）
2. 电商大促实时监控（618/双11期间关键指标实时跟踪）
3. 运维监控大屏（服务QPS、错误率、响应时间实时展示）
4. 活动效果实时评估（秒杀/促销活动的即时效果反馈）
5. IoT设备实时仪表盘（设备在线率、异常率实时统计）

**不适用场景**：
1. T+1简单报表（每天跑一次count算总数就够，没必要上Flink）
2. 低频数据（每天几十条日志，Flink的Checkpoint开销比计算本身还大）

### 扩展思考：如果是你来做大屏，还会加上哪些功能？

1. **实时同比/环比**：今天的GMV vs 昨天的同时段。需要额外的"历史状态存储"
2. **多语言Sink**：Redis + Kafka（给下游微服务） + WebSocket（推送到浏览器）
3. **分级告警**：GMV突降30% → 钉钉告警；热门商品价格异常 → 电话告警
4. **大屏回放**：保存Checkpoint到指定时间，可以在大屏上"回放"历史数据

### 思考题

1. 本章的大屏作业中，所有5个指标共用了同一个Kafka Source。如果其中一个指标的计算逻辑有bug，抛出异常导致整个作业失败——如何做到"单指标故障不影响其他指标"？（提示：每个指标使用独立的作业？还是使用SideOutput或Split失败隔离？）

2. 热门商品Top10使用SlidingWindow+Reduce实现了局部聚合，但最终输出了所有商品的count到Redis。如果要实现"只保留Top10"，应该在哪里做"TopN截断"？（提示：在ProcessWindowFunction中截断、在Redis Sink中截断、还是在Redis之外额外用一个排序服务？各自的优劣是什么？）
