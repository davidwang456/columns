# 第16章：【基础篇综合实战】搭建秒杀服务的单机流量防护系统

## 1 项目背景

基础篇的 15 章即将收尾，我们分别学习了 Sentinel 的七种规则类型和 Dashboard 操作、日志分析、故障排查。但这些都是"拆开学的"——限流是限流、熔断是熔断、热点是热点。在真实的生产场景中，这些能力需要被组合成一个完整的**流量防护系统**。

本章以一个"新品秒杀活动"为背景，要求订单服务在以下约束条件下稳定运行：

- **高并发冲击**：秒杀开始瞬间，QPS 可能从 10 飙升至 2000（200 倍突发流量）
- **热点商品风险**：80% 的流量集中在 3 个爆款 SKU 上
- **下游不稳定**：库存服务可能因 Redis 热点变慢（RT 从 10ms 升到 500ms）
- **安全防刷**：黄牛党可能用脚本 24 小时低频调用下单接口
- **优雅降级**：不能直接返回 500，用户需要看到"排队中"或"已售罄"等友好提示
- **系统自保**：万一所有保护都失效，系统在崩溃前要自动触发保护

你的任务是：从零搭建这个秒杀服务的 Sentinel 防护方案，用 JMeter 压测验证所有规则生效，输出一份基础篇复盘报告。

## 2 项目设计

**小胖**（紧张地搓手）："大师，明天就是秒杀日了。我把基础篇学的限流、熔断、热点、系统保护全加了，但我总觉得心里没底——这些规则配在一起，会不会互相打架？"

**大师**："你这个担心很正常。单条规则单独测没问题，组合在一起就有可能出现'预期之外'的行为。比如你配了 QPS=100，同时又配了热点参数限流=50——那某个热点 skuId 的请求到底被谁拦截？"

**小白**："应该是先到的先拦截？但这样排查起来就麻烦了——如果运维看到 BlockException，无法一眼判断是被 QPS 限制还是热点参数限制。"

**大师**："所以综合实战的关键不是'把规则堆上去'，而是**设计分层防护策略**。我给你一个设计框架：

- 第一层（防火墙）：授权规则 + 系统保护 → 在最外层挡住非法来源和系统过载
- 第二层（流量整形）：QPS 流控 + 关联/链路流控 + 匀速排队 → 把流量削峰填谷
- 第三层（热点隔离）：热点参数限流 → 保护被热点打爆的底层资源
- 第四层（故障熔断）：熔断降级 → 下游出问题时快速失败
- 第五层（兜底）：fallback/blockHandler → 给用户友好提示"

**小胖**："那这些规则的阈值怎么设定？总不能全是拍脑袋吧。"

**大师**："先做阶梯压测，找出服务的性能拐点。假设压测得出：单机 QPS=500 时 CPU=60%、RT=50ms、QPS=700 时 CPU=85%、RT=120ms、QPS=800 时 CPU=95%、RT=5000ms。那么：
- 流控阈值 = 600（拐点前 15%）
- 系统保护 CPU = 80%
- 慢调用熔断阈值 = 120ms（P99）
- 热点参数默认阈值 = 流控阈值的 30%（预留资源给其他 SKU）"

### 3.2 分步实现

**步骤一：秒杀下单核心接口**

```java
@RestController
@RequestMapping("/seckill")
public class SeckillController {

    @Autowired
    private SeckillService seckillService;

    /**
     * 秒杀下单 —— 核心保护接口
     * 规则组合：QPS 限流 + 热点参数 + 熔断降级 + 系统保护 + 授权规则
     */
    @PostMapping("/order")
    @SentinelResource(
        value = "seckillOrder",
        blockHandler = "seckillBlock",
        fallback = "seckillFallback"
    )
    public Result<SeckillResponse> seckill(
            @RequestParam String skuId,
            @RequestParam String userId,
            @RequestParam(defaultValue = "1") Integer quantity) {

        // 委托给 Service 层
        SeckillResponse resp = seckillService.executeSeckill(skuId, userId, quantity);
        return Result.success(resp);
    }

    /**
     * 限流/熔断时的统一处理
     */
    public Result<SeckillResponse> seckillBlock(
            String skuId, String userId, Integer quantity, BlockException e) {

        log.warn("秒杀限流, userId={}, skuId={}, exception={}",
                 userId, skuId, e.getClass().getSimpleName());

        if (e instanceof ParamFlowException) {
            return Result.error("HOT_ITEM",
                "该商品太火爆了！已为您保留排队资格（排队号：" +
                generateQueueId() + "），请 10 秒后查看订单");
        }
        if (e instanceof FlowException) {
            return Result.error("FLOW_LIMIT",
                "秒杀活动参与人数过多，您已进入排队");
        }
        if (e instanceof DegradeException) {
            return Result.error("SERVICE_DEGRADE",
                "系统繁忙，请稍后再来试试吧~");
        }
        if (e instanceof SystemBlockException) {
            return Result.error("SYSTEM_BUSY",
                "系统正在全力处理订单，请 5 秒后重试");
        }
        return Result.error("BLOCK", "抢购未成功，请重试");
    }

    /**
     * 业务异常兜底
     */
    public Result<SeckillResponse> seckillFallback(
            String skuId, String userId, Integer quantity, Throwable t) {
        log.error("秒杀异常, userId={}, skuId={}", userId, skuId, t);
        return Result.error("BIZ_ERROR", "下单失败，已自动重试");
    }

    private String generateQueueId() {
        return "Q" + System.currentTimeMillis() % 100000;
    }
}
```

**步骤二：Service 层熔断保护**

```java
@Service
public class SeckillService {

    @Autowired
    private RestTemplate restTemplate;

    private static final String INVENTORY_URL = "http://localhost:8082/inventory";

    public SeckillResponse executeSeckill(String skuId, String userId, int quantity) {
        // 1. 查询库存 —— 带熔断保护
        int stock = queryStockWithFallback(skuId);
        if (stock < quantity) {
            return SeckillResponse.soldOut(skuId);
        }

        // 2. 扣减库存 —— 带线程数限流
        boolean deducted = deductStockWithProtection(skuId, quantity);
        if (!deducted) {
            log.warn("库存扣减被限流，订单将异步处理, skuId={}, userId={}", skuId, userId);
            return SeckillResponse.pending(skuId);
        }

        // 3. 创建秒杀订单
        return SeckillResponse.success(skuId, "ORDER_" + System.currentTimeMillis());
    }

    @SentinelResource(value = "querySeckillStock", fallback = "queryStockFallback")
    private int queryStockWithFallback(String skuId) {
        String resp = restTemplate.getForObject(
            INVENTORY_URL + "/query?skuId=" + skuId, String.class);
        return Integer.parseInt(resp);
    }

    private int queryStockFallback(String skuId, Throwable t) {
        log.error("查询库存降级, skuId={}", skuId, t);
        return -1; // 未知库存，放行给业务层判断
    }

    @SentinelResource(value = "deductSeckillStock", blockHandler = "deductBlock")
    private boolean deductStockWithProtection(String skuId, int quantity) {
        String resp = restTemplate.postForObject(
            INVENTORY_URL + "/deduct?skuId=" + skuId + "&qty=" + quantity,
            null, String.class);
        return Boolean.parseBoolean(resp);
    }

    private boolean deductBlock(String skuId, int quantity, BlockException e) {
        log.warn("扣减库存被线程数限流, skuId={}", skuId);
        return false;
    }
}
```

**步骤三：统一规则配置**

```java
@Component
public class SeckillRuleConfig {

    @PostConstruct
    public void init() {
        initFlowRules();
        initDegradeRules();
        initParamFlowRules();
        initSystemRules();
        initAuthorityRules();
    }

    private void initFlowRules() {
        List<FlowRule> rules = new ArrayList<>();

        // 秒杀下单：QPS 限流 600
        rules.add(new FlowRule("seckillOrder")
                .setGrade(FLOW_GRADE_QPS).setCount(600));

        // 扣减库存：线程数限流 20（保护数据库连接池，假设连接池 25）
        rules.add(new FlowRule("deductSeckillStock")
                .setGrade(FLOW_GRADE_THREAD).setCount(20));

        // 关联流控：查询库存 QPS 高时限制下单
        rules.add(new FlowRule("seckillOrder")
                .setCount(300)  // 降到 300
                .setStrategy(STRATEGY_RELATE)
                .setRefResource("querySeckillStock"));

        FlowRuleManager.loadRules(rules);
    }

    private void initDegradeRules() {
        List<DegradeRule> rules = new ArrayList<>();

        // 查询库存：慢调用比例熔断（RT > 50ms 且比例 > 50%，熔断 10 秒）
        rules.add(new DegradeRule("querySeckillStock")
                .setGrade(DEGRADE_GRADE_RT)
                .setCount(50)
                .setSlowRatioThreshold(0.5)
                .setMinRequestAmount(10)
                .setStatIntervalMs(10000)
                .setTimeWindow(10));

        // 查询库存：异常比例熔断（异常比例 > 30%，熔断 15 秒）
        rules.add(new DegradeRule("querySeckillStock")
                .setGrade(DEGRADE_GRADE_EXCEPTION_RATIO)
                .setCount(0.3)
                .setMinRequestAmount(10)
                .setStatIntervalMs(10000)
                .setTimeWindow(15));

        DegradeRuleManager.loadRules(rules);
    }

    private void initParamFlowRules() {
        List<ParamFlowRule> rules = new ArrayList<>();

        // 按 skuId（索引 0）做热点限流，默认每个 skuId 200 QPS
        ParamFlowRule rule = new ParamFlowRule("seckillOrder")
                .setParamIdx(0).setCount(200);

        // 爆款商品例外：可以到 400 QPS
        ParamFlowItem hotItem1 = new ParamFlowItem()
                .setObject("SKU_HOT_001").setClassType(String.class.getName())
                .setCount(400);
        ParamFlowItem hotItem2 = new ParamFlowItem()
                .setObject("SKU_HOT_002").setClassType(String.class.getName())
                .setCount(350);
        rule.setParamFlowItemList(Arrays.asList(hotItem1, hotItem2));
        rules.add(rule);

        ParamFlowRuleManager.loadRules(rules);
    }

    private void initSystemRules() {
        List<SystemRule> rules = new ArrayList<>();

        SystemRule rule = new SystemRule();
        rule.setHighestCpuUsage(0.8);     // CPU > 80% 时保护
        rule.setAvgRt(200);                // RT > 200ms 时保护
        rule.setMaxThread(500);            // 入口线程 > 500 时保护
        rule.setQps(1000);                 // 兜底 QPS 上限
        rules.add(rule);

        SystemRuleManager.loadRules(rules);
    }

    private void initAuthorityRules() {
        List<AuthorityRule> rules = new ArrayList<>();

        // 秒杀接口只允许 App 和 Web（拒绝脚本来源）
        AuthorityRule rule = new AuthorityRule();
        rule.setResource("seckillOrder");
        rule.setStrategy(AUTHORITY_BLACK);  // 黑名单模式
        rule.setLimitApp("script,scraper,unknown-source");
        rules.add(rule);

        AuthorityRuleManager.loadRules(rules);
    }
}
```

## 4 项目总结

### 4.1 基础篇核心收获

通过基础篇 16 章的实战，读者应该具备以下能力：

- 能画出 Sentinel 的 Slot Chain 架构图，理解从 `SphU.entry()` 到业务逻辑的完整链路
- 能在 Spring Boot 项目中正确接入 Sentinel（注解 + 编程式 API）
- 能使用 Dashboard 配置和管理规则，看懂实时监控和日志
- 能独立设计一个服务的流控、熔断、热点参数、系统保护、授权组合方案
- 能设计 Sentinel 验收用例，用 JMeter 进行压测验证
- 能排查 90% 的"规则不生效"问题

### 4.2 基础篇能力边界

基础篇覆盖的是 Sentinel 的**单机、静态规则、Dashboard 管理**场景。以下内容将在中级篇和高级篇展开：

- 动态规则持久化（Nacos/Apollo）→ 第 17-18 章
