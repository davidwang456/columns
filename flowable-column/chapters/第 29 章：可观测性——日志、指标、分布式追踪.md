## 1）项目背景

星云生产事故要求 **5 分钟内**回答：实例卡在哪、**哪个下游**慢、**哪台 Pod** 在处理。**本章聚焦**：日志 **MDC**、关键 **Golden metrics**、**TraceId** 与引擎 ID 的绑定；以及 **PII 脱敏** 合规。

## 2.1）场景深化：日志里的「四维」——traceId、tenant、bk、piid

**星云**统一：每一条业务日志至少能聚类 **tenant + businessKey + processInstanceId + traceId**。客服输入订单号即可在日志平台拉全链——这比培训客服看懂 `ACT_RU_*` 更现实。

## 3）项目实战（主代码片段）

### 3.1 MDC 注入监听

```java
public class LoggingExecutionListener implements ExecutionListener {
  @Override
  public void notify(DelegateExecution execution) {
    MDC.put("processInstanceId", execution.getProcessInstanceId());
    MDC.put("businessKey", execution.getProcessInstanceBusinessKey());
    MDC.put("tenantId", execution.getTenantId());
  }
}
```

### 3.2 Micrometer Timer

## 4）项目总结

### 优点

- **MTTR** 显著下降；  
- 容量与异常有据可依。

### 缺点

- 埋点与**成本**（日志量、tracing 存储）。

### 典型使用场景

- 生产、与大客户共营环境。

### 注意事项

- **采样率**；  
- **密钥**不进日志。

### 反例

全量 ERROR 打印堆栈到 stdout 无轮转。**纠正**：**限流 + 脱敏**。

### 常见踩坑

| 现象 | 根因 | 处理 |
|------|------|------|
| trace 断 | async 未传 | OTEL 注入 |
| 指标爆炸 | 高基 label | 控制维度 |
| MDC 泄漏 | 未 clear | filter finally |

### 给测试一句话

对 **日志字段**做契约测试（smoke）。

### 给运维一句话

**SLO/Error budget** 与 **PagerDuty 分级**。

## 附录：星云案例深潜——日志里的身份证与「误发客户」

因 MDC 未脱敏，工程师把含手机号的 trace 发给客户排查——触发隐私事故。之后统一 **日志 scrubber** 与 **访问审计**。可观测性越强，合规责任也越大。
