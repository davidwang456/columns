# 第 25 章：引擎配置全景——ProcessEngineConfiguration 关键项

> 版本说明：Spring Boot 使用 `flowable.*` 前缀映射引擎配置；具体键以你所选 **Flowable 与 Boot 小版本** 的官方文档为准。[VERSION_MATRIX.md](../VERSION_MATRIX.md)

---

## 1）项目背景

星云生产发布前必须经过 **《引擎参数基线表》** 评审：数据库 schema 策略、**history 级别**、异步执行器线程池、是否启用 **IDM**、**事件注册**等。调错一项，可能是「磁盘爆」「延迟高」「升级失败」中的任何一种。**本章聚焦**：常用旋钮含义与**副作用**，并给出变更管理的注意点（而非替代官方文档罗列每一个属性）。

## 2）项目设计（小胖 × 小白 × 大师）

> **角色分工**：**小胖**（爱吃爱玩、不求甚解）用生活化、口语化的方式把话题「开球」，先把问题问出来；**小白**（喜静、喜深入）负责追问原理、边界、风险与可比方案；**大师**（资深技术 Leader）把业务约束与技术选型说透，善打比方、由浅入深。

**小胖**：默认配置能否上生产？

**大师**：小规模可用；一旦量上来，要按 **CPU/内存/连接池/历史保留**逐档调优。

**小白**：`database-schema-update` 呢？

**大师**：预发可以自动；生产建议 **关闭自动**、走 **迁移脚本** 双人复核。

**小胖**：history 用哪档？

**大师**：在 **审计完整性** 与 **磁盘成本**间折中；金融场景常见 **audit**，强监管可能 **full**。

**小白**：关闭异步执行器会怎样？

**大师**：**定时器/async 不工作**——别在产线随手关。

**小胖**：缓存配置？

**大师**：影响内存；**堆**与 **GC** 要结合压测。

## 2.1）场景深化：配置变更的「三角」——研发/运维/DBA

每一次引擎参数变更都应留下：**动机**、**回滚值**、**观测指标前后对比**。**星云**变更单模板里强制勾选：是否影响 **作业调度**、是否影响 **历史写入速率**、是否需要 **维护窗口**。

## 3）项目实战（主代码片段）

### application.yaml 示例

```yaml
flowable:
  database-schema-update: false
  history-level: audit
  async-executor-activate: true

spring:
  datasource:
    hikari:
      maximum-pool-size: 30
```

### Java 自定义 EngineConfigurationConfigurer（如有）

```java
@Configuration
public class FlowableExtraConfig implements EngineConfigurationConfigurer<SpringProcessEngineConfiguration> {
  @Override
  public void configure(SpringProcessEngineConfiguration engineConfiguration) {
    // 例如注册自定义监听器、自定义活动行为类工厂等（进阶）
  }
}
```

### 验收

压测环境采集：**TPS**、**P99 延迟**、**CPU**、**GC pause**、**JDBC wait**。

## 4）项目总结

### 优点

- 参数化让 **同一应用镜像** 在多环境差异化。  
- 为 **容量规划**提供抓手。

### 缺点

- **组合爆炸**；  
- 版本升级默认值可能变化。

### 典型使用场景

- 上线评审、季度容量复盘、事故后加固。

### 注意事项

- **灰度**验证；  
- **备份**先于 schema 变更。

### 反例

从互联网拷贝「神秘魔法 yaml」未经解释就上线。**纠正**：**逐项注释Owner**。

### 常见踩坑

| 现象 | 根因 | 处理 |
|------|------|------|
| 启动失败 | schema 锁 | 维护窗 |
| OOM | 缓存过大 | 限制 |
| job 堆积 | 线程小 | 扩容 |
| 升级差异 | 默认值变 | 读 Release Notes |

### 给测试一句话

配置变更走 **全量回归** + **性能基线对比**。

### 给运维一句话

**配置中心**审计版本；**告警**绑定关键线程池与连接池。

## 附录：星云案例深潜——一次「hist 级别下调」合规警报

为省磁盘，有人提议把 history 从 `full` 调成 `audit`，未通知法务。两周后监管抽查缺字段，项目被迫紧急回滚配置并补数据解释。附录警告：**历史级别=法律证据策略**的一部分，变更要走 **CCR（变更评审会）**。
