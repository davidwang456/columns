# 第 12 章：表达式与 UEL——`${}`、`execution` 与 `task` 里能用什么

> 版本说明：UEL 与 BeanResolver 集成依配置而定。

---

## 1）项目背景

星云流程中，网关条件、监听、服务任务的字段注入，大量使用 **`${ expression }`**。开发者常困惑：**能调用 Spring Bean 吗**？**能访问哪些内置变量**？**null 怎么防**？**本章聚焦** Flowable 的表达式求值上下文：**execution、task、authenticatedUserId**（视场景）等，以及 **Bean 调用边界**。

---

## 2）项目设计（小胖 × 小白 × 大师）

> **角色分工**：**小胖**（爱吃爱玩、不求甚解）用生活化、口语化的方式把话题「开球」，先把问题问出来；**小白**（喜静、喜深入）负责追问原理、边界、风险与可比方案；**大师**（资深技术 Leader）把业务约束与技术选型说透，善打比方、由浅入深。

**小胖**：`${amount > 100000}` 里的 amount 哪来？

**大师**：来自**流程变量**；要找 **当前执行范围** 内的变量。多实例或子执行时名称遮蔽要小心（第 7、14 章）。

**小白**：`${myBean.calc(execution)}` 行不行？

**大师**：常见，但 `myBean` 必须是**可解析 Bean**；注意**副作用**——表达式里别改外部世界太多，难测。

**小胖**：能 new 对象吗？

**大师**：不推荐；复杂逻辑下放 Java。

**小白**：字符串拼接？

**大师**：可用 `${a + b}` 视类型；小心 **数字与字符串**隐式转换。

**小胖**：和 JS 一样吗？

**大师**：**UEL** 不是完整 JS；以文档为准。

**小白**：监听里能用 task 吗？

**大师**：`TaskListener` 有 **task** 对象；`ExecutionListener` 无 **task**。

## 2.1）场景深化：何时必须退出 EL

当表达式跨三行仍读不懂、或需要访问多个远端服务聚合决策时，就应该停止在 EL 里堆逻辑，改为 **Delegate 输出 routingDecision 变量** 或使用 **DMN**。星云 code review 规则是：**EL 内不允许出现三元嵌套超过两层**；超过即重构。这样评审者能把注意力放在业务流程而非字符串编程上。

## 3）项目实战（主代码片段）

### 3.1 网关条件

```xml
<conditionExpression xsi:type="tFormalExpression">
  ${amount >= 100000 &amp;&amp; region == 'EU'}
</conditionExpression>
```

### 3.2 Service Task 字段注入

```xml
<serviceTask id="svc1" flowable:delegateExpression="${demoDelegate}">
  <extensionElements>
    <flowable:field name="threshold">
      <flowable:string>${thresholdVar}</flowable:string>
    </flowable:field>
  </extensionElements>
</serviceTask>
```

### 3.3 JUnit

对同一变量矩阵求值结果做集成断言（通过启动流程或在单元测试内调解析 API，视项目工具）。

### 3.4 curl

表达式本身不通过 curl 测；通过 **启动流程+变量** 间接验证路径。

## 4）项目总结

### 优点

- 声明式、改 BPMN 即可调参。  
- 与 Spring Bean 组合灵活。

### 缺点

- 隐式 NPE；  
- 复杂表达式难读难测。

### 典型使用场景

- 简单数值比较、开关、路由变量。

### 注意事项

- **单元测试覆盖**典型变量组合。  
- 禁止在 EL 里 **远程 IO**。

### 反例

在表达式里写 **数据库查询**。**纠正**：Delegate 或规则引擎。

### 常见踩坑

| 现象 | 根因 | 处理 |
|------|------|------|
| PropertyNotFound | 变量名拼错 | 日志打印 variables |
| 比较 false | 类型不对 | 强转或统一 Long |
| Bean 找不到 | 未暴露 | 检查 `FlowableConfig` |
| 升级行为变 | UEL 版本差异 | 回归测 |

### 给测试一句话

为 **EL 临界组合**建表：**null、边界、多 region**。

### 给运维一句话

表达式**不单独暴露**给用户编辑；避免**任意脚本注入**（第 33 章）。

## 附：延展阅读——单元测试表达式片段的做法

在大型项目里可引入**小型表达式沙箱测试**：把与资金相关的条件抽成常量与纯函数，再在集成测试里用全变量矩阵验证。星云安全组还要求对 `resolveBean` 路径做 **白名单**审查，避免 EL 调用任意 Spring Bean 成为**后门**。

---

## 附录：星云案例深潜——从 `${}` 到「规则平台」的演进

某业务线在 EL 中堆叠十几行判断，MR review 时无人能读完。后来团队把该段搬到 **DMN + 决策评审会**，EL 仅保留 `${routing=='DMN'}` 之类的浅层引用。结果是：**可测试性**与**业务可读性**同步上涨。若你的组织也正走向规则爆炸，请主动发起「表达式瘦身」专题，而不是等问题暴露在生产。
