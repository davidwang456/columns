# 第 33 章：安全、权限与多租户——tenantId、最小权限与数据隔离

> 版本说明：`tenantId` 贯穿 **deployment / instance / task** 查询；配合 OAuth2/JWT。[VERSION_MATRIX.md](../VERSION_MATRIX.md)

---

## 1）项目背景

星云 SaaS **多租客**；法规要求 **租户数据天然隔离**，且 **运维支撑账号** 只能 **break-glass** 访问。**本章聚焦**：引擎 API **tenant 过滤**、与 **BFF 鉴权**配合、**审计日志**以及 **导出脱敏**。

---

## 2）项目设计（小胖 × 小白 × 大师）

> **角色分工**：**小胖**（爱吃爱玩、不求甚解）用生活化、口语化的方式把话题「开球」，先把问题问出来；**小白**（喜静、喜深入）负责追问原理、边界、风险与可比方案；**大师**（资深技术 Leader）把业务约束与技术选型说透，善打比方、由浅入深。

**小胖**：应用层过滤不行吗？

**大师**：不够；**服务层**必须默认 **tenant 上下文**，防 **水平越权**。

**小白**：管理员？

**大师**：**双人授权** + **全量审计** + **时间限制**。

**小胖**：REST 呢？

**大师**：**JWT claims** 映射 tenant；禁止客户端传 tenant 作为唯一依据——要 **签名校验**。

## 2.1）场景深化：「租户级流程定义」与共享模板

**星云**允许 **全局模板** + **租户 overlay**；部署时 **tag** 区分来源。查询 `ProcessDefinition` 要 **union** 策略写清，否则出现 **同 key 多版本混乱**。

## 3）项目实战（主代码片段）

```java
runtimeService.startProcessInstanceByKeyAndTenantId(

```

```java
taskService.createTaskQuery().taskTenantId("tenantA").list();
```

```java
@PreAuthorize("@tenantGuard.matches(#tenantId)")
```

## 4）项目总结

### 优点

- **合规**与 **商业化**基石。

### 缺点

- **研发心智**与 **测试成本**上升。

### 典型使用场景

- 多法人、大型集团、国际化隔离。

### 注意事项

- **导出 CSV** 水印；  
- **日志** scrub PII。

### 反例

REST **仅靠 query string tenantId**。**纠正**：**签权**。

### 常见踩坑

| 现象 | 根因 | 处理 |
|------|------|------|
| 串租 | 漏 filter | AOP |
| 部署串 | 未 tenant | pipeline |
| JWT 伪造 | 无验签 | JWKS |

**租户横切** + **越权尝试**自动化。

### 给运维一句话

## 附录：星云案例深潜——「演示账号」串租事件

售前演示环境复用了生产镜像但未清空种子数据，客户在演示里看到其他租户流程缩略信息。修复：**租户隔离从镜像烘焙开始**；演示账号**独立realm**；并对所有 demo URL **加网络 ACL**。
