# 读者分层与章节推荐

本文配合 [README.md](README.md) 全章目录使用，便于**跨部门推广**：开发、测试、运维各取必读路径，再按需扩展。

各章 **「2）项目设计」** 采用 **小胖（生活化开球）→ 小白（深入追问）→ 大师（由浅入深）** 对话体，内训时可让三位同事分别试读，降低「纯技术宣读」疲劳。

---

## 角色速查

| 角色 | 首要目标 | 次要目标 |
|------|-----------|-----------|
| **开发（初阶）** | 会建模、会调 API、能本地跑通端到端 | 理解事务边界与变量 |
| **开发（进阶）** | 作业/异步、多实例、迁移、扩展点 | 能读配置类与问题定位 |
| **开发（源码向）** | Command/Interceptor 与服务门面关系 | 与排障、自定义扩展联动 |
| **测试** | 任务查询、变量断言、回归分层 | 了解异步/定时对用例稳定性的影响 |
| **运维** | Job、线程池、库表膨胀、集群、可观测性 | CI/CD 中资源发布与回滚 |

---

## 必读章节（按角色）

### 开发（初级）——顺序跟练 1～16

重点建立：**BPMN 元素 ↔ API ↔ 运行时/历史表** 的心智模型。

### 开发（进阶）—— 17～26、31～33

重点：**作业与失败重试、多实例、DMN、Spring 事务、迁移、自定义行为、安全/租户**。

### 开发（源码向）—— 32、33（可与 23、25 交叉）

重点：**命令模式、拦截器链、配置项在源码中的落点**。

### 测试—— 1、2、3、4、5、6、10、24、29、30（可加 17、18）

| 章 | 侧重 |
|----|------|
| 1～6 | 提测对象：流程定义、任务、服务任务、变量基础 |
| 10 | 历史数据校验与验收口径 |
| 17 | 异步/定时导致的「偶现」失败与重现策略 |
| 18 | 多实例完成条件与会签场景用例设计 |
| 24 | 自动化断言与测试数据策略 |
| 29～30 | 与运维联合演练：日志字段、作业堆积 |

**断言建议**：对「流程实例状态、当前活动Id、变量快照、待办列表」分层断言，避免只点 UI。

### 运维—— 3、17、25～30、34（建议加 11）

| 章 | 侧重 |
|----|------|
| 3 | 部署物、版本与多环境差异 |
| 17 | `async`、JobExecutor、死信队列、重试 |
| 25 | 线程池、缓存、历史级别——调参入口 |
| 26～28 | 升级、DB、集群与锁 |
| 29～30 | 监控指标、排障 playbook |
| 34 | 流水线门禁、制品与回滚 |

**观测建议**：统一 `processDefinitionId` / `processInstanceId` / `businessKey` 落在日志/链路中的一致性。

## 推广节奏建议（由浅入深）

1. **第 0 周**：发布模板 + 版本矩阵 + 第 1～3 章样章，收集反馈。  
2. **第 1～4 周**：基础篇周更 1～2 篇，面向全体研发宣导。  
3. **第 5～8 周**：中级篇，**开发 + 测试** 同桌评审（案例来自主线）。  
4. **第 9～周**：高级篇，**开发 + 运维** 同桌评审，沉淀运维手册链接。
