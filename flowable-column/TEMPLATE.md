# Flowable 专栏：统一章节模板（已冻结）

本文件定义全专栏**必须遵守**的结构、字数区间、虚构主线与人设。各章独立成文时不得删减四段标题，可在段内增子标题。

---

## 字数与节奏（冻结）

| 项目 | 要求 |
|------|------|
| 单章总字数 | **3000～5000 字**（中文正文，含小标题；代码与 BPMN/XML 计行不计入字数亦可，但须在文内说明「本段为示意」） |
| 对话轮次 | **8～15 轮**（**小胖 / 小白 / 大师** 交替推进），每轮 2～5 句，避免话剧式冗长 |
| 主代码块 | **至少 1 段**「可运行的主路径」：Java/Kotlin 或 BPMN 片段 + 引擎 API；**另附** REST（curl）或 **JUnit 断言** 二选一 |
| 踩坑清单 | **不少于 4 条**，格式：**现象 / 根因 / 处理** |
| 反例 | **每章 1 个**「看似合理、实则埋雷」的建模或实现，在总结中回扣 correction |

## 四段结构（冻结）

### 1）项目背景

- 使用专栏**统一虚构主线**中的业务切片（见下文），交代：目标、干系人、合规/时限/对接系统。
- 用 1 段说明「本章聚焦的技术对象」与「背景中的哪一环」对应。

### 2）项目设计（小胖 × 小白 × 大师）

- **三人对话**：见下文人设；开章建议用 **引用块**写一句 **角色分工**（引导读者入戏），随后进入问答。
- 对话结束后追加 **一页纸决策表**（表格列建议固定）：

| 输入 | 输出 | 适用场景 | 不适用/慎用 | 与上下游章节关系 |
|------|------|----------|-------------|------------------|

### 3）项目实战（主代码片段）

- **BPMN/DMN/XML 或配置片段**（可复制）+ **引擎服务调用**（主路径）。
- 标明依赖版本：以 [VERSION_MATRIX.md](VERSION_MATRIX.md) 为准，若与示例不同需在章首脚注。

### 4）项目总结

- **优点 / 缺点**（ bullet，各 2～4 条）。
- **典型使用场景**（ 2～3 条，偏业务语言）。
- **注意事项**（与事务、时钟、异步、变量、并发相关的至少 2 条）。
- **常见踩坑**（表格式 现象/原因/办法）。
- **给测试一句话** / **给运维一句话**（中级篇起强制；基础篇建议也写，可简短）。

## 统一虚构业务主线（冻结）

**公司**：星云科技（NeuraTech），智能制造与供应链 SaaS。

**主线故事**（跨多章引用，不必每章讲完）：

1. **采购申请**：部门提需求 → 多级审批 → 合同编号生成。  
2. **合同与履约**：供应商交付延迟触发 **定时升级**、**子流程** 处理例外。  
3. **售后与风控**：客诉工单与 **DMN 规则** 计算赔付区间；必要时 **消息/信号** 联动 CRM。

> 各章从主线中切一小段即可，避免章节之间强依赖导致无法单篇阅读。

## 人设（冻结）：三人行

| 角色 | 定位 | 职责与语气 |
|------|------|------------|
| **小胖** | 爱吃爱玩、不求甚解 | **开球**：用生活化、俏皮、略带「偷懒心态」的问题，把**基础矛盾**抛出来（「加字段不行吗」「晚上火锅还开会吗」）。不求术语准确，但求观众能代入。 |
| **小白** | 喜静、喜深入 | **追问**：在表层问题被点破后，追问**原理、边界、 trade-off、与其他方案对比、线上风险**——话不多，但问到点子上。 |
| **大师** | 资深开发者 / 技术 Leader | **收束与升维**：先交代**业务约束与不变量**，再用**类比**把技术说透（生活、装修、交通、账本皆可）；**由浅入深**，最后指到本章的关键对象（BPMN 元素、引擎 API、运维抓手等）。 |

写作提示：

- 尽量避免第四套人设；若需「运维/测试一句话」，留在段尾专栏位，不进主线斗嘴。  
- **项目背景**叙述中避免再用「小白」指代随机同事，以免与角色 **小白**混淆。

## 趣味性与深度（写作约定）

- **趣味性**：来自**错误假设被纠正**（反例）和**主线梗的回调**，以及 **小胖的生活化类比**；不靠段子灌水。  
- **实战性**：每章代码/配置可在空仓库按版本跑通；复杂章允许「分段可运行」。  
- **深度**：用「边界 + 对比 + 引擎职责」写清原理；源码章单独遵守第 32 章提纲。

## 文件命名

- 章节文件：`chapters/NN-slug.md`，`NN` 为两位序号，`slug` 为英文短名（小写连字符）。  
- 示例：`chapters/第 3 章：第一条流程——从 Modeler 到部署运行（含版本）.md`（文件名与章首 H1 一致；Windows 下文件名中的 `/` 等字符已替换为全角 `／`）。
