## 1. 项目背景

分布式排障需要 **稳定错误结构**（含 `traceId` / `instance`）。RFC 7807 **Problem Details** 是常见选择；本章用自定义 JSON 演示，可迁移到 `quarkus-smallrye-problem`（若采用）。

## 2. 项目设计：大师与小白的对话

**运维**：「用户截图 502，不知道是网关还是应用。」

**大师**：「错误体里要有 **request id**；网关与应用约定字段名。」

**小白**：「所有异常都 500？」

**大师**：「**客户端错误**应 4xx；**未知服务端**才 500。」

**安全**：「能把堆栈返回给客户端吗？」

**大师**：「**不能**；detail 要脱敏。」

### 4.2 错误 DTO

`src/main/java/org/acme/api/ErrorBody.java`：

```java
package org.acme.api;

public record ErrorBody(String type, String title, String detail, String instance) {}
```

### 4.3 Mapper

`src/main/java/org/acme/api/AppExceptionMapper.java`：

```java
package org.acme.api;

import jakarta.ws.rs.core.Response;
import jakarta.ws.rs.ext.ExceptionMapper;
import jakarta.ws.rs.ext.Provider;
import org.jboss.logging.Logger;

@Provider
public class AppExceptionMapper implements ExceptionMapper<IllegalArgumentException> {

    private static final Logger LOG = Logger.getLogger(AppExceptionMapper.class);

    @Override
    public Response toResponse(IllegalArgumentException ex) {
        LOG.warn("bad request", ex);
        var body = new ErrorBody(
            "https://acme.example/problems/bad-request",
            "Bad Request",
            ex.getMessage(),
            ""
        );
        return Response.status(400).entity(body).build();
    }
}
```

### 4.4 Kubernetes：Ingress 自定义错误页（可选）

部分 Ingress 支持 `custom-http-errors`；与应用内错误体互补，讲义讨论即可。

## 5. 课堂实验


| 步骤  | 操作                                            | 预期结果            |
| --- | --------------------------------------------- | --------------- |
| 1   | 资源中 `throw new IllegalArgumentException("x")` | 客户端得 400 + JSON |

## 6. 项目总结


| 维度       | 说明            |
| -------- | ------------- |
| **优点**   | 调用方可解析；可统一风格。 |
| **缺点**   | 过度封装丢信息。      |
| **适用场景** | 公共 API。       |
