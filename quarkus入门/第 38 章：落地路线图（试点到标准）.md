## 1. 项目背景

技术培训需落到 **流程与组织**：否则「学完就忘」。本章输出可执行的 **采纳路线图** 与 **管理层沟通要点**（SLO、风险、回滚）。

---

## 2. 项目设计：大师与小白的对话

**管理层**：「投入产出怎么量化？」

**大师**：「**SLO**：P99 启动、发布失败率、MTTR、单副本内存；对比试点前后。」

**小白**：「一上来全公司推广？」

**大师**：「**试点团队** + **Golden Path** + **平台组响应 SLA**。」

**运维**：「谁维护 Helm chart？」

**大师**：「平台组 owns 模板；业务组填值。」

**测试**：「门禁放哪些测试？」

**大师**：**单元必过** + **精选集成** + **镜像扫描**；Native 可选夜间。」

---

## 3. 知识要点

- Golden Path = 模板 + 文档 + 示例库  
- 技术雷达：试点 → 扩展 → 默认标准  
- 回滚：应用版本 + DB 迁移策略（第 17 章）

## 4. 项目实战

### 4.1 Golden Path 检查清单（Markdown 模板）

```markdown
## Golden Path：新建 Quarkus 服务
- [ ] JDK 版本：____
- [ ] Quarkus Platform：____
- [ ] 最小扩展集：rest, smallrye-health, micrometer-registry-prometheus
- [ ] CI：mvn test + 镜像构建 + Trivy
- [ ] K8s：Deployment/Service/Ingress/探针/资源限制
- [ ] 观测：JSON 日志 + /q/metrics + OTel 采样 __%
- [ ] 配置：ConfigMap/Secret 键名对照表已更新
```

### 4.2 里程碑表（可复制到 Confluence）

| 阶段 | 时间 | 交付物 | 退出标准 |
|------|------|--------|----------|
| M0 | W1 | 培训完成率 80% | 考试或实验通过 |
| M1 | W4 | 1 非关键服务上预发 | 无 P1 缺陷 |
| M2 | W8 | Golden Path v1 + 模板库 | 新项目 100% 使用 |
| M3 | W12 | Native 决策表 + 可选流水线 | 技术委员会签字 |

### 4.3 示例 `pom.xml`（公司父 POM 坐标占位）

```xml
<parent>
  <groupId>com.mycompany</groupId>
  <artifactId>quarkus-parent</artifactId>
  <version>1.0.0</version>
  <relativePath/>
</parent>
```

### 4.4 示例 CI（`.github/workflows/quarkus-service.yml` 骨架）

```yaml
name: quarkus-service
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: '17'
          distribution: 'temurin'
      - run: ./mvnw -B verify
      - run: ./mvnw -B package -DskipTests -Dquarkus.container-image.build=true
      # - uses: aquasecurity/trivy-action@master ...
```

## 5. 课堂实验

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 每组填写 Golden Path 检查清单 | 提交评审 |
| 2 | 选一个真实候选服务做 PRE 评估（风险、数据、依赖） | 一页纸 |
| 3 | 角色扮演：向管理层汇报 5 分钟 | 互相提问 |
| 4 | 制定「回滚演练」日期与参与人 | 日历事件 |

**结业**：学员获得 `docs/column/` 全系列讲义路径作为长期参考。
