# 第 34 章：模块化单体与微服务边界

## 0. 课程卡片

| 项目 | 说明 |
|------|------|
| **建议课时** | 60 分钟（架构研讨为主） |
| **学习目标** | Maven 多模块划分；数据所有权；演进至多 Deployment 的条件 |
| **先修** | 第 1、10 章 |

---

## 1. 项目背景

Quarkus 适合 **模块化单体** 与 **多服务**。K8s 不替你划边界；错误拆分会得到 **分布式单体**。

---

## 2. 项目设计：大师与小白的对话

**小白**：「上 K8s 就要拆微服务。」

**大师**：「**先模块化，再按需拆分**。」

**架构师**：「拆分触发条件？」

**大师**：「独立伸缩、独立发布、团队规模、数据隔离——满足再拆。」

**运维**：「单体一个 Deployment 简单。」

**大师**：「简单是优点；瓶颈在代码边界是否清晰。」

---

## 4. 项目实战

### 4.1 多模块 `pom.xml`（父工程）

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 https://maven.apache.org/xsd/maven-4.0.0.xsd">
  <modelVersion>4.0.0</modelVersion>
  <groupId>org.acme</groupId>
  <artifactId>acme-parent</artifactId>
  <version>1.0.0-SNAPSHOT</version>
  <packaging>pom</packaging>
  <modules>
    <module>acme-domain</module>
    <module>acme-api</module>
    <module>acme-service</module>
  </modules>
  <properties>
    <quarkus.platform.version>3.19.2</quarkus.platform.version>
  </properties>
  <dependencyManagement>
    <dependencies>
      <dependency>
        <groupId>io.quarkus.platform</groupId>
        <artifactId>quarkus-bom</artifactId>
        <version>${quarkus.platform.version}</version>
        <type>pom</type>
        <scope>import</scope>
      </dependency>
    </dependencies>
  </dependencyManagement>
</project>
```

### 4.2 `acme-service/pom.xml`（可执行应用）

```xml
<parent>
  <groupId>org.acme</groupId>
  <artifactId>acme-parent</artifactId>
  <version>1.0.0-SNAPSHOT</version>
</parent>
<artifactId>acme-service</artifactId>
<dependencies>
  <dependency>
    <groupId>org.acme</groupId>
    <artifactId>acme-api</artifactId>
    <version>${project.version}</version>
  </dependency>
  <dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-rest</artifactId>
  </dependency>
</dependencies>
<build>
  <plugins>
    <plugin>
      <groupId>io.quarkus.platform</groupId>
      <artifactId>quarkus-maven-plugin</artifactId>
      <version>${quarkus.platform.version}</version>
      <extensions>true</extensions>
    </plugin>
  </plugins>
</build>
```

### 4.3 Kubernetes

单体：**一个 Deployment**；拆分后：**每服务一个 Deployment + Service**，共享 DB 与否由数据所有权决定。
