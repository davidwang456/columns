## 1. 项目背景

大文件进内存会导致 **OOM**；Ingress、网关、应用三层都有 **body 大小限制**。生产应优先 **流式**与**对象存储**（S3/MinIO）。

---

## 2. 项目设计：大师与小白的对话

**运维**：「Ingress 报 413。」

**大师**：「三层对齐：**客户端、Ingress、Quarkus** `max-body-size`。」

**小白**：「`byte[]` 读全文件。」

**大师**：「内存爆炸捷径。用 `FileUpload` + 转存。」

**安全**：「上传要扫病毒吗？」

**大师**：「视行业要求；至少校验类型与大小。」

## 4. 项目实战

### 4.1 `pom.xml`

```xml
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-rest</artifactId>
</dependency>
<!-- 若使用 resteasy reactive multipart，添加 quarkus-rest-multipart 等，以 guide 为准 -->
<dependency>
  <groupId>io.quarkus</groupId>
  <artifactId>quarkus-rest-multipart</artifactId>
</dependency>
```

### 4.2 `application.properties`

```properties
quarkus.http.limits.max-body-size=20M
```

### 4.3 上传资源（RESTEasy Reactive 风格示例）

`src/main/java/org/acme/UploadResource.java`：

```java
package org.acme;

import jakarta.ws.rs.Consumes;
import jakarta.ws.rs.POST;
import jakarta.ws.rs.Path;
import jakarta.ws.rs.core.MediaType;
import org.jboss.resteasy.reactive.RestForm;
import org.jboss.resteasy.reactive.multipart.FileUpload;

import java.io.IOException;
import java.nio.file.Files;

@Path("/upload")
public class UploadResource {

    @POST
    @Consumes(MediaType.MULTIPART_FORM_DATA)
    public String upload(@RestForm FileUpload file) throws IOException {
        if (file == null) {
            return "no file";
        }
        var target = Files.createTempFile("up-", "-" + file.fileName());
        Files.copy(file.uploadedFile(), target, java.nio.file.StandardCopyOption.REPLACE_EXISTING);
        return "saved-bytes=" + Files.size(target);
    }
}
```

### 4.4 Kubernetes：`Ingress` body 限制

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: api
  annotations:
    nginx.ingress.kubernetes.io/proxy-body-size: "25m"
spec:
  rules:
    - host: files.example.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: api
                port:
                  number: 80
```
