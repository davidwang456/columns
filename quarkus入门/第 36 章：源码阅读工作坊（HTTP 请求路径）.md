| 项目 | 说明 |
|------|------|
| **建议课时** | 120 分钟（工作坊） |

## 1. 项目背景

资深开发应能通过**调用栈**定位延迟与错误。本仓库 HTTP 栈涉及 `independent-projects/resteasy-reactive` 与 `extensions` 集成。工作坊强调**方法**而非背类名。

---

## 2. 项目设计：大师与小白的对话

**小白**：「类太多从哪看？」

**大师**：「从**一条请求**倒着走：断点打在 Resource 方法，栈往上翻。」

**架构师**：「读源码对升级有何用？」

**大师**：「release note + 本地对照 Processor，评估 **breaking**。」

**测试**：「我们要读框架测业务吗？」

**大师**：「不；但要知道**边界行为**（异常映射、线程）。」

## 4. 项目实战

### 4.1 准备

```bash
cd d:/software/workspace/quarkus
# 可选：仅索引相关模块，加快 IDE
```

IDE 导入 Maven 工程，打开示例应用（培训用第 4 章工程即可）以 **remote debug** 附加到 `quarkus:dev`：

```bash
./mvnw quarkus:dev -Dquarkus.debug.port=5005
```

### 4.2 断点清单（学员填表）

| 序号 | 位置 | 观察目的 |
|------|------|----------|
| 1 | 业务 `*Resource` 方法入口 | 确认线程名 |
| 2 | （讲师指定）框架 filter / handler | 请求如何包装 |
| 3 | 序列化写出前 | JSON 编码路径 |

### 4.3 阅读任务（2 选 1）

**任务 A**：在仓库中搜索 `EventLoopContext`，写 200 字说明其与 HTTP 的关系。  
**任务 B**：打开任意 `*Processor.java`（REST 相关），列出 2 个 `BuildItem` 产出。

### 4.4 `pom.xml`

学员应用无需改 pom；工作坊聚焦本仓库源码。
