# 第 9 章：容器镜像构建与交付

## 0. 课程卡片

| 项目 | 说明 |
|------|------|
| **建议课时** | 60～75 分钟（含构建等待） |
| **学习目标** | 使用 `quarkus-container-image-*` 构建镜像；理解 tag 策略；写出可对接 registry 的完整参数 |
| **先修** | 第 1～3 章；本地或 CI 已安装 Docker/Podman |

---

## 1. 项目背景

交付到 Kubernetes 的最终载体是**镜像**。Quarkus 可将镜像构建接入 Maven，减少「手写 Dockerfile 与 jar 路径对不齐」的问题。本章强调：**immutable tag**、**非 latest**、与 CI 变量（`CI_COMMIT_SHORT_SHA`）结合。

---

## 2. 项目设计：大师与小白的对话

**运维**：「镜像 tag 用 `latest` 我拒收。」

**大师**：「用 **git sha** 或 **版本号 + sha**；晋升流水线只推进 digest。」

**小白**：「Jib 和 Dockerfile 扩展选哪个？」

**大师**：「团队熟悉 Docker 多阶段就 Dockerfile；想少维护 Dockerfile 可 Jib——按平台标准选一种写进 Golden Path。」

**安全**：「镜像扫描谁跑？」

**大师**：「CI 里 **Trivy / Grype**；高危 CVE 阻断晋升。」

**架构师**：「Native 镜像构建时间太长怎么办？」

**大师**：**远程构建**、**缓存层**、**仅发布分支构建 Native**——别在每次 PR 上全开。」

## 3. 知识要点

- `quarkus-container-image-docker` 或 `quarkus-container-image-jib`  
- `quarkus.container-image.*` 前缀配置  
- `package -Dquarkus.container-image.build=true` 触发构建

---

## 4. 项目实战

### 4.1 `pom.xml`（Docker 构建）

```xml
<dependencies>
  <dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-container-image-docker</artifactId>
  </dependency>
  <dependency>
    <groupId>io.quarkus</groupId>
    <artifactId>quarkus-rest</artifactId>
  </dependency>
</dependencies>
```

### 4.2 `src/main/docker/Dockerfile.jvm`（由扩展生成时可微调）

若使用默认生成文件，通常位于 `src/main/docker/`。以下为**教学用简化多阶段示例**（路径以 `quarkus-app` 为准）：

```dockerfile
FROM registry.access.redhat.com/ubi9/openjdk-17-runtime:1.20
ENV LANG='en_US.UTF-8' LANGUAGE='en_US:en'
WORKDIR /work/
COPY target/quarkus-app/lib/ /work/lib/
COPY target/quarkus-app/*.jar /work/
COPY target/quarkus-app/app/ /work/app/
COPY target/quarkus-app/quarkus/ /work/quarkus/
EXPOSE 8080
USER 185
ENTRYPOINT ["java", "-jar", "/work/quarkus-run.jar"]
```

### 4.3 `application.properties`

```properties
quarkus.container-image.registry=registry.example.com
quarkus.container-image.group=acme
quarkus.container-image.name=order-service
# 培训可用固定 tag；生产用 CI 注入
quarkus.container-image.tag=${env.CI_COMMIT_SHORT_SHA:local}
quarkus.container-image.builder=docker
```

### 4.4 构建命令

```bash
./mvnw package -DskipTests -Dquarkus.container-image.build=true
```

## 5. 课堂实验

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | `./mvnw package -DskipTests` 确认 `target/quarkus-app/` 存在 | jar 布局正确 |
| 2 | 本地 Docker 运行：`docker run --rm -p 8080:8080 registry.example.com/acme/order-service:local`（按实际 tag） | 可访问 HTTP |
| 3 | 修改 `quarkus.container-image.tag`，连续构建两次，`docker images` 看到两标签 | 理解 immutable |
| 4 | （讨论）若 registry 在内网，Maven 如何认证？ | `docker login` / `~/.docker/config` |

**清理**：`docker rmi` 实验镜像。

## 6. 项目总结

| 维度 | 说明 |
|------|------|
| **优点** | 构建链一体化；与 Quarkus 打包布局一致。 |
| **缺点** | Native 构建资源消耗大；registry 网络问题常见。 |
| **适用场景** | CI/CD、GitOps。 |
| **注意事项** | 基础镜像升级；非 root 用户。 |
| **常见踩坑** | 缓存错误导致旧镜像；架构不一致 arm64/amd64。 |

**延伸阅读**：<https://quarkus.io/guides/container-image>
