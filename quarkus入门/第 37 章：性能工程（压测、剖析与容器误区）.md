## 1. 项目背景

性能问题在 K8s 上常被 **CPU throttle**、**错误 limits**、**邻居 Pod** 干扰。方法论：**对照**、**预热**、**分位延迟**、**剖析**。

## 4. 项目实战

### 4.1 `application.properties`

```properties
quarkus.thread-pool.max-threads=50
```

### 4.2 JVM 容器参数（`Deployment` env）

```yaml
env:
  - name: JAVA_TOOL_OPTIONS
    value: "-XX:MaxRAMPercentage=75.0 -XX:+UseG1GC"
```

### 4.3 压测脚本示例（`hey`）

```bash
hey -n 50000 -c 100 -m GET http://localhost:8080/hello
```

### 4.4 `k6` 片段（`k6-script.js`）

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 50,
  duration: '2m',
  thresholds: {
    http_req_duration: ['p(95)<200'],
  },
};

export default function () {
  const res = http.get('http://localhost:8080/hello');
  check(res, { 'status 200': (r) => r.status === 200 });
  sleep(0.05);
}
```

```bash
k6 run k6-script.js
```

### 4.5 Kubernetes：临时压测 Job（可选）

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: load-gen
spec:
  template:
    spec:
      restartPolicy: Never
      containers:
        - name: hey
          image: rfratto/hey:latest
          args: ["-n", "100000", "-c", "50", "http://api.default.svc.cluster.local/hello"]
```
