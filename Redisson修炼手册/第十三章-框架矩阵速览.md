## 趣味：换马不换缰

小白骑着 **Quarkus** 进城，还在喊：「师父，Redis 定律变了吗？」  
大师：「马换了，**缰绳还是 `Config` 与 key 设计**。别用 **Native 镜像** 省下来的启动时间，去换 **反射没配全的 Codec 惨案**。」

---

## 需求落地（由浅入深）

1. **浅**：脚手架不是 Spring——先找 **官方扩展/redisson-*** 模块，目标仍是 **一个客户端 Bean + 正确 shutdown**。  
2. **中**：对齐 **框架大版本 ↔ Redisson 模块版本**；Hibernate/MyBatis 二级缓存要读 **cache provider 文档**，别混用 **两套缓存实现写同一命名空间**。  
3. **深**：GraalVM Native：**反射、资源、Codec** 常要额外配置；CI 里加 **Testcontainers 冒烟**，避免「本地 JVM 能跑、镜像里哑火」。

---

## 对话钩子

**小白**：我们不是 Spring，是 Quarkus / Micronaut……  
**大师**：集成模块是 **胶水**；**`Config` + `R*` 语义**仍是内核。换框架 ≠ 换 Redis 定律。

**小白**：那我把 Spring 那章跳过？  
**大师**：跳过 **Bean 名字**，别跳过 **锁、队列、Codec、拓扑**——**那些章是 Redis 世界的普通话**。

## 原则

1. **先跑通最小 RedissonClient Bean**，再叠缓存/会话。  
2. **版本矩阵**：框架大版本 ↔ Redisson 模块分支，对齐 [dependency-list.md](../dependency-list.md) 或 BOM。  
3. **Native / AOT**（如 GraalVM）：注意 **反射与 Codec** 配置，按各扩展文档走。

选真实栈：**最小集成** + 打印 **Redisson 版本与连接模式**，纳入 **CI 冒烟**（连 testcontainers Redis）。
