# 第十四章：可观测性、故障演练与上线清单

[← 目录](README.md)

---

## 趣味：蒙眼飙车

**小白**：上线分布式锁了，应该稳了吧？  
**大师**：没有 **持锁时间直方图、获取失败率、等待队列** 的监控，等于 **蒙眼上高速**——还活着纯属路况好。  
运维在窗外喊：「昨晚慢查询谁干的？」——大师：**没有 traceId 的日志，等于没有证人。**

---

## 需求落地（由浅入深）

1. **浅**：上线前检查清单：**连得上、shutdown 有、密码/TLS 对**——这是 **科目一**。  
2. **中**：把 **客户端超时、池水位、重试、反序列化失败** 与 **Redis slowlog、内存、复制积压** 放进同一套大盘；故障演练 **拔节点、拔从、拉长延迟**。  
3. **深**：SRE 问「**最坏多糟**」——你要能指着面板答：**P99、错误预算、锁失败时的降级**。可观测性 **写进架构评审**，不是上线后补丁。

---

## 对话钩子

**小白**：指标要采多少才够？  
**大师**：先保证 **能回答三个问题**：谁慢了？是 Redis 还是客户端？最近发布动了吗？**采一万个没人看的指标，不如五个能定责的。**

---

## 客户端可观测

- **日志**：超时、重连、反序列化失败 → **结构化 + traceId**。  
- **指标**：QPS、P50/P99、连接池 active/idle、重试次数。  
- **Tracing**：把 Redis 段纳入全链路（代理或客户端 span，视栈而定）。

参考：[observability.md](../observability.md)、[fault-tolerance-and-recovery.md](../fault-tolerance-and-recovery.md)。

---

## Redis 侧

- **slowlog**、**latency doctor**、**内存与淘汰**、**复制积压**。  
- **热 key / 大 key**：配合 **proxy 统计** 或 **采样命令**（以运维能力为准）。

---

## 故障注入清单

1. 单节点宕机 / 主从切换  
2. 网络分区（超时飙、部分失败）  
3. 大 key / 热 key  
4. **时钟偏移**（影响 lease、超时判断）

---

## 上线 Checklist（节选）

- [ ] Codec 与 key **版本策略** 文档化  
- [ ] **环境前缀**（prod/stage）与 ACL  
- [ ] TTL、**maxmemory**、淘汰策略与业务对齐  
- [ ] 慢查询与容量 **告警**  
- [ ] 备份与 **RTO/RPO**  
- [ ] 分布式锁：**持锁时间、失败率** 面板
