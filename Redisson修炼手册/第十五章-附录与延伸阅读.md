## 需求落地（由浅入深）

1. **浅**：新项目选 Community 还是 PRO——先看 **功能对比与 license**，别写到一半换坐标 **全员编译红**。  
2. **中**：从 Jedis/Lettuce 迁移：**并存 → 统一 key/Codec 规范 → 热点迁移 → 锁队列单独评审**；**双写验证** 再切读。  
3. **深**：把 **术语表 + 反模式速记** 贴进团队 Wiki——**新人第一周读附录，比第一周读源码更省钱**。

---

## 对话钩子

**小白**：迁移能「一次性切完」吗？  
**大师**：能，**若你同时接受 P1**。业界更稳的路径是 **灰度与双写**——**慢是特征，不是缺陷**。

## Redisson PRO vs Community

- 对比：[feature comparison](https://redisson.pro/feature-comparison.html)。  
- **约定**：本文默认 **Community**；PRO 能力以官网与 **license** 为准，勿混坐标（`org.redisson` vs `pro.redisson`）。

---

## 迁移路径（Jedis / Lettuce / 自研）

1. **并存**：新链路 Redisson，老 key 不动。  
2. **统一 Codec 与 key 规范**，再迁移热点。  
3. **锁/队列** 单独评审——不是 API 1:1 翻译。  
4. **双写验证** → 切读 → 下老代码。

## 术语表（精简）

| 术语 | 含义 |
|------|------|
| `R*` | Redisson 分布式对象类型前缀 |
| watchdog | 客户端侧锁租约 **自动续期**（与 API/配置相关） |
| lease | 租约到期时间 |
| at-least-once | 至少投递一次，**可能重复** |
| MOVED/ASK | 集群槽迁移时的重定向 |

## 「别用错武器」速记（趣味收官）

- List 当 **强可靠 MQ** → 先问 **ACK 与积压**。  
- Pub/Sub 当 **消息队列** → 离线消息 **谁存**？  
- Redis 事务当 **分布式 DB 事务** → 醒醒。  
- 分布式锁当 **正确性唯一支柱** → **幂等 + DB 约束** 请就位。  
- `KEYS *` 在生产 → **掌柜逐桌数筷子数到打烊**。
