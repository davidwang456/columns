# 第2章：环境搭建——单机模式与伪集群部署

## 1. 项目背景

### 业务场景

你刚刚了解了 ZooKeeper 的基本概念和架构原理，现在迫不及待地想在自己电脑上跑起来看看。毕竟，没有什么比亲手启动一个服务更能加深理解了。

你可能面临的选择：
- **单机模式**：只有一个 ZooKeeper 进程，适合开发测试，但无法体验 Leader 选举和集群数据同步
- **伪集群模式**：在一台机器上启动多个 ZooKeeper 进程模拟集群，既能体验集群功能，又不增加硬件成本
- **Docker Compose**：用容器编排一键启动集群，最干净利落的方式

### 痛点放大

不正确的安装配置会带来各种莫名其妙的问题：

- 版本下载错了，遇到不兼容的 API
- zoo.cfg 参数没配对，集群起不来
- myid 文件写错了，节点之间无法通信
- 端口冲突，进程启动失败
- dataDir 权限不对，写入数据时报错

本章的目标很简单：**让你在 10 分钟内，用三种方式把 ZooKeeper 跑起来**。

## 2. 项目设计

### 剧本式交锋对话

**场景**：开发机上，小胖正在下载 ZooKeeper，小白在研究配置文件。

**小胖**：我去官网下载了最新版 3.9.x 的 tar.gz，解压后直接 `./bin/zkServer.sh start`，这就跑起来了吧？

**小白**：等等，你有没有看 `conf/zoo.cfg`？我上次就是直接启动，结果 dataDir 默认是 `/tmp/zookeeper`，重启电脑数据全丢了。

**大师**：小白说得对。单机模式的配置至少有这几个参数必须配：

```
tickTime=2000
dataDir=/usr/local/zookeeper/data
clientPort=2181
maxClientCnxns=60
```

- **tickTime**：ZooKeeper 的基本时间单位（毫秒）。心跳和超时的基础粒度，2000ms 是推荐值。
- **dataDir**：数据存储目录。重要！不要用默认路径。
- **clientPort**：客户端连接端口，默认 2181。
- **maxClientCnxns**：单个 IP 的最大客户端连接数，0 表示不限制。

**小胖**：单机模式太简单了，我想试试集群模式，但就一台电脑怎么办？

**大师**：一台机器也能跑集群，这叫**伪集群模式**。原理很简单——启动多个 ZooKeeper 进程，每个进程用不同的端口和数据目录。

**小白**：那集群之间怎么发现对方？它们怎么知道谁是 Leader 谁是 Follower？

**大师**：看 `zoo.cfg` 里的这些配置：

```properties
server.1=127.0.0.1:2888:3888
server.2=127.0.0.1:2889:3889
server.3=127.0.0.1:2890:3890
```

格式是 `server.id=host:port1:port2`：
- **port1**（2888）：Leader 和 Follower 之间的通信端口（数据同步）
- **port2**（3888）：选举端口（Leader 选举时的通信）

**小胖**：那每个节点怎么知道自己的 id？

**大师**：通过 `dataDir/myid` 文件，里面只写一个数字。比如 server.1 对应 myid = 1。启动时 ZooKeeper 读这个文件就知道自己是第几个节点了。

> **技术映射**：server.id = 每个人分配一个工号，myid = 你的工牌

**小白**：Docker 部署是不是更方便？环境隔离，一键启动，用完就删。

**大师**：没错。用 Docker Compose 可以编排一个完整的 ZooKeeper 集群，而且可以加上一个 Web UI 工具（比如 ZooNavigator），用浏览器就能看到集群状态，非常方便。

> **技术映射**：Docker Compose = 一键组装车间，ZooNavigator = 车间监控大屏

## 3. 项目实战

### 环境准备

- JDK 11+（ZooKeeper 3.9.x 需要 JDK 11）
- 操作系统：macOS / Linux / Windows（WSL2 推荐）
- Docker Desktop（可选，用于 Docker Compose 部署）
- 磁盘空间：500MB+

### 分步实现

#### 方式一：单机模式（裸机）

**步骤目标**：在本地启动一个单机 ZooKeeper 实例。

```bash
# 1. 下载 ZooKeeper 3.9.x
wget https://dlcdn.apache.org/zookeeper/zookeeper-3.9.2/apache-zookeeper-3.9.2-bin.tar.gz
tar -xzf apache-zookeeper-3.9.2-bin.tar.gz
cd apache-zookeeper-3.9.2-bin

# 2. 创建数据目录
mkdir -p data

# 3. 复制配置文件
cp conf/zoo_sample.cfg conf/zoo.cfg

# 4. 修改 zoo.cfg
# 将 dataDir=/tmp/zookeeper 改为 dataDir=./data （或者绝对路径）

# 5. 启动
./bin/zkServer.sh start

# 输出：ZooKeeper JMX enabled by default
# Using config: ./bin/../conf/zoo.cfg
# Starting zookeeper ... STARTED

# 6. 验证
./bin/zkServer.sh status
# 输出：Client port found: 2181. Client address: localhost.
# Mode: standalone

# 7. 连接测试
./bin/zkCli.sh -server 127.0.0.1:2181
# 进入交互模式后执行
ls /
# 输出：[zookeeper]

# 8. 停止
./bin/zkServer.sh stop
```

**可能遇到的坑**：

| 问题 | 原因 | 解决 |
|------|------|------|
| `java: command not found` | JDK 未安装或未配置 PATH | `export JAVA_HOME=... && export PATH=$JAVA_HOME/bin:$PATH` |
| `Address already in use` | 端口被占用 | 检查端口 `lsof -i:2181`，更换端口或 kill 旧进程 |
| `Permission denied` | dataDir 目录无写入权限 | `chmod -R 755 data/` |

#### 方式二：伪集群模式（3 节点）

**步骤目标**：在一台机器上启动 3 个 ZooKeeper 节点，模拟集群。

```bash
# 1. 创建目录结构
mkdir -p cluster/zk1/data cluster/zk1/logs
mkdir -p cluster/zk2/data cluster/zk2/logs
mkdir -p cluster/zk3/data cluster/zk3/logs

# 2. 创建 myid 文件（每个节点唯一 ID）
echo "1" > cluster/zk1/data/myid
echo "2" > cluster/zk2/data/myid
echo "3" > cluster/zk3/data/myid

# 3. 创建配置文件
# zk1 的配置文件：cluster/zk1/zoo.cfg
cat > cluster/zk1/zoo.cfg << 'EOF'
tickTime=2000
initLimit=10
syncLimit=5
dataDir=./cluster/zk1/data
clientPort=2181
server.1=127.0.0.1:2888:3888
server.2=127.0.0.1:2889:3889
server.3=127.0.0.1:2890:3890
EOF

# zk2 的配置文件类似，区别是 dataDir=./cluster/zk2/data, clientPort=2182
# zk3 的配置文件类似，区别是 dataDir=./cluster/zk3/data, clientPort=2183

# 4. 依次启动三个节点
./bin/zkServer.sh start cluster/zk1/zoo.cfg
./bin/zkServer.sh start cluster/zk2/zoo.cfg
./bin/zkServer.sh start cluster/zk3/zoo.cfg

# 5. 查看各节点状态
./bin/zkServer.sh status cluster/zk1/zoo.cfg
# Mode: follower
./bin/zkServer.sh status cluster/zk2/zoo.cfg
# Mode: leader      ← 其中一个会被选为 Leader
./bin/zkServer.sh status cluster/zk3/zoo.cfg
# Mode: follower

# 6. 连接任意节点验证数据同步
./bin/zkCli.sh -server 127.0.0.1:2181
# 创建一个测试节点
create /test-cluster "hello cluster"
# 用另一个客户端连接另一个端口验证
./bin/zkCli.sh -server 127.0.0.1:2182
get /test-cluster
# 输出：hello cluster   ← 数据已同步！
```

#### 方式三：Docker Compose 一键部署

**步骤目标**：用 Docker Compose 启动 3 节点 ZooKeeper 集群 + ZooNavigator UI。

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  zoo1:
    image: zookeeper:3.9
    restart: always
    hostname: zoo1
    container_name: zookeeper-1
    ports:
      - "2181:2181"
    environment:
      ZOO_MY_ID: 1
      ZOO_SERVERS: server.1=0.0.0.0:2888:3888;2181 server.2=zoo2:2888:3888;2181 server.3=zoo3:2888:3888;2181
      ZOO_4LW_COMMANDS_WHITELIST: "*"
    volumes:
      - zoo1-data:/data
      - zoo1-datalog:/datalog

  zoo2:
    image: zookeeper:3.9
    restart: always
    hostname: zoo2
    container_name: zookeeper-2
    ports:
      - "2182:2181"
    environment:
      ZOO_MY_ID: 2
      ZOO_SERVERS: server.1=zoo1:2888:3888;2181 server.2=0.0.0.0:2888:3888;2181 server.3=zoo3:2888:3888;2181
      ZOO_4LW_COMMANDS_WHITELIST: "*"
    volumes:
      - zoo2-data:/data
      - zoo2-datalog:/datalog

  zoo3:
    image: zookeeper:3.9
    restart: always
    hostname: zoo3
    container_name: zookeeper-3
    ports:
      - "2183:2181"
    environment:
      ZOO_MY_ID: 3
      ZOO_SERVERS: server.1=zoo1:2888:3888;2181 server.2=zoo2:2888:3888;2181 server.3=0.0.0.0:2888:3888;2181
      ZOO_4LW_COMMANDS_WHITELIST: "*"
    volumes:
      - zoo3-data:/data
      - zoo3-datalog:/datalog

  zookeeper-ui:
    image: elkozmon/zoonavigator:latest
    container_name: zoonavigator
    ports:
      - "9000:9000"
    environment:
      HTTP_PORT: 9000
    depends_on:
      - zoo1
      - zoo2
      - zoo3

volumes:
  zoo1-data:
  zoo1-datalog:
  zoo2-data:
  zoo2-datalog:
  zoo3-data:
  zoo3-datalog:
```

启动命令：

```bash
docker-compose up -d
# 查看集群状态
docker-compose ps
# 验证集群
docker exec zookeeper-1 zkServer.sh status
# 在浏览器打开 http://localhost:9000，添加连接 ZooKeeper 集群地址：zoo1:2181,zoo2:2181,zoo3:2181
```

### 测试验证

编写一个简单的验证脚本 `verify-cluster.sh`：

```bash
#!/bin/bash
# 验证集群三个节点是否正常工作

PORTS=(2181 2182 2183)
for port in "${PORTS[@]}"; do
  echo "Checking port $port..."
  echo ruok | nc -q 1 127.0.0.1 $port
  echo ""
done

echo "=== Cluster Status ==="
for port in "${PORTS[@]}"; do
  echo stat | nc -q 1 127.0.0.1 $port | grep "Mode"
done
```

期望输出：
```
Checking port 2181...
imok

Checking port 2182...
imok

Checking port 2183...
imok

=== Cluster Status ===
Mode: leader
Mode: follower
Mode: follower
```

## 4. 项目总结

### 优点 & 缺点

| 部署方式 | 优点 | 缺点 |
|---------|------|------|
| 单机模式 | 简单快速，适合开发测试 | 无法体验集群特性，单点故障 |
| 伪集群模式 | 一台机器体验完整集群功能 | 端口管理麻烦，资源争抢 |
| Docker Compose | 环境隔离、一键启停、易清理 | 需要 Docker 环境，网络稍复杂 |

### 适用场景

- **开发调试**：单机模式或 Docker 伪集群
- **学习研究**：伪集群模式体验选举和数据同步
- **生产部署**：裸机多节点或 K8s StatefulSet（后续章节详解）

### 注意事项

- ZooKeeper 3.9.x 需要 JDK 11+，3.5.x 需要 JDK 8+
- Docker 镜像默认开启 Admin Server，端口 8080，注意不要冲突
- 生产环境不要使用伪集群模式，多个进程争抢同一台机器的 CPU/内存/磁盘 IO，性能不可控
- myid 文件中的数字必须和 server.id 配置一致

### 常见踩坑经验

**故障 1：伪集群启动后 Mode 全是 standalone**

现象：三个节点都显示 Mode: standalone，没有选出 Leader。

根因：`zoo.cfg` 中缺少 `server.x` 配置，ZooKeeper 检测到没有配置集群节点，自动以单机模式运行。

**故障 2：Docker 容器启动后马上退出**

现象：Docker Compose 启动后容器自动退出，`docker logs` 显示退出码 1。

根因：`ZOO_SERVERS` 环境变量格式错误。注意 Docker 镜像的 ZOO_SERVERS 格式最后要加 `;2181`，这是 Docker 镜像特有的语法（指定 client 端口）。

### 思考题

1. 为什么 ZooKeeper 推荐奇数节点（3/5/7）？如果部署 4 个节点，和 3 个节点相比有什么不同？
2. 伪集群模式下，如果你同时 `kill` 掉 Follower 和 Leader，剩下的节点能重新选举出新的 Leader 吗？

### 推广计划提示

- **开发**：本章完成了环境搭建，后续可以在本地 ZooKeeper 上做各种实验
- **运维**：建议使用 Docker Compose 部署方式，环境一致性好，方便团队统一开发环境
- **测试**：部署完成后，可以尝试手动 kill 节点，观察集群的自动恢复过程
