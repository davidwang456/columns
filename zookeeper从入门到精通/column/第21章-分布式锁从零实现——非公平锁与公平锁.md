# 第21章：分布式锁从零实现——非公平锁与公平锁

## 1. 项目背景

### 业务场景

第 9 章中直接使用了 Curator 的 `InterProcessMutex` 实现了分布式锁。但 Curator 底层是怎么实现锁的？为什么`InterProcessMutex` 能保证互斥？

本章将从零开始，用 ZooKeeper 原生 API 实现分布式锁。这不仅让你理解 Curator 的内部原理，还能在不需要引入 Curator 的场景下自己实现轻量锁。

### 痛点放大

自己实现分布式锁，第一次尝试往往会踩这些坑：

- **羊群效应（Herd Effect）**：所有等待锁的客户端都 Watcher 同一个节点，锁释放时所有客户端同时被唤醒
- **惊群效应（Thundering Herd）**：同时创建同一个节点导致大量 write 冲突
- **死锁**：获取锁的线程挂了但没有释放锁
- **不可重入**：同一个线程如果递归调用需要锁的方法，会死锁
- **性能差**：每个操作都创建/删除 ZNode，产生大量 write 请求

## 2. 项目设计

### 剧本式交锋对话

**场景**：小胖想自己实现一个分布式锁，用最简单的方案。

**小胖**：分布式锁不就是创建一个临时节点吗？创建成功的获得锁，删除节点释放锁，其他客户端等着。

```java
// 非公平锁——版本 1
void lock() {
    while (true) {
        try {
            zk.create("/lock", ...); // 创建成功 = 获得锁
            break;
        } catch (NodeExistsException e) {
            // 锁已被其他客户端持有，重试
            Thread.sleep(100);
        }
    }
}
```

**大师**：这个方案有两个大问题：

1. **羊群效应**：所有等待锁的客户端都在拼命 `create` 同一个路径，造成大量 write 冲突
2. **忙等待**：`while(true)` + `Thread.sleep(100)` 要么浪费 CPU，要么响应慢

**小白**：那更好的方案是什么？我听说要用临时顺序节点？

**大师**：对，**公平锁的实现就是基于临时顺序节点 + Watcher 前驱节点**。

```
公平锁原理：

1. 每个客户端在 /lock 下创建一个临时顺序节点
   客户端 A → /lock/lock-0000000001
   客户端 B → /lock/lock-0000000002
   客户端 C → /lock/lock-0000000003

2. 排序：序号最小的客户端获得锁
   客户端 A 获得锁（序号 1 最小）

3. 其他客户端 Watcher 前一个节点
   客户端 B 监听 /lock/lock-0000000001
   客户端 C 监听 /lock/lock-0000000002

4. 客户端释放锁 → 删除自己的节点
   客户端 A 删除 /lock/lock-0000000001
   → 客户端 B 的 Watcher 被触发
   → 客户端 B 检查自己的序号（2）是否最小 → 是 → 获得锁

5. 关键：每个客户端只 Watcher 前一个节点，不是所有节点
   避免了羊群效应！
```

**小胖**：哦！我知道了！这样每次锁释放时，只有等待的"下一个人"被唤醒，不是所有人。

> **技术映射**：非公平锁 = 一群人抢一把钥匙，谁抢到谁开门；公平锁 = 取号排队，叫到号的人开门

**小白**：那实现过程中有哪些边界情况？比如客户端获取锁后死了，锁怎么释放？

**大师**：使用**临时节点**而不是持久节点。客户端会话断开 → 临时节点自动删除 → 锁自然释放。这是 ZooKeeper 分布式锁比 Redis 分布式锁更安全的核心原因之一。

**小胖**：可重入呢？如果一个线程调用了两次 `lock()`，总不能死锁吧？

**大师**：可重入的关键是**每个线程维护一个本地计数器**：

```java
ThreadLocal<Integer> reentrantCount = new ThreadLocal<>();

void lock() {
    if (reentrantCount.get() != null && reentrantCount.get() > 0) {
        // 已经持有锁，只是计数器 +1
        reentrantCount.set(reentrantCount.get() + 1);
        return;
    }
    // 真正的获取锁逻辑（创建临时顺序节点）
    doLock();
    reentrantCount.set(1);
}

void unlock() {
    if (reentrantCount.get() == null || reentrantCount.get() <= 0) {
        throw new Exception("没有锁可释放");
    }
    // 计数器 -1
    reentrantCount.set(reentrantCount.get() - 1);
    if (reentrantCount.get() == 0) {
        // 真正的释放锁逻辑（删除节点）
        doUnlock();
    }
}
```

#### 步骤 1：非公平锁实现

创建 `SimpleUnfairLock.java`：

```java
package com.zkdemo.lock;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.Collections;
import java.util.List;
import java.util.concurrent.CountDownLatch;

/**
 * 非公平锁——所有客户端争抢同一个临时节点
 * 优点：实现简单
 * 缺点：羊群效应、忙等待
 */
public class SimpleUnfairLock implements DistributedLock {
    private static final String LOCK_ROOT = "/unfair-locks";
    private final ZooKeeper zk;
    private final String lockPath;
    private final String lockNode;
    private String currentNode;

    public SimpleUnfairLock(ZooKeeper zk, String lockName) {
        this.zk = zk;
        this.lockPath = LOCK_ROOT + "/" + lockName;
        this.lockNode = lockPath;
    }

    @Override
    public void lock() throws Exception {
        // 确保父节点存在
        ensureParentExists();

        while (true) {
            try {
                // 尝试创建临时节点（创建成功 = 获得锁）
                currentNode = zk.create(lockNode, Thread.currentThread().getName().getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);
                System.out.println(Thread.currentThread().getName() + " 获得锁: " + currentNode);
                return;
            } catch (KeeperException.NodeExistsException e) {
                // 锁已被其他客户端持有
                // 注册 Watcher 等待锁释放
                final CountDownLatch latch = new CountDownLatch(1);
                Stat stat = zk.exists(lockNode, event -> {
                    if (event.getType() == Watcher.Event.EventType.NodeDeleted) {
                        latch.countDown();
                    }
                });

                if (stat != null) {
                    System.out.println(Thread.currentThread().getName() + " 等待锁释放...");
                    latch.await(); // 等待锁释放
                }
            }
        }
    }

    @Override
    public void unlock() throws Exception {
        if (currentNode != null) {
            zk.delete(currentNode, -1);
            System.out.println(Thread.currentThread().getName() + " 释放锁");
            currentNode = null;
        }
    }

    private void ensureParentExists() throws Exception {
        Stat stat = zk.exists(LOCK_ROOT, false);
        if (stat == null) {
            try {
                zk.create(LOCK_ROOT, "".getBytes(), ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
            } catch (KeeperException.NodeExistsException e) {
                // 并发创建，忽略
            }
        }
    }
}
```

#### 步骤 2：公平锁实现

创建 `FairDistributedLock.java`：

```java
package com.zkdemo.lock;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.Collections;
import java.util.List;
import java.util.concurrent.CountDownLatch;

/**
 * 公平分布式锁——基于临时顺序节点 + Watcher 前驱节点
 * 优点：无羊群效应、公平排队
 * 缺点：实现复杂
 */
public class FairDistributedLock implements DistributedLock {
    private static final String LOCK_ROOT = "/fair-locks";
    private final ZooKeeper zk;
    private final String lockPath;
    private final ThreadLocal<Integer> reentrantCount = ThreadLocal.withInitial(() -> 0);
    private final ThreadLocal<String> currentNode = new ThreadLocal<>();
    private final ThreadLocal<String> watchedNode = new ThreadLocal<>();

    public FairDistributedLock(ZooKeeper zk, String lockName) {
        this.zk = zk;
        this.lockPath = LOCK_ROOT + "/" + lockName;
    }

    @Override
    public void lock() throws Exception {
        // 可重入检查
        if (reentrantCount.get() > 0) {
            reentrantCount.set(reentrantCount.get() + 1);
            return;
        }

        ensureParentExists();

        // 1. 创建临时顺序节点
        String myNode = zk.create(lockPath + "/lock-",
                Thread.currentThread().getName().getBytes(),
                ZooDefs.Ids.OPEN_ACL_UNSAFE,
                CreateMode.EPHEMERAL_SEQUENTIAL);
        currentNode.set(myNode);
        System.out.println(Thread.currentThread().getName() + " 创建节点: " + myNode);

        // 2. 获取锁路径下的所有子节点
        List<String> children = zk.getChildren(lockPath, false);
        Collections.sort(children);

        // 3. 提取当前节点的序号
        String mySeq = myNode.substring(myNode.lastIndexOf("-") + 1);

        // 4. 检查自己是不是最小的
        if (mySeq.equals(children.get(0).substring(children.get(0).lastIndexOf("-") + 1))) {
            // 序号最小，获得锁
            reentrantCount.set(1);
            System.out.println(Thread.currentThread().getName() + " 获得锁（序号最小）");
            return;
        }

        // 5. 找到前一个节点并注册 Watcher
        String myName = myNode.substring(myNode.lastIndexOf("/") + 1);
        int myIndex = children.indexOf(myName);
        String prevNode = children.get(myIndex - 1);
        String prevPath = lockPath + "/" + prevNode;
        watchedNode.set(prevPath);

        final CountDownLatch latch = new CountDownLatch(1);
        Stat stat = zk.exists(prevPath, event -> {
            if (event.getType() == Watcher.Event.EventType.NodeDeleted) {
                latch.countDown();
            }
        });

        // 双重检查：如果前驱节点已经不存在，重新判断
        if (stat == null) {
            // 前驱节点已经被删除（锁已释放），重新尝试
            lock();
            return;
        }

        System.out.println(Thread.currentThread().getName()
                + " 等待前驱节点释放: " + prevPath);
        latch.await(); // 等待前驱节点被删除

        // 前驱释放后，递归尝试获取锁（重新检查序号）
        reentrantCount.set(0);
        lock();
    }

    @Override
    public void unlock() throws Exception {
        if (reentrantCount.get() <= 0) {
            throw new IllegalMonitorStateException("没有锁可释放");
        }

        reentrantCount.set(reentrantCount.get() - 1);

        if (reentrantCount.get() == 0) {
            String node = currentNode.get();
            if (node != null) {
                zk.delete(node, -1);
                System.out.println(Thread.currentThread().getName() + " 释放锁");
                currentNode.remove();
                watchedNode.remove();
            }
        }
    }

    private void ensureParentExists() throws Exception {
        Stat stat = zk.exists(lockPath, false);
        if (stat == null) {
            try {
                zk.create(LOCK_ROOT, "".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
                zk.create(lockPath, "".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
            } catch (KeeperException.NodeExistsException e) {
                // 并发创建，忽略
            }
        }
    }
}
```

package com.zkdemo.lock;

import org.apache.zookeeper.Watcher;
import org.apache.zookeeper.ZooKeeper;

import java.util.concurrent.CountDownLatch;
import java.util.concurrent.atomic.AtomicInteger;

public class LockTest {
    private static final String ZK_URL = "127.0.0.1:2181";

    public static void main(String[] args) throws Exception {
        CountDownLatch latch = new CountDownLatch(1);
        ZooKeeper zk = new ZooKeeper(ZK_URL, 5000, event -> {
            if (event.getState() == Watcher.Event.KeeperState.SyncConnected) {
                latch.countDown();
            }
        });
        latch.await();

        System.out.println("=== 公平锁测试 ===\n");
        testFairLock(zk);

        System.out.println("\n=== 对比: Curator InterProcessMutex ===");
        testCuratorLock();

        zk.close();
    }

    static void testFairLock(ZooKeeper zk) throws Exception {
        FairDistributedLock lock = new FairDistributedLock(zk, "test");

        // 测试 1：基础加锁/解锁
        System.out.println("--- 测试 1: 基础加锁/解锁 ---");
        lock.lock();
        System.out.println("锁已获取");
        lock.unlock();
        System.out.println("锁已释放\n");

        // 测试 2：可重入
        System.out.println("--- 测试 2: 可重入 ---");
        lock.lock();
        lock.lock(); // 重入
        System.out.println("重入计数: 2");
        lock.unlock();
        lock.unlock();
        System.out.println("重入锁完全释放\n");

        // 测试 3：并发竞争
        System.out.println("--- 测试 3: 并发竞争 ---");
        int threadCount = 5;
        CountDownLatch startSignal = new CountDownLatch(1);
        CountDownLatch doneSignal = new CountDownLatch(threadCount);
        AtomicInteger concurrentMax = new AtomicInteger(0);
        AtomicInteger concurrent = new AtomicInteger(0);

        for (int i = 0; i < threadCount; i++) {
            final int id = i;
            new Thread(() -> {
                try {
                    startSignal.await();
                    FairDistributedLock l = new FairDistributedLock(zk, "test");
                    l.lock();
                    int c = concurrent.incrementAndGet();
                    concurrentMax.updateAndGet(v -> Math.max(v, c));
                    Thread.sleep(200); // 模拟业务
                    concurrent.decrementAndGet();
                    l.unlock();
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    doneSignal.countDown();
                }
            }, "Thread-" + i).start();
        }

        startSignal.countDown();
        doneSignal.await();
        System.out.println("最大并发持有锁数: " + concurrentMax.get()
                + (concurrentMax.get() <= 1 ? " ✓ 互斥性满足" : " ✗ 互斥性失败"));
    }

#### 步骤 4：压测对比

创建 `LockBenchmark.java`：

```java
package com.zkdemo.lock;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.locks.InterProcessMutex;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.zookeeper.Watcher;
import org.apache.zookeeper.ZooKeeper;

import java.util.concurrent.CountDownLatch;
import java.util.concurrent.atomic.AtomicInteger;

public class LockBenchmark {
    private static final String ZK_URL = "127.0.0.1:2181";

    public static void main(String[] args) throws Exception {
        System.out.println("=== 分布式锁基准测试 ===\n");

        CountDownLatch latch = new CountDownLatch(1);
        ZooKeeper zk = new ZooKeeper(ZK_URL, 5000, event -> {
            if (event.getState() == Watcher.Event.KeeperState.SyncConnected) {
                latch.countDown();
            }
        });
        latch.await();

        int threadCount = 20;
        int iterations = 50;

        // 测试公平锁
        System.out.println("1. 公平锁（手写）");
        long fairTime = benchmark(threadCount, iterations, () -> {
            FairDistributedLock lock = new FairDistributedLock(zk, "bench");
            return new LockPair(lock, lock);
        });
        System.out.printf("   总耗时: %d ms%n%n", fairTime);

        // 测试 Curator 锁
        System.out.println("2. Curator InterProcessMutex");
        long curatorTime = benchmark(threadCount, iterations, () -> {
            CuratorFramework client = CuratorFrameworkFactory.newClient(
                    ZK_URL, new ExponentialBackoffRetry(1000, 3));
            client.start();
            InterProcessMutex lock = new InterProcessMutex(client, "/curator-bench");
            return new LockPair(lock::acquire, lock::release);
        });
        System.out.printf("   总耗时: %d ms%n%n", curatorTime);

        zk.close();
    }

    static long benchmark(int threads, int iterations,
                          LockFactory factory) throws Exception {
        AtomicInteger counter = new AtomicInteger(0);
        CountDownLatch doneSignal = new CountDownLatch(threads);
        long start = System.currentTimeMillis();

        for (int i = 0; i < threads; i++) {
            new Thread(() -> {
                try {
                    LockPair pair = factory.create();
                    for (int j = 0; j < iterations; j++) {
                        pair.lock.lock();
                        counter.incrementAndGet();
                        pair.lock.unlock();
                    }
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    doneSignal.countDown();
                }
            }).start();
        }

        doneSignal.await();
        long elapsed = System.currentTimeMillis() - start;
        System.out.printf("   线程: %d, 迭代: %d, 总操作: %d%n",
                threads, iterations, threads * iterations);
        return elapsed;
    }

    interface LockFactory {
        LockPair create() throws Exception;
    }

    record LockPair(AutoCloseable lock, AutoCloseable unlock) {
    }
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 节点创建成功但序号非最小 | 并发创建后还未排序 | 获取所有节点后重新检查 |
| Watcher 未触发 | 前驱节点已被删除（Stat 返回 null） | 双重检查，重新获取锁 |
| 锁释放后 Watcher 仍被触发 | 前驱节点可能不是自己监听的 | 在 Watcher 中检查自己的序号 |

## 4. 项目总结

### 非公平锁 vs 公平锁

| 特性 | 非公平锁 | 公平锁 |
|------|---------|--------|
| 原理 | 争抢同一个临时节点 | 临时顺序节点排队 |
| 羊群效应 | 严重（所有客户端竞争同一节点） | 无（只监听前驱节点） |
| 公平性 | 不公平（谁来谁有，可能"饿死"） | 公平（先来先得） |
| 性能 | 高冲突时差 | 稳定 |
| 实现复杂度 | 简单 | 中等 |
| 典型实现 | 简单锁 | InterProcessMutex |

### 适用场景

- **非公平锁**：锁持有时间很短，竞争不激烈（如缓存更新）
- **公平锁**：锁持有时间较长，或需要公平调度的场景

### 注意事项

- 临时节点自动释放保证不会死锁（即使客户端挂了）
- 可重入使用 ThreadLocal 计数，跨线程不重入
- 连接丢失时锁状态不确定，需要监听 ConnectionStateListener

### 常见踩坑经验

**故障 1：锁释放前连接断开，临时节点被删除**

现象：客户端获取锁后，网络抖动导致临时节点被 ZooKeeper 服务端删除，其他客户端获取了锁，但原客户端还在执行业务。

根因：会话超时后临时节点被自动删除，但原客户端无法感知（线程在执行业务代码，没处理连接事件）。

解决方案：在业务代码中加入"持有锁检测"，定期检查当前会话是否有效。

**故障 2：公平锁的 Watcher 遗漏**

现象：客户端 A 等待锁释放，监听前驱节点 B。节点 B 被删除，但 A 的 Watcher 没有触发。

根因：在 `exists(prevPath, watcher)` 中，如果节点 B 在 Watcher 注册前已经被删除，Watcher 不会触发。

解决方案：`exists` 返回 null 时，说明前驱已经不存在，此时应立即重新检查锁状态，而不是等待。

### 思考题

1. 公平锁实现中，每一个等待的客户端都 Watcher 前一个节点。但如果前一个节点是持久节点（故障节点残留），这个 Watcher 永远不会触发，怎么解决？
2. 非公平锁的"争抢同一个节点"方案在高并发下成功率很低（大量 write 冲突）。如果改成"先创建/queue 节点排队，然后 Watcher 这个节点"，这个方案和公平锁有什么区别？

### 推广计划提示

- **开发**：生产环境直接使用 Curator 的 InterProcessMutex，但理解其底层原理对于排查锁相关的问题非常有帮助
- **测试**：分布式锁的测试必须覆盖：并发竞争、连接断开、会话过期、Watcher 丢失等异常场景
- **运维**：监控 /locks/ 路径下的临时节点数量，异常增长说明可能有锁泄漏
