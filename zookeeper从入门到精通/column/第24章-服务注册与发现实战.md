# 第24章：服务注册与发现实战

## 1. 项目背景

### 业务场景

微服务架构中，服务 A 要调用服务 B，但服务 B 有多个实例，IP 和端口都是动态的（容器化部署、弹性伸缩）。服务 A 怎么知道服务 B 当前有哪些可用的实例？

这就是**服务注册与发现**的核心问题。ZooKeeper 利用**临时节点** + **Watcher** 天然适合实现服务注册与发现：

- 服务启动时在 ZooKeeper 上注册自己的地址（临时节点）
- 服务下线时临时节点自动删除（断连感知）
- 其他服务通过 Watcher 实时感知服务列表变化

### 痛点放大

没有服务发现机制，需要硬编码服务地址：

- 配置文件写死 IP:Port，扩缩容时手动修改
- 负载均衡器（Nginx）需要手动添加/删除后端实例
- 容器化环境中 IP 动态分配，无法预先知道地址

## 2. 项目设计

### 剧本式交锋对话

**场景**：小胖的团队在容器化部署时遇到了服务地址变动的麻烦。

**小胖**：我们的订单服务部署了 3 个实例，IP 每次重启都会变。用户服务调用订单服务时，配置的地址不对，调用失败。

**大师**：这就是服务注册与发现的典型场景。用 ZooKeeper 做，设计如下：

```
/services/{serviceName}/{instanceId} → "ip:port:weight:group:version"

示例：
/services/order-service/192.168.1.1:8080 → {"ip":"192.168.1.1","port":8080,"weight":10}
/services/order-service/192.168.1.2:8080 → {"ip":"192.168.1.2","port":8080,"weight":10}
/services/user-service/192.168.1.3:8081 → {"ip":"192.168.1.3","port":8081,"weight":5}
```

**注册流程**：

```java
// 服务启动时
String path = "/services/order-service/192.168.1.1:8080";
zk.create(path, jsonData.getBytes(), ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);
```

**发现流程**：

```java
// 服务调用时
List<String> instances = zk.getChildren("/services/order-service", true);
// 轮询或加权轮询选择实例
String target = selectInstance(instances);
```

**小白**：用临时节点做心跳检测——服务实例断连后临时节点自动删除，调用方实时感知。这比 Eureka 的心跳机制简单得多。

**大师**：对的。Eureka 需要客户端主动发心跳，服务端超时未收到心跳则剔除。ZooKeeper 利用 TCP 连接本身（Session）来做心跳检测——**连接断开 = 心跳停止 = 服务下线**，更加优雅。

**小胖**：那如果我想区分服务版本呢？比如 v1 和 v2 同时存在？

**大师**：路径中可以加入版本：

```
/services/user-service/v1/192.168.1.3:8081
/services/user-service/v2/192.168.1.4:8082
```

调用方根据配置的服务版本号，选择对应的路径下的实例。

> **技术映射**：服务注册 = 在电话簿上登记你的号码，服务发现 = 查电话簿找号码，临时节点 = 一次性的电话号码（人走了号码自动注销）

### 分步实现

#### 步骤 1：服务注册实现

创建 `ServiceRegistry.java`：

```java
package com.zkdemo.discovery;

import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.imps.CuratorFrameworkState;
import org.apache.curator.framework.state.ConnectionState;
import org.apache.curator.framework.state.ConnectionStateListener;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.zookeeper.CreateMode;

import java.util.Map;

public class ServiceRegistry implements AutoCloseable {
    private static final String SERVICES_ROOT = "/services";
    private final CuratorFramework client;
    private final ObjectMapper mapper = new ObjectMapper();
    private String registeredPath;

    public ServiceRegistry(String connectString) {
        client = CuratorFrameworkFactory.builder()
                .connectString(connectString)
                .retryPolicy(new ExponentialBackoffRetry(1000, 3))
                .build();

        // 监听连接状态：断线重连后重新注册
        client.getConnectionStateListenable().addListener((c, newState) -> {
            if (newState == ConnectionState.RECONNECTED) {
                System.out.println("重连成功，重新注册服务...");
                if (registeredPath != null) {
                    try {
                        register(registeredPath);
                    } catch (Exception e) {
                        System.err.println("重连后注册失败: " + e.getMessage());
                    }
                }
            }
        });

        client.start();
    }

    public void register(String serviceName, String instanceId,
                         Map<String, Object> metadata) throws Exception {
        // 确保根路径存在
        if (client.checkExists().forPath(SERVICES_ROOT) == null) {
            client.create().forPath(SERVICES_ROOT);
        }

        String servicePath = SERVICES_ROOT + "/" + serviceName;
        if (client.checkExists().forPath(servicePath) == null) {
            client.create().forPath(servicePath);
        }

        // 创建临时节点存储实例信息
        String instancePath = servicePath + "/" + instanceId;
        String jsonData = mapper.writeValueAsString(metadata);
        client.create().withMode(CreateMode.EPHEMERAL)
                .forPath(instancePath, jsonData.getBytes());

        this.registeredPath = instancePath;
        System.out.println("服务注册成功: " + instancePath);
        System.out.println("元数据: " + jsonData);
    }

    // 简化注册——只注册 IP:Port
    public void register(String serviceName, String host, int port) throws Exception {
        String instanceId = host + ":" + port;
        Map<String, Object> metadata = Map.of(
                "ip", host,
                "port", port,
                "timestamp", System.currentTimeMillis()
        );
        register(serviceName, instanceId, metadata);
    }

    public void unregister() throws Exception {
        if (registeredPath != null) {
            client.delete().guaranteed().forPath(registeredPath);
            System.out.println("服务注销: " + registeredPath);
            registeredPath = null;
        }
    }

    @Override
    public void close() {
        try {
            unregister();
        } catch (Exception ignored) {
        }
        client.close();
    }
}
```

#### 步骤 2：服务发现实现

创建 `ServiceDiscovery.java`：

```java
package com.zkdemo.discovery;

import com.fasterxml.jackson.core.type.TypeReference;
import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.cache.PathChildrenCache;
import org.apache.curator.framework.recipes.cache.PathChildrenCacheEvent;
import org.apache.curator.framework.recipes.cache.PathChildrenCacheListener;
import org.apache.curator.retry.ExponentialBackoffRetry;

import java.util.*;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.CopyOnWriteArrayList;

public class ServiceDiscovery implements AutoCloseable {
    private static final String SERVICES_ROOT = "/services";
    private final CuratorFramework client;
    private final ObjectMapper mapper = new ObjectMapper();
    private final Map<String, List<ServiceInstance>> cache = new ConcurrentHashMap<>();
    private final Map<String, PathChildrenCache> watchers = new ConcurrentHashMap<>();
    private final List<ServiceChangeListener> listeners = new CopyOnWriteArrayList<>();

    public interface ServiceChangeListener {
        void onChange(String serviceName, List<ServiceInstance> instances);
    }

    public ServiceDiscovery(String connectString) {
        client = CuratorFrameworkFactory.builder()
                .connectString(connectString)
                .retryPolicy(new ExponentialBackoffRetry(1000, 3))
                .build();
        client.start();
    }

    // 监听某个服务的变化
    public void watchService(String serviceName) throws Exception {
        String servicePath = SERVICES_ROOT + "/" + serviceName;

        // 确保服务路径存在
        if (client.checkExists().forPath(SERVICES_ROOT) == null) {
            client.create().forPath(SERVICES_ROOT);
        }
        if (client.checkExists().forPath(servicePath) == null) {
            client.create().forPath(servicePath);
        }

        // 使用 PathChildrenCache 监听子节点变化
        PathChildrenCache cache = new PathChildrenCache(client, servicePath, true);
        cache.getListenable().addListener((curator, event) -> {
            if (event.getType() == PathChildrenCacheEvent.Type.CHILD_ADDED
                    || event.getType() == PathChildrenCacheEvent.Type.CHILD_REMOVED
                    || event.getType() == PathChildrenCacheEvent.Type.CHILD_UPDATED) {
                refreshCache(serviceName);
            }
        });
        cache.start();
        watchers.put(serviceName, cache);

        // 初始化缓存
        refreshCache(serviceName);
    }

    private void refreshCache(String serviceName) {
        try {
            String servicePath = SERVICES_ROOT + "/" + serviceName;
            var children = client.getChildren().forPath(servicePath);
            List<ServiceInstance> instances = new ArrayList<>();

            for (String child : children) {
                String instancePath = servicePath + "/" + child;
                byte[] data = client.getData().forPath(instancePath);
                Map<String, Object> metadata = mapper.readValue(data,
                        new TypeReference<Map<String, Object>>() {});
                instances.add(new ServiceInstance(child, metadata));
            }

            cache.put(serviceName, instances);

            // 通知所有监听器
            for (ServiceChangeListener listener : listeners) {
                listener.onChange(serviceName, instances);
            }

            System.out.println("服务 " + serviceName + " 实例更新: " + instances.size()
                    + " 个实例");
            instances.forEach(inst ->
                    System.out.println("  - " + inst.getId() + " : " + inst.getMetadata()));
        } catch (Exception e) {
            System.err.println("刷新服务缓存失败: " + e.getMessage());
        }
    }

    // 获取服务的所有实例（从本地缓存读取）
    public List<ServiceInstance> getInstances(String serviceName) {
        return cache.getOrDefault(serviceName, List.of());
    }

    // 负载均衡：随机选择一个实例
    public ServiceInstance selectOne(String serviceName) {
        List<ServiceInstance> instances = getInstances(serviceName);
        if (instances.isEmpty()) return null;
        return instances.get(new Random().nextInt(instances.size()));
    }

    // 负载均衡：加权轮询
    public ServiceInstance selectByWeight(String serviceName) {
        List<ServiceInstance> instances = getInstances(serviceName);
        if (instances.isEmpty()) return null;

        int totalWeight = 0;
        for (ServiceInstance inst : instances) {
            Object weight = inst.getMetadata().get("weight");
            totalWeight += (weight instanceof Number) ? ((Number) weight).intValue() : 1;
        }

        int random = new Random().nextInt(totalWeight);
        int cumulative = 0;
        for (ServiceInstance inst : instances) {
            Object weight = inst.getMetadata().get("weight");
            cumulative += (weight instanceof Number) ? ((Number) weight).intValue() : 1;
            if (random < cumulative) {
                return inst;
            }
        }
        return instances.get(instances.size() - 1);
    }

    public void addListener(ServiceChangeListener listener) {
        listeners.add(listener);
    }

    @Override
    public void close() {
        watchers.values().forEach(PathChildrenCache::close);
        client.close();
    }

    public record ServiceInstance(String id, Map<String, Object> metadata) {
        public String getAddress() {
            return metadata.get("ip") + ":" + metadata.get("port");
        }
    }
}
```

#### 步骤 3：集成 Spring Boot

创建 `ZkServiceRegistryConfig.java`：

```java
package com.zkdemo.discovery;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.autoconfigure.condition.ConditionalOnProperty;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import javax.annotation.PostConstruct;
import javax.annotation.PreDestroy;
import java.net.InetAddress;
import java.util.Map;

@Configuration
public class ZkServiceRegistryConfig {
    @Value("${zookeeper.connect-string:127.0.0.1:2181}")
    private String connectString;

    @Value("${spring.application.name:unknown}")
    private String serviceName;

    @Value("${server.port:8080}")
    private int serverPort;

    private ServiceRegistry registry;

    @Bean
    public ServiceDiscovery serviceDiscovery() {
        return new ServiceDiscovery(connectString);
    }

    @PostConstruct
    public void register() throws Exception {
        registry = new ServiceRegistry(connectString);
        String host = InetAddress.getLocalHost().getHostAddress();
        Map<String, Object> metadata = Map.of(
                "ip", host,
                "port", serverPort,
                "weight", 10,
                "group", "default",
                "version", "1.0.0"
        );
        registry.register(serviceName, host + ":" + serverPort, metadata);
    }

    @PreDestroy
    public void unregister() throws Exception {
        if (registry != null) {
            registry.close();
        }
    }
}
```

创建 `RestTemplateConfig.java`（使用服务发现调用其他服务）：

```java
package com.zkdemo.discovery;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.client.RestTemplate;

import java.util.List;

@Configuration
public class RestTemplateConfig {
    private final ServiceDiscovery discovery;

    public RestTemplateConfig(ServiceDiscovery discovery) {
        this.discovery = discovery;
    }

    @Bean
    public RestTemplate restTemplate() {
        return new RestTemplate();
    }

    // 演示：调用 order-service
    public String callOrderService(String path) {
        ServiceDiscovery.ServiceInstance instance = discovery.selectByWeight("order-service");
        if (instance == null) {
            throw new RuntimeException("没有可用的 order-service 实例");
        }
        String url = "http://" + instance.getAddress() + path;
        return restTemplate().getForObject(url, String.class);
    }
}
```

#### 步骤 4：完整演示

创建 `ServiceDiscoveryDemo.java`：

```java
package com.zkdemo.discovery;

import java.util.Map;
import java.util.concurrent.CountDownLatch;

public class ServiceDiscoveryDemo {
    public static void main(String[] args) throws Exception {
        System.out.println("=== 服务注册与发现演示 ===\n");

        // 创建 3 个订单服务实例（模拟启动）
        System.out.println("1. 启动服务并注册...");
        ServiceRegistry order1 = new ServiceRegistry("127.0.0.1:2181");
        order1.register("order-service", "192.168.1.1", 8080);

        ServiceRegistry order2 = new ServiceRegistry("127.0.0.1:2181");
        order2.register("order-service", "192.168.1.2", 8080);

        ServiceRegistry order3 = new ServiceRegistry("127.0.0.1:2181");
        order3.register("order-service", "192.168.1.3", 8080);

        System.out.println("\n2. 启动用户服务并发现订单服务...");
        ServiceDiscovery discovery = new ServiceDiscovery("127.0.0.1:2181");
        discovery.addListener((serviceName, instances) -> {
            System.out.println("\n[通知] " + serviceName + " 实例变化:");
            for (var inst : instances) {
                System.out.println("  - " + inst.getAddress());
            }
        });

        // 监听订单服务
        discovery.watchService("order-service");

        Thread.sleep(1000);
        System.out.println("\n当前订单服务实例:");
        List<ServiceDiscovery.ServiceInstance> instances =
                discovery.getInstances("order-service");
        for (var inst : instances) {
            System.out.println("  → " + inst.getAddress() + " [weight="
                    + inst.getMetadata().get("weight") + "]");
        }

        // 测试负载均衡
        System.out.println("\n3. 负载均衡测试（加权轮询）:");
        for (int i = 0; i < 5; i++) {
            var instance = discovery.selectByWeight("order-service");
            System.out.println("  第 " + (i + 1) + " 次调用 → " + instance.getAddress());
        }

        // 模拟实例下线
        System.out.println("\n4. 模拟实例下线...");
        order1.close();
        Thread.sleep(2000);

        System.out.println("\n下线后剩余实例:");
        instances = discovery.getInstances("order-service");
        for (var inst : instances) {
            System.out.println("  → " + inst.getAddress());
        }

        // 清理
        order2.close();
        order3.close();
        discovery.close();
    }
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 服务注册后找不到 | namespace 导致路径偏差 | 检查 Curator 的 namespace 配置 |
| 实例下线后未及时感知 | PathChildrenCache 刷新延迟 | 临时节点自动删除后触发 |
| 重复注册 | 实例重启后旧节点未清理 | 使用 `creatingParentsIfNeeded` + 自动清理 |

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 实时性 | Watcher 毫秒级通知 | 大集群下 Watcher 风暴 |
| 可靠性 | 临时节点自动感知宕机 | ZK 本身需要维护 |
| 负载均衡 | 客户端灵活选择策略 | 服务端无法集中控制 |
| 数据量 | 存储的元数据可自定义 | 1MB 限制，不适合存大数据 |

### ZooKeeper vs Eureka vs Nacos vs Consul

| 特性 | ZooKeeper | Eureka | Nacos | Consul |
|------|-----------|--------|-------|--------|
| CAP | CP | AP | CP+AP | CP |
| 健康检查 | 长连接（Session） | 心跳 | 心跳+TCP | 多种 |
| KV 存储 | 有（ZNode） | 无 | 有 | 有 |
| 控制台 | 弱（四字命令） | 有 | 有（强大） | 有 |
| 一致性 | 强一致 | 最终一致 | 可选 | Raft |
| 适用规模 | 中小 | 中 | 大 | 大 |

### 适用场景

- 中小规模微服务架构（< 50 个服务）
- 对一致性要求高于可用性的场景
- 团队已有 ZooKeeper 运维经验

**不适用场景**：
- 大规模微服务（> 200 个服务，推荐 Nacos/Consul）
- 需要功能完善的控制台

### 注意事项

- 服务注册使用临时节点，会话过期后自动注销
- 客户端需要缓存服务实例列表（本地缓存），避免每次调用都查询 ZooKeeper
- 服务发现应该使用 Curator 的 PathChildrenCache（自动 Watcher 管理）

### 常见踩坑经验

**故障 1：服务下线后调用方仍然调用已下线的实例**

现象：停止一个服务实例后，其他服务仍然调用该实例，超时失败。

根因：PathChildrenCache 收到子节点删除通知时，调用方本地的负载均衡缓存还没有更新。从通知到更新本地缓存有时间差。

解决方案：在负载均衡选择器中加入"健康检测"——调用前检查实例是否在最新缓存中。

**故障 2：注册中心 ZooKeeper 宕机后，调用方无法获取服务列表**

现象：ZooKeeper 集群故障（如网络分区），调用方无法获取服务实例列表，服务调用全部失败。

根因：服务发现依赖 ZooKeeper，ZooKeeper 不可用时服务发现能力丧失。

解决方案：本地缓存兜底。即使 ZooKeeper 不可用，调用方也应该使用本地缓存的（可能过时的）服务列表继续提供服务。

### 思考题

1. 服务注册与发现中，临时节点自动删除的延迟是多少？一个服务实例异常断连后，从断连到其他服务感知到该实例下线，最快需要多长时间？受哪些因素影响？
2. ZooKeeper 的服务发现方案中，调用方通过 Watcher 感知服务变化。如果 100 个调用方同时 Watcher 同一个服务，服务端会同时推送 100 个通知，造成"Watcher 风暴"。如何设计合理的方案来缓解这个问题？

### 推广计划提示

- **开发**：集成服务发现 SDK 后，调用方通过 `discovery.selectOne("order-service")` 获取实例，不需要硬编码 IP
- **运维**：通过 ZK 四字命令 `ls /services/` 查看当前所有注册的服务
- **架构师**：ZooKeeper 的服务发现方案适用于中小规模，大规模微服务推荐 Nacos 或 Consul
