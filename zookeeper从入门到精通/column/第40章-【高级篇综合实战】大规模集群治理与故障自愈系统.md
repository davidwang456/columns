# 第40章：【高级篇综合实战】大规模集群治理与故障自愈系统

## 1. 项目背景

### 业务场景

一家金融科技公司管理着 100+ 个 ZooKeeper 节点分布在全球 8 个数据中心，用于支持支付、风控、用户认证等核心服务。集群规模大、分布广，运维面临巨大挑战：

- **节点故障频繁**：每天都有节点因磁盘满、OOM、网络抖动等原因宕机
- **扩缩容耗时**：新增一个节点需要手动配置、启动、验证，耗时 30 分钟
- **问题定位慢**：上千个客户端连接，出问题时不知道是谁在做什么
- **容灾演练复杂**：每次演练要手动停止节点，耗时且容易出错

目标：构建一个**自动化集群治理与故障自愈系统**。

### 痛点放大

没有自动化集群治理：

- 节点宕机后需要人工介入排查、恢复，平均耗时 30 分钟
- 集群扩容需要 5 个步骤、跨团队协作，扩一个节点需要 2 小时
- 没有统一的集群操作审计，出了问题找不到责任人

**小白**：具体怎么实现"故障自愈"？节点宕机后系统自动做什么？

**大师**：自愈流程：

```
1. 健康监测引擎发现节点 N 宕机
2. 决策引擎判断：
   a. 是否投票节点：是 → 是否需要补充 Quorum
   b. 是否 Observer：是 → 直接替换，不影响 Quorum
3. 执行替换：
   a. 从备用机器池中分配一台新机器
   b. 安装 ZooKeeper，配置为新节点（相同的 myid）
   c. 启动节点，加入集群
   d. 如果是投票节点，Reconfig 替换
4. 验证：
   a. 等节点同步完成
   b. 确认集群状态正常
5. 通知：发送告警到值班群
```

创建 `HealthMonitor.java`：

```java
package com.zkdemo.operator;

import com.fasterxml.jackson.databind.ObjectMapper;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.net.HttpURLConnection;
import java.net.URL;
import java.time.LocalDateTime;
import java.util.*;
import java.util.concurrent.*;

public class HealthMonitor {
    private static final Logger log = LoggerFactory.getLogger(HealthMonitor.class);
    private static final ObjectMapper mapper = new ObjectMapper();

    private final List<ZkNode> nodes;
    private final ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(2);

    public HealthMonitor(List<ZkNode> nodes) {
        this.nodes = new ArrayList<>(nodes);
    }

    public void start(int intervalSeconds) {
        scheduler.scheduleAtFixedRate(() -> {
            for (ZkNode node : nodes) {
                try {
                    checkNode(node);
                } catch (Exception e) {
                    log.error("检查节点失败: {}", node.getHost(), e);
                }
            }
        }, 0, intervalSeconds, TimeUnit.SECONDS);
    }

    private void checkNode(ZkNode node) {
        try {
            // 1. ruok 检查（基本存活）
            String ruok = sendFourLetterWord(node.getHost(), node.getPort(), "ruok");
            boolean alive = "imok".equals(ruok);

            // 2. mntr 获取详细指标
            String mntr = sendFourLetterWord(node.getHost(), node.getPort(), "mntr");
            Map<String, String> metrics = parseMetrics(mntr);

            // 3. 更新节点状态
            node.setAlive(alive);
            node.setLastCheck(LocalDateTime.now());
            node.setRole(metrics.getOrDefault("zk_server_state", "unknown"));
            node.setConnections(Integer.parseInt(
                    metrics.getOrDefault("zk_num_alive_connections", "0")));
            node.setOutstandingRequests(Integer.parseInt(
                    metrics.getOrDefault("zk_outstanding_requests", "0")));
            node.setAvgLatency(Double.parseDouble(
                    metrics.getOrDefault("zk_avg_latency", "0")));
            node.setZnodeCount(Integer.parseInt(
                    metrics.getOrDefault("zk_znode_count", "0")));
            node.setWatchCount(Integer.parseInt(
                    metrics.getOrDefault("zk_watch_count", "0")));

            log.debug("节点 {}:{} 状态: alive={}, role={}, connections={}",
                    node.getHost(), node.getPort(), alive,
                    node.getRole(), node.getConnections());

        } catch (Exception e) {
            node.setAlive(false);
            log.warn("节点 {}:{} 检查失败: {}",
                    node.getHost(), node.getPort(), e.getMessage());
        }
    }

    private String sendFourLetterWord(String host, int port, String cmd) throws Exception {
        Process p = new ProcessBuilder("nc", "-w", "5", host, String.valueOf(port))
                .start();
        p.getOutputStream().write((cmd + "\n").getBytes());
        p.getOutputStream().flush();

        BufferedReader reader = new BufferedReader(
                new InputStreamReader(p.getInputStream()));
        StringBuilder result = new StringBuilder();
        String line;
        while ((line = reader.readLine()) != null) {
            result.append(line).append("\n");
        }
        p.destroy();
        return result.toString().trim();
    }

    private Map<String, String> parseMetrics(String mntr) {
        Map<String, String> metrics = new HashMap<>();
        for (String line : mntr.split("\n")) {
            int tabIndex = line.lastIndexOf('\t');
            if (tabIndex > 0) {
                String key = line.substring(0, tabIndex).trim();
                String value = line.substring(tabIndex + 1).trim();
                metrics.put(key, value);
            }
        }
        return metrics;
    }

    public void stop() {
        scheduler.shutdown();
    }

    public static class ZkNode {
        private String host;
        private int port;
        private int id;
        private boolean alive;
        private LocalDateTime lastCheck;
        private String role;
        private int connections;
        private int outstandingRequests;
        private double avgLatency;
        private int znodeCount;
        private int watchCount;

        public ZkNode(String host, int port, int id) {
            this.host = host;
            this.port = port;
            this.id = id;
        }

        // getters and setters...
        public String getHost() { return host; }
        public int getPort() { return port; }
        public int getId() { return id; }
        public boolean isAlive() { return alive; }
        public void setAlive(boolean alive) { this.alive = alive; }
        public LocalDateTime getLastCheck() { return lastCheck; }
        public void setLastCheck(LocalDateTime lastCheck) { this.lastCheck = lastCheck; }
        public String getRole() { return role; }
        public void setRole(String role) { this.role = role; }
        public int getConnections() { return connections; }
        public void setConnections(int connections) { this.connections = connections; }
        public int getOutstandingRequests() { return outstandingRequests; }
        public void setOutstandingRequests(int outstandingRequests) { this.outstandingRequests = outstandingRequests; }
        public double getAvgLatency() { return avgLatency; }
        public void setAvgLatency(double avgLatency) { this.avgLatency = avgLatency; }
        public int getZnodeCount() { return znodeCount; }
        public void setZnodeCount(int znodeCount) { this.znodeCount = znodeCount; }
        public int getWatchCount() { return watchCount; }
        public void setWatchCount(int watchCount) { this.watchCount = watchCount; }
    }
}
```

#### 步骤 2：自愈决策引擎

创建 `SelfHealingEngine.java`：

```java
package com.zkdemo.operator;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.util.*;
import java.util.concurrent.TimeUnit;
import java.util.stream.Collectors;

public class SelfHealingEngine {
    private static final Logger log = LoggerFactory.getLogger(SelfHealingEngine.class);

    private final HealthMonitor healthMonitor;
    private final List<HealthMonitor.ZkNode> nodes;
    private final NodeLifecycleManager lifecycleManager;
    private final Set<Integer> healingNodes = new HashSet<>();

    public SelfHealingEngine(HealthMonitor healthMonitor,
                              List<HealthMonitor.ZkNode> nodes,
                              NodeLifecycleManager lifecycleManager) {
        this.healthMonitor = healthMonitor;
        this.nodes = nodes;
        this.lifecycleManager = lifecycleManager;
    }

    // 自愈决策逻辑（每 30 秒执行一次）
    public void evaluate() {
        List<HealthMonitor.ZkNode> deadNodes = nodes.stream()
                .filter(n -> !n.isAlive())
                .collect(Collectors.toList());

        for (HealthMonitor.ZkNode deadNode : deadNodes) {
            if (healingNodes.contains(deadNode.getId())) {
                continue; // 已经在处理中
            }

            log.warn("检测到节点故障: {}:{} (ID={}, role={})",
                    deadNode.getHost(), deadNode.getPort(),
                    deadNode.getId(), deadNode.getRole());

            // 决策逻辑
            HealingAction action = decideAction(deadNode);
            executeAction(deadNode, action);
        }
    }

    private HealingAction decideAction(HealthMonitor.ZkNode deadNode) {
        // 1. 检查集群 Quorum 是否受到影响
        long aliveVotingNodes = nodes.stream()
                .filter(n -> n.isAlive() && !"observer".equals(n.getRole()))
                .count();
        long totalVotingNodes = nodes.stream()
                .filter(n -> !"observer".equals(n.getRole()))
                .count();

        boolean quorumAtRisk = aliveVotingNodes <= totalVotingNodes / 2;

        // 2. 判断故障类型
        if ("observer".equals(deadNode.getRole())) {
            // Observer 故障，不需要维护 Quorum
            return HealingAction.REPLACE;
        } else if (quorumAtRisk) {
            // Quorum 受损，立即恢复
            return HealingAction.EMERGENCY_REPLACE;
        } else {
            // 投票节点故障但 Quorum 仍在
            return HealingAction.REPLACE_AND_REBALANCE;
        }
    }

    private void executeAction(HealthMonitor.ZkNode deadNode, HealingAction action) {
        healingNodes.add(deadNode.getId());

        try {
            switch (action) {
                case REPLACE:
                    log.info("执行替换: 节点 {} (ID={})", deadNode.getHost(), deadNode.getId());
                    lifecycleManager.replaceNode(deadNode);
                    break;

                case EMERGENCY_REPLACE:
                    log.warn("执行紧急替换: 节点 {} (ID={}) - Quorum 受损!",
                            deadNode.getHost(), deadNode.getId());
                    lifecycleManager.emergencyReplaceNode(deadNode);
                    break;

                case REPLACE_AND_REBALANCE:
                    log.info("执行替换并重新平衡: 节点 {} (ID={})",
                            deadNode.getHost(), deadNode.getId());
                    lifecycleManager.replaceNode(deadNode);
                    lifecycleManager.rebalanceCluster();
                    break;
            }
        } catch (Exception e) {
            log.error("自愈操作失败: 节点 {} (ID={})",
                    deadNode.getHost(), deadNode.getId(), e);
        } finally {
            healingNodes.remove(deadNode.getId());
        }
    }

    enum HealingAction {
        REPLACE,                    // 替换节点
        EMERGENCY_REPLACE,          // 紧急替换
        REPLACE_AND_REBALANCE       // 替换并重新平衡
    }
}
```

```java
package com.zkdemo.operator;

import org.apache.curator.framework.CuratorFramework;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.util.List;

public class NodeLifecycleManager {
    private static final Logger log = LoggerFactory.getLogger(NodeLifecycleManager.class);

    private final CuratorFramework curatorClient;
    private final String nodeTemplate;
    private final String backupPoolPath;

    public NodeLifecycleManager(CuratorFramework curatorClient,
                                 String nodeTemplate, String backupPoolPath) {
        this.curatorClient = curatorClient;
        this.nodeTemplate = nodeTemplate;
        this.backupPoolPath = backupPoolPath;
    }

    // 替换故障节点
    public void replaceNode(HealthMonitor.ZkNode deadNode) throws Exception {
        // 1. 从备用机器池分配新机器
        String newHost = allocateBackupNode();

        // 2. 安装配置 ZooKeeper
        configureNewNode(newHost, deadNode.getId(), deadNode.getRole());

        // 3. 启动新节点
        startNode(newHost);

        // 4. 验证
        verifyNode(newHost, deadNode.getPort());
    }

    // 紧急替换（Quorum 受损时）
    public void emergencyReplaceNode(HealthMonitor.ZkNode deadNode) throws Exception {
        // 1. 直接分配备用机器（跳过正常资源分配流程）
        String newHost = allocateEmergencyNode();

        // 2. 快速部署（使用预配置镜像）
        deployFromImage(newHost, deadNode.getId(), deadNode.getRole());

        // 3. 启动并验证
        startNode(newHost);
        verifyNode(newHost, deadNode.getPort());

        // 4. 通知值班人员
        notifyOnCall("紧急节点替换完成: ID=" + deadNode.getId()

        // 分析当前 Leader 分布，如果某个机房有多个 Leader，触发 rebalance
        log.info("集群重新平衡 - 检查 Leader 分布");

                "server.%d=%s:2888:3888:%s;2181",
                nodeId, newHost, role);
        curatorClient.setData().forPath("/zookeeper/config", serverConfig.getBytes());
    }

    public void removeNode(int nodeId) throws Exception {
        // 使用 Reconfig API 动态移除
        curatorClient.setData().forPath("/zookeeper/config",

    }
```

#### 步骤 4：延迟监控（修改 Processor 管道）

创建 `LatencyTraceProcessor.java`：

```java
package com.zkdemo.operator;

import org.apache.zookeeper.server.Request;
import org.apache.zookeeper.server.RequestProcessor;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.util.concurrent.ConcurrentHashMap;

/**
 * 延迟追踪 Processor——记录每个请求在各 Processor 的耗时
 */
public class LatencyTraceProcessor implements RequestProcessor {
    private static final Logger log = LoggerFactory.getLogger("LATENCY");
    private final RequestProcessor nextProcessor;
    private final ConcurrentHashMap<Long, Long> startTimes = new ConcurrentHashMap<>();

    public LatencyTraceProcessor(RequestProcessor nextProcessor) {
        this.nextProcessor = nextProcessor;
    }

    @Override
    public void processRequest(Request request) throws RequestProcessorException {
        long startTime = System.nanoTime();

        // 传递给下一个 Processor
        nextProcessor.processRequest(request);

        // 记录总耗时
        long elapsed = System.nanoTime() - startTime;
        long zxid = request.getHdr() != null ? request.getHdr().getZxid() : 0;

        if (elapsed > 1_000_000) { // 超过 1ms 记录
            log.warn("SLOW_REQUEST: zxid=0x{}, type={}, elapsed={}μs",
                    Long.toHexString(zxid),
                    request.type,
                    elapsed / 1000);
        }

        // 上报 Prometheus（示例）
        // latencyHistogram.observe(elapsed / 1_000_000.0);
    }

    @Override
    public void shutdown() {
        nextProcessor.shutdown();
    }
}
```

```java
package com.zkdemo.operator;

import org.springframework.web.bind.annotation.*;

import java.util.*;
import java.util.stream.Collectors;

@RestController
@RequestMapping("/api/operator")
public class OperatorController {

    private final HealthMonitor healthMonitor;
    private final SelfHealingEngine healingEngine;
    private final NodeLifecycleManager lifecycleManager;

    public OperatorController(HealthMonitor healthMonitor,
                               SelfHealingEngine healingEngine,
                               NodeLifecycleManager lifecycleManager) {
        this.healthMonitor = healthMonitor;
        this.healingEngine = healingEngine;
        this.lifecycleManager = lifecycleManager;
    }

    // 集群拓扑
    @GetMapping("/topology")
    public Map<String, Object> getTopology() {
        Map<String, Object> result = new HashMap<>();
        result.put("totalNodes", 100);
        result.put("aliveNodes", 96);
        result.put("deadNodes", 4);
        result.put("leaders", 8);
        result.put("followers", 60);
        result.put("observers", 32);
        return result;
    }

    // 节点详情
    @GetMapping("/nodes")
    public List<Map<String, Object>> getNodes() {
        List<Map<String, Object>> nodeList = new ArrayList<>();
        // 实际从 HealthMonitor 获取
        for (int i = 1; i <= 100; i++) {
            Map<String, Object> node = new HashMap<>();
            node.put("id", i);
            node.put("host", "10.0.0." + i);
            node.put("port", 2181);
            node.put("role", i <= 60 ? "follower" : i <= 68 ? "leader" : "observer");
            node.put("alive", i > 4);
            node.put("connections", new Random().nextInt(500));
            node.put("latency", new Random().nextInt(50));
            node.put("outstanding", new Random().nextInt(10));
            node.put("znodes", 150000 + new Random().nextInt(50000));
            nodeList.add(node);
        }
        return nodeList;
    }

    // 操作审计
    @GetMapping("/audit")
    public Map<String, Object> getAuditLog(@RequestParam(defaultValue = "50") int limit) {
        Map<String, Object> result = new HashMap<>();
        List<Map<String, String>> logs = new ArrayList<>();

        for (int i = 0; i < limit; i++) {
            Map<String, String> entry = new HashMap<>();
            entry.put("time", "2025-03-14 10:00:" + String.format("%02d", i));
            entry.put("operator", "admin");
            entry.put("action", i % 2 == 0 ? "RECONFIG_ADD" : "RECONFIG_REMOVE");
            entry.put("target", "node-" + (i + 1));
            entry.put("result", i % 5 == 0 ? "FAILED" : "SUCCESS");
            logs.add(entry);
        }

        result.put("logs", logs);
        result.put("total", 5000);
        return result;
    }

    // 一键容灾演练
    @PostMapping("/drill/start")
    public String startDisasterRecoveryDrill(@RequestBody DrillConfig config) {
        System.out.println("开始容灾演练: " + config);
        // 自动按配置停止指定数量的节点
        return "容灾演练已开始，停止 " + config.getNodeCount() + " 个节点";
    }

    @PostMapping("/drill/stop")
    public String stopDisasterRecoveryDrill() {
        // 恢复所有节点
        return "容灾演练已停止，节点已恢复";
    }

    public static class DrillConfig {
        private int nodeCount;
        private String[] nodeIds;
        private int durationSeconds;

        public int getNodeCount() { return nodeCount; }
        public void setNodeCount(int nodeCount) { this.nodeCount = nodeCount; }
        public String[] getNodeIds() { return nodeIds; }
        public void setNodeIds(String[] nodeIds) { this.nodeIds = nodeIds; }
        public int getDurationSeconds() { return durationSeconds; }
        public void setDurationSeconds(int durationSeconds) { this.durationSeconds = durationSeconds; }
    }
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 自愈操作太快导致频繁替换 | 网络抖动导致误判 | 增加"连续 N 次检查失败"才触发的条件 |
| Reconfig 并发冲突 | 多个自愈操作同时执行 | 加分布式锁，串行执行 |
| 备用机器池耗尽 | 故障节点过多 | 设置最大自愈数量，超过则人工介入 |

### 验收标准

### 思考题

1. 自愈系统在替换故障节点时，需要从备用池分配机器并部署 ZooKeeper。如果所有备用机器都用完了（同时有多个节点故障），自愈系统应该怎么做？限制自愈数量还是从其他集群借用？
2. 延迟监控通过在每个节点上插入 `LatencyTraceProcessor` 来实现。但如果所有节点都开启这个 Processor，会对 ZooKeeper 的性能产生多大影响？如何设计"采样"策略来降低性能开销？

### 推广计划提示

- **开发**：ZooKeeper Operator 是治理平台的核心，建议持续迭代
- **运维**：容灾演练功能可以定期执行，检验集群的韧性
- **架构师**：大规模集群治理的核心是"自动化"——减少人工干预，提升恢复速度
