# 第22章：分布式锁进阶——读写锁、信号量与多锁管理

## 1. 项目背景

### 业务场景

第 21 章中实现了互斥锁——同一时刻只有一个客户端可以执行临界区代码。但现实世界中的锁需求更复杂：

- **读写锁**：多个读操作可以并行执行，但写操作必须互斥。适合"读多写少"场景，比如配置读取 + 更新
- **信号量**：允许多个客户端并发访问，但限制最大并发数。适合资源池（如数据库连接池）
- **多锁**：一次需要持有多把锁才能执行。适合跨服务协调

### 痛点放大

只用互斥锁解决不了所有问题：

- **读多写少**：互斥锁让读操作也互相阻塞，读吞吐量降低 90%
- **连接池限流**：需要限制同时访问数据库连接的实例数，互斥锁（1 个）不够用
- **跨服务操作**：转账需要同时锁定"账户 A"和"账户 B"，单一锁不行

## 2. 项目设计

### 剧本式交锋对话

**场景**：小白的电商系统遇到了读写锁的需求——库存扣减和库存查询。

**小白**：我用互斥锁实现了库存扣减，但压测发现读库存的 QPS 上不去。因为读也需要获取互斥锁，导致读读互相阻塞。

**大师**：你需要的不是互斥锁，是**读写锁**。读写锁的规则：

```
读锁（共享锁）：多个客户端可以同时获取读锁
写锁（排他锁）：写锁和任何其他锁互斥

兼容矩阵：
       读锁    写锁
读锁     ✓      ✗
写锁     ✗      ✗
```

**小胖**：这个用 ZooKeeper 怎么实现？

**大师**：核心思路还是临时顺序节点，但在节点路径中标记"读"还是"写"：

```
读写锁的原理：

锁路径：/rw-locks/product-123

读锁节点：/rw-locks/product-123/read-0000000001
读锁节点：/rw-locks/product-123/read-0000000002
写锁节点：/rw-locks/product-123/write-0000000003
读锁节点：/rw-locks/product-123/read-0000000004

获取规则：

读锁获取条件：路径中所有序号小于自己的节点都是"读锁"
  → 读锁可以并发获取

写锁获取条件：自己的序号最小（排在最前面）
  → 写锁必须等待前面所有锁释放
```

> **技术映射**：读写锁 = 图书馆的阅览室，读锁 = 进入阅览室看书（可以多人），写锁 = 书架整理员（里尔图书整理时任何人都不能进入）

**小白**：那分布式信号量呢？比如限制同时最多 5 个实例访问数据库。

**大师**：信号量的实现更直接——**租约（Lease）模式**：

```
信号量原理：

初始化：创建 5 个租约节点 /semaphore/db-conn/lease-0000000001~5

获取信号量：删除一个租约节点（删除成功 = 获得一个许可）
释放信号量：重新创建一个租约节点

如果所有 5 个租约都被删除了 → 资源已满 → 等待
等到某实例释放租约（重新创建节点）→ Watcher 触发 → 获取
```

**小胖**：那多锁呢？比如转账需要同时锁 A 账户和 B 账户。

**大师**：多锁的挑战是**死锁预防**：

```java
// 错误示例：先锁 A 再锁 B（可能死锁）
lock(A); // 线程 1 锁了 A
lock(B); // 线程 2 锁了 B
// 现在线程 1 在等 B，线程 2 在等 A → 死锁！

// 正确方案：全局有序加锁
// 对锁路径排序，总是按相同顺序加锁
lock(min(A, B)); // 先锁较小 ID 的
lock(max(A, B)); // 再锁较大 ID 的
```

### 分步实现

#### 步骤 1：读写锁实现

创建 `ReadWriteLock.java`：

```java
package com.zkdemo.lock;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.Collections;
import java.util.List;
import java.util.concurrent.CountDownLatch;

public class ReadWriteLock {
    private static final String LOCK_ROOT = "/rw-locks";
    private final ZooKeeper zk;
    private final String lockPath;

    public ReadWriteLock(ZooKeeper zk, String lockName) {
        this.zk = zk;
        this.lockPath = LOCK_ROOT + "/" + lockName;
    }

    public class ReadLock implements AutoCloseable {
        private String myNode;
        private boolean acquired = false;

        public void acquire() throws Exception {
            ensureParentExists();

            myNode = zk.create(lockPath + "/read-",
                    "read".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE,
                    CreateMode.EPHEMERAL_SEQUENTIAL);

            if (tryAcquire()) {
                acquired = true;
                return;
            }

            // 等待直到获得锁
            waitForLock();
            acquired = true;
        }

        private boolean tryAcquire() throws Exception {
            List<String> children = zk.getChildren(lockPath, false);
            Collections.sort(children);

            String myName = myNode.substring(myNode.lastIndexOf("/") + 1);
            int myIndex = children.indexOf(myName);

            // 检查前面所有节点：如果前面有写锁，不能获取读锁
            for (int i = 0; i < myIndex; i++) {
                if (children.get(i).startsWith("write-")) {
                    return false; // 前面有写锁
                }
            }
            return true; // 前面没有写锁，可以获取读锁
        }

        private void waitForLock() throws Exception {
            while (!tryAcquire()) {
                List<String> children = zk.getChildren(lockPath, false);
                Collections.sort(children);

                String myName = myNode.substring(myNode.lastIndexOf("/") + 1);
                int myIndex = children.indexOf(myName);

                // 找到最后一个写锁（也是离我最近的那个写锁）
                String watchPath = null;
                for (int i = myIndex - 1; i >= 0; i--) {
                    if (children.get(i).startsWith("write-")) {
                        watchPath = lockPath + "/" + children.get(i);
                        break;
                    }
                }

                if (watchPath == null) break; // 没有写锁，重试

                CountDownLatch latch = new CountDownLatch(1);
                Stat stat = zk.exists(watchPath, event -> {
                    if (event.getType() == Watcher.Event.EventType.NodeDeleted) {
                        latch.countDown();
                    }
                });

                if (stat != null) {
                    latch.await();
                }
            }
        }

        @Override
        public void close() throws Exception {
            if (acquired && myNode != null) {
                zk.delete(myNode, -1);
                acquired = false;
            }
        }
    }

    public class WriteLock implements AutoCloseable {
        private String myNode;
        private boolean acquired = false;

        public void acquire() throws Exception {
            ensureParentExists();

            myNode = zk.create(lockPath + "/write-",
                    "write".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE,
                    CreateMode.EPHEMERAL_SEQUENTIAL);

            if (tryAcquire()) {
                acquired = true;
                return;
            }

            waitForLock();
            acquired = true;
        }

        private boolean tryAcquire() throws Exception {
            List<String> children = zk.getChildren(lockPath, false);
            Collections.sort(children);

            String myName = myNode.substring(myNode.lastIndexOf("/") + 1);
            int myIndex = children.indexOf(myName);

            // 写锁要求：序号最小
            return myIndex == 0;
        }

        private void waitForLock() throws Exception {
            while (!tryAcquire()) {
                List<String> children = zk.getChildren(lockPath, false);
                Collections.sort(children);

                String myName = myNode.substring(myNode.lastIndexOf("/") + 1);
                int myIndex = children.indexOf(myName);

                // 监听前一个节点
                String watchPath = lockPath + "/" + children.get(myIndex - 1);

                CountDownLatch latch = new CountDownLatch(1);
                Stat stat = zk.exists(watchPath, event -> {
                    if (event.getType() == Watcher.Event.EventType.NodeDeleted) {
                        latch.countDown();
                    }
                });

                if (stat != null) {
                    latch.await();
                }
            }
        }

        @Override
        public void close() throws Exception {
            if (acquired && myNode != null) {
                zk.delete(myNode, -1);
                acquired = false;
            }
        }
    }

    public ReadLock readLock() {
        return new ReadLock();
    }

    public WriteLock writeLock() {
        return new WriteLock();
    }

    private void ensureParentExists() throws Exception {
        Stat stat = zk.exists(LOCK_ROOT, false);
        if (stat == null) {
            try {
                zk.create(LOCK_ROOT, "".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
                zk.create(lockPath, "".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
            } catch (KeeperException.NodeExistsException ignored) {
            }
        }
    }
}
```

#### 步骤 2：读写锁测试——秒杀库存场景

创建 `InventoryService.java`：

```java
package com.zkdemo.lock;

import org.apache.zookeeper.Watcher;
import org.apache.zookeeper.ZooKeeper;

import java.util.concurrent.CountDownLatch;

public class InventoryService {
    private final ReadWriteLock rwLock;
    private int stock; // 本地模拟库存

    public InventoryService(ZooKeeper zk, String productId, int initialStock) {
        this.rwLock = new ReadWriteLock(zk, "product-" + productId);
        this.stock = initialStock;
    }

    // 读库存（多个客户端可以并发读）
    public int getStock() throws Exception {
        try (var ignored = rwLock.readLock()) {
            rwLock.readLock().acquire();
            // 模拟读取延迟
            Thread.sleep(50);
            System.out.println(Thread.currentThread().getName()
                    + " 读库存: " + stock);
            return stock;
        }
    }

    // 扣减库存（互斥写）
    public boolean decreaseStock(int amount) throws Exception {
        try (var ignored = rwLock.writeLock()) {
            rwLock.writeLock().acquire();
            // 模拟写入延迟
            Thread.sleep(100);
            if (stock >= amount) {
                stock -= amount;
                System.out.println(Thread.currentThread().getName()
                        + " 扣减库存: " + amount + ", 剩余: " + stock);
                return true;
            }
            System.out.println(Thread.currentThread().getName()
                    + " 扣减失败，库存不足: " + stock);
            return false;
        }
    }

    public static void main(String[] args) throws Exception {
        CountDownLatch latch = new CountDownLatch(1);
        ZooKeeper zk = new ZooKeeper("127.0.0.1:2181", 5000, event -> {
            if (event.getState() == Watcher.Event.KeeperState.SyncConnected) {
                latch.countDown();
            }
        });
        latch.await();

        InventoryService service = new InventoryService(zk, "1001", 10);

        System.out.println("=== 读写锁演示：秒杀库存 ===\n");

        // 启动 3 个读线程（持续读库存）
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                try {
                    for (int j = 0; j < 5; j++) {
                        service.getStock();
                        Thread.sleep(200);
                    }
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }, "Reader-" + i).start();
        }

        // 启动 2 个写线程（并发扣减库存）
        for (int i = 0; i < 2; i++) {
            new Thread(() -> {
                try {
                    for (int j = 0; j < 6; j++) {
                        boolean success = service.decreaseStock(1);
                        if (!success) break;
                        Thread.sleep(300);
                    }
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }, "Writer-" + i).start();
        }

        Thread.sleep(10000);
        zk.close();
    }
}
```

#### 步骤 3：信号量实现

创建 `DistributedSemaphore.java`：

```java
package com.zkdemo.lock;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.List;
import java.util.concurrent.CountDownLatch;

public class DistributedSemaphore {
    private static final String SEM_ROOT = "/semaphores";
    private final ZooKeeper zk;
    private final String semPath;
    private final int maxLeases;
    private final String leasePrefix;

    public DistributedSemaphore(ZooKeeper zk, String name, int maxLeases) {
        this.zk = zk;
        this.semPath = SEM_ROOT + "/" + name;
        this.maxLeases = maxLeases;
        this.leasePrefix = semPath + "/lease-";
    }

    public void initialize() throws Exception {
        Stat stat = zk.exists(SEM_ROOT, false);
        if (stat == null) {
            zk.create(SEM_ROOT, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }
        stat = zk.exists(semPath, false);
        if (stat == null) {
            zk.create(semPath, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }

        // 创建初始租约节点
        List<String> children = zk.getChildren(semPath, false);
        if (children.size() < maxLeases) {
            for (int i = children.size(); i < maxLeases; i++) {
                zk.create(leasePrefix, "lease".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT_SEQUENTIAL);
            }
        }
    }

    public void acquire() throws Exception {
        while (true) {
            List<String> children = zk.getChildren(semPath, false);
            if (children.isEmpty()) {
                // 没有可用租约，等待
                CountDownLatch latch = new CountDownLatch(1);
                Stat stat = zk.exists(semPath, event -> {
                    if (event.getType() == Watcher.Event.EventType.NodeChildrenChanged) {
                        latch.countDown();
                    }
                });
                if (stat != null) {
                    latch.await();
                }
                continue;
            }

            // 获取一个租约（删除一个 lease 节点）
            String leaseNode = semPath + "/" + children.get(0);
            try {
                zk.delete(leaseNode, -1);
                System.out.println(Thread.currentThread().getName()
                        + " 获取信号量: " + leaseNode);
                return;
            } catch (KeeperException.NoNodeException e) {
                // 租约已被其他线程获取，重试
            }
        }
    }

    public void release() throws Exception {
        // 释放租约（重新创建一个租约节点）
        String node = zk.create(leasePrefix, "lease".getBytes(),
                ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT_SEQUENTIAL);
        System.out.println(Thread.currentThread().getName()
                + " 释放信号量，创建租约: " + node);
    }
}
```

#### 步骤 4：多锁管理

创建 `MultiLock.java`：

```java
package com.zkdemo.lock;

import java.util.Arrays;
import java.util.Comparator;
import java.util.List;

/**
 * 多锁管理——一次获取多把锁，按排序防止死锁
 */
public class MultiLock {
    private final FairDistributedLock[] locks;

    public MultiLock(FairDistributedLock... locks) {
        // 按锁路径排序，全局一致顺序防止死锁
        this.locks = Arrays.copyOf(locks, locks.length);
        Arrays.sort(this.locks, Comparator.comparing(Object::toString));
    }

    public void lock() throws Exception {
        // 从小到大加锁
        for (FairDistributedLock lock : locks) {
            lock.lock();
        }
    }

    public void unlock() throws Exception {
        // 从大到小释放（反向释放）
        for (int i = locks.length - 1; i >= 0; i--) {
            locks[i].unlock();
        }
    }
}
```

## 4. 项目总结

### 三种锁的对比

| 锁类型 | 并发度 | 适用场景 | 实现原理 |
|-------|--------|---------|---------|
| 互斥锁 | 1 | 写操作互斥 | 临时顺序节点 + 前驱节点 Watcher |
| 读写锁 | 读 N, 写 1 | 读多写少 | 读锁只检查前面有无写锁 |
| 信号量 | 最多 N | 资源池限流 | 租约节点（创建/删除） |

### 适用场景

- **读写锁**：配置读取 + 更新、缓存刷新、库存查询 + 扣减
- **信号量**：数据库连接池、分布式线程池、API 限流
- **多锁**：跨行转账、跨资源操作

### 注意事项

- 读写锁实现中，读锁需要等待前面所有写锁释放（但不需要等读锁）
- 信号量的租约节点是持久节点，需要手动管理生命周期
- 多锁的全局排序是防止死锁的关键

### 常见踩坑经验

**故障 1：读写锁中读锁长时间阻塞**

现象：写锁释放后，读锁仍然无法获取（等待前面的其他读锁释放）。

根因：读写锁实现中，读锁检查的是"前面是否有写锁"，但如果前面的读锁迟迟不释放，当前读锁仍然无法获取（因为需要等前面的节点都处理完）。

解决方案：这是公平 FIFO 的代价。如果需要非公平读写锁（读优先），可以让读锁跳过其他读锁只等待写锁。

**故障 2：信号量租约节点泄漏**

现象：信号量的租约节点数量越来越少，最终没有可用租约。

根因：获取信号量的客户端崩溃后没有执行 `release()` 重新创建租约节点。

解决方案：使用临时节点作为租约（但临时节点上限和 ZooKeeper 性能有关）。

### 思考题

1. 读写锁中，如果一个写锁排在很多读锁后面，它需要等待所有前面的读锁释放。这被称为"写锁饥饿"。如何优化让写锁优先于后续的读锁？
2. 信号量的租约节点是持久节点，如果获取租约的客户端崩溃了，租约不会自动释放。如何改成使用临时节点做信号量？临时节点用于信号量有什么优点和缺点？

### 推广计划提示

- **开发**：读写锁和信号量是 Curator 已经提供的功能，但理解其底层原理有助于在特殊场景下定制
- **测试**：读写锁的测试需要覆盖：多个读锁并发、写锁等待读锁、写锁互斥等场景
- **架构师**：选型时要考虑 ZooKeeper 锁（CP）和 Redis 锁（AP）在一致性上的差异，根据业务需求选择
