# 第37章：数据存储源码——内存数据库、快照与事务日志

## 1. 项目背景

### 业务场景

ZooKeeper 将数据全部存储在内存中——这是高性能的关键。但这引出一个问题：**数据是如何持久化的？**

ZooKeeper 使用两种文件持久化数据：

- **事务日志（Transaction Log）**：记录每次写操作的增量变化
- **快照（Snapshot）**：某一时刻内存数据库的全量副本

启动时，ZooKeeper 加载最新的快照，然后回放快照之后的所有事务日志，重建内存数据库。

### 源码定位

```
DataTree.java           ← 内存数据库（最核心）
DataNode.java           ← ZNode 节点
FileTxnLog.java         ← 事务日志读写
FileSnap.java           ← 快照文件读写
ZKDatabase.java         ← 数据库管理层（整合 DataTree + 文件）
```

## 2. 项目设计

### 剧本式交锋对话

**场景**：小胖想知道 ZooKeeper 的内存数据库是如何组织的。

**大师**：ZooKeeper 的内存数据库核心是 `DataTree`，就像一个巨大的 `Map`：

```java
// 文件位置：org.apache.zookeeper.server.DataTree

public class DataTree {
    // 核心数据结构：路径 → ZNode 节点
    // key = "/services/order-service/192.168.1.1:8080"
    // value = DataNode（包含 data, stat, children, acl）
    private final NodeHashMap nodes;

    // 临时节点索引：sessionId → 该会话的所有临时节点路径
    // 用于会话过期时快速清理
    private final Map<Long, HashSet<String>> ephemerals =
            new ConcurrentHashMap<>();

    // 路径到节点 ID 的映射（用于 Watcher）
    private final Map<String, Long> nodeIds = new ConcurrentHashMap<>();
}
```

**小白**：那 `DataNode` 里存了什么？

```java
// 文件位置：org.apache.zookeeper.server.DataNode

public class DataNode implements Record {
    // 父节点路径
    DataNode parent;

    // 节点存储的数据（最大 1MB）
    byte[] data;

    // 节点的 ACL（访问控制列表）
    private volatile Set<ACL> acl;

    // Stat 持久化状态（czxid, mzxid, version, ...）
    private volatile StatPersisted stat;

    // 子节点（key = 子节点名，value = 子节点）
    private final Set<String> children = new HashSet<>();
}
```

**小胖**：那事务日志和快照的格式呢？

**大师**：**事务日志**（FileTxnLog）是顺序写的，格式如下：

```
File: log.{firstZxid}

Header: 魔数 + 版本 + dbId
  Txn 1: TxnHeader {zxid, time, type} + Record {create/setData/delete}
  Txn 2: TxnHeader + Record
  ...
  Txn N: ...
```

**快照**（FileSnap）是全量序列化的 DataTree：

```
File: snapshot.{lastZxid}

Header: 魔数 + 版本 + dbId
  Serialized DataTree:
    - 所有持久节点的完整数据
    - 所有临时节点的信息（节点路径 + sessionId）
    - 所有会话的信息
  Checksum (CRC32)
```

#### 步骤 1：DataTree 核心操作

```java
// 文件位置：org.apache.zookeeper.server.DataTree
// 创建节点

public void createNode(String path, byte data[], List<ACL> acl,
                        long ephemeralOwner, int parentCVersion,
                        long zxid, long time) throws KeeperException.NoNodeException {

    // 1. 提取父路径和节点名
    int lastSlash = path.lastIndexOf('/');
    String parentName = path.substring(0, lastSlash);
    String childName = path.substring(lastSlash + 1);

    // 2. 创建 DataNode
    StatPersisted stat = new StatPersisted();
    stat.setCzxid(zxid);
    stat.setMzxid(zxid);
    stat.setCtime(time);
    stat.setMtime(time);
    stat.setVersion(0);
    stat.setCversion(0);
    stat.setAversion(0);
    stat.setEphemeralOwner(ephemeralOwner);
    stat.setPzxid(zxid);

    DataNode node = new DataNode(parentName, data, acl, stat);

    // 3. 放入 nodes Map
    nodes.put(path, node);

    // 4. 如果是临时节点，记录到 ephemerals
    if (ephemeralOwner != 0) {
        HashSet<String> set = ephemerals.computeIfAbsent(
                ephemeralOwner, k -> new HashSet<>());
        set.add(path);
    }

    // 5. 更新父节点的子节点列表
    nodes.get(parentName).addChild(childName);
}

// setData——修改节点数据
public Stat setData(String path, byte data[], int version,
                     long zxid, long time) throws KeeperException.NoNodeException {
    DataNode node = nodes.get(path);
    if (node == null) {
        throw new KeeperException.NoNodeException();
    }

    // 版本号校验（乐观锁）
    synchronized (node) {
        node.stat.setVersion(version);
        node.stat.setMzxid(zxid);
        node.stat.setMtime(time);
        node.data = data;
    }

    // 触发 Watcher
    dataWatches.triggerWatch(path, EventType.NodeDataChanged);

    return node.stat;
}

// deleteNode——删除节点
public void deleteNode(String path, long zxid) {
    // 从 nodes 中移除
    DataNode node = nodes.remove(path);

    // 从父节点的 children 中移除
    String parentPath = getParentPath(path);
    DataNode parentNode = nodes.get(parentPath);
    parentNode.removeChild(getChildName(path));

    // 如果是临时节点，从 ephemerals 中移除
    if (node.stat.getEphemeralOwner() != 0) {
        Set<String> set = ephemerals.get(node.stat.getEphemeralOwner());
        if (set != null) {
            set.remove(path);
        }
    }

    // 触发 Watcher
    dataWatches.triggerWatch(path, EventType.NodeDeleted);
    childWatches.triggerWatch(parentPath, EventType.NodeChildrenChanged);
}
```

#### 步骤 2：事务日志写入（FileTxnLog）

```java
// 文件位置：org.apache.zookeeper.server.persistence.FileTxnLog
// 追加事务日志

public synchronized boolean append(TxnHeader hdr, Record txn)
        throws IOException {
    // 1. 如果当前日志文件已到上限，滚动新文件
    if (logStream == null) {
        logStream = new FileOutputStream(logFile);
        // 写入日志文件头
        writeHeader(logStream, hdr);
    }

    // 2. 计算日志文件大小
    logStream.write(util.padBuffer(logFileWrite));

    // 3. 序列化 TxnHeader + Record
    ByteArrayOutputStream baos = new ByteArrayOutputStream();
    BinaryOutputArchive boa = BinaryOutputArchive.getArchive(baos);
    hdr.serialize(boa, "hdr");
    txn.serialize(boa, "txn");

    // 4. 写入文件
    // 格式: [数据长度][序列化数据][Checksum]
    byte[] bytes = baos.toByteArray();
    logStream.writeInt(bytes.length);
    logStream.write(bytes);
    // CRC32 校验
    Checksum crc = new CRC32();
    crc.update(bytes, 0, bytes.length);
    logStream.writeLong(crc.getValue());

    // 5. 强制刷盘（fsync）！
    logStream.flush();
    FileChannel channel = logStream.getChannel();
    channel.force(false);  // fsync！保证数据落盘

    return true;
}
```

#### 步骤 3：快照生成（FileSnap）

```java
// 文件位置：org.apache.zookeeper.server.persistence.FileSnap
// 创建快照

public synchronized void serialize(DataTree dt, Map<Long, Integer> sessions,
                                    File snapFile) throws IOException {
    // 1. 创建输出流
    OutputStream fos = new BufferedOutputStream(new FileOutputStream(snapFile));

    // 2. 写入文件头
    // 魔数 0x4C616E676C65424F (LittleEndianBO)
    fos.write(util.MAGIC_HEADER);
    // 版本号
    fos.write(util.VERSION_HEADER);
    // dbId
    fos.write(Long.toBytes(snapFile.getDbId()));

    // 3. 序列化 DataTree
    BinaryOutputArchive oa = BinaryOutputArchive.getArchive(fos);

    // 4. 序列化所有 ZNode
    dt.serialize(oa, "tree");

    // 5. 序列化会话信息
    oa.writeInt(sessions.size(), "count");
    for (Map.Entry<Long, Integer> entry : sessions.entrySet()) {
        oa.writeLong(entry.getKey(), "id");
        oa.writeInt(entry.getValue(), "timeout");
    }

    // 6. 写入 Checksum
    Checksum crc = new Adler32();
    crc.update(data, 0, data.length);
    oa.writeLong(crc.getValue(), "crc");

    fos.flush();
    fos.close();
}
```

#### 步骤 5：编写独立工具读取事务日志

创建 `TxnLogReader.java`：

```java
package com.zkdemo.source;

import org.apache.jute.BinaryInputArchive;
import org.apache.zookeeper.server.persistence.FileTxnLog;
import org.apache.zookeeper.txn.TxnHeader;

import java.io.File;
import java.io.FileInputStream;
import java.nio.ByteBuffer;
import java.util.zip.Adler32;
import java.util.zip.Checksum;

/**
 * 独立工具——读取 ZooKeeper 事务日志文件
 * 用于审计和数据分析
 */
public class TxnLogReader {

    public static void main(String[] args) throws Exception {
        if (args.length < 1) {
            System.out.println("用法: TxnLogReader <log-file-path>");
            System.out.println("示例: TxnLogReader /var/lib/zookeeper/data/version-2/log.100000001");
            return;
        }

        File logFile = new File(args[0]);
        if (!logFile.exists()) {
            System.err.println("文件不存在: " + logFile);
            return;
        }

        System.out.println("=== ZooKeeper 事务日志阅读器 ===");
        System.out.println("文件: " + logFile);
        System.out.println("大小: " + logFile.length() + " bytes\n");

        FileInputStream fis = new FileInputStream(logFile);
        BinaryInputArchive ia = BinaryInputArchive.getArchive(fis);

        // 读取文件头
        int magic = ia.readInt("magic");
        int version = ia.readInt("version");
        long dbId = ia.readLong("dbId");
        System.out.printf("魔数: 0x%08X, 版本: %d, DbId: %d%n%n", magic, version, dbId);

        int txnCount = 0;
        long totalDataSize = 0;

        try {
            while (true) {
                // 读取事务头
                TxnHeader hdr = new TxnHeader();
                hdr.deserialize(ia, "hdr");

                // 读取事务体
                int dataLength = ia.readInt("dataLength");
                byte[] data = new byte[dataLength];
                ia.readInt(dataLength, "data", data);

                // 读取 CRC 校验
                long storedCrc = ia.readLong("crc32");

                // 校验 CRC
                Checksum crc = new Adler32();
                crc.update(data, 0, data.length);
                long computedCrc = crc.getValue();

                txnCount++;
                totalDataSize += dataLength;

                System.out.printf("[%d] zxid=0x%x, type=%d, time=%tF %<tT%n",
                        txnCount, hdr.getZxid(), hdr.getType(),
                        hdr.getTime());
                System.out.printf("    数据长度: %d bytes, CRC校验: %s%n",
                        dataLength, storedCrc == computedCrc ? "PASS" : "FAIL");
                System.out.printf("    数据内容: %s%n",
                        new String(data).replaceAll("[\0]", ""));

                if (txnCount >= 100) {
                    System.out.println("\n... (达到显示上限，共 " + txnCount + " 条)");
                    break;
                }
            }
        } catch (Exception e) {
            // 读取完毕
        }

        System.out.printf("\n总计: %d 条事务, 数据量 %d bytes%n",
                txnCount, totalDataSize);
        fis.close();
    }
}
```

#### 步骤 6：数据一致性校验

```java
// 文件位置：org.apache.zookeeper.server.persistence.FileSnap
// 加载快照时的校验

public long deserialize(DataTree dt, Map<Long, Integer> sessions,
                         PlayBackListener listener) throws IOException {
    // 1. 找到最新的快照
    File snapFile = findMostRecentSnapshot();

    // 2. 读取并校验 Checksum
    CheckedInputStream crcIn = new CheckedInputStream(
            new BufferedInputStream(new FileInputStream(snapFile)),
            new Adler32());

    BinaryInputArchive ia = BinaryInputArchive.getArchive(crcIn);

    // 3. 反序列化 DataTree
    dt.deserialize(ia, "tree");

    // 4. 校验 CRC
    long computedCrc = crcIn.getChecksum().getValue();
    long storedCrc = ia.readLong("crc");

    if (computedCrc != storedCrc) {
        throw new IOException("CRC 校验失败！快照已损坏");
    }

    return dt.getLastProcessedZxid();
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 快照加载慢 | DataTree 反序列化耗时 | 增大堆内存、使用 SSD |
| 事务日志积压 | snapCount 太大 | 调小 snapCount |
| CRC 校验失败 | 文件损坏 | 从备份恢复 |

## 4. 项目总结

### 数据存储架构

```
启动时加载:
  Snapshot(snapshot.zxid) → DataTree (全量)
      + replay
  Transaction Logs (log.zxid*) → 增量事务
      = 
  DataTree (完整内存数据库)

运行时写入:
  Write Request → DataTree (内存更新)
      + 
  FileTxnLog.append() (事务日志，fsync)
      +
  周期性: FileSnap.serialize() (快照)
```

### 关键性能点

| 操作 | 瓶颈 | 优化 |
|------|------|------|
| 事务日志追加 | fsync（磁盘 IO） | SSD、独立磁盘 |
| 快照生成 | 全量序列化 + 写入 | 调整 snapCount |
| 启动加载 | 快照反序列化 + 日志回放 | 堆内存、SSD |

### 思考题

1. `DataTree` 中使用 `ConcurrentHashMap<String, DataNode>` 存储所有 ZNode。路径作为 key，查询性能 O(1)。但 `getChildren("/services/order-service")` 需要遍历所有节点吗？如果不是，ZooKeeper 如何高效获取子节点列表？
2. `FileTxnLog.append()` 每次写入都要 fsync。如果修改为每 N 次写才 fsync 一次，性能会提升多少？代价是什么？

### 推广计划提示

- **开发**：DataTree 是理解 ZooKeeper 内存数据库的最好起点
- **运维**：`dataDir` 和 `dataLogDir` 分别放不同磁盘上（事务日志 SSD，快照 HDD 也可以）
- **贡献者**：FileTxnLog 和 FileSnap 是持久化层的核心，修改它们可以改变数据存储格式
