# 第26章：分布式队列与任务调度

## 1. 项目背景

### 业务场景

分布式系统中，不同组件之间需要**异步通信**和**任务编排**：

- **任务队列**：爬虫系统将待抓取的 URL 放入队列，多个 Worker 从队列中获取任务执行
- **优先级队列**：紧急任务优先执行（如支付超时处理）
- **延迟队列**：下单后 30 分钟未支付，自动取消订单
- **任务调度**：复杂任务需要分阶段执行，每个阶段由不同 Worker 处理

ZooKeeper 的顺序节点天然适合实现 FIFO 队列，TTL 节点（3.5+）适合实现延迟队列。

### 痛点放大

不用 ZooKeeper 做队列的替代方案各有缺陷：

- **数据库表做队列**：轮询开销大、高并发下锁竞争激烈
- **Redis List**：RPUSH/LPOP 快但无法做到严格 FIFO 和 ACK 确认
- **RabbitMQ/Kafka**：功能丰富但引入额外组件，小项目太重量级

## 2. 项目设计

### 剧本式交锋对话

**场景**：小胖的爬虫系统需要将 URL 分发给多个 Worker 抓取。

**小胖**：我想用 ZooKeeper 的顺序节点做队列，生产者创建节点，消费者读取节点，先进先出。

**大师**：FIFO 队列的原理很简单——每个任务是一个顺序节点，序号最小的节点最先被消费。

```
分布式 FIFO 队列：

生产者：
  create /queue/task- "url=http://example.com" (PERSISTENT_SEQUENTIAL)
  → 创建节点：/queue/task-0000000001

消费者：
  1. 获取 /queue 的所有子节点，排序
  2. 取序号最小的节点
  3. 读取数据，处理
  4. 删除节点

重复步骤 1-4 不断消费
```

**小白**：但如果消费者在处理任务时挂了，任务不就丢了吗？没有 ACK 机制。

**大师**：好问题。ZooKeeper 队列的可靠性可以通过"两阶段处理"保证：

```
可靠的 ZooKeeper 队列：

1. 消费者获取最小的顺序节点
2. 读取任务数据
3. 在另一个路径创建"处理中"标记（临时节点）:
   /queue/task-0000000001/processing → worker-id
4. 删除原始任务节点（从队列中移除）
5. 执行业务逻辑
6. 如果执行成功 → 删除 processing 标记
7. 如果执行失败 → 重新创建任务节点（放回队列）
8. 如果 Worker 崩溃 → processing 标记自动删除（临时节点）
   恢复程序检查 processing 标记 → 重新入队
```

> **技术映射**：队列 = 医院挂号处，顺序节点 = 挂号单号，处理中标记 = 你在诊室门口排队，ACK = 医生看完病说"好了"

**小胖**：延迟队列呢？如何实现"下单 30 分钟未支付，自动取消"？

**大师**：ZooKeeper 3.5+ 的 TTL 节点正好实现延迟队列：

```java
// 延迟队列——任务节点在 TTL 超时后自动被删除
// 消费者监听节点的删除事件 → 触发取消操作

// 生产者：创建 TTL 节点（30 分钟过期）
CreateMode mode = CreateMode.PERSISTENT_WITH_TTL;
zk.create("/delay-queue/order-12345", orderData, acl, mode, ttlData);

// 消费者：监听 /delay-queue 的子节点删除
zk.getChildren("/delay-queue", watcher);
// 当某个节点因 TTL 被删除 → Watcher 触发 → 执行取消操作
```

## 3. 项目实战

### 环境准备

- ZooKeeper 3.9.x 运行中（TTL 节点需要 `extendedTypesEnabled=true`）
- JDK 11+
- Maven

### 分步实现

#### 步骤 1：FIFO 分布式队列

创建 `DistributedQueue.java`：

```java
package com.zkdemo.queue;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.Collections;
import java.util.List;
import java.util.concurrent.CountDownLatch;

public class DistributedQueue {
    private static final String QUEUE_ROOT = "/queues";
    private final ZooKeeper zk;
    private final String queuePath;
    private final String queuePrefix;

    public DistributedQueue(ZooKeeper zk, String queueName) {
        this.zk = zk;
        this.queuePath = QUEUE_ROOT + "/" + queueName;
        this.queuePrefix = queuePath + "/task-";
    }

    // 初始化队列
    public void init() throws Exception {
        Stat stat = zk.exists(QUEUE_ROOT, false);
        if (stat == null) {
            zk.create(QUEUE_ROOT, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }
        stat = zk.exists(queuePath, false);
        if (stat == null) {
            zk.create(queuePath, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }
    }

    // 生产者：发布任务
    public void publish(byte[] taskData) throws Exception {
        String nodePath = zk.create(queuePrefix, taskData,
                ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT_SEQUENTIAL);
        String taskId = nodePath.substring(nodePath.lastIndexOf("-") + 1);
        System.out.println("[生产者] 发布任务: " + taskId
                + ", 数据: " + new String(taskData));
    }

    // 消费者：消费任务（可靠的——两阶段处理）
    public byte[] consume() throws Exception {
        while (true) {
            List<String> children = zk.getChildren(queuePath, false);
            if (children.isEmpty()) {
                // 队列为空，等待
                System.out.println("[消费者] 队列为空，等待...");
                CountDownLatch latch = new CountDownLatch(1);
                Stat stat = zk.exists(queuePath, event -> {
                    if (event.getType() == Watcher.Event.EventType.NodeChildrenChanged) {
                        latch.countDown();
                    }
                });
                if (stat != null) {
                    latch.await();
                }
                continue;
            }

            // 排序获取最小的节点
            Collections.sort(children);
            String taskName = children.get(0);
            String taskPath = queuePath + "/" + taskName;

            try {
                // 读取任务数据
                byte[] taskData = zk.getData(taskPath, false, null);
                String taskId = taskName.substring(taskName.lastIndexOf("-") + 1);

                // 创建 processing 标记（临时节点，防崩溃）
                String processingPath = taskPath + "/processing";
                zk.create(processingPath, "worker-1".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);

                // 删除原任务节点
                zk.delete(taskPath, -1);

                System.out.println("[消费者] 消费任务: " + taskId
                        + ", 数据: " + new String(taskData));

                return taskData;

            } catch (KeeperException.NoNodeException e) {
                // 节点已被其他消费者消费，继续
                continue;
            }
        }
    }

    // 获取队列大小
    public int size() throws Exception {
        return zk.getChildren(queuePath, false).size();
    }
}
```

#### 步骤 2：优先级队列

创建 `PriorityQueue.java`：

```java
package com.zkdemo.queue;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.*;

public class PriorityQueue {
    private static final String PRIORITY_ROOT = "/priority-queues";
    private final ZooKeeper zk;
    private final String queuePath;

    public PriorityQueue(ZooKeeper zk, String queueName) {
        this.zk = zk;
        this.queuePath = PRIORITY_ROOT + "/" + queueName;
    }

    public void init() throws Exception {
        Stat stat = zk.exists(PRIORITY_ROOT, false);
        if (stat == null) {
            zk.create(PRIORITY_ROOT, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }
        stat = zk.exists(queuePath, false);
        if (stat == null) {
            zk.create(queuePath, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }
    }

    // 按优先级发布：1-最高，5-最低
    public void publish(String priority, byte[] taskData) throws Exception {
        String priorityPath = queuePath + "/p" + priority;
        Stat stat = zk.exists(priorityPath, false);
        if (stat == null) {
            zk.create(priorityPath, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }

        String nodePath = zk.create(priorityPath + "/task-", taskData,
                ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT_SEQUENTIAL);
        String taskId = nodePath.substring(nodePath.lastIndexOf("-") + 1);
        System.out.println("[优先级队列] 发布任务(p" + priority + "): " + taskId);
    }

    // 按优先级消费：先从高优先级队列取
    public byte[] consume() throws Exception {
        for (int p = 1; p <= 5; p++) {
            String priorityPath = queuePath + "/p" + p;
            Stat stat = zk.exists(priorityPath, false);
            if (stat == null) continue;

            List<String> children = zk.getChildren(priorityPath, false);
            if (!children.isEmpty()) {
                Collections.sort(children);
                String taskPath = priorityPath + "/" + children.get(0);
                byte[] data = zk.getData(taskPath, false, null);
                zk.delete(taskPath, -1);
                String taskId = children.get(0).substring(
                        children.get(0).lastIndexOf("-") + 1);
                System.out.println("[消费者] 消费任务(p" + p + "): " + taskId
                        + ", 数据: " + new String(data));
                return data;
            }
        }
        return null;
    }
}
```

#### 步骤 3：分布式爬虫任务调度系统

创建 `DistributedCrawler.java`：

```java
package com.zkdemo.queue;

import org.apache.zookeeper.*;
import org.apache.zookeeper.data.Stat;

import java.util.*;
import java.util.concurrent.CountDownLatch;

/**
 * 分布式爬虫任务调度系统
 * Master：发现 URL 并分片
 * Worker：从队列捞取 URL 爬取
 */
public class DistributedCrawler {
    private static final String CRAWLER_ROOT = "/crawler";
    private final ZooKeeper zk;
    private final String jobName;

    public DistributedCrawler(ZooKeeper zk, String jobName) {
        this.zk = zk;
        this.jobName = CRAWLER_ROOT + "/" + jobName;
    }

    public void init() throws Exception {
        Stat stat = zk.exists(CRAWLER_ROOT, false);
        if (stat == null) {
            zk.create(CRAWLER_ROOT, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }
        stat = zk.exists(jobName, false);
        if (stat == null) {
            zk.create(jobName, "".getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
        }

        // 创建任务目录结构
        for (String dir : new String[]{"pending", "running", "done", "failed"}) {
            String dirPath = jobName + "/" + dir;
            stat = zk.exists(dirPath, false);
            if (stat == null) {
                zk.create(dirPath, "".getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);
            }
        }
    }

    // Master：发现 URL 并添加到待处理队列
    public void discoverUrls(List<String> urls) throws Exception {
        String pendingPath = jobName + "/pending";
        for (String url : urls) {
            String nodePath = zk.create(pendingPath + "/url-",
                    url.getBytes(),
                    ZooDefs.Ids.OPEN_ACL_UNSAFE,
                    CreateMode.PERSISTENT_SEQUENTIAL);
            System.out.println("[Master] 发现 URL: " + url
                    + " → " + nodePath);
        }
    }

    // Worker：从待处理队列获取任务
    public String fetchTask(String workerId) throws Exception {
        String pendingPath = jobName + "/pending";

        while (true) {
            List<String> pendingTasks = zk.getChildren(pendingPath, false);
            if (pendingTasks.isEmpty()) {
                System.out.println("[Worker " + workerId + "] 无待处理任务");
                return null;
            }

            Collections.sort(pendingTasks);
            String taskName = pendingTasks.get(0);
            String taskPath = pendingPath + "/" + taskName;

            try {
                byte[] urlData = zk.getData(taskPath, false, null);
                String url = new String(urlData);

                // 移动到 running 状态
                String runningPath = jobName + "/running/" + taskName;
                zk.create(runningPath, workerId.getBytes(),
                        ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.EPHEMERAL);
                zk.delete(taskPath, -1);

                System.out.println("[Worker " + workerId + "] 获取任务: " + url);
                return url;

            } catch (KeeperException.NoNodeException e) {
                // 其他 Worker 取走了，继续
            }
        }
    }

    // Worker：完成任务
    public void completeTask(String taskName, boolean success) throws Exception {
        String srcPath = jobName + "/running/" + taskName;
        String dstPath = jobName + "/" + (success ? "done" : "failed") + "/" + taskName;

        byte[] data = zk.getData(srcPath, false, null);
        zk.delete(srcPath, -1);
        zk.create(dstPath, data, ZooDefs.Ids.OPEN_ACL_UNSAFE, CreateMode.PERSISTENT);

        System.out.println("任务 " + taskName
                + (success ? " 完成" : " 失败"));
    }

    // Master：查看任务统计
    public void printStats() throws Exception {
        int pending = zk.getChildren(jobName + "/pending", false).size();
        int running = zk.getChildren(jobName + "/running", false).size();
        int done = zk.getChildren(jobName + "/done", false).size();
        int failed = zk.getChildren(jobName + "/failed", false).size();

        System.out.println("\n=== 爬虫任务统计 ===");
        System.out.println("待处理: " + pending);
        System.out.println("进行中: " + running);
        System.out.println("已完成: " + done);
        System.out.println("已失败: " + failed);
        System.out.println("====================\n");
    }

    public static void main(String[] args) throws Exception {
        CountDownLatch latch = new CountDownLatch(1);
        ZooKeeper zk = new ZooKeeper("127.0.0.1:2181", 5000, event -> {
            if (event.getState() == Watcher.Event.KeeperState.SyncConnected) {
                latch.countDown();
            }
        });
        latch.await();

        DistributedCrawler crawler = new DistributedCrawler(zk, "web-crawl");
        crawler.init();

        // Master：发现 URL
        System.out.println("=== Phase 1: 发现 URL ===");
        List<String> urls = Arrays.asList(
                "http://example.com/page1",
                "http://example.com/page2",
                "http://example.com/page3",
                "http://example.com/page4",
                "http://example.com/page5"
        );
        crawler.discoverUrls(urls);

        Thread.sleep(500);

        // Worker：消费任务
        System.out.println("\n=== Phase 2: Worker 处理 ===");
        for (int i = 0; i < 3; i++) {
            int workerId = i + 1;
            new Thread(() -> {
                try {
                    String url = crawler.fetchTask("W" + workerId);
                    if (url != null) {
                        // 模拟爬取
                        Thread.sleep((long) (Math.random() * 2000 + 500));
                        crawler.completeTask("url-", true);
                    }
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }).start();
        }

        Thread.sleep(3000);
        crawler.printStats();

        zk.close();
    }
}
```

创建 `QueueBenchmark.java`：

```java
package com.zkdemo.queue;

import org.apache.zookeeper.Watcher;
import org.apache.zookeeper.ZooKeeper;

import java.util.concurrent.CountDownLatch;
import java.util.concurrent.atomic.AtomicInteger;

public class QueueBenchmark {
    public static void main(String[] args) throws Exception {
        CountDownLatch latch = new CountDownLatch(1);
        ZooKeeper zk = new ZooKeeper("127.0.0.1:2181", 5000, event -> {
            if (event.getState() == Watcher.Event.KeeperState.SyncConnected) {
                latch.countDown();
            }
        });
        latch.await();

        DistributedQueue queue = new DistributedQueue(zk, "benchmark");
        queue.init();

        int taskCount = 500;

        // 生产者
        System.out.println("生产 " + taskCount + " 个任务...");
        long start = System.currentTimeMillis();
        for (int i = 0; i < taskCount; i++) {
            queue.publish(("task-" + i).getBytes());
        }
        long publishTime = System.currentTimeMillis() - start;

        // 消费者
        System.out.println("消费 " + taskCount + " 个任务...");
        start = System.currentTimeMillis();
        for (int i = 0; i < taskCount; i++) {
            queue.consume();
        }
        long consumeTime = System.currentTimeMillis() - start;

        System.out.printf("\n结果: 生产 %d 个任务耗时 %d ms (%d ops/s)%n",
                taskCount, publishTime, taskCount * 1000L / publishTime);
        System.out.printf("结果: 消费 %d 个任务耗时 %d ms (%d ops/s)%n",
                taskCount, consumeTime, taskCount * 1000L / consumeTime);

        zk.close();
    }
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 队列顺序错乱 | 不同线程同时消费未加锁 | 使用 distributed lock 保护消费过程 |
| 任务重复消费 | processing 标记未正确创建 | 使用临时节点保证崩溃后自动清理 |
| ZooKeeper 节点太多 | 大量任务节点未清理 | 消费后及时删除，或设置 autopurge |

## 4. 项目总结

### 队列类型对比

| 特性 | Simple FIFO | 优先级队列 | 延迟队列 (TTL) |
|------|------------|-----------|---------------|
| 排序 | 按创建时间 | 按优先级 + 创建时间 | 按到期时间 |
| 可靠性 | 中 | 中 | 高 |
| 适合场景 | 普通任务队列 | 紧急任务优先 | 超时处理 |
| 复杂度 | 低 | 中 | 中 |

### ZooKeeper vs Kafka / RabbitMQ

| 特性 | ZooKeeper 队列 | Kafka | RabbitMQ |
|------|---------------|-------|----------|
| 吞吐量 | 低（~1000/s） | 极高（百万/s） | 高（~10万/s） |
| 可靠性 | 中 | 高 | 高 |
| 功能性 | 基础 | 完整 | 完整 |
| 运维复杂度 | 低（复用 ZK） | 高 | 中 |

### 适用场景

- **低频任务**：每小时/每天执行一次的定时任务
- **强一致性**：任务状态需要严格追踪
- **轻量级**：不想引入 Kafka/RabbitMQ

### 注意事项

- TTL 节点需要服务端配置 `extendedTypesEnabled=true`
- 大量任务节点（>10万）会占用 ZK 大量内存
- processing 标记使用临时节点防止 Worker 崩溃后任务丢失

### 常见踩坑经验

**故障 1：消费端读取不到新发布的任务**

现象：生产者发布了任务，但消费者一直阻塞等待，获取不到新节点。

根因：Watcher 注册的时机不对。如果先注册 Watcher 再检查队列，如期间生产了任务，Watcher 不会触发。

解决方案：先获取子节点列表，如果为空再注册 Watcher。注册前再检查一次（双重检查）。

**故障 2：TTL 节点到期未删除**

现象：ZooKeeper 中的 TTL 节点超过了设定的 TTL 时间但仍然存在。

根核：TTL 节点的删除依赖于 ZooKeeper 的定时检测机制，不是精确的 TTL + 立即删除。实际上可能存在几秒到几十秒的延迟。

### 思考题

1. 分布式爬虫的任务调度中，如果某个 Worker 爬取任务时挂了，任务会被标记为 running 但永远无法完成。如何设计"任务超时回收"机制来自动处理这个问题？
2. 优先级队列的实现中，如果高优先级队列一直有新任务进入，低优先级队列的任务可能永远得不到处理（"饥饿"问题）。如何设计"老化"机制——任务等待时间越长，优先级动态提升？

### 推广计划提示

- **开发**：ZooKeeper 队列适合低频、一致性要求高的场景；高频场景请使用 Kafka/RabbitMQ
- **运维**：监控 `/queues/` 和 `/crawler/` 路径下的节点数量，异常积压说明消费者处理不过来
- **架构师**：任务调度系统的设计原则——生产快、消费稳、重试有界、状态可追踪
