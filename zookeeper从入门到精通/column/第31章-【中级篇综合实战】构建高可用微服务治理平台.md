# 第31章：【中级篇综合实战】构建高可用微服务治理平台

## 1. 项目背景

### 业务场景

一家电商公司有 80+ 微服务，运行在 200+ 容器实例上。当前的问题是：

- **服务发现不可靠**：上线和下线的服务实例不能及时感知
- **配置管理混乱**：每个服务有自己的配置文件，修改后需要逐个重启实例
- **定时任务冲突**：多个实例同时执行定时任务，导致数据重复处理
- **分布式锁不统一**：每个团队用自己的方式实现分布式锁，有的用数据库，有的用 Redis，问题频出
- **灰度发布困难**：新配置上线缺乏灰度机制，出问题影响全量实例

需要一个统一的**微服务治理平台**，基于 ZooKeeper + Curator + Spring Cloud，提供注册发现、配置管理、任务调度、分布式锁等能力。

### 痛点放大

每次微服务架构的问题排查，都要花大量时间在协调上：

- 服务 A 调用服务 B 超时，是因为服务 B 挂了还是因为负载均衡策略不对？
- 库存服务配置的连接池参数太小，数据库连接不够用，改配置要等 30 分钟部署
- 凌晨的数据清理任务，5 个实例同时执行，数据被重复清理

## 2. 项目设计

### 剧本式交锋对话

**场景**：架构评审会上，小胖提出了微服务治理平台的设计方案。

**小胖**：我们要提供一个"治理 SDK"，所有服务通过 SDK 接入。SDK 提供：服务注册、配置管理、分布式锁、选主、分布式 ID。

**大师**：架构设计如下：

```
┌─────────────────────────────────────────────────────────┐
│                  微服务治理平台架构                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│  │ 服务 A    │  │ 服务 B    │  │ 服务 C    │  │ ...      │ │
│  │ (SDK)     │  │ (SDK)     │  │ (SDK)     │  │ (SDK)     │ │
│  └─────┬────┘  └─────┬────┘  └─────┬────┘  └─────┬────┘ │
│        │              │              │              │      │
│  ┌─────┴──────────────┴──────────────┴──────────────┴───┐ │
│  │              ZooKeeper 集群 (3 节点)                   │ │
│  │  /services/  /config/  /locks/  /leader/  /idgen/    │ │
│  └──────────────────────────┬──────────────────────────┘ │
│                             │                            │
│  ┌──────────────────────────▼──────────────────────────┐ │
│  │              治理平台控制台 (Admin Web)                │ │
│  │  服务拓扑  |  配置管理  |  锁监控  |  任务调度         │ │
│  └─────────────────────────────────────────────────────┘ │
│                                                          │
│  ┌──────────────────────────────────────────────────────┐ │
│  │  Prometheus + Grafana（监控告警）                     │ │
│  └──────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**小白**：SDK 内部的结构是怎样的？要整合这么多功能。

**大师**：SDK 采用**模块化设计**：

```
治理 SDK
├── RegistryModule（服务注册/发现）
│   ├── ServiceRegistry    → 注册当前实例到 /services/{app}/{instanceId}
│   └── ServiceDiscovery   → 监听其他服务的实例变化
├── ConfigModule（配置管理）
│   ├── ConfigCenter        → 读取 + 监听 /config/{app}/{env}/{key}
│   ├── LocalFileCache      → 本地缓存兜底
│   └── GrayRelease         → 灰度发布支持
├── LockModule（分布式锁）
│   ├── ReentrantLock        → 基于 InterProcessMutex
│   ├── ReadWriteLock        → 基于 InterProcessReadWriteLock
│   └── MultiLock            → 多锁管理
├── ElectionModule（选主）
│   ├── LeaderLatchManager   → 持续 Leader 模式
│   └── LeaderSelectorManager → 轮值 Leader 模式
└── IdGenModule（ID 生成）
    └── DistributedIdGenerator → 基于顺序节点或 DistributedAtomicLong
```

### 分步实现

#### 步骤 1：治理 SDK 核心实现

创建 `GovernanceSDK.java`：

```java
package com.zkdemo.governance;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.atomic.AtomicValue;
import org.apache.curator.framework.recipes.atomic.DistributedAtomicLong;
import org.apache.curator.framework.recipes.leader.LeaderLatch;
import org.apache.curator.framework.recipes.leader.LeaderLatchListener;
import org.apache.curator.framework.recipes.leader.LeaderSelector;
import org.apache.curator.framework.recipes.leader.LeaderSelectorListener;
import org.apache.curator.framework.recipes.locks.InterProcessMutex;
import org.apache.curator.framework.recipes.locks.InterProcessReadWriteLock;
import org.apache.curator.framework.state.ConnectionState;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.util.*;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.TimeUnit;

public class GovernanceSDK implements AutoCloseable {
    private static final Logger log = LoggerFactory.getLogger(GovernanceSDK.class);

    private static final String SERVICES_ROOT = "/services";
    private static final String CONFIG_ROOT = "/config";
    private static final String LOCKS_ROOT = "/locks";
    private static final String LEADER_ROOT = "/leader";
    private static final String IDGEN_ROOT = "/idgen";

    private final CuratorFramework client;
    private final String appName;
    private final String env;
    private final String instanceId;

    public GovernanceSDK(String connectString, String appName,
                         String env, String instanceId) {
        this.appName = appName;
        this.env = env;
        this.instanceId = instanceId;

        this.client = CuratorFrameworkFactory.builder()
                .connectString(connectString)
                .sessionTimeoutMs(30000)
                .retryPolicy(new ExponentialBackoffRetry(1000, 3))
                .build();
        client.start();

        log.info("治理 SDK 初始化完成: app={}, env={}, instance={}",
                appName, env, instanceId);
    }

    // ========== 服务注册/发现 ==========

    public void registerService(String host, int port) throws Exception {
        String path = SERVICES_ROOT + "/" + appName + "/" + instanceId;
        Map<String, Object> metadata = new HashMap<>();
        metadata.put("ip", host);
        metadata.put("port", port);
        metadata.put("startTime", System.currentTimeMillis());
        metadata.put("version", getClass().getPackage().getImplementationVersion());

        client.create().creatingParentsIfNeeded()
                .withMode(org.apache.zookeeper.CreateMode.EPHEMERAL)
                .forPath(path, new com.fasterxml.jackson.databind.ObjectMapper()
                        .writeValueAsString(metadata).getBytes());
        log.info("服务注册成功: {}:{}", host, port);
    }

    public List<String> discoverService(String serviceName) throws Exception {
        String path = SERVICES_ROOT + "/" + serviceName;
        if (client.checkExists().forPath(path) == null) {
            return List.of();
        }
        return client.getChildren().forPath(path);
    }

    // ========== 配置管理 ==========

    public String getConfig(String key) throws Exception {
        return getConfig(key, null);
    }

    public String getConfig(String key, String defaultValue) {
        try {
            String path = CONFIG_ROOT + "/" + appName + "/" + env + "/" + key;
            if (client.checkExists().forPath(path) != null) {
                byte[] data = client.getData().forPath(path);
                return new String(data);
            }
        } catch (Exception e) {
            log.warn("读取配置失败: {}", key, e);
        }
        return defaultValue;
    }

    public void setConfig(String key, String value) throws Exception {
        String path = CONFIG_ROOT + "/" + appName + "/" + env + "/" + key;
        if (client.checkExists().forPath(path) != null) {
            client.setData().forPath(path, value.getBytes());
        } else {
            client.create().creatingParentsIfNeeded()
                    .forPath(path, value.getBytes());
        }
        log.info("配置更新: {} = {}", key, value);
    }

    public void watchConfig(String key, ConfigChangeListener listener) throws Exception {
        String path = CONFIG_ROOT + "/" + appName + "/" + env + "/" + key;
        var cache = new org.apache.curator.framework.recipes.cache.NodeCache(client, path);
        cache.getListenable().addListener(() -> {
            var data = cache.getCurrentData();
            if (data != null) {
                listener.onChange(key, new String(data.getData()));
            }
        });
        cache.start();
    }

    public interface ConfigChangeListener {
        void onChange(String key, String newValue);
    }

    // ========== 分布式锁 ==========

    public InterProcessMutex getLock(String lockName) {
        return new InterProcessMutex(client, LOCKS_ROOT + "/" + appName + "/" + lockName);
    }

    public InterProcessReadWriteLock getReadWriteLock(String lockName) {
        return new InterProcessReadWriteLock(client,
                LOCKS_ROOT + "/" + appName + "/" + lockName);
    }

    public boolean tryLock(InterProcessMutex lock, long timeout, TimeUnit unit)
            throws Exception {
        return lock.acquire(timeout, unit);
    }

    public void unlock(InterProcessMutex lock) throws Exception {
        lock.release();
    }

    // ========== Leader 选举 ==========

    public LeaderLatch createLeaderLatch(String taskName) throws Exception {
        LeaderLatch latch = new LeaderLatch(client,
                LEADER_ROOT + "/" + appName + "/" + taskName, instanceId);
        latch.addListener(new LeaderLatchListener() {
            @Override
            public void isLeader() {
                log.info("成为 Leader: {}", taskName);
            }

            @Override
            public void notLeader() {
                log.info("不再是 Leader: {}", taskName);
            }
        });
        latch.start();
        return latch;
    }

    public LeaderSelector createLeaderSelector(String taskName,
                                                LeaderSelectorListener listener) {
        LeaderSelector selector = new LeaderSelector(client,
                LEADER_ROOT + "/" + appName + "/" + taskName, listener);
        selector.autoRequeue();
        selector.start();
        return selector;
    }

    // ========== ID 生成 ==========

    public long generateId() throws Exception {
        DistributedAtomicLong counter = new DistributedAtomicLong(
                client, IDGEN_ROOT + "/" + appName,
                new org.apache.curator.retry.RetryNTimes(10, 50));

        AtomicValue<Long> value = counter.increment();
        if (value.succeeded()) {
            return value.postValue();
        }
        throw new RuntimeException("ID 生成失败");
    }

    @Override
    public void close() {
        client.close();
    }
}
```

#### 步骤 2：治理控制台 REST API

创建 `GovernanceController.java`：

```java
package com.zkdemo.governance;

import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.cache.PathChildrenCache;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.springframework.web.bind.annotation.*;

import java.util.*;
import java.util.stream.Collectors;

@RestController
@RequestMapping("/api/governance")
public class GovernanceController {
    private static final String SERVICES_ROOT = "/services";
    private final CuratorFramework client;
    private final ObjectMapper mapper = new ObjectMapper();

    public GovernanceController() {
        client = CuratorFrameworkFactory.builder()
                .connectString("127.0.0.1:2181")
                .retryPolicy(new ExponentialBackoffRetry(1000, 3))
                .build();
        client.start();
    }

    // 1. 服务拓扑
    @GetMapping("/services")
    public Map<String, Object> getServices() throws Exception {
        Map<String, Object> result = new HashMap<>();

        if (client.checkExists().forPath(SERVICES_ROOT) == null) {
            return result;
        }

        List<String> services = client.getChildren().forPath(SERVICES_ROOT);
        for (String service : services) {
            String servicePath = SERVICES_ROOT + "/" + service;
            List<String> instances = client.getChildren().forPath(servicePath);
            List<Map<String, String>> instanceList = new ArrayList<>();

            for (String instance : instances) {
                String instancePath = servicePath + "/" + instance;
                byte[] data = client.getData().forPath(instancePath);
                instanceList.add(Map.of(
                        "id", instance,
                        "metadata", new String(data)
                ));
            }
            result.put(service, instanceList);
        }
        return result;
    }

    // 2. 配置管理
    @GetMapping("/config/{app}/{env}")
    public List<String> getConfigKeys(@PathVariable String app,
                                       @PathVariable String env) throws Exception {
        String path = "/config/" + app + "/" + env;
        if (client.checkExists().forPath(path) == null) {
            return List.of();
        }
        return client.getChildren().forPath(path);
    }

    @PutMapping("/config/{app}/{env}/{key}")
    public String updateConfig(@PathVariable String app,
                                @PathVariable String env,
                                @PathVariable String key,
                                @RequestBody String value) throws Exception {
        String path = "/config/" + app + "/" + env + "/" + key;
        if (client.checkExists().forPath(path) != null) {
            client.setData().forPath(path, value.getBytes());
        } else {
            client.create().creatingParentsIfNeeded()
                    .forPath(path, value.getBytes());
        }
        return "OK";
    }

    // 3. 锁监控
    @GetMapping("/locks")
    public Map<String, Object> getLocks() throws Exception {
        Map<String, Object> result = new HashMap<>();
        String locksPath = "/locks";

        if (client.checkExists().forPath(locksPath) == null) {
            return result;
        }

        List<String> apps = client.getChildren().forPath(locksPath);
        for (String app : apps) {
            String appPath = locksPath + "/" + app;
            List<String> locks = client.getChildren().forPath(appPath);
            result.put(app, locks);
        }
        return result;
    }

    // 4. 集群状态
    @GetMapping("/cluster/status")
    public Map<String, String> getClusterStatus() {
        return Map.of(
                "status", "running",
                "timestamp", String.valueOf(System.currentTimeMillis())
        );
    }
}
```

#### 步骤 3：Spring Boot 集成

创建 `GovernanceAutoConfiguration.java`：

```java
package com.zkdemo.governance;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.net.InetAddress;

@Configuration
@ConditionalOnGovernanceEnabled
public class GovernanceAutoConfiguration {

    @Value("${zookeeper.connect-string:127.0.0.1:2181}")
    private String connectString;

    @Value("${spring.application.name:unknown}")
    private String appName;

    @Value("${governance.env:dev}")
    private String env;

    @Bean(destroyMethod = "close")
    public GovernanceSDK governanceSDK() throws Exception {
        String host = InetAddress.getLocalHost().getHostAddress();
        String instanceId = host + "@" + appName;

        GovernanceSDK sdk = new GovernanceSDK(connectString, appName, env, instanceId);

        // 自动注册服务
        sdk.registerService(host,
                Integer.getInteger("server.port", 8080));

        return sdk;
    }
}
```

## 4. 项目总结

### SDK 性能指标

| 功能 | 初始化耗时 | 每次调用耗时 | 内存占用 |
|------|-----------|-------------|---------|
| 服务注册 | < 500ms | — | 低 |
| 配置读取 | < 10ms（缓存） | < 1ms（本地） | 低 |
| 分布式锁 | — | < 50ms（正常） | 低 |
| Leader 选举 | < 200ms | — | 低 |
| ID 生成 | — | < 50ms | 低 |

### 适用场景

- 需要治理平台的中型微服务架构（30-100 个服务）
- 对 ZooKeeper 生态已有运维经验的团队

### 注意事项

- SDK 版本和服务端版本需要匹配
- ZooKeeper 故障时，SDK 本地缓存确保基础功能可用
- 治理控制台需要鉴权，避免未授权访问

### 常见踩坑经验

**故障 1：所有服务同时注册导致 ZooKeeper 写放大**

现象：100 个服务同时重启，每个服务都在 ZooKeeper 上创建临时节点，瞬间写 QPS 超过 ZooKeeper 处理能力。

解决方案：服务启动时加入随机延迟（0-5 秒），错峰注册。

**故障 2：配置变更导致所有服务同时刷新连接池**

现象：修改了 `db.url` 配置，80 个服务实例同时检测到配置变化，同时创建新的数据库连接池，数据库被连接打满。

解决方案：配置变更回调中引入"抖动"（jitter），随机延迟 0-3000ms 后再执行。

### 思考题

1. 本章的治理 SDK 中，`generateId()` 使用 `DistributedAtomicLong` 生成全局 ID。如果 ZooKeeper 集群不可用，ID 生成会失败。如何设计一个"本地缓存 + 预分配"方案来缓解这个问题？
2. 治理控制台中显示了所有服务实例的详细信息。如果某个服务的恶意实例注册了虚假信息（如错误的 IP 地址），其他服务可能会调用失败的实例。如何防止这种情况？

### 推广计划提示

- **开发**：所有微服务只需引入治理 SDK，配置连接串，即可获得注册发现、配置管理、分布式锁等能力
- **运维**：治理控制台提供统一的集群管理入口，减少对 ZooKeeper 命令行的依赖
- **架构师**：治理平台是微服务架构的基础设施，建议在项目初期就纳入统一规划
