# 第39章：自定义扩展——动态重新配置与插件开发

## 1. 项目背景

### 业务场景

作为 ZooKeeper 的资深用户，你可能需要：

- **动态扩缩容**：不重启集群，在线添加/移除节点（3.5+ 的 Reconfig）
- **自定义 RequestProcessor**：插入自己的处理逻辑，比如请求审计、延迟监控
- **自定义选举算法**：替换默认的 FLE 实现
- **自定义持久化**：将事务日志存到数据库而不是文件

ZooKeeper 的模块化架构支持这些扩展，本章带你实现一个**请求审计 Processor**。

### 痛点放大

没有自定义扩展能力：

- 扩缩容需要重启整个集群，影响业务
- 无法插入自定义的请求审计或监控逻辑
- 只能依赖 ZooKeeper 的内置功能，无法满足特殊需求

## 2. 项目设计

### 剧本式交锋对话

**场景**：小白需要在 ZooKeeper 上添加请求审计功能，记录所有写操作。

**小白**：第 30 章的审计日志只能记录操作结果，我想把每次写入的请求体也记录下来，用于后续分析。

**大师**：这可以通过自定义 RequestProcessor 实现。我们可以创建一个 `AuditRequestProcessor`，插入到管道的 `PrepRequestProcessor` 和 `SyncRequestProcessor` 之间。

```
自定义 Processor 位置：

PrepRequestProcessor → AuditRequestProcessor (新插入) → SyncRequestProcessor → ...
                              │
                              ▼
                     记录请求路径、数据、操作类型到审计日志
```

**小胖**：那动态 Reconfig 呢？怎么在线添加一个节点？

**大师**：ZooKeeper 3.5+ 的 `reconfig` 功能允许动态修改集群成员：


### 分步实现

#### 步骤 1：自定义 RequestProcessor——审计日志

创建 `AuditRequestProcessor.java`：

```java
package com.zkdemo.extension;

import org.apache.jute.BinaryOutputArchive;
import org.apache.zookeeper.proto.*;
import org.apache.zookeeper.server.*;
import org.apache.zookeeper.txn.*;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.io.ByteArrayOutputStream;
import java.nio.ByteBuffer;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

/**
 * 请求审计 Processor——记录所有写操作到审计日志
 * 插入到 PrepRequestProcessor 和 SyncRequestProcessor 之间
 */
public class AuditRequestProcessor implements RequestProcessor {
    private static final Logger auditLog = LoggerFactory.getLogger("AUDIT");
    private static final DateTimeFormatter FORMATTER =
            DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss.SSS");

    private final RequestProcessor nextProcessor;

    public AuditRequestProcessor(RequestProcessor nextProcessor) {
        this.nextProcessor = nextProcessor;
    }

    @Override
    public void processRequest(Request request) throws RequestProcessorException {
        // 只审计写操作
        if (request.getHdr() != null) {
            String operation = getOperationName(request.getHdr().getType());
            String path = extractPath(request);
            String user = extractUser(request);
            String data = extractData(request);

            auditLog.info("{} | {} | user={} | path={} | data={} | zxid=0x{}",
                    LocalDateTime.now().format(FORMATTER),
                    operation, user, path, data,
                    Long.toHexString(request.getHdr().getZxid()));
        }

        // 传递给下一个 Processor
        nextProcessor.processRequest(request);
    }

    private String getOperationName(int type) {
        return switch (type) {
            case OpCode.create -> "CREATE";
            case OpCode.setData -> "SETDATA";
            case OpCode.delete -> "DELETE";
            case OpCode.setACL -> "SETACL";
            case OpCode.multi -> "MULTI";
            case OpCode.reconfig -> "RECONFIG";
            default -> "UNKNOWN(" + type + ")";
        };
    }

    private String extractPath(Request request) {
        if (request.path != null) return request.path;
        if (request.request != null) {
            try {
                ByteBuffer bb = request.request;
                // 尝试从请求体中提取路径
                // 对于 create/setData 请求，前几个字节是路径
                // 这里简化处理
                return "path-unknown";
            } catch (Exception e) {
                return "parse-error";
            }
        }
        return "unknown";
    }

    private String extractUser(Request request) {
        var authInfo = request.authInfo;
        if (authInfo != null && !authInfo.isEmpty()) {
            return authInfo.get(0).getId();
        }
        return "anonymous";
    }

    private String extractData(Request request) {
        try {
            if (request.getTxn() != null) {
                if (request.getTxn() instanceof CreateTxn createTxn) {
                    return new String(createTxn.getData()).replaceAll("[\0]", "");
                } else if (request.getTxn() instanceof SetDataTxn setDataTxn) {
                    return new String(setDataTxn.getData()).replaceAll("[\0]", "");
                } else if (request.getTxn() instanceof DeleteTxn deleteTxn) {
                    return deleteTxn.getPath();
                }
            }
        } catch (Exception ignored) {}
        return "";
    }

    @Override
    public void shutdown() {
        if (nextProcessor != null) {
            nextProcessor.shutdown();
        }
    }
}
```

#### 步骤 2：将自定义 Processor 插入 Leader 管道

修改 `LeaderZooKeeperServer.java` 的 `setupRequestProcessors()`：

```java
// 文件位置：org.apache.zookeeper.server.quorum.LeaderZooKeeperServer
// 修改 setupRequestProcessors 方法

@Override
protected void setupRequestProcessors() {
    RequestProcessor finalProcessor = new FinalRequestProcessor(this);
    RequestProcessor toBeAppliedProcessor =
            new ToBeAppliedRequestProcessor(this, finalProcessor);
    commitProcessor = new CommitProcessor(toBeAppliedProcessor,
            Long.toString(getServerId()), false, getZooKeeperServerListener());
    commitProcessor.start();

    ProposalRequestProcessor proposalProcessor =
            new ProposalRequestProcessor(this, commitProcessor);
    proposalProcessor.start();

    syncProcessor = new SyncRequestProcessor(this, proposalProcessor);
    syncProcessor.start();

    // ========== 插入自定义 AuditRequestProcessor ==========
    // 在 PrepRequestProcessor 和 SyncRequestProcessor 之间插入
    RequestProcessor auditProcessor = new AuditRequestProcessor(syncProcessor);

    firstProcessor = new PrepRequestProcessor(this, auditProcessor);
    // ====================================================

    ((PrepRequestProcessor) firstProcessor).start();
}
```

#### 步骤 5：自定义选举算法

创建 `CustomElection.java`：

```java
package com.zkdemo.extension;

import org.apache.zookeeper.server.quorum.*;
import org.apache.zookeeper.server.quorum.FastLeaderElection;

/**
 * 自定义选举算法
 * 基于 FLE 但修改比较逻辑：
 * 优先选择 sid 小的节点（而不是默认的 sid 大的节点）
 */
public class CustomElection extends FastLeaderElection {

    public CustomElection(QuorumPeer self, QuorumCnxManager manager) {
        super(self, manager);
    }

    /**
     * 重写 totalOrderPredicate——修改比较逻辑
     * 原始逻辑：epoch > zxid > sid（sid 大的胜）
     * 自定义逻辑：epoch > zxid > sid（sid 小的胜！）
     */
    @Override
    protected boolean totalOrderPredicate(long newId, long newZxid,
                                           long newEpoch, long curId,
                                           long curZxid, long curEpoch) {
        if (newEpoch > curEpoch) {
            return true;
        } else if (newEpoch < curEpoch) {
            return false;
        }
        if (newZxid > curZxid) {
            return true;
        } else if (newZxid < curZxid) {
            return false;
        }
        // 修改点：选择 sid 小的节点
        // 原始：return newId > curId;
        return newId < curId;  // 现在小的获胜！
    }
}
```

在 `QuorumPeer.java` 中注册自定义选举算法：

```java
// 在 startLeaderElection 方法中
public synchronized void startLeaderElection() {
    // 根据配置选择选举算法
    if (electionAlg == 3) {
        // 默认使用 FastLeaderElection
        if (Boolean.getBoolean("zookeeper.useCustomElection")) {
            election = new CustomElection(this, qcm);
        } else {
            election = new FastLeaderElection(this, qcm);
        }
    }
}
```

#### 步骤 6：自定义持久化

创建 `DatabaseTxnLog.java`（将事务日志写入数据库的示例）：

```java
package com.zkdemo.extension;

import org.apache.zookeeper.server.persistence.TxnLog;
import org.apache.zookeeper.txn.TxnHeader;

import javax.sql.DataSource;
import java.io.IOException;
import java.sql.Connection;
import java.sql.PreparedStatement;

/**
 * 基于数据库的事务日志存储
 * 替换默认的 FileTxnLog
 */
public class DatabaseTxnLog implements TxnLog {
    private final DataSource dataSource;

    public DatabaseTxnLog(DataSource dataSource) {
        this.dataSource = dataSource;
    }

    @Override
    public boolean append(TxnHeader hdr, org.apache.jute.Record txn)
            throws IOException {
        try (Connection conn = dataSource.getConnection();
             PreparedStatement ps = conn.prepareStatement(
                     "INSERT INTO txn_log (zxid, time, type, data) VALUES (?, ?, ?, ?)")) {

            ps.setLong(1, hdr.getZxid());
            ps.setLong(2, hdr.getTime());
            ps.setInt(3, hdr.getType());
            ps.setBytes(4, serializeTxn(txn));

            return ps.executeUpdate() > 0;
        } catch (Exception e) {
            throw new IOException("写入事务日志失败", e);
        }
    }

    private byte[] serializeTxn(org.apache.jute.Record txn) throws IOException {
        // 序列化事务体
        java.io.ByteArrayOutputStream baos = new java.io.ByteArrayOutputStream();
        org.apache.jute.BinaryOutputArchive boa =
                org.apache.jute.BinaryOutputArchive.getArchive(baos);
        txn.serialize(boa, "txn");
        return baos.toByteArray();
    }

    @Override
    public void close() throws IOException {
        // 关闭数据库连接池
    }

    // ... 其他 TxnLog 接口实现
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 自定义 Processor 未生效 | 未正确插入管道 | 检查 LeaderZooKeeperServer.setupRequestProcessors |
| Reconfig 失败 | `reconfigEnabled=true` 未配置 | 检查 zoo.cfg |
| 自定义选举导致集群不稳定 | 比较逻辑改变 | 在测试环境充分验证 |

## 4. 项目总结

### 扩展点总结

| 扩展点 | 方式 | 难度 | 用途 |
|-------|------|------|------|
| RequestProcessor | 插入管道 | 中 | 审计、监控、过滤 |
| 选举算法 | 继承 FastLeaderElection | 高 | 自定义选举策略 |
| 持久化 | 实现 TxnLog 接口 | 高 | 数据库存储、云存储 |
| Reconfig | 内置（3.5+） | 低 | 动态扩缩容 |

### 注意事项

- 自定义 Processor 必须调用 `nextProcessor.processRequest()`，否则管道断裂
- Reconfig 需要 `reconfigEnabled=true`，且需要 ACL 或 `skipACL=yes`
- 修改选举算法可能在集群中产生不一致行为，务必充分测试

### 思考题

1. 自定义 `AuditRequestProcessor` 插在 `PrepRequestProcessor` 之后，但此时请求还没有被持久化。如果插入在 `FinalRequestProcessor` 之后会怎么样？这样能记录到真正的"已提交"请求吗？
2. ZooKeeper 的动态 Reconfig 支持在运行时添加/移除节点。如果一个节点正在被移除的同时还有一个正在执行的选举操作，会发生什么？Reconfig 如何保证和选举操作的原子性？

### 推广计划提示

- **开发**：自定义 Processor 是实现 ZooKeeper 扩展功能的最常见方式
- **运维**：动态 Reconfig 让集群扩缩容不再需要停机
- **贡献者**：ZooKeeper 的模块化设计鼓励社区贡献自定义实现
