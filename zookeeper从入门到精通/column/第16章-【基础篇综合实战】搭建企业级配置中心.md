# 第16章：【基础篇综合实战】搭建企业级配置中心

## 1. 项目背景

### 业务场景

一家电商公司有 30+ 微服务，每个服务都依赖配置项：数据库连接串、Redis 地址、消息队列配置、功能开关、限流阈值。当前的问题是：

- 配置散落在每个服务的 `application.properties` 文件中
- 修改配置需要重新打包、部署，周期 30 分钟
- 各环境（dev/staging/prod）的配置经常搞混
- 紧急关闭某个功能开关，需要 30 分钟才能生效
- 没有配置变更的审计日志

目标是基于 ZooKeeper 搭建一套生产级配置中心，实现：

1. **多环境隔离**：dev/staging/prod 配置独立管理
2. **实时推送**：配置变更后 3 秒内推送到所有实例
3. **灰度发布**：新配置先推给 10% 的实例验证
4. **版本管理**：支持配置回滚
5. **权限控制**：不同团队只能操作自己的配置

### 痛点放大

不用配置中心的直接后果：

- **配置错误导致故障**：生产环境的数据库连接串配成了测试环境的，导致数据写入了错误的数据库
- **故障恢复慢**：某个功能开关需要关闭，但需要发版、审批、部署，耗时数小时
- **资源配置浪费**：连接池大小等参数无法动态调整，只能按最大负载配置，日常资源浪费 50%+

## 2. 项目设计

### 剧本式交锋对话

**场景**：架构评审会上，小胖提出了配置中心的方案。

**小胖**：我们直接用 ZooKeeper 做配置中心，把配置放在 ZNode 里，Watcher 监听变化不就行了吗？

**大师**：基础思路是对的，但生产环境需要考虑很多细节：

```
配置路径设计（多环境隔离）:
/config/{app}/{env}/{key}

/config/order-service/prod/db-url = "jdbc:mysql://prod-db:3306/order"
/config/order-service/prod/redis-host = "prod-redis:6379"
/config/order-service/prod/rate-limit = "5000"
/config/order-service/staging/db-url = "jdbc:mysql://staging-db:3306/order"
/config/order-service/dev/db-url = "jdbc:mysql://dev-db:3306/order"

灰度配置:
/config/order-service/prod/db-url              ← 默认配置
/config/order-service/prod/_gray/db-url        ← 灰度配置
/config/order-service/prod/_gray/instances     ← 灰度实例列表
```

**小白**：那客户端怎么知道自己在灰度名单中？

**大师**：每个服务实例启动时，从 ZooKeeper 读取 `/config/{app}/{env}/_gray/instances` 节点，判断自己的 `instanceId` 是否在其中。如果在，读取灰度节点；如果不在，读取默认节点。

灰度实例列表变化时，客户端通过 PathChildrenCache 接收通知，自动切换配置源。

**小胖**：配置回滚怎么做？ZooKeeper 没有"版本树"机制。

**大师**：利用 ZooKeeper 的版本号（dataVersion）实现：
- 每次修改配置，dataVersion +1
- 同步记录变更历史到 `/config/{app}/{env}/_history/{key}`（持久顺序节点）
- 回滚时读取历史版本的数据，调用 `setData` 恢复

**小胖**：客户端怎么保证配置中心不可用时系统不崩溃？

**大师**：客户端采用"三级缓存"策略：

```
第一级：本地内存缓存（最快）
第二级：本地文件缓存（进程重启后恢复）
第三级：ZooKeeper 远程配置（真实来源）

读取顺序：内存 → 文件 → ZooKeeper
写入：ZooKeeper（触发 → Watcher → 更新内存 + 文件）
```

**小白**：权限控制呢？总不能所有服务都能改所有配置吧？

**大师**：ACL 控制 + 路径隔离：

```bash
# 订单服务只能读写自己的配置
addauth digest order-service:password
setAcl /config/order-service digest:order-service:cdrwa,world:anyone:r

# 用户服务只能读写自己的配置
addauth digest user-service:password
setAcl /config/user-service digest:user-service:cdrwa,world:anyone:r

# 运维管理员可以操作所有路径
setAcl /config digest:admin:cdrwa,world:anyone:
```

#### 步骤 2：配置中心服务端实现

创建 `ConfigCenterServerApplication.java`：

```java
package com.zkdemo.configcenter;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@SpringBootApplication
@RestController
@RequestMapping("/api/config")
public class ConfigCenterServerApplication {
    private final CuratorFramework client;

    public ConfigCenterServerApplication() {
        client = CuratorFrameworkFactory.builder()
                .connectString("127.0.0.1:2181")
                .namespace("config")
                .retryPolicy(new ExponentialBackoffRetry(1000, 3))
                .build();
        client.start();
    }

    // 获取配置
    @GetMapping("/{app}/{env}/{key}")
    public String getConfig(@PathVariable String app,
                            @PathVariable String env,
                            @PathVariable String key) throws Exception {
        String path = "/" + app + "/" + env + "/" + key;
        byte[] data = client.getData().forPath(path);
        return new String(data);
    }

    // 更新配置
    @PutMapping("/{app}/{env}/{key}")
    public String updateConfig(@PathVariable String app,
                               @PathVariable String env,
                               @PathVariable String key,
                               @RequestBody String value) throws Exception {
        String path = "/" + app + "/" + env + "/" + key;

        // 保存历史版本
        String historyPath = "/" + app + "/" + env + "/_history/" + key + "/";
        byte[] oldData = client.getData().forPath(path);
        client.create().creatingParentsIfNeeded()
                .forPath(historyPath, oldData);

        // 更新配置
        client.setData().forPath(path, value.getBytes());

        return "updated: " + key + " = " + value;
    }

    // 获取配置历史
    @GetMapping("/{app}/{env}/{key}/history")
    public List<String> getHistory(@PathVariable String app,
                                   @PathVariable String env,
                                   @PathVariable String key) throws Exception {
        String historyPath = "/" + app + "/" + env + "/_history/" + key;
        if (client.checkExists().forPath(historyPath) == null) {
            return List.of();
        }
        return client.getChildren().forPath(historyPath);
    }

    // 回滚配置
    @PostMapping("/{app}/{env}/{key}/rollback/{version}")
    public String rollback(@PathVariable String app,
                           @PathVariable String env,
                           @PathVariable String key,
                           @PathVariable String version) throws Exception {
        String historyPath = "/" + app + "/" + env + "/_history/" + key + "/" + version;
        byte[] historyData = client.getData().forPath(historyPath);
        String configPath = "/" + app + "/" + env + "/" + key;
        client.setData().forPath(configPath, historyData);
        return "rolled back: " + key + " to version " + version;
    }

    public static void main(String[] args) {
        SpringApplication.run(ConfigCenterServerApplication.class, args);
    }
}
```

#### 步骤 3：配置中心客户端 SDK

创建 `ConfigClient.java`：

```java
package com.zkdemo.configcenter;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.cache.ChildData;
import org.apache.curator.framework.recipes.cache.NodeCache;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.io.*;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

public class ConfigClient implements Closeable {
    private static final Logger log = LoggerFactory.getLogger(ConfigClient.class);

    private final CuratorFramework client;
    private final String appName;
    private final String env;
    private final String instanceId;
    private final Map<String, String> memoryCache = new ConcurrentHashMap<>();
    private final Map<String, ConfigEntry> watchers = new ConcurrentHashMap<>();
    private final String fileCachePath;

    public ConfigClient(String connectString, String appName, String env, String instanceId) {
        this.appName = appName;
        this.env = env;
        this.instanceId = instanceId;
        this.fileCachePath = System.getProperty("user.home") + "/.config-cache/" + appName + "/" + env + "/";

        client = CuratorFrameworkFactory.builder()
                .connectString(connectString)
                .namespace("config")
                .retryPolicy(new ExponentialBackoffRetry(1000, 3))
                .build();
        client.start();
    }

    // 初始化并加载配置
    public void init(Map<String, String> defaultConfig) throws Exception {
        File cacheDir = new File(fileCachePath);
        cacheDir.mkdirs();

        for (Map.Entry<String, String> entry : defaultConfig.entrySet()) {
            String key = entry.getKey();
            String path = buildConfigPath(key);

            try {
                // 从 ZooKeeper 读取
                if (client.checkExists().forPath(path) == null) {
                    client.create().creatingParentsIfNeeded()
                            .forPath(path, entry.getValue().getBytes());
                }
                byte[] data = client.getData().forPath(path);
                String value = new String(data);
                updateCache(key, value);

                // 注册监听
                registerWatcher(key, path);
            } catch (Exception e) {
                // ZooKeeper 不可用，从文件缓存加载
                String fileValue = loadFromFileCache(key);
                if (fileValue != null) {
                    updateCache(key, fileValue);
                    log.warn("从文件缓存加载配置 {} = {}", key, fileValue);
                } else {
                    updateCache(key, entry.getValue());
                    log.warn("使用默认配置 {} = {}", key, entry.getValue());
                }
            }
        }
    }

    private void registerWatcher(String key, String path) throws Exception {
        NodeCache cache = new NodeCache(client, path);
        cache.getListenable().addListener(() -> {
            if (cache.getCurrentData() != null) {
                String newValue = new String(cache.getCurrentData().getData());
                String oldValue = memoryCache.get(key);
                if (!newValue.equals(oldValue)) {
                    updateCache(key, newValue);
                    log.info("配置变更: {} = {} → {}", key, oldValue, newValue);
                    onConfigChanged(key, oldValue, newValue);
                }
            }
        });
        cache.start();
        watchers.put(key, new ConfigEntry(cache, path));
    }

    private void updateCache(String key, String value) {
        memoryCache.put(key, value);
        saveToFileCache(key, value);
    }

    private void saveToFileCache(String key, String value) {
        try (FileWriter fw = new FileWriter(fileCachePath + key)) {
            fw.write(value);
        } catch (IOException e) {
            log.error("保存文件缓存失败: {}", key, e);
        }
    }

    private String loadFromFileCache(String key) {
        try (BufferedReader br = new BufferedReader(new FileReader(fileCachePath + key))) {
            return br.readLine();
        } catch (IOException e) {
            return null;
        }
    }

    public String get(String key) {
        return memoryCache.get(key);
    }

    public int getInt(String key, int defaultValue) {
        String value = memoryCache.get(key);
        if (value == null) return defaultValue;
        try {
            return Integer.parseInt(value);
        } catch (NumberFormatException e) {
            return defaultValue;
        }
    }

    public boolean getBoolean(String key, boolean defaultValue) {
        String value = memoryCache.get(key);
        if (value == null) return defaultValue;
        return Boolean.parseBoolean(value);
    }

    public void setConfig(String key, String value) throws Exception {
        String path = buildConfigPath(key);
        client.setData().forPath(path, value.getBytes());
    }

    private String buildConfigPath(String key) {
        return "/" + appName + "/" + env + "/" + key;
    }

    // 配置变更回调——子类可覆盖
    protected void onConfigChanged(String key, String oldValue, String newValue) {
    }

    @Override
    public void close() throws IOException {
        watchers.values().forEach(w -> {
            try {
                w.cache.close();
            } catch (Exception e) {
                // ignore
            }
        });
        client.close();
    }

    private record ConfigEntry(NodeCache cache, String path) {
    }
}
```

#### 步骤 4：订单服务集成配置中心

创建 `OrderServiceWithConfig.java`：

```java
package com.zkdemo.configcenter;

import javax.sql.DataSource;
import java.util.HashMap;
import java.util.Map;

public class OrderServiceWithConfig {
    private final ConfigClient configClient;
    private DataSource dataSource;

    public OrderServiceWithConfig(String instanceId) throws Exception {
        // 创建配置客户端（订单服务，生产环境）
        configClient = new ConfigClient("127.0.0.1:2181",
                "order-service", "prod", instanceId) {
            @Override
            protected void onConfigChanged(String key, String oldValue, String newValue) {
                switch (key) {
                    case "db-url" -> {
                        System.out.println(">>> 刷新数据库连接池: " + newValue);
                        refreshDataSource(newValue);
                    }
                    case "rate-limit" -> {
                        System.out.println(">>> 更新限流阈值: " + newValue);
                        updateRateLimiter(Integer.parseInt(newValue));
                    }
                    case "feature-switch" -> {
                        System.out.println(">>> 功能开关变更: " + newValue);
                    }
                }
            }
        };

        // 初始化配置（默认值）
        Map<String, String> defaults = new HashMap<>();
        defaults.put("db-url", "jdbc:mysql://prod-db:3306/order");
        defaults.put("redis-host", "prod-redis:6379");
        defaults.put("rate-limit", "5000");
        defaults.put("feature-switch", "true");
        defaults.put("order-timeout", "30000");
        defaults.put("max-retries", "3");
        configClient.init(defaults);

        System.out.println("\n=== 订单服务配置加载完成 ===");
        System.out.println("实例 ID: " + instanceId);
        System.out.println("db-url: " + configClient.get("db-url"));
        System.out.println("redis-host: " + configClient.get("redis-host"));
        System.out.println("rate-limit: " + configClient.getInt("rate-limit", 0));
        System.out.println("feature-switch: " + configClient.getBoolean("feature-switch", false));
        System.out.println("\n等待配置变更...");
    }

    private void refreshDataSource(String dbUrl) {
        System.out.println("  关闭旧连接池，创建新连接池: " + dbUrl);
    }

    private void updateRateLimiter(int rate) {
        System.out.println("  更新限流器阈值: " + rate + " req/s");
    }

    public void close() throws Exception {
        configClient.close();
    }

    public static void main(String[] args) throws Exception {
        String instanceId = args.length > 0 ? args[0] : "instance-001";
        OrderServiceWithConfig service = new OrderServiceWithConfig(instanceId);
        Thread.sleep(Long.MAX_VALUE);
    }
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 初始化时 ZooKeeper 不可用 | 网络故障或 ZooKeeper 未启动 | 从文件缓存加载配置 |
| 文件缓存被篡改 | 手动修改了文件缓存 | 校验文件缓存 checksum |
| 灰度实例列表变更通知延迟 | PathChildrenCache 异步初始化 | 启动时主动拉取一次 |
| 配置回滚到错误版本 | 回滚了历史版本的正确数据 | 在回滚前备份当前版本 |

## 4. 项目总结

### 架构总览

```
┌─────────────────────────────────────────────────────┐
│                  ZooKeeper 集群                       │
│  /config/{app}/{env}/{key}                           │
│  /config/{app}/{env}/_gray/{key}                     │
│  /config/{app}/{env}/_history/{key}                  │
└──────────┬──────────────────────────────┬───────────┘
           │                              │
    ┌──────▼──────┐              ┌───────▼──────────┐
    │ Config Admin │              │  Config Client   │
    │ (REST API)   │              │  (SDK 内嵌)      │
    │              │              │                  │
    │ · 读取配置   │              │ · 内存缓存       │
    │ · 更新配置   │◄──Watcher── │ · 文件缓存       │
    │ · 回滚版本   │              │ · 实时回调       │
    │ · 灰度发布   │              │ · 灰度检测       │
    └──────────────┘              └──────────────────┘
```

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 实时性 | Watcher 毫秒级推送 | 一次性 Watcher，复杂场景需重新注册 |
| 可用性 | 三级缓存确保 ZooKeeper 不可用时可用 | 初次启动必须依赖 ZooKeeper |
| 灰度 | 按实例灰度，可控可观测 | 灰度列表需手动维护 |
| 历史 | 顺序节点记录变更历史 | 历史版本只能回滚，不能 diff |

### 适用场景

- 中小规模微服务架构（30-50 个服务）
- 配置项数量适中（每个服务 < 50 个配置项）
- 对配置变更实时性要求高的场景（毫秒级）

**不适用场景**：
- 大规模配置存储（> 10 万配置项，ZooKeeper 内存受限）
- 需要配置 Diff 和复杂的审批流

### 注意事项

- 每个 ZNode 的数据限制 1MB，单个配置项不要超过这个限制
- 文件缓存需要定期清理，避免磁盘占用过多
- 灰度发布功能需要客户端配合实现，纯运维改配置做不到灰度
- ACL 一定要配置，否则任何能连 ZooKeeper 的人都能改配置

### 常见踩坑经验

**故障 1：所有客户端同时收到配置变更通知导致 ZooKeeper 过载**

现象：运维修改了一个配置，100 个服务实例同时触发 NodeCache 回调，每个实例又发起一次 `getData`，ZooKeeper 瞬间 QPS 飙升。

解决方案：在客户端回调中加入"抖动"（jitter），随机延迟 0-500ms 再执行。

**故障 2：文件缓存和 ZooKeeper 配置不一致**

现象：ZooKeeper 正常服务，客户端却用了文件缓存的旧配置。

根因：NodeCache 启动时的初始数据加载是异步的，在初始化完成前读取配置，返回的是文件缓存的旧值。

解决方案：在 `init()` 方法中，等待 NodeCache 初始化完再返回。

### 思考题

1. 本文的灰度发布方案是"白名单"模式——运维手动将实例 ID 加入灰度列表。如何设计一个"按百分比"的灰度策略？比如：新配置自动推送给 10% 的实例。
2. 配置中心的配置变更历史中，有一个"回滚操作"也被记录到历史中了。如果回滚后发现问题，想撤销回滚（也就是恢复回滚前的状态），应该怎么做？

### 推广计划提示

- **开发**：集成配置中心 SDK 后，代码中通过 `configClient.get("key")` 读取配置，`onConfigChanged` 处理变更，不再需要硬编码配置
- **运维**：通过 Admin API 修改配置，观察客户端的实时推送效果
- **测试**：测试覆盖：ZooKeeper 断连后从文件缓存加载、配置变更后回调触发、灰度实例与非灰度实例的配置差异
