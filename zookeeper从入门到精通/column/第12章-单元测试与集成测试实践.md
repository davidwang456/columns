# 第12章：单元测试与集成测试实践

## 1. 项目背景

### 业务场景

你在第 9 章中实现了一个分布式锁服务，在生产环境运行了一个月，一切正常。但今天，运维报告说锁服务出现问题：某些情况下锁无法释放，导致业务线程永久阻塞。

你打开 IDE，想复现这个问题，但你面临一个经典困境——**测试需要 ZooKeeper 集群**。你的开发机上没有跑 ZooKeeper，单元测试必须依赖外部服务。

ZooKeeper 相关的测试有特殊挑战：
- 需要真正的 ZooKeeper 服务端（不是 Mock）
- 分布式锁需要多个客户端并发竞争才能测试
- 网络异常、会话过期需要模拟
- 测试用例之间需要数据隔离

### 痛点放大

没有好的测试策略，ZooKeeper 相关代码的问题很难排查：

- **不能 Mock**：ZooKeeper 的核心行为（Session 管理、Watcher 触发、zxid 递增）无法通过 Mock 验证
- **环境依赖**：每次跑测试都要启动外部 ZooKeeper 服务，CI 流水线也需要配置
- **并发测试难**：分布式锁的竞争条件难以在单线程测试中复现
- **状态残留**：测试用例之间，ZooKeeper 上的数据会互相影响

Apache Curator 的 `curator-test` 模块提供了 `TestingServer` 和 `TestingCluster`，让你在单元测试中**内嵌启动 ZooKeeper 服务端**。

## 2. 项目设计

### 剧本式交锋对话

**场景**：小白在写一个分布式锁的单元测试，但不知道如何模拟 ZooKeeper 环境。

**小白**：我要测试分布式锁的 acquire 和 release，但每次跑测试前都要手动启动 ZooKeeper，太麻烦了。

**大师**：用 Curator 的 `TestingServer`。它启动一个内嵌的 ZooKeeper 服务端，测试结束自动关闭。

```java
@BeforeEach
void setUp() throws Exception {
    // 启动内嵌的 ZooKeeper（随机端口）
    testingServer = new TestingServer();
    client = CuratorFrameworkFactory.newClient(
            testingServer.getConnectString(),
            new ExponentialBackoffRetry(1000, 3));
    client.start();
}

@AfterEach
void tearDown() {
    client.close();
    testingServer.close();
}
```

**小胖**：那 TestingCluster 是干啥的？和 TestingServer 有什么区别？

**大师**：`TestingServer` 启动一个单节点 ZooKeeper，适合大多数测试。`TestingCluster` 启动一个多节点集群（伪集群），适合测试选举、ZAB 协议等集群特性。

```java
// 启动 3 节点集群
TestingCluster cluster = new TestingCluster(3);
cluster.start();

// 获取每个节点的连接串
for (TestingServer server : cluster.getServers()) {
    System.out.println(server.getConnectString());
}
```

**小白**：那测试分布式锁需要多个客户端并发获取锁，怎么写测试？

**大师**：用 `ExecutorService` 模拟多客户端：

```java
@Test
void testConcurrentLock() throws Exception {
    InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");
    int threadCount = 10;
    CountDownLatch latch = new CountDownLatch(threadCount);
    AtomicInteger successCount = new AtomicInteger(0);

    for (int i = 0; i < threadCount; i++) {
        executor.submit(() -> {
            lock.acquire();
            successCount.incrementAndGet();
            lock.release();
            latch.countDown();
        });
    }
    latch.await();
    assertEquals(threadCount, successCount.get());
}
```

**小胖**：模拟网络断开、会话过期这些异常场景呢？

**大师**：`TestingServer` 提供了 `stop()` 和 `restart()` 方法：

```java
@Test
void testSessionExpired() throws Exception {
    InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");
    lock.acquire();

    // 模拟网络分区——停止 ZooKeeper 服务
    testingServer.stop();
    Thread.sleep(15000);  // 等待会话过期
    // 重启 ZooKeeper
    testingServer.restart();

    // Curator 自动重连并创建新会话
    assertThrows(Exception.class, () -> lock.release());
    // 旧锁需要重新获取
}
```

> **技术映射**：TestingServer = 开发环境自带的测试数据库（H2），TestingCluster = 测试环境的多节点集群

### 分步实现

#### 步骤 1：基础 TestingServer 测试

创建 `DistributedLockTest.java`：

```java
package com.zkdemo;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.locks.InterProcessMutex;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.curator.test.TestingServer;
import org.junit.jupiter.api.*;

import java.util.concurrent.CountDownLatch;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.atomic.AtomicInteger;

import static org.junit.jupiter.api.Assertions.*;

class DistributedLockTest {
    private TestingServer testingServer;
    private CuratorFramework client;

    @BeforeEach
    void setUp() throws Exception {
        testingServer = new TestingServer();
        client = CuratorFrameworkFactory.builder()
                .connectString(testingServer.getConnectString())
                .sessionTimeoutMs(5000)
                .retryPolicy(new ExponentialBackoffRetry(500, 3))
                .build();
        client.start();
    }

    @AfterEach
    void tearDown() throws Exception {
        client.close();
        testingServer.close();
    }

    @Test
    @DisplayName("基础加锁/解锁测试")
    void testBasicLock() throws Exception {
        InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");

        // 获取锁
        assertTrue(lock.acquire(1, TimeUnit.SECONDS));
        // 验证锁的路径存在
        assertNotNull(client.checkExists().forPath("/test-lock"));
        // 释放锁
        lock.release();
    }

    @Test
    @DisplayName("可重入锁测试——同一线程可多次获取")
    void testReentrantLock() throws Exception {
        InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");

        // 第一次获取
        lock.acquire();
        // 第二次获取（可重入）
        lock.acquire();
        // 释放两次
        lock.release();
        lock.release();
        // 验证锁已释放
        assertTrue(lock.acquire(1, TimeUnit.SECONDS));
        lock.release();
    }

    @Test
    @DisplayName("并发锁测试——10个线程竞争同一把锁")
    void testConcurrentLock() throws Exception {
        InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");
        int threadCount = 10;
        ExecutorService executor = Executors.newFixedThreadPool(threadCount);
        CountDownLatch latch = new CountDownLatch(threadCount);
        AtomicInteger concurrentCount = new AtomicInteger(0);
        AtomicInteger maxConcurrent = new AtomicInteger(0);

        for (int i = 0; i < threadCount; i++) {
            executor.submit(() -> {
                try {
                    lock.acquire();
                    // 获取锁后递增并发计数
                    int current = concurrentCount.incrementAndGet();
                    maxConcurrent.updateAndGet(v -> Math.max(v, current));
                    Thread.sleep(100); // 模拟业务处理
                    concurrentCount.decrementAndGet();
                    lock.release();
                } catch (Exception e) {
                    fail("锁操作失败: " + e.getMessage());
                } finally {
                    latch.countDown();
                }
            });
        }

        assertTrue(latch.await(30, TimeUnit.SECONDS), "测试超时");
        // 最大同时持有锁的线程数不能超过 1
        assertEquals(1, maxConcurrent.get(), "分布式锁应保证互斥");
        executor.shutdown();
    }

    @Test
    @DisplayName("锁超时测试——获取锁超时返回false")
    void testLockTimeout() throws Exception {
        InterProcessMutex lock1 = new InterProcessMutex(client, "/test-lock");
        InterProcessMutex lock2 = new InterProcessMutex(client, "/test-lock");

        // 线程1获取锁
        lock1.acquire();

        // 线程2尝试获取锁（超时100ms）
        long start = System.currentTimeMillis();
        boolean acquired = lock2.acquire(100, TimeUnit.MILLISECONDS);
        long elapsed = System.currentTimeMillis() - start;

        assertFalse(acquired, "锁已被持有，应获取失败");
        assertTrue(elapsed >= 100, "应等待超时时间才返回");

        lock1.release();
    }

    @Test
    @DisplayName("异常释放测试——未获取锁时释放应抛异常")
    void testReleaseWithoutAcquire() {
        InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");
        assertThrows(Exception.class, lock::release,
                "未获取锁时释放应抛出异常");
    }
}
```

#### 步骤 2：会话过期测试

创建 `SessionExpirationTest.java`：

```java
package com.zkdemo;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.locks.InterProcessMutex;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.curator.test.TestingServer;
import org.junit.jupiter.api.*;

import java.util.concurrent.TimeUnit;

import static org.junit.jupiter.api.Assertions.*;

class SessionExpirationTest {
    private TestingServer testingServer;
    private CuratorFramework client;

    @BeforeEach
    void setUp() throws Exception {
        testingServer = new TestingServer();
        client = CuratorFrameworkFactory.builder()
                .connectString(testingServer.getConnectString())
                .sessionTimeoutMs(3000) // 3秒超时，方便测试
                .retryPolicy(new ExponentialBackoffRetry(500, 3))
                .build();
        client.start();
    }

    @AfterEach
    void tearDown() throws Exception {
        client.close();
        testingServer.close();
    }

    @Test
    @DisplayName("会话过期后旧锁不可用")
    void testLockAfterSessionExpired() throws Exception {
        InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");
        lock.acquire();

        // 停止 ZooKeeper 模拟网络中断
        testingServer.stop();

        // 等待会话过期（Session Timeout 3 秒 + 缓冲）
        Thread.sleep(5000);

        // 重启 ZooKeeper
        testingServer.restart();

        // 等待重连
        Thread.sleep(3000);

        // 旧的锁对象可能已失效
        // 验证：尝试释放旧锁
        assertThrows(Exception.class, () -> {
            lock.release();
        });
    }

    @Test
    @DisplayName("会话过期后需要重新获取锁")
    void testReacquireAfterSessionExpired() throws Exception {
        InterProcessMutex lock = new InterProcessMutex(client, "/test-lock");
        lock.acquire();

        // 停止 ZooKeeper 服务
        testingServer.stop();
        Thread.sleep(5000);
        testingServer.restart();
        Thread.sleep(3000);

        // 由于会话过期，锁已自动释放，可以重新获取
        // 但旧锁对象可能报错，需要用新的客户端
        try {
            lock.release();
        } catch (Exception ignored) {
        }

        // 用新客户端获取锁
        CuratorFramework newClient = CuratorFrameworkFactory.builder()
                .connectString(testingServer.getConnectString())
                .sessionTimeoutMs(5000)
                .retryPolicy(new ExponentialBackoffRetry(500, 3))
                .build();
        newClient.start();

        InterProcessMutex newLock = new InterProcessMutex(newClient, "/test-lock");
        assertTrue(newLock.acquire(5, TimeUnit.SECONDS));
        System.out.println("会话过期后重新获取锁成功!");
        newLock.release();
        newClient.close();
    }
}
```

#### 步骤 3：Watcher 测试 + NodeCache 测试

创建 `WatcherTest.java`：

```java
package com.zkdemo;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.framework.recipes.cache.ChildData;
import org.apache.curator.framework.recipes.cache.NodeCache;
import org.apache.curator.framework.recipes.cache.PathChildrenCache;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.curator.test.TestingServer;
import org.awaitility.Awaitility;
import org.junit.jupiter.api.*;

import java.util.concurrent.TimeUnit;
import java.util.concurrent.atomic.AtomicBoolean;
import java.util.concurrent.atomic.AtomicInteger;

import static org.junit.jupiter.api.Assertions.*;

class WatcherTest {
    private TestingServer testingServer;
    private CuratorFramework client;

    @BeforeEach
    void setUp() throws Exception {
        testingServer = new TestingServer();
        client = CuratorFrameworkFactory.builder()
                .connectString(testingServer.getConnectString())
                .retryPolicy(new ExponentialBackoffRetry(500, 3))
                .build();
        client.start();
        client.create().forPath("/test", "initial".getBytes());
    }

    @AfterEach
    void tearDown() throws Exception {
        client.close();
        testingServer.close();
    }

    @Test
    @DisplayName("NodeCache 监听数据变化")
    void testNodeCache() throws Exception {
        NodeCache cache = new NodeCache(client, "/test");
        AtomicBoolean changed = new AtomicBoolean(false);

        cache.getListenable().addListener(() -> {
            ChildData data = cache.getCurrentData();
            if (data != null && "updated".equals(new String(data.getData()))) {
                changed.set(true);
            }
        });
        cache.start();

        // 修改数据
        client.setData().forPath("/test", "updated".getBytes());

        // 使用 Awaitility 等待异步通知
        Awaitility.await()
                .atMost(5, TimeUnit.SECONDS)
                .untilTrue(changed);

        assertTrue(changed.get(), "NodeCache 应检测到数据变化");
        cache.close();
    }

    @Test
    @DisplayName("PathChildrenCache 监听子节点变化")
    void testPathChildrenCache() throws Exception {
        PathChildrenCache cache = new PathChildrenCache(client, "/test", true);
        AtomicInteger addCount = new AtomicInteger(0);

        cache.getListenable().addListener((curator, event) -> {
            if (event.getType() == PathChildrenCacheEvent.Type.CHILD_ADDED) {
                addCount.incrementAndGet();
            }
        });
        cache.start();

        // 创建子节点
        client.create().forPath("/test/child1", "data1".getBytes());
        client.create().forPath("/test/child2", "data2".getBytes());

        // 等待异步通知
        Awaitility.await()
                .atMost(5, TimeUnit.SECONDS)
                .until(() -> addCount.get() >= 2);

        assertEquals(2, addCount.get(), "应收到两次 CHILD_ADDED 事件");
        cache.close();
    }
}
```

#### 步骤 4：多节点集群测试

创建 `ClusterTest.java`：

```java
package com.zkdemo;

import org.apache.curator.framework.CuratorFramework;
import org.apache.curator.framework.CuratorFrameworkFactory;
import org.apache.curator.retry.ExponentialBackoffRetry;
import org.apache.curator.test.TestingCluster;
import org.apache.curator.test.TestingServer;
import org.junit.jupiter.api.*;

import static org.junit.jupiter.api.Assertions.*;

class ClusterTest {
    private TestingCluster cluster;

    @BeforeEach
    void setUp() throws Exception {
        cluster = new TestingCluster(3);
        cluster.start();
    }

    @AfterEach
    void tearDown() throws Exception {
        cluster.close();
    }

    @Test
    @DisplayName("3节点集群启动后应该有1个Leader和2个Follower")
    void testClusterHasLeader() throws Exception {
        int leaderCount = 0;
        for (TestingServer server : cluster.getServers()) {
            if (server.getInstanceSpec().getServerState().equals("LEADING")) {
                leaderCount++;
            }
        }
        assertEquals(1, leaderCount, "3节点集群应有 1 个 Leader");
    }

    @Test
    @DisplayName("写入数据后所有节点都可读取")
    void testDataReplication() throws Exception {
        // 连接到第一个节点
        CuratorFramework client1 = CuratorFrameworkFactory.builder()
                .connectString(cluster.getConnectString())
                .retryPolicy(new ExponentialBackoffRetry(500, 3))
                .build();
        client1.start();
        client1.create().forPath("/cluster-test", "replicated".getBytes());

        // 用第二个连接串验证（验证不同节点都能读到）
        String connectString2 = cluster.getServers().get(1).getConnectString();
        CuratorFramework client2 = CuratorFrameworkFactory.builder()
                .connectString(connectString2)
                .retryPolicy(new ExponentialBackoffRetry(500, 3))
                .build();
        client2.start();

        byte[] data = client2.getData().forPath("/cluster-test");
        assertEquals("replicated", new String(data), "数据应在所有节点复制");

        client1.close();
        client2.close();
    }

    @Test
    @DisplayName("Leader 宕机后自动选出新 Leader")
    void testLeaderElectionAfterCrash() throws Exception {
        // 找出当前 Leader
        TestingServer leader = null;
        for (TestingServer server : cluster.getServers()) {
            if (server.getInstanceSpec().getServerState().equals("LEADING")) {
                leader = server;
                break;
            }
        }
        assertNotNull(leader, "应有 Leader");

        // Kill Leader
        leader.stop();

        // 等待选举
        Thread.sleep(3000);

        // 验证新 Leader 被选出
        boolean newLeaderElected = false;
        for (TestingServer server : cluster.getServers()) {
            if (server.getInstanceSpec().getServerState().equals("LEADING")) {
                newLeaderElected = true;
                break;
            }
        }
        assertTrue(newLeaderElected, "Leader 宕机后应自动选出新 Leader");
    }
}
```

### 可能遇到的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| `Address already in use` | TestingServer 端口冲突 | 不用手动指定端口，用随机端口 |
| 测试间数据残留 | 测试用例共享同一个 client | `@AfterEach` 删除测试数据或重新创建 client |
| 异步操作测试失败 | Curator 操作是异步的 | 使用 `Awaitility` 等待异步结果 |
| 测试超时 | TestingServer 启动慢 | 增大 `@Timeout` 或预先 `start()` |
| 多线程测试数据竞争 | 锁测试中的共享变量 | 使用 `AtomicInteger` / `CountDownLatch` |

### 测试运行

```bash
# 运行所有测试
mvn test

# 运行单个测试类
mvn test -Dtest=DistributedLockTest

# 运行单个测试方法
mvn test -Dtest=DistributedLockTest#testConcurrentLock
```

## 4. 项目总结

### 优点 & 缺点

| 工具 | 优点 | 缺点 |
|------|------|------|
| TestingServer | 轻量级、启动快、随机端口 | 单节点，无法测试集群特性 |
| TestingCluster | 多节点、可模拟Leader选举 | 启动慢、资源消耗大 |
| Awaitility | 优雅等待异步结果 | 增加测试时间 |
| curator-test | 无外部依赖 | 与特定 Curator 版本绑定 |

### 适用场景

- TestingServer：单机 API 测试、分布式锁测试、Watcher 测试
- TestingCluster：集群选举测试、ZAB 协议测试、故障转移测试

### 注意事项

- TestingServer 默认端口随机，避免与本地运行的 ZooKeeper 冲突
- 测试用例之间使用独立的 namespace 或路径隔离数据
- 异步操作（NodeCache、PathChildrenCache）必须等待初始化完成再触发变更
- 多线程测试建议设置合理的超时，避免 CI 卡死

### 常见踩坑经验

**故障 1：TestingServer 关闭后端口未释放**

现象：连续运行多个测试类，第二个测试类报 `Address already in use`。

根因：TestingServer 没有正确关闭。在 `@AfterAll` 或 `@AfterEach` 中必须调用 `close()`。

**故障 2：PathChildrenCache 在 `start()` 之前注册 Listener 丢失事件**

现象：子节点在 `start()` 前创建，但 Listener 没有收到通知。

根因：`start()` 启动后才开始监听。正确的顺序是：先 `start()`，等待初始化完成，再创建子节点。

### 思考题

1. 在 `testConcurrentLock` 中，我们通过 `maxConcurrent` 确认了锁的互斥性。如果不用分布式锁，还能用什么方式验证锁的互斥性？
2. TestingCluster 在网络分区测试中有限制：所有节点在同一台机器上，网络延迟为零。如何模拟真实的"网络延迟"和"分区"场景？

### 推广计划提示

- **开发**：所有 ZooKeeper 相关的代码都必须有对应的单元测试，TestingServer 让这一步变得简单
- **测试**：集成测试中至少覆盖：CRUD 操作、Watcher 触发、分布式锁竞争选主、会话过期恢复
- **CI/CD**：在 CI 流水线中使用 TestingServer，无需单独部署 ZooKeeper 服务
