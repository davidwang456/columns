# GitLab 实战修炼专栏大纲

> **版本**：GitLab 17.x (CE/EE)
> **面向人群**：开发、运维、测试、架构师
> **总章节**：40 章（基础篇 16 章 / 中级篇 15 章 / 高级篇 9 章）
> **每章独立成文件，字数 3000-5000 字**

---

## 专栏定位

以 GitLab 实战为主线，从核心概念到架构部署，从 CI/CD 流水线到源码扩展，从单机运维到大规模 SRE 落地，全链路贯通。每一章采用「业务痛点 → 三人剧本对话 → 代码/配置实战 → 总结思考」四段式结构，兼顾趣味性、实战性与深度。

---

| 角色 | 建议阅读顺序 | 重点章节 |
|------|-------------|---------|

# 基础篇（第 1-16 章）

> **核心目标**：建立 GitLab 核心概念，掌握安装部署、日常操作与基础 CI/CD 流水线编写。
> **面向角色**：新人开发、测试工程师。

---

## 第1章：GitLab 术语全景与架构原理
**定位**：专栏总览与开篇，建立统一语系。
**核心内容**：
- 术语词典：Git、Repository、Branch、Commit、Merge Request、Pipeline、Job、Runner、Issue、Milestone、Registry、Webhook
- GitLab 整体架构图解：Rails 单体 + Gitaly + Sidekiq + PostgreSQL + Redis + Nginx 协同
- 工作流全景：代码提交 → MR 审查 → CI 构建 → 制品归档 → 自动部署
- 组件职责：GitLab Rails（业务逻辑）、Gitaly（Git RPC）、Sidekiq（异步任务）、PostgreSQL（元数据）、Redis（缓存/队列）
**实战目标**：绘制一张 GitLab 整体架构图，并标注各组件间的数据流向。

## 第2章：GitLab 安装与初始化配置
**定位**：从零搭建一个可用的 GitLab 实例。
**核心内容**：
- 安装方式对比：Omnibus 包、Docker 部署、Kubernetes Helm Chart
- Docker 单机部署实战：docker-compose.yml 编写，端口映射，卷挂载
- 初始化配置：gitlab.rb 关键参数（external_url、SMTP、时区、备份路径）
- 首次登录与 root 密码重置
**实战目标**：用 Docker Compose 在本地 5 分钟内启动一个功能完整的 GitLab CE。

## 第3章：Git 核心操作与 GitLab 工作流
**定位**：扎实的 Git 基本功是 GitLab 的入场券。
**核心内容**：
- Git 对象模型：blob、tree、commit、tag 的底层存储
- 核心操作：clone、add、commit、push、pull、fetch、merge、rebase
- GitLab 工作流模型：Git Flow、GitHub Flow、GitLab Flow 的对比
- .gitignore 与 .gitattributes 的最佳实践
**实战目标**：模拟一个 3 人团队，使用 GitLab Flow 完成一次完整的特性开发与合并。

## 第4章：项目创建与仓库管理
**定位**：掌握 GitLab 项目管理的基础能力。
**核心内容**：
- 项目创建：空白项目、模板项目、导入（GitHub/Bitbucket/URL）
- 仓库设置：默认分支、推送规则、存储路径
- 项目可见性：Private / Internal / Public 的差异与安全边界
- 项目成员与权限体系：Guest / Reporter / Developer / Maintainer / Owner
**实战目标**：创建一个团队项目，配置不同角色的权限，验证各角色操作边界。

## 第5章：分支策略与保护规则
**定位**：分支是团队协作的第一道防线。
**核心内容**：
- 分支策略：main / develop / feature / release / hotfix 的分工
- 保护分支：Allowed to merge、Allowed to push、Code owner approval
- 分支命名规范与自动化清理（Delete merged branches）
- 分支与 Git 钩子（pre-receive hook）的协同
**实战目标**：为生产仓库配置 main 分支保护+强制审批+线性历史，禁止 force push。

## 第6章：Merge Request 基础——代码审查入门
**定位**：Code Review 是 GitLab 的灵魂。
**核心内容**：
- MR 生命周期：Draft → Ready → Review → Approve → Merge
- MR 创建方式：UI、命令行 git push -o、GitLab CLI（glab）
- MR 视图：Changes 选项卡、行内评论、Unresolved threads 阻塞合并
- 合并模式：Merge Commit、Squash and Merge、Rebase and Merge 的差异与选择
**实战目标**：发起一个包含 3 个 commit 的 MR，经历 review → 评论 → 修改 → squash merge 全流程。

## 第7章：Issue 与项目管理
**定位**：从代码仓库到项目管理的跃迁。
**核心内容**：
- Issue 完整功能：标题、描述（Markdown）、标签、指派、里程碑、权重、截止日期
- Issue Board：看板视图、列配置、泳道（按标签/指派人分组）
- Epic（EE）：多项目 Issue 聚合管理与路线图
- Quick Actions：`/assign`、`/label`、`/estimate`、`/spend` 等快捷命令
**实战目标**：为一个虚拟电商项目创建 5 个 Issue + 1 个 Milestone + 看板，模拟一次 Sprint 排期。

## 第8章：Wiki 与文档协作
**定位**：知识沉淀是团队效率的基石。
**核心内容**：
- Wiki 仓库原理：独立 Git 仓库，支持克隆与离线编辑
- Markdown 增强：Mermaid 图表、PlantUML、数学公式、Task List
- 页面组织：目录结构、侧边栏、自定义标题页
- 与 Pages 的协同：静态站点文档发布
**实战目标**：搭建团队技术 Wiki，包含架构图（Mermaid）、接口文档、新人入职指南。

## 第9章：.gitlab-ci.yml 语法基础
**定位**：CI/CD 的敲门砖——一切从配置文件开始。
**核心内容**：
- 配置文件入口：.gitlab-ci.yml 的存放位置与生效规则
- 核心关键字：stages、job、script、image、before_script、after_script
- 变量体系：variables、job 变量、触发变量、预定义变量的优先级
- 条件执行：rules、only/except、when（manual/delayed/on_success）
- CI Lint：流水线语法在线校验
**实战目标**：编写一个 3 阶段流水线（build → test → deploy），包含一个手动触发步骤。

## 第10章：GitLab Runner 注册与管理
**定位**：Runner 是 CI/CD 的执行引擎。
**核心内容**：
- Runner 类型：Shared Runner、Group Runner、Project Runner 的选择策略
- Runner 安装：Docker、Linux/Shell、Kubernetes 三种方式
- Runner 注册：gitlab-runner register 的交互流程与参数
- Executor 选型：Shell vs Docker vs Docker Machine vs Kubernetes 的对比
- 并发控制：concurrent 与 limit 参数调优
**实战目标**：注册一个 Docker Executor Runner，并运行第一个 CI 任务。

## 第11章：Pipeline 流水线可视化与调试
**定位**：看懂流水线运行状态，快速定位失败原因。
**核心内容**：
- Pipeline 视图：流水线图、DAG 依赖图、阶段耗时统计
- Job 日志分析：日志折叠（collapsible sections）、ANSI 颜色、日志流式输出
- 调试技巧：`CI_DEBUG_TRACE=true`、交互式 Web Terminal、SSH 到 Runner
- 流水线状态徽章：在 README 嵌入 build/pipeline 状态
**实战目标**：排查一个因缺少依赖而失败的 job，通过 interactive terminal 进入容器修复。

## 第12章：Artifacts 制品与 Cache 缓存
**定位**：理解 CI/CD 中数据持久化的两种方式。
**核心内容**：
- Artifacts：跨阶段/跨 Pipeline 传递构建产物，配置语法（paths、expire_in、when）
- Cache：加速依赖下载，配置语法（key、paths、policy）
- Artifacts vs Cache 的本质区别：语义、生命周期、存储位置对比表
- 最佳实践：node_modules / Maven 仓库 / Python venv 如何配置缓存
**实战目标**：为一个 Node.js 项目配置合理的 cache（npm）与 artifacts（build/dist），对比有无缓存时的流水线耗时。

## 第13章：环境变量与 Secrets 管理
**定位**：安全地管理流水线中的敏感信息。
**核心内容**：
- 变量层级：Project → Group → Instance → 触发变量，优先级规则
- 变量类型：Variable（普通）、File（文件注入）、Masked（脱敏）、Protected（仅保护分支/标签可用）
- CI/CD Variables vs.gitlab-ci.yml variables 的使用边界
- 手动触发变量：流水线触发器传入自定义参数
**实战目标**：为一组微服务配置 Docker Registry 认证、数据库密码等敏感变量，保护生产分支密钥。

## 第14章：容器镜像仓库基础
**定位**：CI/CD 的产物——容器镜像的存储与分发。
**核心内容**：
- GitLab Container Registry 架构：独立组件 vs 内嵌集成
- 镜像操作：docker login、docker push、docker pull 与 Registry API
- 镜像标签策略：版本号、分支名、commit SHA、latest 的利弊
- 自动清理：cleanup policy（按时间、按数量、按标签正则）
**实战目标**：在 CI 流水线中自动构建 Docker 镜像并推送到 GitLab Registry，配置 30 天自动清理策略。

## 第15章：GitLab 日常运维基础
**定位**：从能用到稳用——日常运维必知必会。
**核心内容**：
- 常用运维命令：gitlab-ctl status/restart/tail/reconfigure
- 日志识别：production.log、gitaly.log、sidekiq.log 的定位
- 常见问题排查：502（GitLab 无响应）、Runner 离线、磁盘满、内存耗尽
- 健康检查：`/health_check` 端点、`gitlab-rake gitlab:check`
**实战目标**：模拟 3 种生产故障（502 错误、Runner 不调度、仓库推送超时），输出排查 SOP。

## 第16章：【基础篇综合实战】搭建团队 DevOps 工作流
**定位**：融会贯通基础篇全部知识。
**核心内容**：
- 场景：为一个 10 人创业团队搭建从代码提交到自动部署的全流程
- 需求：GitLab 仓库 + Issue Board + MR Review + CI 构建 + 镜像推送到 Registry + 自动部署
- 技术栈：GitLab CI + Docker + Node.js + Docker Compose 部署
- 验收标准：代码 push → 自动构建 → 自动测试 → 构建镜像 → 部署成功，全过程 < 5 分钟
