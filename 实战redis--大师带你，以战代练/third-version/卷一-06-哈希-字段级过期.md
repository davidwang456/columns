## 哈希：对象表 + 每列自己的「更漏」

**大师**：以前 Hash 像一张表，整张表共用一个 **key TTL**。如今每列（field）可自带闹钟——**token 一刻钟，user_id 永久**，不必拆成十个 key。

**小白**：闹钟响了列没了，`HGET` 空？

**大师**：正是语义。业务要区分「从未设置」与「过期删除」，有时需 **版本号或哨兵字段**。

---

## 命令族（名称以 `HELP @hash` 为准）

- 设过期：`HEXPIRE` / `HPEXPIRE` / `HEXPIREAT` / `HPEXPIREAT`  
- 读写顺带过期：`HGETEX`、`HSETEX`（注意 **`HSETEX`：过期选项在 `FIELDS` 块之前**）  
- 查询 TTL：`HTTL` / `HPTTL`；绝对时间：`HEXPIRETIME` / `HPEXPIRETIME`  
- 清过期：`HPERSIST`（以文档为准）

---

## 最小示例

```text
HSET session:u1 user_id 42
HSETEX session:u1 PX 900000 FIELDS 1 token "abc"
HTTL session:u1 FIELDS 1 token
```
