## 卷一 · 破剑式新编：一剑仍是缓存与计数

**大师**：剑招未改：**`SET`/`GET`/`INCR`/`MSET`**。改的是对手——从前只有页面片段，如今还有 **序列化对象、令牌、分布式锁的 value**。

**小白**：锁也算字符串？

**大师**：锁的 key/value 仍是字节串；**语义在业务**，Redis 只保证原子写入与过期。

---

## 实战三板斧

1. **缓存**：`SET key value EX ttl`；注意大 value 拖垮网络与 RDB。  
2. **计数**：`INCR` / `INCRBYFLOAT`；热点 key 结合 [卷七-25](卷七-25-可观测与HOTKEYS.md)。  
3. **Bit 入口**：`SETBIT` / `GETBIT`（详见 [卷一-10](卷一-10-位图与位域.md)）。
