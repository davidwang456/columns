## Redis Query Engine：当 Redis 像「能查的文档库」

**大师**：在 **Hash/JSON** 上建**二级索引**，再做**全文、向量（在文档叙事中）、地理、聚合**——这是 **Query Engine** 的路线。**不是**替代 `GET`/`HGET` 的简单缓存，而是**查询引擎**。

**小白**：和 **Vector set** 是一回事吗？

**大师**：**不是同一条兵器**。Vector set 是**原生类型 + `VADD`/`VSIM`**；Query Engine 是**另一套索引与查询语言**，常与 JSON 文档搭配。**两篇文分开读，分开试**。
