## List + Stream 混合战：何时不必请 Kafka

**大师**：**小中台**常见套路：**Stream** 做业务事件与审计；**List** 做**简单工作队列**或**削峰缓冲**；**Pub/Sub** 推配置。Kafka 不是勋章，**过重也是债**。

**小白**：我们 QPS 五千，能上吗？

**大师**：看**持久化配置、副本延迟、热点 key、内存上限**——五千可以是「轻松」，也可以是「作死」。**压测比口水准**。

## 一种务实切法

| 数据 | 载体 |
|------|------|
| 订单状态机事件 | Stream + 消费组 |
| 瞬时通知 | Pub/Sub |
| 夜间批任务队列 | List + BLPOP |
| 重试必须幂等的写入 | Stream + **IDMP**（8.6） |
