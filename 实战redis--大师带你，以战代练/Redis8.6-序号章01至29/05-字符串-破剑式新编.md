# 第 05 章 · 字符串：破剑式新编

> 标题里的「破剑式」只是比喻：**字符串（String）**是 Redis 里最常用、也最容易被用滥的类型；读完不必知道任何武侠出处。

本故事纯属虚构，如有雷同，纯属巧合。

> **版本说明**：String 命令多为 1.x–5.x 即存在；**8.6** 对短字符串 `GET` 等路径有延迟优化官宣，线上以压测为准。

## 本话目标

- 会用 **`SET`/`GET`/`INCR`** 解决缓存与计数，并说出 **TTL 与大 value** 的代价。  
- 知道 **分布式锁**在 Redis 里只占一小段「字符串语义」，边界在业务。

## 步步引导：先别一把梭 `SET`

**小白**：缓存不就是 `SET key 大 JSON` 吗？

**大师**：能跑。但你先回答：**过期时间呢？** 多大算大？**穿透**了怎么办？

**小白**：那我永不过期，业务删……

**大师**：可以，直到**磁盘、网络、RDB**一起教你做人。默认姿势是 **`EX`/`PX` 显式 TTL**，或配合淘汰策略（见 [24-淘汰策略-LRU到LRM.md](24-淘汰策略-LRU到LRM.md)）。

**小白**：热点接口我 `INCR` 计数，全打在一个 key 上？

**大师**：先问 QPS。单 key 可能是**热 key**，要结合分片或本地缓存（见 [25-可观测与HOTKEYS.md](25-可观测与HOTKEYS.md)）。

**小白**：分布式锁我就 `SETNX`？

**大师**：可以入门，但要谈清：**过期、续期、主从切换、token**——Redis 只保证**原子写入**，不保证你业务叫「锁」就万事大吉。

## 小剧场：胖客人压塌楼梯

店小二 Redis 只管门牌（key）上写啥；你塞半兆序列化 blob，等于**胖客人走木楼梯**——网络先喘，持久化再咳。

---

## 一剑仍是缓存与计数

**大师**：剑招未改：**`SET`/`GET`/`INCR`/`MSET`**。改的是对手——页面片段、序列化对象、令牌、限流计数、锁的 value。**剑意**在变。

**小白**：锁也算字符串？

**大师**：锁的 key/value 仍是字节串；**语义在业务**，Redis 只保证原子写入与过期。

---

## 实战三板斧

### 1. 缓存

```text
SET page:/home "<html>...</html>" EX 300
GET page:/home
```

要点：

- **`EX`/`PX` 显式过期**，别靠「永不过期 + 手工删」当常态。  
- 大 value 拖垮网络与 RDB/AOF；**压缩、拆 key、或换 Hash/JSON** 常更划算。

### 2. 计数与限流辅助

```text
INCR api:quota:user:1001
INCRBYFLOAT metrics:tps 0.5
```

热点计数 key 结合 [25-可观测与HOTKEYS.md](25-可观测与HOTKEYS.md) 与业务侧分片（按用户尾号拆 key）。

### 3. Bit 入口

`SETBIT` / `GETBIT` 仍在 String 上操作——详见 [10-位图与位域.md](10-位图与位域.md)。

## 进阶几式（面试常问）

| 命令/模式 | 用处 |
|-----------|------|
| `SET key value NX EX ttl` | 简易分布式锁（注意续期与 fencing token 争议） |
| `MSET` / `MGET` | 减少 RTT；注意原子语义与 key 数量 |
| `GETSET` | 旧值返回 + 新值写入，适合简单状态轮换 |
| `APPEND` / `SETRANGE` | 日志型拼接；小心无限增长 |
| `STRLEN` | 快速判断 blob 是否异常膨胀 |

## 动手试一试（仅练习库）

> 在**可清空**的实例执行；若有 `requirepass` 先 `AUTH`。

```text
SET demo:cache:page  "<h1>hello</h1>" EX 60
TTL demo:cache:page
GET demo:cache:page
INCR demo:api:calls
INCR demo:api:calls
GET demo:api:calls
OBJECT ENCODING demo:cache:page
DEL demo:cache:page demo:api:calls
```

换一批：设一个 **>512MB 的 value**（仅本机 dare），观察 **`SET` 耗时**与 **`INFO memory`**——以后你就知道「胖客人」有多重。

## 实战锦囊

- Key 名用 **`业务:维度:id`**，例如 `user:1001:profile`，别把所有用户塞进一个巨型 key。  
- 跨语言团队统一 **JSON/MessagePack**，拒绝把语言私有序列化塞进 Redis。  
- 缓存穿透：空值短 TTL、[Bloom](12-概率模块-Bloom与伙伴.md)、或合并回源，三选一要有意识。
