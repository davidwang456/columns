# 第 07 章 · 列表：队列与栈

本故事纯属虚构，如有雷同，纯属巧合。

> **版本说明**：List 为经典类型；与 **Stream** 选型见 [13-Stream-消费组与XADD幂等.md](13-Stream-消费组与XADD幂等.md)、[15-List与Stream混合战.md](15-List与Stream混合战.md)。

## 本话目标

- 用 List 搭一个 **FIFO 队列**（含阻塞读）。  
- 能一句话说清：**何时必须换 Stream**。

## 步步引导：队列不是越「高级」越好

**小白**：我们要上消息队列，直接 List 行不行？

**大师**：先问三个问题：**丢一条要不要紧？要不要重放？要不要消费组 ACK？** 都行 → List；有一项不行 → 想 Stream。

**小白**：我用 `BLPOP`，线程一直卡住……

**大师**：检查**超时**、**连接池**、**事件循环模型**——在异步框架里阻塞 IO 是**常见自杀姿势**。

**小白**：任务处理一半进程挂了？

**大师**：用 **`LMOVE` / processing 列表**模式，把「已取出未完成」留痕；恢复时从 processing 捞。仍要记住：**至少一次**要靠业务幂等。

## 小剧场：纸条与账本

面馆纸条（List）丢了就下一位；银行流水（Stream）丢一条要对账——**别拿纸条办按揭**。

---

## 软鞭抽得快，但要认对手

**大师**：**`LPUSH`/`RPOP`/`BLPOP`** 做队列，简单凌厉，像软鞭——**抽得快、招式少**。对手若是「**要重放、要消费组、要幂等生产、要审计**」，换 **Stream**，别硬鞭断石。

**小白**：List 能当消息队列吗？

**大师**：能，**轻量任务、允许丢、允许人工兜底**时极好。要 **PEL、位点、ACK、可观测的消费滞后**，请换 **Stream**（见 [13-Stream-消费组与XADD幂等.md](13-Stream-消费组与XADD幂等.md)）。

**趣味比方**：List 是**面馆叫号纸条**——简单、快；Stream 是**银行流水账**——能查、能追责、能对账。

---

## 队列与栈（方向记清）

| 模式 | 典型组合 | 语义 |
|------|----------|------|
| FIFO 队列 | `LPUSH` + `RPOP`（或对称） | 先进先出 |
| 栈 | 同端 push/pop | 后进先出 |
| 阻塞消费 | `BLPOP` / `BRPOP` | 等货来再醒，省空转 |

---

## 实战要点

### 阻塞读

```text
BLPOP jobs 30
```

注意：**阻塞连接数**、超时、以及客户端线程模型——别在事件循环里一把梭阻塞调用。

### 可靠一点：`LMOVE` / 经典 BRPOPLPUSH 模式

把任务从 `jobs:pending` 移到 `jobs:processing`，处理完 `LREM` 或 `LPUSH` 到 `jobs:done`。宕机时**处理中列表**可找回。

**大师**：这不是 Kafka，**至少一次**语义要靠业务幂等与去重键；若要写侧防重，看 Stream **IDMP**（[13-Stream-消费组与XADD幂等.md](13-Stream-消费组与XADD幂等.md)）。

### 禁止误用

`LRANGE 0 -1` 拉全表当分页——**热 key + O(N)**，线上会狠狠教育你。

---

## 与 Stream / PubSub 一眼对比

| 需求 | List | Stream | Pub/Sub |
|------|------|--------|---------|
| 简单任务队列 | 优 | 可 | 不宜 |
| 消费组、ACK | 无 | 优 | 无 |
| 持久与回溯 | 弱 | 优 | 无 |
| 广播 | 不宜 | 不宜 | 优 |

---

## 踩坑

- **单消费者瓶颈**：列表过长时考虑分片多个 key。  
- **死信**：失败任务要有 `failed` 列表或超时回滚策略。  
- **集群**：多 key 操作需同槽 hash tag（[27-集群与高可用.md](27-集群与高可用.md)）。

## 动手试一试

```text
LPUSH demo:jobs "send-email:1"
LPUSH demo:jobs "send-email:2"
RPOP demo:jobs
LLEN demo:jobs
-- 另开终端：BLPOP demo:jobs 5
```

加练：连续 `RPOP` 直到空，体会 **与 BLPOP 的差异**（CPU 空转 vs 阻塞等待）。

## 实战锦囊

- 生产队列要有 **最大长度**或**监控 `LLEN`**，防止爆内存。  
- 失败任务进 **`jobs:failed`**，别默默丢。  
- 多 key 原子迁移在集群要 **hash tag**（[27-集群与高可用.md](27-集群与高可用.md)）。
