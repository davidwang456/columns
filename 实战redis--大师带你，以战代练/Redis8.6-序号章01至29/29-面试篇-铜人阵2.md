## 一、基础必会（零基础也要答得出口）

### Redis 是什么？典型场景？

**答**：开源**内存数据结构存储**，常用作**缓存、会话、排行榜、消息流、分布式锁（需谨慎设计）**等。特点是**快**、**丰富数据类型**、**可选持久化与高可用**。

### 和 Memcached 比？

**答**：Redis **类型多**（Hash、ZSet、Stream…）、**持久化与复制**成熟、**单线程执行命令**（网络与 IO 多线程版本演进以官方为准）。选型看**功能需求与团队熟悉度**。

### 持久化 RDB vs AOF？

**答**：**RDB** 快照，恢复快，可能丢两次快照间数据；**AOF** 日志，更耐丢，体积与 `fsync` 策略要权衡。详见 [23-持久化与副本.md](23-持久化与副本.md)。

### 缓存三大问题？

**答**：**穿透**（查不存在）、**击穿**（热点过期）、**雪崩**（大量同时过期）。手段：布隆过滤、互斥重建、随机 TTL、限流、多级缓存等。

### 主从复制与一致性？

**答**：默认**异步复制**；主写入成功不等于所有从已同步。故障切换可能丢**最后一小段**；业务要**重试 + 幂等**。`WAIT` 可提高副本确认比例，换延迟。

### 事务、Pipeline、Lua？

**答**：`MULTI/EXEC` 是**命令打包**，不是关系库 ACID 事务；出错处理要理解 **EXEC 前/后**错误区别。**Pipeline** 减少 RTT，不保证原子。**Lua 脚本**在服务端原子执行，适合复杂逻辑，但要控时长避免阻塞。

### 分布式锁用 Redis 怎么说清楚？

**答**：常用 `SET key token NX EX ttl`；讨论**过期续期、主从切换 fence、Redlock 争议**——关键是**证明你知道边界**，不是背站队。

## 二、Redis 8.x / 8.6 增量（本教程重点）

### 1. Hash 字段过期 vs 整 key TTL

**答**：**7.4+** 字段级过期；**8.0+** `HSETEX`；避免整 key 误杀。复制/持久化语义以发行说明为准；上线前**从库演练**。详见 [06-哈希-字段级过期.md](06-哈希-字段级过期.md)。

**追问**：与 String JSON 双写？——版本号、灰度、回滚、以谁为准。

### 2. `XADD` 幂等（8.6）

**答**：**`IDMP`/`IDMPAUTO`** + **`*` 自动 ID**；减轻**生产端重试导致重复消息**；**消费侧幂等**仍要做（`XACK` 前后都可能重复）。详见 [13-Stream-消费组与XADD幂等.md](13-Stream-消费组与XADD幂等.md)。

### 3. `volatile-lrm` / `allkeys-lrm`

**答**：按**最近修改**思路淘汰，与 LRU/LFU 适用场景不同；**压测与监控驱动**选型。详见 [24-淘汰策略-LRU到LRM.md](24-淘汰策略-LRU到LRM.md)。

### 4. `HOTKEYS`（8.6）

**答**：热点发现；注意**开销**，优先**从库/影子环境**；配合 `LATENCY DOCTOR`、客户端埋点。详见 [25-可观测与HOTKEYS.md](25-可观测与HOTKEYS.md)。

### 5. Vector set 与 RAG

**答**：**嵌入在应用/模型侧生成**，Redis **`VADD`/`VSIM`**；维数与模型版本一致；索引重建与双写策略。详见 [20-VectorSet与RAG迷你管道.md](20-VectorSet与RAG迷你管道.md)、[21-语义缓存与路由.md](21-语义缓存与路由.md)。

### 6. JSON 类型 vs 字符串塞 JSON

**答**：迁移路径、子路径更新、与索引/Query Engine 联动。详见 [18-JSON类型.md](18-JSON类型.md)、[19-RedisQueryEngine导论.md](19-RedisQueryEngine导论.md)。

### 7. `BUILD_WITH_MODULES`

**答**：Bloom 等 README 标 **\*** 的能力需对应构建；`MODULE LIST` 自检；Docker 镜像与云托管可能裁剪。详见 [04-编译秘笈-BUILD与模块对照表.md](04-编译秘笈-BUILD与模块对照表.md)、[12-概率模块-Bloom与伙伴.md](12-概率模块-Bloom与伙伴.md)。

### 8. 文档与二进制不一致

**答**：以 **`INFO server` + `COMMAND INFO`** 为准；坦诚版本差异比硬背文档高分。

## 三、场景题速练（自述提纲即可）
