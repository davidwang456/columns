# 第 20 章 · Vector set 与 RAG 迷你管道

本故事纯属虚构，如有雷同，纯属巧合。

> **版本说明**：**Vector set** 见 `modules/vector-sets/README.md`；Redis **8+** 主线叙事为合并进默认构建，仍以 `redis-cli` 实测为准。

## 本话目标

- 口述 **RAG 四步**：切块 → 嵌入 → 向量检索 → LLM。  
- 牢记：**维数与模型版本**不一致 = 事故。

## 项目背景

**贯穿设定**：**某电商平台**的智能客服、帮助中心，需要「用户问法多变 → 找到最相关文档片段 → 再生成回答」的 **RAG** 链路。

**与本章关系**：文档**切块**后 **嵌入** 为向量，写入 **Vector set**（或等价能力）；提问先做**近邻检索**再调 LLM。**模型与维数**须与入库时一致，否则检索失真。向量检索与 **String 页面缓存** 职责不同，勿混在同一套键设计里凑合。

## 步步引导：向量不是魔法灰

**大师**：第一次做 RAG，你心里「最省事」的做法是什么？

**小白**：把整篇文档变成一个向量，问啥都往里扔。

**大师**：省事在第一步。你再往下想：**一篇里其实有好多段**，用户常问的只是其中一两段——若不分块，**召回**会不会像**大海捞针**？

**小白**：……要切块、还要留出处？

**大师**：正是。向量多半只管「**像不像**」；**准不准**，还要看 **元数据、过滤、重排** 与后面的 **LLM** 怎么接力。

**小白**：我换了个 embedding 模型，应该没事吧？

**大师**：模型一变，**坐标系**往往就变。常见功课是 **重建索引** 或 **双写迁移**——悄悄换，线上容易「**同一库，两种口音**」。

**小白**：向量旁边还要塞原文吗？

**大师**：常见是 **向量在 Redis，原文在 DB/对象存储**，中间用 **doc_id** 牵一根线——**各干擅长的活**。

## 小剧场：气味索引

藏书楼按「气味」找相似段落——**气味**是向量，**讲书**还得靠说书先生（LLM）。

---

## Redis 管近邻，模型管「懂」

**大师**：**嵌入（embedding）在应用或模型服务生成**，Redis **`VADD` 存储、`VSIM` 搜近邻**，可选 **`FILTER`** 做属性裁剪。拼 prompt → LLM → 回答，便是**最小 RAG 管道**：**召回靠向量，生成靠模型**。

**小白**：Redis 是向量数据库吗？

**大师**：叫啥随市场。**工程上**问三件事：**延迟、召回、成本**——能答再贴标签。别为了标签把架构绑死。

**趣味比方**：向量集像**藏书楼的「气味索引」**——「哪几本和这句问话气味相近」；真正「讲书」的还是 LLM 这位说书先生。

---

## 极简命令流（维数请与模型一致）

```text
VADD kb VALUES 3 0.1 0.2 0.3 chunk:intro SETATTR {"doc":"readme"}
VSIM kb VALUES 3 0.11 0.19 0.28 COUNT 5 WITHSCORES
```

**大师**：**维数不一致**是一剑穿心的大错；**模型版本变更**通常要**重建索引**或双写迁移。

---

## RAG 迷你管道（口述要能画）

1. 文档切块 → 嵌入 → `VADD`（带 `doc_id`、版本元数据）。  
2. 用户提问 → 嵌入 → `VSIM` 取 TopK chunk。  
3. 拼上下文 → 调模型 → 返回答案 + 引用来源。  
4. 观测：命中率、空召回率、延迟 P99。

## 动手试一试

若本机支持 `VADD`/`VSIM`：

```text
COMMAND INFO VADD
VADD demo:kb VALUES 3 0.1 0.2 0.3 doc:1
VSIM demo:kb VALUES 3 0.11 0.19 0.29 COUNT 3 WITHSCORES
```

故意把第二行改成 **4 维向量**，看报错信息——**记住这种错**，排障快一半。

## 实战锦囊

- 元数据里写 **`model_version` / `embedding_dim` / `chunk_id`**。  
- 监控 **空召回率** 与 **P99**；阈值勿拍脑袋。  
- 合规：**勿把隐私明文当 chunk 文本**；日志脱敏见 [26-安全-ACL与TLS.md](26-安全-ACL与TLS.md)。
