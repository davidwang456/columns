# 第 09 章 · 有序集合：排行榜与延时

本故事纯属虚构，如有雷同，纯属巧合。

> **版本说明**：**8.6** 官宣 ZSet 延迟与内存优化；业务侧仍以压测与监控为准。

## 本话目标

- 用 ZSet 做一个 **TopN 排行榜** 与 **时间窗限流** 雏形。  
- 理解 **score 相同**时的排序规则，避免与产品文案「打架」。

## 项目背景

**贯穿设定**：**某电商平台**的直播礼物榜、按分值排序的榜单，以及「到点才执行」的券或任务，都需要**带分数的有序集合**。

**与本章关系**：**ZSet** 用 `ZREVRANGE` 取 TopN；用**时间戳作 score** 可做延时/到期拉取（与消费配合时注意并发语义）。**与 Geo、向量检索区分**：ZSet 这里是**业务分数或时间序**，不是地球球面距离，也不是语义相似度。

## 步步引导：分数不是随便 float

**大师**：要做排行榜，你打算把什么放进 **score**？

**小白**：时间戳？分数越高越新……

**大师**：可以，很自然。你再确认一件小事：**团队说的是毫秒表还是秒表**？两种都能用，**混用**才会出现「穿越」。

**小白**：延时队列也想用 ZSet，到点再拉。

**大师**：思路通。你再想消费者这一侧：**两个人同时看见同一条**，在业务上允不允许？若不允许，就要设计 **取出与删除**如何**同一步**完成（Lua/事务常出场）。

**小白**：我把整段 JSON 塞进 member，省事。

**大师**：省事在写，费事在**传与存**。常见做法是 **member 短、详情回源**——何时值得「胖 member」，用**体积与 QPS**量一量再定。

## 小剧场：并列状元

榜文写「同分看姓名字典序」——上线前跟策划对齐，否则玩家骂娘。

---

## 榜上有名，分秒必争

**大师**：**`ZADD`/`ZRANGE`/`ZREVRANK`**——排行榜、延时队列（score 用时间戳）、滑动窗口限流（配合 `ZREMRANGEBYSCORE`）。**有序集合（ZSet）**一直是这几类的「扛把子」；**8.6** 还有官宣的性能与内存优化，线上仍以压测为准。

**小白**：score 相同谁在前？

**大师**：按 **member 字典序** tie-break；设计业务名次时要心里有数，别上线才发现「同分并列」和策划案不一致。

**趣味比方**：ZSet 像**武林榜**——每人有个「功力值」（score），榜文按功力排；功力一样，按姓名笔画（member）论先后。

---

## 实战片段一：延时任务

```text
ZADD delays 1735689600000 job:1001
ZRANGEBYSCORE delays 0 1735689605000 LIMIT 0 10
```

处理完 **`ZREM`** 或移入工作 Stream（[13-Stream-消费组与XADD幂等.md](13-Stream-消费组与XADD幂等.md)）。多消费者时注意 **Lua 或事务**保证「取出即删」的原子性。

---

## 实战片段二：滑动窗口限流

思路：用 **当前毫秒时间戳** 为 score，`ZADD` 用户请求记录，`ZREMRANGEBYSCORE` 删掉窗口外，`ZCARD` 看窗口内次数——简单有效；超高 QPS 时考虑 **令牌桶在应用层** 或 **分片 key**。

---

## 实战片段三：排行榜分页

```text
ZREVRANGE leaderboard 0 9 WITHSCORES
```

**别**对超大 ZSet 频繁 `ZRANGE 0 -1`。

## 常用命令速查

| 场景 | 命令 |
|------|------|
| 加分 | `ZINCRBY` |
| 名次 | `ZREVRANK` / `ZRANK` |
| 按分数删 | `ZREMRANGEBYSCORE` |
| 限制长度 | `ZPOPMAX` / 定期裁剪 |
| 并集榜 | `ZUNIONSTORE`（注意权重与聚合） |

## 踩坑

- **member 太大**：长字符串 member 浪费内存与网络。  
- **热点榜**：单一 ZSet 被打穿 → 分赛季、分服、或本地缓存 TopN + Redis 全量。见 [25-可观测与HOTKEYS.md](25-可观测与HOTKEYS.md)。  
- **时间源**：延时队列用 **统一时钟**（NTP），别各节点「各自为政」。

## 动手试一试

```text
ZADD demo:board 100 user:alice 200 user:bob 200 user:carol
ZREVRANGE demo:board 0 9 WITHSCORES
ZREVRANK demo:board user:bob
ZINCRBY demo:board 50 user:alice
```

限流雏形（示意）：用 **当前毫秒** 作 score，`ZADD` + `ZREMRANGEBYSCORE` 清窗口外 + `ZCARD`，在练习库跑一遍计数变化。

## 实战锦囊

- 定期 **裁剪**榜长度（`ZREMRANGEBYRANK` 或异步任务），防止无限涨。  
- 热点榜 **分片** 或 **本地缓存 TopN**，见 [25-可观测与HOTKEYS.md](25-可观测与HOTKEYS.md)。  
- **NTP** 同步，延时任务最怕「各节点表不准」。
