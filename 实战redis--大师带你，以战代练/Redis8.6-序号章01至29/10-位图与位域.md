# 第 10 章 · 位图与位域

本故事纯属虚构，如有雷同，纯属巧合。

> **版本说明**：Bitmap 基于 String；**Bitfield** 为独立数据类型能力，见官方文档。

## 本话目标

- 用 Bitmap 记 **「某日是否打卡」**，并解释 **稠密 vs 稀疏**。  
- 知道 **Bitfield** 适合「多段小整数」，不是万能压缩。

## 项目背景

**贯穿设定**：签到、每日限次领权益等，往往是**大量用户的 0/1 或小整数**，且希望省内存。

**与本章关系**：用户 id **稠密**时，**Bitmap** 按日记「是否签到」很省；id **稀疏（如雪花）** 时 offset 可能把位图撑得过长，需改 Set/HLL 等。**Bitfield** 适合把多段小整数打一包。**先看清 id 分布**，再选位图还是别的结构。

## 步步引导：别用位图装空气

**大师**：要记「今天谁来过」，你最先想到的结构？

**小白**：Bitmap！一位一个人，听起来很省。

**大师**：在**位很密**时，常常极省。你再问一句：**人的编号是连续的小整数，还是雪花那样稀疏的大整数**？

**小白**：……雪花。那 Bitmap 还合适吗？

**大师**：稀疏时，卷轴会被**撑得很长**，省字可能变成**贵字**。这时不妨看看 **Set** 或 **HLL**（[11-HyperLogLog-近似去重.md](11-HyperLogLog-近似去重.md)）——**先问分布，再选结构**。

**小白**：权限我也 `SETBIT`，offset 随便定……

**大师**：不妨先**在纸上**写：最大可能是多少号？一位占多少空间？**算完再写代码**，往往比事后查内存曲线省心。

**小白**：Bitfield 和 Hash 字段好像都能塞小整数？

**大师**：都对。你再想：**要不要人类可读的字段名、要不要单独过期**？若答案是「要」，Hash 往往更顺口；若追求**极致紧凑的一串位**，再看 Bitfield。

三只鸟别买万里卷轴；**墨点密**才配得上 Bitmap。

## 日活与 RBAC 的「针尖功」

**大师**：**Bitmap** 用 `SETBIT` 记「某天某用户是否登录」——省内存、怕稀疏。 **Bitfield** 则在同一串里塞**多个有符号/无符号整数位段**，适合 **RBAC 权限位**、小型计数、游戏 buff 标志位。

**小白**：Bitmap 和 Set 谁省？

**大师**：**极稠密**用 Bitmap；**稀疏**用 Set 或 Bloom（模块）。**错选型比错命令更费钱**——面试里常拿这个考选型。

**趣味比方**：Bitmap 像**千里江山图上的墨点**——密密麻麻才划算；若整卷只有三只鸟，你还画满卷轴，那是**败家**。

## Bitmap 迷你战：按日活打卡

```text
SETBIT dau:20260407 10001 1
GETBIT dau:20260407 10001
BITCOUNT dau:20260407
```

跨日对比可用 `BITOP AND/OR` 做「连续登录」粗算。

---

## Bitfield 迷你战：权限位

**大师**：角色 3 位、权限 5 位，能否挤在一个 **Bitfield** 里？能，但要算清**溢出与升级路径**。上线前用 **`BITFIELD` GET** 做演练，别靠心算。

```text
BITFIELD rbac:u1 SET u8 #0 7
BITFIELD rbac:u1 GET u8 #0
```

（具体语法以 `HELP BITFIELD` 与版本文档为准。）

## 选型表

| 数据形态 | 倾向 |
|----------|------|
| 连续用户 ID、按日统计 | Bitmap |
| 稀疏 ID、随机分布 | Set / HLL（[11-HyperLogLog-近似去重.md](11-HyperLogLog-近似去重.md)） |
| 多个小整数标志 | Bitfield |
| 「是否存在」超大规模 | Bloom（[12-概率模块-Bloom与伙伴.md](12-概率模块-Bloom与伙伴.md)） |

## 踩坑

- Bitmap 最大位受 String 最大长度约束；**规划 ID 空间**。  
- **并发更新**同一 Bitfield 字段需理解命令原子边界。  
- 监控 **内存突增**：错误逻辑可能把 offset 打到天文数字。

## 动手试一试

```text
SETBIT demo:dau:day1 1001 1

BITOP AND demo:dau:overlap demo:dau:day1 demo:dau:day2
```

再试 `BITFIELD`（语法以 `HELP` 为准）：给 `demo:bf:u1` 写一个 **u8** 再读回。

## 实战锦囊

- 设计 **ID 空间**：自增内部门禁卡号适合 Bitmap，随机 UUID 不适合。  
- 监控 **STRING 最大长度**与 **内存**，防止异常 offset。  
- 与 [12-概率模块-Bloom与伙伴.md](12-概率模块-Bloom与伙伴.md) 对照：**「在不在」超大规模**常选 Bloom。
