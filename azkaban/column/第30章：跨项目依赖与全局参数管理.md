# 第30章：跨项目依赖与全局参数管理

## 1. 项目背景

### 业务场景

某公司的数据平台按业务域划分了20个Azkaban项目。但实际的数据流转是跨项目的——"用户行为采集"（项目A）完成后，才能触发"用户画像构建"（项目B）和"推荐模型训练"（项目C）。

目前的临时方案是：项目A的最后一个Job里写了个curl命令调项目B的API。这种方式极其脆弱——curl如果超时了怎么办？项目B的API地址变了怎么办？项目A的作者离职了，下游团队不知道这段逻辑的存在。

另外，20个项目中有许多公共配置——HDFS地址、Yarn队列、告警邮箱——每个项目都copy了一份，地址变更时需要改20个地方。

### 痛点放大

跨项目协作不力时：

1. **隐式依赖**：项目A→项目B的触发逻辑隐藏在脚本中，无人知晓
2. **配置漂移**：同一个参数在10个项目中配置了5个不同的值
3. **协调复杂**：一个上游变更需要通知所有下游团队手动同步

## 2. 项目设计——剧本式交锋对话

**小胖**（翻着项目A的源码）：大师，项目B的团队说他们的Flow这两周没跑过，但我们项目A的日志显示每次都调了他们的API啊……

**大师**：看看是不是他们改了API密钥或者URL？

**小胖**（查看后）：果然！他们上周改了API端口，从8081改成8082了。但我们项目A脚本里还是硬编码的老端口。

**大师**：这就是"跨项目隐式依赖"的代价。上下游之间没有明确定义接口，任何一方的变更都可能导致另一方中断。

**小白**：那有什么好的跨项目协作模式？

**大师**：三个层次：

1. **全局参数层**：所有项目共享的参数集中管理
2. **事件通知层**：上游完成后发布事件，下游订阅事件
3. **服务注册层**：项目的输入/输出以API形式暴露

**小胖**：全局参数Azkaban支持吗？

**大师**：Azkaban的`global.properties`是所有项目都能读的。你可以把公共参数放在这里：

```properties
# conf/global.properties
hdfs.namenode=hdfs://prod-nn:8020
yarn.resourcemanager=prod-rm:8032
yarn.default.queue=etl
alert.email=oncall@company.com
hive.metastore.uri=thrift://prod-hms:9083
```

但`global.properties`是系统级的——修改需要重启Web Server。对于更频繁变化的参数，建议用外部配置中心（如Consul、Apollo）。

**小白**：那跨项目触发怎么规范化？

**大师**：我推荐"事件总线"模式。建立一个中心化的触发器服务，上游项目完成后只需发送一个事件，触发服务根据配置决定激活哪些下游项目。

### 技术映射总结

- **global.properties** = 公司公告栏（全员可见的基础设施信息）
- **外部配置中心** = 云端通讯录（集中存储 + 实时同步 + 变更通知）
- **事件总线** = 邮局系统（上游寄信，邮局分发给所有订阅的下游）

#### 步骤2：外部配置中心集成（Consul）

**目标**：使用Consul管理动态配置，避免重启。

```python
#!/usr/bin/env python3
# consul_config_manager.py —— Consul配置管理器

import consul
import json

class ConsulConfigManager:
    """从Consul读取Azkaban配置"""
    
    def __init__(self, consul_host='localhost', consul_port=8500):
        self.consul = consul.Consul(host=consul_host, port=consul_port)
        self.prefix = 'azkaban/config/'
    
    def get_config(self, key, default=None):
        """读取单个配置"""
        _, data = self.consul.kv.get(f"{self.prefix}{key}")
        if data:
            return data['Value'].decode('utf-8')
        return default
    
    def get_all_project_config(self, project_name):
        """读取某个项目的所有配置"""
        _, keys = self.consul.kv.get(f"{self.prefix}{project_name}/", keys=True)
        config = {}
        
        if keys:
            for key in keys:
                short_key = key.replace(f"{self.prefix}{project_name}/", '')
                value = self.get_config(f"{project_name}/{short_key}")
                config[short_key] = value
        
        return config
    
    def set_config(self, key, value):
        """设置配置（通过Web或CLI操作Consul）"""
        self.consul.kv.put(f"{self.prefix}{key}", value)
    
    def watch_config(self, key, callback):
        """监听配置变更"""
        index = None
        while True:
            idx, data = self.consul.kv.get(f"{self.prefix}{key}", index=index)
            if data and idx != index:
                new_value = data['Value'].decode('utf-8')
                callback(key, new_value)
            index = idx

# ===== 在Job中使用 =====

# type=command
# command=bash -c '
# DB_HOST=$(python3 -c "from consul_config_manager import ConsulConfigManager; print(ConsulConfigManager().get_config(\"db.host\"))")
# echo "DB Host from Consul: $DB_HOST"

#### 步骤3：项目依赖注册中心

**目标**：建立跨项目的依赖关系注册表。

```python
#!/usr/bin/env python3
# dependency_registry.py —— 跨项目依赖注册中心

import json
import os
from datetime import datetime

class DependencyRegistry:
    """跨项目依赖关系注册表"""
    
    def __init__(self, registry_file="dependency_registry.json"):
        self.registry_file = registry_file
        self.registry = self._load()
    
    def _load(self):
        if os.path.exists(self.registry_file):
            with open(self.registry_file, 'r') as f:
                return json.load(f)
        return {"dependencies": {}, "global_params": {}}
    
    def _save(self):
        with open(self.registry_file, 'w') as f:
            json.dump(self.registry, f, indent=2)
    
    def register_dependency(self, upstream_project, upstream_flow,
                           downstream_project, downstream_flow,
                           params_mapping=None):
        """注册跨项目依赖"""
        dep_key = f"{upstream_project}/{upstream_flow}"
        
        if dep_key not in self.registry["dependencies"]:
            self.registry["dependencies"][dep_key] = []
        
        self.registry["dependencies"][dep_key].append({
            "downstream_project": downstream_project,
            "downstream_flow": downstream_flow,
            "params_mapping": params_mapping or {},
            "registered_at": datetime.now().isoformat()
        })
        
        self._save()
        print(f"✓ 注册依赖: {dep_key} → {downstream_project}/{downstream_flow}")
    
    def get_downstreams(self, upstream_project, upstream_flow):
        """获取上游完成时需要触发的下游列表"""
        dep_key = f"{upstream_project}/{upstream_flow}"
        return self.registry["dependencies"].get(dep_key, [])
    
    def get_upstreams(self, downstream_project, downstream_flow):
        """获取下游依赖的上游列表"""
        upstreams = []
        for dep_key, downstreams in self.registry["dependencies"].items():
            for ds in downstreams:
                if (ds["downstream_project"] == downstream_project and 
                    ds["downstream_flow"] == downstream_flow):
                    upstreams.append(dep_key)
        return upstreams
    
    def generate_dependency_graph(self):
        """生成Mermaid格式的依赖图"""
        lines = ["```mermaid", "graph LR"]
        
        for dep_key, downstreams in self.registry["dependencies"].items():
            for ds in downstreams:
                up = dep_key.replace('/', '_')
                down = f"{ds['downstream_project']}_{ds['downstream_flow']}"
                lines.append(f"    {up}[{dep_key}] --> {down}[{ds['downstream_project']}/{ds['downstream_flow']}]")
        
        lines.append("```")
        return '\n'.join(lines)

# ===== 使用示例 =====
if __name__ == '__main__':
    reg = DependencyRegistry()
    
    # 注册跨项目依赖
    reg.register_dependency(
        "log_collection", "daily_collect",
        "user_profile", "daily_build",
        params_mapping={"date": "process_date"}
    )
    
    reg.register_dependency(
        "log_collection", "daily_collect",
        "recommendation", "model_training",
        params_mapping={"date": "training_date"}
    )
    
    reg.register_dependency(
        "user_profile", "daily_build",
        "recommendation", "feature_engineering",
        params_mapping={"user_tags_path": "input_path"}
    )
    
    # 查看依赖图
    print(reg.generate_dependency_graph())
```

#### 步骤4：跨项目触发服务

**目标**：基于依赖注册表自动触发下游项目。

```python
#!/usr/bin/env python3
# cross_project_trigger.py —— 跨项目触发器

import time
import json
from azkaban_client_v2 import AzkabanClient
from dependency_registry import DependencyRegistry

class CrossProjectTrigger:
    """跨项目触发器：上游完成→根据注册表→触发下游"""
    
    def __init__(self, azkaban_url, username, password):
        self.client = AzkabanClient(azkaban_url, username, password)
        self.registry = DependencyRegistry()
        self.processed_execs = set()  # 已处理的执行ID（防重复）
    
    def start(self, poll_interval=30):
        """启动监控循环"""
        print("=== Cross-Project Trigger 启动 ===")
        
        while True:
            try:
                self._check_and_trigger()
            except Exception as e:
                print(f"Error: {e}")
            
            time.sleep(poll_interval)
    
    def _check_and_trigger(self):
        """检查上游Flow是否完成并触发下游"""
        
        # 1. 获取最近完成的Flow（简化实现：查询最近100条记录）
        recent = self.client._get("/executor", params={
            "ajax": "fetchexecflowhistory",
            "start": 0,
            "length": 100
        })
        
        for exec_info in recent.get("executions", []):
            exec_id = exec_info.get("execId")
            
            # 跳过已处理的
            if exec_id in self.processed_execs:
                continue
            
            # 只处理成功的Flow
            if exec_info.get("status") != "SUCCEEDED":
                self.processed_execs.add(exec_id)
                continue
            
            project = exec_info.get("projectName")
            flow = exec_info.get("flowId")
            
            # 2. 查找注册的下游依赖
            downstreams = self.registry.get_downstreams(project, flow)
            
            if not downstreams:
                self.processed_execs.add(exec_id)
                continue
            
            # 3. 触发下游Flow
            for ds in downstreams:
                print(f"触发下游: {ds['downstream_project']}/{ds['downstream_flow']}")
                
                # 解析参数映射
                flow_params = {}
                for src_param, dst_param in ds.get("params_mapping", {}).items():
                    # 从上游执行中获取参数值
                    flow_params[dst_param] = exec_info.get("flowParams", {}).get(src_param, "")
                
                try:
                    downstream_exec_id = self.client.execute_flow(
                        ds["downstream_project"],
                        ds["downstream_flow"],
                        flow_params=flow_params
                    )
                    print(f"  ✓ 下游执行ID: {downstream_exec_id}")
                except Exception as e:
                    print(f"  ✗ 触发失败: {e}")
            
            self.processed_execs.add(exec_id)
    
    def manual_trigger_downstreams(self, upstream_project, upstream_flow, source_params=None):
        """手动触发下游"""
        downstreams = self.registry.get_downstreams(upstream_project, upstream_flow)
        
        results = []
        for ds in downstreams:
            flow_params = {}
            if source_params and ds.get("params_mapping"):
                for src, dst in ds["params_mapping"].items():
                    if src in source_params:
                        flow_params[dst] = source_params[src]
            
            exec_id = self.client.execute_flow(
                ds["downstream_project"],
                ds["downstream_flow"],
                flow_params=flow_params
            )
            results.append({
                "project": ds["downstream_project"],
                "flow": ds["downstream_flow"],
                "exec_id": exec_id
            })
        
        return results

if __name__ == '__main__':
    trigger = CrossProjectTrigger("http://localhost:8081", "trigger_bot", "bot_pass")
    trigger.start()
```

#### 步骤5：配置变更审计

**目标**：跟踪全局参数变更历史。

```python
#!/usr/bin/env python3
# config_auditor.py —— 配置变更审计

import json
import hashlib
from datetime import datetime

class ConfigAuditor:
    """配置变更审计器"""
    
    def __init__(self, audit_file="config_audit.log"):
        self.audit_file = audit_file
    
    def record_change(self, key, old_value, new_value, changed_by, reason=""):
        """记录配置变更"""
        entry = {
            "timestamp": datetime.now().isoformat(),
            "key": key,
            "old_value_hash": hashlib.sha256(str(old_value).encode()).hexdigest()[:8],
            "new_value_hash": hashlib.sha256(str(new_value).encode()).hexdigest()[:8],
            "changed_by": changed_by,
            "reason": reason
        }
        
        with open(self.audit_file, 'a') as f:
            f.write(json.dumps(entry, ensure_ascii=False) + '\n')
        
        print(f"📝 配置变更: {key} by {changed_by}")
    
    def get_change_history(self, key=None, limit=50):
        """获取变更历史"""
        history = []
        with open(self.audit_file, 'r') as f:
            for line in f:
                entry = json.loads(line.strip())
                if key is None or entry["key"] == key:
                    history.append(entry)
        
        return history[-limit:]
    
    def check_consistency(self, projects_configs):
        """检查多个项目间配置一致性"""
        inconsistencies = []
        
        # 按配置key分组检查
        all_keys = set()
        for project, config in projects_configs.items():
            all_keys.update(config.keys())
        
        for key in all_keys:
            values = {}
            for project, config in projects_configs.items():
                if key in config:
                    values[project] = config[key]
            
            # 如果同一个key在不同项目中有不同值
            unique_values = set(values.values())
            if len(unique_values) > 1:
                inconsistencies.append({
                    "key": key,
                    "values": values
                })
        
        return inconsistencies
```

## 4. 项目总结

### 配置管理方案对比

| 方案 | 实时性 | 一致性 | 运维复杂度 | 适用阶段 |
|------|--------|--------|----------|---------|
| 硬编码 | — | ★☆☆ | ★☆☆ | 初期 |
| global.properties | 需重启 | ★★☆ | ★★☆ | 中期 |
| 外部配置中心 | 准实时 | ★★★ | ★★★ | 成熟期 |
| 依赖注册中心 | 事件驱动 | ★★★ | ★★★ | 跨团队协作 |

### 适用场景

- **适用**：多项目共享基础设施的大型数据平台、跨团队数据管道、有严格配置管理要求的场景
- **不适用**：单项目的小型团队、独立运作的数据平台

### 注意事项

- `global.properties`变更需重启Web Server才能生效
- 跨项目触发服务本身是高可用瓶颈——需考虑其自身的HA
- 依赖注册表需要与Git仓库同步（基础设施即代码）

### 常见踩坑经验

1. **global.properties键名冲突**：项目B定义了一个`${data.dir}`参数，与global.properties中的同名键冲突，被悄悄覆盖了。解决：项目级参数使用项目名前缀（如`project_b.data.dir`）。
2. **循环依赖**：项目A→B→C→A，触发服务不断循环触发。解决：在触发服务中添加"最大触发深度"限制。
3. **配置漂移检测不及时**：有人绕过配置中心直接改Azkaban配置，导致registry与实际情况不一致。解决：定期用脚本扫描所有项目的参数配置做一致性检查。
