### 开场一分钟（趣味钩子）

本地环境像**驾校教练车**：能学会起步、换挡、看后视镜，但**别用教练车拉货上高速**。培训环境的意义是**可复现**，不是「我笔记本能跑所以生产没问题」。统一 JDK、时区、hosts，比多装两个炫酷插件重要一百倍。

### 1）项目背景

- **开发**：本地或可复现环境调试客户端；避免「只有我机器能连」的隐性配置（`hosts`、Kerberos、`hbase-site.xml`）。
- **运维**：统一公司标准镜像与参数，避免「我机器能跑」；培训 / CI / POC 用同一套**版本矩阵**。
- **测试**：CI 与手工环境对齐，减少环境问题扯皮；缺陷单要能标注**环境指纹**（版本 + 配置哈希）。
- **若跳过本章**：后半程所有实验会在「环境不对」上浪费 50% 时间。

### 2）项目设计（大师 × 小白）

- **小白**：「我内存 8G，能跑全集群吗？」
- **大师**：「学习可以 **单机 / 伪分布式**；生产是多 RS。先保证 **JDK、HDFS、HBase 版本矩阵**与公司一致。」
- **小白**：「能直接用 Docker 吗？」
- **大师**：「可以，但要固定**镜像版本与端口**，文档写清 `hbase-site.xml` 挂载位置。」
- **小白**：「Windows 上能跑吗？」
- **大师**：「常见是 **WSL2 / Linux VM / Docker**；原生 Windows 容易在路径、权限上踩坑，培训建议 Linux 一致。」
- **小白**：「我调了超大 heap 是不是更稳？」
- **大师**：「笔记本上过大 heap 反而 GC 卡顿；**按官方建议 + 本机内存**来。」
- **段子**：小白说「我克隆了同事的容器但还是连不上。」大师：「你是不是连的是**你心里那台** Master？」

### 3）项目实战

**前置**：JDK、SSH（伪分布式）、或 Docker；磁盘至少预留数 GB 给 HDFS / 本地数据目录。

**步骤 1：启动与验活**

1. 按公司文档或官方 [Quick Start](https://hbase.apache.org/book.html#quickstart) 启动 HBase。
2. `hbase shell` 执行：`status 'simple'`（或当前版本等价命令）。
3. 浏览器打开 Master UI（端口以配置为准），截图或记录 **HBase version**、**RegionServer 列表**。

**步骤 2：环境指纹（写入实验记录）**

记录以下字段（培训收作业用）：

| 项 | 值 |
|----|-----|
| OS |  |
| JDK 版本 | `java -version` |
| HBase 版本 | UI 或 `hbase version` |
| `hbase.rootdir` | 来自 `hbase-site.xml` |
| 是否 Docker | 是 / 否，镜像 tag |

**步骤 3：常见故障 15 min 排演（可选）**

- 故意改错 `localhost` / 主机名，观察错误日志关键字；再改回。
- **验收**：能口述「时间不同步」「权限不足」「端口占用」三类各占一条自查命令。

### 4）项目总结

- **优点**：标准化环境降低沟通成本；新人 onboarding 可复制。
- **缺点**：本地与生产差异仍存在（网络、数据量、安全）。
- **适用**：培训、集成测试、POC。
- **注意**：不要将本地调优参数直接拷生产；Kerberos 环境另有一套 checklist。
- **踩坑**：时间不同步导致奇怪故障；`hosts` 不一致导致 RS 互相找不到。
- **运维检查项**：NTP、hosts、防火墙、目录权限；镜像构建可复现。
- **测试检查项**：环境检查清单入用例库；缺陷单模板含环境指纹。

### 5）课堂自测与作业（讲师可选用）

**自测**

1. 为什么培训环境要固定 **JDK 大版本**？
2. `hbase.rootdir` 指向本地与指向 HDFS 各有什么利弊（学习 vs 生产）？
3. 列出启动失败时你会看的**第一个日志路径**（按你环境实际）。

**作业**

- 写半页「团队标准开发环境」：推荐 Docker Compose 或脚本入口、禁止事项（例如不要共用 rootdir）。
