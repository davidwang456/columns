### 开场一分钟（趣味钩子）

Spark 扫 HBase 像**用联合收割机进超市过道**：不是不能进，但要**预约、限宽、限速**，否则购物车（在线请求）全堵门口。记住：**离线作业的「爽」常是在线业务的「痛」**。

### 1）项目背景

- **开发**：数仓、实时计算需从 HBase **批量或流式**读写；需明确 **Catalog、分区、容错、幂等 sink**。
- **运维**：Spark/Flink 作业资源与 HBase **Region 热点、scanner 超时** 相互影响；需要**业务窗口**与告警联动。
- **测试**：空表、部分 Region 不可用、作业失败重试的数据正确性；压测对在线 SLA 的影响评估。
- **若跳过本章**：离线任务上线当天，在线 P99「集体离家出走」。

### 2）项目设计（大师 × 小白）

- **小白**：「Spark 直接扫 HBase 全表行不行？」
- **大师**：「**行**，但要控 **分区并行度、缓存、列裁剪**；否则 RS 和网络先爆。」
- **小白**：「Flink 实时写呢？」
- **大师**：「注意 **幂等、sink 并行度与 RowKey 设计**；结合 **BufferedMutator** 或连接器提供的批量能力。」
- **小白**：「能白天跑全表吗？」
- **大师**：「看 SLA；默认建议**错峰 + 限流 + 只读副本**（若架构有）。」
- **小白**：「checkpoint 失败重试会重复写吗？」
- **大师**：「会；要有 **幂等键 / 去重 / 事务语义（按引擎）**。」
- **段子**：小白把 `spark.default.parallelism` 调成 Region 数乘 10。大师：「RS 正在写遗书第二页。」

### 3）项目实战

1. 阅读官方 [Spark](https://hbase.apache.org/book.html#spark) 与 [External APIs](https://hbase.apache.org/book.html#external_apis) 相关章节；Flink 查阅当前使用的 **Flink-HBase connector** 文档版本。
2. 做一个 **最小读**：从 HBase 读一张表写入文件或控制台（脱敏）；记录 `TableInputFormat` 或 DataSource API 用法。
3. 列出作业参数：`cache`, `batch`, 连接超时、重试、**RowKey 范围剪枝**（若做了要写明）。

**验收**：可复现命令或作业配置片段 + 一页参数说明 + 「对在线影响自评」（低/中/高）。

### 4）项目总结

- **优点**：与大数据栈统一；适合 ETL、特征回填、离线对账。
- **缺点**：批扫对在线集群冲击大；需窗口与限流。
- **适用**：T+1 报表、批量修复、流式同步（配合 Kafka 等）。
- **注意**：离线任务避开业务高峰；优先 **RowKey 范围** 剪枝；监控 RS 读延迟。
- **踩坑**：默认并行度过高打满单 RS；无幂等的 At-least-once sink。
- **运维检查项**：RS handler 队列与读延迟告警；作业白名单与配额。
- **测试检查项**：空表、部分 Region 不可用时的失败语义；重放一致性。
