# 测试用例模板（专栏通用）

供测试工程师在各章落地「可自动化/可手工」用例时复用。每章至少覆盖 **正常、异常、恢复** 三类之一；与交付语义、再均衡、EOS 相关章节应全覆盖。

## 用例元数据

| 字段 | 说明 |
|---|---|
| 章节 | 如：第 3 章 |
| 用例 ID | 如：CH03-INT-001 |
| 前置条件 | Broker 版本、Topic 配置、客户端版本 |
| 数据准备 | Key 分布、消息条数、是否事务 |
| 步骤 | 可执行操作序列 |
| 期望结果 | 断言（日志、指标、业务状态） |
| 清理 | Topic/Group 是否删除、是否影响共享环境 |

## 场景矩阵（勾选）

- [ ] **正常路径**：端到端发送与消费成功。
- [ ] **重复消费**：同 offset 或同业务键重复投递，业务是否幂等。
- [ ] **乱序**：跨分区或重试导致的乱序是否在允许范围内。
- [ ] **断网/超时**：客户端重试后与 Broker 状态一致。
- [ ] **Broker 重启**：Leader 切换、ISR 收缩期间，生产/消费行为符合预期。
- [ ] **再均衡**：扩缩消费者实例时，Lag 与业务错误率可接受。

## 与部门协作

- **开发**：提供可观测点（日志字段、指标名、业务主键）。
- **运维**：提供变更窗口与告警静默策略，避免演练误报。
- **测试**：将可重复部分沉淀为自动化（如容器化集成、Testcontainers，见第 38 章）。
