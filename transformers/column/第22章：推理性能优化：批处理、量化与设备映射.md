# 第22章：推理性能优化：批处理、量化与设备映射

## 1 项目背景

### 业务场景

客服工单分类服务上线两周后，运维团队发现一个严重问题：单条推理耗时 50ms 看似正常，但在高峰期 QPS 达到 30 时，P95 延迟飙到了 2.8 秒。根因分析发现，服务每次只推理一条，GPU 利用率仅 12%——大量时间花在了 CPU→GPU 数据传输和 kernel launch 开销上，而非实际计算。

与此同时，算法团队在测试新的 7B 参数大模型时，发现 FP32 精度下推理需要 28GB 显存，单张 A10（24GB）加载都装不下，必须上 A100（80GB）。如果部署 3 个不同业务模型，仅GPU成本每月就超 5 万元。

技术总监拍板："在不显著降低准确率的前提下，推理速度提升 3 倍，显存占用降低 4 倍。"

### 痛点放大

模型推理的三大性能瓶颈：

1. **小batch低吞吐**：单条推理GPU利用率低（<15%），batch推理可提升3-5倍吞吐
2. **精度浪费**：FP32推理占用大量显存但大多数场景FP16精度足够，int8量化几乎不影响分类准确率
3. **显存碎片**：大模型单卡装不下，需要手动分配不同层到不同设备，代码复杂易出错

```
单条推理: GPU利用率12%, QPS=20,  延迟=50ms
批处理:   GPU利用率85%, QPS=120, 延迟=60ms (batch=8)

FP32: 28GB 显存, 准确率 91.2%
FP16: 14GB 显存, 准确率 91.1% (几乎无损)
INT8:  7GB 显存, 准确率 90.5% (轻微损失)
INT4:  4GB 显存, 准确率 88.7% (损失较大，需评估)
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周三下午 2:00，War Room。运维小王把 GPU 监控大屏投出来，GPU 利用率那条线几乎贴地。

---

**小胖**:"这GPU利用率咋跟我的心电图似的——一条直线？GPU是不是在摸鱼？"

**小陈**:"不是摸鱼，是单条推理太'轻'了。GPU一次能并行计算几千个操作，但我们每次只给它一条文本，GPU大部分时间在等数据从CPU传过来。就像你有一辆能装50人的大巴，但每次都只拉一个人。"

**小白**:"那就把多条请求攒成一个batch再推理——batch_size=8 能提升3-5倍吞吐。但问题是：如果用户请求来得稀疏（每隔200ms来一个），你让谁等这个batch凑齐？用户体验的延迟就会增加。"

**大师**:"这个矛盾需要**动态批处理**来解决。让我把推理优化的三层武器讲清楚。

**第一层：精度降级（量化）。**

| 精度 | 显存占用 (7B模型) | 推理速度 | 准确率影响 |
|------|------------------|---------|-----------|
| FP32 | 28 GB | 基准 | 最优 |
| FP16 | 14 GB | 1.5-2× | 几乎无损 |
| BF16 | 14 GB | 1.5× | 几乎无损 |
| INT8 | 7 GB | 2-3× | 轻微损失 |
| INT4 | 3.5 GB | 3-5× | 需评估 |

量化原理：把32位浮点数压缩到8位或4位整数。关键是**校准**——用量化前跑一小批数据，统计每层的数值范围，再映射到低精度区间。Transformers库通过 `BitsAndBytesConfig` 实现量化加载。

**第二层：设备映射（device_map）。**

`device_map="auto"` 是HuggingFace的自动设备分配器，它分析模型每层的大小，自动分配到GPU和CPU上：
- 能放GPU的层放GPU
- GPU放不下的溢到CPU
- 还能 `disk offload` 溢到硬盘

比如7B模型在24GB卡上：前28层放GPU（22GB），后4层放CPU（2GB）。推理时CPU部分慢一些但整体延迟增加不到20%。

**第三层：批处理策略。**

三种批处理模式：
- **静态批处理**：固定batch_size=8，凑够8条才推理。延迟稳定但需要等。
- **动态批处理**：设置最大等待时间（如50ms），时间到了不管几条都推理。兼顾延迟和吞吐。
- **连续批处理（Continuous Batching）**：vLLM/TGI的方案，请求不必一起开始一起结束——完成的先返回，新请求随时加入。最先进但实现复杂。"

**技术映射总结**：
- 量化 = 把高清电影压成标清，文件更小播放更快，画质损失可接受
- device_map="auto" = 自动搬家，大件放近的房间（GPU），不常用的放远（CPU）
- 动态批处理 = 拼车出行，等50ms，能拼几个拼几个

```python
# batch_benchmark.py
"""批处理性能基准测试"""

import time
import torch
import numpy as np
from transformers import AutoTokenizer, AutoModelForSequenceClassification

MODEL_PATH = "./models/sentiment_minimal"
tokenizer = AutoTokenizer.from_pretrained(MODEL_PATH)
model = AutoModelForSequenceClassification.from_pretrained(MODEL_PATH)
device = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)
model.eval()

# 测试文本
test_texts = ["这是一个测试文本，用来评估推理性能。"] * 100


def benchmark_batch(batch_sizes: list, num_runs: int = 50):
    """对比不同 batch_size 的吞吐和延迟"""
    print(f"\n{'batch_size':<12} {'吞吐(QPS)':<12} {'P50(ms)':<12} {'P95(ms)':<12} {'GPU利用率'}")
    print("-" * 60)

    for bs in batch_sizes:
        latencies = []

        for _ in range(num_runs):
            # 随机取 bs 条文本
            idx = np.random.randint(0, len(test_texts), bs)
            batch = [test_texts[i] for i in idx]

            inputs = tokenizer(batch, truncation=True, max_length=128,
                               padding=True, return_tensors="pt").to(device)

            start = time.time()
            with torch.no_grad():
                _ = model(**inputs)
            latencies.append((time.time() - start) * 1000 / bs)  # 均摊到每条

        latencies = sorted(latencies)
        p50 = latencies[len(latencies)//2]
        p95 = latencies[int(len(latencies)*0.95)]
        qps = 1000 / p50

        # GPU 利用率（简单估算）
        if device == "cuda":
            util = min(100, bs / 16 * 100)  # 粗略估算
        else:
            util = "N/A"

        print(f"{bs:<12} {qps:<12.0f} {p50:<12.1f} {p95:<12.1f} {util}")


if __name__ == "__main__":
    print(f"设备: {device}")
    benchmark_batch([1, 2, 4, 8, 16, 32])
```

### 3.3 量化推理对比

```python
# quantization_compare.py
"""量化精度对比：FP32 vs FP16 vs INT8 vs INT4"""

import torch
import time
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from transformers import BitsAndBytesConfig
import numpy as np


def load_model_with_precision(model_name: str, precision: str):
    """按精度加载模型"""
    if precision == "fp16":
        return AutoModelForSequenceClassification.from_pretrained(
            model_name, torch_dtype=torch.float16, device_map="auto"
        )
    elif precision == "int8":
        bnb_config = BitsAndBytesConfig(
            load_in_8bit=True,
            llm_int8_threshold=6.0,
        )
        return AutoModelForSequenceClassification.from_pretrained(
            model_name, quantization_config=bnb_config, device_map="auto"
        )
    elif precision == "int4":
        bnb_config = BitsAndBytesConfig(
            load_in_4bit=True,
            bnb_4bit_compute_dtype=torch.float16,
            bnb_4bit_quant_type="nf4",
        )
        return AutoModelForSequenceClassification.from_pretrained(
            model_name, quantization_config=bnb_config, device_map="auto"
        )
    else:  # fp32
        return AutoModelForSequenceClassification.from_pretrained(
            model_name, device_map="auto"
        )


def benchmark_precision(model_name: str):
    """对比不同精度下的显存和速度"""
    tokenizer = AutoTokenizer.from_pretrained(model_name)
    texts = ["测试文本内容" * 20] * 8

    precisions = ["fp32", "fp16", "int8", "int4"]
    results = []

    for prec in precisions:
        print(f"\n加载 {prec} 模型...")
        try:
            model = load_model_with_precision(model_name, prec)
            model.eval()

            # 显存估算
            mem_mb = 0
            if torch.cuda.is_available():
                torch.cuda.reset_peak_memory_stats()
                torch.cuda.empty_cache()

            inputs = tokenizer(texts[:2], truncation=True, max_length=128,
                               padding=True, return_tensors="pt")
            if torch.cuda.is_available():
                inputs = {k: v.cuda() for k, v in inputs.items()}

            # 预热
            with torch.no_grad():
                for _ in range(3):
                    _ = model(**inputs)

            # 计时
            times = []
            inputs_full = tokenizer(texts, truncation=True, max_length=128,
                                    padding=True, return_tensors="pt")
            if torch.cuda.is_available():
                inputs_full = {k: v.cuda() for k, v in inputs_full.items()}

            for _ in range(20):
                start = time.time()
                with torch.no_grad():
                    _ = model(**inputs_full)
                times.append((time.time() - start) * 1000)

            avg_time = np.mean(times)

            if torch.cuda.is_available():
                mem_mb = torch.cuda.max_memory_allocated() / 1024**2

            results.append({
                "precision": prec,
                "latency_ms": round(avg_time, 1),
                "gpu_memory_mb": round(mem_mb, 1) if mem_mb else "N/A",
            })

            del model
            torch.cuda.empty_cache()

        except Exception as e:
            print(f"  {prec} 加载失败: {e}")
            results.append({"precision": prec, "latency_ms": "FAIL", "gpu_memory_mb": "FAIL"})

    print(f"\n{'精度':<10} {'延迟(ms)':<12} {'显存(MB)':<12}")
    print("-" * 40)
    for r in results:
        print(f"{r['precision']:<10} {str(r['latency_ms']):<12} {str(r['gpu_memory_mb']):<12}")


if __name__ == "__main__":
    benchmark_precision("prajjwal1/bert-tiny")
```

### 3.5 动态批处理服务

```python
# dynamic_batch_server.py
"""动态批处理推理服务 —— 平衡延迟与吞吐"""

import asyncio
import time
import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from typing import List, Dict
from collections import deque

MODEL_PATH = "./models/sentiment_minimal"

class DynamicBatchInference:
    """动态批处理推理引擎"""

    def __init__(self, model_path: str, max_batch_size: int = 32,
                 max_wait_ms: float = 50.0):
        self.tokenizer = AutoTokenizer.from_pretrained(model_path)
        self.model = AutoModelForSequenceClassification.from_pretrained(model_path)
        self.device = "cuda" if torch.cuda.is_available() else "cpu"
        self.model.to(self.device)
        self.model.eval()

        self.max_batch_size = max_batch_size
        self.max_wait_ms = max_wait_ms / 1000  # 转换为秒

        # 请求队列
        self.queue = deque()
        self.lock = asyncio.Lock()
        self.batch_ready = asyncio.Event()

    async def predict(self, text: str) -> Dict:
        """异步推理接口"""
        future = asyncio.Future()

        async with self.lock:
            self.queue.append((text, future, time.time()))
            # 如果攒够了 max_batch_size，立即触发
            if len(self.queue) >= self.max_batch_size:
                self.batch_ready.set()

        # 等待结果
        return await future

    async def batch_worker(self):
        """后台 worker：定期取出队列中的请求批量推理"""
        while True:
            # 等待触发（时间到 或 攒够了）
            try:
                await asyncio.wait_for(
                    self.batch_ready.wait(),
                    timeout=self.max_wait_ms,
                )
            except asyncio.TimeoutError:
                pass  # 超时了，不管多少条都推理

            self.batch_ready.clear()

            async with self.lock:
                if not self.queue:
                    continue

                # 取出最多 max_batch_size 条
                batch_items = []
                while self.queue and len(batch_items) < self.max_batch_size:
                    batch_items.append(self.queue.popleft())

            if not batch_items:
                continue

            texts = [item[0] for item in batch_items]
            futures = [item[1] for item in batch_items]
            submit_times = [item[2] for item in batch_items]

            # 批量推理
            inputs = self.tokenizer(
                texts, truncation=True, max_length=128,
                padding=True, return_tensors="pt"
            ).to(self.device)

            infer_start = time.time()
            with torch.no_grad():
                outputs = self.model(**inputs)
                probs = torch.softmax(outputs.logits, dim=-1)
                top_probs, top_labels = torch.max(probs, dim=-1)
            infer_time = (time.time() - infer_start) * 1000

            # 告知每个请求的结果
            now = time.time()
            for i, future in enumerate(futures):
                wait_time = (now - submit_times[i]) * 1000 if submit_times else 0
                future.set_result({
                    "label": self.model.config.id2label.get(str(top_labels[i].item()), "unknown"),
                    "confidence": round(top_probs[i].item(), 4),
                    "batch_size": len(texts),
                    "wait_ms": round(wait_time, 1),
                    "infer_ms": round(infer_time / len(texts), 1),
                })

    async def start(self):
        """启动后台 worker"""
        asyncio.create_task(self.batch_worker())


# ===== FastAPI 集成示例 =====
if __name__ == "__main__":
    from fastapi import FastAPI
    import uvicorn

    app = FastAPI()
    engine = DynamicBatchInference(MODEL_PATH, max_batch_size=16, max_wait_ms=30)

    @app.on_event("startup")
    async def startup():
        await engine.start()

    @app.post("/predict")
    async def predict(text: str = "测试"):
        result = await engine.predict(text)
        return result

    uvicorn.run(app, host="0.0.0.0", port=8001)
```

## 4 项目总结

### 4.1 优点与缺点

| 优化技术 | 优点 | 缺点 |
|---------|------|------|
| **批处理** | 吞吐提升3-5倍，GPU利用率高 | 增加等待延迟，batch内最长文本决定整体耗时 |
| **FP16量化** | 显存减半，几乎无损准确率 | 仅支持Volta+架构GPU，CPU无加速 |
| **INT8/INT4量化** | 显存降到1/4~1/8，CPU也可用 | 准确率有轻微损失（1-3%），生成任务影响大于分类 |
| **device_map="auto"** | 自动分配，代码零改动 | 跨设备通信增加延迟，分配策略非最优 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 高吞吐在线推理 | 动态批处理 + FP16 |
| 消费级GPU跑大模型 | INT4/INT8量化 + device_map |
| CPU推理 | INT8量化 + ONNX Runtime |
| 延迟敏感场景（<20ms） | 静态批处理 + batch_size=4 |

**不适用场景**：
- 准确率要求极高（99%+）的医疗/金融场景→用FP32
- 单条请求的文本长度差异极大（如1字 vs 5000字）→批处理padding浪费严重

### 4.3 注意事项

1. **量化后不能直接 `.to(device)`**：量化模型用 `device_map` 管理，手动 `.to()` 会破坏量化配置
2. **batch内padding**：用 `DataCollatorWithPadding` 动态padding到batch内最长，节省显存
3. **torch.compile**：PyTorch 2.0+支持编译优化，但首次编译慢，适合长期运行的服务

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| INT8模型加载后推理报错 | bitsandbytes与CUDA版本不匹配 | 检查 `bitsandbytes` 版本与CUDA兼容性 |
| 批处理反而更慢 | batch内文本长度差异大，padding浪费 | 按长度排序后分组，长度相近的放同一batch |
| device_map分配不合理 | 默认策略根据参数数量而非实际显存分配 | 设置 `max_memory` 手动约束 |

### 4.6 第21章思考题答案

**第21章思考题1**：
- chunk_size=200：检索更精确但可能丢失上下文；chunk_size=1000：语义信息完整但检索精度下降。500是常见平衡点，具体选择取决于文档类型——技术手册（段落短）用200-300，政策文档（段落长）用500-800。

**第21章思考题2**：
- 多轮对话RAG方案：(1) 维护对话历史 `[(q1,a1), (q2,a2), ...]`；(2) 对当前问题做query重写——将"那第二个呢？"重写为"那第二个方案呢？"（融合上下文）；(3) 用重写后的query做检索；(4) 生成时把历史对话也放入prompt。核心：query重写可以用一个小模型完成，或者在prompt中让LLM先总结对话意图再检索。
