# 第27章：模型版本管理与灰度发布

## 1 项目背景

### 业务场景

算法团队训练了一个新版工单分类模型 v2.1.0，在测试集上 F1 比当前线上 v1.5.0 高了 3 个百分点（0.91 → 0.94）。产品经理兴奋地要求"今晚就上线"。运维团队直接替换了模型文件，重启了推理服务。

第二天早上，客服主管怒气冲冲地打来电话："自动分派系统完全乱了！所有工单都被分到了'其他'类别！这比不分类还糟糕，现在所有工单都堆在人工队列里，8 个客服根本处理不过来！"

紧急排查发现：v2.1.0 训练时 `id2label` 的映射和 v1.5.0 不同——v1.5.0 用 `{0:"投诉", 1:"咨询"...}`，而 v2.1.0 用 `{0:"咨询", 1:"投诉"...}`。标签全乱了。

更糟糕的是，运维人员试图回滚到 v1.5.0，但发现旧的模型文件被覆盖了——因为新旧版本都叫 `model.safetensors`，放在了同一个目录里。

### 痛点放大

模型从训练到上线是一条"脆弱链条"，任一环节断裂就导致事故：

```
v2训练完成 → 文件直接覆盖v1 → v2出BUG → 无旧版本可回滚 → 事故扩大
    │              │
    └── 无版本号    └── 无灰度，100%流量直接切换
```

三大核心痛点：
1. **版本混乱**：模型文件没有规范的版本命名，无法追溯哪个版本对应哪个训练实验
2. **无灰度机制**：新模型一上线就扛 100% 流量，出问题就是全量事故
3. **回滚困难**：旧版本文件被覆盖、Docker镜像被覆盖，回滚需重新训练

## 2 项目设计

### 剧本式交锋对话

**场景**：周三上午 9:00，事故复盘会。大屏播放着昨天的混乱 timeline。技术经理脸色铁青。

---

**小胖**:"这就好比你家装修，直接把旧家具全扔了摆新家具。结果新床塌了——发现旧床已经被垃圾车运走了，今晚只能睡地板。"

**小白**:"模型版本管理的基本要求：每个版本的文件放在独立目录（`models/v1.5.0/`、`models/v2.1.0/`），加载时通过路径区分。但我有个疑问——新模型上线一定要 100% 替换吗？能不能先让 10% 的用户试试新模型，确认没问题再全量？"

**大师**:"这就是灰度发布。让我把模型发布流程的三大核心机制讲清楚。

**机制一：版本命名与元数据。**

模型版本号推荐用语义化版本：`主版本.次版本.修订号`（如 v2.1.0）。
- 主版本：模型架构变更（如 BERT → RoBERTa）
- 次版本：训练数据/策略变更（如数据集从 Q1 换成 Q2）
- 修订号：Bug 修复/参数微调

每个版本目录必须包含一个 `model_meta.json`，记录：
```json
{
  "version": "v2.1.0",
  "base_model": "bert-base-chinese",
  "training_date": "2024-07-15",
  "dataset_version": "tickets_2024Q2_v2",
  "metrics": {"f1": 0.94, "accuracy": 0.93},
  "git_commit": "abc1234",
  "trainer": "小陈"
}
```

**机制二：灰度发布。**

灰度发布的本质是**流量分割**。常见方案：
- **按比例随机**：X% 流量走新模型，(100-X)% 走旧模型。简单但可能同一用户两次请求走不同模型
- **按用户 ID Hash**：`hash(user_id) % 100 < X` 走新模型。同一用户始终走同一模型
- **影子流量**：新模型不直接影响用户，只在后台并行推理，对比新旧模型结果差异

灰度节奏：10%（观察1天）→ 30%（观察半天）→ 50%（观察半天）→ 100%。每步对比新模型 vs 旧模型的关键指标（准确率、延迟、错误率）。

**机制三：回滚策略。**

三个回滚路径，从快到慢：
1. **路由回滚**（秒级）：在网关/负载均衡层把流量切回旧版本，最快
2. **文件回滚**（分钟级）：替换模型目录的文件符号链接
3. **部署回滚**（分钟级）：K8s 回滚到上一个 Deployment 版本

关键原则：**旧版本永远不删除**。保留最近 5 个版本的模型文件和 Docker 镜像。"

**技术映射总结**：
- 版本命名 = 给模型起"身份证号"，可追溯不可伪造
- 灰度发布 = 新菜品试吃，先给 10% 的顾客尝，没问题再全量
- 回滚 = "撤销"键，但前提是旧版本还留着

```python
# model_registry.py
"""模型版本注册表 —— 版本管理 + 元数据 + 查询"""

import os
import json
import shutil
from pathlib import Path
from datetime import datetime
from typing import List, Dict, Optional


class ModelRegistry:
    """本地模型注册表"""

    def __init__(self, registry_dir: str = "./model_registry"):
        self.registry_dir = Path(registry_dir)
        self.registry_dir.mkdir(parents=True, exist_ok=True)

    def register(self, model_src: str, version: str,
                 metadata: Dict = None) -> str:
        """注册模型版本"""
        dest = self.registry_dir / version
        if dest.exists():
            raise FileExistsError(f"版本 {version} 已存在")

        # 复制模型
        shutil.copytree(model_src, str(dest))

        # 写入扩展元数据
        meta = metadata or {}
        meta.update({
            "version": version,
            "registered_at": datetime.now().isoformat(),
            "source_path": model_src,
            "model_files": [f.name for f in dest.iterdir()],
        })
        with open(dest / "model_meta.json", "w", encoding="utf-8") as f:
            json.dump(meta, f, ensure_ascii=False, indent=2)

        print(f"模型已注册: {version} → {dest}")
        return str(dest)

    def get_version(self, version: str) -> Optional[Dict]:
        """获取指定版本信息"""
        path = self.registry_dir / version
        if not path.exists():
            return None
        meta_file = path / "model_meta.json"
        if meta_file.exists():
            with open(meta_file) as f:
                return json.load(f)
        return {"version": version, "path": str(path)}

    def list_versions(self) -> List[Dict]:
        """列出所有已注册版本"""
        versions = []
        for d in sorted(self.registry_dir.iterdir()):
            if d.is_dir() and not d.name.startswith("."):
                info = self.get_version(d.name)
                versions.append(info)
        return versions

    def get_latest(self) -> Optional[str]:
        """获取最新版本路径"""
        versions = self.list_versions()
        if not versions:
            return None
        return versions[-1]["path"]

    def get_best(self, metric: str = "f1") -> Optional[str]:
        """按指标获取最佳版本路径"""
        best_version = None
        best_score = -1
        for v in self.list_versions():
            metrics = v.get("metrics", {})
            score = metrics.get(metric, 0)
            if score > best_score:
                best_score = score
                best_version = v
        return best_version["path"] if best_version else None

    def delete_version(self, version: str, force: bool = False):
        """删除版本（保留最近 5 个的限制）"""
        path = self.registry_dir / version
        if not path.exists():
            raise FileNotFoundError(f"版本 {version} 不存在")
        versions = self.list_versions()
        if len(versions) <= 2 and not force:
            raise ValueError("至少保留 2 个版本，禁止删除")
        shutil.rmtree(str(path))
        print(f"已删除: {version}")


# ===== 使用示例 =====
if __name__ == "__main__":
    registry = ModelRegistry("./model_registry")

    # 注册版本
    registry.register(
        model_src="./models/sentiment_minimal",
        version="v1.5.0",
        metadata={
            "base_model": "bert-base-chinese",
            "training_date": "2024-06-01",
            "metrics": {"f1": 0.91, "accuracy": 0.92},
            "dataset": "tickets_2024Q1",
            "trainer": "小陈",
        }
    )

    # 查询所有版本
    print("\n已注册版本:")
    for v in registry.list_versions():
        metrics = v.get("metrics", {})
        print(f"  {v['version']}: F1={metrics.get('f1', 'N/A')}, "
              f"date={v.get('training_date', 'N/A')}")

    # 获取最新/最佳版本
    print(f"\n最新版本: {registry.get_latest()}")
    print(f"最佳 F1 版本: {registry.get_best('f1')}")
```

### 3.3 灰度发布引擎

```python
# canary_router.py
"""灰度发布路由引擎"""

import hashlib
import time
from enum import Enum
from typing import Dict, Optional, List
from dataclasses import dataclass, field


class TrafficStrategy(Enum):
    RANDOM = "random"       # 随机比例
    USER_HASH = "user_hash" # 用户 ID 哈希
    SHADOW = "shadow"       # 影子流量


@dataclass
class ModelRoute:
    model_path: str
    version: str
    weight: int  # 0-100，和所有 route 的 weight 总和应为 100


@dataclass
class CanaryConfig:
    routes: List[ModelRoute] = field(default_factory=list)
    strategy: TrafficStrategy = TrafficStrategy.RANDOM
    shadow_enabled: bool = False  # 是否开启影子流量
    auto_promote_threshold: Optional[float] = None  # 自动全量的指标阈值


class CanaryRouter:
    """灰度发布路由器"""

    def __init__(self, config: CanaryConfig):
        self.config = config
        self._validate_routes()
        self.stats = {}  # 每个版本的统计

    def _validate_routes(self):
        total_weight = sum(r.weight for r in self.config.routes)
        if total_weight != 100:
            raise ValueError(f"路由权重总和必须为 100，当前: {total_weight}")

    def route(self, user_id: str = None) -> ModelRoute:
        """根据策略选择模型版本"""
        if self.config.strategy == TrafficStrategy.USER_HASH and user_id:
            hash_val = int(hashlib.md5(user_id.encode()).hexdigest(), 16) % 100
        else:
            import random
            hash_val = random.randint(0, 99)

        cumulative = 0
        for route in self.config.routes:
            cumulative += route.weight
            if hash_val < cumulative:
                return route

        return self.config.routes[-1]  # 兜底

    def record_result(self, version: str, metrics: Dict):
        """记录每次推理的结果（用于灰度对比）"""
        if version not in self.stats:
            self.stats[version] = {"count": 0, "errors": 0, "total_latency": 0}
        s = self.stats[version]
        s["count"] += 1
        if metrics.get("error"):
            s["errors"] += 1
        s["total_latency"] += metrics.get("latency_ms", 0)

    def get_comparison(self) -> Dict:
        """获取新老版本对比"""
        result = {}
        for version, s in self.stats.items():
            if s["count"] > 0:
                result[version] = {
                    "count": s["count"],
                    "error_rate": s["errors"] / s["count"],
                    "avg_latency_ms": s["total_latency"] / s["count"],
                }
        return result

    def should_rollback(self, threshold_error_rate: float = 0.05) -> bool:
        """判断是否需要自动回滚（错误率超阈值）"""
        for version, stats in self.get_comparison().items():
            if "canary" in version.lower() and stats["error_rate"] > threshold_error_rate:
                return True
        return False

    def promote_step(self, new_version_weight: int):
        """灰度比例提升"""
        for route in self.config.routes:
            if "canary" in route.version.lower() or "v2" in route.version:
                route.weight = new_version_weight
            else:
                route.weight = 100 - new_version_weight
        self._validate_routes()

    def promote_full(self):
        """全量切换"""
        new_route = [r for r in self.config.routes
                     if "canary" in r.version.lower() or "v2" in r.version]
        old_route = [r for r in self.config.routes
                     if r not in new_route]
        if new_route and old_route:
            new_route[0].weight = 100
            old_route[0].weight = 0

    def rollback(self):
        """回滚到旧版本"""
        for route in self.config.routes:
            if "canary" in route.version.lower() or "v2" in route.version:
                route.weight = 0
            else:
                route.weight = 100


# ===== 使用示例 =====
if __name__ == "__main__":
    config = CanaryConfig(
        routes=[
            ModelRoute(model_path="./models/v1.5.0", version="v1.5.0", weight=90),
            ModelRoute(model_path="./models/v2.1.0", version="v2.1.0-canary", weight=10),
        ],
        strategy=TrafficStrategy.USER_HASH,
    )

    router = CanaryRouter(config)

    # 模拟灰度验证
    print("灰度 10% 验证中...")
    for i in range(100):
        route = router.route(user_id=f"user_{i}")
        # 模拟推理结果
        is_error = (route.version == "v2.1.0-canary" and i % 3 == 0)  # v2 有 33% 错误
        router.record_result(route.version, {
            "error": is_error,
            "latency_ms": 50 if route.version == "v1.5.0" else 80,
        })

    comparison = router.get_comparison()
    print("\n灰度对比结果:")
    for ver, stats in comparison.items():
        print(f"  {ver}: 请求数={stats['count']}, "
              f"错误率={stats['error_rate']:.1%}, "
              f"平均延迟={stats['avg_latency_ms']:.0f}ms")

    # 判断是否需回滚
    if router.should_rollback(threshold_error_rate=0.1):
        print("\n⚠ 错误率超阈值，自动回滚！")
        router.rollback()
    else:
        print("\n✓ 灰度正常，可提升比例")
        router.promote_step(new_version_weight=30)
        for r in router.config.routes:
            print(f"  {r.version}: {r.weight}%")

# server_canary.py
"""集成灰度的推理服务"""

from fastapi import FastAPI, HTTPException, Request, Query
from canary_router import CanaryRouter, CanaryConfig, ModelRoute, TrafficStrategy
from model_registry import ModelRegistry
from typing import Dict

app = FastAPI()

# 模型注册表
registry = ModelRegistry("./model_registry")

# 灰度配置
canary_config = CanaryConfig(
    routes=[
        ModelRoute(model_path=registry.get_version("v1.5.0")["path"],
                   version="v1.5.0", weight=90),
        ModelRoute(model_path=registry.get_version("v2.1.0")["path"],
                   version="v2.1.0", weight=10),
    ],
    strategy=TrafficStrategy.USER_HASH,
)
router = CanaryRouter(canary_config)


@app.post("/predict")
async def predict(text: str, user_id: str = Query("anonymous"), request: Request = None):
    route = router.route(user_id=user_id)

    try:
        # 加载对应版本模型并推理（此处简化）
        # model = load_model(route.model_path)
        # result = model.predict(text)
        result = {"label": "咨询", "confidence": 0.9}

        router.record_result(route.version, {"error": False, "latency_ms": 50})

        return {
            **result,
            "model_version": route.version,
        }
    except Exception as e:
        router.record_result(route.version, {"error": True, "latency_ms": 0})
        raise HTTPException(500, str(e))


@app.get("/canary/status")
async def canary_status():
    return {
        "routes": [{"version": r.version, "weight": r.weight}
                   for r in router.config.routes],
        "comparison": router.get_comparison(),
    }


@app.post("/canary/promote")
async def promote(weight: int = Query(30, ge=0, le=100)):
    router.promote_step(weight)
    return {"status": "ok", "message": f"canary weight → {weight}%"}


@app.post("/canary/full")
async def promote_full():
    router.promote_full()
    return {"status": "ok", "message": "全量切换到新版本"}


@app.post("/canary/rollback")
async def rollback():
    router.rollback()
    return {"status": "ok", "message": "已回滚到旧版本"}


if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

import pytest
from canary_router import CanaryRouter, CanaryConfig, ModelRoute, TrafficStrategy

class TestCanaryRouter:
    def test_route_distribution(self):

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **版本注册表** | 可追溯、防覆盖、一键回滚 | 磁盘占用随版本数增长 |
| **语义化版本号** | 直观表达变更范围 | 需团队约定和遵守 |
| **用户 Hash 灰度** | 同用户始终走同一模型 | 新增用户可能集中在灰度组 |
| **自动回滚** | 错误率高时自动切换 | 误报阈值设置不当导致频繁切换 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 模型常规迭代 | 灰度 10%→30%→100% |
| 高风险变更 | 影子流量先行（不实际影响用户） |
| 线上事故 | 一键回滚 + 事后根因分析 |
| 多业务线模型管理 | 注册表 + 元数据索引 |

**不适用场景**：
- 实时性极高（<10ms）的场景，增加路由层会增加延迟
- 单租户/离线推理场景

### 4.3 注意事项

1. **旧版本保留策略**：至少保留最近 5 个版本的模型文件和 Docker 镜像
2. **灰度验证指标**：至少对比错误率、P95 延迟、准确率（需日志打标）
3. **回滚后的数据一致性**：如果新模型修改了输出格式，回滚后下游系统需兼容旧格式

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 灰度路由不准 | 随机数种子在多进程间未隔离 | 用 user_id hash 替代 random |
| 旧版本加载失败 | 依赖库升级后旧模型不兼容 | 模型元数据中记录依赖版本 |
| 全量后旧版仍接收流量 | 灰度配置未原子更新 | 用分布式配置中心（etcd/consul） |

1. **初级**：在 `ModelRegistry` 中增加 `diff_versions(v1, v2)` 方法，对比两个版本的元数据差异（指标、数据集、训练日期）。

- 数据漂移检测方案：(1) 从训练集中随机抽样 5000 条，用 Embedding 模型编码，保存 Embedding 分布的均值/协方差；(2) 在线推理时每 1000 条计算一次线上数据的 Embedding 分布；(3) 计算两个分布的 Wasserstein 距离，超过阈值（如历史均值的 3σ）则告警；(4) 用 `scipy.stats.wasserstein_distance` 实现。
