# 第19章：长文本处理：滑窗、摘要与层级建模

## 1 项目背景

### 业务场景

法务科技公司的合同审查系统上线后，算法团队发现一个严重问题：约 30% 的合同超过 20 页，word 计数 5000-15000 字不等。BERT 模型的最大输入长度只有 512 个 token，即使换用支持更长上下文的模型（如 Longformer 的 4096 token），仍然无法覆盖完整的合同文本。

更糟糕的是，合同的结构有特殊性：关键信息往往集中在特定段落——"违约条款"在第 15 页，"付款方式"在第 3 页，而"争议解决"在最后一页。简单的"截断前 512 token"会丢失后半部分的重要信息，"截断后 512 token"又丢掉了甲乙方信息和合同标的。

法务团队反馈："你们系统每次只识别了合同前两页的内容，后面的违约条款全漏了，这比不识别还危险——法务以为系统检查了全文。"

### 痛点放大

长文本处理是 Transformer 模型的"阿喀琉斯之踵"：

1. **硬截断问题**：`max_length=512` 导致尾部信息完全丢失，而关键条款（如违约、保密、争议解决）往往在合同末尾
2. **注意力复杂度**：Self-Attention 的复杂度是 O(n²)，n=512 时需要 262K 次计算，n=4096 时需要 16M 次——100 页合同（约 25000 token）根本不可能直接处理
3. **全局 vs 局部**：合同分类需要看全文才能判断"这是采购合同还是劳务合同"，但实体抽取（如金额、日期）通常只需看局部

```
15 页合同 (15000 字, ~8000 tokens):
┌────────────────────────────────────────────────┐
│ 第1页: 甲方、乙方、合同标的                      │ ← 截断前512: 只看这里
│ 第2-3页: 付款方式、交付日期                      │
│ 第4-5页: 质量标准、验收流程                      │
│ ...                                            │
│ 第13页: 违约责任条款                             │ ← 截断后512: 只看这里
│ 第14页: 保密条款                                 │
│ 第15页: 争议解决                                 │
└────────────────────────────────────────────────┘
              两者都丢失了中间信息!
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周四下午 4:00，法务团队会议室。屏幕上展示着两份合同审查结果——一份是人工的（12 个风险点），一份是 AI 的（4 个风险点，且都是前两页的）。

---

**小胖**:"这 AI 审查怎么跟看菜单只看第一页一样？后面的招牌菜全错过了。我就问一句——不能让模型把整份合同都看完吗？"

**小陈**:"不行，BERT 模型最大只能处理 512 个 token。这份合同 15000 字，相当于 8000 多个 token——模型根本不是'不想看'，是'装不下'。"

**小胖**:"那你不能把合同切成小块，一块一块喂给模型吗？"

**小白**:"这就是滑窗切分策略。但切分后有三个问题：第一，怎么切——按字符、按句子、还是按段落？第二，滑窗之间有重叠吗？第三，每个窗口的预测结果怎么合并回完整的合同？"

**大师**:"好，让我把长文本处理的三种主流策略系统化讲清楚。

**策略一：滑窗切分（Sliding Window）。** 把长文档切成固定长度的窗口，相邻窗口之间有 `doc_stride` 重叠。每个窗口独立过模型推理。所有窗口的预测结果通过投票或拼接来合并。

适合：分类任务（每个窗口给一个类别预测，最终投票决定文档类别）、NER 任务（每个窗口的实体预测合并回原文）。

关键参数：`window_size`（窗口大小）和 `stride`（滑动步长）。步长太小 → 窗口过多推理慢；步长太大 → 窗口边界处信息丢失。

**策略二：先摘要后分类（Summarize-then-Classify）。** 先用生成模型对长文档做摘要，把 15000 字压缩到 500 字，再用分类模型对摘要做分类。

适合：文档级分类任务。不适合 NER（摘要会丢失实体位置信息）。

**策略三：层级建模（Hierarchical Modeling）。** 先对每个句子/段落独立编码（用 BERT 句子编码器），得到句子向量序列；再用一个轻量级的 Transformer 或 RNN 对句子向量序列做全局建模。

适合：需要全局理解但细节不重要的任务。存储效率高，但不适合需要精确定位实体边界的任务。

**我们的合同场景最佳方案：** 分段切分（按"第X条"作为分界）+ 滑窗 NER 抽取 + 跨段落合并。既利用了合同的结构化特点（按条款分段），又保证了实体级别的精确性。"

**小胖**:"那 Longformer、BigBird 这些'长文本特化模型'呢？它们不是说能处理 4096 token 吗？"

**大师**:"它们用稀疏注意力（只关注局部窗口 + 少数全局 token）把 O(n²) 降到 O(n)，确实能处理更长的上下文。但 4096 token 对 8000 token 的合同仍然不够，而且这些模型的预训练不如 BERT 成熟。在我们的场景中，它们可以作为滑窗策略的补充——窗口开到 4096 而不是 512，需要的窗口数减少 8 倍。"

**技术映射总结**：
- 滑窗 = 把长文章切成"卡片"，一张一张看，最后拼回来
- 先摘要后分类 = 先看"内容提要"再判断类型
- 层级建模 = 先给每个段落写摘要，再对摘要序列做全局理解
- 稀疏注意力 = 只跟邻居和少数"班长"交流，不是所有人都跟所有人说话

### 3.2 滑窗分类策略

```python
# sliding_window_classify.py
"""长文档分类 —— 滑窗 + 投票策略"""

import torch
import numpy as np
from transformers import AutoTokenizer, AutoModelForSequenceClassification


class LongDocumentClassifier:
    """
    长文档分类器
    策略: 滑窗切分 → 逐窗分类 → 多数投票/置信度加权
    """

    def __init__(self, model_path: str, window_size: int = 400,
                 stride: int = 200, device: str = None):
        self.tokenizer = AutoTokenizer.from_pretrained(model_path)
        self.model = AutoModelForSequenceClassification.from_pretrained(model_path)
        self.window_size = window_size
        self.stride = stride
        self.device = device or ("cuda" if torch.cuda.is_available() else "cpu")
        self.model.to(self.device)
        self.model.eval()

    def _split_windows(self, text: str) -> list:
        """将长文本切分为滑窗"""
        # 先 tokenize 全文获取 token 数
        tokens = self.tokenizer.encode(text, add_special_tokens=False)
        windows = []

        start = 0
        while start < len(tokens):
            end = min(start + self.window_size, len(tokens))
            window_tokens = [self.tokenizer.cls_token_id] + tokens[start:end] + [self.tokenizer.sep_token_id]
            windows.append(window_tokens)
            if end >= len(tokens):
                break
            start += self.stride

        return windows

    def classify_sliding(self, text: str, strategy: str = "vote") -> dict:
        """
        滑窗分类

        Args:
            text: 长文本
            strategy: "vote"(投票) / "max_confidence"(最大置信度) / "weighted"(加权)
        """
        windows = self._split_windows(text)
        all_probs = []
        window_results = []

        for i, window in enumerate(windows):
            # padding
            if len(window) < self.window_size + 2:
                window = window + [self.tokenizer.pad_token_id] * (self.window_size + 2 - len(window))

            input_ids = torch.tensor([window]).to(self.device)
            attention_mask = torch.tensor(
                [[1 if t != self.tokenizer.pad_token_id else 0 for t in window]]
            ).to(self.device)

            with torch.no_grad():
                outputs = self.model(input_ids=input_ids, attention_mask=attention_mask)
                probs = torch.softmax(outputs.logits, dim=-1)

            all_probs.append(probs[0])
            pred_class = torch.argmax(probs, dim=-1).item()
            conf = probs[0][pred_class].item()
            window_results.append({"window": i, "pred": pred_class, "conf": conf})

        all_probs = torch.stack(all_probs)

        if strategy == "vote":
            # 多数投票
            preds = [r["pred"] for r in window_results]
            from collections import Counter
            final_pred = Counter(preds).most_common(1)[0][0]
            final_conf = sum(1 for p in preds if p == final_pred) / len(preds)

        elif strategy == "max_confidence":
            # 最大置信度
            best = max(window_results, key=lambda x: x["conf"])
            final_pred = best["pred"]
            final_conf = best["conf"]

        else:  # weighted
            # 置信度加权平均
            weighted = all_probs.mean(dim=0)
            final_pred = torch.argmax(weighted).item()
            final_conf = weighted[final_pred].item()

        id2label = self.model.config.id2label
        return {
            "label": id2label.get(str(final_pred), f"class_{final_pred}"),
            "label_id": final_pred,
            "confidence": round(final_conf, 4),
            "num_windows": len(windows),
            "window_details": window_results[:5],  # 只展示前 5 个窗口
        }


# ===== 使用示例 =====
if __name__ == "__main__":
    classifier = LongDocumentClassifier(
        model_path="./models/sentiment_minimal",
        window_size=200,
        stride=100,
    )

    # 模拟长文本（15 页合同）
    long_contract = "甲方和乙方签订本合同 " * 50 + "违约条款：违约金为合同总额的30% " * 20
    print(f"原始文本长度: {len(long_contract)} 字符")

    result = classifier.classify_sliding(long_contract, strategy="weighted")
    print(f"\n分类结果: {result['label']} (置信度: {result['confidence']:.2%})")
    print(f"切分窗口数: {result['num_windows']}")
```

### 3.3 长文档 NER 与实体合并

```python
# long_ner.py
"""长文档 NER —— 滑窗提取 + 跨窗口实体合并"""

import re
from typing import List, Dict
from transformers import AutoTokenizer, AutoModelForTokenClassification
import torch


class LongDocumentNER:
    def __init__(self, model_path: str, window_size: int = 256,
                 stride: int = 128):
        self.tokenizer = AutoTokenizer.from_pretrained(model_path)
        self.model = AutoModelForTokenClassification.from_pretrained(model_path)
        self.window_size = window_size
        self.stride = stride
        self.device = "cuda" if torch.cuda.is_available() else "cpu"
        self.model.to(self.device)
        self.model.eval()
        self.id2tag = self.model.config.id2label

    def extract_entities_long(self, text: str) -> Dict[str, List[str]]:
        """
        长文档实体抽取：滑窗 → 逐窗 NER → 跨窗口合并去重
        """
        # 按句子切分（保留位置信息）
        sentences = self._split_sentences(text)

        # 构建滑窗
        windows = []
        for i in range(0, len(sentences), max(1, len(sentences) // (len(text) // self.stride))):
            # 取 stride 个句子作为一个窗口
            window_text = "".join(sentences[i:i + self.stride // 30])
            if window_text.strip():
                windows.append(window_text)

        # 逐窗口 NER
        all_entities = {}
        for window_text in windows:
            entities = self._ner_single_window(window_text)
            for ent_type, values in entities.items():
                all_entities.setdefault(ent_type, []).extend(values)

        # 跨窗口去重与合并
        return self._deduplicate_entities(all_entities)

    def _split_sentences(self, text: str) -> List[str]:
        """简单分句"""
        # 按常见标点分句
        sentences = re.split(r'[。！？；\n]', text)
        return [s.strip() + "。" for s in sentences if s.strip()]

    def _ner_single_window(self, text: str) -> Dict[str, List[str]]:
        """单窗口 NER"""
        inputs = self.tokenizer(
            text, max_length=self.window_size,
            truncation=True, padding="max_length", return_tensors="pt"
        ).to(self.device)

        with torch.no_grad():
            outputs = self.model(**inputs)
            predictions = torch.argmax(outputs.logits, dim=-1)[0]

        tokens = self.tokenizer.convert_ids_to_tokens(inputs["input_ids"][0])
        entities = {}
        current_ent = []
        current_type = None

        for token, pred_id in zip(tokens, predictions):
            tag = self.id2tag.get(pred_id.item(), "O")

            if token in ("[CLS]", "[SEP]", "[PAD]"):
                continue

            if tag.startswith("B-"):
                if current_ent and current_type:
                    entities.setdefault(current_type, []).append("".join(current_ent))
                current_type = tag[2:]
                current_ent = [token.replace("##", "")]
            elif tag.startswith("I-") and current_ent:
                current_ent.append(token.replace("##", ""))
            else:
                if current_ent and current_type:
                    entities.setdefault(current_type, []).append("".join(current_ent))
                current_ent = []
                current_type = None

        # 最后一个实体
        if current_ent and current_type:
            entities.setdefault(current_type, []).append("".join(current_ent))

        return entities

    def _deduplicate_entities(self, entities: Dict[str, List[str]]) -> Dict[str, List[str]]:
        """跨窗口去重（完全相同的删除，包含关系的取长的）"""
        result = {}
        for ent_type, values in entities.items():
            unique = []
            for v in values:
                v = v.strip()
                if len(v) < 2:
                    continue
                # 如果是已有实体的子串，跳过
                is_substring = any(v != u and v in u for u in unique)
                if not is_substring:
                    # 如果已有实体是当前实体的子串，替换
                    unique = [u for u in unique if u not in v]
                    unique.append(v)
            result[ent_type] = unique
        return result


if __name__ == "__main__":
    ner = LongDocumentNER("./models/contract_ner", window_size=256, stride=128)

    text = (
        "甲方：深圳市腾讯计算机系统有限公司。" * 5 +
        "合同金额为人民币伍拾万元整。" * 3 +
        "签订日期：2024年5月20日。" * 3 +
        "违约金不超过合同总金额的30%。" * 2
    )

    entities = ner.extract_entities_long(text)
    print("长文档 NER 结果:")
    for ent_type, values in entities.items():
        print(f"  {ent_type}: {values[:3]}")
```

### 3.4 先摘要后分类策略

```python
# summarize_then_classify.py
"""先摘要后分类 —— 长文本处理策略"""

from transformers import pipeline


class SummarizeThenClassify:
    """先用生成模型做摘要，再用分类模型分类"""

    def __init__(self, summarizer_model: str, classifier_path: str):
        # 摘要模型（生成式）
        self.summarizer = pipeline(
            "summarization",
            model=summarizer_model,
            device=0 if __import__("torch").cuda.is_available() else -1,
        )
        # 分类模型
        from transformers import AutoModelForSequenceClassification, AutoTokenizer
        self.cls_tokenizer = AutoTokenizer.from_pretrained(classifier_path)
        self.cls_model = AutoModelForSequenceClassification.from_pretrained(classifier_path)
        self.device = "cuda" if __import__("torch").cuda.is_available() else "cpu"
        self.cls_model.to(self.device)

    def classify(self, text: str, max_summary_length: int = 300) -> dict:
        """先摘要后分类"""
        # 如果文本很长，先分块摘要再合并
        if len(text) > 2000:
            # 分块摘要
            chunks = [text[i:i+2000] for i in range(0, len(text), 2000)]
            summaries = []
            for chunk in chunks:
                result = self.summarizer(
                    chunk, max_length=max_summary_length // len(chunks),
                    min_length=30, do_sample=False
                )
                summaries.append(result[0]["summary_text"])
            summary = " ".join(summaries)
        else:
            result = self.summarizer(
                text, max_length=max_summary_length, min_length=30, do_sample=False
            )
            summary = result[0]["summary_text"]

        # 对摘要做分类
        import torch
        inputs = self.cls_tokenizer(
            summary, max_length=512, truncation=True,
            padding="max_length", return_tensors="pt"
        ).to(self.device)

        with torch.no_grad():
            outputs = self.cls_model(**inputs)
            probs = torch.softmax(outputs.logits, dim=-1)
            top_prob, top_label = torch.max(probs, dim=-1)

        id2label = self.cls_model.config.id2label
        return {
            "label": id2label.get(str(top_label.item()), "unknown"),
            "confidence": round(top_prob.item(), 4),
            "summary": summary[:200],
            "original_length": len(text),
            "summary_length": len(summary),
        }
```

## 4 项目总结

### 4.1 优点与缺点

| 策略 | 优点 | 缺点 |
|------|------|------|
| **滑窗切分** | 实现简单，与已有模型兼容，适合 NER | 窗口数多导致推理慢，边界信息可能丢失 |
| **先摘要后分类** | 速度快（窗口数少），语义连贯 | 摘要可能丢失关键细节，不适合 NER |
| **层级建模** | 内存效率高，可处理任意长度 | 需要额外训练层级编码器，架构复杂 |
| **长上下文模型** | 原生支持长文本，注意力完整 | 推理慢，预训练不如标准模型充分 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 长合同 NER（需精确定位） | 滑窗 + 跨窗口实体合并 |
| 超长文档分类 | 先摘要后分类 或 滑窗+投票 |
| 多文档问答 | 检索+精读（先缩小范围再逐窗 QA） |
| 日志/代码流分析 | 层级建模（每行/每函数独立编码 + 序列建模） |

**不适用场景**：
- 实时性要求 < 10ms 的长文本任务（滑窗推理次数多）

### 4.3 注意事项

1. **stride 选择**：stride 太小 → 窗口过多浪费算力；stride 太大 → 边界遗漏。经验值：`stride = window_size / 2`
2. **实体合并规则**：跨窗口同名实体去重时，优先保留长的那个；边界不完整的实体（如窗口末尾截断了一半的词）要去掉
3. **摘要质量依赖**：先摘要后分类策略的瓶颈在上游摘要模型，摘要偏了分类就偏了

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 滑窗分类所有窗口预测不一致 | stride 太大，每个窗口看到的语义片段不够 | 增大重叠比例，或改用加权策略 |
| NER 实体被切成两半 | 窗口边界恰好落在实体中间 | 确保 stride > 最大实体长度 |
| 摘要后分类准确率反而不如直接截断 | 摘要模型对特定领域（法律/医学）效果差 | 在领域数据上微调摘要模型 |

### 4.6 第18章思考题答案

**第18章思考题1**：
- r=2：参数最少（~0.08%），训练最快，但效果可能比全量微调低 3-5%；r=32：参数约 1.2%，效果接近全量微调。最优 r 通常在 8-16 之间。

**第18章思考题2**：
- 架构：(1) 服务初始化时加载基座模型到 GPU；(2) 维护一个 adapter 注册表 `{"pre_sales": "path/to/lora1", "after_sales": "path/to/lora2", "complaint": "path/to/lora3"}`；(3) 请求进来时根据 `request.scene` 字段选择对应的 adapter；(4) 用 `model.load_adapter(adapter_path, adapter_name=scene)` 加载并切换（支持同时加载多个 adapter 到内存，`model.set_adapter(scene)` 切换。
