# 第6章：命名实体识别实战：合同关键信息抽取

## 1 项目背景

### 业务场景

某法务科技公司为中小型企业提供合同审查 SaaS 服务。客户上传一份 20 页的采购合同 PDF，系统需要在 30 秒内自动抽取关键字段：甲方公司名、乙方公司名、合同金额、签订日期、合同编号、违约条款，并在网页端高亮展示。

法务团队过去依赖人工逐页标注，一份合同平均耗时 8 分钟。随着客户量从 50 家增长到 500 家，法务团队从 5 人扩到 15 人仍不堪重负。更棘手的是，人工抽取的准确率在疲劳状态下从 98% 骤降到 85%，漏掉的"违约金不超过合同总额 30%"这类条款直接导致了客户的合同纠纷。

CTO 决定引入 NLP 技术自动抽取合同实体，要求准确率 > 95%，覆盖 20+ 种实体类型。

### 痛点放大

命名实体识别（NER）与第 5 章的文本分类完全不同：分类给整段文本贴一个标签，NER 要给每个 token 贴一个标签。核心挑战：

1. **标注粒度问题**：中文没有天然空格分隔，"深圳市腾讯计算机系统有限公司"应该整体标注为一个实体还是拆开？BIO/BIOES 标注体系如何选择？
2. **子词切分与标签对齐**：BERT 的 WordPiece 会把"有限公司"切成 `['有限', '##公司']`，token 数量 > 原始字符数，标签怎么对应？
3. **边界歧义**：合同中出现"本合同总金额为人民币伍拾万元整（￥500,000.00）"，金额的起止边界在哪里？"伍拾万元整"和"￥500,000.00"是同一个实体还是两个？

```
输入: 甲方：深圳市腾讯计算机系统有限公司，合同金额：500万元
                                                     ↓
                   ┌──── token 级别标注 ────┐
Token:  [CLS] 甲 方 ： 深 圳 市 腾 讯 ... 公 司 ， 合 同 金 额 ： 500 万 元 [SEP]
Label:   O    B-甲 B-甲 O B-乙 B-乙 B-乙 ... I-乙 I-乙 O B-金额 I-金额 O B-金额 I-金额 I-金额 O
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周三下午 2:30，法务团队会议室。算法工程师小陈演示了第一版 NER Demo，但标注结果错漏百出。

---

**小胖**（啃着饼干）:"小陈你这个抽取结果好奇怪啊。合同里明明写着'甲方：深圳市腾讯计算机系统有限公司'，你抽出来的是'深圳市'、'腾讯计算机'、'有限公司'三个片段拼不起来。这要是我点外卖，地址给我拆成'朝阳区'、'建国路'、'88号'三行，骑手能找着才怪。"

**小白**（翻看着标注规则文档）:"这就是 token 级别分类的经典问题——子词切分导致 token 和原始文字不对齐。BERT 把'深圳市腾讯计算机系统有限公司'切成了 `['深圳', '##市', '腾', '##讯', '##计算机', '##系统', '##有限', '##公司']`，模型预测每个 sub-token 都有独立的标签，预测结果需要合并回原始实体。"

**大师**:"来，我把这个问题拆成三个层面讲清楚。

**第一个层面：标注体系选型。**

BIO 体系：B（Begin）表示实体开头，I（Inside）表示实体内部，O（Outside）表示非实体。
BIOES 体系：多了 E（End）表示实体结尾，S（Single）表示单个 token 的实体。

比如'深圳市'作为地点实体：
- BIO：`['B-LOC', 'I-LOC', 'I-LOC']`
- BIOES：`['B-LOC', 'I-LOC', 'E-LOC']`

BIOES 多了边界约束（E 后面不能接 I），能有效减少预测错误。但标注成本更高。我们合同场景用 BIO 足够。

**第二个层面：子词对齐。**

WordPiece 把'有限公司'切成 `['有限', '##公司']` 两个 sub-token。训练时，`ignore_index=-100` 的 token 不参与 loss 计算。但预测时所有 sub-token 都有预测结果。合并策略：只取每个词的第一个 sub-token 的标签，或者取多数投票。"

**小胖**:"那如果 `['有限']` 预测为 B-ORG，`['##公司']` 预测为 O，你最后告诉我这个实体是 B-ORG 还是 O？"

**大师**:"这就是为什么用 **E 标签** 或 **置信度加权投票** 来降低这种错误。不过实际工程中，我们通常约定：**同一个词的所有 sub-token 共享标签**（取第一个 token 的预测），因为 BERT 的子词切分是确定性的。"

**小白**:"第三个层面呢？"

**大师**:"**实体后处理。** 模型输出的是 token 级标签，但业务要的是结构化的 JSON：`{'甲方': '深圳市腾讯计算机系统有限公司', '金额': '500万元'}`。后处理需要做三件事：合并连续同标签 token → 去除不完整实体（只有 B 没有 I）→ 规则后处理（如金额数字格式标准化）。"

**小陈**:"还有一个问题：如果合同里既有'人民币500万元'又有'美元200万元'，怎么区分币种？"

**大师**:"这超出了 NER 的范畴，NER 只解决'在哪里'，不解决'是什么关系'。区分币种需要做**关系抽取**或直接设计更细粒度的实体（B-人民币金额 / B-美元金额）。先用 NER 跑通主要实体，后续再叠加关系分类模型。"

**技术映射总结**：
- NER = 给文本中的每个 token 打上 B/I/O 标签，再合并成实体
- 子词切分导致 token 数 > 字符数，需对齐后合并
- BIO 标注体系 = 开局(B) + 连续(I) + 无关(O)
- 后处理 = 合并连续同标签 token → 规则校验 → 结构化 JSON

```python
"""合同 NER 数据集构建"""

from datasets import Dataset, DatasetDict
from transformers import AutoTokenizer

# ===== 模拟合同训练数据 =====
# 格式: (文本, BIO标签列表)
train_data = [
    (
        "甲方：深圳市腾讯计算机系统有限公司",
        ["B-甲方", "I-甲方", "O", "B-乙方", "I-乙方", "I-乙方", "I-乙方", "I-乙方",
         "I-乙方", "I-乙方", "I-乙方", "I-乙方", "I-乙方", "I-乙方", "I-乙方", "I-乙方"]
    ),
    (
        "合同总金额为人民币伍拾万元整",
        ["O", "O", "O", "O", "B-金额", "I-金额", "I-金额", "I-金额", "I-金额", "I-金额"]
    ),
    (
        "签订日期：2024年5月20日",
        ["O", "O", "O", "O", "B-日期", "I-日期", "I-日期", "I-日期", "I-日期", "I-日期"]
    ),
    (
        "合同编号：HT-2024-00188",
        ["O", "O", "O", "O", "B-编号", "I-编号", "I-编号", "I-编号", "I-编号",
         "I-编号", "I-编号", "I-编号", "I-编号", "I-编号", "I-编号", "I-编号"]
    ),
    (
        "违约金不超过合同总金额的百分之三十",
        ["B-条款", "I-条款", "O", "O", "O", "O", "O", "O", "O", "O", "O",
         "O", "O", "O", "O", "O"]
    ),
]

texts, bio_tags = zip(*train_data)

# 构建标签映射
unique_tags = sorted(set(tag for tags in bio_tags for tag in tags))
tag2id = {tag: i for i, tag in enumerate(unique_tags)}
id2tag = {i: tag for tag, i in tag2id.items()}

print("标签映射:")
for tag, idx in tag2id.items():
    print(f"  {tag:12s} → {idx}")
print(f"  标签数: {len(unique_tags)}")

# 获取 tokenizer
model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)

# ===== 关键: 子词对齐 =====
def tokenize_and_align_labels(examples):
    """
    对文本进行 tokenize，并将标签对齐到 sub-token 级别
    - 第一个 sub-token 保留原标签
    - 后续 sub-token 设为 -100（计算 loss 时忽略）
    """
    tokenized_inputs = tokenizer(
        examples["text"],
        max_length=128,
        truncation=True,
        padding="max_length",
        is_split_into_words=False,  # 输入是完整字符串
    )

    labels = []
    for i, label_list in enumerate(examples["labels"]):
        word_ids = tokenized_inputs.word_ids(batch_index=i)
        label_ids = []

        previous_word_idx = None
        for word_idx in word_ids:
            if word_idx is None:
                # [CLS], [SEP], [PAD] 等特殊 token → -100
                label_ids.append(-100)
            elif word_idx != previous_word_idx:
                # 每个词的第一个 sub-token → 使用原标签
                label_ids.append(tag2id[label_list[word_idx]])
            else:
                # 同一词的后续 sub-token → -100（忽略）
                label_ids.append(-100)
            previous_word_idx = word_idx

        labels.append(label_ids)

    tokenized_inputs["labels"] = labels
    return tokenized_inputs


# 构建 Dataset
dataset = Dataset.from_dict({
    "text": list(texts),
    "labels": [list(tags) for tags in bio_tags]
})

tokenized_dataset = dataset.map(tokenize_and_align_labels, batched=True)
tokenized_dataset.set_format("torch")

# 划分训练/验证集
split_dataset = tokenized_dataset.train_test_split(test_size=0.2, seed=42)
ner_dataset = DatasetDict({
    "train": split_dataset["train"],
    "validation": split_dataset["test"]
})

print(f"\n数据集: train={len(ner_dataset['train'])}, val={len(ner_dataset['validation'])}")

# 验证对齐效果
sample = ner_dataset["train"][0]
print(f"\n对齐验证 (第1条样本):")
print(f"  原始文本: {texts[0]}")
input_ids = sample["input_ids"]
labels = sample["labels"]
for j, (tid, lid) in enumerate(zip(input_ids[:20], labels[:20])):
    token = tokenizer.decode(tid)
    tag = id2tag.get(lid.item(), "IGNORE") if lid != -100 else "IGNORE"
    print(f"    [{j:2d}] {token:10s} → {tag}")
```

```python
"""合同 NER 模型训练"""

import os
import numpy as np
from datasets import load_from_disk

    AutoModelForTokenClassification,
    AutoTokenizer,
    Trainer,
    TrainingArguments,
    EarlyStoppingCallback,
)
from seqeval.metrics import classification_report, f1_score
import torch

# ===== 1. 加载数据 =====
# 从上一步保存或直接使用 prepare_ner_data.py 中的变量
model_name = "bert-base-chinese"
num_labels = 7  # 根据实际标签数调整

tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForTokenClassification.from_pretrained(
    model_name, num_labels=num_labels
)

# ===== 2. 评估指标（seqeval） =====
def compute_metrics(eval_pred):
    predictions, labels = eval_pred
    predictions = np.argmax(predictions, axis=2)

    # 移除 -100 (ignore_index) 的标签
    true_predictions = []
    true_labels = []

    for pred_seq, label_seq in zip(predictions, labels):
        pred_tags = []
        label_tags = []

        for p, l in zip(pred_seq, label_seq):
            if l != -100:
                pred_tags.append(id2tag[p])
                label_tags.append(id2tag[l])

        true_predictions.append(pred_tags)
        true_labels.append(label_tags)

    # seqeval 指标
    results = {
        "f1": f1_score(true_labels, true_predictions),
    }
    return results

# ===== 3. 训练 =====
training_args = TrainingArguments(
    output_dir="./output/contract_ner",
    evaluation_strategy="epoch",
    save_strategy="epoch",

    per_device_train_batch_size=8,

    num_train_epochs=5,
    weight_decay=0.01,
    load_best_model_at_end=True,
    metric_for_best_model="f1",
    save_total_limit=2,
    report_to="none",
```

### 3.4 推理与实体后处理

```python
# ner_inference.py
"""NER 推理 + 实体合并后处理"""

import re
import torch
from transformers import AutoTokenizer, AutoModelForTokenClassification
from typing import List, Dict


class ContractNER:
    def __init__(self, model_path: str):
        self.tokenizer = AutoTokenizer.from_pretrained(model_path)
        self.model = AutoModelForTokenClassification.from_pretrained(model_path)
        self.model.eval()
        self.id2tag = self.model.config.id2label
        self.device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
        self.model.to(self.device)

    def predict_tags(self, text: str) -> List[str]:
        """预测每个 token 的 BIO 标签"""
        inputs = self.tokenizer(
            text, max_length=256, truncation=True,
            padding="max_length", return_tensors="pt"
        ).to(self.device)

        with torch.no_grad():
            outputs = self.model(**inputs)
            predictions = torch.argmax(outputs.logits, dim=-1)[0]

        # 解码标签（跳过特殊 token 和 ignore_index）
        tags = []
        tokens = self.tokenizer.convert_ids_to_tokens(inputs["input_ids"][0])
        for token, pred in zip(tokens, predictions):
            if token in ("[CLS]", "[SEP]", "[PAD]"):
                continue
            if pred.item() not in self.id2tag:
                continue
            tags.append(self.id2tag[pred.item()])

        return tags

    def extract_entities(self, text: str) -> Dict[str, List[str]]:
        """
        从文本中提取实体（合并 BIO 标签 → 结构化结果）
        返回: {"甲方": ["深圳市腾讯计算机系统有限公司"], "金额": ["伍拾万元整"], ...}
        """
        tags = self.predict_tags(text)
        entities = {}

        current_entity = None
        current_type = None

        for i, tag in enumerate(tags):
            if tag.startswith("B-"):
                # 保存上一个实体
                if current_entity is not None:
                    entities.setdefault(current_type, []).append("".join(current_entity))

                # 开始新实体
                current_type = tag[2:]  # 去掉 "B-" 前缀
                current_entity = [text[i]] if i < len(text) else []
            elif tag.startswith("I-") and current_entity is not None:
                if i < len(text):
                    current_entity.append(text[i])
            elif tag == "O":
                if current_entity is not None:
                    entities.setdefault(current_type, []).append("".join(current_entity))
                current_entity = None
                current_type = None

        # 保存最后一个实体
        if current_entity is not None:
            entities.setdefault(current_type, []).append("".join(current_entity))

        # 后处理：去重、去空、规则过滤
        for k in list(entities.keys()):
            cleaned = [e.strip() for e in entities[k] if e.strip() and len(e.strip()) >= 2]
            if cleaned:
                entities[k] = list(set(cleaned))
            else:
                del entities[k]

        return entities


# ===== 使用示例 =====
if __name__ == "__main__":
    ner = ContractNER("./models/contract_ner")

    contract_text = (
        "合同编号：HT-2024-00188。甲方：深圳市腾讯计算机系统有限公司，"
        "乙方：北京创新科技有限公司。合同总金额为人民币伍拾万元整（￥500,000.00）。"
        "签订日期：2024年5月20日。违约金不超过合同总金额的百分之三十。"
    )

    entities = ner.extract_entities(contract_text)

    print("=" * 50)
    print("合同关键信息抽取结果")
    print("=" * 50)
    for ent_type, values in entities.items():
        print(f"  {ent_type}: {', '.join(values)}")

    # JSON 输出
    import json
    print("\nJSON 格式:")
    print(json.dumps(entities, ensure_ascii=False, indent=2))
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **BIO 标注** | 简单直观，与分词器天然适配 | 无法表达嵌套实体（如"腾讯公司采购部"中公司和部门两个实体） |
| **BERT+TokenClassification** | 预训练语义理解强，微调收敛快 | 长实体跨越多 token 边界容易预测断裂 |
| **子词对齐ignore_index** | 简洁，不污染 loss 计算 | 同一个词的不同 sub-token 预测可能不一致 |
| **规则后处理** | 补充模型不足，确保格式合法 | 规则过多时维护成本高，与 ML 逻辑耦合 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 合同/发票/病历关键字段抽取 | NER + 规则后处理 |
| 嵌套实体（公司>部门>人名） | 多层 NER 或 Span-based 模型 |
| 开放域实体识别（不限定类别） | 大模型 few-shot prompting |
| 实时流式抽取 | 预加载 + 批量推理 |

**不适用场景**：
- 实体间关系抽取（如"A 是 B 的子公司"），NER 只能告诉你 A 和 B 是实体，不能告诉关系
- 超长文档（>2000 tokens）的直接标注，需配合滑窗分段处理

### 4.3 注意事项

1. **中文单字 token 问题**：bert-base-chinese 将大部分中文字切为单字 token，BIO 标签中 B 和 I 的判定粒度极细
2. **标签顺序敏感**：`B-甲方` 后面突然出现 `I-乙方` 是非法序列，后处理应自动截断
3. **ignore_index 陷阱**：`-100` 是 PyTorch CrossEntropyLoss 的默认 ignore_index，必须与标签映射中未使用 ID 区分

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 所有预测都是 O | 标签映射失误，第一个标签从 0 开始而非 1 | 确保 `num_labels` 与标签数一致，O 标签 ID 也为有效值 |
| loss 为 nan | 某样本全为 -100（全被截断） | 数据过滤掉过短文本，或确保至少有一个有效标签 |
| 实体边界总是差一个字 | 子词对齐时后续 token 用 ignore_index 但预测仍参与后处理 | 后处理只取非 ignore 的 token 做实体合并 |

### 4.5 思考题

1. **初级**：如果训练数据中 "B-条款 I-条款 O" 的标签序列在预测时变成 "B-条款 O O"，后处理应该做什么？（提示：B 后面是 O 代表实体只有 1 个 token，应保留而非丢弃）
2. **进阶**：合同中出现 "甲方：A公司（以下简称'买方'）"，括号内的别名也应该被识别为甲方实体。你如何在 BIO 标注中处理这种情况？

（答案将在第7章末尾给出）

### 4.6 第5章思考题答案

**第5章思考题1**：
- epoch=1：欠拟合，train loss 较高，验证 F1 低
- epoch=3：正常收敛，loss 平稳下降
- epoch=10：过拟合，train loss 趋近于 0 但 val loss 开始回升，F1 下降

**第5章思考题2**：
- BERT 可直接支持 100 分类，但需注意：(1) 分类头从 `(768, 5)` 变为 `(768, 100)`，参数增量可忽略；(2) 保证每个类别至少有 50-100 个样本，否则需考虑层次分类或合并稀疏类；(3) 分类头随机初始化重新训练即可收敛
