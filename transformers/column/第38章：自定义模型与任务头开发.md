# 第38章：自定义模型与任务头开发

## 1 项目背景

### 业务场景

客服工单系统目前有两个独立的模型：一个做分类（投诉/咨询/退款），一个做相似度匹配（新工单与历史工单是否重复）。每个模型各自占用 400MB 显存，推理时要调用两次，延迟叠加 50ms + 30ms = 80ms。

技术经理提出："能不能把两个模型合并成一个？既能做分类又能做匹配，共享 BERT 底座，减少一半显存和延迟。"

小陈研究了 Transformers 的多任务头机制，发现虽然可以手动在一个 BERT 上堆两个分类头，但这样做的模型无法被 `AutoModel` 自动加载——因为它的结构和标准 BERT 不同。他需要把这个"双头 BERT"变成一个标准化的、可被 Transformers 生态识别的自定义模型。

### 痛点放大

开发自定义模型需要理解 Transformers 类的继承体系：

```
PretrainedConfig       ← 自定义 Config（模型超参数）
PreTrainedModel        ← 自定义 Model（模型结构 + 权重加载）
PreTrainedTokenizer    ← 可选的 Tokenizer（通常复用）

必须实现的方法:
  Config:  model_type 属性
  Model:   __init__, forward, _init_weights（可选）
  
可选注册:
  AutoConfig.register(), AutoModel.register()
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周一上午 10:00，AI Lab。小陈正在设计 CustomConfig 的字段。

---

**小胖**:"你又要造轮子了？现成的模型不够用？"

**小陈**:"不是造轮子，是组装新车型。我们用 BERT 的发动机（底座），但加两个方向盘（任务头）——一个管分类，一个管匹配。问题是 Transformers 不认识这辆车，得给它办'行驶证'（注册到 AutoClass）。"

**小白**:"自定义模型的最小实现需要几个文件？我做过一次，至少需要 Config 类 + Model 类，而且要正确设置 `config_class` 和 `model_type`。"

**大师**:"对，让我把自定义模型的完整开发流程讲清楚。

**第一步：定义 PretrainedConfig。** 继承 `PretrainedConfig`，必须设置 `model_type`（唯一标识符）。你可以在 config 中添加任何自定义字段——多任务头的配置、dropout 率、额外的维度参数。保存模型时会自动保存为 `config.json`。

**第二步：定义 PreTrainedModel。** 继承 `PreTrainedModel`，必须设置 `config_class`（关联到你的 Config）。在 `__init__` 中：
1. 加载预训练的 BERT 底座（`AutoModel.from_pretrained` 或手动构建）
2. 添加自定义的任务头（nn.Linear 等）
3. 调用 `self.post_init()`（Transformers 要求，用于权重初始化）

在 `forward()` 中：
1. 输入通过 BERT 底座得到 hidden states
2. 分别送入不同的任务头
3. 返回自定义的 `ModelOutput`（包含所有任务的结果）

**第三步：注册到 AutoClass。** 这样 `AutoModel.from_pretrained("你的模型路径")` 就能自动加载：

```python
AutoConfig.register("my_dual_model", MyDualConfig)
AutoModel.register(MyDualConfig, MyDualModel)
```

**关键：`post_init()` 的作用。** 它调用了 `_init_weights()`——对新增的层做随机初始化。如果你不调用它，新增的 nn.Linear 就用 PyTorch 默认的初始化（不一样！），可能导致训练不稳定。"

**技术映射总结**：
- Config = 模型的"配置单"（发动机型号、马力、座位数）
- Model = 模型的"实际构造"（发动机 + 车身 + 座椅）
- model_type = 车型编号，注册后 AutoClass 靠它识别
- post_init = 出厂质检，确保每个零件初始化正确

### 3.2 双任务自定义模型

```python
# custom_dual_model.py
"""自定义双任务模型 —— 分类 + 匹配"""

import torch
import torch.nn as nn
from typing import Optional, Tuple, Dict
from dataclasses import dataclass

from transformers import (
    PretrainedConfig, PreTrainedModel, AutoModel, AutoConfig, AutoTokenizer,
)
from transformers.modeling_outputs import ModelOutput


# ===== Step 1: 自定义 Config =====
class MyDualConfig(PretrainedConfig):
    model_type = "my_dual_model"  # ★ 唯一标识符

    def __init__(
        self,
        base_model_name: str = "bert-base-chinese",
        num_classify_labels: int = 5,
        match_hidden_size: int = 256,
        classify_dropout: float = 0.1,
        match_dropout: float = 0.1,
        **kwargs
    ):
        super().__init__(**kwargs)
        self.base_model_name = base_model_name
        self.num_classify_labels = num_classify_labels
        self.match_hidden_size = match_hidden_size
        self.classify_dropout = classify_dropout
        self.match_dropout = match_dropout


# ===== Step 2: 自定义 ModelOutput =====
@dataclass
class DualModelOutput(ModelOutput):
    """双任务模型的输出"""
    loss: Optional[torch.FloatTensor] = None
    classify_logits: Optional[torch.FloatTensor] = None
    match_logits: Optional[torch.FloatTensor] = None
    hidden_states: Optional[Tuple[torch.FloatTensor]] = None
    attentions: Optional[Tuple[torch.FloatTensor]] = None


# ===== Step 3: 自定义 PreTrainedModel =====
class MyDualModel(PreTrainedModel):
    config_class = MyDualConfig  # ★ 关联 Config

    def __init__(self, config: MyDualConfig):
        super().__init__(config)

        # 1. BERT 底座
        self.bert = AutoModel.from_pretrained(config.base_model_name)
        hidden_size = self.bert.config.hidden_size

        # 2. 分类任务头
        self.classify_head = nn.Sequential(
            nn.Dropout(config.classify_dropout),
            nn.Linear(hidden_size, hidden_size // 2),
            nn.ReLU(),
            nn.Linear(hidden_size // 2, config.num_classify_labels),
        )

        # 3. 匹配任务头（输出 2分类: 匹配/不匹配）
        self.match_head = nn.Sequential(
            nn.Dropout(config.match_dropout),
            nn.Linear(hidden_size * 3, config.match_hidden_size),  # 拼接 [u; v; |u-v|]
            nn.ReLU(),
            nn.Linear(config.match_hidden_size, 2),
        )

        # 4. 初始化新增的权重
        self.post_init()

    def forward(
        self,
        input_ids: torch.Tensor = None,
        attention_mask: torch.Tensor = None,
        token_type_ids: torch.Tensor = None,
        # 匹配任务需要的第二个文本
        input_ids_2: torch.Tensor = None,
        attention_mask_2: torch.Tensor = None,
        # 标签
        classify_labels: torch.Tensor = None,
        match_labels: torch.Tensor = None,
        # 其他
        output_hidden_states: bool = False,
        output_attentions: bool = False,
        return_dict: bool = True,
    ) -> DualModelOutput:
        """
        前向传播

        分类任务: 单文本 → BERT → classify_head → N 分类
        匹配任务: 双文本 → BERT × 2 → 拼接 → match_head → 2 分类
        """
        # ===== BERT 编码 =====
        bert_outputs = self.bert(
            input_ids=input_ids,
            attention_mask=attention_mask,
            token_type_ids=token_type_ids,
            output_hidden_states=output_hidden_states,
            output_attentions=output_attentions,
            return_dict=True,
        )
        pooled = bert_outputs.pooler_output  # (batch, hidden)

        # ===== 分类任务头 =====
        classify_logits = self.classify_head(pooled)  # (batch, num_classify_labels)

        # ===== 匹配任务头 =====
        match_logits = None
        if input_ids_2 is not None:
            bert_outputs_2 = self.bert(
                input_ids=input_ids_2,
                attention_mask=attention_mask_2,
                return_dict=True,
            )
            pooled_2 = bert_outputs_2.pooler_output

            # 拼接 [u; v; |u-v|]
            diff = torch.abs(pooled - pooled_2)
            match_input = torch.cat([pooled, pooled_2, diff], dim=-1)
            match_logits = self.match_head(match_input)  # (batch, 2)

        # ===== Loss 计算 =====
        loss = None
        if classify_labels is not None or match_labels is not None:
            loss = 0
            if classify_labels is not None:
                loss += nn.CrossEntropyLoss()(classify_logits, classify_labels)
            if match_labels is not None and match_logits is not None:
                loss += nn.CrossEntropyLoss()(match_logits, match_labels)

        if not return_dict:
            return (loss, classify_logits, match_logits) + bert_outputs[2:]

        return DualModelOutput(
            loss=loss,
            classify_logits=classify_logits,
            match_logits=match_logits,
            hidden_states=bert_outputs.hidden_states,
            attentions=bert_outputs.attentions,
        )


# ===== Step 4: 注册到 AutoClass =====
AutoConfig.register("my_dual_model", MyDualConfig)
AutoModel.register(MyDualConfig, MyDualModel)


# ===== 使用示例 =====
if __name__ == "__main__":
    # 创建配置
    config = MyDualConfig(
        base_model_name="prajjwal1/bert-tiny",
        num_classify_labels=5,
    )

    # 创建模型
    model = MyDualModel(config)
    print(f"模型类: {type(model).__name__}")
    total = sum(p.numel() for p in model.parameters())
    print(f"总参数量: {total:,}")

    # 保存模型
    tokenizer = AutoTokenizer.from_pretrained("prajjwal1/bert-tiny")
    model.save_pretrained("./my_dual_model")
    tokenizer.save_pretrained("./my_dual_model")
    print("模型已保存至 ./my_dual_model")

    # ★ 用 AutoModel 重新加载！
    loaded_model = AutoModel.from_pretrained("./my_dual_model")
    print(f"AutoModel 加载成功: {type(loaded_model).__name__}")

    # 单文本推理（分类）
    text1 = "我要投诉这个产品质量问题"
    inputs1 = tokenizer(text1, return_tensors="pt")
    with torch.no_grad():
        out = loaded_model(**inputs1)
    print(f"\n分类 logits shape: {out.classify_logits.shape}")  # (1, 5)
    pred_class = torch.argmax(out.classify_logits, dim=-1).item()
    print(f"预测类别 ID: {pred_class}")

    # 双文本推理（匹配）
    text2 = "商品质量有问题我要投诉"
    inputs2 = tokenizer(text2, return_tensors="pt")
    with torch.no_grad():
        out = loaded_model(
            input_ids=inputs1["input_ids"],
            attention_mask=inputs1["attention_mask"],
            input_ids_2=inputs2["input_ids"],
            attention_mask_2=inputs2["attention_mask"],
        )
    match_pred = torch.argmax(out.match_logits, dim=-1).item()
    print(f"匹配预测: {'匹配' if match_pred == 1 else '不匹配'} "
          f"(logits: {out.match_logits.tolist()})")
```

"""将自定义模型打包为可发布的格式"""

import json
import os

def create_model_card(model_dir: str, model_name: str, description: str,
                      tasks: list, metrics: dict):
    """创建模型卡片 (README.md)"""
    card = f"""---
language: zh
license: apache-2.0
tags:
{chr(10).join(f'- {t}' for t in tasks)}
metrics:
{chr(10).join(f'- {k}: {v}' for k, v in metrics.items())}
---

# {model_name}

{description}

## 架构
- 底座: BERT-base-chinese
- 任务头: 分类 (5 类) + 文本匹配 (2 类)

## 使用方式
```python
from transformers import AutoModel, AutoTokenizer

model = AutoModel.from_pretrained("{model_name}")
tokenizer = AutoTokenizer.from_pretrained("{model_name}")

# 分类任务
inputs = tokenizer("测试文本", return_tensors="pt")
outputs = model(**inputs)
print(outputs.classify_logits)
```
"""
    with open(os.path.join(model_dir, "README.md"), "w", encoding="utf-8") as f:
        f.write(card)


def validate_model_structure(model_dir: str) -> dict:
    """验证模型目录结构完整性"""
    path = Path(model_dir)
    required = ["config.json", "model.safetensors"]
    optional = ["tokenizer_config.json", "vocab.txt", "README.md"]

    result = {"valid": True, "issues": []}

    for f in required:
        if not (path / f).exists() and not list(path.glob("*.safetensors")):
            result["valid"] = False
            result["issues"].append(f"缺少必需文件: {f}")

    config_file = path / "config.json"
    if config_file.exists():
        with open(config_file) as f:
            config = json.load(f)
        if "model_type" not in config:
            result["issues"].append("config.json 缺少 model_type 字段")
        if config.get("model_type") != "my_dual_model":
            result["issues"].append(
                f"model_type 不匹配: 实际={config.get('model_type')}"
            )

    return result


if __name__ == "__main__":

    print(f"模型验证: {'✓ 通过' if result['valid'] else '✗ 有问题'}")

        print(f"  - {issue}")

# test_custom_model.py
import pytest
import torch
import os
from custom_dual_model import MyDualConfig, MyDualModel, DualModelOutput

class TestCustomModel:
    def test_config_creation(self):
        config = MyDualConfig(num_classify_labels=7)
        assert config.model_type == "my_dual_model"
        assert config.num_classify_labels == 7

    def test_model_creation(self):
        config = MyDualConfig(base_model_name="prajjwal1/bert-tiny")
        model = MyDualModel(config)
        assert isinstance(model, MyDualModel)

    def test_forward_classify(self):
        config = MyDualConfig(base_model_name="prajjwal1/bert-tiny")
        model = MyDualModel(config)
        model.eval()
        dummy = torch.randint(0, 1000, (1, 16))
        mask = torch.ones(1, 16)
        out = model(input_ids=dummy, attention_mask=mask)
        assert out.classify_logits.shape == (1, config.num_classify_labels)

    def test_forward_match(self):
        config = MyDualConfig(base_model_name="prajjwal1/bert-tiny")
        model = MyDualModel(config)
        model.eval()
        t1 = torch.randint(0, 1000, (1, 16))
        mask1 = torch.ones(1, 16)
        t2 = torch.randint(0, 1000, (1, 12))
        mask2 = torch.ones(1, 12)
        out = model(input_ids=t1, attention_mask=mask1,
                    input_ids_2=t2, attention_mask_2=mask2)
        assert out.match_logits.shape == (1, 2)

    def test_save_and_load(self, tmp_path):
        config = MyDualConfig(base_model_name="prajjwal1/bert-tiny")
        model = MyDualModel(config)
        save_dir = str(tmp_path / "test_model")
        model.save_pretrained(save_dir)

        from transformers import AutoModel
        loaded = AutoModel.from_pretrained(save_dir)
        assert isinstance(loaded, MyDualModel)

    def test_loss(self):
        config = MyDualConfig(base_model_name="prajjwal1/bert-tiny")
        model = MyDualModel(config)
        t1 = torch.randint(0, 1000, (2, 8))
        mask = torch.ones(2, 8)
        out = model(input_ids=t1, attention_mask=mask,
                    classify_labels=torch.tensor([0, 1]))

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **自定义架构** | 完全控制模型结构，可实现任意多任务组合 | 需要理解 Transformers 的类继承体系 |
| **AutoClass 注册** | 加载方式与标准模型一致 `AutoModel.from_pretrained()` | 注册是全局的，可能污染其他代码 |
| **ModelOutput** | 结构化返回，支持 loss 计算 | 每个任务需要字段设计 |
| **模块打包** | 可发布到 HuggingFace Hub 共享 | 需要完整的文档和测试 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 多任务合并（分类+匹配） | 共享底座 + 多任务头 |
| 特殊模型结构（图神经网络+Transformer） | 继承 PreTrainedModel 自定义 |
| 内部算法快速实验 | 不注册 AutoClass，手动创建 |
| 团队共享标准组件 | 注册 AutoClass + 发布到私有 Hub |

**不适用场景**：
- 对标准模型仅改配置（如改层数）→ 用 `from_pretrained` 传 config 即可
- 仅添加一个 nn.Linear 头 → 直接用 AutoModel + 自定义头，不需要完整自定义

### 4.3 注意事项

1. **`post_init()` 必须调用**：否则新增层的初始化不符合 Transformers 规范
2. **config_class 必须设置**：这是 PreTrainedModel 找到 config 类的唯一方式
3. **model_type 唯一性**：不能与已有模型重名（BERT=bert, GPT-2=gpt2 等已占用）

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 加载报 `Unrecognized model` | model_type 未注册或拼写错误 | 确保 `AutoConfig.register()` 在 `from_pretrained` 之前执行 |
| 权重 shape 不匹配 | config 中的 hidden_size 等参数与保存时不一致 | 保存 config 后再加载，不要手动修改 config 参数 |
| 训练不收敛 | 新增层初始化有问题 | 确认 `post_init()` 被调用 |
