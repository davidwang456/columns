# 第16章：基础篇综合实战：企业客服文本智能分派系统

## 1 项目背景

### 业务场景

经过前面 15 章的学习，团队已经掌握了 Transformers 的核心能力：Pipeline 快速推理、Tokenizer 文本预处理、文本分类/NER/QA 任务训练、Embedding 语义搜索、Trainer 训练框架、模型评估、推理封装和 FastAPI 部署。现在，技术经理要求将这些分散的能力整合为一个完整的"企业客服文本智能分派系统"。

某中型电商公司的客服中心日均处理工单 8000 条。目前的痛点：(1) 人工分派平均耗时 45 秒/条，含人工阅读+判断+转发；(2) 新手分派错误率 22%；(3) 相似问题重复回答，客服效率低；(4) 高峰期工单积压，用户等待超 5 分钟。CTO 要求在双十一大促前上线一套智能分派系统——模型自动分派 + 不确定工单转人工 + 相似历史问题推荐。

### 痛点放大

前面的章节各自独立，但真实系统需要把组件串联起来。几个关键的集成挑战：

1. **多模型协作**：分类模型做分派，Embedding 做相似问题推荐——两个模型如何共存在一个服务中？
2. **数据闭环**：分类模型上线后，线上数据如何回流到训练集？如何触发增量训练？
3. **全链路可观测**：从请求进入 → 模型推理 → 结果返回，每个环节的延迟和状态都需要可追踪
4. **模型训练与部署衔接**：训练好的模型如何一键部署到生产环境？如何做版本管理？

```
用户工单 → API Gateway → ┌─ 分类模型(主) → 自动分派
                         ├─ 置信度 < 阈值 → 转人工
                         └─ Embedding模型 → 推荐相似历史问答
                                │
                          ┌─────▼─────┐
                          │ 数据闭环   │
                          │(人工标注回流)│
                          └─────┬─────┘
                                ▼
                          增量训练 → 新版本模型
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周一上午 10:00，项目 kickoff 会议。大屏上投影着系统架构图。

**小胖**:"终于到综合实战了！前面学了那么多，这下可以组装成一台'变形金刚'了吧？"

**小陈**:"是的。但我有个问题——分类模型和 Embedding 模型能不能放在一个服务里？两个模型加起来 800MB 内存，会不会爆？"

**小白**:"不仅要考虑内存，还要考虑延迟。如果一个请求要串行调用分类 + Embedding 两个模型，延迟就是两者之和（50ms + 30ms = 80ms）。能不能并行调用？另外，模型训练产出后怎么自动化部署？"

**大师**:"好问题。综合实战的核心就是**组件编排**。我来讲系统设计的五个关键决策。

**决策一：单服务 vs 微服务？** 对于日均 8000 条工单（峰值 QPS 约 5），单服务即可承载。一个 FastAPI 服务内加载两个模型（分类 + Embedding）作为全局单例，总内存约 1GB，单机完全够用。如果未来 QPS 破 100，再拆分为微服务不迟。

**决策二：请求处理流程。** 收到工单文本后，并行执行两条路径——路径 A（分类模型）：文本 → tokenize → 分类推理 → 标签+置信度。路径 B（Embedding 模型）：文本 → 向量化 → FAISS 检索 Top 3 相似历史问答。两条路径的结果合并后返回。

**决策三：数据闭环。** 分类置信度 < 0.5 的工单转人工处理后，客服会标注正确类别。这些标注数据定期（每周）导出，合并到训练集，触发增量训练，生成新版本模型，通过灰度发布替换旧模型。

**决策四：限流与容错。** 大促期间 QPS 可能突发 10 倍。需要请求级限流（单 IP QPS 限制）+ 服务级限流（总 QPS 上限）+ 优雅降级（队列满时返回"系统繁忙"）。如果是单 worker 处理不过来，还要考虑多 worker 或异步队列。

**决策五：训练到部署的自动化桥梁。** 训练完成后自动触发模型评估 → 达标则自动注册版本 → 复制模型文件到生产目录 → 通过热加载或重启服务切换模型。这个环节最容易断链，需要脚本自动化。"

**技术映射总结**：
- 单服务多模型 = 合租省房租，小流量没必要拆微服务
- 并行推理 = 分类和 Embedding 同时跑，不互相等待
- 数据闭环 = 线上标注数据回流到训练集，模型持续进化
- 限流容错 = 门口排队发号，超出承载能力就告知"等一等"
- 自动化桥梁 = 工厂流水线，从原材料到成品自动流转

```python
# train_pipeline.py
"""客服工单分类模型 —— 完整训练管道"""

import os
import json
import numpy as np
import pandas as pd
from datasets import Dataset, DatasetDict
from transformers import (
    AutoTokenizer, AutoModelForSequenceClassification,
    Trainer, TrainingArguments, DataCollatorWithPadding,
    EarlyStoppingCallback,
)
from sklearn.metrics import (
    accuracy_score, precision_recall_fscore_support,
    confusion_matrix, classification_report
)
import torch

# ===== 1. 数据准备（模拟真实工单） =====
def prepare_ticket_data():
    np.random.seed(42)
    categories = ["投诉", "咨询", "退款退货", "物流问题", "售后"]

    templates = {
        "投诉": ["{product}质量太差了！我要投诉！", "你们{service}态度极其恶劣",
                "这是第{num}次出问题了，必须给个说法"],
        "咨询": ["请问{product}支持{feature}吗？", "{product}的{feature}怎么操作？",
                "有没有关于{product}的详细说明？"],
        "退款退货": ["收到的{product}跟描述不符，要退款", "我要退{product}，质量问题",
                   "{product}用了一次就坏了，退钱"],
        "物流问题": ["{product}发货太慢了，等了{num}天", "物流显示签收但我没收到{product}",
                   "快递员把{product}弄丢了"],
        "售后": ["{product}的保修怎么申请？", "{product}屏幕碎了能修吗？",
                "购买的{product}需要换货"],
    }

    products = ["手机", "耳机", "充电器", "衣服", "鞋子", "手表"]
    services = ["客服", "售后", "物流", "快递"]
    features = ["退货", "换货", "保修", "退款"]

    texts, labels = [], []
    for cat in categories:
        for _ in range(80):  # 每类 80 条
            tmpl = np.random.choice(templates[cat])
            text = tmpl.format(
                product=np.random.choice(products),
                service=np.random.choice(services),
                feature=np.random.choice(features),
                num=np.random.randint(1, 10),
            )
            texts.append(text)
            labels.append(cat)

    df = pd.DataFrame({"text": texts, "label": labels})
    df = df.drop_duplicates(subset=["text"])
    print(f"数据分布:\n{df['label'].value_counts()}")
    return df


# ===== 2. 构建 Dataset =====
df = prepare_ticket_data()
label2id = {label: i for i, label in enumerate(sorted(df["label"].unique()))}
id2label = {i: label for label, i in label2id.items()}

df["label_id"] = df["label"].map(label2id)

# 分层划分
from sklearn.model_selection import train_test_split
train_df, temp_df = train_test_split(df, test_size=0.3, stratify=df["label"], random_state=42)
val_df, test_df = train_test_split(temp_df, test_size=0.5, stratify=temp_df["label"], random_state=42)

dataset = DatasetDict({
    "train": Dataset.from_pandas(train_df),
    "validation": Dataset.from_pandas(val_df),
    "test": Dataset.from_pandas(test_df),
})
print(f"数据集: train={len(dataset['train'])}, val={len(dataset['validation'])}, test={len(dataset['test'])}")

# ===== 3. Tokenize =====
model_name = "bert-base-chinese"
tokenizer = AutoTokenizer.from_pretrained(model_name)

def tokenize_fn(examples):
    return tokenizer(examples["text"], truncation=True, max_length=128)

tokenized = dataset.map(tokenize_fn, batched=True)

# ===== 4. 模型训练 =====
model = AutoModelForSequenceClassification.from_pretrained(
    model_name, num_labels=len(label2id),
    id2label=id2label, label2id=label2id,
)

def compute_metrics(eval_pred):
    logits, labels = eval_pred
    preds = np.argmax(logits, axis=-1)
    accuracy = accuracy_score(labels, preds)
    precision, recall, f1, _ = precision_recall_fscore_support(
        labels, preds, average="macro", zero_division=0
    )
    return {"accuracy": accuracy, "precision": precision, "recall": recall, "f1": f1}

training_args = TrainingArguments(
    output_dir="./output/ticket_classifier",
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    num_train_epochs=5,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    metric_for_best_model="f1",
    logging_strategy="steps",
    logging_steps=10,
    fp16=torch.cuda.is_available(),
    report_to="none",
    save_total_limit=2,
)

trainer = Trainer(
    model=model, args=training_args,
    train_dataset=tokenized["train"],
    eval_dataset=tokenized["validation"],
    tokenizer=tokenizer,
    data_collator=DataCollatorWithPadding(tokenizer),
    compute_metrics=compute_metrics,
    callbacks=[EarlyStoppingCallback(early_stopping_patience=2)],
)

print("开始训练...")
trainer.train()

# ===== 5. 评估与保存 =====
test_results = trainer.evaluate(tokenized["test"])
print(f"\n测试集结果: {test_results}")

# 生成混淆矩阵
preds_output = trainer.predict(tokenized["test"])
y_pred = np.argmax(preds_output.predictions, axis=-1)
y_true = preds_output.label_ids

cm = confusion_matrix(y_true, y_pred)
print("\n混淆矩阵:")
print(cm)
print("\n分类报告:")
print(classification_report(y_true, y_pred, target_names=list(id2label.values()), zero_division=0))

# 保存最终模型
save_path = "./models/ticket_classifier_final"
os.makedirs(save_path, exist_ok=True)
trainer.save_model(save_path)
tokenizer.save_pretrained(save_path)

# 保存元数据
meta = {
    "label2id": label2id, "id2label": id2label,
    "test_metrics": test_results,
    "model_name": model_name,
    "train_date": pd.Timestamp.now().isoformat(),
}
with open(os.path.join(save_path, "training_meta.json"), "w") as f:
    json.dump(meta, f, ensure_ascii=False, indent=2)
print(f"模型已保存至 {save_path}")
```

### 3.3 系统架构与核心服务

```python
# ticket_system.py
"""企业客服文本智能分派系统 —— 完整生产级服务"""

import time, uuid, logging, asyncio, json, os
from typing import List, Dict, Optional
from contextlib import asynccontextmanager
from collections import defaultdict

import torch, numpy as np
from fastapi import FastAPI, HTTPException, Request, Query
from fastapi.responses import JSONResponse
from pydantic import BaseModel, Field
from transformers import AutoTokenizer, AutoModelForSequenceClassification
from sentence_transformers import SentenceTransformer

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger(__name__)

# ===== 数据模型 =====
class TicketRequest(BaseModel):
    text: str = Field(..., min_length=1, max_length=5000)
    ticket_id: Optional[str] = None

class TicketResponse(BaseModel):
    ticket_id: str
    category: str
    category_confidence: float
    needs_review: bool
    similar_questions: List[Dict]
    latency_ms: float
    trace_id: str

class BatchTicketRequest(BaseModel):
    tickets: List[TicketRequest] = Field(..., min_items=1, max_items=50)

class HealthResponse(BaseModel):
    status: str
    classifier_loaded: bool
    embedder_loaded: bool
    faiss_ready: bool
    queue_depth: int
    uptime_seconds: float

# ===== 简易限流器 =====
class SimpleRateLimiter:
    def __init__(self, max_requests: int, window_seconds: float = 1.0):
        self.max_requests = max_requests
        self.window = window_seconds
        self.clients = defaultdict(list)

    def is_allowed(self, client_id: str) -> bool:
        now = time.time()
        history = self.clients[client_id]
        history[:] = [t for t in history if now - t < self.window]
        if len(history) >= self.max_requests:
            return False
        history.append(now)
        return True

# ===== 全局单例 =====
_classifier_model = None
_classifier_tokenizer = None
_embedder_model = None
_faq_texts = []
_faq_embeddings = None
_start_time = None
_rate_limiter = SimpleRateLimiter(max_requests=20, window_seconds=1.0)
_request_counts = defaultdict(int)

MODEL_PATH = "./models/ticket_classifier_final"
EMBEDDER_NAME = "paraphrase-multilingual-MiniLM-L12-v2"
CONFIDENCE_THRESHOLD = 0.6

@asynccontextmanager
async def lifespan(app: FastAPI):
    global _classifier_model, _classifier_tokenizer
    global _embedder_model, _faq_embeddings, _start_time

    logger.info("系统启动中...")
    _start_time = time.time()

    # 1. 加载分类模型
    logger.info("[1/3] 加载分类模型...")
    _classifier_tokenizer = AutoTokenizer.from_pretrained(MODEL_PATH)
    _classifier_model = AutoModelForSequenceClassification.from_pretrained(MODEL_PATH)
    _classifier_model.eval()
    device = "cuda" if torch.cuda.is_available() else "cpu"
    _classifier_model.to(device)
    logger.info(f"  分类模型加载完成 (device={device})")

    # 2. 加载 Embedding 模型
    logger.info("[2/3] 加载 Embedding 模型...")
    _embedder_model = SentenceTransformer(EMBEDDER_NAME)
    logger.info("  Embedding 模型加载完成")

    # 3. 构建 FAQ 向量索引
    logger.info("[3/3] 构建 FAQ 索引...")
    _faq_texts = _load_faq_database()
    _faq_embeddings = _embedder_model.encode(
        _faq_texts, normalize_embeddings=True
    ).astype(np.float32)
    logger.info(f"  FAQ 索引构建完成 ({len(_faq_texts)} 条)")

    logger.info(f"系统就绪，启动耗时 {time.time() - _start_time:.1f}s")
    yield
    logger.info("系统关闭")

def _load_faq_database():

    return [

app = FastAPI(title="客服文本智能分派系统", version="2.0.0", lifespan=lifespan)

@app.middleware("http")
async def trace_middleware(request: Request, call_next):
    trace_id = str(uuid.uuid4())[:8]
    request.state.trace_id = trace_id
    start = time.time()
    response = await call_next(request)
    elapsed = (time.time() - start) * 1000
    logger.info(f"trace_id={trace_id} path={request.url.path} "
                f"status={response.status_code} latency={elapsed:.1f}ms")
    response.headers["X-Trace-ID"] = trace_id
    return response

@app.get("/health", response_model=HealthResponse)
async def health():
    return HealthResponse(
        status="healthy" if _classifier_model and _embedder_model else "not_ready",
        classifier_loaded=_classifier_model is not None,
        embedder_loaded=_embedder_model is not None,
        faiss_ready=_faq_embeddings is not None,
        queue_depth=sum(_request_counts.values()),
        uptime_seconds=round(time.time() - _start_time, 1) if _start_time else 0,
    )

@app.post("/dispatch", response_model=TicketResponse)
async def dispatch(request: TicketRequest, req: Request):
    global _classifier_model, _classifier_tokenizer
    global _embedder_model, _faq_texts, _faq_embeddings

    ticket_id = request.ticket_id or str(uuid.uuid4())[:8]
    trace_id = getattr(req.state, "trace_id", "unknown")
    client_ip = req.client.host if req.client else "unknown"
    start = time.time()

    # 限流检查
    if not _rate_limiter.is_allowed(client_ip):
        raise HTTPException(429, "请求过于频繁，请稍后重试")

    if not _classifier_model or not _embedder_model:
        raise HTTPException(503, "模型未加载完成")

    _request_counts[ticket_id] = 1

    try:
        # ===== 路径 A: 分类推理 =====
        inputs = _classifier_tokenizer(
            request.text, max_length=128, truncation=True,
            padding="max_length", return_tensors="pt"
        ).to(_classifier_model.device)

        with torch.no_grad():
            outputs = _classifier_model(**inputs)
            probs = torch.softmax(outputs.logits, dim=-1)
            top_prob, top_label = torch.max(probs, dim=-1)

        confidence = top_prob.item()
        label_id = top_label.item()
        id2label = _classifier_model.config.id2label
        category = id2label.get(str(label_id), f"class_{label_id}")
        needs_review = confidence < CONFIDENCE_THRESHOLD

        # ===== 路径 B: 相似问题检索 =====
        query_vec = _embedder_model.encode(
            [request.text], normalize_embeddings=True
        ).astype(np.float32)
        scores = np.dot(_faq_embeddings, query_vec[0])
        top_indices = np.argsort(scores)[::-1][:5]

        similar = []
        for rank, idx in enumerate(top_indices):
            if scores[idx] < 0.3:
                continue
            similar.append({
                "rank": rank + 1,
                "question": _faq_texts[idx][:120],
                "score": round(float(scores[idx]), 4),
            })

        latency = (time.time() - start) * 1000

        # 结构化日志
        logger.info(
            f"dispatch | ticket_id={ticket_id} trace_id={trace_id} "
            f"category={category} confidence={confidence:.3f} "
            f"review={needs_review} similar_count={len(similar)} "
            f"latency={latency:.1f}ms"
        )

        # 低置信度记录到反馈日志（后续人工标注）
        if needs_review:
            _log_low_confidence(request.text, category, confidence, ticket_id)

        return TicketResponse(
            ticket_id=ticket_id, category=category,
            category_confidence=round(confidence, 4),
            needs_review=needs_review,
            similar_questions=similar[:3],
            latency_ms=round(latency, 1),
            trace_id=trace_id,
        )

    except Exception as e:
        logger.error(f"推理失败 ticket_id={ticket_id}: {e}")
        raise HTTPException(500, f"推理服务异常: {str(e)}")
    finally:
        _request_counts.pop(ticket_id, None)

@app.post("/batch_dispatch")
async def batch_dispatch(request: BatchTicketRequest, req: Request):
    """批量分派接口（最多 50 条）"""
    results = []
    for ticket in request.tickets:
        resp = await dispatch(ticket, req)
        results.append(resp.dict())
    return {"results": results, "total": len(results)}


def _log_low_confidence(text: str, category: str, confidence: float, ticket_id: str):
    """记录低置信度样本（用于数据闭环）"""
    feedback_dir = "./data/feedback"
    os.makedirs(feedback_dir, exist_ok=True)
    record = {
        "ticket_id": ticket_id,
        "text": text,
        "predicted_category": category,
        "confidence": confidence,
        "timestamp": time.strftime("%Y-%m-%d %H:%M:%S"),
    }
    with open(os.path.join(feedback_dir, "low_confidence.jsonl"), "a", encoding="utf-8") as f:
        f.write(json.dumps(record, ensure_ascii=False) + "\n")
```

### 3.4 数据闭环脚本

```python
# data_feedback_loop.py
"""数据闭环：线上标注回流 → 增量训练 → 自动注册"""

import os, json, subprocess
from datetime import datetime
import pandas as pd
from datasets import load_from_disk, Dataset, concatenate_datasets

FEEDBACK_DIR = "./data/feedback"
TRAIN_DATA_DIR = "./data/processed_tickets"
MODEL_REGISTRY = "./model_registry"
LOW_CONF_FILE = os.path.join(FEEDBACK_DIR, "low_confidence.jsonl")


def collect_low_confidence():
    """读取低置信度样本，等待人工标注"""
    if not os.path.exists(LOW_CONF_FILE):
        return []
    samples = []
    with open(LOW_CONF_FILE, "r", encoding="utf-8") as f:
        for line in f:
            samples.append(json.loads(line))
    return samples


def annotate_samples(samples):
    """模拟人工标注（实际中由运营团队通过标注平台完成）"""
    # 简单规则模拟：将低置信样本按关键词标注
    keyword_map = {
        "投诉": "投诉", "差": "投诉", "坏": "退款退货", "慢": "物流问题",
        "退款": "退款退货", "问": "咨询", "怎么": "咨询", "保修": "售后",
    }
    annotated = []
    for s in samples:
        label = "咨询"  # 默认
        for kw, cat in keyword_map.items():
            if kw in s["text"]:
                label = cat
                break
        annotated.append({
            "text": s["text"],
            "label": label,
            "source": "feedback",
            "date": datetime.now().strftime("%Y-%m-%d"),
            "original_prediction": s["predicted_category"],
        })
    return annotated


def merge_and_retrain():
    """合并反馈数据并触发增量训练"""
    samples = collect_low_confidence()
    if not samples:
        print("没有待标注的低置信度样本")
        return

    annotated = annotate_samples(samples)
    print(f"标注了 {len(annotated)} 条反馈数据")

    # 加载现有训练数据
    try:
        existing = load_from_disk(TRAIN_DATA_DIR)
        existing_train = existing["train"]
    except:
        existing_train = Dataset.from_list([])

    # 合并
    feedback_ds = Dataset.from_list(annotated)
    merged = concatenate_datasets([existing_train, feedback_ds])
    print(f"数据合并: {len(existing_train)} → {len(merged)} 条")

    # 保存新版本数据
    version = datetime.now().strftime("%Y%m%d_%H%M")
    new_data_path = f"{TRAIN_DATA_DIR}_v{version}"
    merged.save_to_disk(new_data_path)

    # 触发增量训练
    print(f"触发增量训练: {new_data_path}")
    # subprocess.run(["python", "train_pipeline.py", "--data", new_data_path])

    # 清理已处理的反馈
    os.rename(LOW_CONF_FILE, LOW_CONF_FILE + f".processed_{version}")
    print("数据闭环完成")


def register_model(model_path: str, version: str, metrics: dict):
    """注册模型到版本管理"""
    os.makedirs(MODEL_REGISTRY, exist_ok=True)
    entry = {
        "version": version,
        "model_path": model_path,
        "registered_at": datetime.now().isoformat(),
        "metrics": metrics,
    }
    registry_file = os.path.join(MODEL_REGISTRY, "registry.json")
    registry = []
    if os.path.exists(registry_file):
        with open(registry_file) as f:
            registry = json.load(f)
    registry.append(entry)
    with open(registry_file, "w") as f:
        json.dump(registry, f, ensure_ascii=False, indent=2)
    print(f"模型已注册: v{version} → {model_path}")


if __name__ == "__main__":
    merge_and_retrain()
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **单服务多模型** | 部署简单，延迟低（无跨服务网络开销） | 内存占用叠加，更新任一模型需重启整个服务 |
| **并行推理** | 分类和检索同时完成，总延迟≈max(分类, 检索) | 多模型同时推理对 CPU/GPU 资源竞争 |
| **数据闭环** | 线上数据持续回流，模型与时俱进 | 标注质量依赖人工，错误标注会污染模型 |
| **结构化日志** | trace_id 串联全链路，易于排查和监控 | 日志量随时间增长，需设置保留策略 |
| **Docker 化** | 一键部署，环境一致 | 镜像体积需持续优化（CPU 版 PyTorch 可缩小 50%） |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 日均 < 5 万条工单 | 单服务多模型 |
| 日均 > 10 万条 | 微服务化（分类服务 + 检索服务独立部署） |
| 需要热切换模型 | 模型版本路由 + 配置中心 |
| 多租户隔离 | 按租户加载不同模型实例 |

**不适用场景**：
- 需要实时流式输出的场景（当前为请求-响应模式）
- 超大规模 FAQ 库（>10万条）→ FAISS Flat 检索延迟不可接受，需 IVF/HNSW 索引

### 4.3 注意事项

1. **模型内存估算**：分类 BERT（400MB） + Embedding 模型（500MB） + FAISS 索引（100MB）≈ 1GB，确保服务器内存 > 2GB
2. **并发控制**：GPU 推理时建议用信号量限制并发数，避免显存溢出
3. **日志脱敏**：工单文本可能含手机号、订单号等敏感信息，日志中需做脱敏处理

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 启动时 OOM | 两个模型同时加载到 CPU 内存超限 | 分步加载，先加载完一个再加载另一个 |
| 分类和检索结果不一致 | 两个模型用了不同的 tokenizer 预处理 | 确保分类模型的 tokenizer 配置与训练时一致 |
| 数据闭环后模型变差 | 反馈数据分布偏差（如只有投诉类被人工复核） | 监控反馈数据的标签分布，做采样校正 |
| Docker 中 GPU 不可用 | 容器未映射 GPU 或缺少 nvidia-docker | `docker run --gpus all` 并安装 nvidia-container-toolkit |

### 4.5 思考题

1. **初级**：如果要在本系统中加入"敏感词过滤"功能（工单含敏感词自动标记为需人工审核），你会放在请求处理流程的哪个环节？
2. **进阶**：系统目前是单 worker 模式。如果需要支持 10 倍流量，请设计一个水平扩展方案——包含负载均衡、无状态服务、共享模型缓存、以及扩容触发条件。
