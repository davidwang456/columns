# 第12章：模型评估与错误分析

## 1 项目背景

### 业务场景

算法团队花了两周时间训练了一个客服工单分类模型，测试集上的 Accuracy 达到 91%。产品经理信心满满地推进上线，结果灰度期间运营团队反馈："这模型把一半的投诉工单分到了咨询类，投诉用户得不到及时处理，更生气了！"

小陈一脸困惑地打开混淆矩阵一看——"投诉"类别的召回率（Recall）只有 48%，远低于整体的 Accuracy 91%。原因很简单：训练数据中投诉工单仅占 5%，模型学会了"只要预测为咨询，准确率就高"的偷懒策略。但业务上，漏掉一条投诉的代价远远大于把咨询误判为投诉。

技术经理拍板："重新评估模型，这次不能只看 Accuracy。"

### 痛点放大

在真实的业务场景中，模型评估远比"跑个测试集看准确率"复杂得多：

1. **指标选择的陷阱**：Accuracy 在样本均衡时有效，但在投诉占 5% 的极端不平衡场景下，一个永远预测"咨询"的模型 Accuracy 也有 95%。你需要 Precision、Recall、F1、AUC 等多个指标来从不同角度衡量。
2. **业务代价不对等**：把"投诉"误判为"咨询"的代价是客服响应慢（用户愤怒升级），把"咨询"误判为"投诉"的代价只是多看了一眼。阈值选择需要有业务权重。
3. **模型为什么错**：准确率从 91% 降到上线后的 71%，是数据漂移？是标注错误？还是模型本身有盲区？没有系统的错误分析就无法定位问题。

```
Accuracy = 91% 的背后：

混淆矩阵:
              预测咨询  预测投诉  预测其他
真实咨询       720        15       12
真实投诉        24        38       18     ← 48% 的投诉被漏掉！
真实其他        20         8       45

业务影响:
漏掉投诉 → 用户得不到处理 → 升级为平台投诉/12315 → 品牌损失
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周四下午 4:00，复盘会议室。小陈把评估结果投到屏幕上，运营主管张姐的脸色不太好。

---

**小胖**（看着满屏的数字）:"91% 不是挺高的吗？小学考试 91 分我妈都要奖励我一个鸡腿。怎么张姐还不满意？"

**张姐**:"小胖，假如你是医生，体检中心给你打电话说'你 91% 概率没病'——但那个 9% 的概率恰好是癌症没查出来。你觉得这个 91% 的准确率有用吗？"

**小胖**:"哦……这么说好像是不太够。"

**小白**:"这就是为什么只看 Accuracy 是致命的。在分类问题上，至少要同时看 Precision（精确率）和 Recall（召回率）。Precision 问的是：模型说某条是投诉，实际真的是投诉的比例——高了代表不乱误报。Recall 问的是：真实投诉中模型找出了多少——高了代表不遗漏。"

**大师**（打开会议室白板）:"让我们用一句口诀记住这四个核心指标：

**Accuracy = 所有预测正确的 / 总样本**
**Precision = 预测为正且对了的 / 所有预测为正的**（别冤枉好人）
**Recall = 预测为正且对了的 / 所有真实为正的**（别放过坏人）
**F1 = Precision 和 Recall 的调和平均**（两者都别太差）

回到张姐的问题：投诉工单占 5%，Recall 只 48%，意味着超过一半的投诉被当成咨询处理了。这些用户得不到及时回复，负面情绪会发酵。"

**小陈**:"那我怎么知道模型在哪个类别上漏得多、哪个类别上错得多？"

**大师**:"看**混淆矩阵**。它是最直观的错误分析工具——行是真实标签，列是预测标签。对角线上是分对的，非对角线上是分错的。还可以按比例归一化，看每行（每个真实类别）中有多少比例被分到了其他类别。

比混淆矩阵更进一步的，是**错误样本分析**。从错误预测的样本中抽样 50-100 条，人工逐条分析错误原因，通常能发现：
- 标注错误：训练数据的标签就是错的，模型当然学不对
- 边界歧义：这条工单确实同时涉及投诉和咨询，人工也会纠结
- 数据盲区：模型从未见过某类表述（如行业术语），无法正确分类
- 模型能力不足：样本太模糊或需要推理，当前模型容量不够"

**小白**:"那怎么衡量业务上的代价？比如说漏掉投诉的代价是误报咨询的 5 倍。"

**大师**:"这就是**代价矩阵**——你给每个错误类型定义一个代价权重，然后计算加权后的总代价。比如：

|  | 预测咨询 | 预测投诉 | 预测退款 |
|---|---------|---------|---------|
| 真实咨询 | 0 | 1 | 1 |
| 真实投诉 | 5 | 0 | 3 |
| 真实退款 | 1 | 2 | 0 |

投诉→咨询代价=5（最高，漏掉投诉最严重），你可以根据代价阈值自动判断哪些工单该转人工复核。"

**技术映射总结**：
- Accuracy = 总分，但偏科生也能拿高分（样本不平衡时失效）
- Precision = "不乱抓人"，Recall = "不错过坏人"，F1 = 两者平衡
- 混淆矩阵 = 错误记录的"地图"，一眼看出哪些类别之间容易混淆
- 错误分析 = 从"模型不好"到"为什么不好"的侦探工作

### 3.2 多维度模型评估

#### 目标

对训练好的分类模型进行全面的指标评估。

```python
# evaluate_model.py
"""多维度模型评估脚本"""

import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.metrics import (
    accuracy_score, precision_recall_fscore_support,
    confusion_matrix, classification_report, roc_auc_score
)
from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch
from datasets import Dataset

# ===== 1. 加载模型和测试数据 =====
MODEL_PATH = "./models/sentiment_minimal"
tokenizer = AutoTokenizer.from_pretrained(MODEL_PATH)
model = AutoModelForSequenceClassification.from_pretrained(MODEL_PATH)
model.eval()

# 模拟测试数据（含标签分布不均）
test_data = {
    "text": [
        # 咨询类（多数类，10条）
        "这个产品怎么使用？", "请问发货多久到？", "支持货到付款吗？",
        "可以开发票吗？", "有没有优惠券？", "怎么修改订单？",
        "请问退货地址是什么？", "商品什么时候补货？", "支持七天无理由吗？",
        "能不能换颜色？",
        # 投诉类（少数类，5条）
        "你们客服态度太差了", "商品质量严重有问题", "虚假宣传欺骗消费者",
        "三天了还不发货太过分", "收到的货是坏的",
        # 退款类（少数类，5条）
        "我要退款退货", "怎么退钱", "申请退款审核多久",
        "为什么拒绝我的退款", "退款还没到账",
    ],
    "label": [0]*10 + [1]*5 + [2]*5,  # 0=咨询, 1=投诉, 2=退款
}
id2label = {0: "咨询", 1: "投诉", 2: "退款"}

ds = Dataset.from_dict(test_data)

# ===== 2. 批量推理 =====
def predict_batch(texts):
    inputs = tokenizer(texts, truncation=True, max_length=128,
                       padding=True, return_tensors="pt")
    with torch.no_grad():
        outputs = model(**inputs)
        probs = torch.softmax(outputs.logits, dim=-1)
        preds = torch.argmax(outputs.logits, dim=-1)
    return preds.numpy(), probs.numpy()

all_preds, all_probs = predict_batch(ds["text"])
all_labels = np.array(ds["label"])

# ===== 3. 基础指标 =====
print("=" * 60)
print("3. 基础指标")
print("=" * 60)

accuracy = accuracy_score(all_labels, all_preds)

# 宏平均（每个类别权重相同）
macro_p, macro_r, macro_f1, _ = precision_recall_fscore_support(
    all_labels, all_preds, average="macro", zero_division=0
)

# 加权平均（按样本数加权）
weighted_p, weighted_r, weighted_f1, _ = precision_recall_fscore_support(
    all_labels, all_preds, average="weighted", zero_division=0
)

print(f"Accuracy:       {accuracy:.4f}")
print(f"Macro F1:       {macro_f1:.4f}  (每个类别权重相等)")
print(f"Weighted F1:    {weighted_f1:.4f}  (按样本数加权)")

# ===== 4. 每类指标 =====
print("\n" + "=" * 60)
print("4. 各类别指标")
print("=" * 60)

per_class = precision_recall_fscore_support(
    all_labels, all_preds, labels=[0, 1, 2], zero_division=0
)

print(f"{'类别':<10} {'Precision':<12} {'Recall':<12} {'F1':<12} {'样本数':<8}")
print("-" * 55)
for i, name in id2label.items():
    count = sum(1 for l in all_labels if l == i)
    print(f"{name:<10} {per_class[0][i]:<12.4f} {per_class[1][i]:<12.4f} "
          f"{per_class[2][i]:<12.4f} {count:<8}")

# ===== 5. 混淆矩阵可视化 =====
print("\n" + "=" * 60)
print("5. 混淆矩阵")
print("=" * 60)

cm = confusion_matrix(all_labels, all_preds)
class_names = list(id2label.values())

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# 原始计数
sns.heatmap(cm, annot=True, fmt='d', cmap='Blues',
            xticklabels=class_names, yticklabels=class_names, ax=axes[0])
axes[0].set_title("Confusion Matrix (Count)")
axes[0].set_xlabel("Predicted")
axes[0].set_ylabel("True")

# 行归一化（召回率视角：每个真实类别被分到哪了）
cm_normalized = cm.astype('float') / cm.sum(axis=1, keepdims=True)
sns.heatmap(cm_normalized, annot=True, fmt='.2f', cmap='YlOrRd',
            xticklabels=class_names, yticklabels=class_names, ax=axes[1])
axes[1].set_title("Confusion Matrix (Row Normalized)")
axes[1].set_xlabel("Predicted")
axes[1].set_ylabel("True")

plt.tight_layout()
plt.savefig("./output/confusion_matrix.png", dpi=150)
print("混淆矩阵已保存至 ./output/confusion_matrix.png")

# 打印分类报告
print("\n" + classification_report(
    all_labels, all_preds, target_names=class_names, zero_division=0
))
```

### 3.3 错误样本分析

```python
# error_analysis.py
"""错误样本系统化分析"""

import numpy as np
from collections import defaultdict


def analyze_errors(texts, true_labels, pred_labels, pred_probs,
                   id2label, top_k=10):
    """
    系统化分析预测错误

    返回每种错误类型的样本及诊断建议
    """
    errors = []
    error_types = defaultdict(list)

    for i, (text, true, pred, probs) in enumerate(
        zip(texts, true_labels, pred_labels, pred_probs)
    ):
        if true == pred:
            continue  # 预测正确，跳过

        confidence = probs[pred]
        true_confidence = probs[true]
        margin = confidence - true_confidence  # 预测类置信度 - 真实类置信度

        # 错误分类
        if confidence < 0.5:
            error_type = "低置信度错误（模型不确定）"
        elif margin > 0.3:
            error_type = "高置信度错误（模型很确定地答错了）"
        else:
            error_type = "边界混淆（多个类别得分接近）"

        error_info = {
            "index": i,
            "text": text,
            "true_label": id2label[true],
            "pred_label": id2label[pred],
            "confidence": confidence,
            "true_confidence": true_confidence,
            "margin": margin,
            "error_type": error_type,
        }
        errors.append(error_info)
        error_types[error_type].append(error_info)

    # 输出分析报告
    print("=" * 60)
    print("错误分析报告")
    print("=" * 60)
    print(f"总样本数: {len(texts)}, 错误数: {len(errors)}, "
          f"错误率: {len(errors)/len(texts):.2%}\n")

    # 错误类型分布
    print("错误类型分布:")
    for etype, samples in sorted(error_types.items(),
                                 key=lambda x: -len(x[1])):
        print(f"  {etype}: {len(samples)} 条 ({len(samples)/len(errors):.1%})")
    print()

    # 类别混淆统计
    print("最常见的类别混淆 (Top 5):")
    confusion_pairs = defaultdict(int)
    for e in errors:
        pair = f"{e['true_label']} → {e['pred_label']}"
        confusion_pairs[pair] += 1

    for pair, count in sorted(confusion_pairs.items(),
                              key=lambda x: -x[1])[:5]:
        print(f"  {pair}: {count} 次")

    # 高置信度错误样本（最需要关注）
    high_conf_errors = [e for e in errors if e["confidence"] > 0.8]
    if high_conf_errors:
        print(f"\n⚠ 高置信度错误样本 ({len(high_conf_errors)} 条):")
        for e in high_conf_errors[:min(5, len(high_conf_errors))]:
            print(f"  文本: {e['text'][:60]}")
            print(f"  真实: {e['true_label']}, 预测: {e['pred_label']}, "
                  f"置信度: {e['confidence']:.2%}")
            print()

    return errors, error_types


# ===== 使用示例 =====
if __name__ == "__main__":
    # 模拟预测结果
    texts = [
        "客服态度极其恶劣，我要投诉！",       # 真实=投诉
        "这个产品发货速度很快，好评",         # 真实=好评
        "东西收到了，但不是很满意",           # 真实=中评
        "垃圾产品，用一次就坏了",            # 真实=投诉
        "还不错吧，中规中矩",                # 真实=中评
        "物流太慢，但是东西质量可以",         # 真实=中评
    ]
    true_labels = [1, 0, 2, 1, 2, 2]  # 0=好评, 1=投诉, 2=中评
    pred_labels =  [1, 0, 0, 1, 2, 0]  # 模拟部分预测错误
    pred_probs = [
        [0.1, 0.85, 0.05],
        [0.9, 0.05, 0.05],
        [0.45, 0.15, 0.40],  # 边界混淆
        [0.03, 0.92, 0.05],
        [0.05, 0.10, 0.85],
        [0.52, 0.08, 0.40],  # 高置信度错误
    ]
    id2label = {0: "好评", 1: "投诉", 2: "中评"}

    analyze_errors(texts, np.array(true_labels), np.array(pred_labels),
                   np.array(pred_probs), id2label)
```

### 3.4 代价敏感评估

```python
# cost_sensitive_eval.py
"""代价敏感评估 —— 考虑业务成本的模型选择"""

import numpy as np

# 代价矩阵（行=真实，列=预测）
# 值越大代价越高
COST_MATRIX = np.array([
    # 预测→  咨询  投诉  退款
    [0,    1,    1],    # 真实=咨询
    [5,    0,    3],    # 真实=投诉（漏掉代价最高）
    [1,    2,    0],    # 真实=退款
])

id2label = {0: "咨询", 1: "投诉", 2: "退款"}


def compute_cost(true_labels, pred_labels, cost_matrix):
    """计算预测的总代价"""
    total_cost = 0
    per_sample_cost = []

    for true, pred in zip(true_labels, pred_labels):
        cost = cost_matrix[true][pred]
        total_cost += cost
        per_sample_cost.append(cost)

    return total_cost, per_sample_cost


def find_optimal_threshold(probs, true_labels, cost_matrix,
                           num_thresholds=50):
    """
    寻找最优置信度阈值（基于代价最小化）

    思路：对于置信度低于阈值的样本转人工处理（代价=0.5），
    高于阈值的用模型预测。
    """
    best_cost = float("inf")
    best_threshold = 0
    best_preds = None

    for threshold in np.linspace(0, 1, num_thresholds):
        preds = []
        for i, p in enumerate(probs):
            max_prob = np.max(p)
            if max_prob < threshold:
                preds.append(-1)  # 转人工
            else:
                preds.append(np.argmax(p))

        # 计算代价：模型错误按代价矩阵，转人工按固定代价
        cost = 0
        for i, (true, pred) in enumerate(zip(true_labels, preds)):
            if pred == -1:
                cost += 0.5  # 转人工代价（人工审核成本）
            else:
                cost += cost_matrix[true][pred]

        if cost < best_cost:
            best_cost = cost
            best_threshold = threshold
            best_preds = preds

    return best_threshold, best_cost, best_preds


# ===== 使用示例 =====
if __name__ == "__main__":
    # 模拟预测
    np.random.seed(42)
    true_labels = np.array([0]*100 + [1]*20 + [2]*30)
    probs = np.random.dirichlet(np.ones(3), size=150)
    # 让真实类的概率更高
    for i in range(150):
        probs[i, true_labels[i]] += 0.3
    probs = probs / probs.sum(axis=1, keepdims=True)

    naive_preds = np.argmax(probs, axis=1)

    # 不计代价的评估
    naive_cost, _ = compute_cost(true_labels, naive_preds, COST_MATRIX)
    print(f"不计代价（直接用 argmax）: 总代价 = {naive_cost:.1f}")

    # 寻找最优阈值
    best_threshold, best_cost, best_preds = find_optimal_threshold(
        probs, true_labels, COST_MATRIX
    )
    manual_count = sum(1 for p in best_preds if p == -1)
    print(f"最优阈值 = {best_threshold:.2f}")
    print(f"转人工: {manual_count}/{len(true_labels)} 条 ({manual_count/len(true_labels):.1%})")
    print(f"最优代价 = {best_cost:.1f}")
    print(f"相比不计代价节省: {naive_cost - best_cost:.1f}")
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **Accuracy** | 直观易懂，非技术人员能理解 | 样本不平衡时严重失真 |
| **Precision/Recall/F1** | 从不同维度衡量，适合不平衡数据 | 多分类时需选择 macro/micro/weighted 平均方式 |
| **混淆矩阵** | 一眼看出类别混淆模式 | 类别超过 20 个时可视化杂乱 |
| **代价矩阵** | 对齐业务目标，选最优阈值 | 代价值依赖主观判断，需业务方校准 |

### 4.2 适用场景

| 场景 | 推荐指标组合 |
|------|-------------|
| 样本均衡的分类 | Accuracy + F1 |
| 样本不均衡的分类（投诉检测） | Macro F1 + Recall per class + 混淆矩阵 |
| 误报代价远高于漏报的（欺诈检测） | Precision 优先 + 代价矩阵 |
| 漏报代价远高于误报的（疾病筛查） | Recall 优先 + 代价矩阵 |

**不适用场景**：
- 生成类任务（文本生成、翻译等），需要用 BLEU/ROUGE/BERTScore
- 排序类任务，需要用 MRR/NDCG

### 4.3 注意事项

1. **macro vs micro vs weighted**：macro 各类权重相同（小类友好），micro 等于 Accuracy，weighted 按样本数加权（大类主导）
2. **zero_division=0**：当某个类别在预测中从未出现时，Precision/Recall 为 0/0，需要设置 `zero_division=0` 避免 Warning
3. **评估集泄露**：训练、验证、测试三者的样本必须严格互斥，混淆矩阵中验证集指标虚高时优先检查数据泄漏

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| F1 为 0 但 Accuracy 很高 | 小类完全未被预测（模型只输出大类） | 检查类别分布，加 class_weight 或重采样 |
| 验证集指标远高于测试集 | 验证集参与了模型选择导致过拟合 | 严格隔离测试集，只在最终评估时使用 |
| 上线后准确率骤降 | 线上数据分布与测试集不同（数据漂移） | 定期抽样线上数据加入训练，持续监控分布变化 |

### 4.6 第11章思考题答案

**第11章思考题1**：
- 有效 batch_size = 4 × 4 = 16。与直接设 batch_size=16 的区别：(1) 梯度累积的内存占用更小——每次只处理 4 个样本，4 次累积后再更新参数，节省显存；(2) 两者的 loss 数值可能略有不同，因为 batch_size=16 时 batch norm 的统计量（均值和方差）是基于 16 个样本计算的，而梯度累积是分别基于 4 个样本计算的。

**第11章思考题2**：
- 继承 Trainer 并重写 `create_optimizer()` 方法：`def create_optimizer(self): self.optimizer = torch.optim.SGD(self.model.parameters(), lr=self.args.learning_rate, momentum=0.9)`。同时需要重写 `create_scheduler()` 因为默认的 scheduler 是为 AdamW 设计的。
