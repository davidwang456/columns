# 第4章：模型加载、配置文件与权重管理

## 1 项目背景

### 业务场景

算法团队完成了一个客服工单分类模型的训练，准确率达到 91%。模型被保存为一个目录，里面有 `config.json`、`pytorch_model.bin`、`tokenizer_config.json`、`vocab.txt` 等七八个文件。后端工程师小刘需要在 Java 服务中调用这个模型，于是他找到算法工程师小陈。

"这些文件哪个是模型？我怎么加载？"小刘指着目录问。

小陈解释道："`pytorch_model.bin` 是权重文件，`config.json` 是模型配置。你先用 Python 加载，然后通过 HTTP API 提供给 Java 调用。"

小刘按照小陈给的代码跑，但遇到了三个问题：

1. 生产环境是离线服务器，无法从 Hugging Face Hub 下载模型，`from_pretrained("bert-base-chinese")` 直接报 `ConnectionError`。
2. 模型文件有 400MB，每次部署都在 Docker 镜像里拷一份，镜像膨胀到 2GB，CI/CD 构建慢得离谱。
3. 某天线上突然出现 `KeyError: 'classifier.weight'`，排查发现是算法团队更新了模型结构但没有同步更新 config。

### 痛点放大

模型加载是连接训练和推理的桥梁，常见陷阱如下：

1. **在线 vs 离线加载混淆**：`from_pretrained("模型名")` 默认走 Hub 下载，离线环境需指定本地路径 `from_pretrained("/path/to/model")`，但本地路径的结构必须完整。
2. **文件散落风险**：`config.json`、`tokenizer_config.json`、`pytorch_model.bin` 等七八个文件缺一不可，任一缺失都导致加载失败。
3. **权重格式隐患**：PyTorch 的 `.bin` 文件使用 pickle 序列化，存在代码注入风险；Safetensors 格式更安全且加载更快。
4. **配置与权重不匹配**：config 中的 `num_labels` 和权重的分类头维度不一致时，加载会静默出错或用随机初始化权重替代。

```
from_pretrained("model-name")
        │
        ▼
   [检查本地缓存] ──有──→ 加载
        │ 无
        ▼
   [从 Hub 下载] ──网络不通──→ ConnectionError ✗
        │ 成功
        ▼
   [解析 config.json] ──不匹配──→ 权重映射失败 ✗
        │ 匹配
        ▼
   [加载权重文件] ──格式不兼容──→ 加载失败 ✗
        │ 成功
        ▼
   [返回模型对象] ✓
```

本章的目标是：理清 `from_pretrained` 的完整执行路径，掌握离线加载、Safetensors 安全加载和模型缓存管理。

## 2 项目设计

### 剧本式交锋对话

**场景**：周五下午 4:30，小刘和小陈在会议室对着报错日志发愁。

---

**小胖**（端着一碗泡面走进来）:"你俩还在加班？咋了，模型又罢工了？"

**小刘**:"我把小陈给的模型文件放到服务器上，`from_pretrained('bert-base-chinese')` 一直报连接错误。不是已经有模型文件了吗，为什么还要联网？"

**小胖**:"这不是跟打车一样嘛——你站在路边喊'出租车！'（`from_pretrained('名字')`），系统自动帮你从网络召一辆车。但你已经有一辆自己的车停在停车场了，你直接走过去开就行（`from_pretrained('/path/to/local/model')`）。你少了个 '/' 前缀。"

**小白**:"这个比喻说得通。但我有个更深入的问题：本地路径的模型目录里到底需要哪些文件？我下了一个模型，发现里面 config.json、pytorch_model.bin、tf_model.h5、model.safetensors 全都有，哪个是需要的？"

**大师**:"好问题。模型目录的标准结构分两种：

**旧格式（单文件权重）**：
```
bert-custom/
├── config.json           # 模型配置（层数、隐藏维度、词表大小等）
├── pytorch_model.bin     # PyTorch 权重（pickle 格式，有安全风险）
├── tokenizer_config.json # Tokenizer 配置
├── vocab.txt             # 词表文件
└── special_tokens_map.json
```

**新格式（分片权重 + Safetensors）**：
```
bert-custom/
├── config.json
├── model.safetensors           # 安全格式权重（推荐）
├── model-00001-of-00002.safetensors  # 大模型分片
├── model-00002-of-00002.safetensors
├── model.safetensors.index.json      # 分片索引文件
├── tokenizer_config.json
├── tokenizer.json              # Fast tokenizer 专用
├── special_tokens_map.json
└── generation_config.json      # 生成模型专用
```

```
**小刘**:"等一下——`pytorch_model.bin` 和 `model.safetensors` 有什么区别？都是权重文件吧？"

**大师**:"区别大了：

- `pytorch_model.bin` 用 `torch.save()` → Python pickle 序列化。pickle 的反序列化可以执行任意 Python 代码，如果你加载了一个被恶意篡改的 `.bin` 文件，它会执行攻击者注入的代码。
- `model.safetensors` 是专门为安全加载张量设计的格式，只存储纯数据（tensor dtype + shape + raw bytes），不包含可执行代码。加载速度也比 pickle 快，因为它不需要反序列化 Python 对象，直接内存映射。

一线生产环境应该坚持 `safetensors`。"

**小白**:"那 config.json 里的内容怎么检查对不对？我碰到过 config 里写的 `num_labels=2`，但实际权重里的分类头是 `nn.Linear(768, 5)`，加载时居然没报错。"

**大师**:"这正是 `from_pretrained` 的一个微妙行为。当你调用 `from_pretrained(config, weights)`，如果权重的某个 key 的 shape 和 config 定义不匹配：

- **大小不匹配**：Transformers 会打印一条 warning 并**跳过该权重**（用随机初始化替代），这就是为什么你的分类头变成了随机初始化——准确率当然掉到 50%（二分类随机猜）。
- **完全缺失**：`missing_keys` 里会列出来，model 加载后会有 warning。
- **多余权重**：`unexpected_keys` 会列出来，通常是因为 config 定义的任务头不同。

验证方法：`model.load_state_dict(state_dict, strict=False)` 查 `missing_keys` 和 `unexpected_keys`。"

**小胖**:"我还有个问题——我们十几个人团队，经常各自下载同一个模型，公司带宽都被吃满了。有没有办法让大家共享？"

**大师**:"三个方案递进：

1. **设置缓存目录为共享路径**：`export HF_HOME=/mnt/shared/huggingface_cache`，所有人共用缓存。
2. **搭建 HF Hub 镜像代理**：用 `huggingface-cli` 配合 `HF_ENDPOINT` 指向内网代理。
3. **模型版本打包**：把模型文件 + 代码一起打成 Python wheel 包或 Docker volume，通过制品仓库分发。

方案三最可靠，尤其适合离线部署和 CI/CD 场景。"

**技术映射总结**：
- `from_pretrained("model-name")` 走云端 Hub → `from_pretrained("/local/path")` 走本地文件
- `pytorch_model.bin` = pickle 格式（危险加载）→ `model.safetensors` = 安全格式（推荐）
- config 与权重不匹配时不会抛异常，只发 warning + 随机初始化
- 团队共享模型用缓存目录、镜像代理或制品打包

### 3.2 离线模型加载器

#### 目标

实现一个支持在线/离线双模式、自动检测模型格式、记录加载日志的模型加载器。

```python
# model_loader.py
"""通用模型加载器 —— 支持在线/离线、bins/safetensors 自动检测"""

import os
import json
import logging
from pathlib import Path
from typing import Optional, Dict, Tuple
from dataclasses import dataclass

import torch
from transformers import (
    AutoConfig,
    AutoTokenizer,
    AutoModel,
    AutoModelForSequenceClassification,
    PreTrainedModel,
    PreTrainedTokenizer,
    PretrainedConfig,
)

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)


@dataclass
class ModelLoadResult:
    model: PreTrainedModel
    tokenizer: PreTrainedTokenizer
    config: PretrainedConfig
    weight_format: str
    model_size_mb: float
    load_warnings: list


class SafeModelLoader:
    """安全的模型加载器，支持在线/离线双模式"""

    def __init__(self, offline_mode: bool = False, local_models_dir: Optional[str] = None):
        """
        Args:
            offline_mode: True 时不联网
            local_models_dir: 本地模型根目录（离线模式使用）
        """
        self.offline_mode = offline_mode
        self.local_models_dir = Path(local_models_dir) if local_models_dir else None

    def _resolve_path(self, model_id: str) -> str:
        """解析模型路径: 先查本地 → 再查 Hub"""
        # 如果是绝对路径，直接返回
        if os.path.isabs(model_id) and os.path.isdir(model_id):
            return model_id

        # 离线模式：从本地目录找
        if self.offline_mode and self.local_models_dir:
            local_path = self.local_models_dir / model_id.replace("/", "--")
            if local_path.exists():
                logger.info(f"离线模式: 从 {local_path} 加载模型")
                return str(local_path)
            raise FileNotFoundError(
                f"离线模式下未找到模型 {model_id}，"
                f"请将模型放到 {local_path} 目录"
            )

        # 在线模式：让 transformers 自行从 Hub 下载
        return model_id

    def _detect_weight_format(self, model_dir: str) -> Tuple[str, float]:
        """检测权重格式，返回 (格式名, 文件大小MB)"""
        model_path = Path(model_dir)
        total_size = 0

        # 检查 safetensors
        st_files = list(model_path.glob("*.safetensors"))
        if st_files:
            total_size = sum(f.stat().st_size for f in st_files) / (1024 * 1024)
            return "safetensors", total_size

        # 检查 pytorch_model.bin
        bin_file = model_path / "pytorch_model.bin"
        if bin_file.exists():
            total_size = bin_file.stat().st_size / (1024 * 1024)
            return "pytorch_bin", total_size

        return "unknown", 0

    def _validate_files(self, model_dir: str) -> list:
        """验证模型目录文件完整性"""
        model_path = Path(model_dir)
        warnings = []

        required_files = ["config.json"]
        for f in required_files:
            if not (model_path / f).exists():
                warnings.append(f"缺少必需文件: {f}")

        # 检查权重文件
        has_bin = (model_path / "pytorch_model.bin").exists()
        has_st = len(list(model_path.glob("*.safetensors"))) > 0
        has_index = (model_path / "model.safetensors.index.json").exists()

        if not has_bin and not has_st and not has_index:
            warnings.append("未找到任何权重文件 (.bin 或 .safetensors)")

        return warnings

    def _verify_config_weight_match(
        self, config: PretrainedConfig, state_dict: dict
    ) -> list:
        """验证配置与权重的匹配度"""
        warnings = []

        # 检查 num_labels 匹配（分类模型）
        if hasattr(config, "num_labels"):
            classifier_key = None
            for key in state_dict:
                if "classifier" in key and "weight" in key:
                    classifier_key = key
                    break

            if classifier_key:
                weight_shape = state_dict[classifier_key].shape
                if weight_shape[0] != config.num_labels:
                    warnings.append(
                        f"config.num_labels={config.num_labels} 与 "
                        f"分类头权重维度 {weight_shape[0]} 不匹配！"
                    )

        return warnings

    def load(
        self,
        model_id: str,
        task: str = "base",
        torch_dtype: Optional[torch.dtype] = None,
        **kwargs
    ) -> ModelLoadResult:
        """
        加载模型

        Args:
            model_id: 模型名称或本地路径
            task: 任务类型 ('base', 'classification', 'token_classification')
            torch_dtype: 模型精度 (torch.float16, torch.bfloat16 等)
        """
        resolved_path = self._resolve_path(model_id)
        all_warnings = []

        # Step 1: 加载 Config
        logger.info(f"Step 1/4: 加载配置...")
        config = AutoConfig.from_pretrained(
            resolved_path,
            local_files_only=self.offline_mode,
        )
        logger.info(f"  模型架构: {config.architectures if hasattr(config, 'architectures') else '未指定'}")
        logger.info(f"  隐藏层维度: {getattr(config, 'hidden_size', 'N/A')}")
        logger.info(f"  层数: {getattr(config, 'num_hidden_layers', 'N/A')}")

        # Step 2: 加载 Tokenizer
        logger.info(f"Step 2/4: 加载 Tokenizer...")
        tokenizer = AutoTokenizer.from_pretrained(
            resolved_path,
            local_files_only=self.offline_mode,
        )
        logger.info(f"  词表大小: {tokenizer.vocab_size}")

        # Step 3: 检测权重格式
        logger.info(f"Step 3/4: 检测权重格式...")
        if os.path.isdir(resolved_path):
            format_name, size_mb = self._detect_weight_format(resolved_path)
            file_warnings = self._validate_files(resolved_path)
            all_warnings.extend(file_warnings)
            logger.info(f"  权重格式: {format_name}")
            logger.info(f"  模型大小: {size_mb:.1f} MB")
        else:
            format_name = "from_hub"
            size_mb = 0

        # Step 4: 加载模型（优先使用 safetensors）
        logger.info(f"Step 4/4: 加载模型权重...")

        if task == "classification":
            model_class = AutoModelForSequenceClassification
        elif task == "token_classification":
            from transformers import AutoModelForTokenClassification
            model_class = AutoModelForTokenClassification
        else:
            model_class = AutoModel

        model = model_class.from_pretrained(
            resolved_path,
            config=config,
            local_files_only=self.offline_mode,
            torch_dtype=torch_dtype,
            use_safetensors=True,  # 优先使用 safetensors
            **kwargs
        )

        # Step 5: 验证 config 与权重的匹配
        state_dict = model.state_dict()
        match_warnings = self._verify_config_weight_match(config, state_dict)
        all_warnings.extend(match_warnings)

        if all_warnings:
            logger.warning(f"加载警告: {all_warnings}")
        else:
            logger.info("模型加载成功，未发现配置/权重不匹配 ✓")

        return ModelLoadResult(
            model=model,
            tokenizer=tokenizer,
            config=config,
            weight_format=format_name,
            model_size_mb=size_mb,
            load_warnings=all_warnings,
        )


# ===== 使用示例 =====
if __name__ == "__main__":
    # 示例1: 在线加载
    print("=== 在线加载 ===")
    loader = SafeModelLoader(offline_mode=False)
    result = loader.load("distilbert-base-uncased", task="classification")
    print(f"格式: {result.weight_format}, 大小: {result.model_size_mb:.1f}MB")

    # 示例2: 离线加载（假设模型已在本地）
    # loader = SafeModelLoader(
    #     offline_mode=True,
    #     local_models_dir="/mnt/shared/models"
    # )
    # result = loader.load("bert-base-chinese", task="classification")

    # 示例3: FP16 精度加载（节省一半显存）
    print("\n=== FP16 精度加载 ===")
    result_fp16 = loader.load(
        "distilbert-base-uncased",
        task="classification",
        torch_dtype=torch.float16
    )
    print(f"精度: float16, 显存预估: {result_fp16.model_size_mb * 0.5:.0f}MB")

    # 验证推理
    inputs = result_fp16.tokenizer("Hello, this is a test!", return_tensors="pt")
    with torch.no_grad():
        outputs = result_fp16.model(**inputs)
    print(f"推理输出形状: {outputs.logits.shape}")

### 3.3 模型缓存管理工具

```python
# cache_manager.py
"""HuggingFace 模型缓存管理工具"""

import shutil
from pathlib import Path
from huggingface_hub import scan_cache_dir


def list_cached_models():
    """列出所有已缓存的模型及大小"""
    cache = scan_cache_dir()

    print(f"{'模型名称':<55} {'大小':<10} {'修订版':<12}")
    print("-" * 80)

    total_size = 0
    for repo in cache.repos:
        for revision in repo.revisions:
            print(f"{repo.repo_id:<55} {revision.size_on_disk_str:<10} {revision.commit_hash[:8]:<12}")
            total_size += revision.size_on_disk
    print("-" * 80)
    print(f"总缓存大小: {total_size / 1024**3:.1f} GB")


def export_model_to_local(model_id: str, output_dir: str, include_tokenizer: bool = True):
    """
    将 HuggingFace Hub 模型导出到本地目录（用于离线部署）

    Args:
        model_id: 模型 ID，如 "bert-base-chinese"
        output_dir: 本地输出目录
        include_tokenizer: 是否同时导出 tokenizer
    """
    from transformers import AutoConfig, AutoTokenizer, AutoModel

    output_path = Path(output_dir)
    output_path.mkdir(parents=True, exist_ok=True)

    print(f"正在下载模型 {model_id} ...")

    # 下载配置
    config = AutoConfig.from_pretrained(model_id)
    config.save_pretrained(str(output_path))
    print(f"  ✓ config.json")

    # 下载模型（safetensors 格式）
    model = AutoModel.from_pretrained(model_id, use_safetensors=True)
    model.save_pretrained(str(output_path), safe_serialization=True)
    print(f"  ✓ 模型权重 (safetensors)")

    if include_tokenizer:
        tokenizer = AutoTokenizer.from_pretrained(model_id)
        tokenizer.save_pretrained(str(output_path))
        print(f"  ✓ Tokenizer")

    # 计算总大小
    total_size = sum(f.stat().st_size for f in output_path.rglob("*") if f.is_file())
    print(f"\n导出完成！目录: {output_dir}")
    print(f"文件列表:")
    for f in sorted(output_path.iterdir()):
        print(f"  {f.name} ({f.stat().st_size / 1024:.1f} KB)")
    print(f"总大小: {total_size / 1024**2:.1f} MB")


def convert_to_safetensors(model_path: str):
    """将 pytorch_model.bin 转换为 safetensors 格式"""
    import torch
    from safetensors.torch import save_file

    model_dir = Path(model_path)
    bin_file = model_dir / "pytorch_model.bin"
    st_file = model_dir / "model.safetensors"

    if not bin_file.exists():
        print("未找到 pytorch_model.bin")
        return

    print(f"加载 {bin_file} ...")
    state_dict = torch.load(bin_file, map_location="cpu")

    print(f"转换为 safetensors 格式 ...")
    save_file(state_dict, str(st_file))

    bin_size = bin_file.stat().st_size / 1024**2
    st_size = st_file.stat().st_size / 1024**2
    print(f"  pytorch_model.bin: {bin_size:.1f} MB")
    print(f"  model.safetensors:  {st_size:.1f} MB")

    # 删除原始 bin 文件（可选，需确认）
    confirm = input("删除原始 pytorch_model.bin？(y/n): ")
    if confirm.lower() == "y":
        bin_file.unlink()
        print("已删除 pytorch_model.bin")


if __name__ == "__main__":
    import sys

    if len(sys.argv) < 2:
        print("用法:")
        print("  python cache_manager.py list         # 列出缓存")
        print("  python cache_manager.py export <model_id> <output_dir>  # 导出模型")
        print("  python cache_manager.py convert <model_path>  # 转换为 safetensors")
        sys.exit(0)

    cmd = sys.argv[1]
    if cmd == "list":
        list_cached_models()
    elif cmd == "export" and len(sys.argv) >= 4:
        export_model_to_local(sys.argv[2], sys.argv[3])
    elif cmd == "convert" and len(sys.argv) >= 3:
        convert_to_safetensors(sys.argv[2])
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **from_pretrained API** | 一行代码加载任何模型，自动解析配置、权重和 tokenizer | 默认走网络，离线环境需额外配置 |
| **Safetensors** | 加载速度比 pickle 快 2-5 倍，无代码注入风险，支持内存映射 | 老模型可能只有 .bin 格式，需要手动转换 |
| **AutoClass 抽象** | AutoModel/AutoConfig/AutoTokenizer 自动匹配模型类 | 自定义模型需注册才能被 AutoClass 发现 |
| **分片权重** | 大模型（>5GB）自动分片，支持增量下载 | 分片文件数量多（几十个），目录管理复杂 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 云端开发环境 | `from_pretrained("model-name")` + 默认缓存 |
| 内网离线服务器 | 先用 `export_model_to_local` 导出，再 `from_pretrained("/path")` |
| Docker 镜像构建 | 模型文件作为 volume 挂载，不进镜像层 |
| 多团队共享 | 设置共享 `HF_HOME` 或搭建内网镜像代理 |
| 安全敏感场景 | 强制 `use_safetensors=True`，拒绝 pickle 权重 |

**不适用场景**：
- 超大模型（>30GB）没有磁盘 IO 优化（需要 `device_map="auto"` + 磁盘 offload）
- 频繁切换模型的在线服务（每次加载初始化 slow，应预加载为单例）

### 4.3 注意事项

1. **`trust_remote_code=True` 安全风险**：允许执行模型仓库中的自定义代码，仅在完全信任模型来源时使用
2. **`torch_dtype` 与 `device_map` 互斥**：使用 `device_map="auto"` 时不要设置 `torch_dtype`，让 Transformers 自动决定
3. **`resize_token_embeddings` 陷阱**：扩展 tokenizer 词表后必须同步调用 `model.resize_token_embeddings(len(tokenizer))`，否则训练会崩溃

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| `safetensors_rust.SafetensorError` | 下载中断导致文件不完整 | 删除损坏的 .safetensors 文件，加 `resume_download=True` 重新下载 |
| 加载后准确率远低于预期 | config 分类头维度与权重不匹配，分类头随机初始化 | `_verify_config_weight_match` 检测不匹配 |
| Docker 构建每次下载模型 | 模型缓存在 `/root/.cache` 但构建未保留 | 用 Docker volume 挂载，或在 Dockerfile 中 `COPY` 本地模型 |
| GPU 加载后显存撑满 | 以 fp32 加载了 fp16 训练的最佳模型 | 加载时指定 `torch_dtype=torch.float16` |

### 4.5 思考题

1. **初级**：将 `pytorch_model.bin` 转换为 `model.safetensors` 后，文件大小一样吗？如果不一致，为什么？
2. **进阶**：你有一个在 BERT-base 上微调的分类模型，config.json 中 `num_labels=5`。现在需求变为 3 分类，你能不重新训练直接复用这个模型吗？需要做哪些操作？

### 4.6 第3章思考题答案

**第3章思考题1答案**：
- 输入 "Hello 世界！" 被切分为：`['[CLS]', 'hello', '世', '界', '！', '[SEP]']`
- 英文 "Hello" 被保留为完整 token（英文词表收录），但中文"世界"被拆为两个单字 token
- 原因：bert-base-chinese 的中文词表以单字为主，而英文词表以子词为主。对于中英混合文本，英文和数字通常保留为更长 token，中文被拆得更细

**第3章思考题2答案**：
- 医疗领域推荐 SentencePiece + BPE，词表大小 30,000-50,000
- 原因：SentencePiece 空格字符化可保留医学缩写（如 "CT检查" 不会因为空格被拆散）；BPE 按频率合并可自动学习医学子词（如 "高血压" → "高" + "血压" 或保留整体取决于频率）；30000 词表是最常见的平衡点——既覆盖常见术语又不浪费显存
- 不建议太大（>100k）：词表越大 Embedding 层显存越多，且医疗术语长尾没必要全收录
