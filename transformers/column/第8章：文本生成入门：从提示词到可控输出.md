# 第8章：文本生成入门：从提示词到可控输出

## 1 项目背景

### 业务场景

某电商公司的内容运营团队每天要产出 50 条商品营销文案。运营经理小李的日常是：打开竞品网站 → 参考 Top 3 商品的文案 → 手动改写为自家风格 → 交给组长审核。一条 100 字的商品卖点文案，从构思到定稿平均耗时 18 分钟。

上级要求：文案产量翻倍但人力不增加，同时要求文案风格统一、避免抄袭嫌疑。小李向技术团队求助："能不能用 AI 帮我写初稿？我只需要改几个字就能用。"

技术团队用 GPT 系模型（如 Qwen、ChatGLM）跑了一个 Demo，但很快发现三个问题：同样的 prompt 输入三遍，每次输出的文案都不一样；有时候生成的内容跑题到和商品毫无关系；偶尔还会生成"一键联系客服换货"这样不恰当的承诺性文案。

### 痛点放大

文本生成与分类/NER/QA 的根本区别在于：模型输出是"创作"出来的而非"选择"出来的。这带来了全新的控制难题：

1. **随机性 vs 稳定性**：`temperature=0` 时每次输出完全一样（确定性高但缺乏多样性），`temperature=1.5` 时输出天马行空（多样性高但可能跑题）。如何在不同场景选择合适的参数组合？
2. **解码策略选择**：贪心搜索（贪心）最快但容易陷入重复；Beam Search（束搜索）更流畅但多样性低；Top-k/Top-p 采样平衡了质量和创意。三种策略在业务场景下如何选择？
3. **生成结果不可控**：营销文案中出现"假一赔十"、"终身质保"等模型自己编造的承诺，一旦不经审核直接发布就是法律风险。

```
Prompt: "写一段蓝牙耳机的营销文案，突出音质和续航"

┌─────────────────────────────────────────────┐
│ Temperature=0.3: (确定性强)                   │
│ "这款蓝牙耳机音质清晰，续航长达30小时。"        │
│ Temperature=0.8: (平衡)                       │
│ "沉浸式Hi-Fi音质，30小时超长续航，通勤必备。"   │
│ Temperature=1.5: (随机性强 / 可能跑题)          │
│ "音乐在耳边流淌，仿佛是来自遥远星系的呼唤..."    │
└─────────────────────────────────────────────┘
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周五下午 3:00，AI 实验区。小李盯着屏幕上一段明显跑题的 AI 文案哭笑不得。

---

**小胖**（吃着辣条过来）:"哎，这 AI 写的什么玩意？'这款耳机让我仿佛听到了宇宙大爆炸的回声'——这到底是卖耳机还是写科幻小说？"

**小李**:"就是啊，我把 temperature 调到了 1.5，想让它写得更有创意一点，结果创过头了。但调回 0.3 又太死板，每次生成的文案都一模一样。"

**小胖**:"那你就调个中间值呗，比如 0.7？"

**小白**（在旁边调试代码）:"temperature 不只是个滑条调参。你得理解它背后的原理——temperature 控制的是 softmax 输出概率分布的'平滑度'。让我画个图。"

小白在纸上画了三个概率分布：

```
原始 logits 经 softmax 后:
   "音质" 0.50  "续航" 0.30  "宇宙" 0.05  "其他" 0.15

Temperature=0.3 (压低分布，更确定):
   "音质" 0.78  "续航" 0.18  "宇宙" 0.01  "其他" 0.03
   → 模型基本上只会选"音质"

Temperature=1.0 (原始分布):
   "音质" 0.50  "续航" 0.30  "宇宙" 0.05  "其他" 0.15

Temperature=1.5 (拉平分布，更随机):
   "音质" 0.35  "续航" 0.28  "宇宙" 0.12  "其他" 0.25
   → "宇宙"的概率从5%涨到12%，容易出离谱的词
```

**小白**:"temperature 越小，概率分布越尖，模型越倾向于选最高概率的词，输出就越确定。temperature 越大，分布越平，低频词也有机会被选中，输出就更随机。"

**大师**（推开会议室门）:"解释得很好。但小白没提的是——temperature 只是冰山一角。我们来把生成控制的三个核心维度讲透。

**第一维：解码策略（怎么选下一个词）**

- **贪心搜索 (Greedy)**：每步选概率最高的词。速度最快，但容易陷入重复——'这个产品很好很好很好'这种循环就是贪心的典型问题。
- **Beam Search**：每步保留 K 条最可能的路径（Beam 宽度），最终选概率积最高的完整序列。比贪心流畅，但 K 太大会导致输出多样性降低——多条路径趋同。
- **Top-k Sampling**：每步只从概率最高的 k 个候选词中随机采样。k=50 是比较常见的默认值。
- **Top-p (Nucleus) Sampling**：每步从累积概率达到 p 的最小候选词集合中采样。相比固定 k，top-p 能根据当前上下文动态调整候选集大小——当前面词很确定时（如'蓝牙'后面大概率是'耳机'），候选集自动缩小。

**第二维：重复控制**

- `repetition_penalty`（重复惩罚）：对已经出现过的 token 降低概率，值 > 1.0 抑制重复。但值太大（>1.5）会导致正常的篇章结构也被破坏——比如'首先...其次...最后...'这种自然的排比句可能被不恰当地抑制。
- `no_repeat_ngram_size`：禁止连续 N 个 token 重复出现。设为 3 表示不允许任何 3-gram 重复。

**第三维：停止条件**

- `stop_strings` / `stopping_criteria`：遇到特定词时停止生成。比如营销文案不能出现'赔偿'、'保证'等承诺性词汇，一旦生成立即截断。
- `max_new_tokens`：生成的最大 token 数，防止无限制输出。

```

**小胖**（听得云里雾里）:"这么多参数，我到底该用哪个组合？"

**大师**:"一句话总结：**大部分业务场景用 Top-p sampling（p=0.9）+ temperature=0.7 + repetition_penalty=1.1**。需要确定性输出（如 JSON 格式化）用贪心搜索。需要创造性输出（如创意文案）用高 temperature+top-k。不确定就多生成几条再人工筛选。"

**技术映射总结**：
- Temperature = 概率分布的"地平线"，越低越确定，越高越随机
- 贪心搜索 = 永远选最好的 → 快但容易重复
- Beam Search = 保留多条路径 → 流畅但多样性低
- Top-p = 动态候选池 → 平衡质量和多样性
- Repetition Penalty = 给说过的词"降权" → 防止复读机

用同一 prompt，对比不同生成参数组合的输出差异。

```python
# generation_compare.py
"""生成参数对比实验"""

import torch
from transformers import AutoTokenizer, AutoModelForCausalLM

# 使用中文 GPT-2 模型（轻量、快速验证参数效果）
model_name = "uer/gpt2-chinese-cluecorpussmall"

tokenizer = AutoTokenizer.from_pretrained(model_name)
# GPT-2 没有 pad_token，需要手动设置
tokenizer.pad_token = tokenizer.eos_token
model = AutoModelForCausalLM.from_pretrained(model_name)

device = "cuda" if torch.cuda.is_available() else "cpu"
model.to(device)

prompt = "这是一款最新发布的蓝牙耳机，"


def generate_with_params(**gen_kwargs):
    """使用给定参数生成文本"""
    inputs = tokenizer(prompt, return_tensors="pt").to(device)

    with torch.no_grad():
        outputs = model.generate(
            **inputs,
            max_new_tokens=50,
            pad_token_id=tokenizer.eos_token_id,
            do_sample=gen_kwargs.pop("do_sample", True),
            **gen_kwargs,
        )

    generated = tokenizer.decode(outputs[0], skip_special_tokens=True)
    # 只显示生成部分（去掉 prompt）
    return generated[len(prompt):].strip()[:100]


# ===== 1. 贪心搜索 vs 采样 =====
print("=" * 60)
print("1. 贪心搜索 vs 采样")
print("=" * 60)

greedy = generate_with_params(do_sample=False)
print(f"  贪心搜索: {greedy[:80]}")

sampled = generate_with_params(do_sample=True, temperature=0.7)
print(f"  采样(0.7): {sampled[:80]}")

# ===== 2. 不同 Temperature =====
print("\n" + "=" * 60)
print("2. Temperature 影响")
print("=" * 60)

for temp in [0.1, 0.5, 0.7, 1.0, 1.5]:
    result = generate_with_params(temperature=temp, top_p=0.9)
    print(f"  temp={temp:.1f}: {result[:80]}")

# ===== 3. Top-k vs Top-p =====
print("\n" + "=" * 60)
print("3. Top-k vs Top-p")
print("=" * 60)

topk = generate_with_params(temperature=0.7, top_k=10)
print(f"  top_k=10: {topk[:80]}")

topp = generate_with_params(temperature=0.7, top_p=0.9)
print(f"  top_p=0.9: {topp[:80]}")

# ===== 4. Repetition Penalty =====
print("\n" + "=" * 60)
print("4. 重复惩罚")
print("=" * 60)

# 故意用贪婪搜索来触发重复
no_penalty = generate_with_params(
    do_sample=False, no_repeat_ngram_size=0, repetition_penalty=1.0
)
print(f"  无惩罚: {no_penalty[:100]}")

with_penalty = generate_with_params(
    do_sample=False, no_repeat_ngram_size=3, repetition_penalty=1.2
)
print(f"  有惩罚: {with_penalty[:100]}")

# ===== 5. Beam Search =====
print("\n" + "=" * 60)
print("5. Beam Search")
print("=" * 60)

for beam in [1, 3, 5]:
    result = generate_with_params(
        do_sample=False, num_beams=beam, num_return_sequences=1
    )
    print(f"  beam={beam}: {result[:80]}")

### 3.3 营销文案生成器（生产版）

```python
# copy_generator.py
"""可控文案生成器 —— 支持多种生成策略、安全过滤"""

import re
import torch
from transformers import AutoTokenizer, AutoModelForCausalLM
from dataclasses import dataclass
from typing import Optional, List


@dataclass
class GenerationResult:
    text: str
    truncated: bool
    stop_reason: str  # "max_length" | "stop_token" | "safety_filter"


class CopyGenerator:
    # 敏感承诺词黑名单
    FORBIDDEN_PATTERNS = [
        r"假一赔\S{0,2}",    # 假一赔十/假一赔百
        r"终身质保",
        r"无条件退款",
        r"绝对\S{1,4}",      # 绝对有效/绝对安全
        r"保证.*效果",
        r"无效退款",
    ]

    STOP_TOKENS = ["赔偿", "投诉", "律师函"]

    def __init__(self, model_name: str = "uer/gpt2-chinese-cluecorpussmall"):
        self.tokenizer = AutoTokenizer.from_pretrained(model_name)
        self.tokenizer.pad_token = self.tokenizer.eos_token
        self.model = AutoModelForCausalLM.from_pretrained(model_name)
        self.device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
        self.model.to(self.device)

    def _check_safety(self, text: str) -> bool:
        """检查生成文本是否包含敏感词"""
        for pattern in self.FORBIDDEN_PATTERNS:
            if re.search(pattern, text):
                return False
        return True

    def _check_stop_tokens(self, text: str) -> Optional[str]:
        """检查是否触发了停止词"""
        for token in self.STOP_TOKENS:
            if token in text:
                return token
        return None

    def generate(
        self,
        prompt: str,
        max_new_tokens: int = 100,
        temperature: float = 0.7,
        top_p: float = 0.9,
        top_k: int = 50,
        repetition_penalty: float = 1.1,
        no_repeat_ngram_size: int = 3,
        style: str = "balanced",
    ) -> GenerationResult:
        """
        可控文案生成

        Args:
            prompt: 输入 prompt
            style: 生成风格
                - "creative": 创意模式（高温度+top-k采样）
                - "balanced": 平衡模式（中温度+top-p采样）
                - "deterministic": 确定性模式（贪心搜索）
        """
        # 风格预设
        if style == "creative":
            temperature = 1.0
            top_p = 0.95
            top_k = 100
            do_sample = True
        elif style == "deterministic":
            temperature = 1.0  # 贪心不用 temperature
            do_sample = False
        else:  # balanced
            do_sample = True

        inputs = self.tokenizer(
            prompt, return_tensors="pt", truncation=True, max_length=256
        ).to(self.device)

        gen_kwargs = {
            "max_new_tokens": max_new_tokens,
            "pad_token_id": self.tokenizer.eos_token_id,
            "eos_token_id": self.tokenizer.eos_token_id,
            "no_repeat_ngram_size": no_repeat_ngram_size,
            "repetition_penalty": repetition_penalty,
        }

        if do_sample:
            gen_kwargs.update({
                "do_sample": True,
                "temperature": temperature,
                "top_p": top_p,
                "top_k": top_k,
            })
        else:
            gen_kwargs["do_sample"] = False

        with torch.no_grad():
            outputs = self.model.generate(**inputs, **gen_kwargs)

        full_text = self.tokenizer.decode(outputs[0], skip_special_tokens=True)
        generated = full_text[len(prompt):].strip()

        # 安全检查
        if not self._check_safety(generated):
            generated = generated[:50] + "\n[⚠ 自动截断：检测到敏感承诺词]"
            return GenerationResult(
                text=generated, truncated=True, stop_reason="safety_filter"
            )

        # 停止词检查
        stop_token = self._check_stop_tokens(generated)
        if stop_token:
            # 截断到停止词之前
            idx = generated.find(stop_token)
            generated = generated[:idx].strip()
            return GenerationResult(
                text=generated, truncated=True, stop_reason=f"stop_token:{stop_token}"
            )

        is_truncated = len(generated) >= max_new_tokens
        return GenerationResult(
            text=generated,
            truncated=is_truncated,
            stop_reason="max_length" if is_truncated else "eos",
        )

    def generate_multiple(
        self, prompt: str, n: int = 3, **kwargs
    ) -> List[GenerationResult]:
        """生成多条候选，供人工筛选"""
        results = []
        for i in range(n):
            result = self.generate(prompt, **kwargs)
            results.append(result)
        return results


# ===== 使用示例 =====
if __name__ == "__main__":
    generator = CopyGenerator()

    # 营销示例
    prompts = [
        "请为以下商品写一段50字以内的营销文案：无线蓝牙耳机，支持降噪，续航30小时",
        "请用一句话描述这款产品的核心卖点：智能手表，心率监测，IP68防水",
    ]

    for prompt in prompts:
        print(f"\n{'='*50}")
        print(f"Prompt: {prompt}")

        # 生成3条候选
        candidates = generator.generate_multiple(prompt, n=3, style="balanced")
        for i, cand in enumerate(candidates):
            status = "✓" if not cand.truncated else f"⚠ {cand.stop_reason}"
            print(f"  [{i+1}] {status}: {cand.text[:80]}")
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **贪心搜索** | 速度最快，结果可复现 | 极易陷入重复循环 |
| **Beam Search** | 全局较优，输出流畅 | 计算量大（K越大越慢），多样性低 |
| **Top-p Sampling** | 自适应候选集，平衡质量和多样性 | 对 prompt 敏感，低质 prompt 会导致低质输出 |
| **Repetition Penalty** | 有效抑制重复，参数简单 | 过高会破坏正常篇章结构 |

### 4.2 适用场景

| 场景 | 推荐策略 |
|------|---------|
| JSON/结构化输出 | Greedy + low temperature (0.2) |
| 客服标准回复 | Balanced: top_p=0.9, temperature=0.7 |
| 创意文案/广告语 | Creative: top_k=50, temperature=1.0 |
| 翻译/摘要 | Beam Search (beam=4~6) |
| 聊天对话 | Top-p with high temperature (0.8~1.0) |

**不适用场景**：
- 需要严格格式的输出（如 SQL、代码片段），建议用 constrained decoding + grammar
- 需要引用事实的任务（如"2024年GDP增长率是多少"），生成模型易产生幻觉，应改用 RAG

### 4.3 注意事项

1. **do_sample=False 时 temperature/top_p/top_k 无效**：很多新手困惑为什么改了 temperature 输出还是一样，因为贪心模式下 sampling 参数不生效
2. **eos_token 不设置会导致无限生成**：必须设置 `eos_token_id` 或 `max_new_tokens`，否则模型会一直生成到内存爆掉
3. **中文 GPT-2 的 pad_token 问题**：GPT-2 系列的 tokenizer 默认没有 pad_token，需要手动 `tokenizer.pad_token = tokenizer.eos_token`，否则 batch 推理报错

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 输出中文乱码/英文 | 模型词表不包含中文（如英文 GPT-2） | 换用中文模型（如 uer/gpt2-chinese-cluecorpussmall） |
| 每隔几句就重复 | 贪心搜索 + 无重复惩罚 | 启用 `repetition_penalty=1.1` + `no_repeat_ngram_size=3` |
| 输出突然截断在奇怪位置 | eos_token 被意外触发（如"。"也被映射为 eos） | 检查 tokenizer 的 eos_token 定义 |

### 4.6 第7章思考题答案

**第7章思考题1**：
- 使用 Embedding 语义检索（第9章将讲解）先对用户问题做向量化，与 500 篇文档做相似度排序，取 Top 5。再对 Top 5 逐篇做 QA。相比逐篇 QA（500次推理），先召回再精排只需 6 次推理，速度提升 80 倍。

**第7章思考题2**：
- 方案：(1) 更新帮助文档中的退款时间描述；(2) 重新构建包含新答案的 QA 训练数据；(3) 增量微调模型（在全量微调和仅训练 QA Head 之间选择）；(4) 在生产环境做 A/B 测试——新模型、旧模型按流量灰度对比答案准确率。
