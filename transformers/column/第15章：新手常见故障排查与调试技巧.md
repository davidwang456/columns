## 1 项目背景

### 业务场景

新人算法工程师小王入职第二周，被分配了一个任务：复现团队的客服工单分类模型，然后在新数据集上微调。但小王从第一天开始就陷入了故障泥潭：

- 第一天：`pip install transformers` 后 `torch.cuda.is_available()` 返回 False，排查到晚上才发现装的是 CPU 版 PyTorch
- 第二天：模型下载到 80% 中断，重试三次都失败，最后发现是公司网络限制了到 `huggingface.co` 的连接
- 第三天：训练跑起来了，但 loss 一直在 0.69 附近震荡，排查发现 learning rate 设为 0.1（应该 2e-5）
- 第四天：训练终于收敛了，但测试集上所有预测都是同一个类——标签映射写反了
- 第五天：部署到测试环境后 CPU 使用率 100%，原来是在 Docker 中下了 GPU 版 PyTorch，掉到 CPU 模式慢 20 倍

小王在周报中写道："本周解决 5 个故障，实际写代码时间仅占 20%。"

### 痛点放大

NLP 新人最耗时的不是写代码，而是排查问题。核心痛点：

1. **环境问题**：CUDA 版本、PyTorch 版本、Transformers 版本三者兼容性复杂，报错信息不直观
2. **模型下载问题**：网络超时、文件不完整、缓存损坏，每次报错都不一样
3. **训练不收敛**：loss NaN、loss 不降、验证集指标虚高——排查需要逐一检查数据、参数、模型
4. **推理异常**：线上输出与本地不一致、延迟突然飙升、OOM

**小胖**:"那有什么通用的 debug 技巧吗？比如打印中间变量？"

**大师**:"三个最常用的调试技巧：

1. **打印 shape**：每次写出新代码时，打印 tensor 的 shape 验证维度正确。比如 tokenize 后马上 `print(input_ids.shape)`，模型输出后 `print(logits.shape)`。

2. **最小复现脚本**：不要把问题放在完整的 500 行训练脚本里排查。抽出一个 20 行的最小脚本——只有加载模型、tokenize 一条数据、前向传播一步。如果这跑不通，问题就在这 20 行里。

3. **transformers logging 级别**：设置 `transformers.logging.set_verbosity_debug()` 可以看到模型加载的详细日志——哪个权重被跳过了，哪个参数被随机初始化了。"

**技术映射总结**：
- 排查优先级 = 环境 > 数据 > 模型 > 参数
- 最小复现 = 用 Tiny 模型 + 5 条数据，快速缩小问题范围
- 日志 = 你的"眼睛"，debug logging 比 print 更结构化

```python
# debug_toolkit.py
"""新人故障排查工具箱"""

import torch
import transformers
import sys
from typing import Optional


class DebugToolkit:
    """故障排查工具箱"""

    # ===== 1. 环境诊断 =====
    @staticmethod
    def diagnose_environment() -> dict:
        """全面环境诊断"""
        report = {
            "python_version": sys.version,
            "torch_version": torch.__version__,
            "transformers_version": transformers.__version__,
            "cuda_available": torch.cuda.is_available(),
            "cuda_version": torch.version.cuda if torch.cuda.is_available() else "N/A",
            "gpu_count": torch.cuda.device_count() if torch.cuda.is_available() else 0,
        }

        # 常见问题检测
        issues = []

        # 检测1: 是否装了 CPU 版 PyTorch
        if not torch.cuda.is_available():
            try:
                import subprocess
                result = subprocess.run(["nvidia-smi"], capture_output=True, text=True)
                if result.returncode == 0:
                    issues.append(
                        "⚠ GPU 驱动存在但 PyTorch 检测不到 CUDA → 可能装了 CPU 版 PyTorch\n"
                        "   解决: pip uninstall torch && pip install torch --index-url "
                        "https://download.pytorch.org/whl/cu118"
                    )
            except FileNotFoundError:
                pass  # 确实没有 GPU

        # 检测2: tokenizers 是否为 Rust 版本
        try:
            import tokenizers
            report["tokenizers_version"] = tokenizers.__version__
        except ImportError:
            issues.append("⚠ tokenizers 未安装 → pip install tokenizers")

        report["issues"] = issues
        return report

    # ===== 2. 模型下载诊断 =====
    @staticmethod
    def diagnose_model_download(model_name: str = "prajjwal1/bert-tiny") -> dict:
        """诊断模型下载问题"""
        report = {}
        try:
            from transformers import AutoConfig
            config = AutoConfig.from_pretrained(model_name)
            report["download_ok"] = True
            report["config"] = str(config.__class__.__name__)
        except Exception as e:
            report["download_ok"] = False
            report["error"] = str(e)

            # 常见原因及方案
            if "ConnectionError" in str(e) or "MaxRetryError" in str(e):
                report["suggestion"] = (
                    "网络不通。方案: 1) 设置 HF_ENDPOINT=https://hf-mirror.com "
                    "2) 检查代理设置 3) 验证 huggingface.co 是否可达"
                )
            elif "403" in str(e):
                report["suggestion"] = (
                    "模型需要授权（gated model）。方案: 1) huggingface-cli login "
                    "2) 在 HF 网站上申请访问权限"
                )
            else:
                report["suggestion"] = f"未知错误，请检查网络连接和模型名称"
        return report

    # ===== 3. 训练数据诊断 =====
    @staticmethod
    def diagnose_training_data(dataset, tokenizer) -> dict:
        """诊断训练数据质量"""
        report = {}

        # 检测1: 数据量
        report["train_size"] = len(dataset.get("train", []))

        # 检测2: 标签分布
        from collections import Counter
        if "train" in dataset:
            labels = [example.get("label", example.get("labels", -1))
                      for example in dataset["train"]]
            label_counts = Counter(labels)
            report["label_distribution"] = dict(label_counts)

            # 标签不均衡检测
            total = sum(label_counts.values())
            for label, count in label_counts.items():
                ratio = count / total
                if ratio < 0.05:
                    report.setdefault("warnings", []).append(
                        f"标签 {label} 占比仅 {ratio:.1%}，可能导致该类被忽略"
                    )

        # 检测3: 文本长度分布
        if "train" in dataset and tokenizer:
            lengths = []
            for example in dataset["train"]:
                text = example.get("text", "")
                length = len(tokenizer.encode(text))
                lengths.append(length)

            if lengths:
                report["avg_length"] = sum(lengths) / len(lengths)
                report["max_length"] = max(lengths)
                over_limit = sum(1 for l in lengths if l > 512)
                report["over_512"] = over_limit

                if over_limit > len(lengths) * 0.1:
                    report.setdefault("warnings", []).append(
                        f"{over_limit}/{len(lengths)} 条文本超过512 tokens，将被截断"
                    )

        return report

    # ===== 4. 训练过程诊断 =====
    @staticmethod
    def diagnose_training_loss(losses: list) -> dict:
        """诊断训练 loss 异常"""
        report = {"loss_history": losses[-10:] if len(losses) > 10 else losses}

        if not losses:
            return report

        # 检测1: loss NaN
        if any(torch.isnan(torch.tensor(l)) for l in losses):
            report["issue"] = "NaN"
            report["suggestion"] = (
                "Loss 出现 NaN。按优先级排查:\n"
                "  1. 降低 learning_rate (建议 1e-5 ~ 3e-5)\n"
                "  2. 检查数据是否有 NaN/Inf\n"
                "  3. 加 gradient clipping: max_grad_norm=1.0\n"
                "  4. 降低 batch_size，排除显存问题"
            )
            return report

        # 检测2: loss 不下降（最后 10 步无明显下降）
        if len(losses) >= 10:
            recent = losses[-10:]
            if max(recent) - min(recent) < 0.01 and min(recent) > 0.5:
                report["issue"] = "不收敛"
                report["suggestion"] = (
                    "Loss 不下降。按优先级排查:\n"
                    "  1. learning_rate 是否合适 (太大或太小都不行)\n"
                    "  2. 是否正确设置了 model.train()\n"
                    "  3. 标签映射是否正确（label2id 映射是否错位）\n"
                    "  4. 数据是否混乱（随机打乱后标签不对应）"
                )

        return report

    # ===== 5. 推理结果诊断 =====
    @staticmethod
    def diagnose_predictions(predictions: list, labels: list) -> dict:
        """诊断推理结果异常"""
        from collections import Counter
        report = {}

        pred_counts = Counter(predictions)
        report["pred_distribution"] = dict(pred_counts)

        # 检测1: 全部同一个类
        if len(pred_counts) == 1:
            report["issue"] = "预测单一"
            report["suggestion"] = (
                "所有预测都是同一个类别。按优先级排查:\n"
                "  1. 检查 id2label 映射是否正确\n"
                "  2. 检查模型是否用了正确的 checkpoint\n"
                "  3. 模型可能坍塌（训练时标签极度不均衡）"
            )

        # 检测2: 中文全 [UNK]
        report["check_unk"] = (
            "如果输出大量 [UNK]，检查 tokenizer 与模型是否匹配，"
            "是否使用了中文模型"
        )

        # 检测3: 线上与本地不一致
        report["check_consistency"] = (
            "确保推理时的 tokenizer 配置（max_length、truncation、padding）"
            "与训练时完全一致"
        )

        return report


# ===== 使用示例 =====
if __name__ == "__main__":
    print("=" * 60)
    print("🔧 环境诊断")
    print("=" * 60)
    env_report = DebugToolkit.diagnose_environment()
    for key, value in env_report.items():
        if key != "issues":
            print(f"  {key}: {value}")
    if env_report.get("issues"):
        print("\n发现问题:")
        for issue in env_report["issues"]:
            print(f"  {issue}")

    print("\n" + "=" * 60)
    print("🔧 模型下载诊断")
    print("=" * 60)
    download_report = DebugToolkit.diagnose_model_download()
    print(f"  下载状态: {'✓ 正常' if download_report['download_ok'] else '✗ 失败'}")
    if not download_report["download_ok"]:
        print(f"  建议: {download_report.get('suggestion', '无')}")
```

### 3.3 最小复现脚本

```python
# minimal_test.py
"""最小复现脚本 —— 20 行验证训练流程"""

import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

MODEL = "prajjwal1/bert-tiny"  # 最小的 BERT 模型（仅 4.4MB）

def minimal_train_test():
    """最小训练可行性测试"""
    print("Step 1: 加载模型...")
    tokenizer = AutoTokenizer.from_pretrained(MODEL)
    model = AutoModelForSequenceClassification.from_pretrained(
        MODEL, num_labels=2
    )

    print(f"  模型参数: {sum(p.numel() for p in model.parameters()):,}")

    print("Step 2: Tokenize 一条数据...")
    text = "This is a test sentence."
    inputs = tokenizer(text, return_tensors="pt")
    print(f"  input_ids shape: {inputs['input_ids'].shape}")
    print(f"  tokens: {tokenizer.convert_ids_to_tokens(inputs['input_ids'][0])}")

    print("Step 3: 前向传播...")
    labels = torch.tensor([1])  # 假标签
    outputs = model(**inputs, labels=labels)
    print(f"  loss: {outputs.loss.item():.4f}")
    print(f"  logits shape: {outputs.logits.shape}")

    print("Step 4: 反向传播...")
    outputs.loss.backward()
    grad_norm = sum(p.grad.norm().item() for p in model.parameters() if p.grad is not None)
    print(f"  gradient norm: {grad_norm:.4f}")

    print("\n✓ 最小训练流程验证通过！环境和代码无问题。")
    return True


if __name__ == "__main__":
    try:
        minimal_train_test()
    except Exception as e:
        print(f"\n✗ 最小验证失败: {e}")
        print("请优先排查环境和依赖问题")
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **排查 SOP** | 结构化、有优先级，不盲目试错 | 无法覆盖所有边缘故障（如特定 GPU 型号兼容性） |
| **最小复现脚本** | 20 行代码快速定位问题层级 | 对于分布式训练/大模型 OOM 等问题无法完全模拟 |
| **日志级别控制** | debug 级别能看到模型加载和权重匹配细节 | 生产环境开 debug 会产生海量日志 |
| **DebugToolkit** | 自动化诊断，减少人工排查时间 | 诊断结果可能误报（如环境正常但 GPU 型号特定 bug） |

### 4.2 适用场景

| 场景 | 推荐工具/方法 |
|------|-------------|
| 新人搭建环境 | `diagnose_environment()` |
| 模型下载失败 | `diagnose_model_download()` + 镜像配置 |
| 训练不收敛 | `diagnose_training_data()` + `diagnose_training_loss()` |
| 推理结果异常 | `diagnose_predictions()` + 最小复现脚本 |

### 4.3 注意事项

1. **排查顺序**：永远先从环境检查开始，再检查数据，最后检查代码和参数——80% 的问题在前两步就能解决
2. **记录每次故障**：团队建立一个"踩坑文档"，每次解决一个新 bug 就补充 SOP，减少重复劳动
3. **版本锁定**：生产环境锁定 `transformers`、`torch`、`tokenizers` 的具体版本，避免自动升级引入新 bug

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决优先级 |
|---------|------|----------|
| `torch.cuda.is_available()` = False 但 GPU 存在 | CPU 版 PyTorch | ★★★★★ |
| 训练 loss 一直 0.69 | lr 太大或标签映射错 | ★★★★★ |
| 下载到一半中断 | 网络不稳定 | ★★★★ |
| 预测全是同一个类 | id2label 错位 | ★★★★ |
| Docker 中推理极慢 | 装了 GPU 版 PyTorch 但容器无 GPU | ★★★ |

### 4.6 第14章思考题答案

**第14章思考题1**：
- `await asyncio.sleep(0.1)`：不阻塞事件循环，其他请求可以并发处理，响应时间 ≈ 100ms（推理时间 + sleep 时间）
- `time.sleep(0.1)`：阻塞事件循环，当前请求处理期间无法处理其他请求，响应时间也是 100ms，但这 100ms 内服务对任何其他请求都无响应。在生产中这是致命的。

**第14章思考题2**：
- 热切换方案：(1) 维护一个当前版本配置（Redis/配置文件/环境变量）；(2) 后台协程每隔 30 秒检查配置是否变化；(3) 发现新版本后，在后台异步加载新版模型，完成后原子替换全局 model 引用；(4) 使用 `asyncio.Lock` 确保替换期间请求不被路由到半加载的模型。注意：新旧模型同时存在于内存中约 5-10 秒，需确保服务器内存充裕。
