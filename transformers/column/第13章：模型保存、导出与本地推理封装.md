# 第13章：模型保存、导出与本地推理封装

## 1 项目背景

### 业务场景

算法团队训练好客服工单分类模型后，后端工程师小李需要把这个模型集成到 Spring Boot 服务中。小陈把模型文件打包发给了小李——一个文件夹，里面有 `config.json`、`pytorch_model.bin`、`tokenizer_config.json`、`vocab.txt` 等 7 个文件。

小李打开一看就懵了："这些文件哪个是模型？怎么在 Java 里加载 Python 写的模型？"

小陈提议用 Python 写一个推理脚本，暴露 HTTP 接口给 Java 调用。于是小陈写了一个 `predict.py`，每次调用都 `AutoModel.from_pretrained()` 加载一次模型——结果首次调用耗时 8 秒（加载模型），后续每次调用耗时 200ms，生产流量一上来，P95 延迟直接飙到 5 秒。

测试团队也在抱怨：每次运行自动化测试都要重新下载模型（Github Actions 上网络不稳定），测试经常因模型下载超时而失败。

### 痛点放大

模型从训练完成到可以稳定被业务系统调用，中间有一道"最后一公里"工程难题：

1. **加载慢**：`from_pretrained()` 每次调用都重新加载，8 秒的冷启动在生产中不可接受
2. **文件散乱**：7 个文件缺一不可，部署时容易遗漏导致生产事故
3. **依赖耦合**：推理脚本依赖 `transformers` 库 + `torch`（合计 2GB），部署包过于臃肿
4. **没有标准接口**：每个模型各自写一套推理代码，预处理/后处理逻辑不一致

```
训练产出: 模型文件 (config.json, pytorch_model.bin, vocab.txt, ...)
              │
              ├── 方案A: 每次加载 (8s冷启动, 不可用)
              ├── 方案B: 全局单例 (快, 但无标准化封装)
              └── 方案C: 推理SDK封装 (目标!)
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周五下午 3:30，后端开发区和 ML 区交界处。小李和小陈围着一台服务器。

---

**小胖**（路过，手里拿着一盒蛋挞）:"你俩又在 debug？模型不是训好了吗，直接部署就行了呗。"

**小李**:"直接部署？我把小陈给的 7 个文件丢到服务器上，每次请求过来我都得 `from_pretrained()` 加载一次，一个请求等了 8 秒。用户早跑了。"

**小胖**:"那你不能只加载一次吗？就像食堂的打饭阿姨——每天早上把菜做好，中午直接打。不用每个学生来了都重新炒一盘鱼香肉丝吧？"

**小陈**:"小胖说得对，但实现上有讲究。我需要把模型加载为**全局单例**——服务启动时加载一次，所有请求复用同一个模型对象。问题在于：Python 的模块导入天然是单例，但 Flask/FastAPI 在多 worker 模式下每个 worker 有自己的进程空间，每个都要加载一遍。"

**小白**:"还有更麻烦的——模型部署不仅仅是加载的问题。你怎么保证预处理和后处理逻辑一致？怎么管理模型版本？怎么做到热切换模型而不中断服务？"

**大师**（合上笔记本走过来）:"这是模型工程化的问题。我来讲讲标准方案。

**第一层：模型文件组织。** 训练完成后，用 `trainer.save_model()` + `tokenizer.save_pretrained()` 把所有文件保存到一个目录。这个目录就是模型的'发布包'。目录里至少要有：`config.json`（结构）、`model.safetensors`（权重）、`tokenizer_config.json`（分词器配置）、`vocab.txt` 或 `tokenizer.json`（词表）。

**第二层：推理类封装。** 把加载、预处理、推理、后处理封装到一个 Python 类中。这个类在服务启动时初始化一次（全局单例），后续调用只执行推理。封装的好处：预处理逻辑和训练时完全一致（同一个 tokenizer 配置），后处理逻辑可以带置信度阈值、标签映射等业务规则。

**第三层：模型版本管理。** 用目录命名区分版本：`models/ticket_classifier/v1/`、`models/ticket_classifier/v2/`。推理类初始化时传入路径，`from_pretrained(path)` 加载对应版本。切换版本只需改配置，重启服务即可——不需要重新部署代码。"

**小胖**:"那有没有办法不依赖 Python 和 PyTorch 也能推理？我们的前端是 Node.js，后端是 Java。"

**大师**:"有，但要看场景。对 Transformer 来说，主流的跨语言方案：
- **ONNX Runtime**：把 PyTorch 模型导出为 `.onnx` 格式，用 ONNX Runtime 在 Java/C#/Node.js 中加载推理。适合分类、NER 等 Encoder 模型，导出简单，性能接近原生 PyTorch。
- **Triton Inference Server**：NVIDIA 的方案，支持多框架多模型管理，适合大规模推理服务。
- **直接 Python 微服务**：最灵活也最常用。Python 写推理服务暴露 HTTP/gRPC 接口，其他语言调用。对于小团队最实用。"

**技术映射总结**：
- 模型目录 = 模型的"发布包"，所有文件缺一不可
- 全局单例 = "早上做好的菜"，服务启动加载一次，请求复用
- 推理类封装 = "标准化接口"，预处理 + 推理 + 后处理一体
- ONNX = "跨语言翻译器"，把 PyTorch 模型转成通用格式

### 3.2 推理类封装

#### 目标

将模型加载、预处理、推理、后处理封装为可复用的 Python 类。

```python
# inference_package.py
"""标准推理封装 —— 单例加载、预处理 & 后处理一体"""

import os
import time
import logging
from typing import List, Dict, Optional, Union
from dataclasses import dataclass

import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)


@dataclass
class PredictResult:
    label: str
    label_id: int
    confidence: float
    latency_ms: float
    needs_review: bool


class ModelInference:
    """
    通用模型推理封装

    用法:
        model = ModelInference("./models/ticket_classifier_v2")
        result = model.predict("我要投诉！")
    """

    def __init__(self, model_path: str, device: str = None):
        """
        Args:
            model_path: 模型目录路径
            device: "cpu" / "cuda" / "cuda:0" / None（自动选择）
        """
        if not os.path.exists(model_path):
            raise FileNotFoundError(f"模型目录不存在: {model_path}")

        self.model_path = model_path
        self.device = device or ("cuda" if torch.cuda.is_available() else "cpu")

        # 加载（这一步慢，只在初始化时执行一次）
        logger.info(f"加载模型: {model_path}")
        start = time.time()

        self.tokenizer = AutoTokenizer.from_pretrained(model_path)
        self.model = AutoModelForSequenceClassification.from_pretrained(model_path)
        self.model.to(self.device)
        self.model.eval()

        self.id2label = self.model.config.id2label
        self.label2id = self.model.config.label2id
        self.num_labels = self.model.config.num_labels

        elapsed = time.time() - start
        logger.info(f"模型加载完成，耗时 {elapsed:.1f}s, 设备: {self.device}")

    def predict(
        self,
        text: str,
        confidence_threshold: float = 0.5,
        max_length: int = 128,
    ) -> PredictResult:
        """单条预测"""
        start = time.time()

        # 预处理
        inputs = self.tokenizer(
            text,
            max_length=max_length,
            truncation=True,
            padding="max_length",
            return_tensors="pt",
        ).to(self.device)

        # 推理
        with torch.no_grad():
            outputs = self.model(**inputs)
            probs = torch.softmax(outputs.logits, dim=-1)

        # 后处理
        top_prob, top_label = torch.max(probs, dim=-1)
        confidence = top_prob.item()
        label_id = top_label.item()
        label = self.id2label.get(str(label_id), f"class_{label_id}")

        latency = (time.time() - start) * 1000

        return PredictResult(
            label=label,
            label_id=label_id,
            confidence=round(confidence, 4),
            latency_ms=round(latency, 1),
            needs_review=confidence < confidence_threshold,
        )

    def predict_batch(
        self,
        texts: List[str],
        confidence_threshold: float = 0.5,
        batch_size: int = 32,
        max_length: int = 128,
    ) -> List[PredictResult]:
        """批量预测"""
        results = []

        for i in range(0, len(texts), batch_size):
            batch = texts[i:i + batch_size]

            inputs = self.tokenizer(
                batch,
                max_length=max_length,
                truncation=True,
                padding=True,
                return_tensors="pt",
            ).to(self.device)

            with torch.no_grad():
                outputs = self.model(**inputs)
                probs = torch.softmax(outputs.logits, dim=-1)

            top_probs, top_labels = torch.max(probs, dim=-1)

            for prob, label_id in zip(top_probs, top_labels):
                confidence = prob.item()
                lid = label_id.item()
                label = self.id2label.get(str(lid), f"class_{lid}")

                results.append(PredictResult(
                    label=label,
                    label_id=lid,
                    confidence=round(confidence, 4),
                    latency_ms=0,  # batch 模式下不单独计时
                    needs_review=confidence < confidence_threshold,
                ))

        return results

    def get_model_info(self) -> Dict:
        """获取模型元信息"""
        config = self.model.config
        param_count = sum(p.numel() for p in self.model.parameters())

        return {
            "model_path": self.model_path,
            "architecture": config.architectures if hasattr(config, "architectures") else "unknown",
            "num_labels": self.num_labels,
            "labels": list(self.id2label.values()),
            "parameters": f"{param_count/1e6:.1f}M",
            "device": self.device,
            "tokenizer_vocab_size": len(self.tokenizer),
        }


# ===== 使用示例 =====
if __name__ == "__main__":
    # 初始化（服务启动时执行一次）
    model = ModelInference("./models/sentiment_minimal")

    # 查看模型信息
    info = model.get_model_info()
    print(f"模型信息: {info}")

    # 单条推理
    print("\n单条推理:")
    result = model.predict("这个产品质量太差了，我要投诉！")
    print(f"  文本: 这个产品质量太差了，我要投诉！")
    print(f"  预测: {result.label} (置信度: {result.confidence:.2%})")
    print(f"  耗时: {result.latency_ms}ms")
    print(f"  需人工: {result.needs_review}")

    # 批量推理
    print("\n批量推理:")
    texts = [
        "客服态度很好",
        "商品质量太差了",
        "发货速度很快",
    ]
    results = model.predict_batch(texts)
    for text, r in zip(texts, results):
        print(f"  {text} → {r.label} ({r.confidence:.2%})")
```

### 3.3 模型格式转换

```python
# export_onnx.py
"""PyTorch 模型导出为 ONNX 格式"""

import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification


def export_to_onnx(model_path: str, output_path: str):
    """将 PyTorch 模型导出为 ONNX 格式"""
    tokenizer = AutoTokenizer.from_pretrained(model_path)
    model = AutoModelForSequenceClassification.from_pretrained(model_path)
    model.eval()

    # 准备示例输入（用于 ONNX 追踪计算图）
    dummy_text = "这是一个测试文本"
    dummy_inputs = tokenizer(
        dummy_text,
        max_length=128,
        truncation=True,
        padding="max_length",
        return_tensors="pt",
    )

    # 导出
    torch.onnx.export(
        model,
        (
            dummy_inputs["input_ids"],
            dummy_inputs["attention_mask"],
        ),
        output_path,
        input_names=["input_ids", "attention_mask"],
        output_names=["logits"],
        dynamic_axes={
            "input_ids": {0: "batch_size"},
            "attention_mask": {0: "batch_size"},
            "logits": {0: "batch_size"},
        },
        opset_version=14,
        do_constant_folding=True,
    )

    print(f"ONNX 模型已导出至: {output_path}")

    # 验证 ONNX 模型
    import onnxruntime as ort

    session = ort.InferenceSession(output_path)
    onnx_outputs = session.run(
        None,
        {
            "input_ids": dummy_inputs["input_ids"].numpy(),
            "attention_mask": dummy_inputs["attention_mask"].numpy(),
        },
    )

    # 对比 PyTorch 和 ONNX 的输出
    with torch.no_grad():
        torch_outputs = model(**dummy_inputs)

    diff = torch.abs(
        torch.tensor(onnx_outputs[0]) - torch_outputs.logits
    ).max().item()

    print(f"PyTorch vs ONNX 最大差异: {diff:.6f}")
    if diff < 1e-3:
        print("验证通过 ✓")
    else:
        print("警告: 差异较大，请检查导出配置")


# ===== ONNX 推理示例 =====
def onnx_inference(onnx_path: str, tokenizer_path: str, texts: list):
    """使用 ONNX Runtime 进行推理（无需 PyTorch）"""
    import onnxruntime as ort
    from transformers import AutoTokenizer
    import numpy as np

    tokenizer = AutoTokenizer.from_pretrained(tokenizer_path)
    session = ort.InferenceSession(onnx_path)

    inputs = tokenizer(
        texts,
        max_length=128,
        truncation=True,
        padding=True,
        return_tensors="np",
    )

    outputs = session.run(
        None,
        {
            "input_ids": inputs["input_ids"],
            "attention_mask": inputs["attention_mask"],
        },
    )

    logits = outputs[0]
    probs = np.exp(logits) / np.exp(logits).sum(axis=-1, keepdims=True)
    preds = np.argmax(probs, axis=-1)
    confidences = np.max(probs, axis=-1)

    return preds, confidences


if __name__ == "__main__":
    export_to_onnx(
        model_path="./models/sentiment_minimal",
        output_path="./models/sentiment_minimal/model.onnx",
    )

    # 测试 ONNX 推理
    preds, confs = onnx_inference(
        onnx_path="./models/sentiment_minimal/model.onnx",
        tokenizer_path="./models/sentiment_minimal",
        texts=["这个产品很好", "太差了要投诉"],
    )
    print(f"\nONNX 推理结果: {preds}, 置信度: {confs}")
```

### 3.4 模型版本管理工具

```python
# model_registry.py
"""模型版本注册与管理"""

import os
import json
import shutil
from pathlib import Path
from datetime import datetime
from typing import Optional


class ModelRegistry:
    """本地模型注册表"""

    def __init__(self, registry_dir: str = "./model_registry"):
        self.registry_dir = Path(registry_dir)
        self.registry_dir.mkdir(parents=True, exist_ok=True)

    def register(self, model_path: str, version: str,
                 metadata: Optional[dict] = None) -> str:
        """
        注册一个模型版本

        Args:
            model_path: 源模型目录
            version: 版本号，如 "v1.2.0"
            metadata: 额外元数据（训练指标、日期、数据集等）
        Returns:
            注册后的模型路径
        """
        target_dir = self.registry_dir / version

        # 复制模型到注册表
        if target_dir.exists():
            raise FileExistsError(f"版本 {version} 已存在")

        shutil.copytree(model_path, str(target_dir))

        # 写入元数据
        meta = metadata or {}
        meta.update({
            "version": version,
            "registered_at": datetime.now().isoformat(),
            "source_path": str(model_path),
        })

        with open(target_dir / "registry_meta.json", "w", encoding="utf-8") as f:
            json.dump(meta, f, ensure_ascii=False, indent=2)

        print(f"模型已注册: {version} → {target_dir}")
        return str(target_dir)

    def list_versions(self) -> list:
        """列出所有已注册的版本"""
        versions = []
        for d in sorted(self.registry_dir.iterdir()):
            if d.is_dir():
                meta_file = d / "registry_meta.json"
                meta = {}
                if meta_file.exists():
                    with open(meta_file) as f:
                        meta = json.load(f)

                versions.append({
                    "version": d.name,
                    "path": str(d),
                    "registered_at": meta.get("registered_at", "unknown"),
                })
        return versions

    def get_latest(self) -> Optional[str]:
        """获取最新版本路径"""
        versions = self.list_versions()
        if not versions:
            return None
        return versions[-1]["path"]


# ===== 使用示例 =====
if __name__ == "__main__":
    registry = ModelRegistry("./model_registry")

    # 注册 v1.0
    registry.register(
        model_path="./models/sentiment_minimal",
        version="v1.0.0",
        metadata={"accuracy": 0.91, "dataset": "tickets_2024Q1"}
    )

    # 查看版本列表
    print("已注册版本:")
    for v in registry.list_versions():
        print(f"  {v['version']}: {v['registered_at']}")

    # 获取最新版本
    latest = registry.get_latest()
    print(f"最新版本: {latest}")
```

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **推理封装** | 预处理+推理+后处理统一，调用方无需关心内部细节 | 类内部状态（如 tokenizer 配置）需要与训练时严格一致 |
| **全局单例** | 加载一次秒级响应，内存占用可控 | 无法在线热切换模型，需重启服务 |
| **ONNX 导出** | 跨语言、跨平台，无需 PyTorch 依赖 | 仅支持静态图，动态 shape 和自定义算子支持有限 |
| **模型版本管理** | 可追溯、可回滚 | 多版本并存占用额外磁盘空间 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 小团队快速上线 | Python 推理封装 + FastAPI HTTP 服务 |
| Java/Go/C# 后端集成 | ONNX Runtime 推理 |
| 多模型多版本管理 | ModelRegistry + 配置化路由 |
| 边缘设备部署 | ONNX / CoreML / TensorFlow Lite |

**不适用场景**：
- 超大模型（>7B）单卡部署：需配合量化或 vLLM 等专用推理框架
- 需要流式输出（chat streaming）：需单独处理 SSE/WebSocket

### 4.3 注意事项

1. **tokenizer 配置与训练一致**：推理时 `max_length`、`truncation`、`padding` 参数必须与训练时相同
2. **`id2label` 写入 config**：保存模型时把 `id2label` 写入 `config.json`，否则推理时 label_id 无法映射回业务标签
3. **ONNX opset 版本**：Transformers 4.x 导出的模型建议 `opset_version=14`，过低会导致某些算子不支持

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 推理输出全是同一个标签 | `id2label` 缺失，读取时用了默认的 "LABEL_0" 等 | 训练保存时传 `model.config.id2label = id2label` |
| ONNX 推理结果与 PyTorch 相差很大 | 动态轴配置错误或 attention_mask 传递有误 | 对比中间层输出定位差异 |
| 多 worker 模式下 OOM | 每个 worker 进程独立加载模型 | 限制 worker 数量或使用共享内存（较复杂） |

### 4.5 思考题

1. **初级**：在 `ModelInference` 中，如果用 `device="cuda"` 初始化，但你的机器没有 GPU，会发生什么？如何让代码自动降级到 CPU？
2. **进阶**：你有 v1 和 v2 两个版本的分类模型。v2 的 `id2label` 与 v1 相同但准确率更高。请设计一个**灰度发布**方案——10% 流量走 v2，90% 走 v1，逐步切换并对比指标。
