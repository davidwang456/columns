# 第31章：中级篇综合实战：企业知识库问答与模型服务平台

## 1 项目背景

### 业务场景

某金融科技公司内部有超过 3000 份合规文档、产品手册和运维 SOP，分散在 Confluence、SharePoint 和本地文件服务器中。员工查找信息平均耗时 12 分钟——有时翻遍三个系统都找不到。风控部门甚至因为没找到最新版的合规条款，导致一次监管检查中被扣分。公司面临严重的知识管理危机：新员工入职需要 2 周才能熟悉内部文档体系，老员工离职时大量隐性知识流失。

CTO 在阅读了基础篇和中级篇之后提出了一个大胆的目标："用我们学过的技术，搭建一个内部 AI 知识平台。员工用自然语言提问，系统从文档中找到答案，生成可引用来源的回复。模型和知识库要支持版本管理和灰度更新。这个平台不是一次性项目——未来要成为公司所有 AI 能力的统一入口。"

技术团队梳理了需要的组件：文档切分 + Embedding 索引 + FAISS 检索 + RAG 生成 + FastAPI 服务 + 异步队列 + 限流熔断 + Prometheus 监控 + 版本管理 + 灰度发布 + 安全网关——几乎覆盖了中级篇的全部知识点。

### 痛点放大

综合实战的挑战不在于单个组件的实现，而在于**系统集成**和**运维闭环**：

1. **数据流串联**：文档上传 → 切分 → 向量化 → 索引构建 → 检索 → 生成 → 返回，任一环节挂了都会影响最终效果
2. **多模型共存**：Embedding 模型和生成模型共享 GPU，显存和调度需要协调
3. **知识库更新**：文档新增/修改后如何增量更新索引而不影响在线服务？
4. **生产运维**：系统上线后如何监控问答质量？如何发现检索漂移？如何灰度切换模型？
5. **安全合规**：金融行业的文档涉及客户隐私，AI 回答绝不能泄露敏感信息

**小陈**:"架构我大致理清楚了，但有个顾虑——Embedding 模型（500MB）和生成模型（1GB）都放 GPU 上，显存够吗？之前在单卡上同时加载两个模型经常 OOM。"

**小白**:"而且知识库有 3000 份文档，每份切 10 个 chunk 就是 3 万个向量。检索延迟能控制在 50ms 以内吗？如果文档更新了，索引重建需要多久？重建期间服务会中断吗？另外，安全审计怎么接入？金融行业对AI输出有合规要求。"

**大师**:"好，让我把综合实战的五个关键架构决策讲清楚。

**决策一：模型资源调度。** 单张 24GB GPU 同时加载 Embedding + 生成模型可行，但需要精细控制：
- Embedding 模型（Sentence-BERT 约 500MB FP16）放在 GPU
- 生成模型用 INT4 量化（1GB→约 300MB），两个模型共享 GPU
- 如果显存不够，Embedding 模型放 GPU（检索频率高需低延迟），生成模型放 CPU（生成频率低可容忍稍慢）
- 后期可迁移到 vLLM 做生成推理（Continuous Batching，吞吐提升 6 倍）

**决策二：索引增量更新。** 不能每次加一篇文档就全量重建索引。方案：
- 维护一个"主索引"（全量）和一个"增量索引"（最近 24 小时新增的文档）
- 搜索时同时查询两个索引，合并去重
- 每天凌晨低峰期将增量合并到主索引，重建后原子替换
- 使用双缓冲：重建新索引 → 验证 → 原子替换指针，全程不影响在线查询

**决策三：问答质量闭环。** 上线后如何知道 RAG 回答得好不好？
- 在 UI 上给每条回答加"有帮助/无帮助"按钮（用户反馈）
- 后台每天抽样 50 条无帮助的回答，人工分析是检索问题还是生成问题
- 将反馈数据用于 Embedding 模型和生成模型的增量微调
- 建立质量看板：引用准确率、拒答率、用户满意度趋势

**决策四：部署与灰度。** 
- FastAPI 单服务挂载所有组件（前期 QPS 不高）
- 模型版本通过环境变量 `MODEL_VERSION=v2.1.0` 控制
- 灰度：nginx upstream + 两套服务（stable + canary），按 user_id hash 分流
- 回滚：网关层切换路由到旧版本服务，秒级生效

**决策五：安全合规。** 金融行业特殊要求：
- 所有回答必须标注引用来源（可审计）
- 输出必须经过 PII 脱敏（身份证号、银行卡号等）
- Prompt Injection 防护：用户输入不能覆盖系统指令
- 审计日志保留 90 天，记录每次问答的输入/输出/trace_id"

**技术映射总结**：
- 模型资源调度 = 合租室友分配房间，GPU 放高频模型，CPU 放低频模型
- 增量索引 + 双缓冲 = 日记本+活页夹，新内容写活页，晚上整理进日记本，不减慢白天翻页
- 反馈闭环 = 餐厅的意见卡，差评多的菜改进，好评多的保持
- 灰度部署 = 新菜品试吃，10% 顾客先尝，没问题再全量
- 安全合规 = 银行金库的三道门，层层检查

```python
# document_indexer.py
"""文档上传 → 切分 → 向量化 → 索引构建"""

import os, json, hashlib, time
from pathlib import Path
from typing import List, Dict, Optional
from dataclasses import dataclass

import numpy as np
from sentence_transformers import SentenceTransformer
import faiss


@dataclass
class Document:
    doc_id: str
    title: str
    content: str
    source: str
    version: str
    uploaded_at: str


class DocumentProcessor:
    """文档处理器：支持 PDF/TXT/Markdown"""

    @staticmethod
    def parse_file(file_path: str) -> str:
        ext = Path(file_path).suffix.lower()
        if ext == ".txt" or ext == ".md":
            with open(file_path, "r", encoding="utf-8") as f:
                return f.read()
        elif ext == ".pdf":
            from pypdf import PdfReader
            reader = PdfReader(file_path)
            return "\n".join(page.extract_text() or "" for page in reader.pages)
        else:
            raise ValueError(f"不支持的文件格式: {ext}")

    @staticmethod
    def split_text(text: str, chunk_size: int = 500,
                   chunk_overlap: int = 80) -> List[Dict]:
        """智能切分：优先在段落/句子边界切分"""
        chunks = []
        start = 0
        chunk_idx = 0

        while start < len(text):
            end = min(start + chunk_size, len(text))
            if end < len(text):
                for sep in ["\n\n", "\n", "。", "；", ". "]:
                    last_sep = text.rfind(sep, start, end)
                    if last_sep > start + chunk_size * 0.5:
                        end = last_sep + len(sep)
                        break

            chunk_text = text[start:end].strip()
            if len(chunk_text) >= 50:
                chunks.append({
                    "chunk_id": chunk_idx,
                    "text": chunk_text,
                })
                chunk_idx += 1
            start = end - chunk_overlap

        return chunks


class KnowledgeIndex:
    """知识库向量索引 —— 支持增量更新和双缓冲"""

    def __init__(self, embedder_model: str = "paraphrase-multilingual-MiniLM-L12-v2"):
        self.embedder = SentenceTransformer(embedder_model)
        self.processor = DocumentProcessor()

        # 主索引 + 增量索引
        self.main_chunks: List[Dict] = []
        self.main_index: Optional[faiss.Index] = None
        self.delta_chunks: List[Dict] = []
        self.delta_index: Optional[faiss.Index] = None

        self.documents: Dict[str, Document] = {}
        self._lock = False  # 简单的锁标志

    def add_document(self, file_path: str, title: str,
                     source: str = "", version: str = "v1") -> str:
        """添加文档到索引"""
        content = self.processor.parse_file(file_path)
        doc_id = hashlib.md5(content.encode()).hexdigest()[:12]

        # 检查是否已存在相同内容
        if doc_id in self.documents:
            return doc_id

        doc = Document(
            doc_id=doc_id, title=title, content=content,
            source=source or file_path, version=version,
            uploaded_at=time.strftime("%Y-%m-%d %H:%M:%S"),
        )
        self.documents[doc_id] = doc

        # 切分并添加到增量索引
        chunks = self.processor.split_text(content)
        for c in chunks:
            c["doc_id"] = doc_id
            c["title"] = title
            c["source"] = source

        self.delta_chunks.extend(chunks)
        self._rebuild_delta_index()

        print(f"文档已添加: {title} ({len(chunks)} chunks, doc_id={doc_id})")
        return doc_id

    def _rebuild_delta_index(self):
        """重建增量索引"""
        if not self.delta_chunks:
            return
        texts = [c["text"] for c in self.delta_chunks]
        embeddings = self.embedder.encode(texts, normalize_embeddings=True)
        dim = embeddings.shape[1]
        self.delta_index = faiss.IndexFlatIP(dim)
        self.delta_index.add(embeddings.astype(np.float32))

    def merge_delta_to_main(self):
        """将增量索引合并到主索引（通常在低峰期执行）"""
        if not self.delta_chunks:
            return

        print(f"合并增量索引: {len(self.delta_chunks)} chunks → 主索引")
        self.main_chunks.extend(self.delta_chunks)
        self.delta_chunks = []
        self.delta_index = None

        # 重建主索引
        if self.main_chunks:
            texts = [c["text"] for c in self.main_chunks]
            embeddings = self.embedder.encode(texts, normalize_embeddings=True,
                                              show_progress_bar=True)
            dim = embeddings.shape[1]
            self.main_index = faiss.IndexFlatIP(dim)
            self.main_index.add(embeddings.astype(np.float32))

        print(f"合并完成: 主索引 {len(self.main_chunks)} chunks")

    def search(self, query: str, top_k: int = 5,
               min_score: float = 0.3) -> List[Dict]:
        """混合搜索：主索引 + 增量索引"""
        query_vec = self.embedder.encode(
            [query], normalize_embeddings=True
        ).astype(np.float32)

        all_results = []

        # 搜索主索引
        if self.main_index and self.main_chunks:
            scores, indices = self.main_index.search(query_vec, top_k * 2)
            for score, idx in zip(scores[0], indices[0]):
                if idx >= 0 and idx < len(self.main_chunks):
                    all_results.append({
                        **self.main_chunks[idx],
                        "score": float(score),
                        "from": "main",
                    })

        # 搜索增量索引
        if self.delta_index and self.delta_chunks:
            scores, indices = self.delta_index.search(query_vec, top_k * 2)
            for score, idx in zip(scores[0], indices[0]):
                if idx >= 0 and idx < len(self.delta_chunks):
                    all_results.append({
                        **self.delta_chunks[idx],
                        "score": float(score),
                        "from": "delta",
                    })

        # 去重（同 doc_id 同 chunk_id 取高分）
        seen = set()
        unique_results = []
        for r in sorted(all_results, key=lambda x: x["score"], reverse=True):
            key = (r.get("doc_id", ""), r.get("chunk_id", -1))
            if key not in seen and r["score"] >= min_score:
                seen.add(key)
                unique_results.append(r)

        return unique_results[:top_k]

    def get_stats(self) -> Dict:
        return {
            "main_chunks": len(self.main_chunks),
            "delta_chunks": len(self.delta_chunks),
            "documents": len(self.documents),
            "total_chunks": len(self.main_chunks) + len(self.delta_chunks),
        }
```

# knowledge_platform.py

# ===== 数据模型 =====
class AskRequest(BaseModel):
    question: str = Field(..., min_length=1, max_length=1000)
    user_id: str = Field("anonymous")

class FeedbackRequest(BaseModel):

# ===== 简易安全网关（内嵌） =====
class SecurityFilter:
    FORBIDDEN_INPUT = [
        r"忽略.*指令", r"你是.{1,5}角色", r"system\s*:", r"<\|.*?\|>",
        r"忘记.*设定", r"ignore.*instruction",
    ]
    PII_PATTERNS = {
        "phone": r'1[3-9]\d{9}',
        "id_card": r'\d{17}[\dXx]',
        "bank_card": r'\d{16,19}',
    }

    @classmethod
    def check_input(cls, text: str) -> tuple:
        import re
        for pattern in cls.FORBIDDEN_INPUT:
            if re.search(pattern, text, re.IGNORECASE):
                return False, f"检测到潜在注入攻击"
        return True, ""

    @classmethod
    def mask_output(cls, text: str) -> str:
        import re
        for pii_type, pattern in cls.PII_PATTERNS.items():
            text = re.sub(pattern, f"[已脱敏:{pii_type}]", text)
        return text


# ===== 全局组件 =====

_generator_model = None
_generator_tokenizer = None
_start_time = None

app = FastAPI(title="企业知识库问答平台 v3.0", lifespan=lifespan)

@app.middleware("http")
async def log_middleware(request: Request, call_next):
    trace_id = str(uuid.uuid4())[:8]
    request.state.trace_id = trace_id
    start = time.time()
    response = await call_next(request)
    elapsed = (time.time() - start) * 1000
    logger.info(f"trace_id={trace_id} path={request.url.path} "
                f"status={response.status_code} latency={elapsed:.1f}ms")
    response.headers["X-Trace-ID"] = trace_id
    return response


@app.post("/ask", response_model=AskResponse)
async def ask(request: AskRequest, req: Request):
    """知识库问答"""
    trace_id = getattr(req.state, "trace_id", str(uuid.uuid4())[:8])
    start = time.time()

    # 安全输入检查
    is_safe, reason = SecurityFilter.check_input(request.question)
    if not is_safe:
        logger.warning(f"安全拦截: trace_id={trace_id} reason={reason}")
        raise HTTPException(403, f"输入包含不安全内容: {reason}")

    # 1. 检索
    chunks = _index.search(request.question, top_k=request.top_k)
    if not chunks:
        return AskResponse(
            answer="抱歉，知识库中未找到相关信息。建议您联系相关团队或查阅最新版本文档。",
            sources=[], confidence=0,
            latency_ms=round((time.time()-start)*1000, 1),
            trace_id=trace_id, model_version=MODEL_VERSION,
        )

    # 2. 生成（RAG Prompt）
    context_parts = []
    for i, c in enumerate(chunks):
        source_info = f"{c.get('title','文档')}"
        context_parts.append(f"[来源{i+1}] {source_info}\n{c['text'][:400]}")
    context = "\n\n".join(context_parts)

    prompt = (
        f"你是一个企业知识库助手。请严格根据以下参考资料回答用户问题。\n"
        f"规则：1) 只能使用参考资料中的信息 2) 如果资料不足以回答，请明确说无法回答\n"
        f"      3) 回答时引用来源编号，如 [来源1]\n\n"
        f"参考资料:\n{context}\n\n"
        f"用户问题: {request.question}\n"
        f"回答:"
    )

    inputs = _generator_tokenizer(prompt, return_tensors="pt",
                                   truncation=True, max_length=1536)
    gen_start = time.time()
    with torch.no_grad():
        outputs = _generator_model.generate(
            **inputs, max_new_tokens=200, temperature=0.3,
            top_p=0.9, do_sample=True,
            pad_token_id=_generator_tokenizer.eos_token_id,
            repetition_penalty=1.1,
        )
    gen_time = (time.time() - gen_start) * 1000

    answer = _generator_tokenizer.decode(outputs[0], skip_special_tokens=True)
    answer = answer.split("回答:")[-1].strip()

    # 3. 输出脱敏
    answer = SecurityFilter.mask_output(answer)

    latency = round((time.time() - start) * 1000, 1)

    # 4. 审计日志
    _log_audit(trace_id, request.question, answer, request.user_id,
               [c.get("title","") for c in chunks], latency)

    return AskResponse(
        answer=answer,
        sources=[{"title": c.get("title",""), "score": c["score"],
                  "chunk_id": c.get("chunk_id")} for c in chunks[:3]],
        confidence=round(chunks[0]["score"], 4) if chunks else 0,
        latency_ms=latency, trace_id=trace_id, model_version=MODEL_VERSION,
    )

## 4 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **组件复用** | 直接复用前 30 章的代码模块（RAG/安全/灰度/监控） | 模块耦合处需要适配代码 |
| **安全网关** | 输入输出全链路安全，PII 脱敏 | 增加约 5ms 的请求处理延迟 |
| **增量索引** | 文档实时可检索，不影响在线服务 | 合并操作需在低峰期执行 |
| **审计日志** | 满足金融合规要求，可追溯 | 日志量随 QPS 增长，需归档策略 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 企业内部知识库（< 10000 文档） | 单服务模式 |
| 大规模公网服务（> 100 QPS） | 微服务拆分（检索服务 + 生成服务 + API网关） |
| 高安全要求场景（金融/医疗/政务） | 安全网关 + 审计日志 + 数据脱敏 + 私有化部署 |

**不适用场景**：
- 实时数据问答（股票/新闻），索引更新延迟不满足
- 多媒体知识库（需要视频/音频理解能力 → 需接入多模态模型）

### 4.3 注意事项

1. **文档增量更新**：避免在线重建索引导致查询超时，用双缓冲策略
2. **RAG 幻觉控制**：prompt 中必须强化"不知道就说不知道"，temperature 建议 ≤0.3
3. **成本核算**：记录每次问答的 token 消耗，监控日/月成本
4. **模型版本兼容**：切换生成模型时确保 tokenizer 配置一致

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| 检索结果不包含新文档 | 增量索引未合并到主索引 | 检查定时合并任务是否正常执行 |
| 生成内容与引用不符 | LLM 忽略 prompt 中的引用要求 | 降低 temperature、增加 few-shot 示例 |
| 服务内存持续增长 | 模型推理后的缓存未释放 | 定期 `torch.cuda.empty_cache()`，设置 Python GC |
| Prompt Injection 绕过检测 | 正则规则库不全 | 定期更新检测规则，接入 LLM 作为二级审核 |

### 4.5 思考题

1. **初级**：在知识平台中增加一个 `/quality_report` 接口——统计最近 24 小时的帮助率、平均置信度、拒答率、Top 10 热门问题。
2. **进阶**：设计一个**自动评估系统**——每天自动抽样 100 条问答，用另一个 LLM 作为"裁判"评估答案质量（准确性、完整性、引用正确性各 1-5 分），输出评分报告并自动触发告警（连续 3 天评分下降 > 10%）。
