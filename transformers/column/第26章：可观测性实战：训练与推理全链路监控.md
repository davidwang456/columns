# 第26章：可观测性实战：训练与推理全链路监控

## 1 项目背景

### 业务场景

客服工单分类系统上线半年后，已稳定运行。某个周三下午 3:00，运营团队反馈："自动分派的准确率好像突然变差了，投诉工单基本都被分到了咨询组"。算法团队紧急排查，但遇到了致命问题——没有历史记录。不知道准确率是什么时候开始下降的，不知道是模型退化还是数据漂移，甚至不确定下降了多少。唯一能做的就是在当前测试集上重新评估，但测试集是半年前的，无法代表当前线上分布。

与此同时，训练侧也有隐患：实习生小李跑了一组超参数实验后离职了，没人知道那组最好的 checkpoint 是怎么训出来的——用了什么 lr、跑了几个 epoch、训练数据是哪个版本。

技术总监拍板："必须建立可观测性体系——训练和推理的每一个关键指标都要可追踪、可回溯、可报警。"

### 痛点放大

ML 系统的可观测性比传统软件更复杂——需要同时监控训练管道和推理服务两个维度：

```
训练侧盲区:                      推理侧盲区:
┌──────────────────┐            ┌──────────────────┐
│ 哪个实验最优？     │            │ 线上准确率变化？    │
│ loss 为什么震荡？  │            │ 延迟为什么飙升？    │
│ 数据版本是什么？   │            │ 输出分布是否漂移？  │
│ 超参数记录在哪？   │            │ 显存是否泄漏？      │
└──────────────────┘            └──────────────────┘
          ↓                              ↓
    实验追踪缺失                    线上监控空白
```

## 2 项目设计

### 剧本式交锋对话

**场景**：周四上午 10:00，复盘会议室。大屏幕上一片空白——没有历史监控数据。运维小王急得直挠头。

---

**小胖**:"所以我们就像在黑屋子里开车，连仪表盘都没有？那不出事才怪。至少得有个速度表（QPS）和油量表（显存）吧？"

**小白**:"不仅仅是实时指标，还需要历史回溯。训练侧的实验追踪和推理侧的线上监控是两个不同的体系，但都需要统一的可观测性基础设施。"

**大师**:"好，让我把 ML 可观测性的四个支柱讲清楚。

**支柱一：训练实验追踪。** 每次训练实验必须记录：超参数（lr、batch_size、epochs）、数据集版本（哪个切分、多少条）、模型版本（base model + commit hash）、训练曲线（loss/指标随时间变化）、最终 checkpoint 路径。工具选型：TensorBoard 免费简单适合单机，Weights & Biases (WandB) 功能全适合团队协作，MLflow 开源可私有化部署适合合规要求高的企业。

**支柱二：推理服务监控。** 黄金指标（RED 方法）：
- **Rate（请求量）**：QPS、每分钟请求数
- **Errors（错误率）**：4xx/5xx 比例、模型推理失败率、超时率
- **Duration（延迟）**：P50/P95/P99 延迟、TTFT（首个 token 时间）、TPOT（每 token 时间）

外加 ML 特有指标：模型输出分布（各类别占比）、置信度分布、输入文本长度分布。这些是发现数据漂移的"金丝雀"。

**支柱三：结构化日志。** 每条请求打一条 JSON 日志，字段至少包含：
```json
{
  "trace_id": "a1b2c3d4",
  "model_version": "v2.1.0",
  "input_hash": "sha256...",
  "input_length": 128,
  "predicted_label": "投诉",
  "confidence": 0.87,
  "latency_ms": 52.3,
  "error": null
}
```
所有日志接入 ELK 或 Grafana Loki，可按任意字段查询和聚合。

**支柱四：告警规则。** 至少配置以下 5 条：
1. P95 延迟 > 200ms 持续 5 分钟 → 告警
2. 5xx 错误率 > 1% 持续 3 分钟 → 紧急告警
3. GPU 显存利用率 > 95% 持续 10 分钟 → 告警（可能 OOM）
4. 模型输出中某类别占比突变 > 30% → 告警（数据漂移）
5. 推理服务健康检查连续失败 3 次 → 紧急告警"

**小胖**:"这么多指标，谁来盯？"

**大师**:"不需要人盯。通过 Grafana 大盘可视化 + Prometheus AlertManager 自动告警，人只关心告警。日常关注的只有一张大盘和几条告警规则。"

**技术映射总结**：
- 实验追踪 = 菜谱记录——每个菜（实验）用了什么料、火候多大都要记
- RED 指标 = 餐厅的翻台率、投诉率、出餐速度
- 结构化日志 = 每桌客人的点菜单和出菜记录
- 告警 = 厨房烟感报警器，响了就去处理

### 3.2 训练实验追踪

```python
# training_observer.py
"""训练可观测性：TensorBoard + WandB + 自定义 Callback"""

import os
import json
import torch
import numpy as np
from datetime import datetime
from transformers import TrainerCallback, Trainer
from torch.utils.tensorboard import SummaryWriter
from typing import Dict, Any


class ExperimentTracker:
    """训练实验追踪 —— 统一管理 TensorBoard 和 WandB"""

    def __init__(self, experiment_name: str, use_wandb: bool = False,
                 use_tensorboard: bool = True, log_dir: str = "./logs"):
        self.experiment_name = experiment_name
        self.run_id = datetime.now().strftime("%Y%m%d_%H%M%S")

        # TensorBoard
        self.tb_writer = None
        if use_tensorboard:
            tb_dir = os.path.join(log_dir, "tensorboard", f"{experiment_name}_{self.run_id}")
            self.tb_writer = SummaryWriter(tb_dir)

        # WandB
        self.wandb_run = None
        if use_wandb:
            import wandb
            self.wandb_run = wandb.init(
                project="ticket-classifier",
                name=f"{experiment_name}_{self.run_id}",
                config={},  # 后续填充
            )

        # 实验元数据
        self.metadata = {
            "experiment_name": experiment_name,
            "run_id": self.run_id,
            "start_time": datetime.now().isoformat(),
            "metrics_history": [],
            "hyperparameters": {},
        }

    def log_hparams(self, hparams: Dict[str, Any]):
        """记录超参数"""
        self.metadata["hyperparameters"] = {k: str(v) for k, v in hparams.items()}
        if self.wandb_run:
            self.wandb_run.config.update(hparams)

    def log_metrics(self, metrics: Dict[str, float], step: int, prefix: str = ""):
        """记录指标"""
        prefixed = {f"{prefix}/{k}" if prefix else k: v
                    for k, v in metrics.items()}

        # TensorBoard
        if self.tb_writer:
            for k, v in prefixed.items():
                self.tb_writer.add_scalar(k, v, step)

        # WandB
        if self.wandb_run:
            self.wandb_run.log(prefixed, step=step)

        # 历史记录
        self.metadata["metrics_history"].append({
            "step": step, "metrics": metrics, "timestamp": datetime.now().isoformat()
        })

    def log_model_info(self, model):
        """记录模型信息"""
        total_params = sum(p.numel() for p in model.parameters())
        trainable_params = sum(p.numel() for p in model.parameters() if p.requires_grad)
        self.metadata["model_info"] = {
            "total_params": total_params,
            "trainable_params": trainable_params,
            "trainable_ratio": trainable_params / total_params,
        }

    def save_metadata(self):
        """保存实验元数据到 JSON"""
        self.metadata["end_time"] = datetime.now().isoformat()
        os.makedirs("./experiments", exist_ok=True)
        path = f"./experiments/{self.experiment_name}_{self.run_id}.json"
        with open(path, "w", encoding="utf-8") as f:
            json.dump(self.metadata, f, ensure_ascii=False, indent=2)
        print(f"实验元数据已保存: {path}")

    def close(self):
        if self.tb_writer:
            self.tb_writer.close()
        if self.wandb_run:
            self.wandb_run.finish()


class ObservabilityCallback(TrainerCallback):
    """Trainer 可观测性 Callback —— 自动记录更多细节"""

    def __init__(self, tracker: ExperimentTracker):
        self.tracker = tracker
        self._epoch_losses = []

    def on_log(self, args, state, control, logs=None, **kwargs):
        if logs:
            step = state.global_step
            self.tracker.log_metrics(
                {k: v for k, v in logs.items() if isinstance(v, (int, float))},
                step=step,
                prefix="train"
            )

    def on_evaluate(self, args, state, control, metrics=None, **kwargs):
        if metrics:
            self.tracker.log_metrics(metrics, step=state.global_step, prefix="eval")

    def on_save(self, args, state, control, **kwargs):
        """记录 checkpoint 保存"""
        self.tracker.metadata.setdefault("checkpoints", []).append({
            "step": state.global_step,
            "epoch": state.epoch,
            "path": args.output_dir,
        })

    def on_train_end(self, args, state, control, **kwargs):
        self.tracker.save_metadata()
        self.tracker.close()


# ===== 使用示例 =====
if __name__ == "__main__":
    tracker = ExperimentTracker("bert_ticket_classifier_v3", use_tensorboard=True)

    tracker.log_hparams({
        "learning_rate": 2e-5,
        "batch_size": 16,
        "epochs": 3,
        "model": "bert-base-chinese",
        "dataset_version": "tickets_2024Q2_v2",
    })

    # 模拟训练指标
    for step in range(0, 500, 50):
        tracker.log_metrics({
            "loss": 0.5 * np.exp(-step/200) + 0.1,
            "learning_rate": 2e-5 * (1 - step/500),
        }, step=step, prefix="train")

    # 在 Trainer 中使用:
    # trainer = Trainer(
    #     ...,
    #     callbacks=[ObservabilityCallback(tracker)],
    #     report_to=["tensorboard"],  # Trainer 内置 TensorBoard 支持
    # )

    tracker.save_metadata()
    print("实验追踪示例完成")

# 查看 TensorBoard:
# tensorboard --logdir=./logs/tensorboard
```

### 3.3 推理服务 Prometheus 指标

```python
# inference_metrics.py
"""推理服务 Prometheus 指标导出"""

import time
import psutil
import torch
from prometheus_client import Counter, Histogram, Gauge, Info, CollectorRegistry, generate_latest
from typing import Optional


class InferenceMetrics:
    """推理服务 Prometheus 指标"""

    def __init__(self, registry: Optional[CollectorRegistry] = None):
        self.registry = registry or CollectorRegistry()

        # 请求计数器
        self.requests_total = Counter(
            "inference_requests_total", "总请求数",
            ["model_version", "status"], registry=self.registry
        )

        # 延迟直方图
        self.latency_seconds = Histogram(
            "inference_latency_seconds", "推理延迟（秒）",
            ["model_version"], buckets=[0.01, 0.025, 0.05, 0.1, 0.25, 0.5, 1.0, 2.5, 5.0],
            registry=self.registry
        )

        # 输入长度直方图
        self.input_length = Histogram(
            "inference_input_length", "输入文本长度（字符）",
            ["model_version"], buckets=[10, 50, 100, 200, 500, 1000, 2000, 5000],
            registry=self.registry
        )

        # 置信度直方图
        self.confidence = Histogram(
            "inference_confidence", "预测置信度",
            ["model_version"], buckets=[0.1, 0.3, 0.5, 0.7, 0.8, 0.9, 0.95, 0.99],
            registry=self.registry
        )

        # 队列深度
        self.queue_depth = Gauge(
            "inference_queue_depth", "当前队列深度",
            ["model_version"], registry=self.registry
        )

        # GPU 显存
        self.gpu_memory_gb = Gauge(
            "inference_gpu_memory_gb", "GPU 显存使用量 (GB)",
            ["gpu_id"], registry=self.registry
        )

        # 模型信息
        self.model_info = Info(
            "inference_model", "模型信息", registry=self.registry
        )

    def record_request(self, model_version: str, status: str,
                       latency_ms: float, input_len: int,
                       confidence: float):
        """记录一次推理请求"""
        self.requests_total.labels(model_version=model_version, status=status).inc()
        self.latency_seconds.labels(model_version=model_version).observe(latency_ms / 1000)
        self.input_length.labels(model_version=model_version).observe(input_len)
        self.confidence.labels(model_version=model_version).observe(confidence)

    def update_queue_depth(self, model_version: str, depth: int):
        self.queue_depth.labels(model_version=model_version).set(depth)

    def update_gpu_memory(self):
        """更新 GPU 显存指标"""
        if torch.cuda.is_available():
            for i in range(torch.cuda.device_count()):
                allocated = torch.cuda.memory_allocated(i) / 1024**3
                self.gpu_memory_gb.labels(gpu_id=str(i)).set(allocated)

    def set_model_info(self, model_path: str, version: str):
        self.model_info.info({
            "model_path": model_path,
            "version": version,
        })


# ===== FastAPI 集成示例 =====
if __name__ == "__main__":
    from fastapi import FastAPI, Response

    app = FastAPI()
    metrics = InferenceMetrics()

    # 模拟模型信息
    metrics.set_model_info("./models/v2", "v2.0.1")

    @app.get("/metrics")
    async def get_metrics():
        """Prometheus 指标端点"""
        metrics.update_gpu_memory()
        metrics.update_queue_depth("v2.0.1", 5)  # 模拟队列深度
        return Response(content=generate_latest(metrics.registry),
                        media_type="text/plain")

    @app.post("/predict")
    async def predict(text: str = "测试"):
        start = time.time()
        # ... 模型推理 ...
        latency = (time.time() - start) * 1000

        metrics.record_request(
            model_version="v2.0.1",
            status="success",
            latency_ms=latency,
            input_len=len(text),
            confidence=0.92,
        )
        return {"label": "咨询", "confidence": 0.92}

    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

### 3.4 告警规则配置

```yaml
# prometheus_rules.yml
groups:
  - name: inference_alerts
    rules:
      # 规则 1: P95 延迟过高
      - alert: HighLatency
        expr: histogram_quantile(0.95, rate(inference_latency_seconds_bucket[5m])) > 0.2
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "推理 P95 延迟超过 200ms"
          description: "模型 {{ $labels.model_version }} 的 P95 延迟为 {{ $value }}s"

      # 规则 2: 错误率过高
      - alert: HighErrorRate
        expr: |
          rate(inference_requests_total{status="error"}[5m])
          / rate(inference_requests_total[5m]) > 0.01
        for: 3m
        labels:
          severity: critical
        annotations:
          summary: "推理错误率超过 1%"

      # 规则 3: GPU 显存即将耗尽
      - alert: GPUMemoryHigh
        expr: inference_gpu_memory_gb > 20  # 假设 24GB 卡
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "GPU 显存使用超过 20GB"

      # 规则 4: 输出分布异常
      - alert: OutputDistributionDrift
        expr: |
          abs(
            rate(inference_requests_total{status="success"}[1h])
            - rate(inference_requests_total{status="success"}[1h] offset 24h)
          ) > 0.3
        for: 15m
        labels:
          severity: warning
        annotations:
          summary: "模型输出分布可能发生漂移"

      # 规则 5: 服务不可用
      - alert: ServiceDown
        expr: up{job="inference"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "推理服务不可用"
```

## 4 项目总结

### 4.1 优点与缺点

| 工具 | 优点 | 缺点 |
|------|------|------|
| **TensorBoard** | 免费、PyTorch原生、可视化直观 | 不支持团队协作、无模型版本管理 |
| **WandB** | 实验对比方便、团队协作、自动记录 | 云服务有数据出境风险、私有化部署收费 |
| **MLflow** | 开源可私有化、端到端管理 | UI不如WandB直观、配置稍复杂 |
| **Prometheus+Grafana** | 业界标准、告警成熟、生态丰富 | 学习曲线陡、Histogram配置有坑 |

### 4.2 适用场景

| 场景 | 推荐方案 |
|------|---------|
| 个人/小团队实验追踪 | TensorBoard + JSON 元数据 |
| 团队协作实验管理 | WandB (Teams) 或 MLflow |
| 推理服务监控 | Prometheus + Grafana + 自定义指标 |
| 日志分析 | ELK (Elasticsearch + Logstash + Kibana) 或 Grafana Loki |

**不适用场景**：
- 离线/涉密环境（WandB 云端不可用 → 用 MLflow 私有化部署）

### 4.3 注意事项

1. **Histogram buckets 设计**：buckets 分布要覆盖 P50-P99 范围，否则分位值估算偏差大
2. **指标基数爆炸**：不要用高基数标签（如 trace_id），每个唯一标签值创建一个时间序列
3. **日志量与存储**：JSON 日志每条约 200 bytes，1 万 QPS × 86400 秒 ≈ 172GB/天，需设置保留策略

### 4.4 常见踩坑经验

| 故障现象 | 根因 | 解决方案 |
|---------|------|---------|
| Prometheus 内存爆掉 | 标签基数过高 | 移除 trace_id 等高基数标签 |
| TensorBoard 打不开 | 日志目录权限问题 | `tensorboard --logdir=./logs --bind_all` |
| WandB 上传失败 | 网络代理未配置 | 设置 `WANDB_BASE_URL` 或使用私有化部署 |

### 4.5 思考题

1. **初级**：在 `training_observer.py` 中增加 GPU 显存利用率记录——每 100 步记录一次 `torch.cuda.memory_allocated()` 和 `torch.cuda.max_memory_allocated()`。
2. **进阶**：设计一个**数据漂移检测**系统——对比训练集和线上数据的 Embedding 分布，当分布差异超过阈值时自动告警。（提示：用 Wasserstein 距离或 KL 散度衡量分布差异）
