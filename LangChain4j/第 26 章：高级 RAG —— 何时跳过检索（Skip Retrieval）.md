# 第 26 章：高级 RAG —— 何时跳过检索（Skip Retrieval）

## 1. 项目背景

并非每一次用户输入都值得触发 **向量检索**：「你好」「谢谢」「解释一下你能做什么」等 **元对话**若也强行 RAG，既 **浪费 embed/检索成本**，也可能 **引入随机文档噪声** 污染回答。**Skip Retrieval** 的模式是：在 **路由/压缩之后、进入向量库之前**，做一次 **轻量判断** —— **本轮是否需要私域知识**；若不需要，则 **直接进入纯聊天模型**。

示例：

- `langchain4j-examples/rag-examples/src/main/java/_3_advanced/_06_Advanced_RAG_Skip_Retrieval_Example.java`

本章与第 22 章 **路由**、第 21 章 **压缩** 组成「**检索前处理三件套**」：**变形 / 分型 / 是否检索**。

## 2. 项目设计：大师与小白的对话

**小白**：这和「查询路由」有什么不同？

**大师**：**路由**是 **多库选一（或多）**；**跳过**是 **二元决策：查或不查**。可以 **先判断跳过**，若否再 **进入路由**。

**小白**：用规则还是用小分类器？

**大师**：**冷启动**常用规则（关键词、意图标签）；数据多了再上 **小模型**或 **轻量二分类**。

**小白**：跳过会不会漏掉「表面闲聊、实则含业务」的问题？

**大师**：会，这是 **核心风险**。要靠 **日志回放**、**误跳过标注**、以及 **显式澄清**（「是否与 X 政策有关？」）。

**小白**：用户需要知道「本轮没查知识库」吗？

**大师**：看产品：**显式**增强可解释性；**隐式**更简洁。B 端可 **在 debug 面板展示**。

**小白**：成本角度？

**大师**：应记录 **`saved_retrieval_calls`** 与 **`false_negative` 投诉率** 以验证策略 ROI。

**小白**：如何与工具调用协同？

**大师**：某些问题 **不需要文档**但需要 **查询订单 API**——跳过 RAG **不意味着**跳过工具；要 **分层决策**。

**小白**：国际化？

**大师**：**问候语模板**因语言而异；规则需 **多语言覆盖**或 **语言检测前置**。

**小白**：如何把误判样本喂回训练？

**大师**：沉淀 **`skip_decision` 日志** + **人工标注** + **周期性重训/改规则**。

## 3. 项目实战：主代码片段

> **场景入戏**：每条闲聊都强行 RAG，像 **问路去洗手间前先查一遍世界地图**——又慢又可能 **指到北极**。**Skip retrieval** 是门口保安：**纯唠嗑放行，带工牌进档案室**。

打开 [`_06_Advanced_RAG_Skip_Retrieval_Example.java`](../../langchain4j-examples/rag-examples/src/main/java/_3_advanced/_06_Advanced_RAG_Skip_Retrieval_Example.java)：

1. **书签**标出 **skip 判断**与 **特征来源**（正则 / 小模型 / 意图分类）。  
2. **剧本杀**：写 **10 句中文**——5 句应 skip、5 句禁止 skip，互相 **盲测**命中率。  
3. **产品按钮**：误判时，用户能否 **一键「强制查知识库」**？——没有这钮，运营 **电话会被打爆**。

#### 指标玩笑

给 Grafana 起个 panel 名：`skip_rate_vs_happiness`——**skip 率**与 **NPS** 的相关性，可能让你 **笑不出来**。

## 4. 项目总结

### 优点

- **省钱省延迟**，改善 **闲聊体验**。  
- 降低 **无关文档**注入风险。

### 缺点

- **误判**会损害 **专业场景可信度**。  
- **策略迭代**需要 **运营与分析**配合。

### 适用场景

- 「通用助理 + 垂直知识库」**混合入口**。  
- **高成本向量库**需要 **精确触发**时。

### 注意事项

- **监控跳过率**突增/突降。  
- **与营销话术**区分，避免 **跳过过多业务问题**。

### 常见踩坑

1. **仅靠关键词**误判 **省略主语**的专业问题。  
2. **无观测**导致 **错误永不暴露**。  
3. **与第 27 章多路检索**逻辑打架（跳过过早）。
