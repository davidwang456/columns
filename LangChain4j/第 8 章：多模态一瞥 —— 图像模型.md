# 第 8 章：多模态一瞥 —— 图像模型

## 1. 项目背景

### 业务场景（拟真）

市场部希望在运营后台「**一句话出活动配图**」，与现有 **文本客服** 共用同一 Java 后端团队。图像能力与 **Chat** 的差异在于：**计费单位**、**输出形态（二进制/URL）**、**延迟**、**合规（版权、肖像、敏感内容）** 均需单独清单。

### 痛点放大

若 **与 Chat 混同一套监控与预算**：**成本**上图像任务可能 **一夜打穿配额**；**运维**上 **临时 URL** 被当前端永久资源导致 **403/过期**；**合规**上未过审素材 **直接对公众展示** 引发法务风险。LangChain4j 将图像能力抽象为 **`ImageModel`**；教程 `_02_OpenAiImageModelExamples.java` 是 JVM 团队 **最小连通性实验**。

## 2. 项目设计：小胖、小白与大师的对话

**小胖**：文生图是不是跟「点外卖配图」一样，等得久一点？

**小白**：`ImageModel` 也能用 **BOM** 对齐吗？`response.content().url()` 谁负责下载？

**大师**：BOM 仍管 **版本**；注意 **artifact 勿重复引**。URL **由你的系统下载**——考虑 **时效、内网拉取、病毒扫描、CDN 费用**。**技术映射**：**ImageModel = 生成契约，资产入库 = 你自己的管道**。

**小胖**：生成的图能直接上广告吗？

**小白**：和 Chat **共用 Key** 吗？**CI 每次都跑真生成** 太贵怎么办？

**大师**：**条款与品牌合规** 先于技术；**账户常共用**但 **计费 SKU 不同**——账单要 **分行项目**。CI 用 **Mock `ImageModel`**、**nightly** 真调或 **低成本沙箱**。**技术映射**：**合规与成本 = 业务策略，不是模型返回值**。

**小胖**：失败重试呢？

**大师**：图像易 **429/长任务**——**退避 + 队列**，勿在 **用户请求线程** 长时间阻塞同步。**技术映射**：**异步任务 + 配额治理**。

## 3. 项目实战

### 环境准备

- [`_02_OpenAiImageModelExamples.java`](../../langchain4j-examples/tutorials/src/main/java/_02_OpenAiImageModelExamples.java)。

### 分步实现

```java
ImageModel model = OpenAiImageModel.builder()
        .apiKey(ApiKeys.OPENAI_API_KEY)
        .modelName(DALL_E_3)
        .build();

Response<Image> response = model.generate(
        "Swiss software developers with cheese fondue, a parrot and a cup of coffee");

System.out.println(response.content().url());
```

| 步骤 | 目标 | 操作 |
|------|------|------|
| 1 | 随机性 | 同 prompt **双击运行**，观察 URL 是否变化 |
| 2 | 下载与哈希 | `HttpClient` 拉取并 **SHA-256**，再决定是否入库 |
| 3 | 元数据 | 查 `Response` 是否含 **revised_prompt** 等（视版本） |

**可能遇到的坑**：**临时 URL** 当永久链；**未设预算**；**无人工审核** 对公众发图。

### 测试验证

- 断言 **非空 URL 或字节**；stub **内容策略拒绝**；不把大图塞进 JUnit 报告。

## 4. 项目总结

### 优点与缺点（与同类做法对比）

| 维度 | LangChain4j `ImageModel` | 直连 HTTP | 专用 Python 服务 |
|------|--------------------------|-----------|-------------------|
| 依赖治理 | 与 BOM 一致 | 自管 | 双栈 |
| 单测 | 易 mock 接口 | 中 | 中 |
| 合规责任 | 仍在业务 | 仍在业务 | 仍在业务 |
| 典型缺点 | 延迟/成本 | 重复代码 | 联调成本 |

### 适用场景

- 营销草稿、配图探索、内部设计协作；与文本 **资源隔离** 部署。

### 不适用场景

- **高 QPS 同步阻塞** 热路径；**强实时视频**——非本章范围。

### 注意事项

- **NSFW/政策**与地区；**留存** 病毒扫描与哈希去重。

### 常见踩坑经验（生产向根因）

1. **临时 URL** 当前端永久资源。  
2. **未设预算告警**，图像打穿配额。  
3. **无审核** 对公众自动发图。

### 进阶思考题

1. 如何将 **图像元数据** 与 **文本审计日志** 关联同一 `traceId`？  
2. **多模态聊天**（图+文）与 **独立 ImageModel** 在 **消息模型** 上的分界？（提示：关注 `data.message` 包演进。）

### 推广计划提示（多部门）

| 角色 | 建议阅读顺序 | 协作要点 |
|------|----------------|----------|
| **开发** | 本章 + 存储管道 | **异步任务** + **对象存储 ACL** |
