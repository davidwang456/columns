# 第 8 章：多模态一瞥 —— 图像模型

## 1. 项目背景

「大模型」不仅包含文本聊天，也包含 **图像生成、图像编辑、视觉理解** 等能力。LangChain4j 将「面向某一种输出模态的客户端」抽象为独立接口，例如 **`ImageModel`**。与 `ChatModel` 的显著差异在于：**计费单位**可能不是 chat token；**输出**是二进制或 URL；**延迟**往往更长；**合规**（版权、肖像、敏感内容）更敏感。

教程 `_02_OpenAiImageModelExamples.java`（`langchain4j-examples/tutorials/src/main/java/_02_OpenAiImageModelExamples.java`）展示 `OpenAiImageModel` + `DALL_E_3` + `Response<Image>`，是 JVM 团队在混排「**文生图**」与「**文本客服**」前的最小连通性实验。本章目标不是成为视觉算法专家，而是建立 **配置、调用、存储、审核** 的工程清单。

## 2. 项目设计：大师与小白的对话

**小白**：图像模型也能用 BOM 统一管理版本吗？

**大师**：可以尝试，但要注意 **artifact 仍分模块**：`langchain4j-open-ai` 里同时有聊天与图像 Builder；**依赖树**里别引重复。

**小白**：`response.content().url()` 拿到 URL 后谁负责下载？

**大师**：**你的系统**。要考虑 **URL 时效**、**内网拉取是否允许**、**病毒扫描**与 **CDN 回源费用**。

**小白**：我们能直接把生成的图用在广告里吗？

**大师**：**先看服务商条款与品牌合规**；再看你们内部 **素材审批**。技术只解决「生成」，不解决「可商用」。

**小白**：和 Chat 共用 API Key 吗？

**大师**：常共用同一账户，但 **配额/计费维度** 可能不同——运维看账单要**分产品行**。

**小白**：失败重试怎么做？

**大师**：图像生成长任务更易 **429/配额**；要 **退避** 与 **队列**，不要同步阻塞在用户请求线程里太久。

**小白**：测试环境每次跑 CI 都生成太贵怎么办？

**大师**：**Mock `ImageModel`**、或 **只在 nightly** 调真接口、或使用 **低成本/沙箱模型**。

**小白**：为什么示例里用 `Response<Image>`？

**大师**：与文本侧类似，保留 **元数据扩展点**（如 revised_prompt、usage）；具体字段以版本 Javadoc 为准。

**小白**：以后要接入「图像+文本多模态聊天」怎么办？

**大师**：会涉及 **不同消息类型** 与 **厂商多模态 API**——保持关注 `data.message` 包里与图像相关的消息构造。

## 3. 项目实战：主代码片段

> **场景入戏**：文本 `ChatModel` 像**发短信**；`ImageModel` 像**下单洗照片**——等得更久、钱按**张/像素**算，还可能被「内容政策」**整单拒掉**。**别在热路径同步阻塞**印 T 恤。

```java
ImageModel model = OpenAiImageModel.builder()
        .apiKey(ApiKeys.OPENAI_API_KEY)
        .modelName(DALL_E_3)
        .build();

Response<Image> response = model.generate(
        "Swiss software developers with cheese fondue, a parrot and a cup of coffee");

System.out.println(response.content().url());
```

**仓库锚点**：[`_02_OpenAiImageModelExamples.java`](../../langchain4j-examples/tutorials/src/main/java/_02_OpenAiImageModelExamples.java)。

#### 闯关任务

| 难度 | 动手 | 趣味点 |
|------|------|--------|
| ★ | 把 prompt 改成你们公司 **标语风格** + 品牌主色（文字描述） | 体会「**美术总监**」其实是**调参** |
| ★★ | 连续 **双击运行** 同 prompt，观察 URL **是否变化** | **随机性**在图像侧同样存在 |
| ★★★ | 用 `HttpClient` 把 URL **拉下来算 SHA-256**，再决定 **是否入库** | **临时链接触发防盗链 403**——预演生产坑 |

#### 挖深一层

- **计费维度**：图像与 Chat **往往不同 SKU**——财务看账单要对行项目。  
- **`Response<Image>`**：留心是否还有 **revised_prompt** 等元数据（视版本），可用于 **审计**。  
- **合规**：生成的人脸、商标、版权素材——**上线前过法务**，技术实现只是最后一米。

## 4. 项目总结

### 优点

- 与 LangChain4j **同一套依赖治理** 引入图像能力。  
- `ImageModel` 抽象利于 **单元测试替身**。

### 缺点

- **延迟与成本**不适合同步阻塞在高 QPS 路径。  
- **合规与审核**完全在业务侧。

### 适用场景

- 营销草稿、配图探索、内部设计协作。  
- 与文本链路 **独立部署** 的场景（资源隔离）。

### 注意事项

- **NSFW / 政策**：地区与账号等级会影响可用性。  
- **留存**：下载后病毒扫描与哈希去重。

### 常见踩坑

1. 把 **临时 URL** 当永久链接挂在前端。  
2. **未设预算告警**，图像任务把月度配额一夜打穿。  
3. 在 **无人工审核** 下对公众自动发图。
