# 第 31 章：文档加载与解析链路

## 1. 项目背景

RAG 上限常卡在：**原始文件读不出来**。PDF 扫描件、复杂表格、加密 Office、站内网页登录态，都需要 **document-loader**（从哪里拉字节）与 **document-parser**（如何变成 `Document`）协同。Aggregator 在 `document-loaders/` 与 `document-parsers/` 下提供 **S3、Azure Blob、Playwright、PdfBox、Tika** 等模块组合。

## 2. 项目设计：大师与小白的对话

**小白**：为何不用 `Files.readString`？

**大师**：**编码**、**二进制**、**分页**、**附件**都需要专门解析；否则会 **静默丢内容**。

**小白**：网页抓取和 API 调用的区别？

**大师**：**API** 稳定、合规清晰；**抓取**要处理 **JS 渲染、登录、robots**。

**小白**：大文件怎么办？

**大师**：**流式**读取 + **分块解析**；批任务设 **超时**与 **最大体积**。

**小白**：OCR 成本？

**大师**：**按页计费** + **CPU**；要在 **业务价值**与 **成本**间取舍。

**小白**：病毒与宏？

**大师**：**杀毒扫描**、禁用宏执行、沙箱解析。

## 3. 项目实战：主代码片段

> **场景入戏**：**Loader/Parser** 是 **食材预处理**：给模型喂 **带壳核桃**，再贵的 **米其林厨师**也只能拍出 **「香脆不明物体」**——**OCR/表格**翻车时，向量库里只有 **噪音 embeddings**。

从 [`Naive_RAG_Example.java`](../../langchain4j-examples/rag-examples/src/main/java/_2_naive/Naive_RAG_Example.java) 回顾：

```java
Document document = loadDocument(toPath(documentPath), new TextDocumentParser());
```

**闯关**：  
1. 数 **`loadDocument` 到 `embed`** 之间 **经过哪些类型**（`Document` → `TextSegment` → …）？  
2. 若换成 **PDF**，应替换为 **PdfBox/Tika** 等 parser（查 `document-parsers/*` 模块 README）。在 `pom.xml` 增加 **最少依赖** 列表供评审——并约定：**入库前 min 字符数**门禁，防下面「延伸案例」里的 **silent failure**。

### 延伸案例（情景演练）：扫描版 PDF 与「silent failure」

某制造业把 **质量体系 SOP** 批量入库做 RAG。首批 **200 份 PDF** 中，**60%** 为扫描件。团队最初用 **纯文本拉取**，解析结果 **近乎空串**，却仍进入 **嵌入流水线**——线上表现为：**能问答，但全是无依据胡编**。排查路径：**(1)** 随机抽样源文件，发现 **每页为图片**；**(2)** 引入 **`langchain4j-document-parser-apache-tika`** 及 **OCR 步骤**（或外包给具备 OCR 的云解析），并在 **ingest 质量门槛**中增加 **`minCharsPerPage`**；**(3)** 对不合格文档进入 **人工队列**。

第二个波折：**中英混排表格** 在解析后列错位，导致检索到 **上下行拼接的错误「事实」**。缓解策略：**表格密集文档**走 **专用解析管道**（如先转 HTML/结构化导出），或在入库阶段打 **`layoutConfidence` metadata**，检索阶段 **降权**低置信片段。该案例说明：**Loader/Parser 是比向量模型更靠上的「天花板」**——建议在培训中配一张 **「解析前后字符数对比」** 折线图，作为上线门禁。

## 4. 项目总结

### 优点

- **模块化**，按需引用。**缺点**：**依赖树**膨胀。

### 适用场景

- 知识库 **批量入湖**、合规文档 **归档检索**。

### 注意事项

- **版权**与 **许可**。  
- **解析缓存**失效策略。

### 常见踩坑

1. **解析结果空**未发现，直接索引。  
2. **HTML** 含导航栏噪声。  
3. **编码错**导致中文乱码索引。
