# LangChain4j 由浅入深（38 章）

本目录为配套**独立 Markdown 章节**，与本地仓库 [`langchain4j`](../../langchain4j) 及 [`langchain4j-examples`](../../langchain4j-examples) 中的示例代码对应。每章建议阅读时间 **45～90 分钟**（含动手）；**深度样章**按 [template.md](template.md) 建议篇幅控制在 **3000～5000 字**。

## 章节体例（与 template.md 对齐）

撰写与阅读时，请按下列结构理解每章（细节字数见 [template.md](template.md)）：

| 章节 | 内容要点 |
|------|----------|
| **1. 项目背景** | 业务场景（真实或拟真）、**痛点放大**（性能/一致性/可维护性）；可配简单流程图。 |
| **2. 项目设计** | **三角色剧本式对话**：**小胖**（生活化比喻）、**小白**（原理/边界/风险）、**大师**（选型与约束）。**话轮结构**：小胖开球 → 小白追问（2～3 轮）→ 大师解答并引出子话题，循环 **2～3 次**；关键处可插入**技术映射**（生活比方 → 技术术语）。 |
| **3. 项目实战** | 环境准备（依赖/BOM/版本/最小配置）；**分步**实现（步骤目标、带注释代码、运行结果文字描述、坑与解法）；**测试验证**（curl/JUnit 等）；**完整代码清单**指向仓库路径或附录。 |
| **4. 项目总结** | **优点与缺点**（可与同类技术对比，**表格**更佳）；**适用 / 不适用**场景；**注意事项**（配置、版本、安全）；**生产踩坑**（典型故障与根因）；**进阶思考题**（2 道，答案可在下一章或附录）；**跨部门推广**（开发/运维/测试阅读顺序与协作）。 |

文末可保留 **测试 / 运维检查清单**（与 `template.md` 中的「推广计划」呼应）。详细合并逻辑见 [CHAPTER_MERGE_LOG.md](CHAPTER_MERGE_LOG.md)。

| 级别 | 章节范围 | 内容侧重 | 面向角色 |
|------|----------|----------|----------|
| **基础篇** | 第 1～13 章 | 核心概念、单机、简单 API、初级实战 | 新人开发、测试 |
