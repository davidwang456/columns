# 第 17 章：结构化输出与业务类型映射

## 1. 项目背景

下游系统要 `OrderDTO`、`RiskLabel`、JSON 数组——不是散文。LangChain4j 在 `service.output` 包提供 `StringListOutputParser`、`PojoOutputParser` 等，把模型文本拉回 Java 类型系统。失败情形包括：漏字段、markdown 围栏、locale 数字格式。

## 2. 项目设计：小胖、小白与大师的对话

**小胖**：模型返回文本我用 Jackson 解析不就完了？为啥还要搞个 Parser？

**小白**：跟厂商 response_format: json 什么关系？解析失败要重试吗？enum 映射不上咋办？

**大师**：直接 Jackson 确实可行，但你要在每个调用点写一遍噪声处理、重试、错误修复——`PojoOutputParser` 把这些汇聚到一处。厂商原生 JSON mode **优先用它约束输出**，解析器再兜底处理部分响应。失败走有限次重试，超阈值降级人工。enum 必须提供 UNKNOWN 兜底。**技术映射**：**解析层 = 契约恢复层，不是安全授权层**。

### 步骤 1：解析为字符串列表

```java
import dev.langchain4j.model.output.StringListOutputParser;
import dev.langchain4j.model.chat.ChatModel;
import dev.langchain4j.model.openai.OpenAiChatModel;
import static dev.langchain4j.model.openai.OpenAiChatModelName.GPT_4_O_MINI;

public class ParserDemo {

    public static void main(String[] args) {
        ChatModel model = OpenAiChatModel.builder()
                .apiKey(System.getenv("OPENAI_API_KEY"))
                .modelName(GPT_4_O_MINI)
                .build();

        // StringListOutputParser 自动去掉编号、分割行
        StringListOutputParser parser = new StringListOutputParser();

        String prompt = "List three benefits of using Java for backend development.";
        String response = model.chat(prompt);

        System.out.println("Raw response:");
        System.out.println(response);

        try {
            java.util.List<String> list = parser.parse(response);
            System.out.println("\nParsed list:");
            list.forEach(item -> System.out.println(" - " + item));
        } catch (Exception e) {
            System.out.println("Parse failed: " + e.getMessage());
        }
    }
}
```

**预期输出**：
```
Raw response:
1. Strong type system and compile-time safety
2. Mature ecosystem with Spring, Hibernate, etc.
3. Excellent performance with JIT compilation

Parsed list:
 - Strong type system and compile-time safety
 - Mature ecosystem with Spring, Hibernate, etc.
 - Excellent performance with JIT compilation
```

### 步骤 2：解析为 POJO

```java
import dev.langchain4j.service.AiServices;
import dev.langchain4j.service.SystemMessage;
import dev.langchain4j.service.UserMessage;
import dev.langchain4j.service.output.PojoOutputParser;
import dev.langchain4j.service.output.OutputParsingException;

class Person {
    String name;
    int age;
    String occupation;
    
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + ", occupation='" + occupation + "'}";
    }
}

interface PersonExtractor {
    @UserMessage("Extract person information from: {{it}}")
    Person extractPerson(String text);
    
    @SystemMessage("Always respond in valid JSON without markdown formatting")
    @UserMessage("Extract person information from: {{it}}")
    Person extractPersonJson(String text);
}

public class PojoParserDemo {

    public static void main(String[] args) {
        ChatModel model = OpenAiChatModel.builder()
                .apiKey(System.getenv("OPENAI_API_KEY"))
                .modelName(GPT_4_O_MINI)
                .build();

        PersonExtractor extractor = AiServices.builder(PersonExtractor.class)
                .chatModel(model)
                .build();

        String text = "John is a 28-year-old software engineer from New York.";

        try {
            Person person = extractor.extractPerson(text);
            System.out.println("Extracted: " + person);
        } catch (OutputParsingException e) {
            System.out.println("Parse failed: " + e.getMessage());
            // 处理逻辑：重试、修复提示、降级人工
        }
    }
}
```

**预期输出**：
```
Extracted: Person{name='John', age=28, occupation='software engineer'}
```

### 步骤 4：有限重试循环

```java
int maxRetries = 3;
for (int i = 0; i < maxRetries; i++) {
    try {
        Person person = extractor.extractPerson(text);
        System.out.println("Success: " + person);
        break;
    } catch (OutputParsingException e) {
        System.out.println("Attempt " + (i + 1) + " failed, retrying...");
        if (i == maxRetries - 1) {
            System.out.println("All retries exhausted. Sending to manual queue.");
        }
    }
}
```

| 难度 | 动手 | 过关标准 |

|----|------|------|
| 未处理 markdown 代码块 | 模型给 \`\`\`json 包裹 | 提示要纯 JSON |

## 4. 项目总结

### 优点与缺点

| 维度 | PojoOutputParser 等 | 裸 Jackson | 仅 prompt 要 JSON |
|------|--------------------|-----------|-----------------|
| 集中治理 | 高 | 低 | 低 |
| 脆弱性 | 中（可监控） | 中 | 高 |
| 与 JSON mode | 可组合 | 可组合 | 视厂商 |

### 适用 / 不适用场景

**适用**：表单预填、风控标签、抽取式 NLU、对接工单/CRM。

**不适用**：超大嵌套一次生成（应分步或工具）。

### 常见踩坑

1. 未处理 markdown 代码块
2. 盲目重试放大成本
3. 解析器当安全边界
