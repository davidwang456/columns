# 第23章：概率统计与数据可视化实战

---

## 1. 项目背景

某数据培训机构正在用 Manim 制作"数据分析思维"系列动画课程。讲师阿杰负责"月度销售数据分析"一集，需要用动画展示一份真实销售数据的月度趋势、同比变化、异常点标注和结论卡片。

阿杰拿到了 Excel 里的销售数据：12 个月的销售额、同比增长率、以及 3 个月的异常数据（促销活动导致激增）。他原计划用第 20 章学的 `Axes.plot` 画折线图，但很快发现需求远超基本操作：

1. **需要混合图表**：折线图（销售额趋势）和柱状图（同比增长率）共用一个 x 轴但 y 轴不同——需要"双 Y 轴"布局。
2. **需要数据读取**：Excel 数据需要用 Python 读取并转换为 Manim 可消费的格式。
3. **需要异常标注**：3 个月的异常点需要在折线图上用不同颜色标注，并在下方弹出解释卡片。
4. **需要动画叙事**：不是一次性画出整条折线——而是从左到右逐步展开、在关键点停顿解释、最后汇总结论。

这个痛点指向了 Manim 的数据可视化实战能力：`BarChart`、`LineGraph`、`Axes` 的多图表组合、CSV/JSON 数据读入和"数据故事"的叙事节奏。

## 2. 剧本式交锋对话

> **场景**：阿杰的屏幕上，左边的 CSV 文件和右边的 BarChart 代码像两个世界。

**小胖**（拆着一包薯条）：

"阿杰，你这屏幕左边是 Excel 表格，右边是 Manim 代码——你是在做动画还是在做数据 ETL？数据不能直接喂给 Manim 吗？"

**阿杰**（苦笑）：

"不能。Manim 不认识 Excel……也不认识 CSV。我得手动把 12 个数字一个个敲进代码里。12 个月还好，如果是 365 天呢？"

**小白**（打开一个 Python 脚本）：

"这用 Python 原生能力就够了。两行代码把 CSV 读成 Python list，然后传给 `BarChart`：

```python
import csv
with open("data/sales.csv") as f:
    reader = csv.DictReader(f)
    values = [float(row["revenue"]) for row in reader]

chart = BarChart(values=values, ...)
```

"Manim 不管数据从哪来——它只要一个 list。你用 `csv` 读、`json` 读、甚至用 `pandas` 读完转成 list，Manim 都接受。"

**大师**（补充）：

"阿杰的问题更深一层——他需要的是**双 Y 轴**：左边 Y 轴标销售额（范围 0-500 万），右边 Y 轴标增长率（范围 -20% ~ +50%）。这是 `Axes` 做不到的——`Axes` 只有一个 y_range。

解决方案是用两个**独立的 `Axes`**，x_range 对齐，y_range 各自独立。使它们共享同一个画面位置，x 轴重叠但 y 轴错开。"

> **技术映射**：通过创建 `axes1 = Axes(y_range=[0, 500], ...)` 和 `axes2 = Axes(y_range=[-20, 50], ...)` 并让两者 `x_length` 相同来对齐 x 轴。`axes2.shift(RIGHT * length_diff)` 将第二个 y 轴放在右侧。

**小胖**（嚼着薯条）：

"那 `BarChart` 和 `LineChart` 呢？我看它们名字像开箱即用的，但实际好像有很多坑。"

**小白**：

"对。`BarChart` 和 `Axes.bar_chart()` 是两个不同的东西。前者是独立组件，后者是 Axes 的方法。推荐用 `Axes.bar_chart()`——因为它和轴的坐标系自动绑定，更好维护。

`BarChart(values, bar_names=...)` 创建独立的柱状图，适合不需要显式轴的场景（比如对比几个离散分类）。而 `axes.plot_bar_chart(values, ...)` 返回绑定在 axes 上的柱状条，能和其他曲线共享同一个坐标空间。"

**大师**：

"最后——数据故事的结构。和算法动画不同，数据动画的叙事节奏是：**趋势线型 → 关键转折 → 异常原因 → 总结预判**。技术实现上对应四个 play 段落，每个段落之间插入 `wait(1)` 让观众消化。"

**小胖**（薯条快吃完了）：

"阿杰，你刚说 CSV 直接读——但真实数据哪有那么干净？缺字段的、负销售额的、乱码的，你怎么处理？"

**小白**：

"数据清洗是数据可视化的一半工作量。Manim 不会帮你做清洗——得在 Python 层处理。我推荐一个数据加载流水线：

```python
def clean_sales_data(rows):
    cleaned = []
    for row in rows:
        try:
            rev = float(row.get('revenue', 0))
            if rev < 0:
                rev = 0  # 负值归零或标记
            cleaned.append(rev)
        except (ValueError, TypeError):
            cleaned.append(0)  # 脏数据用 0 占位
    return cleaned
```

"这里的关键是——清洗逻辑和渲染逻辑分离。不要在图表的 `always_redraw` 中做清洗，那是性能灾难。"

**大师**（补充）：

"另一个实战问题——数据量太大时（比如 365 天的日数据），在一条折线上显示 365 个点太密。这时需要**降采样**或**聚合**。例如按月聚合日均值，或者用 `ax.plot(lambda x: monthly_avg[int(x)], x_range=[1,12])` 每单位一个采样点。"

> **技术映射**：数据清洗是 Python 原生能力，在 Manim 的 `construct()` 之外完成。清洗后的结果传给 `BarChart`/`plot_line_graph`。

### 3.2 分步实现

> **本章实战目标**：制作一个"月度销售数据分析"动画，包含双 Y 轴混合图表和异常标注。

#### 步骤一：双 Y 轴折线+柱状混合图

**步骤目标**：构建双 Y 轴布局，左轴销售额（折线），右轴增长率（柱状）。

```python
# scenes/chapter23_dual_axis.py
from manim import *

class DualAxisDemo(Scene):
    def construct(self):
        # 左 Y 轴：销售额
        left_axes = Axes(
            x_range=[0, 13, 1], y_range=[0, 800, 100],
            x_length=10, y_length=5.5,
            axis_config={"color": BLUE, "include_tip": True},
            y_axis_config={"label_direction": UP},
        )
        left_axes.to_edge(LEFT, buff=0.8)

        left_label = Text("销售额(万元)", font_size=20, color=BLUE)
        left_label.next_to(left_axes.y_axis.get_top(), UP, buff=0.1)

        # 右 Y 轴：同比增长率
        right_axes = Axes(
            x_range=[0, 13, 1], y_range=[-20, 60, 10],
            x_length=10, y_length=5.5,
            axis_config={"color": GREEN, "include_tip": True,
                         "tip_compression_factor": 0.05},
            y_axis_config={"label_direction": UP},
        )
        right_axes.move_to(left_axes.get_origin())

        right_label = Text("增长率(%)", font_size=20, color=GREEN)
        right_label.next_to(right_axes.y_axis.get_top(), UP, buff=0.1)

        title = Text("月度销售趋势分析", font_size=32, color=WHITE)
        title.to_edge(UP, buff=0.3)

        self.play(Write(title), Create(left_axes), Write(left_label),
                  Create(right_axes), Write(right_label), run_time=2)

        # 折线：销售额
        months = list(range(1, 13))
        revenues = [320, 280, 410, 350, 380, 520, 490, 440, 510, 580, 750, 680]

        line = left_axes.plot_line_graph(
            x_values=months, y_values=revenues,
            line_color=BLUE, add_vertex_dots=False,
        )

        # 柱状：增长率
        growth = [5.2, -12.5, 46.4, -14.6, 8.6, 36.8,
                  -5.8, -10.2, 15.9, 13.7, 29.3, -9.3]
        bars = right_axes.plot_bar_chart(
            x_values=months, y_values=growth,
            bar_colors=[GREEN if g > 0 else RED for g in growth],
            bar_fill_opacity=0.6,
        )

        self.play(Create(line), Create(bars), run_time=3)
        self.wait(1)

        self.play(FadeOut(VGroup(title, left_axes, right_axes,
                                  left_label, right_label, line, bars)), run_time=1.5)
```

#### 步骤三：完整数据动画——标注异常点

**步骤目标**：在折线图上标注 3 个异常月（3月、6月、11月），并用文字卡片解释原因。

```python
class SalesAnalysisFull(Scene):
    def construct(self):
        months_list = list(range(1, 13))
        revenues = [320, 280, 410, 350, 380, 520, 490, 440, 510, 580, 750, 680]
        growths = [5.2, -12.5, 46.4, -14.6, 8.6, 36.8,
                   -5.8, -10.2, 15.9, 13.7, 29.3, -9.3]

        axes = Axes(
            x_range=[0, 13, 1], y_range=[0, 800, 100],
            x_length=10, y_length=5.5,
            axis_config={"color": BLUE, "include_tip": True},
        )
        axes.to_edge(LEFT, buff=0.8).shift(DOWN * 0.3)

        title = Text("月度销售数据（万元）", font_size=30, color=BLUE).to_edge(UP, buff=0.3)
        self.play(Write(title), Create(axes), run_time=2)

        # 折线（从左到右逐步展开）
        line = axes.plot_line_graph(
            x_values=months_list, y_values=revenues,
            line_color=BLUE, add_vertex_dots=True,
            vertex_dot_radius=0.06,
        )
        self.play(Create(line), run_time=3)
        self.wait(0.5)

        # 标注异常月：3月、6月、11月
        anomalies = [
            (3, 410, "3月 · 春季促销"),
            (6, 520, "6月 · 618大促"),
            (11, 750, "11月 · 双11爆发"),
        ]

        for month, val, desc in anomalies:
            # 高亮点
            dot = Dot(axes.coords_to_point(month, val),
                      color=RED, radius=0.12)
            self.play(GrowFromCenter(dot), run_time=0.5)

            # 说明卡片
            card = RoundedRectangle(
                width=3.5, height=0.8, corner_radius=0.1,
                color=RED, fill_opacity=0.15, stroke_width=1.5,
            )
            card_text = Text(desc, font_size=22, color=WHITE)
            card_text.move_to(card.get_center())
            note = VGroup(card, card_text)

            # 放置卡片在点的上方偏右
            note.next_to(dot, UP + RIGHT * 0.5, buff=0.2)
            self.play(FadeIn(note, scale=0.9, shift=UP * 0.2), run_time=1)
            self.wait(1)
            self.play(FadeOut(note), run_time=0.5)

        # 结论
        conclusion = Text(
            "全年趋势向上，促销月贡献峰值",
            font_size=28, color=GREEN,
        )
        conclusion.next_to(axes, DOWN, buff=0.6)
        self.play(Write(conclusion), run_time=1.5)
        self.wait(2)

        self.play(FadeOut(VGroup(title, axes, line, conclusion)), run_time=1.5)
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 双 Y 轴 | 销售额+增长率同框对比 | 两个 Axes 的手工对齐需谨慎 |
| 数据导入 | Python 原生 csv/json/pandas 灵活 | Manim 不内置数据源抽象 |
| 异常标注 | 高亮点 + 弹出卡片叙事性强 | 卡片位置需要手动计算避让 |
| 动画叙事 | 从左到右展开 + 停顿解释的节奏感 | 每个停顿增加总时长 |
| 柱状+折线 | `plot_bar_chart` + `plot_line_graph` 组合 | 两种图形 API 风格不一致 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 销售/财务报告 | 月度/季度趋势 + 同比分析 |
| 运营指标 | 日活、转化率、留存率趋势 |
| 教育数据 | 学生成绩分布、及格率统计 |
| 科学实验 | 实验数据的可视化 |
| 社交媒体 | 粉丝增长、互动率走势 |

**不适用场景**：需要交互式筛选和钻取的大屏 BI（应用 Tableau/PowerBI）、百万级数据点的散点图（应用 Datashader/WebGL）。

### 注意事项

1. **`plot_line_graph` 的 `x_values` 必须为递增序列**：如果 x 值乱序，折线会画出奇怪的"回环"。
2. **双 Y 轴中两个 Axes 的 `x_length` 必须完全一致**：否则 x 轴刻度不对齐。
3. **`plot_bar_chart` 的柱子宽度由 x_range 和间距自动计算**：不能像 SVG/CSS 那样手动调宽。如果柱子太密或太疏，调整 `x_range` 的步长。

### 常见踩坑经验

**故障一：`plot_line_graph` 的折线在 x 轴边界外溢**

根因：`x_values` 超出了 `axes.x_range` 的范围。

解决：确保所有数据点落在 x_range 内，或设置 `x_range` 略大于数据范围（如数据 1-12，设 `x_range=[0, 13]`）。

**故障二：柱状条颜色传列表后渲染全部黑色**

根因：`bar_colors` 必须是 `ManimColor` 对象列表，不是 hex 字符串列表。

解决：`[ManimColor("#00FF00") if g>0 else ManimColor("#FF0000") for g in growth]`。

**故障三：动画中双 Y 轴的标签重叠**

根因：两个 y 轴共用同一垂直空间，标签自动放置可能冲突。

解决：将右侧 y 轴的 `label_direction` 设为 `LEFT`（标签放在轴线左侧），或将右侧轴整体向右平移。
