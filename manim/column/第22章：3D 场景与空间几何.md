# 第22章：3D 场景与空间几何

---

## 1. 项目背景

某大学数学系副教授正在用 Manim 录制"多元微积分"MOOC 课程。第一讲的主题是"多元函数的梯度下降"——需要展示一个三维曲面（z = f(x,y)），以及在曲面上移动的一个点，沿梯度方向滑向最低点。

教授的前几章都是用 `Axes` + `plot` 做的二维动画，但三维曲面需要完全不同的技术栈——`ThreeDScene`、`ThreeDAxes`、`Surface`、`ParametricFunction`，以及一套全新的三维摄像机控制（phi、theta、zoom）。

他试着自己摸索，结果第一版渲染出来是一个扁平的多边形——完全看不出三维感。问了同事才知道：他用了 `Scene` 而不是 `ThreeDScene`，Camera 默认是正交投影而非透视投影，所以没有近大远小的深度感。

切换到 `ThreeDScene` 后，曲面能渲染了，但新问题层出不穷：曲面的颜色太深透明度太低导致后面的网格线全被遮挡、摄像机角度不会调（phi/theta 的参数含义不明）、想在曲面上标注一个点但不知道该用哪个坐标映射函数。

这些痛点正是 3D Manim 的学习门槛。本章要系统讲解 `ThreeDScene` 的基础操作、曲面绘制、3D 坐标映射、摄像机控制以及"2D 标注叠加到 3D 场景"的技巧。

## 2. 剧本式交锋对话

> **场景**：教授的屏幕上是一个扁平的蓝色色块，像压扁的塑料布。小胖在吃蛋黄酥。

**小胖**（掰开蛋黄酥，碎渣掉了一桌）：

"教授，您这 3D 曲面怎么看起来像 2D 的？像个煎饼果子平摊在锅里。说好的'三维'呢，深度哪去了？"

**教授**（无奈）：

"我用的是普通 `Scene`。我以为把它改成 `ThreeDScene` 就行了，结果改了之后曲面跑出画面外了——phi 和 theta 参数到底是什么？"

**小白**（已经调出了正确的 3D 示例）：

"核心区别在于：`ThreeDScene` 使用了**透视投影**（perspective projection），远处的物体会变小；而 `Scene` 是**正交投影**，没有深度感。但光是继承类还不够——你需要设置摄像机的角度：

- **phi**：摄像机绕着**水平轴**旋转的角度（上下俯仰）。0° = 正对 XZ 平面，90° = 正上方俯视。
- **theta**：摄像机绕着**垂直轴**旋转的角度（左右旋转）。0° = 正对 X 轴，90° = 正对 Y 轴。
- **zoom**：摄像机距离原点的距离。值越小越近（物体越大）。

典型的三维教学视角：`self.set_camera_orientation(phi=70 * DEGREES, theta=-45 * DEGREES)`。phi 越大越俯视，theta 控制左右旋转。"

**大师**（补充）：

"我再补充坐标系——`ThreeDAxes` 与 `Axes` 的关键差别是**第三维（z 轴）**。它的 `coords_to_point(x, y, z)` 返回 3D 画面坐标。而 `Surface` 需要的核心参数是 `func(u, v) → (x, y, z)`，u 和 v 是参数域上的两个变量。

举个典型用法——画一个 z = x² + y² 的碗状曲面：

```python
surface = Surface(
    lambda u, v: axes.c2p(u, v, u**2 + v**2),
    u_range=[-3, 3], v_range=[-3, 3],
    resolution=(30, 30),
)
```

`resolution=(30, 30)` 决定了曲面的网格密度——越大越光滑但性能越差。"

> **技术映射**：`ThreeDAxes.coords_to_point(x, y, z)` 返回三维向量 `(px, py, pz)`，带深度信息。2D 的 Axes 只返回 `(px, py, 0)`。

**小胖**（蛋黄酥快吃完了）：

"那我想在 3D 曲面上标注一些 2D 文字——比如在谷底标'最小值'，怎么弄？"

**大师**：

"这是 3D 场景的经典混搭问题。3D 曲面上每一点是 (x, y, z) 三维坐标，而 2D 文字只能放在 (x, y) 平面上。你需要做的是——**3D 坐标转 2D 投影**：

```python
point_3d = axes.c2p(x0, y0, z0)
label = MathTex("最小值").move_to(point_3d)
# ThreeDScene 中 Mobject 的 z 坐标不会影响其 2D 位置
# 但可以手动将 z 分量映射为缩放因子（模拟近大远小）
```

不过更简单的做法是用 `Dot3D` 做 3D 标注点（它在这个三维坐标上渲染），然后在点上附加一个 2D 标签——标签的 `move_to` 会使用点在屏幕上的投影位置。"

## 3. 项目实战

### 3.1 环境准备

沿用基础篇环境。3D 功能是 Manim 标准模块，无需额外依赖。

#### 补充对话：3D 渲染性能剖析

**小胖**（蛋黄酥终于吃完了）：

"我还有一个灵魂拷问——3D 渲染是不是特别慢？我用 Surface 画了一个 100×100 的曲面，电脑风扇直接起飞。"

**大师**：

"问得好。3D 渲染的性能杀手就三个：

1. **Surface resolution 过高**：40×40=1600 面片是合理上限，100×100=10000 面片会让 Cairo 渲染器严重卡顿。
2. **三个 Axes 都开了刻度**：3D Axes 的每个刻度标签都是独立 Mobject。关掉不需要显示的轴刻度（`x_axis_config={"numbers_to_include": []}`）能省 30% 的对象数。
3. **`ambient_camera_rotation` 期间播放动画**：旋转本身就要重绘每一帧，如果同时有 `play` 动画，帧率直接腰斩。

如果必须用高分辨率曲面，考虑切换到 OpenGL 渲染器（`manim -p --renderer=opengl`），它利用 GPU 做面片渲染，对大规模曲面友好得多。`ThreeDScene` 还支持 `self.renderer.camera.light_source.move_to([...])` 调整光源位置以改善曲面立体感。"

> **技术映射**：Cairo 是 CPU 渲染，每个面片是一个独立 VMobject；OpenGL 是 GPU 渲染，面片在显存中批处理。高分辨率曲面建议用 `--renderer=opengl`。

### 3.2 分步实现

> **本章实战目标**：制作一个"二元函数梯度下降"的 3D 动画，展示曲面、等高线和下降路径。

---

#### 步骤一：ThreeDScene 入门

**步骤目标**：掌握 `ThreeDScene` 的创建、摄像机设置和基础的 3D 对象。

```python
# scenes/chapter22_3d_basics.py
from manim import *

class ThreeDBasics(ThreeDScene):
    def construct(self):
        # 设置摄像机视角
        self.set_camera_orientation(phi=70 * DEGREES, theta=-45 * DEGREES)

        # 3D 坐标轴
        axes = ThreeDAxes(
            x_range=[-5, 5, 1], y_range=[-5, 5, 1], z_range=[-2, 2, 1],
            x_length=8, y_length=8, z_length=4,
        )

        title = Text("3D 场景入门", font_size=36, color=BLUE)
        title.to_corner(UL, buff=0.3)
        self.add_fixed_in_frame_mobjects(title)

        self.play(Create(axes), run_time=2)

        # 旋转摄像机展示三维感
        self.begin_ambient_camera_rotation(rate=0.15)
        self.wait(2)
        self.stop_ambient_camera_rotation()

        # 手动调整到新视角
        self.move_camera(phi=30 * DEGREES, theta=30 * DEGREES, run_time=2)
        self.wait(1)

        self.play(FadeOut(axes), FadeOut(title), run_time=1.5)
        self.stop_ambient_camera_rotation()
```

**关键理解**：
- `self.add_fixed_in_frame_mobjects(title)` 让标题不随 3D 旋转而旋转——始终固定在屏幕上方。
- `self.begin_ambient_camera_rotation(rate=0.15)` 让摄像机自动匀速旋转，适合展示 3D 物体的全貌。
- `self.move_camera(phi=..., theta=..., run_time=...)` 平滑过渡到新视角。

---

#### 步骤二：3D 曲面与等高线

**步骤目标**：绘制 z = sin(r)/r 的曲面，并在底部叠加等高线。

```python
class SurfaceDemo(ThreeDScene):
    def construct(self):
        self.set_camera_orientation(phi=75 * DEGREES, theta=-60 * DEGREES)

        axes = ThreeDAxes(
            x_range=[-5, 5, 1], y_range=[-5, 5, 1], z_range=[-1, 1, 0.5],
            x_length=8, y_length=8, z_length=3,
        )
        axes.z_axis.add_labels({"0": MathTex("0").scale(0.8)})

        # 曲面：z = sin(sqrt(x²+y²)) / sqrt(x²+y²)
        surface = Surface(
            lambda u, v: axes.c2p(
                u, v,
                0.1 if np.sqrt(u**2+v**2) < 1e-6
                else np.sin(np.sqrt(u**2+v**2)) / np.sqrt(u**2+v**2)
            ),
            u_range=[-4, 4], v_range=[-4, 4],
            resolution=(40, 40),
            fill_opacity=0.8,
            checkerboard_colors=[BLUE_D, BLUE_E],
            stroke_width=0.5,
        )

        title = Text("3D 曲面：z = sin(r)/r", font_size=28, color=BLUE)
        title.to_corner(UL, buff=0.3)
        self.add_fixed_in_frame_mobjects(title)

        self.play(Create(axes), Create(surface), run_time=3)

        # 旋转展示
        self.begin_ambient_camera_rotation(rate=0.12)
        self.wait(3)
        self.stop_ambient_camera_rotation()
        self.wait(1)

        self.play(FadeOut(axes), FadeOut(surface), FadeOut(title), run_time=1.5)
```

**可能遇到的坑**：

1. **`resolution` 太高导致渲染慢**：40×40=1600 个面片。更高分辨率（如 100×100）在低清预览下能渲染但高清模式下可能吃光内存。
2. **`axes.c2p` 是 `coords_to_point` 的简写**：在 3D 场景中返回三维坐标向量，带 z 分量。
3. **`checkerboard_colors` 是双色棋盘**：每一小块面片的颜色在两种颜色之间交替。`BLUE_D`、`BLUE_E` 是 Manim 预设的深浅蓝。

#### 步骤三：梯度下降 3D 动画

**步骤目标**：展示参数点沿梯度方向在 3D 曲面上滑向最低点的过程。

```python
class GradientDescent3D(ThreeDScene):
    def construct(self):
        self.set_camera_orientation(phi=70 * DEGREES, theta=-50 * DEGREES)

        axes = ThreeDAxes(
            x_range=[-3, 3, 1], y_range=[-3, 3, 1], z_range=[0, 10, 2],
            x_length=7, y_length=7, z_length=4,
        )
        axes.shift(LEFT * 1.5 + DOWN * 0.5)

        # 损失曲面 f(x,y) = x² + 2y²
        surface = Surface(
            lambda u, v: axes.c2p(u, v, u**2 + 2*v**2),
            u_range=[-2.8, 2.8], v_range=[-2.8, 2.8],
            resolution=(35, 35),
            fill_opacity=0.7,
            checkerboard_colors=[BLUE_D, BLUE_E],
            stroke_width=0.3,
        )

        title = Text("梯度下降：f(x,y) = x² + 2y²", font_size=26, color=BLUE)
        title.to_corner(UL, buff=0.3)
        self.add_fixed_in_frame_mobjects(title)

        self.play(Create(axes), Create(surface), run_time=3)
        self.wait(0.5)

        # 参数点
        x_val = ValueTracker(2.5)
        y_val = ValueTracker(2.0)

        moving_dot = always_redraw(lambda: Sphere(
            radius=0.1,
            fill_color=RED,
        ).move_to(axes.c2p(x_val.get_value(), y_val.get_value(),
                           x_val.get_value()**2 + 2*y_val.get_value()**2)))

        # 下降轨迹
        path_points = []
        def get_path():
            if len(path_points) < 2:
                return VGroup()
            pts = [axes.c2p(x, y, x**2+2*y**2) for (x, y) in path_points]
            return VMobject().set_points_as_corners(pts).set_color(YELLOW).set_stroke(width=3)

        trail = always_redraw(get_path)

        self.add(moving_dot, trail)

        # 梯度下降迭代
        x, y = 2.5, 2.0
        lr = 0.15
        for step in range(25):
            grad_x, grad_y = 2 * x, 4 * y
            new_x = x - lr * grad_x
            new_y = y - lr * grad_y
            path_points.append((x, y))

            self.play(
                x_val.animate.set_value(new_x),
                y_val.animate.set_value(new_y),
                run_time=0.2,
            )
            x, y = new_x, new_y

        self.wait(0.5)

        # 旋转展示最终位置
        self.begin_ambient_camera_rotation(rate=0.1)
        self.wait(2)
        self.stop_ambient_camera_rotation()

        self.play(FadeOut(VGroup(axes, surface, moving_dot, trail, title)), run_time=1.5)
```

**运行命令**：

```bash
manim -pqm scenes/chapter22_gradient_3d.py GradientDescent3D
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 3D 视觉 | 透视投影 + 旋转展示增强空间感 | 深色背景下曲面细节容易丢失 |
| Surface | 网格密度可调，数学函数直接映射 | CPU 渲染下大分辨率曲面很慢 |
| 摄像机 | phi/theta/zoom 三参数直观控制 | 自动旋转速度单位不直观（rate=0.1 是 rad/s） |
| 固定标签 | `add_fixed_in_frame_mobjects` 让 2D 文字不随旋转飘 | 固定标签与 3D 内容之间没有连线引导 |
| ThreeDAxes | z 轴扩展让坐标系完整 | 标注和刻度在旋转时可能被遮挡 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 多元微积分 | 曲面、梯度、方向导数 |
| 优化算法 | 梯度下降、牛顿法在曲面上的路径 |
| 物理场 | 势能曲面、电场分布 |
| 3D 几何 | 球面、柱面、锥面的教学 |
| 数据可视化 | 三维散点图、网格曲面 |

**不适用场景**：需要大屏实时交互的 3D（使用 three.js/WebGL）、影视级 3D 渲染（使用 Blender）、带纹理贴图的 3D 模型（使用 Unity/Unreal）。

### 注意事项

1. **`ThreeDAxes.z_range` 影响轴的缩放**：z_range 配合 z_length 决定 Z 轴的单位长度。设置不当时 Z 轴可能压缩或拉伸。
2. **`surface.resolution` 是二次开销**：`resolution=(N, N)` 意味着 N² 个面片。N=50 时 2500 面片是一个合理的上限。
3. **摄像机旋转期间不要 play 其他动画**：`ambient_camera_rotation` 期间播放其他动画可能导致画面撕裂。

### 常见踩坑经验

**故障一：3D 曲面渲染出来是二维平面**

根因：忘了继承 `ThreeDScene` 而是继承了 `Scene`。

解决：类定义改为 `class MyScene(ThreeDScene)`。

**故障二：表面颜色块状不均匀**

根因：`resolution` 太低导致面片过大。

解决：提高 `resolution` 到 (40, 40) 或更高，或设置 `fill_opacity=0.9` 减少棋盘格对比度。

**故障三：`c2p` 返回的坐标在摄像机旋转后偏移**

根因：`c2p` 返回的是画布绝对坐标，摄像机旋转后这个坐标会变。

解决：不要在摄像机旋转期间依赖之前计算的 `c2p` 值，改用 `always_redraw` 每帧重新计算。

### 思考题

1. 在 `GradientDescent3D` 的曲面底部（Z=0 平面）叠加等高线——用 `ParametricFunction` 在 `ThreeDAxes` 的底平面绘制 `x²+2y²=constant` 的椭圆族。提示：等高线在 XY 平面绘制，z 坐标恒为 0。

2. 改造 `GradientDescent3D` 加入动量法（Momentum）：维护速度变量 `vx, vy`，每一步更新 `vx = beta*vx - lr*grad_x` 和 `x = x + vx`。在动画中对比标准梯度下降（红色路径）和动量梯度下降（绿色路径）的收敛速度差异。
