## 1. 项目背景

经过中级篇 14 章（第 17-30 章）的学习，读者已经掌握了复杂动画编排、ValueTracker 驱动、3D 场景、数据图表、物理模拟、组件化工程、音画同步、批量渲染和性能优化等核心技能。现在是时候将这些技能拧成一股绳——独立交付一个完整的**数据可视化教学视频工程**。

本章模拟一个真实的业务场景：某在线教育团队需要为"Python 数据分析入门"课程制作一套 10 集的动画教学视频，每集 2-3 分钟，主题涵盖折线图、柱状图、散点图、直方图、饼图、箱线图、热力图、时间序列、相关性分析和综合仪表盘。

这个项目需要整合中级篇的以下核心技能：
- 数据图表（第 20、23 章）：Axes、BarChart、plot_line_graph
- 复杂编排（第 17 章）：AnimationGroup/Succession 多层嵌套
- ValueTracker 驱动（第 19 章）：数据随时间变化的动态图表
- 组件化工程（第 25 章）：TitleCard、StepIndicator、InfoBox
- 音画同步（第 27 章）：manim-voiceover 旁白对齐
- 批量渲染（第 28 章）：build_v2.py 结构化报告
- 性能优化（第 29 章）：大场景的合批与缓存
- 插件生态（第 30 章）：networkx、pandas 数据导入

```python
# data/loader.py
import csv, json
from pathlib import Path

class DataLoader:
    def __init__(self, data_dir="data"):
        self.data_dir = Path(data_dir)

    def load_csv(self, filename):
        with open(self.data_dir / filename, encoding="utf-8") as f:
            return list(csv.DictReader(f))

    def load_json(self, filename):
        with open(self.data_dir / filename, encoding="utf-8") as f:
            return json.load(f)
```

"10 个场景全部通过 `DataLoader` 取数据——数据源变化时只改加载器，不改场景代码。"

### 3.2 第一集完整代码

> 本节参考第 16 章的"分段编码"模式，将第 1 集拆为 6 个自然段落逐段编写。

**步骤目标**：制作"月度销售趋势分析"第一集，展示中级篇综合技能的整合。

**分段编码——第 1 段：开场标题（0-8s）**：

```python
# ===== 第 1 段：开场标题 (0-8s) =====
class TitleCard(VGroup):
    def __init__(self, title, subtitle=""):
        super().__init__()
        box = RoundedRectangle(width=10, height=1.8, corner_radius=0.15,
                               color=PRIMARY, fill_opacity=0.08, stroke_width=2)
        t = Text(title, font_size=36, color=PRIMARY, weight=BOLD)
        t.move_to(box.get_center() + UP * 0.2)
        self.add(box, t)
        if subtitle:
            s = Text(subtitle, font_size=22, color=TEXT_MUTED)
            s.next_to(t, DOWN, buff=0.15)
            self.add(s)

class Ep01SalesTrend(Scene):
    def construct(self):
        card = TitleCard("月度销售趋势分析", "第1集 · 折线图与柱状图")
        card.to_edge(UP, buff=0.4)
        self.play(FadeIn(card, scale=0.95), run_time=1.5)
        self.wait(0.5)
```

**第 2 段：加载数据 + Axes 绘制（8-18s）**：

#### 补充实战：更多图表类型的模板化实现

有了第 1 集的工程模板，后续图表类型的开发变成"填空"工作。以下是几个典型图表的极简模板：

**第 3 集 - 直方图模板**：
```python
class Ep03Histogram(Scene):
    def construct(self):
        card = TitleCard("直方图与分布分析", "第3集 · 数据分布可视化")
        card.to_edge(UP, buff=0.4)
        self.play(FadeIn(card, scale=0.95))

        # 模拟考试成绩数据
        scores = np.random.normal(72, 12, 200).clip(0, 100)
        hist, bins = np.histogram(scores, bins=10, range=(0, 100))

        axes = Axes(x_range=[0, 100, 10], y_range=[0, max(hist)+5, 10],
                     x_length=9, y_length=5)
        axes.to_edge(DOWN, buff=0.6)
        self.play(Create(axes), run_time=2)

        bars = axes.plot_bar_chart(
            values=list(hist),
            bar_colors=[PRIMARY] * len(hist),
            bar_fill_opacity=0.7,
        )
        self.play(LaggedStart(*[GrowFromEdge(b, DOWN) for b in bars], lag_ratio=0.1), run_time=2)
        self.wait(1.5)
```

**第 7 集 - 热力图模板**：
```python
class Ep07Heatmap(Scene):
    def construct(self):
        card = TitleCard("相关性热力图", "第7集 · 变量关联分析")
        card.to_edge(UP, buff=0.4)
        self.play(FadeIn(card, scale=0.95))

        # 生成 6×6 相关性矩阵
        matrix_data = np.random.rand(6, 6)
        for i in range(6): matrix_data[i,i] = 1.0

        # 用 Rectangle 矩阵实现热力图
        cells = VGroup()
        n = 6; size = 0.8
        for i in range(n):
            for j in range(n):
                val = matrix_data[i, j]
                rect = Square(side_length=size, color=interpolate_color(BLUE, RED, val),
                              fill_opacity=val, stroke_width=0.5)
                rect.move_to([(j - n/2 + 0.5) * size, (n/2 - i - 0.5) * size, 0])
                cells.add(rect)
        cells.move_to(ORIGIN + DOWN * 0.3)
        self.play(LaggedStart(*[FadeIn(c) for c in cells], lag_ratio=0.05), run_time=3)
        self.wait(1.5)
```

这些模板验证了工程化的核心价值——新增一集只需要 50-80 行业务代码，其余全部复用基础设施。

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 工程模板 | 10 集视频共享基础设施，单集代码 < 150 行 | 初始搭建模板需要 2-4 小时 |
| 数据驱动 | 修改 CSV 即可更新动画内容 | CSV 结构变化时需要同步改代码 |
| 组件复用 | TitleCard/InfoBox 跨集统一 | 组件样式固化后特殊场景需要额外覆盖 |
| 生产流水线 | build_v2.py 一键批量渲染 | 配置和调试流水线本身需要中级篇全部知识 |
| 可扩展性 | 新增第 11 集只需加 CSV + 场景文件 | 场景过多后 build_v2.py 串行渲染时间线性增长 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 在线课程 | 标准化课件的大规模生产 |
| 数据报告 | 月度/季度数据报告的自动化生成 |
| 教学实验 | 不同参数条件下的对比分析 |
| 企业培训 | 统一风格的内部培训视频 |
| 开源文档 | 配套视频教程的持续更新 |

### 注意事项

1. **DataLoader 的编码处理**：Windows 生成的 CSV 可能是 GBK 编码，`DataLoader` 需支持 `encoding` 参数。
2. **组件库的版本管理**：`components/` 包的修改会影响所有场景。建议在改组件后批量渲染所有集做回归测试。
3. **字幕文件的命名**：`.srt` 文件按集编号命名（`ep01.srt`），方便与视频文件对应归档。

### 常见踩坑经验

**故障一：第 5 集的组件样式和第 1 集不同**

根因：组件代码在项目进行中被修改，但早期集没重新渲染。

解决：改组件后运行 `python build_v2.py h` 批量重渲染全部集。

**故障二：CSV 文件中的空行导致 DataLoader 异常**

根因：Excel 导出的 CSV 文件末尾有多余空行。

解决：在 `load_csv` 中过滤空行：`[row for row in reader if row]`。

**故障三：批量渲染中第 7 集带旁白，其他集无旁白不一致**

根因：配音录制进度落后，但渲染脚本没有区分"有旁白/无旁白"场景。

解决：在 `build_v2.py` 中通过配置文件标记每集是否含旁白。

**故障四：CSV 中数值字段带千位分隔符（如 "1,200"）导致 float() 报错**

根因：Excel 导出时自动添加了千位分隔符。

解决：在 `load_csv` 中预处理：`value.replace(",", "")` 再去 `float()`。

**故障五：10 集渲染到一半硬盘满了**

根因：每集 1080p 约 50MB，10 集约 500MB，加上 LaTeX 缓存和中间帧 PNG，轻松超 2GB。

解决：定期清理 `build/tex_cache/` 和渲染中间产物。在 `build_v2.py` 末尾调用 `clean_build()`。
