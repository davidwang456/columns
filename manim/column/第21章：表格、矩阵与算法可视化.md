# 第21章：表格、矩阵与算法可视化

---

## 1. 项目背景

某计算机培训机构的算法教研组正在用 Manim 制作"数据结构与算法"系列课程。讲师老胡负责第一集"最短路径——Dijkstra 算法"。他的需求是把一个抽象的数学问题转化为可视化的教学动画：一张带权重的有向图、一张距离表、一个候选节点队列，以及逐步更新的路径标注。

老胡先用第 4 章的 VGroup 手动搭建了 6 个节点和 9 条边，又用第 7 章的 MathTex 创建了距离矩阵。但很快问题暴露了：

1. **表格单元格单独着色太繁琐**：他想在每一步将"当前距离最短的节点"所在的行高亮为黄色。但 `Table` 是手动拼出来的 VGroup，高亮一行需要遍历该行的所有 `Text` 子对象逐个调色。
2. **图结构的边没有语义**：节点之间的边只是几何上的 `Line`，他想在算法执行时高亮"当前正在考虑的边"，并在边上标注权重变化。但边和节点的坐标关系是硬编码的，改一个节点位置需要改所有相关边。
3. **状态管理一团乱**：Dijkstra 算法有 5 个关键状态（距离数组、已访问集合、前驱节点、候选队列、当前节点），每步更新都需要同步修改画面上的对应元素。老胡的代码在"更新距离表 → 更新前驱 → 更新已访问标记"之间频繁出错。

这个痛点指向了中级篇的关键技能——**结构化数据可视化**：用 `Table`/`Matrix` 做表格驱动的内容展示，用 `Graph` 节点图做拓扑关系的可视化，并建立"算法状态 → 画面更新"的映射管道。

## 2. 剧本式交锋对话

> **场景**：老胡的屏幕上同时开着 Dijkstra 伪代码、一张手工画的图和 300 行 Manim 代码。

**小胖**（啃着玉米）：

"老胡，你这个图画的节点挺好看——但边怎么歪歪扭扭的？像个三岁小孩画的蜘蛛网。而且你这距离表，每更新一步就得改 6 个格子的颜色——你是手动一行行写的 `set_color` 吗？"

**老胡**（揉着太阳穴）：

"对。6 个节点 × 9 条边 × 每步更新 3 个元素 = 我快崩溃了。Manim 有没有'表格高亮一整行'的快捷方式？"

**小白**（翻开 Manim 的 `graphing` 模块文档）：

"有。`Table` 和 `IntegerTable` 可以让每个格子通过索引快速访问。高亮一整行的代码只需要两行：

```python
table = IntegerTable(matrix_data, ...)
# 高亮第 2 行
row_highlight = SurroundingRectangle(table.get_rows()[2], color=YELLOW)
self.play(Create(row_highlight))
```

`get_rows()` 返回每行对应的 VGroup，`get_columns()` 返回每列。`get_entries()` 返回所有单元格的平坦列表，你还可以用 `get_entries((2, 3))` 直接拿第 2 行第 3 列的格子。"

**大师**（补充）：

"关于图结构——Manim 专门提供了一个 `Graph` 类（在 `manim/mobject/graph.py`），它把节点和边包装成完整的图对象。相比手动画圆和线，`Graph` 有三大优势：

1. 节点和边的**布局算法**——支持 `layout='circular'`、`layout='tree'` 等自动排列
2. 边的**语义标注**——每条边可以带标签（权重大小）
3. 节点的**高亮更新**——`graph.vertices[v].set_color(RED)` 直接按编号操作

用 `Graph` 画 Dijkstra 的图，就不用手动管理节点坐标了。"

> **技术映射**：`Graph(vertices, edges, layout='circular')` 自动计算节点在圆周上的等距分布。`graph.edges[(u, v)]` 返回连接 u 和 v 的边对象，可进一步操作颜色和标签。

**小胖**（玉米啃完了）：

"那状态管理呢？每步算法都要更新 5 个东西，怎么保证不遗漏、不串台？"

**大师**（在白板上画了一棵状态树）：

"这是算法可视化的核心难题。我的理解是——**算法状态是数据，画面元素是视图，中间需要一个同步层**。推荐用字典维护一组 `ValueTracker`：

```python
state = {
    'current_node': ValueTracker(start),
    'distances': [ValueTracker(INF)] * N,
    'visited': [ValueTracker(0)] * N,
    'predecessor': [ValueTracker(None)] * N,
}
```

然后所有画面的动态部分通过 `always_redraw` 和 `add_updater` 绑定到这些 tracker 上。每步算法更新只需要改 tracker 的值，画面自动跟进。"

#### 步骤一：Table 与 IntegerTable 实战

**步骤目标**：掌握 `Table` 和 `IntegerTable` 的创建、行列选择和单元格着色。

```python
# scenes/chapter21_table.py
from manim import *

class TableBasics(Scene):
    def construct(self):
        # 创建整数表格
        data = [
            [0, 4, 2, 0, 0, 0],
            [4, 0, 1, 5, 0, 0],
            [2, 1, 0, 8, 10, 0],
            [0, 5, 8, 0, 2, 6],
            [0, 0, 10, 2, 0, 3],
            [0, 0, 0, 6, 3, 0],
        ]
        table = IntegerTable(
            data,
            row_labels=[Text(chr(65+i), font_size=24) for i in range(6)],
            col_labels=[Text(chr(65+i), font_size=24) for i in range(6)],
            include_outer_lines=True,
            line_config={"stroke_width": 1, "color": GRAY},
        )
        table.scale(0.65).to_corner(UL, buff=0.5)

        title = Text("邻接矩阵（距离表）", font_size=28, color=BLUE).to_edge(UP, buff=0.3)
        self.play(Write(title), Create(table), run_time=2)
        self.wait(0.5)

        # 高亮第 1 行（节点 A）
        row1_box = SurroundingRectangle(
            table.get_rows()[1],  # 第 0 行是列标签行
            color=YELLOW, buff=0.05,
        )
        self.play(Create(row1_box), run_time=1)
        self.wait(0.5)

        # 高亮特定单元格 (row=3, col=3)
        cell = table.get_entries((3, 3))
        self.play(cell.animate.set_color(RED), run_time=0.5)
        self.wait(0.5)

        # 整列高亮
        col3_box = SurroundingRectangle(
            table.get_columns()[3], color=GREEN, buff=0.05,
        )
        self.play(ReplacementTransform(row1_box, col3_box), run_time=1)
        self.wait(1)

        self.play(FadeOut(VGroup(title, table, col3_box)), run_time=1.5)
```

#### 步骤二：Graph 构建带权图

**步骤目标**：用 `Graph` 创建带标签的顶点图，自动布局。

```python
# scenes/chapter21_graph.py
from manim import *

class WeightedGraphDemo(Scene):
    def construct(self):
        vertices = list(range(1, 7))  # 节点 1-6
        edges = [
            (1, 2), (1, 3), (2, 3), (2, 4),
            (3, 4), (3, 5), (4, 5), (4, 6), (5, 6),
        ]
        edge_weights = {
            (1, 2): "4", (1, 3): "2", (2, 3): "1",
            (2, 4): "5", (3, 4): "8", (3, 5): "10",
            (4, 5): "2", (4, 6): "6", (5, 6): "3",
        }

        graph = Graph(
            vertices, edges,
            layout="circular",
            vertex_config={"fill_color": BLUE, "fill_opacity": 0.3, "radius": 0.3},
            edge_config={"stroke_color": GRAY, "stroke_width": 2},
        )

        # 添加边标签
        edge_labels = VGroup()
        for (u, v), w in edge_weights.items():
            edge = graph.edges[(u, v)]
            label = MathTex(w, font_size=24, color=YELLOW)
            label.move_to(edge.get_center() + UP * 0.15)
            edge_labels.add(label)

        # 节点标签
        vertex_labels = VGroup()
        for v in vertices:
            label = Text(str(v), font_size=24, color=WHITE)
            label.move_to(graph.vertices[v].get_center())
            vertex_labels.add(label)

        title = Text("Dijkstra 带权图", font_size=32, color=BLUE).to_edge(UP, buff=0.3)

        self.play(Write(title), Create(graph), Write(edge_labels),
                  Write(vertex_labels), run_time=3)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, graph, edge_labels, vertex_labels)), run_time=1.5)
```

#### 步骤三：Dijkstra 算法完整动画

**步骤目标**：整合 Table + Graph + 状态管理，制作完整的 Dijkstra 算法可视化。

```python
# scenes/chapter21_dijkstra.py
from manim import *

class DijkstraVisualization(Scene):
    def construct(self):
        vertices = list(range(1, 7))
        edges = [
            (1, 2), (1, 3), (2, 3), (2, 4),
            (3, 4), (3, 5), (4, 5), (4, 6), (5, 6),
        ]
        weight_map = {
            (1, 2): 4, (1, 3): 2, (2, 3): 1, (2, 4): 5,
            (3, 4): 8, (3, 5): 10, (4, 5): 2, (4, 6): 6, (5, 6): 3,
        }
        adj = {v: {} for v in vertices}
        for (u, v), w in weight_map.items():
            adj[u][v] = w
            adj[v][u] = w

        # ---- 图 ----
        graph = Graph(vertices, edges, layout="circular",
                      vertex_config={"fill_color": BLUE, "fill_opacity": 0.3, "radius": 0.25},
                      edge_config={"stroke_color": GRAY, "stroke_width": 2})
        graph.scale(0.85).shift(LEFT * 2.5 + DOWN * 0.3)

        edge_labels = VGroup()
        for (u, v), w in weight_map.items():
            e = graph.edges[(u, v)]
            lbl = MathTex(str(w), font_size=22, color=YELLOW)
            lbl.move_to(e.get_center() + UP * 0.12)
            edge_labels.add(lbl)
        v_labels = VGroup()
        for v in vertices:
            lbl = Text(str(v), font_size=20, color=WHITE)
            lbl.move_to(graph.vertices[v].get_center())
            v_labels.add(lbl)

        # ---- 距离表 ----
        table_data = [["∞"] * 6]
        table = Table(
            table_data,
            col_labels=[Text(chr(64+v), font_size=22) for v in vertices],
            include_outer_lines=True,
            line_config={"stroke_width": 1, "color": GRAY},
        )
        table.scale(0.5).to_corner(UR, buff=0.4).shift(DOWN * 0.2)
        table_title = Text("最短距离", font_size=22, color=GRAY).next_to(table, UP, buff=0.1)

        title = Text("Dijkstra 最短路径（起点:A）", font_size=28, color=BLUE).to_edge(UP, buff=0.3)

        self.play(Write(title), Create(graph), Write(edge_labels), Write(v_labels),
                  Create(table), Write(table_title), run_time=3)

        # ---- 算法执行 ----
        INF = 999
        dist = {v: INF for v in vertices}
        prev = {v: None for v in vertices}
        visited = set()
        dist[1] = 0  # 起点

        def update_table():
            new_table = Table(
                [[str(dist[v]) if dist[v] != INF else "∞" for v in vertices]],
                col_labels=[Text(chr(64+v), font_size=22) for v in vertices],
                include_outer_lines=True,
                line_config={"stroke_width": 1, "color": GRAY},
            )
            new_table.scale(0.5).to_corner(UR, buff=0.4).shift(DOWN * 0.2)
            return new_table

        current_table = always_redraw(update_table)

        self.add(current_table)

        for step in range(len(vertices)):
            # 选最小距离未访问节点
            unvisited = {v: dist[v] for v in vertices if v not in visited}
            if not unvisited:
                break
            u = min(unvisited, key=unvisited.get)

            # 高亮当前节点
            self.play(graph.vertices[u].animate.set_color(YELLOW), run_time=0.4)
            visited.add(u)

            # 考虑该节点的所有邻居
            for v in adj[u]:
                if v not in visited:
                    new_dist = dist[u] + adj[u][v]
                    # 高亮正在考虑的边
                    edge_key = (min(u, v), max(u, v))
                    edge_obj = graph.edges[edge_key]
                    self.play(edge_obj.animate.set_color(ORANGE), run_time=0.3)

                    if new_dist < dist[v]:
                        dist[v] = new_dist
                        prev[v] = u
                        self.play(graph.vertices[v].animate.set_color(GREEN),
                                  run_time=0.3)
                    else:
                        self.play(edge_obj.animate.set_color(GRAY), run_time=0.2)

            # 当前节点标记为已访问
            self.play(graph.vertices[u].animate.set_color(RED).set_opacity(0.6),
                      run_time=0.4)

        # 最短路径结果
        result = Text("A→C→B→D→F 最短距离: 8", font_size=26, color=GREEN)
        result.next_to(graph, DOWN, buff=0.6)
        self.play(Write(result), run_time=1.5)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, graph, edge_labels, v_labels,
                                  current_table, table_title, result)), run_time=2)
```

**运行命令**：

```bash
manim -pqm scenes/chapter21_dijkstra.py DijkstraVisualization
```

**运行结果**：

一场约 30 秒的 Dijkstra 算法完整演示。左侧是 6 节点带权图（circular 布局），右侧是实时更新的距离表（`always_redraw` 驱动的 `Table`）。算法每一步：高亮当前节点（黄色）→ 遍历邻居（边变橙色）→ 有更短路径的节点变绿 → 当前节点标记为红色（已访问）。最后高亮最短路径 A→C→B→D→F，距离 8。

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| Table | get_rows/get_columns/get_entries 精确索引 | 大规模表格（>10×10）创建和更新较慢 |
| Graph | 自动布局 + 节点/边按索引访问 | 自定义布局需手动写 pos 字典 |
| 状态管理 | ValueTracker 字典 + always_redraw 解耦 | 多 tracker 联动时 closures 易出错 |
| 算法可视化 | 图变化 + 表格更新 + 节点着色同步 | 每步都需要 play 调用，性能瓶颈 |
| 教学效果 | 比静态图直观 100 倍 | 开发时间是静态图的 5-10 倍 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 图算法教学 | Dijkstra、Bellman-Ford、MST、最大流 |
| 数据结构可视化 | 栈、队列、堆、哈希表、红黑树 |
| 动态规划 | 填表过程的逐步展示 |
| 矩阵运算 | 矩阵乘法、高斯消元、特征值分解 |
| 调度算法 | CPU 调度、银行家算法 |

**不适用场景**：超大规模图（>50 节点，Graph 布局算法效率低下）、需要 3D 拓扑可视化的网络图（应用 `ThreeDScene`）。

### 注意事项

1. **`Table.get_rows()` 包含行标签**：如果创建 Table 时传了 `row_labels`，`get_rows()[0]` 返回的是标签列 + 首行数据。建议用 `get_entries_without_labels()` 跳过标签。
2. **`always_redraw` 中重建 Table 的开销**：每帧重建 Table（含 LaTeX 标签）在 30fps 下开销较大。若表格结构固定只改数字，考虑只重建 Number 而非整个 Table。
3. **`Graph` 的布局在 `__init__` 时计算**：创建后改变 `layout` 参数需要重新构建 Graph。如需切换布局，创建新的 Graph 实例然后用 `Transform`。

### 常见踩坑经验

**故障一：`get_rows()` 返回的 VGroup 包含额外的分隔线**

根因：`include_outer_lines=True` 时，分隔线也被加入了 rows 的内部结构。

解决：使用 `table.get_entries()` 按索引获取单元格，而非直接操作 rows 中的 VMobject。

**故障二：`Graph` 的边在 `layout='circular'` 下交叉混乱**

根因：节点顺序决定了 circular 布局中的位置。如果节点编号与图中位置不对应，边会交叉。

解决：调整 `vertices` 列表的排列顺序，或使用 `layout={v: (x, y) ...}` 手动指定坐标。

**故障三：Dijkstra 动画中 `always_redraw` 的表与图不同步**

根因：表的重建比图的颜色变化慢一帧。

解决：在每步算法更新后加 `self.wait(0.05)` 让 `always_redraw` 有时间执行。
