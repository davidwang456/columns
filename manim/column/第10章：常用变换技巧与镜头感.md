# 第10章：常用变换技巧与镜头感

---

## 1. 项目背景

某数学科普自媒体团队的负责人老张正在审核手下新人制作的"函数从表达式到图像"动画。新人的实现方式很直接：表达式出现 → 等 2 秒 → 坐标轴出现 → 等 2 秒 → 曲线上场 → 结束。

老张看完后评价："逻辑是对的，但像在看菜谱——每一项都告诉你,但没有任何过渡和关联。观众看完会记住'这里有公式、这里有坐标轴、这里有曲线',但不会觉得'哦，原来这条曲线就是从这个表达式变出来的'。"

老张进一步指出："好的教学动画，不仅要把东西展示出来，还要让观众'看见'知识之间的关系。比如表达式右边的那堆字母，它们是怎么变成曲线上的点的？最好有一个镜头告诉你'这个字母对应曲线的这个特征'——这就叫**镜头感**。"

这个痛点正是 Manim 进阶的核心——**动画的叙事性**。除了基础的 `Create`/`FadeIn`/`Transform` 之外，Manim 还提供了一系列高级变换技巧来制造"视觉关联"：

- `TransformMatchingTex`：让两个公式之间"公共的部分保留，不同的部分 morph"
- `TransformMatchingShapes`：让两个几何图形之间"相似的边保留，差异的部分 morph"
- `FocusOn` / `Indicate` / `ApplyWave`：引导观众视线到重点区域
- `MovingCameraScene`：镜头推拉摇移，创造空间纵深感

本章的目标就是把这些技巧系统化，从"能动的画面"升级为"会说话的画面"。

---

## 2. 剧本式交锋对话

> **场景**：老张把一段"跳跃切换"式的动画和新版"过渡流畅"的动画并排播放。小胖、小白和大师都在看。

**小胖**（啃着巧克力棒）：

"老张，左边这段我看了 5 遍都没搞懂——公式和曲线到底啥关系？就一个闪变就完事了？右边这个是怎么让公式'变'成曲线的？"

**老张**（点击暂停，指着代码）：

"右边用的是 `TransformMatchingTex`。普通的 `Transform` 是把整个公式作为一个整体来 morph——所有字符一起模糊、一起变成目标。`TransformMatchingTex` 会做字符级匹配——它在两个 LaTeX 公式之间找'相同的字符'，相同的就原地不动，不同的才 morph。这样观众就能看到'哪些部分变了，哪些没变'——这就是视觉关联。"

**小白**（翻看源码文档）：

"`TransformMatchingTex` 的原理很精巧。它把两个 `MathTex` 对象都拆成 token 列表（每个字符一个 token），然后用 `difflib.SequenceMatcher` 找出最长公共子序列。公共的 token 保持原位，新增/删除的 token 才执行 fade out/fade in 或 morph。如果两个公式有相同的结构（比如 `x^2` 变成 `(x-1)^2`），相同部分越多，观众越容易看出'这里加了一个 -1'。"

**大师**：

"小白把原理讲清楚了。我来补充一个使用场景矩阵——什么时候用哪种变换："

| 场景 | 推荐变换 | 原因 |
|------|----------|------|
| 公式 a → 公式 b，结构相近 | `TransformMatchingTex` | 突出公共部分的变化 |
| 圆 → 方 → 三角等几何形态 | `TransformMatchingShapes` | 顶点数不同的形状 morph |
| 颜色强调某个对象 | `Indicate` / `Flash` | 短暂吸引视线 |
| 区域性关注 | `FocusOn` | 镜头推近，周围暗化 |
| 过渡/换页效果 | `ApplyWave` / `ShowPassingFlash` | 暗示画面切换 |
| 两个对象完全不同的转换 | `ReplacementTransform` | 旧对象消失→新对象出现 |

> **技术映射**：`TransformMatchingTex` 的匹配算法位于 `manim/animation/transform_matching_parts.py`。`TransformMatchingShapes` 继承同一思路，但比较的是贝塞尔点集的形状相似度而非字符。

**小胖**（举着巧克力棒）：

"等等，`MovingCameraScene` 是什么？我上次看到 3Blue1Brown 的视频里，镜头会推进到某个细节然后拉回来——那个效果帅炸了！"

**大师**（切换到摄像头控制的演示代码）：

"好眼光。`MovingCameraScene` 是 `Scene` 的子类，它自带一个可以移动、缩放、旋转的 Camera。你可以在任何 `play` 语句里用 `self.camera.frame.animate.move_to(...)` 或 `.scale(...)` 来移动镜头。典型用法：

- **推镜头**：`self.camera.frame.animate.scale(0.3).move_to(detail)` → 聚焦到细节
- **拉镜头**：`self.camera.frame.animate.scale(1.5).move_to(ORIGIN)` → 展示全景
- **平移**：`self.camera.frame.animate.shift(RIGHT * 5)` → 跟随运动对象

最关键的一点：**镜头运动应该和内容动画同步**。不能先移镜头再出现内容——那样观众在看空画面。正确的模式是：`self.play(FadeIn(detail), self.camera.frame.animate.scale(0.3).move_to(detail))`。"

> **技术映射**：`MovingCameraScene` 在 `setup()` 中创建了一个 `self.camera.frame`（一个 `Mobject`），Camera 用这个 frame 来裁剪画面。`frame.scale(0.3)` 相当于放大 1/0.3 ≈ 3.3 倍。

**小白**（追问）：

"那如果有多个焦点要依次推过去，是不是需要写一堆 `play` 语句？"

**大师**：

"对，但可以封装。比如定义一个方法 `focus_on(obj)`，内部实现 `self.play(FocusOn(obj), run_time=0.5)`。然后你的主流程就变成：讲课 → `focus_on(公式左半边)` → 讲解 → `focus_on(公式右半边)` → 讲解 → `focus_on(全图)` → 总结。镜头服务内容，而不是反过来。"

### 3.2 分步实现

#### 步骤一：TransformMatchingTex 实践

**步骤目标**：对比普通 `Transform` 和 `TransformMatchingTex` 的视觉效果差异。

```python
# scenes/chapter10_transform_tex.py
from manim import *

class TransformTexDemo(Scene):
    def construct(self):
        # --- 左侧：普通 Transform ---
        left_title = Text("普通 Transform", font_size=24, color=RED)
        left_title.to_edge(LEFT, buff=0.8).shift(UP * 2.5)

        f1_left = MathTex(r"y = x^2", font_size=48, color=WHITE)
        f1_left.next_to(left_title, DOWN, buff=0.8)

        f2_left = MathTex(r"y = (x-1)^2 + 3", font_size=42, color=WHITE)
        f2_left.next_to(left_title, DOWN, buff=0.8)

        self.play(Write(left_title), Write(f1_left), run_time=1.5)
        self.wait(0.3)
        self.play(Transform(f1_left, f2_left), run_time=2)
        self.wait(0.5)

        # --- 右侧：TransformMatchingTex ---
        right_title = Text("TransformMatchingTex", font_size=24, color=GREEN)
        right_title.to_edge(RIGHT, buff=0.8).shift(UP * 2.5)

        f1_right = MathTex(r"y = x^2", font_size=48, color=WHITE)
        f1_right.next_to(right_title, DOWN, buff=0.8)

        self.play(Write(right_title), Write(f1_right), run_time=1.5)
        self.wait(0.3)
        self.play(TransformMatchingTex(f1_right, MathTex(
            r"y = (x-1)^2 + 3", font_size=42, color=WHITE
        )), run_time=2)
        self.wait(1.5)

        self.play(FadeOut(VGroup(left_title, f1_left,
            right_title, f1_right)), run_time=1.5)
```

**运行结果**：

左侧用普通 `Transform`——整个 `y = x^2` 整体 morph 成 `y = (x-1)^2 + 3`，有点模糊和抖动。右侧用 `TransformMatchingTex`——`y =`、`x`、`^2` 这些公共部分保持原位，只有新增的 `(-1)` 和 `+ 3` 从原来的位置"长出来"。观众能一眼看出公式的变化点。

#### 步骤二：镜头推拉实战

**步骤目标**：用 `MovingCameraScene` 实现从全景 → 局部细节 → 回归全景的镜头运动。

```python
# scenes/chapter10_camera.py
from manim import *

class CameraDemo(MovingCameraScene):
    def construct(self):
        # ---- 创建一个大场景 ----
        # 背景网格
        grid = NumberPlane(
            x_range=[-10, 10, 1], y_range=[-6, 6, 1],
            background_line_style={"stroke_opacity": 0.3},
        )
        self.add(grid)

        # 散布一些对象
        dots = VGroup()
        for i in range(30):
            dot = Dot(
                [np.random.uniform(-8, 8), np.random.uniform(-4, 4), 0],
                radius=0.05,
                color=interpolate_color(RED, BLUE, i / 30),
            )
            dots.add(dot)

        title = Text("镜头推拉演示", font_size=36, color=BLUE)
        title.to_edge(UP, buff=0.3)

        self.play(Write(title), FadeIn(dots), run_time=2)
        self.wait(0.5)

        # ---- 镜头推近：聚焦某个区域 ----
        target_dot = Dot(RIGHT * 3 + UP * 1, color=YELLOW, radius=0.3)
        target_label = Text("目标点", font_size=18, color=YELLOW)
        target_label.next_to(target_dot, UP, buff=0.15)
        target_group = VGroup(target_dot, target_label)

        self.play(
            FadeIn(target_group),
            self.camera.frame.animate.scale(0.3).move_to(target_dot.get_center()),
            run_time=2,
        )
        self.wait(1)

        # ---- 镜头拉回全景 ----
        self.play(
            self.camera.frame.animate.scale(1.0 / 0.3).move_to(ORIGIN),
            run_time=2,
        )
        self.wait(0.5)

        # ---- 镜头跟随移动 ----
        moving_circle = Circle(radius=0.5, color=GREEN, fill_opacity=0.5)
        self.play(FadeIn(moving_circle), run_time=0.5)

        self.play(
            moving_circle.animate.shift(RIGHT * 5),
            self.camera.frame.animate.scale(0.4).move_to(moving_circle.get_center() + RIGHT * 2),
            run_time=2,
        )
        self.wait(0.3)

        self.play(
            moving_circle.animate.shift(UP * 3 + LEFT * 3),
            self.camera.frame.animate.move_to(moving_circle.get_center()),
            run_time=2,
        )
        self.wait(1)

        self.play(FadeOut(VGroup(title, dots, target_group, moving_circle, grid)),
                  run_time=2)
```

**运行结果**：

画面首先展示大场景（30 个彩色圆点散布在坐标网格上），然后镜头推进到黄色目标点（缩放约 3.3 倍），停留 1 秒后拉回全景。接着一个绿色圆圈向右移动，镜头跟随追踪，最终停在圆圈上方。整个流程清晰展示了 `MovingCameraScene` 的三个核心能力：缩放、移动和跟随。

**可能遇到的坑**：

1. **`self.camera.frame.scale(0.3)` 是缩小 frame → 放大画面**：这是新手最易混淆的点。frame 越小，摄像机"视野"越窄，放大倍率越高。`frame.scale(0.5) = 2 倍放大`。
2. **Camera 移动后画面内容可能超出视野**：如果 frame 移到了没有内容的位置，画面会变空白。确保移向有内容的位置。
3. **Camera 移动的 run_time 不宜太长**：镜头移动超过 3 秒会让观众失去耐心。通常 1-2 秒是合适的。

#### 步骤三：综合实战——函数开口变化动画

**步骤目标**：整合公式变换 + 曲线动画 + 镜头推拉，制作一个完整的教学片段。

```python
# scenes/chapter10_parabola_opening.py
from manim import *

class ParabolaOpening(MovingCameraScene):
    def construct(self):
        # ---- 开场：公式展示 ----
        title = Text("二次函数开口变化", font_size=36, color=BLUE)
        title.to_edge(UP, buff=0.3)

        formula = MathTex(r"y = a x^2", font_size=48, color=WHITE)
        formula.next_to(title, DOWN, buff=1.0)

        self.play(Write(title), Write(formula), run_time=2)
        self.wait(0.5)

        # ---- 创建坐标轴和曲线 ----
        axes = Axes(
            x_range=[-5, 5, 1], y_range=[-1, 10, 2],
            x_length=9, y_length=5,
            axis_config={"color": BLUE, "include_tip": True},
        )
        axes.shift(DOWN * 0.5)

        self.play(FadeOut(formula), Create(axes), run_time=1.5)
        self.wait(0.3)

        # 初始曲线 a=1
        a_tracker = ValueTracker(1)
        def get_curve():
            return axes.plot(
                lambda x: a_tracker.get_value() * x ** 2,
                x_range=[-3, 3], color=YELLOW, stroke_width=3,
            )

        curve = always_redraw(get_curve)
        curve_label = always_redraw(lambda: MathTex(
            f"a = {a_tracker.get_value():.1f}",
            font_size=32, color=YELLOW,
        ).next_to(axes.coords_to_point(-2, 8), LEFT, buff=0.2))

        self.play(Create(curve), Write(curve_label), run_time=2)
        self.wait(0.5)

        # ---- a 从 1 变为 2：曲线开口变小 ----
        formula_a2 = MathTex(r"y = 2x^2", font_size=40, color=ORANGE)
        formula_a2.to_corner(UR, buff=0.6)
        self.play(
            TransformMatchingTex(
                MathTex(r"y = x^2", font_size=40, color=YELLOW).move_to(
                    formula_a2.get_center()),
                formula_a2,
            ),
            run_time=1.5,
        )
        self.wait(0.2)

        # 镜头推近 + a 值变化
        self.play(
            a_tracker.animate.set_value(2),
            self.camera.frame.animate.scale(0.6).move_to(axes.coords_to_point(0, 4)),
            run_time=2,
        )
        self.wait(0.5)

        # ---- a 从 2 变为 3：开口更小 ----
        formula_a3 = MathTex(r"y = 3x^2", font_size=40, color=RED)
        formula_a3.move_to(formula_a2.get_center())
        self.play(
            TransformMatchingTex(formula_a2, formula_a3),
            a_tracker.animate.set_value(3),
            run_time=2,
        )
        self.wait(0.5)

        # 镜头拉回全景
        self.play(
            self.camera.frame.animate.scale(1 / 0.6).move_to(ORIGIN),
            run_time=2,
        )
        self.wait(0.5)

        # ---- 收敛结论 ----
        conclusion = Text(
            "|a| 越大，抛物线开口越窄",
            font_size=32, color=GREEN,
        )
        conclusion.next_to(axes, DOWN, buff=0.8)
        self.play(Write(conclusion), run_time=1.5)
        self.play(Indicate(conclusion, color=GREEN), run_time=1.5)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, axes, curve, curve_label,
            formula_a3, conclusion)), run_time=2)
```

**运行命令**：

```bash
manim -pqm scenes/chapter10_parabola_opening.py ParabolaOpening
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 匹配变换 | TransformMatchingTex/Shapes 提升公式类和几何类动画的叙事质量 | 仅限同类型对象（Tex→Tex, VMobject→VMobject） |
| 视线引导 | Indicate/Flash/FocusOn 精确定位观众注意力 | 过度使用会让动画显得"花哨" |
| 镜头控制 | MovingCameraScene 支持推拉移跟 | 镜头移动与内容同步需要手动管理 |
| always_redraw | 实现数值驱动的实时画面更新 | 性能开销大，不适合大量对象 |
| 视觉连贯 | 多种变换组合可打造专业级教学视频 | 缺少"动画编排器"预览功能，需反复渲染调试 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 数学定理证明 | 公式逐步变形推导 |
| 函数分析 | 参数变化对图像的影响 |
| 算法详解 | 代码→数据结构的视觉映射 |
| 产品讲解 | 聚焦重点功能模块 |
| 机理展示 | 从宏观到微观的层层递进 |

**不适用场景**：需要真实 3D 镜头旋转的场景（Camera 无法绕 Z 轴旋转）、需要景深模糊的效果。

### 注意事项

1. **`TransformMatchingTex` 的两个公式 Token 数必须能够匹配**：如果两个公式 token 相差太大（比如 `a` 和 `\begin{bmatrix}...\end{bmatrix}`），匹配会退化，效果接近普通 Transform。建议公式结构保持相似。
2. **`MovingCameraScene` 的 `self.camera.frame` 是一个实际 Mobject**：可以像普通对象一样 `animate`、`set_color`（但不渲染到画面）。注意：如果你同时移动 frame 和内容对象，视觉上会产生"双倍位移"。
3. **`always_redraw` 在 camera 缩放时可能抖动**：因为 `always_redraw` 每次重建对象，可能触发 LaTeX 重编译。建议配合缓存使用。

### 常见踩坑经验

**故障一：`TransformMatchingTex` 报 `IndexError: list index out of range`**

根因：两个 MathTex 的 token 列表长度差太大，匹配算法找不到映射。

解决：在关键位置插入 `\phantom{}` 让两个公式的 token 数相近，或用 `TransformMatchingShapes` 替代。

**故障二：Camera 移动后 `always_redraw` 的对象消失**

根因：`always_redraw` 重建对象时使用了依赖 camera 位置的坐标（如 `to_corner(UR)`），在 camera 移动后坐标系变了。

解决：`always_redraw` 内部使用固定坐标而非相对坐标，或者用 `camera.frame.get_center()` 作为参考。

**故障三：多个 TransformMatchingTex 串行执行时性能暴降**

根因：每个匹配需要执行 `difflib.SequenceMatcher`，复杂度 O(n²)。连续多个长公式会累积耗时。

解决：将多步公式变形合并为单次较大的 Morph，或者用 `ReplacementTransform` 替代不需要精确匹配的步骤。

### 思考题

1. 在 `ParabolaOpening` 的基础上增加 `|a| < 1` 的情况（如 a = 0.5），让开口从窄（a=3）→ 正常（a=1）→ 宽（a=0.5）连续变化，每次变化配合镜头推拉。提示：用 `a_tracker.animate.set_value(0.5)` 驱动曲线平滑变化。

2. 设计一个 `class FocusFlow(Scene)`：在画面上展示 5 个步骤卡片（Step1–Step5），按顺序用 `FocusOn` 聚焦每个卡片，同时镜头推近当前卡片、拉远已完成卡片。要求封装成方法 `focus_step(i)`，自动计算镜头位置。提示：将 5 个卡片存为列表，`focus_step(i)` 中调用 `self.camera.frame.animate.scale(0.5).move_to(cards[i].get_center())`。
