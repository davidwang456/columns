# 第26章：代码演示与技术架构动画

---

## 1. 项目背景

某互联网公司的技术布道师团队接到任务：为即将召开的技术峰会制作一段"消息队列削峰填谷"的科普动画。受众是几千名开发者和架构师，需要在 3 分钟内讲清楚：什么是消息队列、为什么需要它、以及它在高并发场景下如何工作。

布道师小刘先用 Manim 画了服务节点和箭头，但很快发现技术动画跟数学动画有本质差别——技术动画的核心元素不是坐标轴和函数曲线，而是**代码片段**、**流程图**、**架构时序图**和服务节点之间的消息流。

小刘面对的具体问题是：
1. **代码怎么展示**：需要逐行展示一段伪代码，并在特定行高亮、放大、替换。用 `Text` 手动排版太原始——50 行代码要 50 个 Text 对象，行号和缩进全要手动算。
2. **架构图怎么画**：需要画出一个包含六个服务节点、三个数据库和两条消息队列的微服务架构图。手动画圆角矩形和箭头，调整间距耗了一个上午。
3. **时序图怎么动**：客户端→网关→服务端→数据库的消息流需要按照时间顺序展示。第 17 章的 `Succession` 可以串行动画，但节点和消息的坐标关系需要动态维护。

这些痛点指向中级篇的重要技能——**技术动画的工程化表达**。本章要系统讲解 `Code` 对象、流程图/架构图的编排模式和时序动画的状态机设计。

## 2. 剧本式交锋对话

> **场景**：小刘的屏幕上，左侧是 50 行伪代码的 Text 尝试，右侧是一个手动画的架构图，节点像被风吹歪的积木。

**小胖**（拆着一包麻辣豆皮）：

"小刘你这个架构图——你不说我还以为是拼多多砍一刀的界面。六个服务节点有的宽有的窄，间距一会儿大一会儿小，箭头还歪的。你是不是每个节点手动 `move_to` 的？"

**小刘**（揉着发酸的手腕）：

"对。六个节点我算了半天坐标。每次想'在服务 A 和服务 B 之间加一个箭头'，就得重新算两个节点边框的位置。代码写了一百行，只画了六个方块和五根箭头。"

**小白**（调出 Manim 的 `Code` 类文档）：

"技术动画的布局问题，Manim 准备了两个关键工具。

第一个是 `Code` 类——它直接读取源码文件或字符串，渲染出带语法高亮的代码块。代码行的字号、背景色、行号都在参数里控制。高亮某一行只需 `code.background_mobject[line_no].set_color(YELLOW)`。比 50 个 `Text` 对象先进一个时代。

第二个是架构图编排——用 `VGroup` + `arrange` 统一管理节点，节点之间用 `Arrow` 连接，并用 `always_redraw` 绑定箭头位置到节点。改节点位置只需改一次 `arrange`，所有箭头自动跟上。"

**大师**（在白板上画了一个架构图的 VGroup 树）：

"技术架构图的核心数据结构是**节点列表**和**边列表**。把这两者分开管理：

```python
nodes = VGroup(Node("Gateway"), Node("ServiceA"), Node("ServiceB"), Node("DB"))
nodes.arrange(RIGHT, buff=1.0)  # 一行代码完成布局

edges = [(0, 1), (0, 2), (1, 3), (2, 3)]  # 边的索引对
arrows = VGroup()
for u, v in edges:
    arrow = Arrow(nodes[u].get_right(), nodes[v].get_left(), ...)
    arrows.add(arrow)
```

"之后要改布局——改一行 `arrange`；要扩容节点——在列表末尾追加一个 Node。坐标管理被 VGroup 完全接管了。"

> **技术映射**：`Code` 类是 Manim 内置的代码渲染对象，内部调用 Pygments 做语法解析和着色。`Code("file.py", language="python", line_spacing=0.8)` 自动处理行号、缩进和语法高亮。

**小胖**（麻辣豆皮辣得嘶嘶的）：

"那时序图那种'消息从左飞到右'的动态效果呢？节点不动，消息箭头从左往右飞——这`always_redraw`够用吗？"

**大师**：

"时序图的消息动画用 `ValueTracker` 控制'消息进度'最合适。

消息飞行分三步：
1. 消息出现（箭头从发送方生长到接收方）
2. 消息被处理（接收方高亮）
3. 消息消失（箭头淡出）

每一步用一个 `Animation` 封装，三步用 `Succession` 串起来。多段消息（Client→Gateway、Gateway→Service、Service→DB）再用外层 `Succession` 包一层——结构就是第 17 章学的嵌套编排，只是对象变成了技术节点。

我称之为**时序路由表**——用一张表格记录每段消息的发送方、接收方、内容和颜色，然后用循环自动生成动画序列。一种'用数据驱动动画'的方式。"

#### 步骤一：Code 对象——代码高亮与行选择

**步骤目标**：掌握 `Code` 的创建、语法高亮、行选择高亮和背景着色。

```python
# scenes/chapter26_code.py
from manim import *

class CodeHighlight(Scene):
    def construct(self):
        # 从字符串创建代码块
        code_str = '''
import pika

connection = pika.BlockingConnection(
    pika.ConnectionParameters('localhost'))
channel = connection.channel()
channel.queue_declare(queue='orders')

def callback(ch, method, properties, body):
    print(f" [x] Received {body}")
    process_order(body)
    ch.basic_ack(delivery_tag=method.delivery_tag)

channel.basic_consume(
    queue='orders', on_message_callback=callback)
channel.start_consuming()
        '''
        code = Code(
            code=code_str.strip(),
            language="python",
            font="Consolas",
            font_size=20,
            line_spacing=0.6,
            background="rectangle",
            background_config={"stroke_color": GRAY, "stroke_width": 1,
                               "fill_opacity": 0.08},
        )
        code.to_edge(LEFT, buff=0.5).shift(DOWN * 0.2)

        title = Text("消息队列消费者代码（Python）", font_size=28, color=BLUE)
        title.to_edge(UP, buff=0.3)

        self.play(Write(title), FadeIn(code, shift=UP * 0.3), run_time=2)
        self.wait(0.5)

        # 高亮第 3 行（connection 创建）
        self.play(
            code.background_mobject[2].animate.set_color(YELLOW_A).set_opacity(0.25),
            run_time=0.8,
        )
        self.wait(0.8)

        # 高亮第 7-9 行（callback 函数）
        for line_no in [7, 8, 9]:
            self.play(
                code.background_mobject[line_no].animate.set_color(GREEN_A).set_opacity(0.25),
                run_time=0.3,
            )
        self.wait(1)

        self.play(FadeOut(VGroup(title, code)), run_time=1.5)
```

**运行结果**：

代码块以 VS Code 风格的深色背景渲染，Python 语法高亮（关键字蓝色、字符串绿色、函数名黄色等）。第 3 行和 7-9 行依次覆盖半透明色块作为高亮标注。可用 `code.background_mobject[i]` 按行号索引。

#### 步骤二：微服务架构图

**步骤目标**：用 VGroup + arrange 构建可维护的微服务架构图。

```python
# scenes/chapter26_architecture.py
from manim import *

class MicroserviceArchitecture(Scene):
    def construct(self):
        def make_service(name, color=BLUE):
            box = RoundedRectangle(width=2.2, height=1.0, corner_radius=0.12,
                                   color=color, fill_opacity=0.1, stroke_width=2)
            label = Text(name, font_size=20, color=color)
            label.move_to(box.get_center())
            return VGroup(box, label)

        # 节点
        client   = make_service("Client",  BLUE)
        gateway  = make_service("Gateway", YELLOW)
        mq       = make_service("MQ",      ORANGE)
        svc_a    = make_service("Svc-A",   GREEN)
        svc_b    = make_service("Svc-B",   GREEN)
        db       = make_service("DB",      PURPLE)

        # 行排列
        row1 = VGroup(client, gateway, mq).arrange(RIGHT, buff=1.2).shift(UP * 2)
        row2 = VGroup(svc_a, svc_b, db).arrange(RIGHT, buff=1.2).shift(DOWN * 0.5)

        title = Text("微服务消息架构", font_size=30, color=WHITE).to_edge(UP, buff=0.3)
        self.play(Write(title))
        self.play(FadeIn(row1, scale=0.9), FadeIn(row2, scale=0.9), run_time=2)

        # 边：Client→Gateway、Gateway→MQ、MQ→SvcA、MQ→SvcB、SvcA→DB
        arrows = VGroup(
            Arrow(client.get_right(), gateway.get_left(), buff=0.1, color=GRAY),
            Arrow(gateway.get_right(), mq.get_left(), buff=0.1, color=GRAY),
            Arrow(mq.get_bottom(), svc_a.get_top(), buff=0.1, color=GRAY),
            Arrow(mq.get_bottom() + RIGHT*0.3, svc_b.get_top(), buff=0.1, color=GRAY),
            Arrow(svc_a.get_bottom(), db.get_top() + LEFT*0.3, buff=0.1, color=GRAY),
        )
        self.play(LaggedStart(*[Create(a) for a in arrows], lag_ratio=0.15), run_time=2)

        # 标注：削峰填谷
        label_mq = Text("削峰填谷", font_size=20, color=ORANGE)
        label_mq.next_to(mq, UP, buff=0.2)
        self.play(Write(label_mq), mq.animate.set_color(ORANGE), run_time=1.5)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, row1, row2, arrows, label_mq)), run_time=1.5)
```

#### 步骤三：消息队列削峰填谷完整动画

**步骤目标**：整合代码块 + 架构图 + 时序消息流，制作完整技术动画。

```python
# scenes/chapter26_mq_flow.py
from manim import *

class MQFlowAnimation(Scene):
    def construct(self):
        # ---- 阶段1：问题引入 ----
        title = Text("消息队列削峰填谷", font_size=34, color=BLUE, weight=BOLD)
        title.to_edge(UP, buff=0.4)

        # 流量曲线：瞬时峰值 vs 平滑消费
        axes = Axes(x_range=[0, 10, 1], y_range=[0, 200, 50],
                     x_length=8, y_length=3,
                     axis_config={"color": BLUE, "include_tip": True})
        axes.to_edge(DOWN, buff=0.8)

        # 瞬时流量（红色尖峰）
        spike_values = [20,30,50,180,150,90,60,40,30,20]
        spike_line = axes.plot_line_graph(
            x_values=list(range(10)), y_values=spike_values,
            line_color=RED, add_vertex_dots=True,
        )
        spike_label = Text("瞬时流量", font_size=20, color=RED)
        spike_label.next_to(spike_line.get_end(), UR, buff=0.2)

        # 平滑消费（绿色虚线）
        smooth_values = [50]*10
        smooth_line = axes.plot_line_graph(
            x_values=list(range(10)), y_values=smooth_values,
            line_color=GREEN, add_vertex_dots=False,
        )
        smooth_line.use_stroke(color=GREEN, width=3).set_opacity(0.7)
        smooth_label = Text("平滑消费", font_size=20, color=GREEN)
        smooth_label.next_to(axes.c2p(8, 50), DOWN, buff=0.3)

        self.play(Write(title), Create(axes), run_time=2)
        self.play(Create(spike_line), Write(spike_label), run_time=2)
        self.wait(0.3)
        self.play(Create(smooth_line), Write(smooth_label), run_time=2)
        self.wait(1)

        # ---- 阶段2：削峰填谷原理图 ----
        self.play(FadeOut(VGroup(spike_line, spike_label, smooth_line, smooth_label)),
                  run_time=1)

        # 简化版架构图
        box_producer = RoundedRectangle(width=2, height=1.2, corner_radius=0.1,
                                        color=RED, fill_opacity=0.1).shift(LEFT*4+UP*0.5)
        box_mq = RoundedRectangle(width=2, height=1.2, corner_radius=0.1,
                                  color=ORANGE, fill_opacity=0.15)
        box_consumer = RoundedRectangle(width=2, height=1.2, corner_radius=0.1,
                                        color=GREEN, fill_opacity=0.1).shift(RIGHT*4+UP*0.5)

        lbl_prod = Text("生产者", font_size=22, color=RED).move_to(box_producer)
        lbl_mq = Text("消息队列", font_size=22, color=ORANGE).move_to(box_mq)
        lbl_cons = Text("消费者", font_size=22, color=GREEN).move_to(box_consumer)

        arr1 = Arrow(box_producer.get_right(), box_mq.get_left(), buff=0.1, color=GRAY)
        arr2 = Arrow(box_mq.get_right(), box_consumer.get_left(), buff=0.1, color=GRAY)

        self.play(FadeIn(box_producer), Write(lbl_prod), run_time=1)
        self.play(FadeIn(box_mq), Write(lbl_mq), run_time=1)
        self.play(FadeIn(box_consumer), Write(lbl_cons), run_time=1)
        self.play(Create(arr1), Create(arr2), run_time=1.5)

        # 高亮"削峰填谷"关键点
        self.play(Indicate(box_mq, color=YELLOW, scale_factor=1.1), run_time=1.5)

        conclusion = Text("MQ 将瞬时峰值平滑为持续消费", font_size=26, color=GREEN)
        conclusion.to_edge(DOWN, buff=0.6)
        self.play(Write(conclusion), run_time=1.5)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, axes, box_producer, box_mq, box_consumer,
                                  lbl_prod, lbl_mq, lbl_cons, arr1, arr2,
                                  conclusion)), run_time=2)
        self.wait(0.5)
```

**运行命令**：

```bash
manim -pqm scenes/chapter26_mq_flow.py MQFlowAnimation
```

**运行结果**：

两段式技术动画：第一段展示流量曲线（红色瞬时峰值 vs 绿色平滑消费），第二段展示生产者→消息队列→消费者的架构流程图，用高亮强调 MQ 的"削峰"角色。

**可能遇到的坑**：

1. **`Code.background_mobject` 索引从 0 开始**：第一行代码对应 `background_mobject[0]`。空行也会占据一个索引。
2. **架构图中箭头位置在节点缩放后需更新**：`Arrow` 的起点/终点是在创建时确定的。如果之后移动了节点，箭头不会自动更新。方案：在节点移动的同一个 `play` 中重新创建箭头。

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| Code 对象 | 一行代码加载任意语言源码 + 语法高亮 | 不支持自定义着色方案（只能按 Pygments 主题） |
| 架构图 | VGroup + arrange 自动布局，节点增删方便 | 复杂拓扑（交叉边、分层布局）需要手动调坐标 |
| 时序动画 | Succession 嵌套表达消息时间轴 | 消息数量多时 Succession 嵌套层级深 |
| 代码高亮 | background_mobject 按行索引着色 | 单行内无法只着色部分 token |
| 信息密度 | 技术动画可同时展示代码+架构+流程 | 三个元素挤在一起会导致观众认知过载 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 技术峰会演讲 | 架构演进、技术选型、系统设计 |
| 代码教程 | 逐行讲解算法实现 |
| 系统架构审查 | 微服务拓扑、数据流的可视化 |
| 故障复盘 | 故障传播链路的动画演示 |
| 开源项目宣传 | 架构一图胜千言 |

**不适用场景**：实时代码编辑器演示（应使用录制屏幕）、需要精确 UML 规范的技术文档（应用 PlantUML/Graphviz）。

### 注意事项

1. **`Code` 的字号与画面比例**：在 1080p 下 `font_size=20` 的代码约每行 8 像素高，20 行代码占据约 3 个单位高度。注意不要超过画面高度。
2. **架构图的 `arrange` 只能等间距**：如需不等间距布局（如 Gateway 与 MQ 之间要留更大空位），用 `arrange` 后单独 `shift` 调整。
3. **代码文件编码**：`Code` 读取文件时默认用 UTF-8。Windows 上的 GBK 编码文件需在传入前用 Python 提前转码。

### 常见踩坑经验

**故障一：`Code` 背景矩形覆盖了代码行号**

根因：`background="rectangle"` 创建的矩形包围了整个代码区域。行号在左侧，可能被矩形挡住。

解决：`background_config={"buff": 0.1}` 增大矩形内边距。

**故障二：架构图节点对齐后箭头错位**

根因：`arrange` 按节点的包围盒中心对齐，但箭头按 `get_right()`/`get_left()` 取的是 "经过 arrange 前" 的坐标。

解决：在 `arrange` 之后创建箭头，或创建箭头后再做一次 `always_redraw` 绑定。

**故障三：时序动画中消息消失后节点颜色没有恢复**

根因：`Indicate` 动画结束后没有还原对象的颜色——它保留了闪烁结束的颜色。

解决：`Indicate` 后加一个 `.set_color(original_color)` 或在 `Indicate` 的 `AnimationGroup` 中加 `restore`。

### 思考题

1. 封装一个 `class ArchitectureDiagram`：接收 `nodes`（字典：name→position）和 `edges`（列表：(from, to, label)），自动创建节点 VGroup、箭头和边标签。支持 `highlight_node(name)` 和 `highlight_edge(from, to)` 方法。提示：继承 `VGroup`，在 `__init__` 中创建所有元素。

2. 在 `MQFlowAnimation` 中加入消息堆积的可视化：当生产速率高于消费速率时，在 MQ 节点上方显示一个实时增长的数字（代表队列深度），并用红色/黄色/绿色表示健康状态。提示：`DecimalNumber` + `ValueTracker` 控制队列深度。
