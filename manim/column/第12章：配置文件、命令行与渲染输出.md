# 第12章：配置文件、命令行与渲染输出

---

## 1. 项目背景

某教育科技公司的内容生产团队已经用 Manim 制作了 30 集课程动画。随着项目规模膨胀，操作层面的问题集中爆发：

1. **低清预览 vs 高清发布**：开发阶段需要快速预览（480p），生产阶段需要导出 1080p 甚至 4K。每人每次都要手敲 `-ql` / `-qh` / `-qk`，常有同事在 CI 中少打一个字母导致渲染了高清版浪费 20 分钟。

2. **批量渲染噩梦**：课程主管要求"今晚把第 1–30 集全部重新渲染一遍，统一 1080p 30fps"——有人写了一个 `for` 循环的 bash 脚本，漏掉了第 17 集，第 22 集报错没被发现。

3. **配置文件混乱**：三台机器分别有 `manim.cfg`、`custom_manim.cfg` 和根本没有配置文件；同一段代码在 A 机器输出 720p、B 机器输出 1080p，排查了半天才发现是配置文件没同步。

4. **渲染残留**：管理员发现 `media/` 目录已经积攒了 50GB 的历史渲染数据，不知道哪些可以删、哪些是最终产物。

这些痛点的本质是 Manim 项目的**工程化管理**能力薄弱。本章要建立从配置到命令到产物归档的完整工程流，让动画制作从"手工单件"升级为"可批量复现的生产线"。

```mermaid
flowchart TD
    A["manim.cfg"] --> B["全局配置"]
    C["CLI 参数"] --> B
    B --> D["Scene.render()"]
    D --> E["media/ 产出"]
    E --> F["videos/ 视频"]
    E --> G["images/ 截图"]
    E --> H["Tex/ 公式缓存"]
    F --> I["归档 & 发布"]
```

## 2. 剧本式交锋对话

> **场景**：运维小马在服务器上清理 media/ 目录，已经删了 30GB 还看不到底。小胖在旁边吃泡面。

**小胖**（吸溜一口泡面）：

"马哥，你这服务器硬盘比我的手机还满——不就几个动画文件吗，怎么能吃到 50GB？"

**小马**（指着 `du -sh media/*` 的结果）：

"你看——`Tex/` 目录占了 18GB，全是 LaTeX 编译缓存，同一道公式每次渲染都生成一遍 SVG；`videos/` 下每个场景有 480p、720p、1080p 三个版本，历史迭代版本的也都没删。这目录根本就是个黑洞。"

**小白**（已经在翻 Manim 源码中的 CLI 模块）：

"LaTeX 缓存膨胀的原因是 Manim 默认用公式内容的哈希值命名缓存目录。你每次修改公式字符串——哪怕只是改一个空格——哈希值就变了，Manim 会重新编译 LaTeX 并保存到新的缓存目录，旧的不删除。如果你有 200 个公式，改动 5 轮，就是 1000 个缓存目录。"

**大师**（放下手里的运维清单）：

"配置管不好，缓存就是灾难。我给团队定了个规矩——**三条配置铁律**：

铁律一——**全局配置只写一次**：在项目根目录维护 `manim.cfg`，所有机器共用同一份，提交到 Git。禁止每人手写自己的配置。

铁律二——**缓存集中管理**：在 `manim.cfg` 中指定 `media_dir = ./build/media`，和源码目录隔离开，方便统一清理。

铁律三——**输出分版本归档**：最终产物从 `media/` 拷贝到 `output/` 目录并按版本号管理，`media/` 目录只当临时构建区，随时可删除。"

> **技术映射**：Manim 的配置优先级为：CLI 参数 > 环境变量 > manim.cfg > 默认值。`manim.cfg` 是 `configparser` 格式的 INI 文件，支持 `[CLI]`、`[tex]`、`[camera]`、`[directories]` 等段。

**小胖**（突然想到什么）：

"等等，马哥你刚才说三台机器配置不同。那如果我在 Windows 和 macOS 上跑同一份代码，manim.cfg 里的字体路径不一样怎么办？Windows 上是 `C:\Windows\Fonts\simhei.ttf`，macOS 上是 `/System/Library/Fonts/PingFang.ttc`。"

**大师**：

"好问题！这就是平台差异化配置的经典难题。解决思路有三条：

方案一——**相对路径 + 项目内置字体**：把字体文件直接放到 `assets/fonts/` 下，提交到 Git（注意字体授权）。`manim.cfg` 中写相对路径，所有平台通用。

方案二——**条件配置**：用 Python 脚本生成 `manim.cfg` 前检测平台并写入不同值。比如在 `build.py` 导入前动态检测 `platform.system()` 并设置 config。

方案三——**环境变量注入**：`manim.cfg` 中写 `${FONT_PATH}`，CI 环境中通过环境变量注入实际路径。

生产环境建议方案一——把关键字体打包到项目中，消除平台依赖。"

> **技术映射**：Manim 的配置文件不支持变量替换或条件逻辑，它是静态 INI 格式。平台差异需要在 Python 层（而非配置层）处理。

**小胖**（放下泡面碗）：

"那批量渲染呢？我们每次手工敲 `manim -pqh scene.py SceneA SceneB SceneC`，敲错了就白等半小时。"

**大师**：

"批量渲染要用 Python 脚本驱动。最简单的方案——一个 `build.py`：扫描 `scenes/` 目录下所有 `.py` 文件，用正则找出所有 `class.*\(Scene\)` 定义的场景类名，然后用 `subprocess` 逐个调用 `manim` 命令。高级版可以加：失败重试、进度条、耗时统计、产物校验。

不过——`build.py` 不是一个随便写的 for 循环。它必须处理三种情况：
1. 单个场景渲染失败 → 记录出错日志，继续渲染下一个
2. 全部场景渲染完成 → 汇总成功/失败统计
3. CI 环境 vs 本地环境 → 用不同的 `-q` 参数"

#### 步骤二：编写自动化构建脚本

**步骤目标**：创建 `build.py`，支持单场景和批量渲染，含错误处理和耗时统计。

```python
# build.py
"""
Manim 批量渲染脚本
用法:
  python build.py                    # 渲染全部场景
  python build.py --quality h        # 高清
  python build.py --scene HelloManim # 单场景
  python build.py --dry-run          # 只列出场景，不渲染
"""
import subprocess
import re
import time
import sys
from pathlib import Path

SCENES_DIR = Path("scenes")
OUTPUT_DIR = Path("build/media")

def discover_scenes(directory: Path):
    """扫描 scenes/ 目录，发现所有 Scene 类名"""
    scenes = []
    for py_file in sorted(directory.glob("*.py")):
        content = py_file.read_text(encoding="utf-8")
        # 匹配 class Xxx(Scene) 或 class Xxx(MovingCameraScene) 等
        found = re.findall(r"class\s+(\w+)\(\w*Scene\w*\)", content)
        for name in found:
            scenes.append((py_file, name))
    return scenes

def render_scene(file_path, scene_name, quality="m"):
    """渲染单个场景，成功返回 True"""
    cmd = [
        sys.executable, "-m", "manim",
        "-q", quality,
        "--media_dir", str(OUTPUT_DIR),
        str(file_path), scene_name,
    ]
    start = time.time()
    result = subprocess.run(cmd, capture_output=True, text=True)
    elapsed = time.time() - start
    if result.returncode == 0:
        print(f"  [OK] {scene_name} ({elapsed:.1f}s)")
        return True
    else:
        print(f"  [FAIL] {scene_name} ({elapsed:.1f}s)")
        print(f"  stderr: {result.stderr[-300:]}")
        return False

def main():
    quality = "m"
    target_scene = None
    dry_run = False

    args = sys.argv[1:]
    i = 0
    while i < len(args):
        if args[i] == "--quality" and i + 1 < len(args):
            quality = args[i + 1]; i += 2
        elif args[i] == "--scene" and i + 1 < len(args):
            target_scene = args[i + 1]; i += 2
        elif args[i] == "--dry-run":
            dry_run = True; i += 1
        else:
            i += 1

    scenes = discover_scenes(SCENES_DIR)
    print(f"发现 {len(scenes)} 个场景")

    if dry_run:
        for file_path, name in scenes:
            print(f"  {file_path} -> {name}")
        return

    if target_scene:
        scenes = [(f, n) for f, n in scenes if n == target_scene]
        if not scenes:
            print(f"未找到场景: {target_scene}")
            return

    total = len(scenes)
    ok = 0
    total_start = time.time()
    for file_path, name in scenes:
        print(f"[{ok + 1}/{total}] {name}")
        if render_scene(file_path, name, quality):
            ok += 1

    total_elapsed = time.time() - total_start
    print(f"\n完成: {ok}/{total} 成功, 耗时 {total_elapsed:.1f}s")
    sys.exit(0 if ok == total else 1)

if __name__ == "__main__":
    main()
```

**运行命令**：

```bash
python build.py                    # 批量渲染所有场景，720p
python build.py --quality l        # 低清快速预览
python build.py --dry-run           # 只列出，不渲染
python build.py --scene HelloManim # 只渲染指定场景
```

#### 步骤三：产物管理与归档

**步骤目标**：编写一个产物检查和归档脚本。

```python
# scripts/archive.py
"""
将 build/media 中的最终产物归档到 output/ 目录
"""
import shutil
from pathlib import Path

BUILD_DIR = Path("build/media")
OUTPUT_DIR = Path("output")

def archive_version(version: str):
    """将当前 build 产物归档为指定版本"""
    dest = OUTPUT_DIR / version
    if dest.exists():
        print(f"警告: {dest} 已存在，跳过")
        return
    dest.mkdir(parents=True)

    # 只拷贝 .mp4 文件（最终视频）
    mp4_count = 0
    for mp4 in BUILD_DIR.rglob("*.mp4"):
        rel_path = mp4.relative_to(BUILD_DIR)
        target = dest / rel_path
        target.parent.mkdir(parents=True, exist_ok=True)
        shutil.copy2(mp4, target)
        mp4_count += 1
        print(f"  {rel_path} ({mp4.stat().st_size // 1024 // 1024}MB)")

    print(f"\n归档完成: {mp4_count} 个 MP4 文件 → {dest}")

def clean_build():
    """清理 build 缓存和低清预览"""
    for pattern in ["480p15", "480p30", "tex_cache"]:
        p = BUILD_DIR / "**" / pattern
        for d in list(BUILD_DIR.rglob(pattern)):
            if d.is_dir():
                shutil.rmtree(d)
                print(f"  已删除: {d}")
    print("清理完成")

if __name__ == "__main__":
    import sys
    if len(sys.argv) > 1:
        archive_version(sys.argv[1])
    else:
        print("用法: python scripts/archive.py v1.0.0")
```

### 3.3 完整代码清单

```python
# 扩展 build.py —— 带多环境配置支持

import os
import subprocess
import sys
from pathlib import Path

PROJECT_ROOT = Path(__file__).parent

# 多环境配置
ENV_CONFIGS = {
    "dev":  {"quality": "l", "preview": False, "frame_rate": 15, "background_color": "#0D1117"},
    "prod": {"quality": "h", "preview": False, "frame_rate": 30, "background_color": "#0D1117"},
    "ci":   {"quality": "h", "preview": False, "frame_rate": 30, "background_color": "#0D1117"},
}

def render_scene_with_env(scene_file, scene_name, env="dev"):
    """根据环境名选择渲染参数"""
    cfg = ENV_CONFIGS.get(env, ENV_CONFIGS["dev"])
    cmd = [
        sys.executable, "-m", "manim",
        "-q", cfg["quality"],
        "--frame_rate", str(cfg["frame_rate"]),
        "--background_color", cfg["background_color"],
        "-o", f"{scene_name}_{env}",
        str(Path("scenes") / scene_file),
        scene_name,
    ]
    if not cfg["preview"]:
        cmd.remove("--preview") if "--preview" in cmd else None
    
    # 从环境变量读取额外配置
    extra_args = os.environ.get("MANIM_EXTRA_ARGS", "").split()
    cmd.extend(extra_args)
    
    result = subprocess.run(cmd, capture_output=True, text=True, cwd=PROJECT_ROOT)
    return result.returncode == 0

# 用法示例
if __name__ == "__main__":
    env = sys.argv[1] if len(sys.argv) > 1 else "dev"
    print(f"使用环境配置: {env} -> {ENV_CONFIGS.get(env)}")
    render_scene_with_env("hello_manim.py", "HelloManim", env)
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 配置管理 | manim.cfg 单文件控制全局，支持团队共享 | 配置项分散在多个段中，新手不知道某选项属于哪个段 |
| CLI 参数 | -q/l/m/h 快速切换质量，-p 自动预览 | 参数名短，记忆成本高 |
| 批量渲染 | Python 脚本可扩展（失败重试、并发等） | subprocess 调用开销大，同等条件下比 click CLI 慢 |
| 产物管理 | 输出目录统一，归档脚本自动化 | 文件命名依赖 Manim 的内部规则，不能自定义 |
| 缓存机制 | LaTeX 缓存跳过已编译公式 | 缓存不自动清理，长期积累 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 课程批量生产 | 章节化课程的批量渲染 |
| CI/CD 流水线 | GitHub Actions / Jenkins 中自动渲染 |
| 多环境协作 | 统一配置文件保证团队输出一致 |
| 版本归档 | 按课程版本号归档 MP4 |
| 快速迭代 | -pql 快速预览 → -pqh 高清确认 |

**不适用场景**：需要精细逐帧控制的实时交互应用（使用 ManimGL 的预览模式）、需要自定义 FFmpeg 编码参数的场景（当前 CLI 不支持）。

### 注意事项

1. **CLI 优先级覆盖**：`manim -ql scene.py -qh` 后面 `-qh` 会覆盖 `-ql`。Manim 的 CLI 参数按出现顺序处理，后者覆盖前者。
2. **`--media_dir` 的路径计算**：如果指定了绝对路径的 `media_dir`，Manim 仍然会在内部按 `module_name/scene_name/quality/` 的子目录结构组织输出。不是"所有文件平铺在指定目录"。
3. **渲染过程中的 Ctrl+C**：`build.py` 使用 `subprocess.run`，Ctrl+C 是否能优雅退出取决于操作系统。Linux 下子进程通常会收到 SIGINT，Windows 下可能不会。

### 常见踩坑经验

**故障一：自定义 `manim.cfg` 不生效**

根因：Manim 按以下顺序查找配置：当前目录 > 用户目录 > 默认值。如果你的 `manim.cfg` 不在当前工作目录，Manim 会使用用户目录的配置文件。

解决：始终在项目根目录执行 `manim` 命令，或用 `--config_file path/to/manim.cfg` 显式指定。

**故障二：批量渲染时 LaTeX 报错但单场景正常**

根因：批量渲染时 `subprocess` 的工作目录可能不是项目根目录，导致 LaTeX 找不到自定义模板。

解决：`subprocess.run(cmd, cwd=PROJECT_ROOT)` 显式设置工作目录。

**故障三：`build.py` 捕获不到 Manim 失败**

根因：Manim 在部分错误下返回码为 0（如 `--version` 或渲染缓存命中但无输出）。

解决：不仅检查 `returncode`，还需检查输出目录下是否真的生成了 `.mp4` 文件。

### 思考题

1. 改造 `build.py`，增加 `--workers N` 参数，支持多进程并行渲染。每个进程渲染一个场景，先到先得。提示：使用 `concurrent.futures.ProcessPoolExecutor`。

2. 设计一个 `class BuildConfig` 管理多环境配置。例如：`BuildConfig("dev")` 使用 `-ql --preview`，`BuildConfig("prod")` 使用 `-qh`，`BuildConfig("ci")` 使用 `-qh --no_preview`。提示：在 Python 中定义字典 `configs`，按环境名选择参数。
