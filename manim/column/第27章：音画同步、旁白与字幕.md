# 第27章：音画同步、旁白与字幕

---

## 1. 项目背景

某教育科技公司已经用 Manim 制作了 20 集数学动画课程。前 20 集都是"纯画面动画"——没有声音、没有旁白、没有字幕。学生需要一边看动画一边对照 PDF 讲义才能理解内容。

教研组收到学生反馈——"如果能像 3Blue1Brown 那样有配音讲解，一边看一边听，效果会好很多。"于是团队决定为第 21 集添加旁白、背景音乐和字幕。

执行过程却远不如预期顺利：

1. **音画不同步**：旁白在动面前讲完，或动画播完了旁白还在说——原因是旁白的语速和动画的 `run_time` 对不上。
2. **字幕格式混乱**：有人手动写 `.srt` 文件，时间码全靠目测；有人用剪映导出字幕，再手动校准。
3. **音频格式兼容**：有人录的旁白是 `.wav`，有人是 `.mp3`，有人是 `.m4a`。塞进 Manim 后有的能播有的不能。
4. **重录噩梦**：旁白老师读了 5 遍第 3 段的台词，每次都要重新对齐动画时间码。

这个痛点指向了 Manim 的**音视频整合能力**——`add_sound` 的音频导入、旁白脚本与动画节奏的协同设计、以及 `manim-voiceover` 等自动化工具的使用。

## 2. 剧本式交锋对话

> **场景**：剪辑师小梅的屏幕上同时开着 Audacity（音频编辑）、VS Code（Manim 代码）和剪映（字幕时间轴）。三个窗口都在疯狂闪烁。

**小胖**（啃着一根鸭脖）：

"小梅你这个工作台像航空母舰指挥舱——三个屏幕三个软件，你是在做动画还是在玩三屏联动的游戏？就不能全部在 Manim 里搞定吗？"

**小梅**（头发已经抓乱了）：

"我也想啊。Manim 的 `add_sound` 我只能把音频文件塞进去，但闷头塞进去之后不知道它在哪一秒开始播——跟动画完全对不上。我只能在剪映里对着成片手动打时间码。"

**小白**（已经装好了 `manim-voiceover` 插件）：

"你的痛点是**音画同步的自动化**。Manim 官方的 `add_sound` 只是最基础的 FFmpeg 混音——你把音频文件路径塞进去，渲染成片时 FFmpeg 把它混到音轨里。它不会帮你做任何'对齐'。

真正的解决方案是 **`manim-voiceover` 插件**——它把配音录制、时间码提取、字幕生成和自动节奏控制整合成一个工作流：

```python
from manim_voiceover import VoiceoverScene
from manim_voiceover.services.recorder import RecorderService

class MyScene(VoiceoverScene):
    def construct(self):
        self.set_speech_service(RecorderService())
        with self.voiceover("圆的面积等于 π 乘以半径的平方") as tracker:
            self.play(Write(formula), run_time=tracker.duration)
```

"`tracker.duration` 就是这个旁白的实际时长——你把它直接赋给 `run_time`，动画和旁白就自动同步了！"

**大师**：

"小白说的 `manim-voiceover` 是专业级解决方案。但如果你不想引入额外依赖，原生的手动对齐也有套路——

第一步：把旁白录成一段 MP3，在 Audacity 里标注每个段落的起止时间码。

第二步：用 Python 把时间码读出来，映射为 Manim 的 `wait()` 和 `run_time`：

```python
segments = [
    ("0:00", "0:06", "标题出现"),
    ("0:07", "0:18", "圆绘制"),
    ("0:19", "0:35", "扇形切分"),
]
```

第三步：在每个 `play` 和 `wait` 里按时间表精确设定 `run_time`。虽然手工但可控。"

> **技术映射**：`manim-voiceover` 通过 `tracker.duration` 获取每段配音的精确时长（秒），自动对齐动画节奏。支持的 TTS 包括 Azure、Google Cloud、Coqui AI 以及本地录音。

**小胖**（鸭脖啃得只剩骨头）：

"那字幕呢？字幕文件和时间码怎么生成？总不能一刀一刀手切吧。"

**小白**：

"字幕有两个工作流：

**工作流一（手动）**：写好 `.srt` 文件，用剪映或 ArcTime 在视频成片上打时间码，导出 `.srt`，再烧录到视频里。

**工作流二（自动化）**：用 `manim-voiceover` 的 `--create-subtitles` 参数自动生成 `.srt`。它通过语音识别 API（如 Whisper）或者 TTS 服务内部的时间戳自动生成每句字幕的时间码。

烧录到视频推荐用 FFmpeg 命令：`ffmpeg -i video.mp4 -vf subtitles=subtitle.srt output.mp4`。"

## 3. 项目实战

### 3.1 环境准备

#### 补充对话：专业级音频工作流

**小梅**（继续翻着 Audacity 的时间轴）：

"你们说的这些我大概懂了。但实际工作中，旁白老师录 3 遍，第 1 遍节奏太慢，第 2 遍嘴瓢了，第 3 遍才过关。每次重录我都要重新对齐动画时间码——这过程有没有更智能的方案？"

**大师**：

"有。专业音画同步的工作流是——**先定动画，后配旁白**。而不是反过来。

具体流程：
1. 先用 TTS（文字转语音）生成临时旁白，渲染第一版'配音版动画'。
2. 旁白老师看着这个临时版本，跟着画面的节奏朗读正式旁白。这样语速和情绪自然跟着画面走。
3. 录制正式旁白后，用 Python 脚本提取每句旁白的时间戳（通过 silence detection 或 Whisper 语音识别），自动更新动画的 run_time 和 wait。

这个流程的核心优势是——**画面是锚点**，旁白围绕画面展开，而不是旁白是锚点让画面去追。"

> **技术映射**：`pydub` 库的 `AudioSegment.detect_silence()` 可检测音频中的静音段，自动分割出一句句旁白的时间码。

```bash

```

### 3.2 分步实现

> **本章实战目标**：为一段 60 秒的数学动画添加旁白、背景音乐和字幕，实现音画同步。

---

#### 步骤一：手动 add_sound 音画同步

**步骤目标**：掌握 `add_sound` 的使用和手动时间码对齐方法。

```python
# scenes/chapter27_audio_manual.py
from manim import *

class AudioSyncManual(Scene):
    def construct(self):
        # 1. 标题 (0:00 - 0:06)
        title = Text("勾股定理的证明", font_size=40, color=BLUE, weight=BOLD)
        self.play(Write(title), run_time=2)
        self.wait(0.5)

        # 2. 直角三角形 (0:07 - 0:18)
        tri = Polygon(
            LEFT * 2 + DOWN, RIGHT * 2 + DOWN,
            LEFT * 2 + UP * 1.5,
            color=YELLOW, fill_opacity=0.15, stroke_width=2,
        )
        a_label = MathTex("a", font_size=36, color=RED).next_to(tri.get_left(), LEFT)
        b_label = MathTex("b", font_size=36, color=GREEN).next_to(tri.get_bottom(), DOWN)
        c_label = MathTex("c", font_size=36, color=ORANGE).move_to(
            (tri.get_right() + tri.get_top()) / 2 + UR * 0.2)

        self.play(FadeOut(title), run_time=0.5)
        self.play(Create(tri), run_time=1.5)
        self.play(Write(a_label), Write(b_label), Write(c_label), run_time=2)
        self.wait(1)

        # 3. 面积推导 (0:19 - 0:45)
        formula = MathTex(r"a^2", r"+", r"b^2", r"=", r"c^2", font_size=48)
        formula[0].set_color(RED)
        formula[2].set_color(GREEN)
        formula[4].set_color(ORANGE)
        formula.next_to(tri, DOWN, buff=1.2)

        self.play(Write(formula), run_time=3)
        self.wait(1)

        # 高亮结论
        box = SurroundingRectangle(formula, color=YELLOW, buff=0.15)
        self.play(Create(box), run_time=1.5)
        self.wait(2)

        # 结束 (0:46 - 0:60)
        self.play(FadeOut(VGroup(tri, a_label, b_label, c_label, formula, box)), run_time=2)
        self.wait(0.5)

        # 添加背景音乐（需先将 bgm.mp3 放入 assets/audio/ 目录）
        # self.add_sound("assets/audio/bgm.mp3", time_range=(0, 60))
```

**运行命令**：

```bash
manim -pqm scenes/chapter27_audio_manual.py AudioSyncManual
```

**`add_sound` 使用注意事项**：

- `add_sound(file_path, time_range=(start, end))` 将音频片段混入最终视频的对应时间段。
- `time_range` 省略时默认播放完整音频文件。
- 音频格式要求 FFmpeg 能解码即可（MP3、WAV、OGG 都支持）。

#### 步骤二：用 manim-voiceover 自动化

**步骤目标**：使用 `VoiceoverScene` + `RecorderService` 录制旁白并自动对齐。

```python
# scenes/chapter27_voiceover.py
from manim import *
from manim_voiceover import VoiceoverScene
from manim_voiceover.services.recorder import RecorderService

class VoiceoverPythagorean(VoiceoverScene):
    def construct(self):
        # 设置录音服务（会打开系统麦克风）
        self.set_speech_service(RecorderService())

        # 标题段——与旁白同步
        with self.voiceover("今天我们来学习勾股定理的几何证明。") as tracker:
            title = Text("勾股定理的证明", font_size=40, color=BLUE, weight=BOLD)
            self.play(Write(title), run_time=tracker.duration)

        # 三角形段
        with self.voiceover("首先，我们画一个直角三角形，三个边分别称为 a、b、c。") as tracker:
            tri = Polygon(LEFT*2+DOWN, RIGHT*2+DOWN, LEFT*2+UP*1.5,
                          color=YELLOW, fill_opacity=0.15, stroke_width=2)
            self.play(Create(tri), run_time=tracker.duration)

        # 公式段
        with self.voiceover(
            "根据勾股定理，两条直角边的平方和等于斜边的平方，"
            "即 a 的平方加 b 的平方等于 c 的平方。"
        ) as tracker:
            formula = MathTex(r"a^2", r"+", r"b^2", r"=", r"c^2", font_size=48)
            formula[0].set_color(RED); formula[2].set_color(GREEN); formula[4].set_color(ORANGE)
            formula.next_to(tri, DOWN, buff=1.2)
            self.play(Write(formula), run_time=tracker.duration)

        self.wait(1)
        self.play(FadeOut(VGroup(tri, formula)), run_time=1)
```

#### 步骤三：字幕生成与烧录

**步骤目标**：用 FFmpeg 将字幕文件烧录到动画中。

使用 `manim-voiceover` 自动生成字幕：

```bash
# 渲染时生成字幕文件
manim -pqh scenes/chapter27_voiceover.py VoiceoverPythagorean --create-subtitles

# 用 FFmpeg 烧录字幕
ffmpeg -i media/videos/chapter27_voiceover/2160p60/VoiceoverPythagorean.mp4 \
       -vf "subtitles=media/subtitles/VoiceoverPythagorean.srt" \
       -c:a copy output_with_subtitles.mp4
```

如果手动创建字幕文件（`.srt` 格式）：

```srt
1
00:00:00,000 --> 00:00:06,000
今天我们来学习勾股定理的几何证明。

2
00:00:07,000 --> 00:00:18,000
首先，我们画一个直角三角形，三个边分别称为a、b、c。

3
00:00:19,000 --> 00:00:45,000
根据勾股定理，a的平方加b的平方等于c的平方。
```

**可能遇到的坑**：

1. **音频文件路径**：`add_sound` 从 Manim 执行目录的相对路径加载。使用绝对路径或确保在项目根目录运行 `manim`。
2. **录音设备选择**：`RecorderService` 默认使用系统默认麦克风。在嘈杂环境中，使用 TTS 服务（如 Azure TTS）替代录音可获得更清晰的旁白。
3. **subtitle 烧录时间码**：`.srt` 中的时间码必须与视频的实际时间线一致。如果动画的 `run_time` 和 `wait` 修改过，需要重新校准时码。

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| add_sound | 简单直接，一行代码混音 | 无音画同步机制，需手动计时 |
| manim-voiceover | tracker.duration 自动对齐 | 引入额外依赖，录音质量依赖环境 |
| TTS 服务 | Azure/Google TTS 旁白稳定可重复 | 费用和网络依赖，中文语调有时生硬 |
| 字幕生成 | 自动生成 .srt（含准确时间码） | 语音识别在嘈杂音频或多音字上可能出错 |
| 烧录字幕 | FFmpeg 一行命令嵌入硬字幕 | 硬字幕无法关闭，需提供软字幕版本 |

### 适用场景

| 场景 | 说明 |
|------|------|
| MOOC 课程 | 标准配音的教学视频 |
| 产品演示 | 有旁白解说的功能演示 |
| 技术分享 | 带解说的架构讲解视频 |
| 有声绘本 | 画面+朗读的儿童教育内容 |
| 广告宣传 | 配音+音效+画面的促销短片 |

**不适用场景**：需要多语言配音和切换字幕的国际化视频（建议用软字幕 + 播放器切换，而非硬烧录）、需要立体声空间定位的 VR 音频。

### 注意事项

1. **`add_sound` 的时间范围是相对于视频起点**：`time_range=(5, 10)` 表示音频从视频的第 5 秒开始播放到第 10 秒。不指定则完整播放。
2. **TTS 服务的网络延迟**：首次调用 Azure TTS 可能需要 1-3 秒的 API 响应时间。在渲染脚本中设置合理的超时。
3. **字幕字符集**：`.srt` 文件必须用 UTF-8 编码。Windows 记事本保存的 `.srt` 可能默认是 GBK，导致中文乱码。

### 常见踩坑经验

**故障一：渲染后视频没有声音**

根因：`add_sound` 需要 FFmpeg 支持对应音频格式。某些 FFmpeg 编译版本缺少 MP3/AAC 解码器。

解决：用 `ffmpeg -codecs | grep mp3` 检查解码器是否安装。或提前将音频转为 WAV 格式。

**故障二：旁白录了 5 分钟后发现音量太小**

根因：麦克风增益不够，或环境噪音掩盖了人声。

解决：在 Audacity 中做"归一化"（Effect → Normalize）提升音量到 -3dB。或录制时离麦克风 10-15 厘米。

**故障三：`manim-voiceover` 渲染中途卡住**

根因：RecorderService 在等待用户录音但麦克风权限未授权。

解决：检查操作系统的麦克风隐私设置，确保终端程序有录音权限。
