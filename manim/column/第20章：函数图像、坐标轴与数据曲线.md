# 第20章：函数图像、坐标轴与数据曲线

## 1. 项目背景

某数据科学培训团队正在制作一套"机器学习基础"视频课程。第一集的主题是"梯度下降"——需要在坐标轴上展示损失函数曲面、参数迭代路径和损失值的变化曲线。课程编辑小杨的任务是把讲师描述的计算过程转化为可视动画。

小杨在第 6 章学过基础的 `Axes` 和 `plot`，但这个场景的要求远超基础用法：

1. **多坐标系并排**：左边是参数空间的等高线图，右边是损失函数随迭代次数的下降曲线。两个 Axes 的尺度、范围和标注完全独立。
2. **动态采样**：参数点在迭代过程中位置变化，需要动态标注坐标值。
3. **面积和填充**：需要在曲线上标注"误差面积"——用半透明色填充曲线下方的区域。
4. **参数空间映射**：参数点 (w, b) 在参数空间移动的同时，损失曲线上的点也要同步下降——这是两个 Axes 之间的联动。

这些需求正是 `Axes` 的高级玩法。本章要系统地讲解多 Axes 布局、`plot_parametric_curve`、`get_area` 面积填充、`coords_to_point` 与 `point_to_coords` 的双向转换，以及多坐标系联动。

## 2. 剧本式交锋对话

> **场景**：小杨的屏幕上，左边等高线、右边损失曲线、中间好多标注箭头——但处处都在"错位"。

**小胖**（撕开一根烤肠）：

"小杨你这个画面挺高级——像航天控制中心的多屏幕。但左边这个红色的点明明在等高线上，右边对应位置的损失值却对不上。你这个是双坐标系不同步了吧？"

**小杨**（焦头烂额）：

"对！左边的 Axes 范围是 x [0, 10] y [0, 5]，右边是 x [0, 100] y [0, 20]。我在左边 `coords_to_point(3, 4)` 得到画面坐标，但不知道怎么把它映射到右边的 Axes 上。"

**小白**（画了一张双 Axes 映射图）：

"你混淆了**画面坐标**和**数学坐标**。`coords_to_point` 只负责在**同一个** Axes 的内部做数学→画面转换。不同 Axes 之间没有直接映射——你得先 `point_to_coords` 把画面坐标转回数学坐标，再去另一个 Axes 里 `coords_to_point`。"

```python
# 错误：跨越两个 Axes 直接用画面坐标
y_wrong = axes2.coords_to_point(*axes1.coords_to_point(3, 4)[:2])

# 正确：先回数学坐标再转发
math_x, math_y = axes1.point_to_coords(screen_point)
screen_on_axes2 = axes2.coords_to_point(math_x, math_y)
```

**大师**：

"小白说的对。我来补充——多 Axes 联动的最佳实践是用**一个公共的数学状态**来驱动两个 Axes 的可视化。

在梯度下降场景里，公共状态就是 `(w, b, loss)` 三个数值。左边的 Axes 每次根据 `(w, b)` 更新参数点的位置，右边的 Axes 根据 `(iteration, loss)` 更新损失曲线上的点。两边都是从同一组 `ValueTracker` 拿数据，自然同步。"

> **技术映射**：`Axes` 的核心方法是 `coords_to_point(x, y)`（数学→画面）和 `point_to_coords(p)`（画面→数学）。前者用于放置 Mobject，后者用于读取鼠标/点击位置。

**小胖**（举着烤肠）：

"还有一个问题——面积填充怎么画？我想在损失曲线下方涂一层半透明的红色，表示'误差'。"

**小白**：

"用 `axes.get_area(graph, x_range=[...])`。它返回一个填充了颜色的 VMobject，默认覆盖曲线到 x 轴之间的区域。更灵活的是 `axes.get_riemann_rectangles()`——画出黎曼矩形来近积分。前者适合展示面积概念，后者适合讲解积分原理。"

**大师**（补充）：

"面积填充还有一个参数容易被忽略——`bounded_graph`。如果两条曲线之间的区域有意义（比如预测值和真实值之间的差距），可以用 `axes.get_area(curve1, bounded_graph=curve2)` 画两条曲线之间的区域。"

#### 步骤一：多 Axes 布局与坐标映射

**步骤目标**：掌握两个 Axes 的并排布局和 coordinate 转换。

```python
# scenes/chapter20_multi_axes.py
from manim import *

class MultiAxesDemo(Scene):
    def construct(self):
        # ---- 左 Axes：参数空间 ----
        left_axes = Axes(
            x_range=[-1, 6, 1], y_range=[-2, 10, 2],
            x_length=5, y_length=5,
            axis_config={"color": BLUE, "include_tip": True},
        )
        left_axes.to_edge(LEFT, buff=0.8).shift(DOWN * 0.2)
        left_title = Text("参数空间 (w, b)", font_size=22, color=BLUE)
        left_title.next_to(left_axes, UP, buff=0.2)

        # ---- 右 Axes：损失函数 ----
        right_axes = Axes(
            x_range=[0, 50, 10], y_range=[0, 10, 2],
            x_length=5, y_length=5,
            axis_config={"color": GREEN, "include_tip": True},
        )
        right_axes.to_edge(RIGHT, buff=0.8).shift(DOWN * 0.2)
        right_title = Text("损失下降曲线", font_size=22, color=GREEN)
        right_title.next_to(right_axes, UP, buff=0.2)

        self.play(Create(left_axes), Create(right_axes),
                  Write(left_title), Write(right_title), run_time=2)

        # 映射演示：取左边 Axes 上一个数学坐标，转换后在右侧标注
        math_x, math_y = 3, 4
        left_point_screen = left_axes.coords_to_point(math_x, math_y)
        left_dot = Dot(left_point_screen, color=RED)
        left_label = MathTex(f"({math_x},{math_y})", font_size=20, color=RED)
        left_label.next_to(left_dot, UR, buff=0.1)

        # 反向读取：确认转换正确
        read_x, read_y = left_axes.point_to_coords(left_point_screen)

        self.play(FadeIn(left_dot), Write(left_label), run_time=1.5)
        self.wait(1.5)

        self.play(FadeOut(VGroup(left_axes, right_axes, left_title, right_title,
                                  left_dot, left_label)), run_time=1.5)
```

#### 步骤二：面积填充与曲线标注

**步骤目标**：掌握 `get_area` 和曲线下方的面积标注。

```python
class AreaFillDemo(Scene):
    def construct(self):
        axes = Axes(
            x_range=[0, 5, 1], y_range=[0, 6, 1],
            x_length=8, y_length=5,
            axis_config={"color": BLUE, "include_tip": True},
        ).shift(DOWN * 0.3)

        title = Text("曲线下方的面积", font_size=32, color=BLUE).to_edge(UP, buff=0.4)
        self.play(Write(title), Create(axes), run_time=1.5)

        # 曲线
        graph = axes.plot(lambda x: x ** 0.5 + 1, x_range=[0, 4],
                          color=YELLOW, stroke_width=3)

        # 填充曲线下方面积
        area = axes.get_area(graph, x_range=[0.5, 3.5],
                             color=YELLOW, opacity=0.3)

        self.play(Create(graph), run_time=2)
        self.wait(0.3)

        # 面积填充动画
        self.play(FadeIn(area), run_time=1.5)
        self.wait(0.5)

        # 黎曼矩形近似
        rects = axes.get_riemann_rectangles(
            graph, x_range=[0.5, 3.5], dx=0.4,
            color=ORANGE, fill_opacity=0.5, stroke_width=1,
        )
        self.play(ReplacementTransform(area, rects), run_time=2)
        self.wait(0.5)

        # 标注：矩形逼近面积
        note = Text("用矩形面积之和近似积分", font_size=22, color=ORANGE)
        note.next_to(axes, DOWN, buff=0.5)
        self.play(Write(note), run_time=1)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, axes, graph, rects, note)), run_time=1.5)
```

#### 步骤三：梯度下降动画

**步骤目标**：展示一个二次损失函数的梯度下降过程，包含参数空间路径和损失下降曲线联动。

```python
# scenes/chapter20_gradient_descent.py
from manim import *
import numpy as np

class GradientDescent(Scene):
    def construct(self):
        # ---- 左上：参数空间等高线 ----
        contour_axes = Axes(
            x_range=[-3, 3, 1], y_range=[-3, 3, 1],
            x_length=5, y_length=4.5,
            axis_config={"color": BLUE, "include_tip": False},
        )
        contour_axes.to_corner(UL, buff=0.6).shift(DOWN * 0.2)
        contour_title = Text("参数空间", font_size=22, color=BLUE)
        contour_title.next_to(contour_axes, UP, buff=0.1)

        # 等高线（同心圆简化版）
        circles = VGroup()
        for r in np.arange(0.5, 3.5, 0.5):
            c = Circle(
                radius=r * contour_axes.x_length / (contour_axes.x_range[1] - contour_axes.x_range[0]),
                color=BLUE, stroke_opacity=0.2, stroke_width=1,
            )
            c.move_to(contour_axes.coords_to_point(0, 0))
            circles.add(c)

        # 参数点
        w_tracker = ValueTracker(2.5)
        b_tracker = ValueTracker(2.0)

        param_dot = always_redraw(lambda: Dot(
            contour_axes.coords_to_point(w_tracker.get_value(), b_tracker.get_value()),
            color=RED, radius=0.08,
        ))

        # ---- 右下：损失下降曲线 ----
        loss_axes = Axes(
            x_range=[0, 20, 2], y_range=[0, 8, 2],
            x_length=5, y_length=3,
            axis_config={"color": GREEN, "include_tip": True},
        )
        loss_axes.to_corner(DR, buff=0.6).shift(UP * 0.3)
        loss_title = Text("损失函数值", font_size=22, color=GREEN)
        loss_title.next_to(loss_axes, UP, buff=0.1)

        # 历史损失值
        loss_history = []

        # 动态更新损失曲线和参数轨迹
        def get_trajectory():
            if len(loss_history) < 2:
                return VGroup()
            points = []
            for i, loss in enumerate(loss_history):
                points.append(loss_axes.coords_to_point(i + 1, loss))
            poly = VMobject().set_points_as_corners(points)
            poly.set_color(YELLOW).set_stroke(width=2)
            return poly

        loss_curve = always_redraw(get_trajectory)
        loss_dot = always_redraw(lambda: Dot(
            loss_axes.coords_to_point(
                len(loss_history),
                loss_history[-1] if loss_history else 0
            ),
            color=RED, radius=0.06,
        ))

        self.play(Create(contour_axes), Create(loss_axes),
                  Write(contour_title), Write(loss_title),
                  Create(circles), run_time=2)

        self.add(param_dot, loss_curve, loss_dot)
        self.wait(0.3)

        # ---- 梯度下降迭代 ----
        w, b = 2.5, 2.0  # 初始参数
        lr = 0.3  # 学习率

        for step in range(15):
            # 损失函数 f(w,b) = w² + b²（极简示例）
            loss = w ** 2 + b ** 2
            loss_history.append(loss)

            # 梯度: [2w, 2b]
            dw, db = 2 * w, 2 * b
            w -= lr * dw
            b -= lr * db

            self.play(
                w_tracker.animate.set_value(w),
                b_tracker.animate.set_value(b),
                run_time=0.25,
            )

        self.wait(0.5)

        # 标注收敛点
        final_dot = Dot(contour_axes.coords_to_point(0, 0), color=GREEN, radius=0.1)
        self.play(GrowFromCenter(final_dot), run_time=1)

        conclusion = Text("收敛到最优解 (0, 0)", font_size=24, color=GREEN)
        conclusion.next_to(final_dot, DOWN, buff=0.3)
        self.play(Write(conclusion), run_time=1)
        self.wait(1.5)

        self.play(FadeOut(VGroup(contour_axes, loss_axes, contour_title,
            loss_title, circles, param_dot, loss_curve, loss_dot,
            final_dot, conclusion)), run_time=2)
```

**运行命令**：

```bash
manim -pqm scenes/chapter20_gradient_descent.py GradientDescent
```

**运行结果**：

画面左上角是参数空间等高线（蓝色同心圆），右下角是损失随迭代步数下降的曲线（黄色折线）。红色点初始在 (2.5, 2.0)，经 15 次梯度下降迭代后收敛到原点 (0, 0)，左上的参数点沿梯度方向螺旋下降，右下的损失曲线同步更新。全程两个 Axes 联动。

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 多 Axes | 独立坐标范围，适合并排对比 | 无内置联动机制，需手动写桥接 |
| 坐标转换 | coords_to_point/point_to_coords 双工转换 | 转换函数命名不够直观 |
| 面积填充 | get_area/get_riemann_rectangles 语义清晰 | 填充区域边界可能被曲线覆盖而不完整 |
| 参数空间 | 数学坐标和画面坐标分离，逻辑清晰 | 初学者容易混淆两种坐标 |
| 数据曲线 | 支持 parametric 和 implicit 多种曲线类型 | plot_implicit_curve 在复杂函数上性能差 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 梯度下降/优化 | 参数路径 + 损失曲线并排 |
| 积分教学 | 面积填充 + 黎曼矩形 + 精确值对比 |
| 函数族对比 | 多个函数在同一 Axes 或不同 Axes |
| 数据可视化 | 折线图、面积图、散点图 |
| 物理场 | 等高线、势能曲面、力场 |

**不适用场景**：3D 曲面可视化（使用 `ThreeDScene` + `Surface`）、实时大屏监控（需要 Web 渲染后端）。

### 注意事项

1. **`x_length` / `y_length` 与 Axes 的物理尺寸**：`plot` 创建的曲线按"数学坐标"采样，其"画面长度"取决于 `x_length`/`y_length`。改变 Axes 的物理尺寸会改变曲线的视觉形状。
2. **`get_area` 的 `x_range` 不能超曲线范围**：如果 `get_area(graph, x_range=[0, 10])` 但 `graph` 只覆盖 [2, 8]，[0, 2] 和 [8, 10] 区域可能是空白的。
3. **多 Axes 中的图例管理**：为每个曲线添加 `label` 并在 Axes 之间保持一致的颜色映射。

### 常见踩坑经验

**故障一：两个 Axes 之间的坐标映射"对不上"**

根因：画面坐标是双通道 (x, y)，而 `coords_to_point` 将数学 (x, y) 映射到画面的三维向量。传递时只取了前两个分量但量纲不对。

解决：始终走"数学坐标"桥接，避免用画面坐标做中介。

**故障二：`get_riemann_rectangles` 在曲线快速梯度处失准**

根因：均匀采样 `dx` 在陡坡处精度不足。

解决：减小 `dx`，或自定义 `x_range` 在陡坡处使用更密的采样。

**故障三：`always_redraw` 中 `get_area` 每帧重建导致抖动**

根因：`get_area` 依赖 `graph` 对象，而 `graph` 也是 `always_redraw` 的产物，两者重建时机可能差一帧。

解决：在同一个 `always_redraw` 中同时创建 graph 和 area。

### 思考题

1. 改造 `GradientDescent`，改用动量法（Momentum）：`v = β*v + (1-β)*dw`，并对比动量法和标准梯度下降在损失曲线上哪个收敛更快。提示：增加一个 `v_tracker` 记录速度。

2. 使用 `plot_implicit_curve` 绘制一组"心形线"：`(x² + y² - 1)³ - x²y³ = 0`。观察不同分辨率下曲线的平滑度。提示：`axes.plot_implicit_curve(lambda x, y: (x**2 + y**2 - 1)**3 - x**2 * y**3)`。
