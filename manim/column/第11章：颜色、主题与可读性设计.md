# 第11章：颜色、主题与可读性设计

---

## 1. 项目背景

某在线教育平台的内容团队已经用 Manim 连续制作了三个月的课程动画，累计产出 200 多集视频。初期大家各做各的，配色方案五花八门——有人用正红色标注重点，有人用玫红色；有人用纯白文字，有人在白色文字上加黄色描边导致看不清；有人给所有元素设满饱和色，画面像霓虹灯广告牌。

教研审核员看完最近一批作品后反馈："6 个不同老师做的 6 节课，放在一起看像是 6 个不同平台出的。学生学完一课跳到下一课，视觉风格突变，认知负担很大。"

更严重的是"可读性问题"——有节课在黑底上用了深蓝色文字，投到投影仪上完全看不见；有节课把公式设成 48px 字号但视频只渲染了 480p，字幕糊成一团。

这个痛点指向 Manim 项目的**视觉工程化**——不能每人凭审美感觉写颜色号，需要建立一套可量化、可复用、跨章节统一的**主题体系**。本章要讲清三件事：
1. Manim 的颜色体系（常量/十六进制/HSL/渐变/interpolate）
2. 主题库的工程化封装（`theme.py`）
3. 字号/对比度/留白的可读性设计

---

## 2. 剧本式交锋对话

> **场景**：大屏幕上并排播放着三集不同老师的手游课动画。颜色风格差異大到像三部不同作品。

**小胖**（拆开一根棒棒糖）：

"我说句实话——这三集放一起，感觉在看三本不同杂志。第一集像是《科学画报》的稳重蓝，第二集像是《时尚芭莎》的荧光粉，第三集干脆是霓虹灯管配色。你们是不是每人随机敲了个十六进制？"

**设计师小林**（苦笑）：

"差不多。我写了 `#4A90D9`，小王写了 `#3B82F6`，老李用了 `BLUE`——三个人三个蓝色，放在一起肉眼根本看不出来差别，但风格确实不统一。"

**小白**（调出 Manim 源码中的颜色定义）：

"Manim 预设了 40 多种颜色常量——`BLUE`、`RED`、`GREEN`、`YELLOW` 等。但问题是这些常量只是 `ManimColor` 对象，背后是固定色值。比如 `BLUE` 其实是 `#58C4DD`——一个偏青的蓝。而 `PURE_BLUE` 才是 `#0000FF`。如果你不知道这些色值的视觉差异，光看常量名容易误用。"

**大师**（站起来走到白板前）：

"颜色管理的核心原则是——**语义化，不是色值化**。不要在代码里直接写 `#FC6255`，写 `ACCENT_COLOR`。这样改色板时只需改一处定义，所有场景统一更新。我推荐团队维护一个 `theme.py`，定义三级颜色角色："

| 角色 | 变量名 | 用途 | 示例色 |
|------|--------|------|--------|
| 主色 | PRIMARY | 标题、主按钮、重点框 | #4C9AFF |
| 强调色 | ACCENT | 步骤高亮、动画焦点 | #FFB020 |
| 成功色 | SUCCESS | 正确答案、归位确认 | #47D38D |
| 危险色 | DANGER | 错误、警告、禁止 | #FF4D4F |
| 辅助文字 | TEXT_MUTED | 次要描述、注释 | #94A3B8 |
| 背景色 | BG_CARD | 卡片填充色 | rgba(255,255,255,0.08) |

> **技术映射**：Manim 的 `ManimColor` 内部存储 RGBA 四个通道（0-1 浮点数）。`interpolate_color(c1, c2, t)` 在两种颜色间按 t 线性插值。

**小胖**（舔着棒棒糖）：

"那字号和可读性呢？我上次看一个 4K 视频投到老投影仪上，字小的跟蚂蚁似的。"

**小白**：

"这是个播放端适配问题。Manim 默认的 `frame_width=14.2`、高清渲染 1920 像素，每单位 ≈ 135 像素。一个 `font_size=36` 的 Text 实际在画面上约 36 单位——但单位不是像素。实际像素高度 = 36 / 14.2 × 1920 ≈ 4863……不对，Text 的 `font_size` 是 Pango 的点数（pt），1pt ≈ 1.333 像素。所以 36pt 约等于 48 像素高。在 1080p 视频中占约 4.4% 画面高度——手机上勉强可读，老投影仪可能不够。"

**大师**：

"字号规范我建议定一个层级表：标题 40pt、小节标题 32pt、正文 26pt、注释 20pt、微注 16pt。这套层级映射到画面上，能保证 1080p 输出的最小字号不低于 20 像素，投到 720p 投影仪上也勉强可读。

另外，**对比度**问题不能忽视。黑底白字对比度最高；黑底灰字（如 `GRAY`）在投影仪上可能完全看不见。建议文本色不低于 `#CCCCCC`（80%白），注释不低于 `#888888`（53%白）。"

> **技术映射**：对比度可以用 `(L1+0.05)/(L2+0.05)` 计算相对亮度比。WCAG AA 标准要求正文 > 4.5:1，大字号 > 3:1。

### 3.1 环境准备

沿用第 2 章搭建的环境。在项目中创建 `constants/theme.py` 主题模块：

# constants/theme.py
from manim import ManimColor

# ======== 1. 调色板 ========
PRIMARY   = ManimColor("#4C9AFF")

### 3.2 分步实现

> **本章实战目标**：基于统一主题库，制作一段包含标题、卡片、步骤指示器、警示框和对比实验的"主题应用示范"动画。

---

#### 步骤一：建立主题基类 Scene

**步骤目标**：创建一个 `ThemedScene`，自动设置背景色和全局默认样式。

```python
# scenes/chapter11_themed.py
from manim import *
from constants.theme import *  # 如果 PYTHONPATH 正确的话
# 否则内联定义常量

class ThemedScene(Scene):
    """所有项目场景的基类，统一背景和主题"""
    def setup(self):
        super().setup()
        # 设置背景色
        self.camera.background_color = BACKGROUND

    def make_card(self, text, width=10, height=2, **kw):
        """创建主题卡片"""
        card = RoundedRectangle(
            width=width, height=height,
            corner_radius=CORNER_RADIUS,
            color=PRIMARY, fill_opacity=0.06, stroke_width=STROKE_NORMAL,
            **kw,
        )
        label = Text(text, font_size=FONT_BODY, color=TEXT_PRIMARY)
        label.move_to(card.get_center())
        return VGroup(card, label)

    def make_step_indicator(self, number, text):
        """创建步骤指示器"""
        circle = Circle(radius=0.3, color=ACCENT, fill_opacity=0.2)
        num = Text(str(number), font_size=FONT_SMALL, color=ACCENT)
        num.move_to(circle.get_center())
        label = Text(text, font_size=FONT_SMALL, color=TEXT_SECONDARY)
        label.next_to(circle, DOWN, buff=SPACING_SM)
        return VGroup(circle, num, label)


class ThemeShowcase(ThemedScene):
    def construct(self):
        # 1. 主标题
        title = Text("统一主题演示", font_size=FONT_TITLE, color=PRIMARY, weight=BOLD)
        title.to_edge(UP, buff=SPACING_LG)
        self.play(Write(title))

        # 2. 信息卡片
        card = self.make_card("欢迎使用 Manim 主题系统")
        card.next_to(title, DOWN, buff=SPACING_LG)
        self.play(FadeIn(card, scale=0.95), run_time=1.5)
        self.wait(0.3)

        # 3. 步骤指示器
        steps = VGroup(
            self.make_step_indicator(1, "初始化"),
            self.make_step_indicator(2, "加载数据"),
            self.make_step_indicator(3, "执行渲染"),
            self.make_step_indicator(4, "输出文件"),
        )
        steps.arrange(RIGHT, buff=1.5)
        steps.next_to(card, DOWN, buff=SPACING_LG)
        self.play(LaggedStart(*[FadeIn(s) for s in steps], lag_ratio=0.2), run_time=2)
        self.wait(0.5)

        # 4. 对比实验：同一底色，不同主题色
        self.play(FadeOut(VGroup(card, steps)), run_time=1)

        palettes = [
            ("主题 A（蓝）", PRIMARY),
            ("主题 B（绿）", SUCCESS),
            ("主题 C（橙）", ACCENT),
        ]

        swatches = VGroup()
        for i, (name, col) in enumerate(palettes):
            rect = RoundedRectangle(
                width=3.0, height=1.5, corner_radius=CORNER_RADIUS,
                color=col, fill_opacity=0.15, stroke_width=2,
            )
            txt = Text(name, font_size=FONT_SMALL, color=col)
            txt.move_to(rect.get_center())
            pair = VGroup(rect, txt)
            swatches.add(pair)

        swatches.arrange(DOWN, buff=0.5)
        swatches.move_to(card.get_center())
        self.play(LaggedStart(*[FadeIn(s) for s in swatches], lag_ratio=0.2), run_time=2)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, swatches)), run_time=1.5)
```

**运行结果**：

深色背景下，蓝色标题、半透明卡片、橙色步骤指示器依次出现，然后展示三组不同主题色的卡片对比。所有颜色、字号、间距都从 `theme.py` 统一调用。

#### 步骤二：颜色渐变与插值实战

**步骤目标**：展示 `interpolate_color` 和 `color_gradient` 的实际效果。

```python
class ColorInterpolation(ThemedScene):
    def construct(self):
        title = Text("颜色渐变与插值", font_size=FONT_TITLE, color=PRIMARY)
        title.to_edge(UP, buff=SPACING_LG)
        self.play(Write(title))

        # 从蓝到橙的渐变条
        n_bars = 12
        bars = VGroup()
        for i in range(n_bars):
            t = i / (n_bars - 1)
            col = interpolate_color(PRIMARY, ACCENT, t)
            bar = Rectangle(
                width=0.7, height=0.5 + t * 2.5,
                color=col, fill_opacity=0.8, stroke_width=0,
            )
            label = Text(f"{t:.2f}", font_size=FONT_TINY, color=TEXT_MUTED)
            label.next_to(bar, DOWN, buff=0.1)
            bars.add(VGroup(bar, label))
        bars.arrange(RIGHT, buff=0.15, aligned_edge=DOWN)
        bars.next_to(title, DOWN, buff=SPACING_LG)
        self.play(LaggedStart(*[GrowFromCenter(b) for b in bars], lag_ratio=0.1), run_time=2)
        self.wait(1.5)

        self.play(FadeOut(VGroup(title, bars)), run_time=1.5)
```

#### 步骤三：可读性对比实验

**步骤目标**：通过多组对比演示字号、对比度、留白的可读性差异。

```python
class ReadabilityDemo(ThemedScene):
    def construct(self):
        # 对比组 1：字号层级
        title = Text("可读性对比：字号 × 对比度 × 留白", font_size=FONT_TITLE, color=PRIMARY)
        title.to_edge(UP, buff=SPACING_LG)
        self.play(Write(title))

        sizes_left = VGroup()
        for i, (label, sz) in enumerate([
            ("hero 56pt", 56), ("title 40pt", 40),
            ("heading 32pt", 32), ("body 26pt", 26),
            ("small 20pt", 20), ("tiny 16pt", 16),
        ]):
            t = Text(label, font_size=sz, color=TEXT_PRIMARY)
            t.shift(UP * (2.5 - i * 1.0) + LEFT * 4)
            sizes_left.add(t)

        # 对比组 2：对比度
        contrasts_right = VGroup()
        for i, (label, col, alpha) in enumerate([
            ("TEXT_PRIMARY", TEXT_PRIMARY, None),
            ("TEXT_SECONDARY", TEXT_SECONDARY, None),
            ("TEXT_MUTED", TEXT_MUTED, None),
            ("#555 低对比度", "#555555", None),
            ("透明 50%", TEXT_PRIMARY, 0.5),
            ("透明 30%", TEXT_PRIMARY, 0.3),
        ]):
            t = Text(label, font_size=FONT_BODY, color=col)
            if alpha:
                t.set_opacity(alpha)
            t.shift(UP * (2.5 - i * 1.0) + RIGHT * 3)
            contrasts_right.add(t)

        self.play(
            LaggedStart(*[Write(s) for s in sizes_left], lag_ratio=0.1),
            LaggedStart(*[Write(c) for c in contrasts_right], lag_ratio=0.1),
            run_time=3,
        )
        self.wait(2)
        self.play(FadeOut(VGroup(title, sizes_left, contrasts_right)), run_time=1.5)
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 颜色管理 | 语义化变量 + ManimColor 全链路统一 | 颜色常量数量多时查找不便 |
| 主题封装 | ThemedScene 统一入口，新场景零配置 | 基类耦合较高，非标准场景需要额外处理 |
| 尺寸层级 | 6 级字号适配主流播放设备和分辨率 | 小字号（16pt）在 480p 下几乎不可读 |
| 渐变工具 | interpolate_color / color_gradient 灵活 | 不支持 HSL 空间的平滑过渡 |
| 对比度 | 量化控制文本可见性 | 投影仪/手机屏的 gamma 差异无法自动适应 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 系列课程 | 多集视频统一的品牌视觉 |
| 多团队协作 | 不同开发者用同一调色板 |
| 品牌宣传 | 公司 Logo 色系的规范应用 |
| 学术演示 | 专业严肃的暗色主题 |
| 直播/录播 | 适配不同播放设备的字号方案 |

**不适用场景**：单次使用的临时实验动画（不需要完整的主题工程化）、需要与外部品牌规范对接的项目（可能使用 CSS 变量等其他体系）。

### 注意事项

1. **`ManimColor` 是 Alpha 预乘的**：`ManimColor("#FF0000")` 的 alpha 通道默认是 1.0。设置 `fill_opacity` 会覆盖 alpha，两者作用机制不同。
2. **字体颜色和 `stroke` 的配合**：深色背景上，`stroke` 应设为 0 或接近于背景色；浅色背景上，深色 `stroke` 可增加可读性。
3. **主题模块的导入路径**：确保 `constants/theme.py` 在 `sys.path` 中（或在 `manim.cfg` 的 `[custom_folders]` 段中配置）。

### 常见踩坑经验

**故障一：`ManimColor` 赋值后颜色不变**

根因：对已创建的对象设置 `obj.color = PRIMARY` 不会触发重绘。必须用 `obj.set_color(PRIMARY)` 或 `obj.animate.set_color(PRIMARY)`。

**故障二：半透明元素叠加后画面发灰**

根因：多个 `fill_opacity=0.1` 的元素重叠，alpha 值加法叠加。

解决：减少层数，或对底层 VGroup 设置整体 opacity 而非每个子对象。

**故障三：字号层级在 -pql 和 -pqh 下表现不一致**

根因：低清渲染分辨率不同，字号虽然用 pt 指定，但实际像素大小与渲染分辨率相关。

解决：在开发阶段用 -pqh 确认最终效果，或统一渲染高清版。
