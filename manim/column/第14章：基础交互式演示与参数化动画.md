# 第14章：基础交互式演示与参数化动画

---

## 1. 项目背景

某高中数学教研组用 Manim 制作了"二次函数图像分析"系列动画。初期版本是硬编码的——函数参数直接写死在代码里：`lambda x: 2 * x ** 2 - 3 * x + 1`。当教研组长要求"改一下 a 的值，从 2 改成 -1，重新渲染看看开口方向"时，编辑小刘需要手动改代码 → 重新渲染 → 等待 3 分钟 → 检查效果 → 不满意再改。

这个循环每次约 5 分钟，调整 10 次参数就是一个小时。小刘抱怨："能不能像 Desmos 那样，拖动滑块就能看效果？我不能拖滑块，但起码能不能改一行数字就自动生成所有版本？"

这个痛点暴露了 Manim 的**代码执行模型**——它是批处理渲染（batch render），不是交互式应用（interactive app）。虽然不能"拖动滑块实时预览"，但可以通过**参数化设计**大幅降低迭代成本：

- 将场景封装为带参数的 Python 函数或类
- 用 `ValueTracker` 驱动动态值在动画中变化
- 用循环批量生成多组参数版本的动画
- 用 `always_redraw` 让对象跟随数值实时变化

本章就要讲清这四招，把"改参数 → 等渲染 → 看结果"的迭代周期从 5 分钟压缩到 30 秒。

---

## 2. 剧本式交锋对话

> **场景**：小刘对着三个同样内容但参数不同的渲染窗口发呆。小胖在吃芒果干。

**小胖**（嚼着芒果干）：

"小刘，你屏幕上这仨抛物线跟克隆人似的——除了一个开口向上，一个开口向下，一个特别窄。为啥不写成'参数 a 从列表 [1, -1, 0.5] 里自动生成三个版本'呢？"

**小刘**（揉着眼）：

"我也想啊。但 Manim 的 `construct` 方法是固定的，场景类名也是固定的。我总不能写三个 `class Parabola_A1(Scene)`、`class Parabola_Am1(Scene)`、`class Parabola_A05(Scene)` 吧？"

**小白**（已经在命令行里敲完了测试）：

"不需要三个类。你写一个 `class Parabola(Scene)`，在 `construct` 里接收一个参数 `a`，然后在脚本末尾用 `if __name__ == "__main__"` 来遍历参数列表即可。关键技巧是——**把参数挂到类属性上，在 `construct` 里用 `self.a` 访问**。"

```python
class Parabola(Scene):
    a = 1  # 默认值
    def construct(self):
        curve = axes.plot(lambda x: self.a * x**2, ...)
```

"然后在命令行里用 `manim scene.py Parabola` 渲染默认值，或者写一个 `for a in [1, -1, 0.5]` 循环批量调 `manim`。"

**大师**（点头）：

"小白说的对。不过我要补充一个更强大的工具——**`ValueTracker` + `always_redraw`**。它让你在同一个渲染中动态改变参数，并且画面自动更新，不需要重新渲染。"

（大师在白板上画出流程图）

```python
a_tracker = ValueTracker(1)
curve = always_redraw(lambda: axes.plot(
    lambda x: a_tracker.get_value() * x**2, ...
))
# 在 play 中改变 tracker 值 → 画面自动更新
self.play(a_tracker.animate.set_value(2), run_time=2)
```

"这就是参数化设计的高级形态——参数不是静态的，而是在动画中**随时间变化**。一个 8 秒的动画可以展示 a 从 -2 到 2 的连续变化，所有中间帧的抛物线都精确对应当时的参数值。"

> **技术映射**：`always_redraw(func)` 创建一个"Reactive Mobject"——每一帧，Manim 都会调用 `func()` 重新生成对象。`ValueTracker.animate.set_value(x)` 会让 tracker 的值从当前值平滑过渡到 x，这个过程触发了每一帧的 `always_redraw`。

**小胖**（芒果干差点噎着）：

"等等——那如果我要展示 a 和 b 两个参数同时变呢？a 控制开口，b 控制左右平移，两个 tracker 同时动？"

**大师**：

"可以！你甚至可以创建三个 tracker 分别控制 a、b、c，然后在 one `always_redraw` 的 lambda 中同时读取三个值。但要注意性能——`always_redraw` 在每一帧都会重建 Mobject，如果你的函数内部做了大量计算（如数值积分），60fps 下每秒 60 次重建可能会让渲染变慢。"

> **本章实战目标**：制作一个"二次函数参数探索器"——用一个可调参数的动画，展示 a, b, c 三个参数对二次函数图像的影响。

#### 步骤一：函数封装参数化场景

**步骤目标**：学会将 Scene 构造为可接受外部参数的形式。

```python
# scenes/chapter14_parametric.py
from manim import *

class ParametricParabola(Scene):
    """参数化二次函数场景——a 控制开口方向和大小"""
    a = 1  # 可通过类属性赋值修改

    def construct(self):
        axes = Axes(
            x_range=[-5, 5, 1], y_range=[-3, 8, 1],
            x_length=9, y_length=5.5,
            axis_config={"color": BLUE, "include_tip": True},
        )
        axes.to_edge(DOWN, buff=0.5)

        title = Text(f"y = {self.a}x²", font_size=36, color=YELLOW)
        title.to_edge(UP, buff=0.4)

        graph = axes.plot(lambda x: self.a * x ** 2, color=YELLOW, stroke_width=3)

        self.play(Write(title), Create(axes), Create(graph), run_time=2)
        self.wait(1.5)
        self.play(FadeOut(VGroup(title, axes, graph)), run_time=1)


# 在脚本末尾批量生成多版本
if __name__ == "__main__":
    for val in [1, 2, -1, 0.5]:
        ParametricParabola.a = val
        # 用 subprocess 调用 manim 命令行
```

**运行结果**：

通过修改 `ParametricParabola.a = val` 后再渲染，得到 4 段不同参数值的抛物线动画。

#### 步骤二：ValueTracker + always_redraw 动态参数

**步骤目标**：在单个动画中展示参数连续变化。

```python
# scenes/chapter14_value_tracker.py
from manim import *

class DynamicParabola(Scene):
    def construct(self):
        # 坐标轴
        axes = Axes(
            x_range=[-5, 5, 1], y_range=[-10, 10, 2],
            x_length=9, y_length=5.5,
            axis_config={"color": BLUE, "include_tip": True},
        )
        axes.to_edge(DOWN, buff=0.5)

        self.play(Create(axes), run_time=1.5)

        # ValueTracker —— 驱动参数 a
        a_tracker = ValueTracker(1)

        # always_redraw：曲线根据 a_tracker 的值实时更新
        curve = always_redraw(lambda: axes.plot(
            lambda x: a_tracker.get_value() * x ** 2,
            x_range=[-3, 3],
            color=YELLOW, stroke_width=3,
        ))

        # 公式标签也实时更新
        label = always_redraw(lambda: MathTex(
            f"a = {a_tracker.get_value():.1f}",
            font_size=32, color=YELLOW,
        ).to_corner(UR, buff=0.5))

        self.add(curve, label)
        self.wait(0.5)

        # a 从 1 → 2（开口变窄）
        self.play(a_tracker.animate.set_value(2), run_time=2)
        self.wait(0.5)

        # a 从 2 → -2（开口翻转）
        self.play(a_tracker.animate.set_value(-2), run_time=2)
        self.wait(0.5)

        # a 从 -2 → 0.5（开口变宽）
        self.play(a_tracker.animate.set_value(0.5), run_time=2)
        self.wait(0.5)

        self.play(a_tracker.animate.set_value(1), run_time=1.5)
        self.wait(1)

        self.play(FadeOut(VGroup(axes, curve, label)), run_time=1.5)
```

**运行命令**：

```bash
manim -pqm scenes/chapter14_value_tracker.py DynamicParabola
```

**运行结果**：

一个约 10 秒的动画，抛物线在坐标轴上实时变化——a 从 1 到 2（开口收窄）、到 -2（翻转向下）、到 0.5（开口加宽）、回到 1。右上角的 a 值标签同步更新。全程只渲染了一次。

#### 步骤三：多参数协同——a/b/c 三参数探索器

**步骤目标**：同时控制三个参数，展示二次函数全参数空间。

```python
# scenes/chapter14_abc_explorer.py
from manim import *

class ABCExplorer(Scene):
    def construct(self):
        axes = Axes(
            x_range=[-6, 6, 1], y_range=[-8, 8, 2],
            x_length=8, y_length=5,
            axis_config={"color": BLUE, "include_tip": True},
        )
        axes.to_edge(DOWN, buff=0.5)
        self.play(Create(axes), run_time=1.5)

        a_tracker = ValueTracker(1)
        b_tracker = ValueTracker(0)
        c_tracker = ValueTracker(0)

        # 实时更新曲线
        curve = always_redraw(lambda: axes.plot(
            lambda x: (a_tracker.get_value() * x ** 2
                     + b_tracker.get_value() * x
                     + c_tracker.get_value()),
            x_range=[-5, 5],
            color=YELLOW, stroke_width=3,
        ))

        # 参数面板（左上角）
        panel = always_redraw(lambda: VGroup(
            MathTex(f"a = {a_tracker.get_value():.1f}",
                    font_size=28, color=RED),
            MathTex(f"b = {b_tracker.get_value():.1f}",
                    font_size=28, color=GREEN),
            MathTex(f"c = {c_tracker.get_value():.1f}",
                    font_size=28, color=BLUE),
        ).arrange(DOWN, buff=0.2, aligned_edge=LEFT).to_corner(UL, buff=0.5))

        self.add(curve, panel)
        self.wait(0.5)

        # a 变 → 开口变化
        self.play(a_tracker.animate.set_value(2), run_time=1.5)
        self.wait(0.3)

        # b 变 → 左右平移
        self.play(b_tracker.animate.set_value(-3), run_time=1.5)
        self.wait(0.3)

        # c 变 → 上下平移
        self.play(c_tracker.animate.set_value(4), run_time=1.5)
        self.wait(0.3)

        # 同时变
        self.play(
            a_tracker.animate.set_value(-1),
            b_tracker.animate.set_value(2),
            c_tracker.animate.set_value(-2),
            run_time=2.5,
        )
        self.wait(1.5)

        self.play(FadeOut(VGroup(axes, curve, panel)), run_time=1.5)
```

**运行结果**：

抛物线在坐标轴内实时变化，左上角实时显示三个参数值。先单独改变 a（开口变化），再单独改变 b（水平位移），再单独改变 c（垂直位移），最后三者同时变。整个过程在同一个渲染中完成。

**可能遇到的坑**：

1. **`always_redraw` 的性能**：每次 `always_redraw` 都会重新 `plot`（包含数值积分或采样），在 60fps 下每秒 60 次。如果 `plot` 函数很重（如使用 `scipy.integrate`），帧率会暴跌。
2. **tracker 值变化太快导致曲线动画断裂**：如果 `a_tracker` 从 1 到 2 用时 0.5 秒但采样帧数不足，可能看到"跳帧"。解决：保证 >=1 秒的过渡时间。
3. **`always_redraw` 中创建 LaTeX**：每个 `MathTex` 都需要 LaTeX 编译——虽然 Manim 有缓存，但首次创建时会阻塞帧渲染。

#### 补充：批量参数扫描实战

**步骤目标**：编写一个脚本，自动遍历参数生成多版本动画并合成对比画面。

```python
# scenes/chapter14_batch_scan.py
"""批量参数扫描：自动生成多组 a 值的抛物线对比"""
from manim import *

class BatchParabola(Scene):
    """在同一个画面中展示多个 a 值对应的抛物线"""
    def construct(self):
        axes = Axes(
            x_range=[-4, 4, 1], y_range=[-2, 8, 2],
            x_length=6, y_length=4,
            axis_config={"color": BLUE, "include_tip": True},
        )
        axes.to_edge(DOWN, buff=0.5)

        title = Text("二次函数对比：y = a·x²", font_size=32, color=WHITE)
        title.to_edge(UP, buff=0.4)

        self.play(Write(title), Create(axes), run_time=1.5)

        # 一次性绘制多条曲线，每个 a 值不同颜色
        a_values = [2.0, 1.0, 0.5, -0.5, -1.0, -2.0]
        colors = [RED, ORANGE, YELLOW, GREEN, TEAL, BLUE]

        curves = VGroup()
        legends = VGroup()
        for a, col in zip(a_values, colors):
            curve = axes.plot(
                lambda x, a=a: a * x ** 2,
                x_range=[-3, 3],
                color=col, stroke_width=2,
            )
            legend = VGroup(
                Dot(radius=0.08, color=col),
                MathTex(f"a={a}", font_size=20, color=col),
            ).arrange(RIGHT, buff=0.15)
            curves.add(curve)
            legends.add(legend)

        legends.arrange(DOWN, buff=0.12, aligned_edge=LEFT)
        legends.to_corner(UR, buff=0.5)

        self.play(
            LaggedStart(*[Create(c) for c in curves], lag_ratio=0.1),
            LaggedStart(*[FadeIn(l) for l in legends], lag_ratio=0.1),
            run_time=3,
        )
        self.wait(2)

        self.play(FadeOut(VGroup(title, axes, curves, legends)), run_time=1.5)
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 参数化 | 类属性/函数参数实现代码复用 | 场景类不支持构造函数传参 |
| ValueTracker | 驱动一个动画中的连续变化 | 每个 tracker 占一帧的 CPU 时间 |
| always_redraw | 数值→画面的响应式绑定 | 性能开销不可忽略，需要谨慎使用 |
| 批量生成 | Python 脚本批量渲染多组参数 | 需要手动管理产物命名和归档 |
| 教学表现 | 可视化参数对函数的影响，远超静态图 | 无 Desmos 式的交互体验 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 数学教学 | 函数参数(a/b/c)对图像的影响 |
| 物理实验 | 参数扫描（频率、振幅、阻尼） |
| 算法分析 | 不同参数下的算法行为 |
| 设计对比 | 不同配色/字号/布局的 A/B 对比 |
| 论文配图 | 生成多组参数的对比图 |

**不适用场景**：需要精细手动逐帧调整的艺术动画（`always_redraw` 是全局刷新，无法局部修改）、实时交互式应用。

### 注意事项

1. **`always_redraw` 中的 lambda 闭包**：lambda 捕获的是外部变量的**引用**，不是值。如果 lambda 中使用了 for 循环的迭代变量，所有 `always_redraw` 可能会共享同一个最终值。解决方法见下文陷阱。
2. **tracker 的 `animate.set_value` 是平滑过渡**：`a_tracker.animate.set_value(2, run_time=3)` 会在 3 秒内从当前值平滑过渡到 2。如果只想瞬间跳变，用 `a_tracker.set_value(2)` 不带 `animate`。
3. **多个 `always_redraw` 的顺序**：它们按添加顺序依次执行。如果一个 `always_redraw` 依赖另一个的结果，注意执行顺序。

### 常见踩坑经验

**故障一：`always_redraw` 中的 lambda 捕获了错误的值**

根因（经典 Python 坑）：`for i in range(5): obj = always_redraw(lambda: f(i))` —— 所有 lambda 共享同一个闭包变量 `i`，最终都等于 4。

解决：`lambda i=i: f(i)` 使用默认参数捕获当前值。

**故障二：tracker 值动画期间 `print` 返回值不变**

根因：`print` 在 play 执行前就调用了，tracker 还没开始变化。

解决：在 `always_redraw` 的 lambda 中打印，或使用 `self.wait(0.1)` 后再打印。

**故障三：大量 `always_redraw` 导致渲染时间爆炸**

根因：每个 `always_redraw` 每帧都重建 Mobject，对象的 LaTeX、SVG、plot 等都可能触发开销。10+ 个 `always_redraw` 在高清 60fps 下可能是灾难。

解决：合并多个 `always_redraw` 为一个，或者降低帧率（`--frame_rate 15`）。
