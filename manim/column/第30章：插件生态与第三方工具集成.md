# 第30章：插件生态与第三方工具集成

## 1. 项目背景

经过前 29 章的学习，读者已经掌握了 Manim 的核心能力。但随着项目复杂度提升，一些需求开始超出 Manim 原生的边界：

1. **slides 导出**：教育团队需要将动画转换为交互式 PPT 风格的幻灯片播放，支持暂停/跳转/回放，而不是一个线性播放的视频。
2. **物理模拟**：物理教研室需要模拟刚体碰撞——刚体的运动学方程比粒子复杂得多，从头手写 400 行代码。
3. **Jupyter 集成**：数据团队希望在 Jupyter Notebook 中实时预览 Manim 动画的中间帧，而不是每次改参数都要等完整渲染。
4. **networkx 图算法**：算法教研组需要画大规模社交网络传播图，200+ 节点的布局计算用手写的 circular layout 不可能完成。

这些需求都指向同一个答案——**Manim 的插件生态**。Manim Community 维护着十几个活跃插件，覆盖了 slides 导出、物理引擎、语音合成、Jupyter 交互、图论可视化等场景。学会发现和使用这些插件，能让你的生产能力提升一个数量级。

## 2. 剧本式交锋对话

> **场景**：团队四人各自面对一个新需求，都不知道从何下手。

**小胖**（抱着一袋焦糖瓜子）：

"我想做个社交网络分析动画——100 个节点互相连着线，让度高的节点大一些放在中间，度低的放边沿。光是画这 100 个节点，我是不是得手写半天坐标算法？"

**小白**（已经在 pip 安装了 `networkx` + Manim）：

"不需要。`networkx` 是一个专业的图计算库，它内置了 spring layout、kamada-kawai、spectral 等十几种布局算法。你只需要：

```python
import networkx as nx
G = nx.barabasi_albert_graph(100, 3)  # 生成 100 节点幂律网络
pos = nx.spring_layout(G)  # 力导向布局
# 把 pos 字典转成 Manim 坐标
graph = Graph(list(G.nodes), list(G.edges),
              layout={n: (pos[n][0]*4-2, pos[n][1]*3-1.5) for n in G.nodes})
```

"3 行代码做完 100 个节点的自动布局。比手动画圆算坐标先进一个时代。"

**大师**（补充）：

"networkx 是数据层，Manim 是渲染层。两个库的分工非常明确——networkx 负责图的数据结构、算法和布局计算；Manim 负责节点渲染、边绘制和动画编排。这两者的配合是 Manim 生态中最经典的第三方集成案例。"

> **技术映射**：`nx.spring_layout(G)` 返回 `{node_id: (x, y)}` 字典。Manim 的 `Graph(layout=...)` 直接消费这个字典。

**小米**（团队的新人）：

"我想做一个可以暂停、跳转、回放的交互式课件——观众可以自己控制播放节奏，比线性视频灵活。Manim 能做到吗？"

**小白**（调出 `manim-slides` 文档）：

"`manim-slides` 插件就是为这个场景生的。它把你的 Manim 动画转换为一组 HTML/JavaScript 幻灯片——每个 `self.wait()` 变成一个'暂停点'，观众可以用键盘上下键翻页、空格键播放当前动画。

安装和使用超级简单：

```bash
pip install manim-slides
manim-slides scene.py MyScene  # 自动渲染所有段落到 HTML 文件
```

`manim-slides` 会识别你代码中的 `self.wait()` 和 `play()` 段落边界，自动打断为可翻页的幻灯片。比导出 MP4 后再用 PPT 拼接方便得多。"

**大师**（补充）：

"manim-slides 的亮点是**自动检测段落边界**——不需要你在代码里额外标记'这里翻页'。每个 `self.wait()` 之后就是下一个幻灯片。如果你想要更精细的控制，可以设置 `self.next_slide()` 来显式标记翻页点。"

**小胖**（瓜子壳掉了一地）：

"那物理模拟呢？上次我花了两天手写刚体碰撞的方程。有没有现成的物理引擎？"

**小白**：

"`manim-physics` 插件就是你的救星。它封装了常见的物理效果：

- `Gravity`: 给所有对象添加向下加速度
- `RigidBody`: 模拟刚体碰撞和旋转
- `ElectricField`: 电荷和电场可视化
- `Spring`: 弹簧连接两个物体

一行代码激活物理世界：

```python
from manim_physics import *
class PhysicsDemo(SpaceScene):
    def construct(self):
        ball = Circle(radius=0.2).shift(UP * 2)
        self.add(ball)
        self.make_rigid_body(ball)  # 转换成受重力影响的刚体
        self.wait(3)  # 球自由落体 3 秒
```

`SpaceScene` 是 `Scene` 的子类，内置了 PyMunk 物理引擎。"

**小胖**（瓜子终于嗑完了）：

"这些插件听着都很好——但我有个核心顾虑：万一我用的 `manim-slides 5.x` 跟 `manim 0.21` 不兼容怎么办？或者两个插件之间互相冲突？怎么管理这个版本地狱？"

**大师**：

"这是插件生态的经典难题。Manim 社区用的策略是——

1. **锁定主版本**：在 `requirements.txt` 中固定 `manim==0.20.1`，所有插件选择支持这个版本的。
2. **虚拟环境隔离**：每个项目用独立的 venv，插件冲突不影响其他项目。
3. **渐进式引入**：不要一次性装所有插件。先装 networkx + Manim（基础组合），跑通后再引入 manim-slides，最后 manim-physics。

如果有两个插件都依赖同一个库的不同版本，用 `pip install --upgrade` 升级到两者共同支持的版本。或者在 `setup.py/pyproject.toml` 中使用宽松版本约束（`>=1.0,<2.0`）。

最稳健的方案是——**只装你随时在用的插件**。不要为了'以后可能用'预装一堆，徒增兼容性风险。"

> **技术映射**：`pip freeze > requirements.txt` 锁定当前所有依赖的精确版本。`pip install -r requirements.txt` 精确复现环境。

**小白**（最后补充）：

"除了 pip 安装的插件，还有一个重要的集成方向——**Jupyter Notebook**。Manim 官方提供了 `%%manim` 魔术命令，可以在 Jupyter Cell 中直接渲染动画并嵌入输出：

```python
%%manim -qm -v WARNING MyScene
class MyScene(Scene):
    def construct(self):
        ...
```

执行后会直接在 notebook 的输出区域显示渲染好的视频。对数据科学家和快速实验非常友好。"

#### 步骤一：networkx + Manim 社交网络动画

**步骤目标**：用 networkx 生成社交网络图并用 Manim 渲染。

```python
# scenes/chapter30_network.py
from manim import *
import networkx as nx

class SocialNetwork(Scene):
    def construct(self):
        # 1. 用 networkx 生成社交网络
        G = nx.watts_strogatz_graph(30, 4, 0.3)  # 小世界网络

        # 2. 计算力导向布局
        pos = nx.spring_layout(G, k=0.5, iterations=50)

        # 3. 计算节点大小（按度中心性）
        centrality = nx.degree_centrality(G)
        node_sizes = {n: 0.08 + centrality[n] * 0.3 for n in G.nodes}

        # 4. 创建 Manim Graph
        vertices = list(G.nodes)
        edges = [(e[0], e[1]) for e in G.edges]

        manim_graph = Graph(
            vertices, edges,
            layout={n: (pos[n][0]*5, pos[n][1]*4, 0) for n in G.nodes},
            vertex_config={
                "fill_color": BLUE, "fill_opacity": 0.3,
                "stroke_width": 1, "radius": 0.12,
            },
            edge_config={"stroke_color": GRAY, "stroke_width": 0.5, "stroke_opacity": 0.4},
        )

        title = Text("社交网络分析：小世界网络", font_size=28, color=BLUE).to_edge(UP, buff=0.3)

        self.play(Write(title), Create(manim_graph), run_time=3)
        self.wait(1)

        # 高亮度最高的 3 个节点
        top_nodes = sorted(centrality, key=centrality.get, reverse=True)[:3]
        for node in top_nodes:
            self.play(manim_graph.vertices[node].animate.set_color(YELLOW), run_time=0.3)

        self.wait(1.5)
        self.play(FadeOut(VGroup(title, manim_graph)), run_time=1.5)
```

**运行结果**：

一个 30 节点的小世界网络在画面中展开。节点按 `spring_layout` 自动排布——连接紧密的节点靠近，稀疏的节点远离。度中心性最高的 3 个节点依次高亮为黄色。

#### 步骤二：manim-slides 交互式课件

**步骤目标**：将上一章的场景转换为可翻页的交互式 slides。

```python
# scenes/chapter30_slides.py
from manim import *
# 安装 manim-slides 后可以使用 Slide 基类
# from manim_slides import Slide

class NetworkSlides(Scene):
    """注意：Slide 是 manim-slides 的基类，这里用 Scene 模拟"""
    def construct(self):
        import networkx as nx

        # Slide 1: 标题
        title = Text("小世界网络", font_size=40, color=BLUE, weight=BOLD)
        self.play(Write(title))
        self.wait(0.5)
        self.play(FadeOut(title))
        # self.next_slide()  # manim-slides 中的翻页标记

        # Slide 2: 网络结构
        G = nx.watts_strogatz_graph(25, 4, 0.2)
        pos = nx.spring_layout(G, seed=42)
        graph = Graph(
            list(G.nodes), [(e[0], e[1]) for e in G.edges],
            layout={n: (pos[n][0]*5, pos[n][1]*4, 0) for n in G.nodes},
            vertex_config={"fill_color": BLUE, "fill_opacity": 0.3, "radius": 0.1},
            edge_config={"stroke_color": GRAY, "stroke_width": 0.5, "stroke_opacity": 0.3},
        )
        self.play(Create(graph), run_time=3)
        self.wait(0.5)
        # self.next_slide()

        # Slide 3: 信息传播模拟
        # 模拟 SIR 模型传播
        infected = {0}
        recovered = set()
        for step in range(5):
            new_infected = set()
            for node in infected:
                for neighbor in G.neighbors(node):
                    if neighbor not in infected and neighbor not in recovered:
                        if np.random.random() < 0.3:
                            new_infected.add(neighbor)
            # 标记传播
            for node in new_infected:
                self.play(graph.vertices[node].animate.set_color(RED), run_time=0.1)
            infected |= new_infected
            self.wait(0.3)

        self.wait(1)
        self.play(FadeOut(graph), run_time=1.5)
```

**运行命令**：

```bash
# 使用 manim-slides 渲染
manim-slides scenes/chapter30_slides.py NetworkSlides

# 渲染为 HTML slides
manim-slides convert scenes/chapter30_slides.py NetworkSlides slides_output/
```

#### 步骤三：manim-physics 物理模拟

**步骤目标**：用 manim-physics 创建带重力的物理世界。

```python
# scenes/chapter30_physics.py
# 需要 pip install manim-physics
from manim import *

# 安装 manim-physics 后可用：
# from manim_physics import SpaceScene, RigidBody

class PhysicsDemo(Scene):
    """使用 manim-physics 的 SpaceScene 实现物理模拟"""
    # 如果安装了 manim-physics，改为：
    # class PhysicsDemo(SpaceScene):
    def construct(self):
        title = Text("物理模拟：自由落体", font_size=32, color=BLUE).to_edge(UP, buff=0.3)
        self.play(Write(title))

        # 地面
        ground = Line(LEFT*5 + DOWN*3, RIGHT*5 + DOWN*3, color=GRAY, stroke_width=4)
        self.add(ground)

        # 用标准 Manim 模拟自由落体效果（不依赖 manim-physics）
        ball = Circle(radius=0.2, color=RED, fill_opacity=0.8)
        ball.move_to(UP * 3 + LEFT * 3)

        # 用 ValueTracker + Updater 模拟重力
        y_tracker = ValueTracker(3)
        vy_tracker = ValueTracker(0)

        def update_ball(mob, dt):
            vy = vy_tracker.get_value() - 9.8 * dt  # g = 9.8 m/s²
            y = y_tracker.get_value() + vy * dt
            if y <= -3 + 0.2:  # 碰到地面
                y = -3 + 0.2
                vy = -vy * 0.7  # 弹性系数 70%
            vy_tracker.set_value(vy)
            y_tracker.set_value(y)
            mob.move_to(LEFT * 3 + UP * y)

        ball.add_updater(update_ball)
        self.add(ball)

        self.wait(4)  # 球自由落体 4 秒

        # 如果使用 manim-physics，上述 20 行可以简化为：
        # ball = Circle(radius=0.2).shift(UP*3+LEFT*3)
        # self.add(ball)
        # self.make_rigid_body(ball)
        # self.wait(4)

        self.play(FadeOut(VGroup(title, ground, ball)), run_time=1)
```

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| manim-slides | 自动检测段落边界，HTML 跨平台可交互 | 离线的 slides 文件较大（含视频帧） |
| manim-physics | 一行代码激活重力/碰撞/电场 | 物理精度受帧率限制（不是固定 dt 仿真） |
| networkx | 10+ 种布局算法、复杂网络模型 | 超大规模图（>10 万节点）的布局计算慢 |
| manim-voiceover | TTS + 自动字幕 + tracker.duration | 见第 27 章总结 |
| Jupyter 预览 | `%%manim` 魔术命令实时预览 | 更新延迟 1-2 秒（需重新渲染帧） |

### 适用场景

| 场景 | 说明 |
|------|------|
| 网络分析教学 | 社交网络、引文网络的可视化 |
| 教学课件 | 交互式 slides 替代线性视频 |
| 物理教学 | 力学、电磁学实验的简化模拟 |
| 数据科学 | Jupyter 中实时数据可视化 |
| 技术演讲 | 带旁白和字幕的演讲动画 |

**不适用场景**：需要高精度物理仿真的科研（使用专业仿真软件如 COMSOL）、需要超大规模实时渲染的 3D 游戏（使用 Unity/Unreal）。

### 注意事项

1. **插件版本锁**：在 `requirements.txt` 中锁定 `manim==0.20.1`、`manim-slides==5.1.6` 等，避免插件主版本不兼容。
2. **manim-slides 的段落**：`Slide.next_slide()` 必须在所有 `play()` 之后调用。如果在 play 中途调用，幻灯片会断在动画中间。
3. **manim-physics 不支持 3D**：`SpaceScene` 只在 2D 平面模拟物理。

### 常见踩坑经验

**故障一：networkx 的 spring_layout 每次运行结果不同**

根因：spring_layout 的初始位置是随机的。

解决：设置 `pos = nx.spring_layout(G, seed=42)` 固定随机种子。

**故障二：manim-slides 渲染的 HTML 在浏览器中空白**

根因：浏览器不允许本地文件访问视频资源（CORS 限制）。

解决：用 `python -m http.server 8000` 在 slides 目录启动本地服务器访问。

**故障三：manim-physics 和自定义 Updater 冲突**

根因：`make_rigid_body` 会给对象注册自己的 updater，如果你也注册了自定义 updater，两个 updater 可能互相覆盖。

解决：在 `make_rigid_body` 之前注册自定义 updater，或使用 `obj.clear_updaters()` 清理冲突。
