# 第25章：可复用组件与场景工程化

---

## 1. 项目背景

经过前 24 章的积累，读者已经掌握了 Manim 从基础到中级的大量技术。但随着项目规模的增长，一个新问题浮现了：**代码复用**。

团队成员小江统计了他们最近 10 个场景的代码，发现了惊人的重复率——"标题卡片"出现了 47 次，"步骤指示器"出现了 28 次，"代码块展示"出现了 15 次。每次都是复制粘贴前一个场景的代码，稍作修改。

更严重的是，当设计规范更新时（比如"标题卡片改用圆角 0.2、颜色改为 #4A90D9"），所有 47 个场景都要逐一修改。这已经不是动画制作的效率问题，而是软件工程的维护问题了。

本章要讲的核心技能是**组件化工程**——如何将重复的视觉元素封装为可复用的类（自定义 Mobject/Scene），如何在团队中共享和维护这些组件，以及如何设计稳定的组件 API。这标志着从"脚本编写者"到"动画工具开发者"的思维升级。

## 2. 剧本式交锋对话

> **场景**：小江在 Git 上打开了 10 个场景文件，并用 diff 工具高亮了重复代码——红彤彤一片。

**小胖**（捧着一桶爆米花）：

"小江你这个 diff 像圣诞树——红的绿的满屏都是。10 个场景 80% 的代码都一样？你们这是在写代码还是在做 Ctrl+C 大赛？"

**小江**（苦笑）：

"没办法。每个场景都需要标题、卡片、步骤指示器、注释框。我总不能每次从零写吧——复制粘贴是最快的。"

**小白**（已经打开了一个 `components.py` 文件）：

"你要的是**组件库**。Manim 的 `Mobject` 和 `Scene` 都是 Python 类——你可以继承它们创建自己的组件。比如一个 `TitleCard` 类，内部封装了 RoundedRectangle + Text + 动画逻辑，外部只暴露一个简洁的接口：

```python
class TitleCard(VGroup):
    def __init__(self, title, subtitle="", **kwargs):
        super().__init__(**kwargs)
        self.box = RoundedRectangle(
            width=10, height=1.8, corner_radius=0.15,
            color=PRIMARY, fill_opacity=0.08, stroke_width=2,
        )
        self.title = Text(title, font_size=36, color=PRIMARY, weight=BOLD)
        self.title.move_to(self.box.get_center() + UP * 0.2)
        self.subtitle = Text(subtitle, font_size=22, color=TEXT_MUTED)
        self.subtitle.next_to(self.title, DOWN, buff=0.15)
        self.add(self.box, self.title, self.subtitle)
```

"之后任何场景只需要 `card = TitleCard("线性代数", "第二章·矩阵运算")` ——一行代码替代原来的 15 行。"

**大师**（点头）：

"组件化的核心原则是——**对外封装复杂度，对内管理一致性**。好的组件有三个特征：

1. **构造简单**：参数不超过 5 个，有合理的默认值。
2. **自包含**：不依赖外部全局变量（颜色从 `theme.py` 读取而非硬编码），可以在任何 Scene 中独立使用。
3. **可组合**：一个 `VGroup` 可以包含 `TitleCard` + `StepIndicator` + `CodeBlock`，外层再统一做动画。"

> **技术映射**：继承 `VGroup` 或 `VMobject` 创建自定义组件。`__init__` 中创建子 Mobject 并 `self.add()` 加入自身。组件的 `construct` 动画逻辑可以封装为方法（如 `card.show(scene)`）。

**小胖**（爆米花撒了一腿）：

"那我还有个问题——这么多组件放在哪？总不能塞在一个 3000 行的文件里。"

**大师**：

"组件库的文件组织建议：

```
components/
├── __init__.py        # 统一导出
├── cards.py           # 标题卡片、信息卡片、答案卡片
├── indicators.py      # 步骤指示器、进度条
├── code_blocks.py     # 代码块、语法高亮封装
├── layouts.py         # 布局辅助类
└── transitions.py     # 转场动画封装
```

每个文件保持 100-200 行，`__init__.py` 负责导出所有组件。使用方只需要 `from components import TitleCard, StepIndicator`。"

#### 补充对话：组件 API 设计原则

**小胖**（爆米花撒了一腿）：

"那我还有个疑问——`show(scene)` 这个 API，为什么是传 scene 进去，而不是让组件自己拿全局 scene？为什么不用 `self.scene`？"

**小白**：

"这是为了避免**隐式依赖**。如果组件内部通过 `self.scene` 访问全局场景，那意味它只能用在特定的上下文里——换一个 scene 名字就炸了。通过参数显式传入，组件在任何 Scene 的子类中都能用，独立性强。"

**大师**：

"我再补充组件 API 的三条金科玉律：

1. **方法 = 动词**：`show()`、`hide()`、`highlight()`——暴露行为而非内部状态。
2. **参数 < 5**：超过 5 个参数时，用字典或 `**kwargs` 传递可选配置。`show(self, scene, scale=0.95, shift=UP*0.2, run_time=1.5)` —— 用默认值覆盖 90% 场景，需要时再传。
3. **返回 self**：让方法链成为可能——`card.to_edge(UP).show(self).highlight(2)`。

这三条让组件的使用体验逼近自然语言。"

> **技术映射**：Python 的 `**kwargs` 参数可以接受任意关键字参数并透传给底层 Manim API（如 `scene.play(..., **kwargs)`），使组件方法保持灵活性。

#### 步骤一：创建组件库

**步骤目标**：实现 `TitleCard`、`StepIndicator`、`InfoBox` 三个组件。

```python
# components/__init__.py
from .cards import TitleCard, InfoBox
from .indicators import StepIndicator, StepFlow

# components/cards.py
from manim import *
from constants.theme import *  # 假设 theme.py 在 PYTHONPATH 中
# 如果不方便导入，内联色板：
_PRIMARY = ManimColor("#4C9AFF")
_TEXT_MUTED = ManimColor("#94A3B8")

class TitleCard(VGroup):
    """标题卡片：标题 + 副标题 + 圆角矩形背景"""
    def __init__(self, title, subtitle="", width=10, height=1.8, **kwargs):
        super().__init__(**kwargs)
        self.box = RoundedRectangle(
            width=width, height=height, corner_radius=0.15,
            color=_PRIMARY, fill_opacity=0.08, stroke_width=2,
        )
        self.title_text = Text(title, font_size=36, color=_PRIMARY, weight=BOLD)
        self.subtitle_text = Text(subtitle, font_size=22, color=_TEXT_MUTED)

        if subtitle:
            self.title_text.move_to(self.box.get_center() + UP * 0.25)
            self.subtitle_text.next_to(self.title_text, DOWN, buff=0.15)
        else:
            self.title_text.move_to(self.box.get_center())

        self.add(self.box, self.title_text, self.subtitle_text)

    def show(self, scene, **play_kwargs):
        """播放显示动画"""
        scene.play(FadeIn(self, scale=0.95), **play_kwargs)
        return self

    def hide(self, scene, **play_kwargs):
        """播放隐藏动画"""
        scene.play(FadeOut(self), **play_kwargs)
        scene.remove(self)
        return self


class InfoBox(VGroup):
    """信息卡片：左侧竖条 + 文字"""
    def __init__(self, text, color=_PRIMARY, width=9, **kwargs):
        super().__init__(**kwargs)
        self.strip = Rectangle(
            width=0.08, height=1.5, color=color,
            fill_opacity=1, stroke_width=0,
        )
        self.text = Text(text, font_size=24, color=WHITE)
        self.text.next_to(self.strip, RIGHT, buff=0.3)
        self.strip.align_to(self.text, UP)
        self.add(self.strip, self.text)


# components/indicators.py
class StepIndicator(VGroup):
    """步骤指示器：圆形编号 + 标签"""
    def __init__(self, number, label, color=ManimColor("#F5A623"), **kwargs):
        super().__init__(**kwargs)
        self.circle = Circle(radius=0.35, color=color, fill_opacity=0.2)
        self.number_text = Text(str(number), font_size=22, color=color, weight=BOLD)
        self.number_text.move_to(self.circle.get_center())
        self.label = Text(label, font_size=20, color=GRAY)
        self.label.next_to(self.circle, DOWN, buff=0.2)
        self.add(self.circle, self.number_text, self.label)


class StepFlow(VGroup):
    """步骤流：多个 StepIndicator 水平排列"""
    def __init__(self, steps, **kwargs):
        """steps: list of (number, label_text)"""
        super().__init__(**kwargs)
        for num, label in steps:
            self.add(StepIndicator(num, label))
        self.arrange(RIGHT, buff=1.5)

    def show(self, scene, **play_kwargs):
        scene.play(
            LaggedStart(*[FadeIn(s, scale=0.5) for s in self],
                        lag_ratio=0.2),
            **play_kwargs,
        )
        return self
```

#### 步骤二：在场景中使用组件

**步骤目标**：在两个不同的 Scene 中复用组件，验证 API 一致性。

```python
# scenes/chapter25_lesson.py
from manim import *
from components import TitleCard, InfoBox, StepFlow

class LessonUsingComponents(Scene):
    def construct(self):
        # 标题卡片
        card = TitleCard("矩阵乘法", "第二章·矩阵与线性变换")
        card.to_edge(UP, buff=0.4)
        card.show(self)
        self.wait(0.5)

        # 步骤流
        steps = StepFlow([(1, "理解定义"), (2, "逐元计算"), (3, "验证结果")])
        steps.next_to(card, DOWN, buff=0.8)
        steps.show(self)
        self.wait(0.5)

        # 信息框
        info = InfoBox("矩阵乘法不满足交换律：AB ≠ BA 一般成立",
                       color=ManimColor("#F5A623"))
        info.next_to(steps, DOWN, buff=1.0)
        self.play(FadeIn(info, shift=UP * 0.3), run_time=1)
        self.wait(1.5)

        self.play(FadeOut(VGroup(card, steps, info)), run_time=1.5)
        self.wait(0.5)


class AnotherLesson(Scene):
    """另一个场景复用同一组件库"""
    def construct(self):
        card = TitleCard("导数与切线", "第三章·微积分基础")
        card.to_edge(UP, buff=0.4)
        card.show(self)
        self.wait(1)

        info = InfoBox("导数定义为函数变化率的极限值", color=ManimColor("#47D38D"))
        info.next_to(card, DOWN, buff=1.0)
        self.play(FadeIn(info, scale=0.95), run_time=1)
        self.wait(1.5)

        card.hide(self)
        self.play(FadeOut(info), run_time=1)
        self.wait(0.5)
```

# tests/test_components.py
"""组件库基本验证（手动运行）"""
from manim import *
from components import TitleCard, StepIndicator, InfoBox, StepFlow

def test_title_card():
    """验证 TitleCard 可以创建并包含子对象"""
    card = TitleCard("测试", "副标题")
    assert len(card.submobjects) >= 2  # box + title
    return "PASS"

def test_step_indicator():
    """验证 StepIndicator 可以创建"""
    step = StepIndicator(1, "测试步骤")
    assert len(step.submobjects) >= 1
    return "PASS"

def test_step_flow():
    """验证 StepFlow 排列正确"""
    flow = StepFlow([(1, "A"), (2, "B"), (3, "C")])
    assert len(flow.submobjects) == 3
    return "PASS"

if __name__ == "__main__":
    for name, func in [("TitleCard", test_title_card),
                        ("StepIndicator", test_step_indicator),
                        ("StepFlow", test_step_flow)]:
        try:
            result = func()
            print(f"[{result}] {name}")
        except Exception as e:
            print(f"[FAIL] {name}: {e}")

## 4. 项目总结

### 优点 & 缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 代码复用 | 一行替代 15 行，修改一处全局生效 | 组件 API 设计不当会导致"修改一个参数，破坏 10 个场景" |
| 视觉一致性 | 所有场景的标题/卡片/步骤完全一致 | 一致性强也意味着"个性弱"，特殊场景需要单独覆盖 |
| 测试能力 | 组件可用 Python 单元测试验证 | Manim 的 Scene 对象不适合单元测试（需要完整渲染） |
| 团队协作 | 组件库是设计规范的可执行版本 | 需要组件文档和示例，否则新人不知道有哪些组件 |
| 扩展性 | 继承 VGroup 后可像普通 Mobject 一样操作 | 嵌套组件树过深会影响渲染性能 |

### 适用场景

| 场景 | 说明 |
|------|------|
| 课程系列 | 10+ 集的系列课程需统一风格 |
| 多人协作 | 不同开发者共用同一套组件 |
| 品牌规范 | 企业品牌色和设计规范的代码化 |
| 快速原型 | 用组件快速拼装场景，验证教学逻辑 |
| 开源项目 | 发布组件库供社区使用 |

**不适用场景**：单次使用的临时实验动画（工程化代价大于收益）、需要高度定制视觉风格的艺术动画。

### 注意事项

1. **组件的 `__init__` 中不要播放动画**：组件构造只负责创建 Mobject 结构，动画逻辑放在方法（`show()`）中。这遵循"创建与播放分离"原则。
2. **组件不持有 Scene 引用**：组件应该不依赖全局 Scene 对象。`show(scene)` 方法接收 Scene 作为参数，保持组件独立性。
3. **颜色从 theme.py 单向导入**：组件库只是 `theme.py` 的消费者。不要在组件中修改颜色常量，避免副作用。

### 常见踩坑经验

**故障一：组件在 show 前调用 `add` 到 Scene 导致基类初始化异常**

根因：`show` 方法中的 `FadeIn` 依赖对象已被 Scene 知晓，但如果组件构造时 Scene 尚未完成 setup，对象可能在未完全初始化的上下文中。

解决：在 `show` 开头调用 `scene.add(self)` 确保对象在 Scene 中。

**故障二：跨文件导入组件报 `ModuleNotFoundError`**

根因：Manim 的 Python 脚本执行目录与 `components/` 的相对路径不一致。

解决：在 `manim.cfg` 中配置 `[custom_folders]` 或使用 `sys.path.insert(0, str(PROJECT_ROOT))`。

**故障三：修改组件后所有场景需要重新渲染才能看到效果**

根因：Manim 没有"热加载"机制——代码改了必须重新运行。

解决：用 `build.py` 批量渲染，或只在组件库修改后批量重建。
