## 2. 剧本式交锋对话

> **场景**：老赵在评审会上发布了基础篇综合实战任务。桌上放着一份需求文档和一部样片。

**小胖**（嘴里叼着棒棒糖，翻着需求文档）：

"老赵你这不是为难我吗？从零做一个完整的科普视频，万一我把 15 章学的全忘光了怎么办？我现在连 `play` 和 `wait` 的顺序都要想半天。"

**老赵**（指了指墙上的"基础篇知识地图"）：

"所以才要有这张图。任何一段 Manim 动画都可以用 5 个问题来解构——

1. **这段画面展示什么？** → 创建什么 Mobject
2. **它们怎么出现？** → 用什么 Animation
3. **它们放在哪？** → to_edge / next_to / arrange
4. **这段持续多久？** → run_time + wait
5. **这段结束后画面怎么收？** → FadeOut + remove

你把每一段用这 5 个问题过一遍，代码自然就出来了。忘掉公式没关系，用这张决策树推动思考。"

**小白**（手里已经摊开了一个镜头脚本表格）：

"我建议在做代码之前，先用第 15 章学的镜头脚本模板把整个视频的每一帧规划出来。我的经验是——花 30 分钟写镜头脚本，省下 3 小时的返工。"

**大师**（点头）：

"对。基础篇综合实战的目的不是考察你'记了多少 API'，而是看你**能不能把想法翻译成画面，再把画面翻译成代码**。具体做法分四步：

第一步——**选题拆解**：选一个数学主题，用文字描述它的证明/推导过程，拆成 4-6 个自然段落。

第二步——**镜头脚本**：为每个段落画四列表格（时间戳/画面/旁白/动画效果），确保每段只讲一个点。

第三步——**分段编码**：每段对应代码中的一个 `# ===== 第 N 段 =====` 注释块，逐段写、逐段测试。

第四步——**集成验收**：把 4-6 段拼成一个完整的 Scene，用第 12 章的 `build.py` 渲染高清终版，用第 15 章的三道验收题做质量检查。"

> **技术映射**：基础篇综合实战本质是"软件工程"中最小的瀑布式开发流程——需求（选题）→ 设计（镜头脚本）→ 实现（分段编码）→ 测试（验收）。

**小胖**（放下棒棒糖，拿出笔记本）：

"行，那我选什么主题？圆的面积？勾股定理？我总不能选个太难的自己都推导不明白。"

**大师**：

"选题原则——**选你自己能讲明白的**。建议圆的面积公式推导，它天然有四段视觉过程：圆 → 切分成扇形 → 重新排列 → 逼近长方形。每段有清晰的视觉对应，非常适合 Manim 表达。"

### 3.2 镜头脚本设计

| 时间戳 | 画面内容 | 旁白/字幕要点 | 动画效果 | 章节 |
|--------|----------|-------------|----------|------|
| 0-8s | 标题"圆的面积公式推导" + 副标题 | "如何推导 S = πr²？" | Write 逐字 + FadeIn | 第1段 |

### 3.3 分段编码

#### 第1段：开场标题（0-8s）

```python
# ==================== 第 1 段：开场 ====================
title = Text("圆的面积公式推导", font_size=48, color=PRIMARY, weight=BOLD)
title.to_edge(UP, buff=0.5)

subtitle = Text("从切分到重组 —— 一个优雅的几何证明",
               font_size=24, color=TEXT_MUTED)
subtitle.next_to(title, DOWN, buff=0.3)

self.play(Write(title), run_time=1.5)
self.play(FadeIn(subtitle, shift=UP * 0.2), run_time=1)
self.wait(0.8)
self.play(FadeOut(subtitle), run_time=0.5)
```

#### 第2段：绘制圆与半径标注（8-25s）

```python
# ==================== 第 2 段：圆 + 半径 ====================
self.play(title.animate.to_corner(UR).scale(0.55), run_time=1)

circle = Circle(radius=2.5, color=YELLOW, stroke_width=3)
circle.move_to(ORIGIN + DOWN * 0.3)

# 圆心
center_dot = Dot(circle.get_center(), color=WHITE, radius=0.05)

# 半径线（从圆心到圆周水平右侧）
radius_line = Line(circle.get_center(),
                    circle.get_center() + RIGHT * 2.5,
                    color=GRAY, stroke_width=2)
r_label = MathTex("r", font_size=36, color=GRAY)
r_label.next_to(radius_line.get_center(), DOWN, buff=0.15)

self.play(Create(circle), run_time=2)
self.play(FadeIn(center_dot), Create(radius_line), Write(r_label), run_time=2)
self.wait(1)
```

#### 第3段：切分扇形（25-55s）

**步骤目标**：展示圆被 8 等分的切割过程，每瓣扇形用不同颜色区分。

```python
# ==================== 第 3 段：切分成 8 个扇形 ====================
n_slices = 8
slices = VGroup()
slice_colors = [RED, ORANGE, YELLOW, GREEN, TEAL, BLUE, PURPLE, PINK]

for i in range(n_slices):
    start_angle = i * TAU / n_slices
    end_angle = (i + 1) * TAU / n_slices
    sector = Sector(
        radius=2.5, start_angle=start_angle, angle=TAU / n_slices,
        color=slice_colors[i], fill_opacity=0.6, stroke_width=1.5,
        arc_center=circle.get_center(),
    )
    slices.add(sector)

# 先让扇形边缘可见（分割线）
dividers = VGroup()
for i in range(n_slices):
    angle = i * TAU / n_slices
    div = Line(
        circle.get_center(),
        circle.get_center() + np.array([np.cos(angle), np.sin(angle), 0]) * 2.5,
        color=WHITE, stroke_width=1, stroke_opacity=0.5,
    )
    dividers.add(div)

# 移除之前的圆、半径和圆心
self.play(FadeOut(radius_line), FadeOut(r_label), FadeOut(center_dot), run_time=0.5)

# 展示分割线
self.play(LaggedStart(*[Create(d) for d in dividers], lag_ratio=0.15), run_time=2)
self.wait(0.3)

# 扇区依次着色出现
self.play(
    LaggedStart(*[FadeIn(s) for s in slices], lag_ratio=0.15),
    FadeOut(circle),
    run_time=2,
)
self.wait(1)
```

#### 第4段：扇形重组为近似矩形（55-80s）

**步骤目标**：将 8 个扇区搬运到右侧，交替翻转拼成矩形。

```python
# ==================== 第 4 段：重组为矩形 ====================
self.play(FadeOut(dividers), run_time=0.5)

# 将 8 个扇区搬运重组
# 奇数索引翻转向下，偶数保持向上，交替排列
rearranged = VGroup()
target_x = RIGHT * 2.5
for i in range(n_slices):
    s = slices[i].copy()
    if i % 2 == 1:
        s.rotate(PI, about_point=s.get_arc_center())
    s.move_to(target_x + (i - n_slices / 2 + 0.5) * RIGHT * 0.55 + UP * 0.2)
    rearranged.add(s)

self.play(
    ReplacementTransform(slices, rearranged),
    run_time=3,
)
self.wait(0.5)

# 矩形边框
rect = Rectangle(
    width=4.5, height=2.5,
    color=WHITE, stroke_width=1.5, stroke_opacity=0.4,
)
rect.move_to(rearranged.get_center())
self.play(Create(rect), run_time=1)
self.wait(1)
```

#### 第5段：标注矩形尺寸（80-105s）

```python
# ==================== 第 5 段：矩形尺寸标注 ====================

# 宽度标注：πr（半周长）
width_brace = Brace(rect, direction=DOWN, color=TEXT_MUTED)
width_label = MathTex(r"\pi r", font_size=36, color=GRAY)
width_label.next_to(width_brace, DOWN, buff=0.15)

# 高度标注：r
height_brace = Brace(rect, direction=RIGHT, color=TEXT_MUTED)
height_label = MathTex("r", font_size=36, color=GRAY)
height_label.next_to(height_brace, RIGHT, buff=0.15)

self.play(Create(width_brace), Write(width_label), run_time=1.5)
self.wait(0.3)
self.play(Create(height_brace), Write(height_label), run_time=1.5)
self.wait(1)

# 标注文字
note = Text("宽 = 半周长 = πr\n高 = 半径 = r",
            font_size=22, color=TEXT_SECONDARY, line_spacing=0.5)
note.to_corner(DL, buff=0.5)
self.play(Write(note), run_time=1.5)
self.wait(1)
```

#### 第6-7段：面积推导与结论（105-150s）

```python
# ==================== 第 6 段：面积公式推导 ====================
self.play(FadeOut(note), run_time=0.5)

# 矩形面积公式
area_formula = MathTex(
    r"S", r"=", r"\pi r", r"\times", r"r", font_size=48,
)
area_formula[0].set_color(PRIMARY)   # S
area_formula[2].set_color(YELLOW)    # πr
area_formula[4].set_color(GREEN)      # r
area_formula.next_to(rect, DOWN, buff=1.5)

self.play(Write(area_formula), run_time=2)
self.wait(0.5)

# 化简
simplified = MathTex(r"S", r"=", r"\pi r^2", font_size=56)
simplified[0].set_color(PRIMARY)
simplified[2].set_color(ACCENT)
simplified.move_to(area_formula.get_center())

self.play(TransformMatchingTex(area_formula, simplified), run_time=2)
self.wait(0.5)

# ==================== 第 7 段：结论 ====================
final_box = SurroundingRectangle(simplified, color=ACCENT, buff=0.2)
self.play(Create(final_box), run_time=1)

conclusion = Text("圆的面积公式得证", font_size=32, color=ACCENT)
conclusion.next_to(final_box, DOWN, buff=0.6)
self.play(Write(conclusion), run_time=1.5)
self.play(Indicate(simplified, color=YELLOW, scale_factor=1.1), run_time=1.5)
self.wait(2)

# 收束
self.play(FadeOut(VGroup(title, rearranged, rect,
    width_brace, width_label, height_brace, height_label,
    simplified, final_box, conclusion)), run_time=2)
self.wait(0.5)
```

from manim import *
import numpy as np

# 内联色板（对应 theme.py）

### 注意事项

1. **`Sector` 的弧心必须显式指定**：`Sector(radius=..., arc_center=...)`。如果不指定 `arc_center`，扇形默认以 ORIGIN 为中心创建。
2. **`ReplacementTransform` 和 `copy()` 配合**：本例中 `rearranged` 是通过 `slices[i].copy()` 创建的，确保原始扇区和重组扇区是不同对象，`ReplacementTransform` 才能正常工作。
3. **画面元素守衡**：每段结束时清理不再需要的元素（FadeOut + remove），避免越往后画面越拥挤。

### 常见踩坑经验

**故障一：扇形重组后颜色变暗/异常**

根因：`copy()` 后旋转了 180°，但颜色没有正确保留（部分颜色空间在旋转后重新计算）。

解决：在 `copy()` 后立即调用 `.set_color(slices[i].get_color())` 显式设置。

**故障二：`TransformMatchingTex` 报 token 不匹配**

根因：`S = \pi r \times r` 和 `S = \pi r^2` 的 token 数差异较大。

解决：确实可能出现匹配退化。如果报错，改用 `ReplacementTransform` 替代。

**故障三：大场景渲染失败（FFmpeg 内存溢出）**

根因：2 分半 × 30fps = 4500 帧，高清渲染需要大量内存存放临时帧。

解决：增大系统临时文件空间，或设 `--write_to_movie=true --disable_caching` 跳过帧缓存。

### 思考题

1. 扩展本场景，将扇区数量从 8 增加到 16，观察"近似矩形"的视觉变化。当 n → ∞ 时矩形是否越来越精确？提示：将 n_slices 改成一个函数参数，批量渲染 n=4, 8, 16, 32 四个版本。

2. 改造本场景的扇形重组算法，用 `self.play(slices[i].animate.rotate(PI).move_to(target))` 而不是 `copy()`，实现"扇区在原位翻转并平移"的效果（视觉上更像真实的切分-翻转-拼接操作）。提示：`rotate` 的 `about_point` 参数应设为扇区的弧心。
