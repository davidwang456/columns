# 第11章：生产案例复盘（综合）-导航栈调优一页纸方法论

> 本章目标字数：3000–5000。统一环境见 [ENV.md](../ENV.md)。

## 1 项目背景

### 业务场景

**现象**：机器人**转角震荡**、**全局路径频繁重算**。一页纸方法论：**观察 → 分层假设 → 最小实验 → 数据闭环**。

### 痛点放大

1. **只盯单点参数**：看到晃动就调控制器，忽略传感器鬼影、TF 跳变与 costmap 闪烁。
2. **复盘没有证据链**：会议里只有口头描述，没有 bag、参数 diff、版本向量与日志截图。
3. **修复不可防再发**：当天靠经验调好，下一次换货架材质、地图分辨率或 RMW 版本又复现。

### 模板（可直接用于复盘会议）

| 层级 | 查什么 | 工具 |
|------|--------|------|
| 传感器 | 时间戳/丢帧 | `ros2 topic hz`, `drop rate` |
| TF/位姿 | 跳变 | `tf2_monitor` |
| 地图/代价 | 膨胀/分辨 | RViz layers |
| 规划 | 搜索耗时 | planner logs |
| 控制 | 跟踪误差 | `controller_server` metrics |

## 2 项目设计

### 剧本对话

**小胖**：先调控制器 PID 行不行？销售说「客户只抱怨晃」。

**小白**：**全局路径频繁重算**，我先 git 回滚 **nav2** 上一版？

**大师**：先分层：**现象→测量→假设→最小实验**。若 **定位 covariance** 在静止时仍抖，或 **激光数据**有镜面鬼影，**costmap 局部障碍**会闪 —— **controller** 只是在追一条**每秒都在变的伪障碍边界**。此时调 **PID** 只会让震荡频率变化。**盲目回滚**可能掩盖**环境变更**（新货架材质）。正确顺序：**传感器/TF → 地图与层 → planner → controller**。

**技术映射**：**排障栈** = **由下（观测）而上（决策）**。

---

**小胖**：一页纸模板谁写？写太细开会吵死人。

**大师**：模板的价值是**强制对齐证据**：每条假设都挂 **`ros2 topic hz`、bag 片段、参数 diff**。没有数据的讨论叫**玄学复盘**。

---

**大师**：**根因**找到后，定义 **防再发**：是 **SOP（贴膜/清洁）**、**参数默认**还是 **算法鲁棒**——三者成本不同。

## 3 项目实战

本节不提供「可一键跑」的独立仓库，而给出与 **B04** 同构的 ****SOP：分层实验**——你可把客户现场或 **Gazebo** 任意 **Nav2** 场景代入。

### 环境准备

- **ROS 2 Humble** + **Nav2** 已能 **`ros2 launch ...`** 起栈（参见 [M04](第29章：Nav2 栈概览与行为树入门.md)）。
- 准备一个 **rosbag**（或**录屏+参数快照**），至少含 **`/scan` 或点云**、**`/tf`**、**`/cmd_vel`** / **`odom`**。
- 工具：**`ros2 topic hz`**、**RViz2**、**rqt**（按需）。

### 分步实现

#### 步骤 1：冻结现场向量

- **目标**：复盘前 **锁版本** —— 否则「**当时**」不可复现。
- **操作**：记录 **`ros2 doctor --report`**、**`apt list --installed | grep ros-humble`** 一段、**所有 YAML git hash**、**机器人 URDF commit**。
- **预期输出**：一页 **「版本向量」**（与 [A02](第02章：rmw 与 DDS 实现切换（Fast-DDS-Cyclone 等）.md)、[A10](第10章：SRE-故障演练、SLA、告警与回滚.md) 一致）。
- **坑与解法**：客户口头「最新」→ 强制要 **镜像 digest** 或 **deb 列表**。

#### 步骤 2：传感器层 — 镜面/丢帧

- **目标**：排除 **鬼影障碍** 再碰控制器。
- **命令/操作**：

```bash
ros2 topic hz /scan
# RViz：LaserScan 点云是否尖刺、是否随车身节奏闪动
```

- **预期输出**：若 **scan** 在**静止车体**时仍剧烈跳变 → **优先怀疑镜面/玻璃/雨雾**。
- **坑与解法**：物理 **贴膜/清洁** 往往比调 **`inflation_radius`** 便宜（剧本）。

#### 步骤 3：TF / 定位层

- **目标**：确认 **map→odom→base** 无 **阶跃**。
- **命令**：

```bash
ros2 run tf2_ros tf2_echo map base_link
# 或按现场 frame 名调整
```

- **预期输出**：**平移/旋转** 连续变化；若 **跳变** → 查 **定位、传感器时间戳**（[M06](第31章：传感器驱动与标定流程.md)）。
- **坑与解法**：**TF 与 Costmap** 参数打架 → 回到 [B09](第21章：TF2-坐标系与静态变换.md)。

#### 步骤 4：代价地图与规划层

- **目标**：用 **RViz** 图层看 **局部障碍**是否闪烁。
- **操作**：逐层关闭 **Voxel / Inflation**，对比 **global plan** 是否仍每秒重算。
- **预期输出**：**局部闪烁**而全局静态 → **感知/动态层**；**全局也抖** → **定位或 map**。
- **坑与解法**：盲调 **DWA 权重** → 剧本已否。

#### 步骤 5：控制层 — 最后才动 PID**

- **目标**：**`cmd_vel` 震荡** 时记录 **角速度方差**。
- **命令**（话题名按现场）：

```bash
ros2 topic echo /cmd_vel --no-arr
# 手工采 30s 样本 或 用 rosbag2 录一段 offline 统计
```

- **预期输出**：**前四层无异常** 时，再比较 **PID/约束** 改动前后 **方差**。
- **坑与解法**：无 **bag** 对比 → **无法向客户证明**「我们修了」。

### 完整代码清单

- **虚构案例**（供写作参考）：**激光镜面反射 → 局部障碍闪烁 → 控制器震荡**；**根因**：货架不锈钢；**验证**：**哑光处理 + obstacle 层滤波**；**指标**：**`cmd_vel` 角速度方差** ↓（示例 **70%**）。
- **真实项目**：以你司 **ticket 模板** 附 **`rosbag` + 参数 diff**。

### 测试验证

- **验收**：能填写文中 **「模板表」**每一行并给出**引用证据**（log/bag/截图）。
- **防再发**：对应 **SOP / 默认参数 / 算法** 三选一 owner（见剧本）。

## 4 项目总结

### 优点与缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 工程价值 | **生产案例复盘（综合）：导航栈调优一页纸方法论** 能把隐性经验显式化，便于新人复现与团队协作。 | 需要配套环境、日志与记录表，否则容易停留在概念层。 |
| 可维护性 | 通过标准命名、参数、Launch 或测试约束降低沟通成本。 | 规则一多会增加初期学习曲线。 |
| 可观测性 | 便于用 CLI、日志、bag 或监控指标定位问题。 | 指标若没有业务阈值，仍可能变成“看起来很多但不能决策”。 |
| 扩展性 | 可与前后章节串联，逐步走向真实系统。 | 跨 RMW、跨发行版或跨硬件时需要重新验证边界。 |

### 适用场景

- 团队需要把 **生产案例复盘（综合）：导航栈调优一页纸方法论** 从个人经验沉淀为可复用流程。
- 新人、测试与运维需要用同一套命令与术语对齐问题现象。
- 项目进入联调阶段，需要记录参数、话题、日志与验收结果。
- 需要为后续源码阅读、性能优化或生产复盘提供上下文。

### 不适用场景

- 只做一次性演示且不需要交接、回归或复盘的临时脚本。
- 现场约束尚未明确时，不宜把 **生产案例复盘（综合）：导航栈调优一页纸方法论** 的示例参数直接当作生产标准。

### 注意事项

- **版本兼容**：所有命令以 Humble 与 [ENV.md](../ENV.md) 为基线，其他发行版需查 `--help` 与官方文档。
- **配置边界**：不要把实验参数直接带入生产；先记录硬件、RMW、QoS、网络与时钟条件。
- **安全边界**：涉及远程调试、容器权限、证书或硬件接口时，先按最小权限原则收敛。

### 常见踩坑经验

1. **只看现象不记录环境**：同一命令在不同 RMW、Domain、QoS 或硬件上结果不同。根因通常是缺少版本与环境快照。
2. **一次改多个变量**：参数、Launch、网络与代码同时变化，导致无法归因。解决方法是每次只改一项并保存日志或 bag。
3. **忽略跨角色交接**：开发能跑通但测试/运维无法复现。根因是缺少最小验收命令、预期输出与失败处理路径。
