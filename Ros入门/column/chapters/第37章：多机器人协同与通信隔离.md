# M12 · 多机器人协同与通信隔离

> 本章目标字数：3000–5000。统一环境见 [ENV.md](../ENV.md)。

## 1 项目背景

### 业务场景

仓储 **中央调度** 下发任务到多车：需 **fleet_msgs**、**共享地图**、**互斥区域（mutex zone）** 或 **交通管制**。ROS 2 不内置「车队」，但可组合 **topic + service + DDS 域** + **外部调度器**。

### 痛点放大

1. **单机 Nav2** 直接复制 N 份 → CPU 爆。
2. **全局互知 pose** 需要 **fleet bridge**。
3. **安全交叉路口** 仅靠 local planner 不够。

**本章目标**：架构分层图；**多域 vs 多 namespace** 选型；引用 **Open-RMF**（概念）。

## 2 项目设计

### 剧本对话

**小胖**：多车协同 = 多发几个 **`/goal`** 吗？

**小白**：**中央调度**和 **分布式各走各的**，哪种更香？

**大师**：多机器人本质是**资源争用**：**空间（通道）**、**充电桩**、**任务优先级**。光有 ROS graph **不能天然避碰**——要么**上层调度器**给任务时窗与路径预约，要么**分布式共识**（难度高）。实践常见分层：**域/namespace 隔离研发噪声** → **fleet API（HTTP/GRPC）发任务** → **单车 Nav2 本地执行**。**Open-RMF** 给的是**参考架构**，不是开箱即用产品经理按钮。

**技术映射**：**Fleet 层** = **业务编排**；**ROS 层** = **单车执行与传感**。

---

**小胖**：**互斥区**在 ROS 里咋表达？搞个 **Service 抢锁**？

**大师**：可以是 **`/lock/zone_A`** 一类的 **分布式锁服务**（要考虑**超时/死锁**），也可以用**上层 MES** 裁决——关键是**谁在失效时释放锁**（车趴窝、通信断）。

---

**大师**：先在仿真里起 **两车 + 两 namespace**，再谈**协同算法**——否则 demo 永远停在 PPT。

## 3 项目实战

### 环境准备

与 [ENV.md](../ENV.md) 一致：**Ubuntu 22.04 + ROS 2 Humble**；**仿真** 建议 **两台 TB3** 或 **多实例 Gazebo**（资源要求高时可 **分两台机 + 同域**）。

本章额外依赖：按仿真选型安装 **`turtlebot3_gazebo`** / **`nav2_bringup`**；**Open-RMF** 为 **可选阅读**（不强制 apt）。

### 分步实现

#### 步骤 1：两车不同 `ROS_DOMAIN_ID`（隔离）

- **目标**：体验 **DDS 域** 作为 **硬隔离**（与 [M01](第26章：DDS 发现、域（Domain）与跨机通信.md) 呼应）。
- **命令**：**终端 A** `export ROS_DOMAIN_ID=0` 起 **车 1**；**终端 B** `export ROS_DOMAIN_ID=1` 起 **车 2**。
- **预期输出**：**两车互不可见**（默认）；各自 **`ros2 topic list`** 独立。
- **坑与解法**：需要 **协同** 时 **域隔离反成障碍** —— 进入 **步骤 2**。

#### 步骤 2：桥接节点（练习级）

- **目标**：在 **应用层** 把 **域 A 的 topic** 转发到 **域 B**（或汇总到 **中央**），理解 **「桥接 = 新信任边界」**。
- **命令**：写 **最小 rclpy 节点**：**订阅域 0** 的 `/scan`（需 **独立进程+环境**）再 **发布到域 1**（通常 **两进程 + 命名管道/SHM** 更简单 —— 本章练习可用 **两终端各一域 + relay 脚本** 简化）。
- **预期输出**：**域 1** 侧能看见 **转发后的 `/relay/scan`**。
- **坑与解法**：**延迟、时钟、QoS** 全需重做 —— **生产** 多用 **同一域 + namespace**（[M03](第28章：命名空间、重映射与多实例部署.md)）或 **RMW 级方案**。

#### 步骤 3：同域多机器人 **namespace**

- **目标**：**推荐路径**：**同 `ROS_DOMAIN_ID`** + **`/bot1`、`/bot2` 前缀**（本章正文）。
- **命令**：**`PushRosNamespace`** 启动两套 **Nav2**（[M03](第28章：命名空间、重映射与多实例部署.md)、[M04](第29章：Nav2 栈概览与行为树入门.md)）。
- **预期输出**：`ros2 topic list` **前缀清晰**；**TF** 使用 **frame_prefix** 避免冲突。
- **坑与解法**：**只改 topic 不改 TF** → **RViz 全乱**。

#### 步骤 4（课外）：Open-RMF

- **目标**：了解 **交通调度** 与 **ROS 图** 的边界。
- **命令**：阅读 **Open-RMF** **traffic editor** 入门；**不强制**跑通。
- **预期输出**：能说出 **何时需要 RMF** vs **仅 Nav2**。
- **坑与解法**：**部署复杂** —— **评估成本** 后再引入。

### 完整代码清单

- **`relay_bridge.py`**（占位）：**双域转发** 演示（**仅实验**）。
- **`multi_bot.launch.py`**（占位）：**双 namespace** 仿真入口。
- Git 占位：**待补充**。

### 测试验证

- **手工验收**：**同域 + 双 namespace** 下 **两车 topic 不串**；**异域** 下 **默认不互通**。
- **可选**：**中央调度 vs 分布式** 各写 **一条** 适用场景（呼应思考题）。
