# 第39章：多机多卡训练、容错与成本优化

## 1. 项目背景

算法团队正在训练一个 GPT-2 级别的文本生成模型（1.5B 参数），需要 8 张 A100 (80GB) 训练约两周。公司内部 Kubernetes 集群有一个 GPU 资源池，但面临三个现实难题：

- **机器不是 24/7 独占的**：白天有人用，晚上空闲——训练任务需要"弹性伸缩"，自动利用闲置 GPU
- **机器会故障**：Spot/Preemptible 实例随时可能被回收——任务必须能从 checkpoint 恢复而不丢失太多进度
- **预算有限**：全量用按需实例（On-demand）太贵，需要混用按需 + 竞价实例来降低成本

架构师老周设计了一个方案——用 MultiWorkerMirroredStrategy 做多机多卡同步训练，配合 K8s Job + 自动 checkpoint 恢复 + Volcano 调度器实现弹性训练。但上线的第一周就遇到问题：一个节点在凌晨 3 点被回收，训练停了 4 个小时直到人工介入才恢复；另一次 4 台 A100 并行训练，速度只比 2 台快 30%（理想是 2x）——网络带宽瓶颈。

**痛点放大**：多机多卡训练不只是"把 MirroredStrategy 换成 MultiWorkerMirroredStrategy"——它涉及节点间通信效率、故障恢复、资源调度、成本核算等系统工程问题。生产级训练需要"容错 + 弹性 + 成本最优"三位一体。

## 2. 项目设计

**小胖**（看着 K8s YAML 文件发懵）：多机多卡不就是 MirroredStrategy 加了 Multi 前缀吗？为什么还要搞 K8s、Volcano、弹性训练这一大堆？

**大师**：因为 MirroredStrategy 假设你有一个"永远不坏的 4 GPU 机器"。真实的生产环境里——机器是共享的、可能被抢占的、可能网络抖动的。你需要在四个维度上做好"生产级加固"：

**技术映射**：多机多卡训练 = 同步更新算法 + 节点间通信 + 故障恢复 + 资源编排。MultiWorkerMirroredStrategy 解决前两个，checkpoint+编排解决后两个。

**小白**：通信这部分——AllReduce 的 Ring 和 Hierarchical Copy 有什么区别？为什么 4 机训练可能比 2 机只快了 30%？

**大师**：AllReduce 有三种主流策略：

- **Ring AllReduce**：N 个节点形成环，每个节点只和前后邻居通信。通信量 = 2(N-1)/N × 数据量。适合节点数多（如 > 4）的场景。
- **Hierarchical Copy**：先在机内（NVLink 高速）聚合，再在机间（网络）聚合。如果机内有 8 张卡，机间只有 2 台机器——Hierarchical 比 Ring 更高效（利用 NVLink 的 600GB/s 带宽先做本地聚合）。
- **Recursive Halving-Doubling**：分治递归的 AllReduce。前半程"折半"，后半程"倍增"。

**大师**：你 4 机 32 卡的速度只比 2 机 16 卡快了 30%——说明瓶颈在网络带宽（机间网络 10Gbps 远低于 NVLink 600GB/s）。优化手段：(1) 加大 batch_size → 减少参数同步频率；(2) 用梯度压缩（如 1-bit SGD）降低通信量；(3) 机内多卡优先用 NVLink 聚合再跨机。

**技术映射**：Ring AllReduce 适合纯同质节点；Hierarchical AllReduce 适合"机内 NVLink + 机间网络"的层次结构；梯度压缩以精度换带宽。

**小胖**：容错呢？训练了 3 天，第 4 天节点挂了，不能前功尽弃吧？

**大师**：容错三步走：

1. **Checkpoint 保全**：每个 epoch 或每 N step 保存完整的 checkpoint（模型权重 + 优化器状态 + 训练步数）。用 `tf.train.CheckpointManager` 管理。
2. **Preemption 检测**：K8s 在回收 Pod 前会发 SIGTERM 信号——训练脚本捕获这个信号，在退出前紧急保存一次 checkpoint。
3. **自动恢复**：Job 重新调度后，`tf.train.latest_checkpoint()` 找到最新 checkpoint，恢复权重 + 优化器状态，从断点继续。

4. **幂等训练**：恢复后 skip 已经训练过的 step，确保不会"重复训练"某些数据导致 bias。

**小白**：成本怎么算？按需实例 vs Spot 实例差了多少？

**大师**（拿计算器）：按需 A100 大约 $3/h，Spot 大约 $1/h。训练两周 = 336 小时 × 8 卡。纯按需：$3 × 8 × 336 = $8,064。纯 Spot：$1 × 8 × 336 = $2,688。但 Spot 可能被回收——如果有 10% 的训练时间被浪费在"中断→恢复"上，实际成本约 $2,988。还是比按需便宜 63%。

**步骤一：MultiWorkerMirroredStrategy 训练模板**

目标：展示多机训练的标准代码模板（单机模拟多 Worker）。

```python
import tensorflow as tf
import os
import json
import numpy as np

# === 多机训练配置（通过 TF_CONFIG 环境变量） ===
# 实际生产环境由 K8s Operator 自动注入
def get_tf_config_example():
    """模拟 TF_CONFIG（实际由 K8s/调度器注入）"""
    tf_config = {
        "cluster": {
            "worker": [
                "worker-0.example.com:12345",
                "worker-1.example.com:12345",
                "worker-2.example.com:12345",
                "worker-3.example.com:12345",
            ],
        },
        "task": {"type": "worker", "index": 0},  # 当前 Pod 的身份
    }
    return tf_config

# === 训练脚本 ===
def create_distributed_model(global_batch_size, learning_rate):
    """在 strategy.scope() 内创建模型"""
    strategy = tf.distribute.MultiWorkerMirroredStrategy()

    with strategy.scope():
        model = tf.keras.Sequential([
            tf.keras.layers.Dense(1024, activation="relu", input_shape=(512,)),
            tf.keras.layers.Dense(1024, activation="relu"),
            tf.keras.layers.Dense(10, activation="softmax"),
        ])

        # 学习率缩放：lr = base_lr * sqrt(num_workers)
        scaled_lr = learning_rate * np.sqrt(strategy.num_replicas_in_sync)

        model.compile(
            optimizer=tf.keras.optimizers.Adam(scaled_lr),
            loss="sparse_categorical_crossentropy",
            metrics=["accuracy"],
        )
    return model

# 训练（单机模拟）
strategy = tf.distribute.MultiWorkerMirroredStrategy()
print(f"副本数 (单机模拟): {strategy.num_replicas_in_sync}")

with strategy.scope():
    model = tf.keras.Sequential([
        tf.keras.layers.Dense(512, activation="relu", input_shape=(256,)),
        tf.keras.layers.Dense(10, activation="softmax"),
    ])
    model.compile(optimizer="adam", loss="sparse_categorical_crossentropy",
                  metrics=["accuracy"])

X = np.random.randn(1000, 256).astype(np.float32)
y = np.random.randint(0, 10, 1000)

# 注意: 全局 batch_size = per_worker_batch × num_workers
model.fit(X, y, batch_size=32 * strategy.num_replicas_in_sync,
          epochs=2, verbose=1)
```

**步骤二：容错 Checkpoint + 自动恢复**

目标：实现带 Preemption Hook 的容错训练循环。

```python
import tensorflow as tf
import signal
import sys
import os
import tempfile

tmpdir = tempfile.mkdtemp()
ckpt_dir = os.path.join(tmpdir, "training_checkpoints")
os.makedirs(ckpt_dir, exist_ok=True)

# === 容错训练 ===
model = tf.keras.Sequential([
    tf.keras.layers.Dense(64, activation="relu", input_shape=(32,)),
    tf.keras.layers.Dense(1, activation="sigmoid"),
])
optimizer = tf.keras.optimizers.Adam(1e-3)

ckpt = tf.train.Checkpoint(model=model, optimizer=optimizer, step=tf.Variable(0))
ckpt_mgr = tf.train.CheckpointManager(ckpt, ckpt_dir, max_to_keep=3)

# 尝试从最新 checkpoint 恢复
latest = ckpt_mgr.latest_checkpoint
if latest:
    ckpt.restore(latest).expect_partial()
    print(f"✅ 从 checkpoint 恢复: {latest}")
    print(f"   恢复步数: {ckpt.step.numpy()}")
else:
    print(f"无 checkpoint 可恢复，从零开始训练")

# === Preemption Hook（捕获 SIGTERM 做紧急保存） ===
save_on_exit = True

def emergency_save(signum, frame):
    """收到 SIGTERM 时紧急保存 checkpoint"""
    global save_on_exit
    save_path = ckpt_mgr.save()
    print(f"\n⚠️  收到终止信号, 紧急保存 checkpoint: {save_path}")
    save_on_exit = False  # 防止重复保存
    sys.exit(0)

signal.signal(signal.SIGTERM, emergency_save)

# 训练循环（带定期保存）
X = np.random.randn(1000, 32).astype(np.float32)
y = np.random.randint(0, 2, 1000).astype(np.float32)
ds = tf.data.Dataset.from_tensor_slices((X, y)).batch(32)

total_steps = 30
for step, (xb, yb) in enumerate(ds.take(total_steps)):
    # 跳过已训练过的 step（幂等恢复）
    if step < ckpt.step.numpy():
        continue

    with tf.GradientTape() as tape:
        logits = model(xb, training=True)
        loss = tf.reduce_mean(
            tf.keras.losses.binary_crossentropy(yb, logits)
        )

    grads = tape.gradient(loss, model.trainable_variables)
    optimizer.apply_gradients(zip(grads, model.trainable_variables))

    ckpt.step.assign_add(1)

    if (step + 1) % 10 == 0:
        checkpoint_path = ckpt_mgr.save()
        print(f"Step {step+1}: loss={loss:.4f}, checkpoint→{checkpoint_path}")

print(f"\n训练完成，最终 step: {ckpt.step.numpy()}")

# 清理
import shutil
shutil.rmtree(tmpdir, ignore_errors=True)
```

## 4. 项目总结

### 4.1 分布式训练方案对比

| 方案 | 卡数范围 | 同步模式 | 容错 | 成本 |
|------|---------|---------|------|------|
| **MirroredStrategy** | 1-8 (单机) | 同步 | 无 | 低 |
| **MultiWorkerMirrored** | 4-64 (多机) | 同步 | Checkpoint | 中 |
| **ParameterServerStrategy** | 10-200+ (多机) | 异步 | PS 冗余 | 高 |
| **Horovod** | 4-512 | 同步 | Checkpoint + Elastic | 中 |
| **DeepSpeed/Megatron** | 8-数千 | 同步 + 模型并行 | Checkpoint + Elastic | 极高 |

### 4.2 适用场景

1. **大模型预训练（BERT/GPT）**：MultiWorkerMirroredStrategy + NCCL + 梯度压缩
2. **推荐模型大规模 Embedding**：ParameterServerStrategy + 冷热分离
3. **弹性训练（Spot 实例）**：K8s + Volcano + 自动 checkpoint 恢复
4. **成本敏感场景**：混合 Spot+On-demand + 弹性调度

**不适用场景**：
1. 模型极小（< 100MB）——分布式通信开销大于计算收益
2. GPU 资源固定且专享——容错和弹性机制的成本高于价值

### 4.3 注意事项

- **`TF_CONFIG` 的环境变量注入**：手动配置极易出错（Index 错配、地址写错），务必由 K8s Operator 自动注入
- **checkpoint 必须在共享存储上**：所有 Worker 共享 NFS/HDFS/PVC——否则某个 Worker 挂了恢复后没有同步的最新权重
- **AllReduce 通信拓扑**：NCCL 自动选择最优拓扑（Ring / Tree / Collnet），但你可以用 `NCCL_DEBUG=INFO` 查看实际选择，用 `NCCL_ALGO` 手动覆盖
- **梯度压缩（Gradient Compression）的精度损失**：Top-K sparsification 或 1-bit quantization 会引入噪声——需要 warmup + 逐步增加压缩比

### 4.4 常见踩坑经验

1. **坑**：MultiWorkerMirroredStrategy 训练 hang 住，所有 Worker 卡在 step 0。
   **根因**：NCCL 通信初始化失败——网络防火墙阻断了 Worker 之间的 12345+ 端口，或 Hostname 解析失败。
   **解决**：`nc -zv worker-1 12345` 验证端口可达；设置 `NCCL_SOCKET_IFNAME=eth0` 指定网卡；`NCCL_DEBUG=INFO` 输出 NCCL 初始化日志。

2. **坑**：Spot 实例频繁回收，从 checkpoint 恢复后 loss 突然异常高。
   **根因**：DataLoader 的 shuffle buffer 在恢复后重新初始化——模型"重读"了之前已经训练过的数据，导致 overfitting。
   **解决**：DataLoader 也记录状态（如 shard + offset），恢复时 skip 已经消费过的数据（`dataset.skip(resumed_step)`）。

3. **坑**：8 机训练速度和 2 机一样（无 scaling）。
   **根因**：模型计算量小于通信量——每 step 计算 100ms，但 AllReduce 通信需要 200ms，GPU 有一半时间在等通信。
   **解决**：增大模型（hidden dim）；或使用梯度累积——N step 才做一次 AllReduce，降低通信频次。

### 4.5 思考题

1. 你的训练任务需要在 Spot 实例上训练 7 天。假设 Spot 平均每 24 小时被回收一次，恢复时间约 5 分钟。你在 checkpoint 的频率（每 30 分钟 vs 每 2 小时）和训练总时间之间存在怎样的 tradeoff？如何确定最优的 checkpoint 频率？

2. MultiWorkerMirroredStrategy 的 AllReduce 默认是同步的——最慢的 Worker 决定了每步的训练速度（straggler problem）。你如何在训练过程中检测和缓解 straggler？列出了至少两种方法。
