# 第18章：自定义训练循环与混合训练策略

## 1. 项目背景

风控团队的风控模型（信用卡欺诈检测）在线上表现不稳定——上周准确率 96%，这周掉到 91%。算法工程师阿杰排查后发现，欺诈团伙的行为模式在不断变化：以前欺诈交易多集中在凌晨小额支付，现在改成白天大额家电消费——模型见到的"新欺诈"和训练时的分布差异越来越大。

阿杰提出做一个"在线困难样本挖掘"方案：每个 epoch 结束后，用当前模型对所有训练样本打分，找出那些"模型搞错的样本"和"模型不确定的样本"（预测概率在 0.4-0.6 之间），在下一轮训练中给这些样本更高的采样权重。同时，欺诈样本太少（约 0.5%），阿杰需要用 Focal Loss 配合动态的样本权重调整。

这个需求 `model.fit` 搞不定——原因有三：(1) 需要 epoch 间动态调整样本权重；(2) 需要自定义梯度累积（显存只够 batch_size=32，但想要 256 的等效效果）；(3) 需要对不同层设置不同的学习率（backbone 用 1e-4，分类头用 1e-3）。

**痛点放大**：`model.fit` 的便利性建立在"标准训练范式"之上——固定 batch、固定 lr、固定样本权重。但生产级模型训练经常需要"混合策略"——梯度累积、动态采样、分层学习率、对抗训练、自定义正则化——这些需求超出了 `model.fit` 的能力边界。

## 2. 项目设计

**小胖**（嚼着口香糖）：又回到手写训练循环？第 4 章不是已经写过了吗——`GradientTape` + `optimizer.apply_gradients`，这不就行了？

**大师**：第 4 章的手写训练循环只是"能跑起来"。生产级的自定义训练循环需要解决更多问题：梯度怎么累积？怎么对不同层用不同学习率？怎么在 epoch 间动态更新样本权重？怎么把自定义逻辑和 Keras 的 `Callback`/`metrics` 体系无缝对接？

**技术映射**：生产级自定义训练循环 = `train_step` 覆写 + 梯度策略 + 样本动态选择 + Callback 兼容，而非简单的 `for epoch in range(N): ...`。

**小白**（眼睛一亮）：等等，你说"覆写 `train_step`"？我只知道可以完全手写循环，覆写 `train_step` 是什么意思？

**大师**：好问题！这是两种不同的控制粒度：

**方式 A：覆写 `train_step`**（推荐）。你继承 `keras.Model`，只覆写 `train_step(data)` 方法。`model.fit()` 的其余逻辑——epoch 循环、Callback 触发、验证、日志——全部保留。

```python
class MyModel(keras.Model):
    def train_step(self, data):
        x, y = data
        with tf.GradientTape() as tape:
            y_pred = self(x, training=True)
            loss = self.compute_loss(y, y_pred)

        grads = tape.gradient(loss, self.trainable_variables)
        self.optimizer.apply_gradients(zip(grads, self.trainable_variables))

        # 更新 metrics
        for metric in self.metrics:
            metric.update_state(y, y_pred)
        return {m.name: m.result() for m in self.metrics}
```

**方式 B：完全手写循环**。你自己管理 `for epoch` / `for batch` / Callback 调用 / 验证逻辑。灵活度最大，但你会丢失 `model.fit` 的所有便利。

**大师**：方式 A 是 90% 场景的最优解——大部分时候你只需要改"一个 batch 怎么训练"，不想自己写"多个 epoch 怎么循环"。方式 B 适合 GAN、强化学习等需要交替更新多个模型的场景。

**小胖**：那梯度累积具体怎么在 `train_step` 里实现？

**大师**：梯度累积的核心原理很简单——你不是每个 mini-batch 都更新参数，而是把 N 个 mini-batch 的梯度加起来，最后统一更新一次。效果等同于 batch_size × N 的大批量训练。

```python
def train_step(self, data):
    x, y = data
    with tf.GradientTape() as tape:
        y_pred = self(x, training=True)
        loss = self.compute_loss(y, y_pred) / self.accum_steps  # 关键：除以累积步数

    grads = tape.gradient(loss, self.trainable_variables)
    # 累加梯度
    for i, g in enumerate(grads):
        self.accum_grads[i].assign_add(g)

    self.accum_count.assign_add(1)
    if self.accum_count % self.accum_steps == 0:
        # 统一更新参数
        self.optimizer.apply_gradients(zip(self.accum_grads, self.trainable_variables))
        # 清空累加器
        for g in self.accum_grads:
            g.assign(tf.zeros_like(g))
        self.accum_count.assign(0)
```

**技术映射**：梯度累积 = 多步小 batch 累加梯度后统一更新，用时间换空间。`loss / accum_steps` 确保梯度尺度与直接大 batch 一致。

**小白**：那分层学习率呢？怎么让 backbone 学得慢、head 学得快？

**大师**：TensorFlow 的 `optimizer.apply_gradients` 本身不支持分层学习率——但你可以手动缩放梯度：

```python
# backbone 变量学习率缩小 10 倍
backbone_vars = self.backbone.trainable_variables
head_vars = self.classifier_head.trainable_variables

scaled_grads = []
for var, grad in zip(self.trainable_variables, grads):
    if var in backbone_vars:
        scaled_grads.append(grad * 0.1)  # backbone lr = base_lr * 0.1
    else:
        scaled_grads.append(grad)

self.optimizer.apply_gradients(zip(scaled_grads, self.trainable_variables))
```

### 3.2 分步实现

**步骤一：梯度累积 + Focal Loss 的自定义训练模型**

目标：实现一个带梯度累积和 Focal Loss 的二进制分类模型。

```python
import tensorflow as tf
from tensorflow import keras
import numpy as np

class AccumulationModel(keras.Model):
    """支持梯度累积的二分类模型"""

    def __init__(self, accum_steps=4, focal_gamma=2.0, focal_alpha=0.25, **kwargs):
        super().__init__(**kwargs)
        self.accum_steps = accum_steps
        self.focal_gamma = focal_gamma
        self.focal_alpha = focal_alpha

        # 子层
        self.dense1 = keras.layers.Dense(64, activation="relu")
        self.bn = keras.layers.BatchNormalization()
        self.dropout = keras.layers.Dropout(0.3)
        self.dense2 = keras.layers.Dense(32, activation="relu")
        self.out_layer = keras.layers.Dense(1, activation="sigmoid")

        # 累加器（在 build 中创建）
        self.accum_grads = None
        self.accum_count = None

        # 指标
        self.loss_metric = keras.metrics.Mean(name="loss")
        self.acc_metric = keras.metrics.BinaryAccuracy(name="acc")

    def build(self, input_shape):
        super().build(input_shape)
        # 在 build 中初始化累加器（此时 trainable_variables 已确定）
        self.accum_grads = [tf.Variable(tf.zeros_like(v), trainable=False)
                           for v in self.trainable_variables]
        self.accum_count = tf.Variable(0, trainable=False, dtype=tf.int32)

    def call(self, inputs, training=None):
        x = self.dense1(inputs)
        x = self.bn(x, training=training)
        x = self.dropout(x, training=training)
        x = self.dense2(x)
        return self.out_layer(x)

    def focal_loss(self, y_true, y_pred):
        eps = 1e-7
        y_pred = tf.clip_by_value(y_pred, eps, 1 - eps)
        pt = tf.where(tf.equal(y_true, 1), y_pred, 1 - y_pred)
        focal_weight = tf.pow(1 - pt, self.focal_gamma)
        ce = -tf.math.log(pt)
        return tf.reduce_mean(self.focal_alpha * focal_weight * ce)

    def train_step(self, data):
        x, y = data

        with tf.GradientTape() as tape:
            y_pred = self(x, training=True)
            loss = self.focal_loss(y, y_pred) / self.accum_steps

        grads = tape.gradient(loss, self.trainable_variables)

        # 累加梯度
        for i, g in enumerate(grads):
            if g is not None:
                self.accum_grads[i].assign_add(g)

        self.accum_count.assign_add(1)

        # 累积满后统一更新
        if self.accum_count % self.accum_steps == 0:
            self.optimizer.apply_gradients(
                zip(self.accum_grads, self.trainable_variables))
            for g in self.accum_grads:
                g.assign(tf.zeros_like(g))

        # 更新指标（用未除 accum_steps 的 loss）
        self.loss_metric.update_state(loss * self.accum_steps)
        self.acc_metric.update_state(y, y_pred)
        return {"loss": self.loss_metric.result(), "acc": self.acc_metric.result()}

    @property
    def metrics(self):
        return [self.loss_metric, self.acc_metric]

# 测试
np.random.seed(42)
X = np.random.randn(2000, 20).astype(np.float32)
y = (np.random.rand(2000) > 0.8).astype(np.float32).reshape(-1, 1)

model = AccumulationModel(accum_steps=4)
model.compile(optimizer=keras.optimizers.Adam(1e-3))
model.fit(X, y, epochs=5, batch_size=32, validation_split=0.2, verbose=1)
```

**步骤二：分层学习率 + 动态困难样本挖掘**

目标：实现"backbone 低速学 + head 高速学 + 每个 epoch 找出困难样本加权"。

```python
class DynamicHardMiningModel(keras.Model):
    """支持分层学习率 + 动态困难样本挖掘"""

    def __init__(self, backbone_lr_scale=0.1, hard_mining_ratio=0.3, **kwargs):
        super().__init__(**kwargs)
        self.backbone_lr_scale = backbone_lr_scale
        self.hard_mining_ratio = hard_mining_ratio

        # backbone（慢学）
        self.backbone = keras.Sequential([
            keras.layers.Dense(128, activation="relu"),
            keras.layers.BatchNormalization(),
            keras.layers.Dropout(0.3),
            keras.layers.Dense(64, activation="relu"),
        ], name="backbone")

        # head（快学）
        self.head = keras.layers.Dense(1, activation="sigmoid", name="head")

        # 样本权重（动态更新）
        self.sample_weights = None

        self.loss_metric = keras.metrics.Mean(name="loss")
        self.auc_metric = keras.metrics.AUC(name="auc")

    def call(self, inputs, training=None):
        features = self.backbone(inputs, training=training)
        return self.head(features)

    def get_backbone_vars(self):
        return self.backbone.trainable_variables

    def get_head_vars(self):
        return self.head.trainable_variables

    def set_sample_weights(self, X_train, y_train):
        """用当前模型找出困难样本，赋更高权重"""
        probs = self.predict(X_train, verbose=0, batch_size=512).flatten()
        # 困难样本 = 预测概率接近 0.5 的（模型不确定）
        hardness = 1.0 - 2.0 * np.abs(probs - 0.5)  # 0.5→1.0, 0或1→0
        # 排序取 top hard_mining_ratio 的样本
        threshold = np.percentile(hardness, 100 * (1 - self.hard_mining_ratio))
        weights = np.ones_like(hardness)
        weights[hardness > threshold] = 2.0  # 困难样本权重加倍
        self.sample_weights = weights.astype(np.float32)
        print(f"动态权重更新: {int(np.sum(weights > 1))} 个困难样本权重 ×2")

    def train_step(self, data):
        x, y = data

        with tf.GradientTape() as tape:
            y_pred = self(x, training=True)
            # 加权 BCE
            bce = keras.losses.binary_crossentropy(y, y_pred)
            if self.sample_weights is not None:
                sw = tf.gather(self.sample_weights, tf.range(tf.shape(y)[0]))
                loss = tf.reduce_mean(bce * sw)
            else:
                loss = tf.reduce_mean(bce)

        grads = tape.gradient(loss, self.trainable_variables)
        all_vars = self.trainable_variables

        # 分层学习率缩放
        backbone_vars = set(self.get_backbone_vars())
        scaled_grads = []
        for var, grad in zip(all_vars, grads):
            if grad is not None:
                if var in backbone_vars:
                    scaled_grads.append(grad * self.backbone_lr_scale)
                else:
                    scaled_grads.append(grad)

        self.optimizer.apply_gradients(zip(scaled_grads, all_vars))

        self.loss_metric.update_state(loss)
        self.auc_metric.update_state(y, y_pred)
        return {"loss": self.loss_metric.result(), "auc": self.auc_metric.result()}

    @property
    def metrics(self):
        return [self.loss_metric, self.auc_metric]

# 训练（带每 epoch 动态样本挖掘）
model = DynamicHardMiningModel(backbone_lr_scale=0.1, hard_mining_ratio=0.3)
model.compile(optimizer=keras.optimizers.Adam(1e-3))

X_train, X_val = X[:1600], X[1600:]
y_train, y_val = y[:1600], y[1600:]

for epoch in range(5):
    print(f"\n=== Epoch {epoch+1} ===")
    if epoch >= 1:  # 从第 2 个 epoch 开始动态挖掘
        model.set_sample_weights(X_train, y_train)
    model.fit(X_train, y_train, batch_size=64, validation_data=(X_val, y_val),
              verbose=1, shuffle=False)  # shuffle=False 配合 sample_weights 按索引匹配
```

**步骤三：完全手写训练循环（丢失 model.fit，获得完全自由）**

目标：实现 GAN 风格的"两个模型交替训练"框架。

```python
# 模拟 GAN 风格交替训练
class Generator(keras.Model):
    def __init__(self, latent_dim=100):
        super().__init__()
        self.dense1 = keras.layers.Dense(128, activation="relu")
        self.dense2 = keras.layers.Dense(64, activation="relu")
        self.dense3 = keras.layers.Dense(20, activation="tanh")

    def call(self, z):
        return self.dense3(self.dense2(self.dense1(z)))

class Discriminator(keras.Model):
    def __init__(self):
        super().__init__()
        self.dense1 = keras.layers.Dense(64, activation="relu")
        self.dense2 = keras.layers.Dense(1, activation="sigmoid")

    def call(self, x):
        return self.dense2(self.dense1(x))

# 初始化
generator = Generator()
discriminator = Discriminator()
g_opt = keras.optimizers.Adam(1e-4)
d_opt = keras.optimizers.Adam(1e-4)
bce = keras.losses.BinaryCrossentropy()

ds = tf.data.Dataset.from_tensor_slices(X).batch(64)

for epoch in range(3):
    for step, real_data in enumerate(ds):
        batch_size = tf.shape(real_data)[0]

        # === 训练判别器 ===
        z = tf.random.normal([batch_size, 100])
        fake_data = generator(z, training=True)

        with tf.GradientTape() as d_tape:
            real_pred = discriminator(real_data, training=True)
            fake_pred = discriminator(fake_data, training=True)
            d_loss = bce(tf.ones_like(real_pred), real_pred) + \
                     bce(tf.zeros_like(fake_pred), fake_pred)

        d_grads = d_tape.gradient(d_loss, discriminator.trainable_variables)
        d_opt.apply_gradients(zip(d_grads, discriminator.trainable_variables))

        # === 训练生成器（交替） ===
        z = tf.random.normal([batch_size, 100])
        with tf.GradientTape() as g_tape:
            fake_data = generator(z, training=True)
            fake_pred = discriminator(fake_data, training=True)
            g_loss = bce(tf.ones_like(fake_pred), fake_pred)  # 欺骗判别器

        g_grads = g_tape.gradient(g_loss, generator.trainable_variables)
        g_opt.apply_gradients(zip(g_grads, generator.trainable_variables))

    print(f"Epoch {epoch+1}: d_loss={d_loss:.4f}, g_loss={g_loss:.4f}")
```

### 3.3 训练策略选择决策树

```
你的训练需求：
├─ 标准训练（固定 batch/lr/权重）
│  └─→ model.fit()                      ← 90% 场景
├─ 需要改 batch 级别的行为（梯度累积/自定义 loss/分层 lr）
│  └─→ 覆写 train_step + model.fit()    ← 推荐
├─ 需要改 epoch 级别的行为（动态采样/动态权重）
│  └─→ 覆写 train_step + 外层 for epoch  ← 中级复杂度
└─ 需要交替更新多个模型（GAN/RL/Meta-Learning）
   └─→ 完全手写循环                       ← 终极自由
```

## 4. 项目总结

### 4.1 训练模式对比

| 方面 | `model.fit()` | 覆写 `train_step` | 完全手写循环 |
|------|--------------|-------------------|-------------|
| 代码量 | 3 行 | 30-50 行 | 100+ 行 |
| Callback 兼容 | 原生支持 | 原生支持 | 需手动调用 |
| 分布式训练 | 自动 | 自动（需确保 train_step 兼容） | 需手动处理 |
| 梯度累积 | ❌ | ✅ | ✅ |
| 分层学习率 | ❌ | ✅ | ✅ |
| 动态样本权重 | ⚠️ 有限 | ✅ | ✅ |
| 多模型交替训练 | ❌ | ❌ | ✅ |
| 调试难度 | 低 | 中 | 高 |
| 出错风险 | 低 | 中 | 高 |

### 4.2 适用场景

1. **梯度累积** → GPU 显存不足但需要大 batch_size 时
2. **分层学习率** → 迁移学习（backbone 慢、head 快）、多任务权重调节
3. **动态样本挖掘** → 不均衡分类、在线学习、主动学习
4. **Focal Loss/自定义 loss** → 第 7 章已涉及，此处可与梯度策略组合
5. **GAN/强化学习** → 需要交替更新多个网络参数

**不适用场景**：
1. 标准 CNN 图像分类——`model.fit()` 三条线解决，无需手写循环
2. 团队代码审查能力不足——手写训练循环的 bug 可能比模型本身的 bug 更难排查

### 4.3 注意事项

- **梯度累积的 loss 缩放**：`loss = loss / accum_steps` 是必须的，否则梯度尺度会是单步的 accum_steps 倍
- **`train_step` 中的 `self.compiled_metrics`**：如果覆写 `train_step`，需要手动调用 `self.compiled_loss` 和 `self.compiled_metrics.update_state()`，否则 Callback 看到的 val 指标可能不准确
- **`@tf.function` 装饰 `train_step`**：Keras 默认用 `@tf.function` 包装 `train_step`，如果你覆写了它，内部不能有不可图化的操作（如 `print`、`.numpy()`），用 `tf.print` 替代
- **动态 sample_weight 的 index 对齐**：如果用了 `dataset.shuffle()`，样本顺序会打乱，sample_weight 必须跟着 shuffle——建议把 sample_weight 作为 dataset 的第三个元素传入

### 4.4 常见踩坑经验

1. **坑**：梯度累积训练几个 step 后 loss 突然 NaN。
   **根因**：`accum_steps` 内的 loss 累加导致反向传播的中间结果不断叠积，结合 BatchNorm 的 batch 统计量在小 batch 下极度不稳定。
   **解决**：将 BN 的 `momentum` 调大到 0.99（使统计量更平滑），或在梯度累积时冻结 BN 的统计量更新。

2. **坑**：分层学习率的 backbone 参数完全不更新。
   **根因**：`var in backbone_vars` 的比较是基于 Python 对象的 identity，而非名称。如果 backbone 是 `keras.Sequential` 的子模型，`trainable_variables` 返回的是内部变量的引用，可能不在 `backbone_vars` 集合中。
   **解决**：用 `var.name` 字符串匹配，或使用 `var in backbone.trainable_variables` 做集合查找。

3. **坑**：动态困难样本挖掘后，model.fit 的 `validation_data` 的 loss 持续上升。
   **根因**：困难样本挖掘只改了训练集权重，但如果验证集本身也高度不均衡（比如大部分是易分类样本），模型过分关注困难样本会导致对简单样本的精度下降。
   **解决**：hard_mining_ratio 不要设置过高（建议 0.1-0.3），同时监控验证集上"简单样本"和"困难样本"各自的指标。

### 4.5 思考题

1. 你需要在 8GB 显存的 GPU 上训练一个需要 batch_size=512 的模型，但显存只够 bs=32。设计一套包含梯度累积 + 混合精度训练的方案，并估算有效 batch_size、训练速度变化和可能的精度影响。

2. 本章的动态困难样本挖掘使用了固定阈值（hard_mining_ratio=0.3）。请设计一个"自适应"策略——根据模型当前的验证集表现自动调整 hard_mining_ratio，避免在过拟合阶段还继续挖掘困难样本。
