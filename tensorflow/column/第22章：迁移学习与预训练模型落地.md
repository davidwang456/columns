# 第22章：迁移学习与预训练模型落地

## 1. 项目背景

一家工业制造企业需要上线一个"PCB 板缺陷检测"系统——流水线上安装摄像头，自动检测电路板是否有焊点缺陷（虚焊、短路、漏焊等 6 类缺陷）。企业只有 1500 张标注图片（每类约 250 张），远不够训练一个从零开始的深度 CNN 模型。

算法工程师小林尝试从零训练一个 ResNet-50，结果训练集准确率 95%，验证集只有 61%——典型的小样本过拟合。数据增强、Dropout、Label Smoothing（第 21 章的调优手段）全上了一遍，验证准确率也只提升到 68%。

业务方的总监很着急："我们工厂等不及了，能不能用什么现成的模型？就用 ImageNet 上训练好的那个！"小林说："ImageNet 上训练的是猫、狗、车，根本没见过电路板焊点啊。"总监："那让它见一见不行吗？"

**痛点放大**：工业场景的数据标注成本极高（一张 PCB 缺陷图片需要专业工程师花 5-10 分钟标注），而"从零训练"对数据量的需求是万级甚至十万级。迁移学习（Transfer Learning）的核心价值在于——利用在 ImageNet（1400 万张图片、1000 类）上训练好的模型作为起点，让它把"看到边缘、纹理、形状"的底层视觉能力迁移到你的任务上，只需要少量标注数据微调即可。

## 2. 项目设计

**小胖**（困惑地看着两个模型架构）：等等，你说 ImageNet 上训练的模型根本没见过电路板，那它怎么帮我们检测缺陷？这不就像让一个只会开小轿车的人去开卡车——虽然都是车，但完全不一样啊？

**大师**：你这个比喻其实很到位——只不过迁移学习比你想象的更厉害。一个会开小轿车的人去开卡车，他至少已经掌握了"方向盘怎么转""油门和刹车在哪""怎么看后视镜"这些基本技能。他不需要从零学"交通规则是什么"，只需要适应"车身更宽""刹车距离更长""转弯半径更大"这些差异。

ImageNet 上训练的 CNN 也是一样——前几层学到的"边缘检测""纹理识别""形状判断"这些底层视觉能力是通用的——无论你检测的是猫还是电路板焊点，"边缘就是边缘""纹理就是纹理"。迁移学习就是把前几层"冻结"（保留通用视觉能力），只让后面几层去学习"什么是虚焊""什么是正常焊点"。

**技术映射**：迁移学习 = 预训练模型的底层特征 + 任务特定的顶层。底层卷积层提取通用的纹理/边缘/形状特征，这些特征可以跨领域复用。顶层全连接层学习任务特定的分类决策。

**小白**（在白板上画了两个阶段）：那我怎么知道哪些层该冻结，哪些该训练？

**大师**：有三级策略，从保守到激进：

**Level 1: Feature Extractor（特征提取器）**
```python
base_model = EfficientNetB0(weights="imagenet", include_top=False)
base_model.trainable = False  # 冻结所有层
# 只在顶层加你自己的分类头
```
→ 适合：数据量 < 1000 张或与 ImageNet 差异大（如医学影像、工业检测）

**Level 2: Partial Fine-tuning（部分微调）**
```python
base_model = EfficientNetB0(weights="imagenet", include_top=False)
# 只解冻最后 30% 的层
for layer in base_model.layers[:-int(len(base_model.layers)*0.3)]:
    layer.trainable = False
```
→ 适合：数据量 1000-5000 张，任务与 ImageNet 有中等差异

**Level 3: Full Fine-tuning（全模型微调）**
```python
base_model.trainable = True  # 所有层参与训练
```
→ 适合：数据量 > 5000 张，任务与 ImageNet 比较接近（如猫狗分类就是 ImageNet 的子集）

**小胖**：那 Keras Applications 里那么多模型——ResNet50、EfficientNet、MobileNet——选哪个？

**大师**：按需求选：
- **追求最高精度** → EfficientNetB3/B4（神经架构搜索出的最优模型）
- **资源受限（移动端/边缘设备）** → MobileNetV2/V3（轻量级，速度快）
- **经典可靠、论文引用多** → ResNet50（学术界和工业界广泛验证）
- **兼顾速度和精度** → EfficientNetB0（性价比之王）

```python
# Keras Applications 主要模型速查
from tensorflow.keras.applications import (
    ResNet50, ResNet101,           # ResNet 系列
    EfficientNetB0, EfficientNetB3,  # EfficientNet 系列
    MobileNetV2, MobileNetV3Small,   # 移动端
    DenseNet121,                   # 密集连接
)
```

**小白**：预训练模型的输入预处理（preprocessing）很坑——每个模型的归一化方式不一样，搞错了效果差很多。

**大师**：这个坑非常常见！Keras Applications 每个模型都有对应的 `preprocess_input` 函数：

```python
# ImageNet 的预处理通常是：
# - 从 [0,255] 缩放到 [0,1] 或 [-1,1]
# - 减去 ImageNet 数据集的 mean（RGB 三个通道不同）
from tensorflow.keras.applications.efficientnet import preprocess_input

# 错误做法：自己手动除以 255
img = img / 255.0  # ❌ EfficientNet 期望的是另一种 scale

# 正确做法：用模型自带的 preprocess_input
img = preprocess_input(img)  # ✅
```

**技术映射**：每个预训练模型有自己特定的 `preprocess_input`，必须使用对应模型的处理函数，它保证了输入数据的统计分布与训练时的 ImageNet 一致。

### 3.2 分步实现

**步骤一：Feature Extractor — 冻结全 backbone**

目标：用 EfficientNetB0 做特征提取，只训练顶部分类头。

```python
import tensorflow as tf
from tensorflow import keras
import numpy as np

# 模拟 1500 张 PCB 缺陷图
np.random.seed(42)
n_train, n_val = 1200, 300
img_size = (224, 224)
n_classes = 6

# 生成模拟图片数据
X_train = np.random.rand(n_train, *img_size, 3).astype(np.float32)
X_val = np.random.rand(n_val, *img_size, 3).astype(np.float32)
y_train = np.random.randint(0, n_classes, n_train)
y_val = np.random.randint(0, n_classes, n_val)

print(f"训练集: {X_train.shape}, 验证集: {X_val.shape}")

# === Level 1: Feature Extractor ===
base_model = keras.applications.EfficientNetB0(
    input_shape=(*img_size, 3),
    include_top=False,        # 去掉原始的分类头
    weights="imagenet",       # 加载 ImageNet 预训练权重
    pooling="avg",            # 全局平均池化 → 固定维度
)
base_model.trainable = False  # 冻结所有层

# 分类头
inputs = keras.Input(shape=(*img_size, 3))

# 注意：必须用 EfficientNet 自己的预处理！
x = keras.applications.efficientnet.preprocess_input(inputs)
x = base_model(x, training=False)
x = keras.layers.Dropout(0.5)(x)
outputs = keras.layers.Dense(n_classes, activation="softmax")(x)

model_1 = keras.Model(inputs, outputs, name="feature_extractor")
model_1.compile(
    optimizer=keras.optimizers.Adam(1e-3),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

print(f"可训练参数: {model_1.count_params():,}")
print(f"其中 backbone 参数: {base_model.count_params():,} (全部冻结)")

history_1 = model_1.fit(
    X_train, y_train, batch_size=32, epochs=10,
    validation_data=(X_val, y_val), verbose=1,
    callbacks=[keras.callbacks.EarlyStopping(patience=3, restore_best_weights=True)],
)

val_acc_l1 = max(history_1.history["val_accuracy"])
print(f"\nLevel 1 (Feature Extractor) 最佳验证准确率: {val_acc_l1:.4f}")
```

**步骤二：部分 Fine-tuning — 解冻底层**

目标：在 Feature Extractor 基础上，解冻最后 20% 层进行微调。

```python
# 从 Level 1 继续 —— 解冻部分层
base_model.trainable = True  # 先全部解冻

# 冻结前 80% 的层
freeze_until = int(len(base_model.layers) * 0.8)
for layer in base_model.layers[:freeze_until]:
    layer.trainable = False

print(f"\n总层数: {len(base_model.layers)}, 冻结前 {freeze_until} 层")
print(f"可训练层数: {sum(l.trainable for l in base_model.layers)}")

# Fine-tuning 必须用更小的学习率（否则会破坏预训练权重）
model_2 = keras.Model(inputs, outputs, name="partial_finetune")
model_2.compile(
    optimizer=keras.optimizers.Adam(1e-5),  # lr 降低 100 倍
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

history_2 = model_2.fit(
    X_train, y_train, batch_size=32, epochs=15,
    validation_data=(X_val, y_val), verbose=1,
    callbacks=[keras.callbacks.EarlyStopping(patience=5, restore_best_weights=True)],
)

val_acc_l2 = max(history_2.history["val_accuracy"])
print(f"\nLevel 2 (Partial Fine-tune) 最佳验证准确率: {val_acc_l2:.4f}")
```

**步骤三：多模型对比与模型选型**

目标：对比 ResNet50、EfficientNetB0、MobileNetV2 在相同条件下的表现。

```python
def experiment_model(app_module, model_name, img_size=(224, 224)):
    base = app_module(
        input_shape=(*img_size, 3),
        include_top=False,
        weights="imagenet",
        pooling="avg",
    )
    base.trainable = False

    inp = keras.Input(shape=(*img_size, 3))
    x = app_module.preprocess_input(inp) if hasattr(app_module, 'preprocess_input') else inp
    x = base(x, training=False)
    x = keras.layers.Dropout(0.5)(x)
    out = keras.layers.Dense(n_classes, activation="softmax")(x)

    model = keras.Model(inp, out)
    model.compile(optimizer=keras.optimizers.Adam(1e-3),
                  loss="sparse_categorical_crossentropy", metrics=["accuracy"])

    history = model.fit(X_train, y_train, batch_size=32, epochs=10,
                        validation_data=(X_val, y_val), verbose=0,
                        callbacks=[keras.callbacks.EarlyStopping(patience=3, restore_best_weights=True)])
    return {
        "params": model.count_params(),
        "val_acc": max(history.history["val_accuracy"]),
        "epochs": len(history.history["val_accuracy"]),
    }

# 为公平比较，ResNet50 的输入尺寸固定为 224
results = {}
for app_fn, name in [
    (keras.applications.EfficientNetB0, "EfficientNetB0"),
    (keras.applications.ResNet50, "ResNet50"),
    (keras.applications.MobileNetV2, "MobileNetV2"),
]:
    results[name] = experiment_model(app_fn, name)
    print(f"{name:<16}: val_acc={results[name]['val_acc']:.4f}, "
          f"params={results[name]['params']:,}, epochs={results[name]['epochs']}")
```

**步骤四：小样本训练技巧——渐进式解冻**

目标：避免一次性解冻全部层导致预训练知识被"冲刷"。

```python
def progressive_unfreeze(base_model, X_train, y_train, X_val, y_val, epochs_per_stage=5):
    """渐进式解冻：每次解冻最后 25% 的层，逐步 fine-tune"""
    base_model.trainable = True
    total_layers = len(base_model.layers)
    stages = 4  # 分 4 步解冻

    inp = keras.Input(shape=(*img_size, 3))
    x = keras.applications.efficientnet.preprocess_input(inp)
    x = base_model(x)
    out = keras.layers.Dense(n_classes, activation="softmax")(x)
    model = keras.Model(inp, out)

    for stage in range(1, stages + 1):
        # 解冻最后 (stage * 25)% 的层
        freeze_before = int(total_layers * (1 - stage * 0.25))
        for layer in base_model.layers[:freeze_before]:
            layer.trainable = False
        for layer in base_model.layers[freeze_before:]:
            layer.trainable = True

        trainable = sum(l.trainable for l in base_model.layers)
        print(f"  Stage {stage}: {trainable}/{total_layers} 层可训练")

        lr = 1e-4 / (stage)  # 越往后学习率越小（保护早期层的预训练知识）
        model.compile(optimizer=keras.optimizers.Adam(lr),
                      loss="sparse_categorical_crossentropy", metrics=["accuracy"])

        history = model.fit(X_train, y_train, batch_size=32, epochs=epochs_per_stage,
                            validation_data=(X_val, y_val), verbose=0)

    return max(history.history["val_accuracy"])

_ = progressive_unfreeze(base_model, X_train, y_train, X_val, y_val)
```

### 3.3 迁移学习决策树

```
你的数据量和任务:
├─ 数据 < 500 张
│  └─→ Feature Extractor（冻结全部，只训分类头）
│       + 强数据增强 + 高 Dropout
├─ 数据 500-5000 张
│  └─→ Feature Extractor → 验证 → Partial Fine-tune（解冻最后 20-30%）
├─ 数据 5000-50000 张
│  └─→ Partial → Full Fine-tune（从 80% 冻结逐步到全解冻）
└─ 数据 > 50000 张
   └─→ 考虑从零训练 或 Full Fine-tune with warmup
```

## 4. 项目总结

### 4.1 Keras Applications 模型对比

| 模型 | Top-1 Acc | 参数量 | 推理速度 | 适用场景 |
|------|-----------|--------|---------|----------|
| **MobileNetV2** | 71.3% | 3.5M | 极快 | 移动端、边缘设备 |
| **MobileNetV3** | 75.2% | 5.4M | 极快 | 移动端升级版 |
| **ResNet50** | 74.9% | 25.6M | 快 | 经典基线，论文引用多 |
| **EfficientNetB0** | 77.1% | 5.3M | 快 | 性价比王（精度/参数双优） |
| **EfficientNetB3** | 81.4% | 12.2M | 中 | 服务器端高精度 |
| **DenseNet121** | 75.0% | 8.0M | 中 | 特征复用，小样本友好 |

### 4.2 适用场景

1. **工业缺陷检测**：50-500 张标注 → EfficientNetB0 + Feature Extractor
2. **医学影像诊断**：100-1000 张 → EfficientNetB3 + Partial Fine-tune
3. **移动端分类**：MobileNetV3 + 量化（第 26 章）
4. **多类目商品分类**：EfficientNetB0 → Full Fine-tune（数据量通常足够）
5. **相似域迁移**：猫狗 → 野生动物 → full fine-tune；自然图像 → 医学影像 → feature extractor

**不适用场景**：
1. 目标域与 ImageNet 差异极大（如 CT 切片是单通道灰度图，ImageNet 是 RGB）——预训练权重帮助有限，但仍有加速收敛的作用
2. 完全不同的模态（如音频频谱图）——虽可尝试，但 ImageNet 预训练带来改善的可能性较低

### 4.3 注意事项

- **`preprocess_input` 不可省略或混用**：ResNet 用 `tf.keras.applications.resnet.preprocess_input`（caffe 模式），EfficientNet 用 `efficientnet.preprocess_input`（[0,1] 模式），混用导致输入分布完全错误
- **Fine-tuning 时需注意 BatchNorm 的行为**：如果冻结了 BN 层，推理时它会用预训练的移动统计量而非当前数据——可能不匹配。建议在 fine-tune 时设 `base_model.trainable=True` 让 BN 的统计量也更新
- **解冻后学习率需大幅降低**：一般从 1e-3 降到 1e-5 或 1e-6，否则预训练权重在几步梯度更新后就被"冲刷"了
- **模型许可（License）风险**：Keras Applications 的预训练权重来自开源社区训练，商业使用前确认许可证。EfficientNet 的权重是 Apache 2.0 许可，可以商用

### 4.4 常见踩坑经验

1. **坑**：Feature Extractor 训练几轮后验证准确率横向不动（65% 左右）。改大分类头→没用，改大 lr→过拟合。
   **根因**：数据与 ImageNet 差异太大（如工业 CT 图 vs. 自然场景），预训练的底层特征对任务帮助有限，需要更多可训练层。
   **解决**：从 Feature Extractor 改为 Partial Fine-tune，解冻最后 30-50% 层。

2. **坑**：Fine-tuning 后验证准确率反而比 Feature Extractor 更低。
   **根因**：学习率太大（用 1e-3 直接 fine-tune），预训练权重被破坏；或数据量太少 + 解冻太多 → 严重过拟合。
   **解决**：lr 至少降到 1e-5；数据 < 500 张时不要解冻超过 20% 的层。

3. **坑**：`model.save()` 保存模型后重新加载，推理结果与保存前不一致。
   **根因**：包含了 `preprocess_input` 的函数式模型在 TF 版本升级后，预处理函数的序列化可能失效（因为它内部不是纯 Keras Layer）。
   **解决**：把预处理放在模型外部（在 tf.data pipeline 中完成），模型只包含 backbone + 分类头。

### 4.5 思考题

1. 你有一个在 ImageNet 上预训练的 ResNet50，需要迁移到"手写汉字识别"（3755 类，每类 50 张图）。请设计完整的迁移学习方案，包括：输入尺寸选择、预处理方式、冻结策略、学习率设置、数据增强策略。为什么这样设计？

2. 如果你的任务数据是"灰度医学图像"（单通道），而预训练模型期望 RGB（三通道），你会怎么处理？对比"灰度图复制三通道"和"修改第一层卷积接受单通道输入并随机初始化"两种方案。

### 4.6 推广计划提示

- **新人开发**：先在 Feature Extractor 模式下跑通（最安全），验证 pipeline 无误后尝试 Partial Fine-tune
- **算法工程师**：团队的 `model_zoo.py` 应维护常用预训练模型的加载函数（含正确的 preprocess_input），避免每次手写
- **测试工程师**：迁移学习场景的测试重点是"源域 vs 目标域"的差异——增加目标域特有的对抗样本（如 PCB 板的不同光照/角度），确保模型未退化
