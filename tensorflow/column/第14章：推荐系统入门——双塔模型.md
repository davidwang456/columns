# 第14章：推荐系统入门——双塔模型

## 1. 项目背景

一家在线影视平台的推荐团队面临一个挑战——平台有 5 万部电影和 200 万活跃用户，首页需要为每个用户展示 20 部"你可能喜欢"的电影。如果每次推荐都从 5 万部电影里用复杂模型打分，200 万用户 × 5 万电影 = 1000 亿次计算——实时算完全不可能。

推荐架构师老吴把推荐拆成了两个阶段：**召回（Recall）** 和 **排序（Ranking）**。召回阶段从 5 万部电影中快速选出 500 部候选，排序阶段用更精细的模型对这 500 部打分，选 Top 20 展示。

老吴交给算法新人小孙的任务是：做一个召回模型——输入用户特征（年龄、性别、历史观影记录、地域等）和电影特征（类型、导演、年份、平均评分等），输出一个分数表示用户对这个电影的兴趣程度。

小孙一开始用第 5 章学到的 MLP——把用户特征和电影特征拼接后塞给 Dense 层：

```python
merged = concatenate([user_vec, item_vec]) → Dense(128) → Dense(1)
```

模型离线 AUC 有 0.78，看起来不错。但上线时发现：每次有用户请求，需要对 5 万部电影逐部打分，每部都跑一遍 Dense 层——延迟远超 100ms 的 SLA。

老吴指出：你需要**双塔模型（Two-Tower）**——用户塔产出用户向量，物品塔产出物品向量，5 万部电影的向量可以提前算好存进向量数据库（如 Faiss/Milvus）。在线请求时，只需算一次用户向量，然后在向量库里做 ANN（近似最近邻检索），找 Top 500——整个过程不到 10ms。

**痛点放大**：推荐系统中"特征拼接+MLP"的交互式模型虽然精度高，但推理时需要对所有候选物品逐一计算，不适合大规模召回。双塔模型将用户和物品独立编码为向量，物品向量可离线预计算，在线只需一次用户编码 + 向量检索，实现"10ms 内从百万级物品中召回"。

## 2. 项目设计

**小胖**（翻着电影列表）：推荐还不简单？用户喜欢喜剧就给他推喜剧，喜欢动作片就推动作片。标签匹配就完了，搞什么双塔模型？

**大师**：那你给一个"喜欢《肖申克的救赎》和《当幸福来敲门》、但不喜欢《变形金刚》"的用户推什么？标签匹配会说他喜欢"剧情片"——然后把所有剧情片都推给他，包括《教父》（不错）和《小时代》（灾难）。这里的关键是——"用户喜欢什么"和"电影是什么风格"不是几个标签能描述的，而是一个高维的语义空间。

**小胖**：高维语义空间？这又是什么高端词汇……

**大师**：这么说吧——如果把每部电影比作一个点，你希望"风格相似的电影在空间里距离很近"。比如《肖申克的救赎》和《绿里奇迹》同属"监狱背景+人性救赎"，它们在空间中的坐标应该很近。同时，每个用户也在同一个空间中——如果用户 A 常看"人性救赎类"，他的坐标会靠近这个电影簇。

**技术映射**：双塔模型通过两个独立的神经网络（用户塔和物品塔），将用户和物品映射到同一向量空间。在这个空间中，向量距离（余弦相似度/点积）代表"用户对这个物品的兴趣程度"。

**小白**（在纸上画了两个塔）：那两个塔具体长什么样？用户塔和物品塔是同一个网络吗？

**大师**：通常不是同一个，因为它们处理的特征不同：
- **用户塔**：输入用户特征（年龄、性别、历史观看的电影 ID 序列、地域等），输出一个 128 维向量。用户特征里有序列（看过哪些电影），所以塔里可能有 Embedding + Average Pooling。
- **物品塔**：输入电影特征（类型、导演、年份、简介文本等），输出同样 128 维的向量。电影特征是静态的（类型、标签等），塔通常是 Embedding + MLP。

关键点——两个塔输出的向量必须在同一个向量空间中，维度相同。这样用户向量和电影向量才能做相似度计算。

**技术映射**：双塔模型约束用户塔和物品塔输出同维度向量（如都为 128 维），相似度用余弦相似度或点积计算。Loss 函数通常用 sampled softmax 或 triplet loss——正样本（用户点击过的电影）的相似度逼近 1，负样本逼近 0 或 -1。

**小白**：那负采样是怎么回事？我听说推荐系统里负样本的选择很讲究。

**大师**：对，这是一个大坑。直观地想——用户点击过的电影是"正样本"，没点过的都是"负样本"。但你有 5 万部电影，用户只看过 50 部，剩下的 49950 部你难道全当负样本？不行——计算量太大而且包含很多"假负样本"（用户其实会喜欢但没机会看到）。

所以需要**负采样**——从 5 万部中采一小批（比如 100 部）作为负样本。采样策略很关键：
- **随机负采样**：最简单的，从所有电影中随机采。缺点——大部分随机采的电影用户本来就不会喜欢，对模型学习"区分真正相关的和无关的"帮助有限。
- **流行度负采样**：按电影热度加权采样——热门电影作为负样本的机会更高。这样模型学到的是"区分用户真实兴趣和大众热门"。
- **困难负样本挖掘（Hard Negative Mining）**：用当前模型对全量电影打分，取"模型觉得不错但用户实际没点过"的那批作为负样本——这些才是最有信息量的。

**技术映射**：负采样是双塔模型训练的核心瓶颈。随机采 → 学习效率低；流行度采 → 推荐多样性可能受损；hard negative mining → 训练效果最好但计算成本高，一般每 N 个 epoch 做一次。

**小胖**：那怎么评估召回模型的好坏？AUC 靠谱吗？

**大师**：AUC 有参考价值但不够。召回阶段的关键指标是**召回率@K**——"在 Top K 个召回结果中，有多少是用户真正喜欢的？"具体操作是：从全量电影中随机采 1000 部（含 1 部用户真看过的），看模型能不能把这部真看过的排在 Top K。如果能——命中率（Hit Rate）高。

常用评估指标：

| 指标 | 含义 | 计算方式 |
|------|------|----------|
| Hit Rate@K | Top K 中是否有正样本 | `hit = 1 if any(pos in topK) else 0` |
| Recall@K | 正样本中被召回的占比 | `hit_pos / total_pos` |
| MRR | 第一个正样本的排名的倒数均值 | `mean(1 / rank_of_first_pos)` |
| NDCG@K | 考虑排名的排序质量 | 折损累积增益（高位命中更重要） |

### 3.2 分步实现

**步骤一：双塔模型构建**

目标：用 Functional API 构建用户塔 + 物品塔 + 点积相似度。

```python
import tensorflow as tf
from tensorflow import keras
import numpy as np

# --- 参数设定 ---
embed_dim = 32       # 用户/物品向量维度
user_feat_dim = 10   # 用户特征维度
item_feat_dim = 8    # 物品特征维度

# === 用户塔 ===
user_input = keras.Input(shape=(user_feat_dim,), name="user_features", dtype=tf.float32)
u = keras.layers.Dense(128, activation="relu", name="user_dense1")(user_input)
u = keras.layers.BatchNormalization(name="user_bn")(u)
u = keras.layers.Dropout(0.3, name="user_dropout")(u)
u = keras.layers.Dense(64, activation="relu", name="user_dense2")(u)
u = keras.layers.Dense(embed_dim, name="user_embedding")(u)
user_output = keras.layers.Lambda(
    lambda x: tf.math.l2_normalize(x, axis=1),  # L2 归一化
    name="user_l2_norm"
)(u)

# === 物品塔 ===
item_input = keras.Input(shape=(item_feat_dim,), name="item_features", dtype=tf.float32)
i = keras.layers.Dense(128, activation="relu", name="item_dense1")(item_input)
i = keras.layers.BatchNormalization(name="item_bn")(i)
i = keras.layers.Dropout(0.3, name="item_dropout")(i)
i = keras.layers.Dense(64, activation="relu", name="item_dense2")(i)
i = keras.layers.Dense(embed_dim, name="item_embedding")(i)
item_output = keras.layers.Lambda(
    lambda x: tf.math.l2_normalize(x, axis=1),
    name="item_l2_norm"
)(i)

# === 相似度计算 ===
# 点积 → sigmoid → 0~1 之间的匹配分
dot_product = keras.layers.Dot(axes=-1, normalize=False)([user_output, item_output])
score = keras.layers.Dense(1, activation="sigmoid", name="match_score")(
    tf.expand_dims(dot_product, -1)
)

# === 组装模型 ===
two_tower = keras.Model(
    inputs=[user_input, item_input],
    outputs=score,
    name="two_tower_model"
)

two_tower.compile(
    optimizer=keras.optimizers.Adam(learning_rate=1e-3),
    loss="binary_crossentropy",
    metrics=["accuracy", keras.metrics.AUC(name="auc")],
)

two_tower.summary()
```

**步骤二：模拟训练数据 + 负采样**

目标：构造用户-电影交互数据，实施负采样训练。

```python
np.random.seed(42)
n_users = 5000
n_items = 500
n_interactions = 30000  # 正样本数

# 生成用户特征矩阵
user_features = np.random.randn(n_users, user_feat_dim).astype(np.float32)

# 生成物品特征矩阵
item_features = np.random.randn(n_items, item_feat_dim).astype(np.float32)
# 给不同品类加偏置（模拟不同风格）
for i in range(n_items):
    if i < n_items // 3:       # 动作片
        item_features[i, 0] += 2.0
    elif i < 2 * n_items // 3: # 喜剧片
        item_features[i, 1] += 2.0
    else:                      # 纪录片
        item_features[i, 2] += 2.0

# 生成正样本交互：用户特征和物品特征相似的更容易产生交互
pos_users = np.random.randint(0, n_users, n_interactions)
pos_items = np.random.randint(0, n_items, n_interactions)

# 每个正样本配 4 个负样本（随机采样）
neg_ratio = 4
all_user_ids = []
all_item_ids = []
all_labels   = []

for idx in range(n_interactions):
    uid = pos_users[idx]
    iid = pos_items[idx]
    # 正样本
    all_user_ids.append(uid)
    all_item_ids.append(iid)
    all_labels.append(1.0)
    # 负样本
    for _ in range(neg_ratio):
        neg_iid = np.random.randint(0, n_items)
        while neg_iid == iid:  # 避免负样本和正样本相同
            neg_iid = np.random.randint(0, n_items)
        all_user_ids.append(uid)
        all_item_ids.append(neg_iid)
        all_labels.append(0.0)

# shuffle
idx = np.random.permutation(len(all_labels))
train_users = user_features[np.array(all_user_ids)[idx]]
train_items = item_features[np.array(all_item_ids)[idx]]
train_labels = np.array(all_labels)[idx].astype(np.float32)

print(f"训练样本总数: {len(train_labels)} (正样本: {train_labels.sum():.0f})")
print(f"train_users shape: {train_users.shape}")
print(f"train_items shape: {train_items.shape}")

# 训练
history = two_tower.fit(
    [train_users, train_items], train_labels,
    batch_size=512, epochs=5,
    validation_split=0.1,
    verbose=1,
)
```

**步骤三：离线评估——Recall@K 与 Hit Rate**

目标：实现标准的召回评估流程，计算 TopK 命中率。

```python
# === 提取用户塔和物品塔（分离两个子模型） ===
user_encoder = keras.Model(
    inputs=two_tower.get_layer("user_features").input,
    outputs=two_tower.get_layer("user_l2_norm").output,
    name="user_encoder"
)
item_encoder = keras.Model(
    inputs=two_tower.get_layer("item_features").input,
    outputs=two_tower.get_layer("item_l2_norm").output,
    name="item_encoder"
)

# 预计算所有物品向量
all_item_vectors = item_encoder.predict(item_features, verbose=0)  # (n_items, embed_dim)

# === 召回评估 ===
def evaluate_recall(user_encoder, item_vectors, user_features, user_item_pairs,
                    k_values=[10, 20, 50, 100], pool_size=500):
    """
    对每个 (user, true_item) 对：
    1. 随机采样 pool_size 个物品（含 true_item）
    2. 计算用户向量与这些物品向量的相似度
    3. 排序，看 true_item 在 TopK 中的命中情况
    """
    rng = np.random.default_rng(42)
    hits = {k: 0 for k in k_values}
    total = len(user_item_pairs)

    for uid, true_iid in user_item_pairs:
        # 随机采样候选池
        candidates = rng.choice(len(item_vectors), pool_size - 1, replace=False)
        candidates = np.append(candidates, true_iid)  # 确保正样本在池中
        rng.shuffle(candidates)

        # 用户向量
        user_vec = user_encoder.predict(user_features[uid:uid+1], verbose=0)  # (1, embed_dim)

        # 候选物品向量
        candidate_vecs = item_vectors[candidates]  # (pool_size, embed_dim)

        # 余弦相似度
        similarities = np.dot(user_vec, candidate_vecs.T).flatten()  # (pool_size,)

        # 排序（降序）
        ranked = candidates[np.argsort(similarities)[::-1]]

        # 检查 true_iid 的排名
        for k in k_values:
            if true_iid in ranked[:k]:
                hits[k] += 1

    print(f"\n=== 召回评估结果 ({total} 个测试样本, pool_size={pool_size}) ===")
    print(f"{'K':>6} {'Hit Rate':>10} {'Recall@K':>10}")
    print("-" * 28)
    for k in k_values:
        hit_rate = hits[k] / total
        print(f"{k:>6} {hit_rate:>10.4f} {hit_rate:>10.4f}")

# 构造测试对（选 200 对正样本）
test_pairs = list(zip(pos_users[:200], pos_items[:200]))
evaluate_recall(user_encoder, all_item_vectors, user_features, test_pairs)
```

运行输出（示例）：
```
=== 召回评估结果 (200 个测试样本, pool_size=500) ===
     K   Hit Rate  Recall@K
----------------------------
    10     0.1250     0.1250
    20     0.2250     0.2250
    50     0.4150     0.4150
   100     0.6150     0.6150
```

**步骤四：困难负样本挖掘（Hard Negative Mining）**

目标：用当前模型找出"高相似但非正样本"的物品作为新一轮训练的负样本。

```python
def hard_negative_mining(user_encoder, item_vectors, user_features,
                          pos_pairs, top_n=10):
    """
    对每个 (user, true_item) 找出 top_n 个模型打高分但不是正样本的物品，
    作为困难负样本。
    """
    hard_negatives = []
    for uid, true_iid in pos_pairs[:500]:  # 取 500 对做挖掘
        user_vec = user_encoder.predict(user_features[uid:uid+1], verbose=0)
        similarities = np.dot(user_vec, item_vectors.T).flatten()
        # 排除正样本
        similarities[true_iid] = -np.inf
        # 取 top_n 高分物品
        top_ids = np.argsort(similarities)[::-1][:top_n]
        for neg_iid in top_ids:
            hard_negatives.append((uid, neg_iid))
    return hard_negatives

hard_negs = hard_negative_mining(user_encoder, all_item_vectors,
                                  user_features, test_pairs, top_n=5)
print(f"挖掘到 {len(hard_negs)} 个困难负样本")

# 将这些困难负样本加入训练集，继续训练 2 个 epoch
hard_users = user_features[np.array([p[0] for p in hard_negs])]
hard_items = item_features[np.array([p[1] for p in hard_negs])]
hard_labels = np.zeros(len(hard_negs), dtype=np.float32)

# 合并原训练集 + 困难负样本
augmented_users = np.vstack([train_users, hard_users])
augmented_items = np.vstack([train_items, hard_items])
augmented_labels = np.hstack([train_labels, hard_labels])

two_tower.fit(
    [augmented_users, augmented_items], augmented_labels,
    batch_size=512, epochs=2, validation_split=0.1, verbose=1,
)

# 重新评估
print("\n=== 困难负样本 mining 后重新评估 ===")
evaluate_recall(user_encoder, all_item_vectors, user_features, test_pairs)
```

### 3.3 双塔模型部署流程

```
训练阶段:
  用户特征+物品特征 → 双塔模型 → 训练 loss
  ↓ 训练完成后
  提取用户塔 encoder
  提取物品塔 encoder

离线阶段:
  全量物品 → 物品塔 → 物品向量库（存入Faiss/Milvus）
  索引构建（IVF/HNSW）

在线阶段:
  用户请求 → 用户塔 → 用户向量 → ANN检索 → Top 500 候选 → 排序模型 → Top 20
```

## 4. 项目总结

### 4.1 双塔 vs 交互式模型

| 方面 | 双塔模型 | 交互式模型（特征拼接+DNN） |
|------|----------|--------------------------|
| 推理速度 | 极快（仅一次用户编码+向量检索） | 慢（每对用户-物品需跑 DNN） |
| 精度 | 稍低（用户和物品独立编码，缺少交互信息） | 高（用户和物品特征可深度交叉） |
| 物品侧更新 | 离线重算物品向量即可 | 需要全量对计算 |
| 适用阶段 | 召回（百万→千级） | 排序（千级→百级） |
| 部署复杂度 | 需要向量数据库（Faiss/Milvus） | 直接模型 Serving |

### 4.2 适用场景

1. **大规模召回**：百万级物品库中快速筛选候选（如 YouTube 视频召回）
2. **冷启动物品**：新物品只要有特征就能生成向量，即刻参与召回
3. **跨域推荐**：用户塔和物品塔可分别接入不同特征域（文本、图像、行为序列等）
4. **相似物品推荐**：用物品塔编码当前物品，在向量库中找邻居
5. **搜索/广告粗排**：搜索 query 和广告分别编码，快速过滤不相关候选

**不适用场景**：
1. 需要复杂的用户-物品交叉特征（如"用户年龄 × 电影年代"的交互）——这种信息在双塔中被拆散，应交给排序模型
2. 候选集很小（< 1000）——直接上精排模型即可，召回不是瓶颈

### 4.3 注意事项

- **用户塔和物品塔的输出向量维度必须一致**：通常取 64-256 维，维度越大表达能力越强但检索越慢
- **L2 归一化不可省略**：归一化后点积 = 余弦相似度，向量检索（如 Faiss 的 Inner Product）才能正确工作
- **负采样策略比模型结构更重要**：随机负采样 → 难例挖掘 → 流行度采样，三个阶段的模型效果差距可能比换模型结构更大
- **物品向量需要定期重算**：物品特征变化（如评分更新、标签新增）后，需要重新编码全量物品并重建索引

### 4.4 常见踩坑经验

1. **坑**：Recall@K 在训练机上很高，上线后暴跌。
   **根因**：离线评估时用的候选池太小（pool_size=200），用全量 5 万时分布完全不同。
   **解决**：离线评估的 pool_size 至少设为在线候选集的 50%-100%，或直接用"暴力全量排序 + TopK 召回率"评估。

2. **坑**：双塔模型训练 loss 降不下去，一直在 0.69（相当于随机）。
   **根因**：负样本质量太差——随机采的负样本跟用户兴趣本来就不相关，模型轻松区分，学不到细粒度的区分能力。
   **解决**：从第 3 个 epoch 开始混入困难负样本（hard negatives），数量为先前的 10%-20%。

3. **坑**：物品塔输出向量的 L2 范数差异巨大（有的 0.1，有的 100），检索时被大范数的物品主导。
   **根因**：训练时只约束正样本相似度 → 1，没有约束向量的范数均匀分布。
   **解决**：用户塔和物品塔最后一层统一加 L2 归一化（Lambda layer），并在 loss 中加一个 tiny L2 正则化项防止向量坍缩。

### 4.5 思考题

1. 双塔模型训练时，为了加速通常只用随机负采样。但有一类"困难负样本"——当前模型已经给了高分但用户并未点击的物品。如何在训练过程中高效地挖掘这些困难负样本？请设计一个"在线困难负样本 mining"的方案，并讨论它的计算开销。

2. 你有一个双塔模型已上线，物品塔输出的向量存在 Faiss 中。现在新增了 1000 部电影，它们需要在 1 分钟内可被召回。你会如何设计这个"增量更新"流程？物品向量索引的热更新有哪些工程挑战？

### 4.6 推广计划提示

- **新人开发**：从简单的随机负采样 + 双塔开始，验证 pipeline 跑通后再迭代负采样策略
- **算法工程师**：定期（如每周）做一次困难负样本挖掘，重新训练 M 个 epoch，是性价比最高的迭代方式
- **平台工程师**：向量数据库（Faiss/Milvus）的索引构建和在线检索是召回链路的关键，需单独做容量规划和延迟 benchmark
