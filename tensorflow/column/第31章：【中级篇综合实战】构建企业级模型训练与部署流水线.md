# 第31章：【中级篇综合实战】构建企业级模型训练与部署流水线

## 1. 项目背景

某电商公司技术总监在全公司会议上提出了一个目标："半年内，我们的推荐模型从训练到上线要从 2 周缩短到 2 天。"目前团队的现状是——算法工程师在 Jupyter Notebook 上调参，训练完手动导出 SavedModel，复制到共享文件夹，然后发邮件通知运维部署。数据版本混乱（"上周的训练数据是 `data_v3_final.csv` 还是 `data_v3_final2.csv`？"），实验无法复现，线上回滚需要手动操作。

架构师老周受命设计一套"企业级模型训练与部署流水线（MLOps Pipeline）"，目标是把中级篇（第 17-30 章）学到的所有能力——Model Subclassing（第 17 章）、自定义训练（第 18 章）、高性能数据管道（第 19 章）、TensorBoard（第 20 章）、模型调优（第 21 章）、迁移学习（第 22 章）、分布式训练（第 23-24 章）、Serving（第 25 章）、模型测试（第 27 章）、可观测性（第 28 章）、TFX（第 29 章）、安全治理（第 30 章）——串联成一条可演示的端到端流水线。

**痛点放大**：单点技术掌握了但无法串联 = 无法交付真正的端到端 ML 系统。企业级 MLOps 需要"数据版本 → 实验管理 → 分布式训练 → 质量门禁 → 自动部署 → 线上监控 → 回滚"的全链路自动化。

## 2. 项目设计

**小胖**（脑补了一个巨大的工厂流水线）：这不就是把前面 14 章的技术串起来吗？数据管道 → 训练 → 导出 → 部署 → 监控，每个环节我都写了独立代码！

**大师**：每个环节的代码确实都写了——但它们之间的"接口"是手工拼接的。你训练完导出 SavedModel 之后，怎么通知 Serving 加载？Serving 加载失败怎么回滚？你怎么知道今天晚上的自动训练用的是最新的数据版本？这些"胶水"才是 MLOps 最难的地方。

**技术映射**：MLOps 的核心不是单个组件，而是组件之间的自动衔接——数据版本管理、实验追踪、模型注册、自动部署、线上监控的"握手协议"。

**小白**：那架构应该怎么设计？哪些组件是必须的？

**大师**：看这个架构。一条数据从左边流入，经过 6 个层级后，变成右边的在线服务 + 监控大盘。每一个层级都是中级篇一章的内容。串联它们的关键是**标准化接口**——数据层产出 TFRecord，训练层消费 TFRecord 产出 SavedModel，部署层消费 SavedModel 产出 API，监控层消费 API 的 metrics。

**技术映射**：MLOps 流水线的标准化接口 = TFRecord（数据格式）+ SavedModel（模型格式）+ Prometheus Metrics（监控格式）。三种格式各自都有标准的读写方式，组件间解耦。

**小胖**：那有没有一个"总控脚本"一键跑通全流程？

**大师**：有——但你不需要自己写。中级篇的核心交付物不是代码，而是一场**可以演示的端到端 Demo**。你需要展示的是：

1. 修改一行配置（比如把 `LEARNING_RATE` 从 1e-3 改成 5e-4），触发自动训练
2. 训练过程中监控 TensorBoard，看到曲线实时更新
3. 训练完成后，模型自动推送到 Serving 目录
4. 用 `curl` 或 Python 客户端验证新模型已经在线
5. Grafana 大盘显示新旧模型的 QPS/延迟对比
6. 模拟一次"新模型延迟飙升" → 触发告警 → 自动回滚到上一版本

这 6 步如果你能在 15 分钟内演示完，并且第二次能复现——你就交付了一条合格的 MLOps 流水线。

**步骤二：流水线编排脚本（一键运行）**

目标：用 `run_pipeline.sh` 串联所有步骤，支持单步执行和断点续跑。

```python
# run_pipeline.py —— 流水线编排器
import subprocess
import sys
import os
import config

PIPELINE_STEPS = [
    ("build_data",     "python pipeline/build_data.py"),
    ("train_model",    "python pipeline/train_model.py"),
    ("evaluate_model", "python pipeline/evaluate_model.py"),
    ("export_model",   "python pipeline/export_model.py"),
    ("run_tests",      "pytest tests/ -v --tb=short"),
    ("deploy_model",   "python pipeline/deploy_model.py"),
]

def run_pipeline(start_from=None, stop_before=None):
    """编排运行流水线"""
    started = (start_from is None)
    results = {}

    for step_name, cmd in PIPELINE_STEPS:
        if start_from and step_name == start_from:
            started = True
        if stop_before and step_name == stop_before:
            break
        if not started:
            print(f"[SKIP] {step_name}")
            continue

        print(f"\n{'='*60}")
        print(f"[RUN]  {step_name}")
        print(f"[CMD]  {cmd}")
        print(f"{'='*60}")

        result = subprocess.run(cmd, shell=True, capture_output=False)
        if result.returncode != 0:
            print(f"[FAIL] {step_name} 失败 (exit code: {result.returncode})")
            results[step_name] = "FAILED"
            break
        else:
            results[step_name] = "SUCCESS"

    # 汇总
    print(f"\n{'='*60}")
    print(f"流水线执行汇总")
    print(f"{'='*60}")
    for step, status in results.items():
        status_icon = "✅" if status == "SUCCESS" else "❌"
        print(f"  {status_icon} {step}: {status}")

    return all(s == "SUCCESS" for s in results.values())

if __name__ == "__main__":
    import argparse
    parser = argparse.ArgumentParser()
    parser.add_argument("--start-from", type=str, help="从某一步开始")
    parser.add_argument("--stop-before", type=str, help="在某一步之前停止")
    args = parser.parse_args()
    success = run_pipeline(args.start_from, args.stop_before)
    sys.exit(0 if success else 1)
```

**步骤三：质量门禁集成 + 自动部署**

目标：模型评估不达标则阻止部署，达标则自动推送到 Serving。

```python
# pipeline/evaluate_model.py —— 质量门禁检查
import tensorflow as tf
from tensorflow import keras
import numpy as np
from sklearn.metrics import accuracy_score, roc_auc_score
import config
import sys
import os

def evaluate_and_gate(model_path=None):
    """评估模型并执行质量门禁"""
    if model_path is None:
        model_path = os.path.join(config.MODEL_DIR, "latest.keras")

    model = keras.models.load_model(model_path)

    # 加载 Golden Dataset（固定测试集）
    # 实际项目中从固定路径加载，此处模拟
    np.random.seed(config.TRAIN_CONFIG["random_seed"])
    X_golden = np.random.randn(1000, config.MODEL_INPUT_DIM).astype(np.float32)
    y_golden = np.random.randint(0, config.MODEL_NUM_CLASSES, 1000)

    # 评估
    loss, acc = model.evaluate(X_golden, y_golden, verbose=0)
    y_pred_probs = model.predict(X_golden, verbose=0)
    y_pred = np.argmax(y_pred_probs, axis=1)

    # 计算 AUC (one-vs-rest)
    from sklearn.preprocessing import label_binarize
    y_bin = label_binarize(y_golden, classes=range(config.MODEL_NUM_CLASSES))
    try:
        auc = roc_auc_score(y_bin, y_pred_probs, average="macro", multi_class="ovr")
    except ValueError:
        auc = 0.5  # 退化情况

    # 门禁检查
    checks = {
        "accuracy": (acc, config.QUALITY_GATES["min_accuracy"]),
        "auc": (auc, config.QUALITY_GATES["min_auc"]),
    }

    all_passed = True
    print(f"\n=== 质量门禁报告 ===")
    print(f"{'指标':<12} {'当前值':>8} {'阈值':>8} {'结果':>8}")
    print("-" * 40)
    for name, (actual, threshold) in checks.items():
        passed = actual >= threshold
        status = "✅ PASS" if passed else "❌ FAIL"
        print(f"{name:<12} {actual:>8.4f} {threshold:>8.4f} {status:>8}")
        if not passed:
            all_passed = False

    if not all_passed:
        print("\n模型未通过质量门禁，阻止部署！")
        sys.exit(1)
    else:
        print("\n模型通过所有质量门禁 ✅")

# pipeline/deploy_model.py —— 自动部署到 Serving
def deploy_to_serving(model_source, model_name, version):
    """将模型部署到 Serving 目录"""
    import shutil
    serving_dir = os.path.join(config.SERVING_CONFIG["model_base_path"], str(version))
    os.makedirs(os.path.dirname(serving_dir), exist_ok=True)

    if os.path.exists(serving_dir):
        shutil.rmtree(serving_dir)
    shutil.copytree(model_source, serving_dir)

    print(f"模型已部署到: {serving_dir}")
    print(f"Serving 将自动加载新版本 (热加载)")
    print(f"验证命令: curl http://localhost:{config.SERVING_CONFIG['rest_api_port']}/v1/models/{model_name}")

    # 清理旧版本（保留最近 3 个版本）
    base = config.SERVING_CONFIG["model_base_path"]
    versions = sorted([int(d) for d in os.listdir(base) if d.isdigit()], reverse=True)
    for old_ver in versions[3:]:
        old_path = os.path.join(base, str(old_ver))
        shutil.rmtree(old_path)
        print(f"清理旧版本: {old_path}")
```

### 3.7 自动回滚——Serving 版本目录 + Prometheus Alertmanager

```python
# pipeline/auto_rollback.py —— 自动回滚脚本
import subprocess
import sys
import config

def rollback_to_previous(model_name):
    """删除当前版本目录，Serving 自动回退到上一版本"""
    base = config.SERVING_CONFIG["model_base_path"]
    versions = sorted(
        [d for d in os.listdir(os.path.join(base, model_name)) if d.isdigit()],
        reverse=True,
    )

    if len(versions) < 2:
        print("错误: 无可回滚的旧版本")
        return False

    current = versions[0]
    previous = versions[1]

    # 删除当前版本（触发 Serving 自动回退）
    current_path = os.path.join(base, model_name, str(current))
    shutil.rmtree(current_path)

    print(f"回滚完成: 版本 {current} → 版本 {previous}")
    return True

# Prometheus Alertmanager webhook 触发点
```

### 4.2 适用场景

1. **企业级 MLOps 平台搭建**：以本章架构为蓝图，适配公司技术栈
2. **团队 Onboarding 结业项目**：新成员完成中级篇学习后，独立搭一条流水线作为结业考核
3. **向管理层汇报的 Demo**：15 分钟演示"从配置修改到上线监控"的全流程

**不适用场景**：
1. 个人研究或快速实验——本章的工程化成本高于收益
2. 团队 < 3 人——分工不足，流水线的维护成本可能高于手工作坊

### 4.3 注意事项

- **数据版本管理**：训练数据、Golden Dataset、特征配置三者必须版本绑定。推荐用 DVC（Data Version Control）或 Git LFS
- **流水线的幂等性**：每个 step 必须支持重复运行（不会重复训练/重复部署）。实现方法：检查输出文件是否存在且 hash 未变时 skip
- **Serving 热加载的"影子期"**：新模型加载后 GPU 预热的 5-10 秒内推理延迟可能偏高——灰度初期忽略这段延迟

### 4.4 思考题

1. 你的流水线在"训练完成 → 自动部署"中间隔了一个"人工审批"环节——监管要求高风险模型必须人工点击"通过"才能上线。你如何在不阻塞流水线的前提下实现这个"半自动"门禁？（提示：考虑 Serving 的 model_version_policy）

2. 你有 3 个模型同时训练（推荐/风控/搜索），它们共享同一个 Serving 实例。当一个模型的自动部署触发时，如何保证不影响其他两个模型的正常服务？
