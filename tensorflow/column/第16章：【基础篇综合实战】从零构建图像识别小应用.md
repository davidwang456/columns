一家零售连锁门店提出需求：在收银台安装一个简易摄像头，当顾客拿着商品结账时，系统自动识别商品品类（饮料、零食、日用品、文具、生鲜 5 类），辅助收银员快速录入。信息部负责人找到算法团队："给我们做一个商品图片识别 Demo，不需要多精准——能分对 5 个大类，准确率 85% 以上就行。但要求整套东西能跑起来：有人拍照、系统出结果、结果能给别人演示。"

## 2. 项目设计

**小胖**（拿着一袋薯片走向"收银台"——小赵的工位）：这个识别系统也太简单了吧？用第 10 章的 CNN 加第 6 章的 tf.data 不就完了？我之前每章的代码都能跑啊。

**大师**（走过来看了一眼小胖的屏幕）：你那每章的代码确实是能跑——但它们之间的依赖关系一团乱。你的训练脚本里硬编码了数据路径 `/home/xiaopang/test_imgs/`，换到赵哥的电脑上就跑不了。你的超参数散落在 5 个文件里，想调个学习率要找半天。最重要的是——你交付给信息部的是什么东西？一个 `.ipynb` 文件？人家怎么演示？

**小胖**：那……我把所有代码写到一个 `train.py` 里不就得了？

**大师**：对于一个需要演示、需要给别人跑、可能需要后续维护的项目，"所有代码放一个文件"是最糟糕的选择。你需要的是一个标准的项目结构：

```
product_classifier/
├── README.md           ← 别人看了就知道怎么跑
├── requirements.txt    ← 锁定依赖版本（第 2 章）
├── config.py           ← 所有超参数和路径集中管理
├── data_pipeline.py    ← tf.data 管道（第 6 章）
├── model.py            ← CNN 模型定义（第 10 章）
├── train.py            ← 训练脚本（第 8 章 Callback）
├── evaluate.py         ← 评估脚本（混淆矩阵/准确率）
├── export.py           ← 导出 SavedModel（第 9 章）
├── infer.py            ← 单张图片推理 Demo
├── data/
│   └── (放原始图片，按品类分子目录)
└── outputs/
    ├── models/         ← 训练好的模型
    └── logs/           ← TensorBoard 日志
```

**技术映射**：工程化项目结构遵循"关注点分离"原则——配置/数据/模型/训练/评估/导出各司其职。每个文件职责单一，方便测试、修改和团队协作。

**小白**（看着这个目录结构若有所思）：每个文件具体的职责边界在哪？比如 `train.py` 和 `model.py` 都跟模型有关，怎么划分？

**大师**：好问题。划分原则是——`model.py` 只负责"定义模型结构"（一个函数，输入 config，输出 keras.Model），不包含任何训练逻辑。`train.py` 调用 `model.py` 的函数拿到模型，然后负责"编译 + 训练 + 保存"。这样一来，如果你想换一个模型（比如从 CNN 换成 MobileNet），只需要改 `model.py`，`train.py` 完全不用动。

**小白**：那 `config.py` 里应该放什么？

**大师**：所有"可能会变但又不想每次改代码"的东西。包括：数据路径、图片尺寸、batch_size、learning_rate、模型层数/通道数、训练 epochs、类别名称列表。一个硬性标准——如果你的代码里出现了任何数字字面量（除了 0 和 1），它就应该来自 config。

**技术映射**：配置文件是项目"可复现性"和"可维护性"的基础。把参数从代码中抽离，可以让你在不改代码的情况下调整超参数；也可以通过 Git 追踪不同实验的配置差异。

**小胖**：那推理脚本 `infer.py` 怎么让非技术人员用？人家又不懂 Python。

**大师**：最简单的方案——包装成一个命令行工具。信息部的人只需要在命令行输入：

```bash
python infer.py --image 饮料.jpg
```

输出就是：`预测结果: 饮料 (置信度 92.3%)`。如果后面需要做成 Web API（比如用 Flask 包装），`infer.py` 的核心推理逻辑可以直接复用。

**步骤二：数据管道（`data_pipeline.py`）**

目标：从目录按品类读取图片，构建训练/验证 tf.data 管道。

```python
# data_pipeline.py —— 数据加载与增强
import tensorflow as tf
import config

AUTOTUNE = tf.data.AUTOTUNE

def load_and_preprocess(path, label):
    """从文件读取、解码、resize、归一化"""
    img = tf.io.read_file(path)
    img = tf.io.decode_jpeg(img, channels=3)
    img = tf.image.resize(img, config.IMG_SIZE)
    img = tf.cast(img, tf.float32) / 255.0
    return img, label

def augment(image, label):
    """训练数据增强"""
    image = tf.image.random_flip_left_right(image)
    image = tf.image.random_brightness(image, max_delta=0.1)
    image = tf.image.random_contrast(image, lower=0.9, upper=1.1)
    return image, label

def get_datasets(data_dir, batch_size=None):
    """从目录加载图片，按子目录名作为类别标签"""
    if batch_size is None:
        batch_size = config.BATCH_SIZE

    # 获取所有图片路径和标签
    paths, labels = [], []
    class_names = sorted(os.listdir(data_dir))  # 按字母序 = 0,1,2...
    for label_idx, cls_name in enumerate(class_names):
        cls_dir = os.path.join(data_dir, cls_name)
        if not os.path.isdir(cls_dir):
            continue
        for fname in os.listdir(cls_dir):
            if fname.lower().endswith((".jpg", ".jpeg", ".png")):
                paths.append(os.path.join(cls_dir, fname))
                labels.append(label_idx)

    # 创建 Dataset
    ds = tf.data.Dataset.from_tensor_slices((paths, labels))
    ds = ds.shuffle(buffer_size=len(paths))
    ds = ds.map(load_and_preprocess, num_parallel_calls=AUTOTUNE)
    return ds

def build_train_val_pipeline(data_dir=None):
    """构建训练和验证管道"""
    if data_dir is None:
        data_dir = config.DATA_DIR

    full_ds = get_datasets(data_dir)
    ds_size = sum(1 for _ in full_ds)
    val_size = int(ds_size * config.VALIDATION_SPLIT)
    train_size = ds_size - val_size

    full_ds = get_datasets(data_dir)  # re-create dataset

    train_ds = full_ds.take(train_size)
    train_ds = train_ds.map(augment, num_parallel_calls=AUTOTUNE)
    train_ds = train_ds.batch(config.BATCH_SIZE).prefetch(AUTOTUNE)

    val_ds = full_ds.skip(train_size)
    val_ds = val_ds.batch(config.BATCH_SIZE).prefetch(AUTOTUNE)

    return train_ds, val_ds, train_size, val_size
```

**步骤三：模型定义（`model.py`）**

目标：构建一个三层 CNN + GAP 的分类模型。可解析 `config.CONV_FILTERS` 灵活调整深度。

```python
# model.py —— CNN 分类模型定义
from tensorflow import keras
import config

def create_product_classifier():
    """创建一个 CNN 商品分类模型"""
    inputs = keras.Input(shape=(*config.IMG_SIZE, 3))

    x = inputs
    for filters in config.CONV_FILTERS:
        x = keras.layers.Conv2D(filters, 3, activation="relu", padding="same")(x)
        x = keras.layers.Conv2D(filters, 3, activation="relu", padding="same")(x)
        x = keras.layers.MaxPooling2D(2)(x)
        x = keras.layers.Dropout(config.DROPOUT_RATE)(x)

    x = keras.layers.GlobalAveragePooling2D()(x)
    x = keras.layers.Dense(config.DENSE_UNITS, activation="relu")(x)
    x = keras.layers.Dropout(config.DROPOUT_RATE + 0.2)(x)
    outputs = keras.layers.Dense(config.NUM_CLASSES, activation="softmax")(x)

    model = keras.Model(inputs, outputs, name="product_classifier")
    return model

if __name__ == "__main__":
    model = create_product_classifier()
    model.summary()
```

**步骤四：训练脚本（`train.py`）**

目标：整合数据管道、模型、Callback，完成训练并保存模型。

```python
# train.py —— 训练入口
import os, random, numpy as np, tensorflow as tf
from tensorflow import keras
import config
from data_pipeline import build_train_val_pipeline
from model import create_product_classifier

# === 1. 固定随机种子 ===
random.seed(config.RANDOM_SEED)
np.random.seed(config.RANDOM_SEED)
tf.random.set_seed(config.RANDOM_SEED)

# === 2. 数据管道 ===
print("Building data pipeline...")
train_ds, val_ds, train_n, val_n = build_train_val_pipeline()
print(f"Train samples: {train_n}, Val samples: {val_n}")

# === 3. 构建模型 ===
print("Creating model...")
model = create_product_classifier()

model.compile(
    optimizer=keras.optimizers.Adam(learning_rate=config.LEARNING_RATE),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

# === 4. Callbacks ===
callbacks = [
    keras.callbacks.EarlyStopping(
        monitor="val_loss", patience=config.EARLY_STOP_PATIENCE, restore_best_weights=True
    ),
    keras.callbacks.ReduceLROnPlateau(
        monitor="val_loss", factor=0.5, patience=config.REDUCE_LR_PATIENCE, min_lr=1e-6
    ),
    keras.callbacks.ModelCheckpoint(
        filepath=os.path.join(config.MODEL_DIR, "best_model.keras"),
        monitor="val_accuracy", save_best_only=True, mode="max",
    ),
    keras.callbacks.TensorBoard(
        log_dir=config.LOG_DIR, histogram_freq=1,
    ),
    keras.callbacks.CSVLogger(os.path.join(config.OUTPUT_DIR, "training_log.csv")),
]

# === 5. 训练 ===
print("Starting training...")
history = model.fit(
    train_ds,
    validation_data=val_ds,
    epochs=config.EPOCHS,
    callbacks=callbacks,
    verbose=1,
)

# === 6. 保存最终模型 ===
model.save(os.path.join(config.MODEL_DIR, "final_model.keras"))
print(f"Training complete. Model saved to {config.MODEL_DIR}")
```

**步骤五：评估脚本（`evaluate.py`）**

目标：加载模型，输出准确率和类别混淆矩阵。

```python
# evaluate.py —— 模型评估
import tensorflow as tf
from tensorflow import keras
import numpy as np
from sklearn.metrics import classification_report, confusion_matrix
import config
from data_pipeline import get_datasets

def evaluate_model(model_path=None):
    if model_path is None:
        model_path = os.path.join(config.MODEL_DIR, "best_model.keras")

    model = keras.models.load_model(model_path)

    # 评估数据集
    val_ds = get_datasets(config.DATA_DIR)
    val_ds = val_ds.batch(config.BATCH_SIZE).prefetch(tf.data.AUTOTUNE)

    all_preds, all_labels = [], []
    for imgs, lbls in val_ds:
        preds = model.predict(imgs, verbose=0)
        all_preds.extend(np.argmax(preds, axis=1))
        all_labels.extend(lbls.numpy())

    acc = np.mean(np.array(all_preds) == np.array(all_labels))
    print(f"\n总体准确率: {acc:.2%}")

    print("\n分类报告:")
    print(classification_report(all_labels, all_preds, target_names=config.CLASS_NAMES))

    print("混淆矩阵:")
    cm = confusion_matrix(all_labels, all_preds)
    header = "        " + " ".join(f"{n:<6}" for n in config.CLASS_NAMES)
    print(header)
    for i, row in enumerate(cm):
        print(f"{config.CLASS_NAMES[i]:<6} " + " ".join(f"{v:<6}" for v in row))

    return acc, cm
```

**步骤六：推理脚本（`infer.py`）**

目标：命令行工具，接收图片路径，输出 Top3 预测结果。

```python
# infer.py —— 单张图片推理
import sys, os, argparse
import tensorflow as tf
import numpy as np
import config

def load_model(model_path=None):
    if model_path is None:
        model_path = os.path.join(config.MODEL_DIR, "best_model.keras")
    return tf.keras.models.load_model(model_path)

def preprocess_image(image_path):
    """加载并预处理单张图片"""
    img = tf.io.read_file(image_path)
    img = tf.io.decode_jpeg(img, channels=3)
    img = tf.image.resize(img, config.IMG_SIZE)
    img = tf.cast(img, tf.float32) / 255.0
    return tf.expand_dims(img, 0)  # (1, H, W, 3)

def predict(model, image_path, top_k=3):
    """预测并返回 TopK 结果"""
    img_tensor = preprocess_image(image_path)
    probs = model.predict(img_tensor, verbose=0)[0]
    top_indices = np.argsort(probs)[::-1][:top_k]

    print(f"\n图片: {image_path}")
    print(f"{'排名':<4} {'类别':<8} {'置信度':<10}")
    print("-" * 24)
    for rank, idx in enumerate(top_indices, 1):
        cls_name = config.CLASS_NAMES[idx] if idx < len(config.CLASS_NAMES) else f"Class_{idx}"
        print(f"{rank:<4} {cls_name:<8} {probs[idx]:.2%}")
    return [(config.CLASS_NAMES[i], probs[i]) for i in top_indices]

if __name__ == "__main__":
    parser = argparse.ArgumentParser(description="商品图片识别")
    parser.add_argument("--image", type=str, required=True, help="图片路径")
    parser.add_argument("--model", type=str, default=None, help="模型路径（可选）")
    args = parser.parse_args()

    model = load_model(args.model)
    predict(model, args.image)
```

### 3.4 批量推理优化——模型常驻内存

目标：解决 `infer.py` 每次调用都重新加载模型的问题，实现"一次加载、多次推理"。

```python
# batch_infer.py —— 批量推理（模型常驻内存）
import tensorflow as tf
import numpy as np
import os
import time
import config

class ProductClassifier:
    """模型常驻内存的分类器（可复用）"""

    def __init__(self, model_path=None):
        if model_path is None:
            model_path = os.path.join(config.MODEL_DIR, "best_model.keras")

        load_start = time.time()
        self.model = tf.keras.models.load_model(model_path)
        self.load_time = time.time() - load_start
        print(f"模型加载完成 ({self.load_time:.1f}s)")

        # 记录类别名
        self.class_names = config.CLASS_NAMES

    def preprocess(self, image):
        """预处理（支持 PIL Image / numpy array / 文件路径）"""
        if isinstance(image, str):
            img = tf.io.read_file(image)
            img = tf.io.decode_jpeg(img, channels=3)
        elif hasattr(image, 'numpy'):
            img = tf.constant(image)
        else:
            img = tf.convert_to_tensor(image)

        img = tf.image.resize(img, config.IMG_SIZE)
        img = tf.cast(img, tf.float32) / 255.0
        return tf.expand_dims(img, 0)

    def predict_single(self, image, top_k=3):
        """单张图片预测"""
        img_tensor = self.preprocess(image)
        start = time.time()
        probs = self.model.predict(img_tensor, verbose=0)[0]
        latency = (time.time() - start) * 1000  # ms

        top_indices = np.argsort(probs)[::-1][:top_k]
        results = []

        print(f"\n{'='*50}")
        print(f"预测结果 (延迟: {latency:.1f}ms)")
        print(f"{'='*50}")
        print(f"{'排名':<6} {'类别':<10} {'置信度':<10}")
        print(f"{'-'*26}")
        for rank, idx in enumerate(top_indices, 1):
            cls = self.class_names[idx] if idx < len(self.class_names) else f"Class_{idx}"
            print(f"{rank:<6} {cls:<10} {probs[idx]:.2%}")
            results.append({"rank": rank, "class": cls, "confidence": float(probs[idx])})

        return results

    def predict_batch(self, image_paths, top_k=3):
        """批量图片预测——显著快于逐张 predict_single"""
        batch_tensors = []
        for path in image_paths:
            tensor = self.preprocess(path)
            batch_tensors.append(tensor[0])  # 去掉 batch 维度

        batch_input = tf.stack(batch_tensors, axis=0)  # (N, H, W, 3)
        start = time.time()
        probs = self.model.predict(batch_input, verbose=0)
        total_latency = (time.time() - start) * 1000

        results = []
        for i in range(len(image_paths)):
            top_indices = np.argsort(probs[i])[::-1][:top_k]
            results.append([{
                "class": self.class_names[idx] if idx < len(self.class_names) else f"Class_{idx}",
                "confidence": float(probs[i][idx])
            } for idx in top_indices])

        avg_latency = total_latency / len(image_paths)
        print(f"\n批量预测: {len(image_paths)} 张, 总耗时={total_latency:.1f}ms, 平均={avg_latency:.1f}ms/张")
        return results

# === 批量 vs 逐张性能对比 ===
classifier = ProductClassifier()

# 生成 50 张测试图片
import tempfile
tmpdir = tempfile.mkdtemp()
test_paths = []
for i in range(50):
    fpath = os.path.join(tmpdir, f"test_{i:04d}.jpg")
    img = (np.random.rand(*config.IMG_SIZE, 3) * 255).astype(np.uint8)
    tf.io.write_file(fpath, tf.io.encode_jpeg(img))
    test_paths.append(fpath)

# 逐张推理
start = time.time()
for path in test_paths:
    classifier.predict_single(path, top_k=1)
single_time = time.time() - start

# 批量推理
start = time.time()
classifier.predict_batch(test_paths, top_k=1)
batch_time = time.time() - start

print(f"\n{'='*50}")
print(f"推理模式对比 (50 张图片)")
print(f"{'='*50}")
print(f"逐张推理: {single_time:.1f}s ({(single_time/50*1000):.1f}ms/张)")
print(f"批量推理: {batch_time:.1f}s ({(batch_time/50*1000):.1f}ms/张)")
```

## 4. 项目总结

### 4.1 技术要点回顾

本章串联了基础篇 15 章的核心技能：

| 章节 | 技能 | 在本章的应用 |
|------|------|-------------|
| 第 2 章 | 环境管理 + 随机种子 | `requirements.txt` + `RANDOM_SEED` 配置 |
| 第 3 章 | Tensor 操作 | `preprocess_image()` 中的 resize/cast/expand_dims |
| 第 5 章 | Keras Functional API | `model.py` 中的 Input → Conv → GAP → Dense |
| 第 6 章 | tf.data 管道 | `data_pipeline.py` 中 map/batch/shuffle/prefetch |
| 第 7 章 | 损失函数与优化器 | `compile(loss="sparse_categorical_crossentropy",...)` |
| 第 8 章 | Callback 管理 | EarlyStopping/ModelCheckpoint/ReduceLROnPlateau/TensorBoard |
| 第 9 章 | 模型保存 | `model.save("best_model.keras")` / `load_model()` |
| 第 10 章 | CNN 模型 | Conv2D + MaxPooling + GAP 三 Block 架构 |
| 第 15 章 | 故障排查 | NaN 检查、shape 验证、训练曲线解读 |

### 4.2 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| 项目结构 | 关注点分离，每个文件 < 100 行，易于理解和修改 | 文件数量多，新手可能觉得"过度设计" |
| 配置集中 | `config.py` 改一行即可切换数据路径、模型大小 | 配置项增多后需文档化每个参数的含义 |
| 命令行推理 | 非技术人员也能使用 | 单张推理延迟较高（需加载模型），实际生产应常驻服务 |
| 可复现性 | 固定种子 + requirements.txt | GPU 上的浮点累加顺序仍有微弱随机性 |

### 4.3 适用场景

1. **团队新人 Onboarding**：以此项目为模板，替换数据和模型即可开始自己的第一个 TensorFlow 项目
2. **快速 Demo 演示**：1 天内搭出一个可演示的图像分类应用
3. **教学培训**：作为基础篇的结课项目，覆盖全部核心 API

**不适用场景**：
1. 生产级高并发推理服务——应使用 TensorFlow Serving（第 25 章）
2. 百万级图片——需要 TFRecord + 分布式训练（第 19 章、第 23 章）

### 4.4 思考题

1. 当前项目只用了图片数据。如果业务方要求同时输入"商品条码"作为辅助特征，你需要在 `model.py` 中做哪些修改？请画出修改后的模型架构图。

2. `infer.py` 每次推理都要重新 `load_model()`，这在批量推理时非常低效。请把它改造成一个"模型常驻内存 + 批量推理"的版本，并计算你在 100 张图片上节省了多少时间。
