# 第2章：环境搭建与第一个可复现实验

## 1. 项目背景

算法实习生小王入职了一家零售企业，mentor 给他的第一个任务很简单："在本机搭建 TensorFlow 环境，跑通 MNIST 手写数字识别，把训练结果记录下来，下周让你的同事也能在你的机器上跑出一模一样的结果。"

小王心想这还不简单？打开命令行 `pip install tensorflow`，报错 `ERROR: Could not find a version that satisfies the requirement tensorflow`。一查才知道自己用的 Python 3.13 还没有对应的 TensorFlow 版本。降级到 Python 3.10，再次安装，成功了！兴奋地跑代码——`ImportError: DLL load failed while importing _pywrap_tensorflow_internal`。又折腾了一下午，发现是 CUDA 版本与显卡驱动不匹配。

好不容易环境跑通了，小王在 Jupyter Notebook 上训练了一个 MNIST 模型，准确率 98.2%，截图发给 mentor。mentor 问："把代码和环境依赖发我，我在我电脑上跑一下。"小王把 `.ipynb` 文件丢过去，mentor 跑了 20 分钟，Loss 一直 NaN，准确率 10%。——原来小王用了 GPU 而 mentor 的笔记本没有 GPU，小王代码里又没有固定随机种子。

**痛点放大**：深度学习项目从一开始就面临三个"不可复现"的陷阱：环境不可复现（Python 版本、包版本、CUDA 版本不一致）、代码不可复现（随机种子未固定、依赖隐式版本）、数据不可复现（数据预处理逻辑散落在各个 Cell 中）。这些问题会随项目复杂度指数级放大。

本章从零开始，搭建一个"换台机器也能跑出同样结果"的 TensorFlow 项目模板，并完成 MNIST 分类实验。

## 2. 项目设计

**小胖**（晃着手里的可乐瓶）：我说，装个 TensorFlow 咋这么烦？pip install 不行，conda install 也不行，官网还说要匹配 CUDA、cuDNN 版本，这一堆东西是啥关系啊？不就跟我要喝可乐——拧开瓶盖就行了，结果你说得先配好瓶盖材质、瓶口螺纹规格、碳酸浓度计，这不是有病吗？

**大师**（忍俊不禁）：你这个比喻其实挺对的——TensorFlow 就是一个"碳酸饮料系统"。Python 是瓶子，TensorFlow 是可乐液，CUDA 是瓶盖的密封圈，cuDNN 是瓶口的特殊设计让你喝得更快（加速卷积等操作），NVIDIA Driver 是开瓶器。任何一环不匹配，要么漏气（DLL 报错），要么打不开（import 失败）。

**小胖**：那直接喝罐装的不就完了？搞什么 Docker？

**大师**：对！Docker 就是"罐装可乐"——TensorFlow 官方把 Python、TensorFlow、CUDA、cuDNN 打包在一个镜像里，你 `docker run` 就行，根本不用自己配环境。但罐装虽方便，你要是想自己调配方呢？比如加柠檬（自定义 CUDA Kernel）、换甜度（特定版本的依赖包），还得回到手动配置。

**技术映射**：Docker 镜像解决了环境一致性问题；手动 pip/conda 配置需要严格遵循 NVIDIA CUDA 兼容矩阵（https://www.tensorflow.org/install/source#gpu）。

**小白**（盯着屏幕上的错误日志）：那我还有个问题——为什么一定要固定随机种子？随机不就是为了让模型学到"可能性"吗？固定种子不就变死了？

**大师**（从抽屉里拿出一副扑克牌）：好问题。随机性在训练中确实必要——它防止模型走极端（比如每次都按同一个顺序学习样本，产生偏差）。但你想象一下：你训练了一个模型，准确率 98%；你同事用同样的代码和数据训练，准确率 85%。老板问你俩"这模型到底靠不靠谱？"你怎么回答？固定随机种子，就像洗牌时记住洗牌的顺序——你俩各洗一次，大概率牌序不一样（结果不同）。但如果你约定好"第 1 张和第 33 张互换，第 2 张和第 17 张互换..."，那无论洗多少次，结果完全一样。

**小白**：也就是说随机种子是"可复现的随机"？

**大师**：对。计算机里的"随机"其实是伪随机——只要算法和种子相同，输出的随机序列一模一样。TensorFlow、NumPy、Python 内置 random 库各有各的随机数生成器，所以需要分别固定：

```python
import random, numpy as np, tensorflow as tf
random.seed(42)
np.random.seed(42)
tf.random.set_seed(42)
```

**技术映射**：固定随机种子是机器学习实验可复现性的必要条件，不固定种子导致的准确率波动（±2%）可能掩盖模型改进的真实效果。

**小胖**（凑过来看屏幕）：那 conda 和 virtualenv 这俩东西是干啥的？为啥不能直接 pip install 到系统里？

**大师**：问得好。假设你同时做两个项目——项目 A 需要 TensorFlow 2.16 + NumPy 1.26，项目 B 需要 TensorFlow 2.10 + NumPy 1.24。如果你把两个版本的包都装到系统 Python 里，它们会互相"打架"——哪个版本最终生效取决于安装顺序。conda/virtualenv 就是给每个项目一个"独立的小房间"——A 项目在 301，B 项目在 302，彼此看不到对方的东西，互不干扰。

**小白**：那 conda 和 virtualenv 怎么选？

**大师**：virtualenv 轻量，只隔离 Python 包；conda 隔离 Python + C 库（包括 CUDA、cuDNN）。如果你只用 Python 生态，virtualenv + pip 足够；如果你需要精确控制 CUDA 版本或混用 Python + R + Julia，conda 更合适。我们项目中用 `python -m venv`（virtualenv 内建版）就够。

**技术映射**：虚拟环境是 Python 项目的标准隔离实践，`python -m venv` 零额外依赖，`requirements.txt` 锁定依赖版本。

## 3. 项目实战

### 3.1 环境准备

**操作系统兼容说明**：
- Windows：支持 TensorFlow 2.x（无 GPU 加速的 DLL 问题较少）
- Linux（Ubuntu 20.04/22.04）：GPU 支持最完善
- macOS：仅支持 CPU 版（Apple Silicon 需通过 `tensorflow-macos` 和 `tensorflow-metal`）

**最小化配置（CPU 版，所有平台通用）**：

```bash
# 1. 检查 Python 版本（需 3.9-3.12）
python --version

# 2. 创建虚拟环境
python -m venv tf_mnist_env

# 3. 激活虚拟环境
# Windows:
tf_mnist_env\Scripts\activate
# Linux/Mac:
source tf_mnist_env/bin/activate

# 4. 安装 TensorFlow CPU 版
pip install tensorflow==2.16.1

# 5. 验证安装
python -c "import tensorflow as tf; print('TF version:', tf.__version__); print('GPU available:', tf.config.list_physical_devices('GPU'))"
```

**GPU 版补充配置**（Windows/Linux，NVIDIA GPU）：

```bash
# 确认 CUDA 和 cuDNN 兼容性
# TensorFlow 2.16 → CUDA 12.3, cuDNN 8.9
# 参考: https://www.tensorflow.org/install/source#gpu
nvidia-smi                       # 查看驱动版本和 CUDA 支持上限
pip install tensorflow[and-cuda]==2.16.1  # 自动包含 CUDA 依赖
```

> **坑点1**：Windows 安装 GPU 版报 `DLL load failed` → 安装 Microsoft Visual C++ Redistributable (vc_redist.x64.exe)。
>
> **坑点2**：`pip install tensorflow` 报找不到匹配版本 → Python 版本过高/过低，检查 Python 版本是否在 TF 支持范围内。

### 3.2 创建可复现项目模板

项目目录结构：

```
mnist_project/
├── README.md              # 运行说明
├── requirements.txt       # 固定依赖版本
├── config.py              # 超参数与路径配置
├── train.py               # 训练脚本
├── model.py               # 模型定义
└── results/               # 训练输出目录
    └── .gitkeep
```

**分步实现**：

**步骤一：`requirements.txt` — 锁定依赖版本**

目标：确保 `pip install -r requirements.txt` 在任何机器上安装完全相同的包版本。

```text
tensorflow==2.16.1
numpy==1.26.4
matplotlib==3.8.4
```

```bash
# 生成当前环境的依赖锁定文件（可选）
pip freeze > requirements.txt
```

**步骤二：`config.py` — 集中管理超参数**

目标：所有可调参数集中一处，修改后自动影响训练和评估。

```python
# config.py —— 超参数与配置集中管理
import os

RANDOM_SEED = 42

TRAIN_CONFIG = {
    "batch_size": 64,
    "epochs": 5,
    "learning_rate": 1e-3,
    "validation_split": 0.2,
}

MODEL_CONFIG = {
    "hidden_units": [128, 64],
    "dropout_rate": 0.2,
    "num_classes": 10,
}

PATH_CONFIG = {
    "results_dir": os.path.join(os.path.dirname(__file__), "results"),
    "model_path": os.path.join(os.path.dirname(__file__), "results", "mnist_model.keras"),
}
```

**步骤三：`model.py` — 模型定义**

目标：将模型构建独立成函数，方便单元测试和后续替换。

```python
# model.py —— 模型定义
import tensorflow as tf
from tensorflow import keras
from config import MODEL_CONFIG

def create_mnist_model():
    """构建一个简单的 MNIST 分类 MLP"""
    model = keras.Sequential([
        keras.layers.Flatten(input_shape=(28, 28)),
        keras.layers.Dense(MODEL_CONFIG["hidden_units"][0], activation="relu"),
        keras.layers.Dropout(MODEL_CONFIG["dropout_rate"]),
        keras.layers.Dense(MODEL_CONFIG["hidden_units"][1], activation="relu"),
        keras.layers.Dropout(MODEL_CONFIG["dropout_rate"]),
        keras.layers.Dense(MODEL_CONFIG["num_classes"], activation="softmax"),
    ])
    return model

if __name__ == "__main__":
    model = create_mnist_model()
    model.summary()
```

**步骤四：`train.py` — 完整训练脚本**

目标：从数据加载到模型保存的全流程，可复现。

```python
# train.py —— 可复现的 MNIST 分类训练脚本
import random
import numpy as np
import tensorflow as tf
from tensorflow import keras
import os

from config import RANDOM_SEED, TRAIN_CONFIG, PATH_CONFIG
from model import create_mnist_model

# ============================================================
# 0. 固定随机种子（可复现的核心）
# ============================================================
random.seed(RANDOM_SEED)
np.random.seed(RANDOM_SEED)
tf.random.set_seed(RANDOM_SEED)

# 确保 GPU 确定性（牺牲少量性能换取可复现性）
# 仅当使用 GPU 时需要以下两行：
# tf.config.experimental.enable_op_determinism()  # TF 2.16+

# ============================================================
# 1. 加载数据 & 预处理
# ============================================================
(x_train, y_train), (x_test, y_test) = keras.datasets.mnist.load_data()

# 归一化到 [0, 1] 区间
x_train = x_train.astype("float32") / 255.0
x_test = x_test.astype("float32") / 255.0

print(f"训练集: {x_train.shape}, 测试集: {x_test.shape}")
print(f"标签分布 - 训练集: {np.bincount(y_train)}")
print(f"标签分布 - 测试集: {np.bincount(y_test)}")

# ============================================================
# 2. 构建模型
# ============================================================
model = create_mnist_model()
model.summary()

# ============================================================
# 3. 编译模型
# ============================================================
model.compile(
    optimizer=keras.optimizers.Adam(TRAIN_CONFIG["learning_rate"]),
    loss="sparse_categorical_crossentropy",
    metrics=["accuracy"],
)

# ============================================================
# 4. 训练
# ============================================================
os.makedirs(PATH_CONFIG["results_dir"], exist_ok=True)

history = model.fit(
    x_train, y_train,
    batch_size=TRAIN_CONFIG["batch_size"],
    epochs=TRAIN_CONFIG["epochs"],
    validation_split=TRAIN_CONFIG["validation_split"],
    verbose=2,
)

# ============================================================
# 5. 评估 & 保存
# ============================================================
test_loss, test_acc = model.evaluate(x_test, y_test, verbose=0)
print(f"\n测试集准确率: {test_acc:.4f} ({test_acc*100:.2f}%)")
print(f"测试集损失:   {test_loss:.4f}")

model.save(PATH_CONFIG["model_path"])
print(f"模型已保存至: {PATH_CONFIG['model_path']}")

# ============================================================
# 6. 输出实验结果摘要（方便团队比对）
# ============================================================
print("\n" + "=" * 50)
print("实验摘要")
print("=" * 50)
print(f"随机种子:    {RANDOM_SEED}")
print(f"训练轮数:    {TRAIN_CONFIG['epochs']}")
print(f"批次大小:    {TRAIN_CONFIG['batch_size']}")
print(f"学习率:      {TRAIN_CONFIG['learning_rate']}")
print(f"最终训练准确率: {history.history['accuracy'][-1]:.4f}")
print(f"最终验证准确率: {history.history['val_accuracy'][-1]:.4f}")
print(f"测试集准确率:   {test_acc:.4f}")
```

### 3.4 常见安装错误速查表

| 错误信息 | 可能原因 | 解决方法 |
|----------|----------|----------|
| `Could not find a version that satisfies the requirement tensorflow` | Python 版本不在 TF 支持范围 | 使用 Python 3.9-3.12 |
| `ImportError: DLL load failed while importing _pywrap_tensorflow_internal` | 缺少 VC++ Redistributable | 安装 vc_redist.x64.exe |
| `Could not load dynamic library 'cudart64_12.dll'` or `libcudnn.so.8` | CUDA/cuDNN 版本不匹配 | 对照官方兼容矩阵安装 |
| `Cannot register 2 metrics with the same name` | 代码中重复注册了同名指标 | 检查 `model.compile(metrics=[...])` 中是否有重复字符串 |
| `OOM when allocating tensor with shape [...]` | GPU 显存不足 | 减小 batch_size 或换 CPU 执行 |

## 4. 项目总结

### 4.1 优点与缺点

| 方面 | 优点 | 缺点 |
|------|------|------|
| **venv 虚拟环境** | Python 内建，零额外依赖，创建速度快 | 不管理 CUDA 等系统级依赖 |
| **固定随机种子** | 实现训练结果完全可复现，方便 A/B 实验 | 可能因固定种子掩盖采样偏差；GPU 上需要额外设置确定性 |
| **配置文件分离** | 超参数集中管理，团队协作修改不冲突 | 配置文件变更需要同步更新文档和测试 |
| **requirements.txt** | pip 原生支持，CI/CD 友好 | 仅锁定顶层依赖，不保证依赖的依赖版本（推荐 pip-tools 或 poetry）|

### 4.2 适用场景

1. **团队新成员 onboarding**：一键搭建环境，即刻跑通第一个实验
2. **CI/CD 自动化训练**：固定种子 + 固定依赖，确保流水线可重复
3. **学术实验对比**：给定相同数据和参数，任何人跑出的结果一致
4. **教学演示**：助教和环境切换零成本

**不适用场景**：
1. 需要在不同 CUDA 版本间切换的 GPU 密集型项目（Docker 更合适）
2. 需要同时管理 R/Julia 等多语言环境（conda 更合适）

### 4.3 注意事项

- **TensorFlow 与 Python 版本严格绑定**：TF 2.16 仅支持 Python 3.9-3.12，每次升级 TF 版本前先查阅 [官方 Build Configurations](https://www.tensorflow.org/install/source#tested_build_configurations)
- **GPU 确定性**：GPU 浮点运算是不完全确定的（浮点加法不满足结合律），`enable_op_determinism()` 会降低性能，仅在需要严格可复现时开启
- **requirements.txt 的局限性**：它不锁定依赖的依赖，生产环境推荐使用 `pip-tools` 的 `pip-compile` 或 Poetry 生成完全锁定的锁文件

### 4.4 常见踩坑经验

1. **坑**：生产服务器 import tensorflow 成功，但模型加载后推理结果全是 NaN。
   **根因**：生产服务器 Python 虽是 3.10，但 NumPy 版本被其他服务升级到了不兼容的 2.x（API 变更导致 TensorFlow 内部数值异常）。
   **解决**：使用 `pip freeze` 或 `pip-tools` 完全锁定所有传递依赖版本，部署时严格安装锁定版本。

2. **坑**：四人团队各自用 Jupyter Notebook 训练，准确率 94%~98% 不等，无法判断谁的改有效。
   **根因**：每人只固定了 `tf.random.set_seed()`，但没有固定 `np.random.seed()` 和 `random.seed()`，数据预处理中的随机行为仍不一致。
   **解决**：模板化三段种子设置，代码 review 时检查 seed 语句。

3. **坑**：相同代码和种子，A 机器（GPU）准确率 97.2%，B 机器（CPU）准确率 97.0%。
   **根因**：GPU 上 `Dropout` 的随机数实现与 CPU 不同，加上浮点累加顺序不同导致微小差异。
   **解决**：这种 <0.5% 的差异是可接受的，但需要在实验报告中注明硬件环境。

### 4.5 思考题

1. 如果项目需要同时支持 TensorFlow 2.16（Python 3.10-3.12）和 TensorFlow 2.8（Python 3.7-3.10），你会怎样设计环境管理方案？请对比 Docker 和 conda 两种思路。

2. 下面的代码能在不同机器上产生相同的准确率吗？如果不能，缺了什么？
   ```python
   tf.random.set_seed(42)
   model = keras.Sequential([...])
   model.compile(...)

   ```
