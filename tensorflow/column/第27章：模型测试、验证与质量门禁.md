# 第27章：模型测试、验证与质量门禁

## 1. 项目背景

算法团队刚上线了一个新版本的商品推荐模型。上线三天后，运维团队发现了一个致命问题——推荐 API 的 P99 延迟从 80ms 飙升到 3 秒，部分请求直接超时。排查后发现：新模型多了一层 Cross-Attention，导致 SavedModel 推理图比旧版本多了 4 个 `tf.while_loop` 算子，在 Serving 的热加载过程中触发了一个罕见的图优化 bug。

如果团队在模型交付前有一套测试套件——(1) 单元测试验证模型的输入输出形状；(2) Serving 推理一致性测试确保 SavedModel 的输出与训练环境一致；(3) 性能回归测试在 CI 中对比新旧模型的推理延迟——这个故障在 PR 阶段就能被拦截。

**痛点放大**：传统软件的 CI/CD 有完善的测试体系（单元测试 / 集成测试 / E2E 测试），但机器学习项目的测试往往"跑通 training 就算通过"。模型测试需要覆盖四个维度：(1) 功能性测试（输出格式对不对）；(2) 数值性测试（精度/指标是否退化）；(3) 鲁棒性测试（异常输入会不会崩）；(4) 性能测试（推理延迟是否超标）。

## 2. 项目设计

**小胖**（不以为然）：模型还需要测试？训练完跑个 `model.evaluate()`，准确率 OK 就上线呗。我们又不是写 Java，不需要单元测试吧？

**大师**：那我问你——你上周更新的模型，输入特征从 50 维改成了 48 维，你上线前验证过 SavedModel 的输入签名变了没有？后端工程师调用 Serving 时传了 50 维的向量，API 是报一个清晰的错误提示，还是直接崩了？

**小胖**（沉默）……后者。后端半夜打电话骂了我一顿。

**大师**：这就是模型测试的第一层——**功能性测试**。它不是测"准确率是不是 95%"，而是测"模型能不能正常工作"。包括：输入 shape 是否正确解析、输出 shape 是否符合预期、错误的输入是否返回清晰的报错而非崩掉。

**技术映射**：模型功能性测试 = 验证模型的基本契约（contract）——输入输出 dtype、shape、合理的值范围。这是模型测试的"单元测试"等价物。

**小白**：那第二层——数值性测试——具体测什么？不就是准确率吗？

**大师**：数值性测试的核心是**回归测试（Regression Test）**——选一批固定的测试数据（Golden Dataset），确保新模型的指标不低于旧模型的某个阈值。比如旧模型在这批数据上的 AUC=0.85，新模型要求 AUC ≥ 0.84（允许 1% 的浮动，但不能退化 3%+）。这个阈值就是"质量门禁（Quality Gate）"。

```
CI Pipeline:
  PR 提交 → 单元测试(功能) → 训练 → 回归测试(指标) → 性能测试(延迟)
           ↓ 失败            ↓ 失败    ↓ 失败
         阻止合并         告警通知   阻止上线
```

**小白**：那推理一致性测试呢？训练环境的 `model.predict()` 和 Serving 的推理结果一定一样吗？

**大师**：不一定！第 9 章我们讲过——Dropout 和 BN 的 training 参数、SavedModel 的图优化、TFLite 的量化——都可能让训练环境和推理环境产生差异。推理一致性测试就是用同一批输入，对比(1) `model.predict()`、(2) Serving REST/gRPC、(3) TFLite 推理——三者输出的差异应 < 1e-5（float32）或 < 0.02（int8 量化）。差异大了，说明某个环节出了转换错误。

**技术映射**：推理一致性测试 = 验证模型在不同运行环境（训练 Python / Serving / TFLite）下输出一致。不一致通常意味着预处理差异、training 参数泄漏或图优化 bug。

**小胖**：鲁棒性测试又是啥？给模型喂一堆垃圾数据？

**大师**：差不多！鲁棒性测试就是"变态测试"——输入全零向量、全 NaN、超大数值（1e10）、空字符串、超长序列——预期是模型不崩（不 segfault、不 OOM），最好能返回合理的兜底结果（如全零输入 → 输出 uniform 概率分布）。这种测试在生产环境中特别重要——用户永远会传入你意料之外的数据。

**步骤一：模型功能测试（单元测试）**

目标：为模型编写 pytest 测试用例，验证输入输出契约。

```python
# test_model_functional.py —— 模型功能测试
import pytest
import tensorflow as tf
from tensorflow import keras
import numpy as np
import os
import tempfile

# --- 被测试的模型 ---
def create_test_model():
    """创建用于测试的推荐模型"""
    user_input = keras.Input(shape=(15,), dtype=tf.float32, name="user_features")
    item_input = keras.Input(shape=(12,), dtype=tf.float32, name="item_features")
    merged = keras.layers.Concatenate()([user_input, item_input])
    x = keras.layers.Dense(32, activation="relu")(merged)
    output = keras.layers.Dense(1, activation="sigmoid", name="score")(x)
    model = keras.Model(inputs=[user_input, item_input], outputs=output)
    model.compile(optimizer="adam", loss="binary_crossentropy")
    return model

# --- 测试用例 ---

def test_model_input_output_shapes():
    """测试 1: 验证模型的输入输出形状"""
    model = create_test_model()
    user = tf.constant(np.random.randn(3, 15).astype(np.float32))
    item = tf.constant(np.random.randn(3, 12).astype(np.float32))

    output = model({"user_features": user, "item_features": item}, training=False)

    assert output.shape == (3, 1), f"期望 (3, 1), 实际 {output.shape}"

def test_model_output_range():
    """测试 2: sigmoid 输出应在 [0, 1] 范围内"""
    model = create_test_model()
    user = tf.constant(np.random.randn(10, 15).astype(np.float32))
    item = tf.constant(np.random.randn(10, 12).astype(np.float32))

    output = model({"user_features": user, "item_features": item}, training=False)

    assert tf.reduce_min(output) >= 0.0, f"最小值 {tf.reduce_min(output)} < 0"
    assert tf.reduce_max(output) <= 1.0, f"最大值 {tf.reduce_max(output)} > 1"

def test_model_invalid_input_handling():
    """测试 3: 输入 shape 不对时应给出清晰的错误"""
    model = create_test_model()
    wrong_shape = tf.constant(np.random.randn(3, 10).astype(np.float32))  # 应为 15 维

    with pytest.raises((ValueError, tf.errors.InvalidArgumentError)):
        model({"user_features": wrong_shape, "item_features": tf.constant(np.random.randn(3, 12))})

def test_model_save_load_roundtrip():
    """测试 4: 模型保存后加载，输出应一致"""
    model = create_test_model()

    tmpdir = tempfile.mkdtemp()
    save_path = os.path.join(tmpdir, "test_model.keras")
    model.save(save_path)

    loaded = keras.models.load_model(save_path)

    user = tf.constant(np.random.randn(4, 15).astype(np.float32))
    item = tf.constant(np.random.randn(4, 12).astype(np.float32))

    orig_output = model({"user_features": user, "item_features": item}, training=False)
    loaded_output = loaded({"user_features": user, "item_features": item}, training=False)

    assert np.allclose(orig_output.numpy(), loaded_output.numpy(), atol=1e-5), \
        "保存加载后输出不一致"

    import shutil
    shutil.rmtree(tmpdir, ignore_errors=True)

def test_gradient_connectivity():
    """测试 5: 所有可训练变量都有梯度"""
    model = create_test_model()
    user = tf.constant(np.random.randn(4, 15).astype(np.float32))
    item = tf.constant(np.random.randn(4, 12).astype(np.float32))

    with tf.GradientTape() as tape:
        output = model({"user_features": user, "item_features": item}, training=True)
        loss = tf.reduce_mean(output)

    grads = tape.gradient(loss, model.trainable_variables)
    none_grads = [v.name for v, g in zip(model.trainable_variables, grads) if g is None]

    assert len(none_grads) == 0, f"以下变量梯度为 None: {none_grads}"

print("所有模型功能测试定义完成。运行: pytest test_model_functional.py -v")
```

**步骤二：回归测试 + 质量门禁**

目标：在 CI 中自动验证新模型的指标不低于基线。

```python
# test_regression.py —— 模型回归测试
import pytest
import numpy as np
import tensorflow as tf
from tensorflow import keras
from sklearn.metrics import roc_auc_score

# 配置质量门禁阈值
QUALITY_GATES = {
    "auc": 0.72,         # AUC 最低要求
    "max_degradation": 0.03,  # 相比基线的最大退化
}

# 固定的 Golden Dataset（应被版本控制）
np.random.seed(42)
GOLDEN_USER = np.random.randn(500, 15).astype(np.float32)
GOLDEN_ITEM = np.random.randn(500, 12).astype(np.float32)
GOLDEN_LABEL = np.random.randint(0, 2, 500).astype(np.float32)

BASELINE_AUC = None  # 从上次训练结果中读取

def test_auc_meets_minimum():
    """验证 AUC ≥ 最低阈值"""
    model = create_test_model()
    # 模拟训练
    model.fit(
        {"user_features": GOLDEN_USER[:400], "item_features": GOLDEN_ITEM[:400]},
        GOLDEN_LABEL[:400], epochs=1, batch_size=32, verbose=0,
    )

    preds = model.predict(
        {"user_features": GOLDEN_USER, "item_features": GOLDEN_ITEM}, verbose=0
    ).flatten()

    auc = roc_auc_score(GOLDEN_LABEL, preds)
    assert auc >= QUALITY_GATES["auc"], f"AUC {auc:.4f} < 最低要求 {QUALITY_GATES['auc']}"
    print(f"  AUC = {auc:.4f} ✓ (≥ {QUALITY_GATES['auc']})")

def test_no_significant_degradation():
    """验证相比基线无显著退化"""
    global BASELINE_AUC
    model = create_test_model()
    model.fit(
        {"user_features": GOLDEN_USER[:400], "item_features": GOLDEN_ITEM[:400]},
        GOLDEN_LABEL[:400], epochs=1, batch_size=32, verbose=0,
    )

    preds = model.predict(
        {"user_features": GOLDEN_USER, "item_features": GOLDEN_ITEM}, verbose=0
    ).flatten()
    current_auc = roc_auc_score(GOLDEN_LABEL, preds)

    if BASELINE_AUC is not None:
        degradation = BASELINE_AUC - current_auc
        assert degradation <= QUALITY_GATES["max_degradation"], \
            f"AUC 退化 {degradation:.4f} > 最大允许 {QUALITY_GATES['max_degradation']}"
        print(f"  AUC 退化 = {degradation:.4f} ✓ (≤ {QUALITY_GATES['max_degradation']})")

    BASELINE_AUC = current_auc

from test_model_functional import create_test_model
print("回归测试定义完成。运行: pytest test_regression.py -v")
```

**步骤三：推理一致性测试**

目标：确保训练环境、SavedModel、TFLite 三者的输出一致。

```python
import tensorflow as tf
from tensorflow import keras
import numpy as np
import os
import tempfile

def test_inference_consistency():
    """验证 Keras predict / SavedModel / TFLite 三种推理输出一致"""
    model = create_test_model()

    # 固定测试数据
    user_test = tf.constant(np.random.randn(10, 15).astype(np.float32))
    item_test = tf.constant(np.random.randn(10, 12).astype(np.float32))
    test_input = {"user_features": user_test, "item_features": item_test}

    # 1. Keras 基准
    keras_output = model.predict(test_input, verbose=0)

    # 2. SavedModel
    tmpdir = tempfile.mkdtemp()
    sm_path = os.path.join(tmpdir, "saved_model")
    model.save(sm_path)
    loaded_sm = tf.saved_model.load(sm_path)
    sm_fn = loaded_sm.signatures["serving_default"]
    sm_output = sm_fn(user_features=user_test, item_features=item_test)
    sm_output = list(sm_output.values())[0].numpy()

    # 3. TFLite
    converter = tf.lite.TFLiteConverter.from_saved_model(sm_path)
    tflite_model = converter.convert()
    interpreter = tf.lite.Interpreter(model_content=tflite_model)
    interpreter.allocate_tensors()

    input_details = interpreter.get_input_details()
    interpreter.set_tensor(input_details[0]["index"], user_test.numpy())
    interpreter.set_tensor(input_details[1]["index"], item_test.numpy())
    interpreter.invoke()
    tflite_output = interpreter.get_tensor(interpreter.get_output_details()[0]["index"])

    # 一致性检查
    sm_diff = np.max(np.abs(keras_output - sm_output))
    tflite_diff = np.max(np.abs(keras_output - tflite_output))

    print(f"Keras vs SavedModel 最大差异: {sm_diff:.2e}")
    print(f"Keras vs TFLite     最大差异: {tflite_diff:.2e}")

    assert sm_diff < 1e-5, f"SavedModel 输出不一致: diff={sm_diff:.2e}"
    assert tflite_diff < 1e-5, f"TFLite 输出不一致: diff={tflite_diff:.2e}"

    import shutil
    shutil.rmtree(tmpdir, ignore_errors=True)

from test_model_functional import create_test_model
test_inference_consistency()
print("\n推理一致性测试通过 ✅")
```

**步骤四：鲁棒性测试**

目标：验证模型在异常输入下不崩溃。

```python
import tensorflow as tf
import numpy as np
import pytest

def test_robustness_edge_cases():
    """鲁棒性测试：异常输入不崩溃"""
    model = create_test_model()

    edge_cases = {
        "全零输入": (np.zeros((3, 15), dtype=np.float32),
                    np.zeros((3, 12), dtype=np.float32)),
        "全一输入": (np.ones((3, 15), dtype=np.float32),
                    np.ones((3, 12), dtype=np.float32)),
        "负值输入": (-np.ones((3, 15), dtype=np.float32),
                    -np.ones((3, 12), dtype=np.float32)),
        "大数值输入": (np.ones((3, 15), dtype=np.float32) * 1e3,
                      np.ones((3, 12), dtype=np.float32) * 1e3),
        "极小值输入": (np.ones((3, 15), dtype=np.float32) * 1e-6,
                      np.ones((3, 12), dtype=np.float32) * 1e-6),
    }

    for name, (user_inp, item_inp) in edge_cases.items():
        try:
            output = model(
                {"user_features": tf.constant(user_inp),
                 "item_features": tf.constant(item_inp)},
                training=False,
            )
            # 检查输出没有 NaN 或 Inf
            assert not tf.reduce_any(tf.math.is_nan(output)), f"{name}: 输出包含 NaN"
            assert not tf.reduce_any(tf.math.is_inf(output)), f"{name}: 输出包含 Inf"
            print(f"  {name}: OK (output range: [{tf.reduce_min(output):.4f}, {tf.reduce_max(output):.4f}])")
        except Exception as e:
            pytest.fail(f"{name}: 模型崩溃: {e}")

from test_model_functional import create_test_model
test_robustness_edge_cases()
print("\n鲁棒性测试通过 ✅")
```

## 4. 项目总结

### 4.1 测试类型速查

| 测试类型 | 测什么 | 工具 | CI 阶段 | 失败动作 |
|---------|--------|------|---------|---------|
| **功能测试** | shape/dtype/值范围/save-load | pytest + tf | PR 阶段 | 阻止合并 |
| **回归测试** | AUC/accuracy ≥ 历史基准 | pytest + sklearn | PR 阶段 | 告警通知 |
| **推理一致性** | Keras ≈ Serving ≈ TFLite | 自定义脚本 | 部署前 | 阻止部署 |
| **鲁棒性测试** | 异常输入不崩溃 | pytest | PR 阶段 | 阻止合并 |
| **性能回归** | 推理延迟 ≤ 基线 × 1.2 | 压测脚本 | 部署前 | 阻止部署 |
| **数据质量** | 缺失率/分布漂移 | Great Expectations | 训练前 | 阻止训练 |

### 4.2 适用场景

1. **PR 提交流水线**：功能测试 + 回归测试（5 分钟内完成，快速反馈）
2. **Nightly 全量测试**：功能 + 回归 + 鲁棒性 + 一致性（30-60 分钟）
3. **发布前门禁**：全量测试 + 性能回归 + 人工审批
4. **Golden Dataset 管理**：选取 500-2000 条代表性数据作为固定测试集，与代码一起版本控制

**不适用场景**：
1. 极其早期的实验探索阶段（模型还在快速迭代，阈值定的过紧会拖慢实验节奏）
2. 推理环境无法在 CI 中复现（如特殊硬件 TPU）——此时一致性测试可在上线后的 smoke test 中进行

### 4.3 注意事项

- **Golden Dataset 必须固定且独立**：不能从训练集中采样，也不能每次重新随机生成——确保回归测试结果具有可比性
- **质量阈值的设定**：第一次上线前是"手动批准"的，上线后根据稳定期的指标设一个合理的下限（如 `best_AUC - 0.02`）
- **性能回归测试的环境一致性**：CI 的 GPU 型号、CUDA 版本应与生产环境一致，否则延迟数据没有参考价值
- **测试数据的隐私问题**：Golden Dataset 中的数据不应包含真实用户的原始信息（应经过脱敏处理）

### 4.4 常见踩坑经验

1. **坑**：CI 中的回归测试频繁误报——每次跑 AUC 都差 2%+，但实际模型没问题。
   **根因**：Golden Dataset 只有 50 条，样本量太小导致 AUC 波动大。
   **解决**：Golden Dataset 至少 500 条；如果需要评估 Recall@K（更稳定），至少 2000 条。

2. **坑**：推理一致性测试中，SavedModel 和 Keras 输出差异 > 1e-3。
   **根因**：模型中有 Dropout 且 SavedModel 保存时没有显式设 `training=False`（见第 9 章）。
   **解决**：保存前确保模型处于 eval 模式；保存 Signature 函数中显式传入 `training=False`。

3. **坑**：鲁棒性测试中，NaN 输入导致测试框架本身崩溃（而非被测模型返回错误）。
   **根因**：pytest 的 `assert` 直接抛异常，测试报告不清晰。
   **解决**：用 `try/except` 包裹异常输入推理，捕获 error 并输出清晰的错误信息后再 `pytest.fail()`。

### 4.5 思考题

1. 你需要在 CI 中加入"模型推理延迟回归测试"。你的 Serving 容器在 CI 中启动需要 ~30 秒，测试只有 ~5 秒。你会如何在 CI 中高效地设计这个测试，避免每次 PR 都等待 30 秒的容器启动？（提示：考虑 Docker layer caching 和预热）

2. 假设你的 Golden Dataset 有 1000 条数据，但 A/B 测试发现离线 AUC 提升 2% 并不能保证线上 CTR 提升。这是为什么？Golden Dataset 的哪些局限性导致了这个问题？如何改善？

### 4.6 推广计划提示
