# 第13章：时间序列预测实战

## 1. 项目背景

某新闻门户网站的运维团队需要提前做容量规划——根据历史访问量预测未来 7 天的流量，以便提前扩容或缩容服务器，避免流量洪峰导致服务崩溃或低谷期资源浪费。

数据分析师小杨拿到了过去两年（730 天）的每日 PV（页面浏览量）数据。她一开始用第 8 章学到的 MLP 回归模型——把日期编码为"星期几""月份""是否节假日"等特征，用 Dense 层预测 PV。模型在训练集上 R²=0.95，看起来很好。但一到测试集（最近 90 天），R² 掉到了 -0.3——比直接用均值预测还差！

小杨排查后发现：她对数据做了"随机切分"——用 `train_test_split(random_state=42)` 把 730 天的数据随机打乱后分成了训练集和测试集。问题在于，时间序列有自相关性——今天的流量和昨天高度相关。随机切分导致训练集里混入了"测试集前一天"的数据（也就是未来信息），模型在训练时就"偷看"到了答案。这种现象叫**数据泄漏（Data Leakage）**，是时间序列预测中最常见也最致命的错误。

**痛点放大**：时间序列预测与普通回归有三个根本区别——(1) 数据必须按时间顺序切分，不能用随机 shuffle；(2) 模型需要"记住"历史模式——用过去 N 天的数据预测未来 M 天（滑动窗口），而非静态特征一步到位；(3) 归一化方式必须防泄漏——不能把未来的均值/方差用来归一化过去的数据。

## 2. 项目设计

**小胖**（看着一张歪歪扭扭的折线图）：不就是一个数字预测下一个数字嘛！跟"今天 30 度，明天大概也 30 度"一样简单，用昨天的值当成今天的预测不就完了？

**大师**：那你看看这张图——国庆假期前一天 PV 是 500 万，国庆当天跳水到 180 万，节后第一天又猛涨到 700 万。"昨天 500 万 → 今天 180 万"这个规律你有把握说"每次从 500 万跌到 180 万之后都会怎样"吗？而且国庆这种突发事件一年就一次，你拿什么数据训练它？

**小胖**：……那怎么搞？难道要我把过去所有节假日的数据都找出来？

**大师**：对，但这只是特征工程的一部分。时间序列预测的核心是**滑动窗口**——你不是输入"昨天的一个数字"，而是输入"过去 14 天的 14 个数字"，让模型自己去发现里面有没有周期性、有没有趋势、有没有突发事件后的反弹模式。

**技术映射**：滑动窗口将原始时间序列转换为监督学习样本——用 `[X_{t-n}, ..., X_{t-1}, X_t]` 预测 `[X_{t+1}, ..., X_{t+m}]`。n 是回顾窗口（lookback），m 是预测步长（forecast horizon）。

**小白**（在本子上画了一个窗口滑动的示意图）：那窗口大小怎么定？用过去 7 天预测未来 1 天，还是过去 30 天预测未来 7 天？

**大师**：这需要结合业务理解和实验。经验法则：
- **回看窗口（lookback）**：至少覆盖一个完整的周期。如果你的数据有"周周期"（周末流量低），回看窗口至少 7 天；有"月周期"至少 30 天。但窗口太大模型难训练——建议取最大周期 × 2 或 ×3。
- **预测步长（forecast）**：越短越准。预测未来 1 天比预测未来 7 天容易得多。如果业务只需要 7 天预测，你有两种方案——(1) 训练一个模型直接输出 7 个值（多步预测），(2) 训练一个"单步预测"模型，预测出 1 天后把它当作已知再预测下 1 天（递归预测）。递归预测误差会逐步累积，但实现简单。

**技术映射**：多步预测的监督学习构造——用 `X[t-30:t]` 预测 `y[t+1:t+8]`（8 维输出，代表未来 7 天+1）。

**小白**：那数据切分和归一化的"防泄漏"具体怎么操作？

**大师**：防泄漏有铁三条：

1. **时间顺序切分**：训练集 = 最早的 80% 数据，验证集 = 接下来 10%，测试集 = 最近 10%。绝对不能用 `train_test_split` 打乱！
2. **归一化只 fit 训练集**：`scaler.fit(train_data)` 之后，用同一个 scaler 去 transform 验证集和测试集。不能把所有数据混在一起 fit。
3. **窗口构造在切分之后**：先用时间切好数据集，再对每个子集独立构造窗口。不能在切分前构造窗口——否则一个窗口可能跨越训练集和测试集的边界。

```python
# 正确的时间切分
train = data[:int(0.7 * len(data))]
val   = data[int(0.7 * len(data)):int(0.85 * len(data))]
test  = data[int(0.85 * len(data)):]

scaler = StandardScaler().fit(train)  # 只 fit 训练集
train_norm = scaler.transform(train)
val_norm   = scaler.transform(val)
test_norm  = scaler.transform(test)
```

**技术映射**：时间序列的数据泄漏来源有三——(1) 随机切分让未来信息混入训练集；(2) 归一化用全量数据计算统计量；(3) 窗口构造前未先切分导致窗口跨边界。

**小胖**：那 LSTM 在这里面是干嘛的？不就能记住"过去"吗？

**大师**：对！LSTM 天然适合时间序列——它的门控机制可以自动学习"7 天前的周末效应""30 天前的月周期"，不需要你手工提取"星期几"这种特征。但注意——LSTM 不是万能的，如果数据只有简单的趋势+噪声，一个 Dense 层（拿了前 7 天数据拼成的向量）就能搞定，LSTM 反而容易过拟合。Conv1D 也是一个强有力的竞争者——用一维卷积在时间轴上滑动，可以高效捕获局部时间模式。

### 3.2 分步实现

**步骤一：滑动窗口数据构造**

目标：理解如何把原始时间序列转化为监督学习的 (X, y) 样本。

```python
import numpy as np
import pandas as pd
import tensorflow as tf
from tensorflow import keras
from sklearn.preprocessing import StandardScaler

# 1. 模拟 730 天的 PV 数据（含趋势+周期+节假日+噪声）
np.random.seed(42)
days = 730
t = np.arange(days)

# 趋势：缓慢增长
trend = 100 + t * 0.05
# 周周期：周末流量低 20%
weekly = 15 * np.sin(2 * np.pi * t / 7)
# 年周期：年中高，年初年末低
yearly = 30 * np.sin(2 * np.pi * t / 365)
# 节假日效应（国庆、春节等）
holiday = np.zeros(days)
holiday[270:277] -= 40     # 模拟国庆 7 天流量下跌
holiday[30:37]   -= 35     # 模拟春节
# 噪声
noise = np.random.randn(days) * 5

pv = trend + weekly + yearly + holiday + noise
pv = np.maximum(pv, 10)  # 最低量不低于 10

print(f"PV 范围: [{pv.min():.1f}, {pv.max():.1f}]")
print(f"前 7 天 PV: {pv[:7].round(1).tolist()}")

# 2. 时间顺序切分
train_size = int(0.7 * days)
val_size = int(0.15 * days)
test_size = days - train_size - val_size

train_pv = pv[:train_size]
val_pv   = pv[train_size:train_size+val_size]
test_pv  = pv[train_size+val_size:]

print(f"训练集: {len(train_pv)} 天 | 验证集: {len(val_pv)} 天 | 测试集: {len(test_pv)} 天")

# 3. 归一化（只在训练集上 fit）
scaler = StandardScaler()
train_norm = scaler.fit_transform(train_pv.reshape(-1, 1)).flatten()
val_norm   = scaler.transform(val_pv.reshape(-1, 1)).flatten()
test_norm  = scaler.transform(test_pv.reshape(-1, 1)).flatten()

# 4. 滑动窗口函数
def create_sequences(data, lookback=14, forecast=7):
    """将 1D 数组转为 (X, y) 样本
    X 形状: (n_samples, lookback, 1)  —— 过去 lookback 天
    y 形状: (n_samples, forecast)      —— 未来 forecast 天
    """
    X, y = [], []
    for i in range(len(data) - lookback - forecast + 1):
        X.append(data[i:i+lookback])
        y.append(data[i+lookback:i+lookback+forecast])
    return np.array(X), np.array(y)

lookback = 14  # 用过去 14 天
forecast = 7   # 预测未来 7 天

X_train, y_train = create_sequences(train_norm, lookback, forecast)
X_val,   y_val   = create_sequences(val_norm, lookback, forecast)
X_test,  y_test  = create_sequences(test_norm, lookback, forecast)

# reshape: (样本数, lookback, 1)
X_train = X_train[..., np.newaxis]
X_val   = X_val[..., np.newaxis]
X_test  = X_test[..., np.newaxis]

print(f"\nX_train shape: {X_train.shape}, y_train shape: {y_train.shape}")
print(f"X_val shape:   {X_val.shape}, y_val shape:   {y_val.shape}")
print(f"X_test shape:  {X_test.shape}, y_test shape:  {y_test.shape}")
print(f"预测目标: 未来 {forecast} 天的流量")
```

```
前 7 天 PV: [125.3, 119.8, 121.5, 132.1, 138.2, 132.7, 119.4]
训练集: 511 天 | 验证集: 109 天 | 测试集: 110 天

X_train shape: (491, 14, 1), y_train shape: (491, 7)
X_val shape:   (89, 14, 1), y_val shape:   (89, 7)
X_test shape:  (90, 14, 1), y_test shape:  (90, 7)
```

**步骤二：LSTM vs Conv1D vs Dense 对比**

目标：比较三种模型在时间序列预测上的表现。

```python
from tensorflow import keras

def build_ts_model(model_type, lookback=14, forecast=7):
    inp = keras.Input(shape=(lookback, 1))

    if model_type == "dense":
        x = keras.layers.Flatten()(inp)
        x = keras.layers.Dense(64, activation="relu")(x)
        x = keras.layers.Dense(32, activation="relu")(x)

    elif model_type == "conv1d":
        x = keras.layers.Conv1D(32, 3, activation="relu", padding="same")(inp)
        x = keras.layers.Conv1D(32, 3, activation="relu", padding="same")(x)
        x = keras.layers.GlobalAveragePooling1D()(x)
        x = keras.layers.Dense(32, activation="relu")(x)

    elif model_type == "lstm":
        x = keras.layers.LSTM(64, return_sequences=True)(inp)
        x = keras.layers.LSTM(32)(x)

    else:  # bilstm
        x = keras.layers.Bidirectional(
            keras.layers.LSTM(64, return_sequences=True))(inp)
        x = keras.layers.Bidirectional(
            keras.layers.LSTM(32))(x)

    x = keras.layers.Dropout(0.2)(x)
    out = keras.layers.Dense(forecast)(x)

    model = keras.Model(inp, out)
    model.compile(optimizer=keras.optimizers.Adam(1e-3), loss="mse", metrics=["mae"])
    return model

results = {}
for name in ["dense", "conv1d", "lstm", "bilstm"]:
    tf.random.set_seed(42)
    model = build_ts_model(name, lookback, forecast)
    history = model.fit(
        X_train, y_train,
        epochs=30, batch_size=32,
        validation_data=(X_val, y_val),
        callbacks=[keras.callbacks.EarlyStopping(patience=5, restore_best_weights=True)],
        verbose=0,
    )
    val_mae = model.evaluate(X_test, y_test, verbose=0)[1]  # MAE（归一化后的）
    results[name] = {"test_mae_norm": val_mae, "params": model.count_params()}

# 把归一化后的 MAE 还原到原始尺度
from sklearn.metrics import mean_absolute_error
print(f"\n{'Model':<10} {'Norm MAE':>10} {'Params':>10}")
print("-" * 32)
for name, r in results.items():
    print(f"{name:<10} {r['test_mae_norm']:10.4f} {r['params']:10,}")
```

**步骤四：数据泄漏检测实验**

目标：通过对比正确切分 vs 随机切分的测试集效果，直观展示数据泄漏的危害。

```python
from sklearn.model_selection import train_test_split

# === 正确方式：时间顺序切分 ===
def time_split_eval():
    train = pv[:train_size]
    test  = pv[-test_size:]
    scaler = StandardScaler().fit(train.reshape(-1,1))
    train_n = scaler.transform(train.reshape(-1,1)).flatten()
    test_n  = scaler.transform(test.reshape(-1,1)).flatten()
    X_tr, y_tr = create_sequences(train_n, lookback, 1)
    X_te, y_te = create_sequences(test_n, lookback, 1)
    X_tr = X_tr[..., np.newaxis]; X_te = X_te[..., np.newaxis]
    model = build_ts_model("lstm", lookback, 1)
    model.fit(X_tr, y_tr, epochs=10, batch_size=32, verbose=0)
    return model.evaluate(X_te, y_te, verbose=0)[1]  # test mae

# === 错误方式：随机切分（数据泄漏） ===
def random_split_eval():
    X_all, y_all = create_sequences(
        StandardScaler().fit_transform(pv.reshape(-1,1)).flatten(), lookback, 1
    )
    X_all = X_all[..., np.newaxis]
    X_tr, X_te, y_tr, y_te = train_test_split(X_all, y_all, test_size=0.15,
                                               random_state=42)  # ❌ 随机切分
    model = build_ts_model("lstm", lookback, 1)
    model.fit(X_tr, y_tr, epochs=10, batch_size=32, verbose=0)
    return model.evaluate(X_te, y_te, verbose=0)[1]

print("=== 数据泄漏对比实验 ===")
for name, func in [("时间顺序切分 ✅", time_split_eval), ("随机切分 ❌", random_split_eval)]:
    mae_vals = []
    for _ in range(3):
        tf.random.set_seed(42)
        mae_vals.append(func())
    print(f"  {name}: MAE = {np.mean(mae_vals):.4f} ± {np.std(mae_vals):.4f}")
```

运行输出：
```
=== 数据泄漏对比实验 ===
  时间顺序切分 ✅: MAE = 0.5234 ± 0.0123
  随机切分 ❌:     MAE = 0.0832 ± 0.0056
```

> 随机切分的 MAE 虚假地降低了 **6 倍以上**——这是因为模型在训练时已经"看过"测试集前后的数据，典型的数据泄漏。

### 3.3 时间序列防泄漏 Checklist

| # | 检查项 | 正确做法 |
|---|--------|----------|
| 1 | 数据切分方式 | 按时间顺序切分，训练集 < 验证集 < 测试集（时间上） |
| 2 | 归一化 | `scaler.fit(train)` → `transform(val)` → `transform(test)` |
| 3 | 窗口顺序 | 先切分数据，再对每段独立构造窗口 |
| 4 | 特征工程 | 不能使用未来的数据计算特征（如"未来 7 天均值"） |
| 5 | 交叉验证 | 使用时序交叉验证（TimeSeriesSplit），不能用 K-Fold |

## 4. 项目总结

### 4.1 时间序列模型对比

| 模型 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Dense (MLP)** | 简单、训练快、可解释 | 无法利用时序结构 | 短窗口 + 明显 pattern |
| **Conv1D** | 局部模式感知、并行训练 | 缺乏长期记忆 | 带周期性的趋势预测 |
| **LSTM** | 天然适合序列、长期记忆 | 训练慢、对超参数敏感 | 复杂时序（多周期叠加） |
| **BiLSTM** | 同时利用前向+后向信息 | 推理时不可用后向（非因果） | 离线标注/分类场景 |

### 4.2 适用场景

1. **流量/负载预测**：网站 PV、API QPS、服务器 CPU 预测，用于弹性伸缩
2. **销售/库存预测**：商品销量预测未来 N 天需求，驱动补货
3. **金融时间序列**：股价/汇率预测（注意：金融数据的信噪比极低）
4. **IoT 传感器预测**：温度、湿度、振动数据趋势监测
5. **异常检测**：预测值与真实值的残差超过阈值 → 异常告警

**不适用场景**：
1. 没有明显时间依赖的数据（纯随机游走，任何模型都无用）
2. 需要精确逐点预测的混沌系统（天气、地震等复杂物理过程，单变量时序不够）

### 4.3 注意事项

- **归一化必须按时间切分 fit**：如果把全量数据一起归一化，均值可能包含了未来的信息
- **LSTM 的 `stateful` 模式慎用**：stateful 让 LSTM 跨 batch 保持状态，适合单个长序列的分段训练，但 shuffle 后会导致状态错乱
- **多变量预测场景**：如果除了 PV 还有"星期几""是否节假日"等特征，需在滑动窗口中同时构造多维特征
- **预测区间 vs 点预测**：业务决策往往需要置信区间（"明天 PV 在 100-150 万之间，概率 95%"），而非一个点值。可考虑用分位数回归预测上下界

### 4.4 常见踩坑经验

1. **坑**：测试集 loss 远低于训练集，以为是"天生泛化好"。
   **根因**：数据泄漏——测试集的时间段刚好在训练集的"影子"里（如春节+节后一周分别分入了训练+测试，节前节后的关联性被模型偷学了）。
   **解决**：确保测试集在时间上完全在训练集之后，且中间留出足够的"空白期"（至少等于 lookback 窗口）。

2. **坑**：LSTM 预测的结果恒为一条水平线。
   **根因**：学习率太高或初始化不当，LSTM 的 forget gate 一开始就完全关闭，隐藏状态几乎不变，输出退化成了"均值预测"。
   **解决**：降低 lr 到 1e-4，对 LSTM 的 forget gate bias 初始化为 1（`LSTM(64, unit_forget_bias=True)`）。

3. **坑**：多步预测（7 天）中，第 1 天很准，第 7 天误差暴增 3 倍。
   **根因**：直接多步预测模型把 7 天的输出当做 7 个独立的回归目标，缺乏"步间依赖"。
   **解决**：使用 Seq2Seq 架构（encoder=历史窗口，decoder=逐天自回归预测），或使用分位数损失/自定义损失给远步更高权重。

### 4.5 思考题

1. 假设你要预测未来 7 天的流量，现有两种方案：(A) 训练一个模型直接输出 7 个值；(B) 训练一个单步预测模型，预测出 Day 1 后把它当作已知去预测 Day 2，以此迭代。请分析两种方案各自的优缺点，并设计实验对比。

2. 时间序列预测中，如果测试集 RMSE 为 10，但业务允许的误差是 ±5%，请问这个模型能否上线？如果不确定，还需要哪些信息来判断？
