# 第7章：损失函数、优化器与评估指标

## 1. 项目背景

风控团队需要上线一个"交易欺诈检测"模型——用户每发起一笔支付，模型需要判断这笔交易是否可疑。业务方给了 50 万条历史交易数据，阿杰打开一看：正常交易 49.2 万条，欺诈交易只有 8000 条——正负样本比例约 1:60。

阿杰用标准交叉熵损失函数训练了一个二分类模型，accuracy 高达 98.4%。他兴奋地拿给业务方看，业务方却说："你这个模型我们不敢用——我们测了最近 100 笔欺诈交易，你的模型只识别出了 3 笔。"阿杰仔细一看：模型几乎把所有交易都预测为"正常"（负类），因为"正常"样本占了 98.4%——模型只要"偷懒"全猜"正常"，准确率就是 98.4%！

于是阿杰换了策略：先给正负样本加权，让每个欺诈样本的 loss 权重是正常样本的 60 倍。模型开始关注少数类了，但新的问题出现了——召回率（Recall）上来了，但精确率（Precision）跌到了 8%——模型疯狂"喊狼来了"，把大量正常交易误报为欺诈。

最后，阿杰发现问题的根源在于：他还在用 Accuracy 作为评估指标。对于极不均衡的分类问题，Accuracy 是完全失真的——正确的指标应该是 AUC、F1-Score，并且需要使用面向不均衡样本设计的损失函数（如 Focal Loss 或加权交叉熵）。

**痛点放大**：损失函数决定了模型"优化什么"、优化器决定了模型"怎么优化"、评估指标决定了"怎么判断好坏"——这三者构成训练的"三件套"。选错了损失函数（如不均衡用交叉熵），模型学到的是偏置；选错了优化器（如稀疏梯度用 SGD），模型永远不收敛；选错了评估指标（如不均衡用 Accuracy），你可能一直觉得模型很好，上线才发现完全不能用。

## 2. 项目设计

**小胖**（啃着鸡腿）：损失函数、优化器、评估指标——这三个词我都听烂了。不就是一个算错误多大，一个纠正错误，一个打完分看成绩吗？有啥好选的？直接用默认的不就完了？

**大师**：好，那我来考考你。一个高中生考试，数学满分 150，英语满分 150。你想让他总分最高——这是你的"损失函数"。现在有两个老师帮他复习：A 老师让他"先把数学错题全做对"，B 老师让他"每天背 200 个单词"。同样的时间，哪个效果更好？这取决于他现在哪科弱——这就是"优化器"的选择。最后你用总分排名看他进步没——这是"评估指标"。

**小胖**：哦……但我怎么知道他现在哪科弱？这不就跟模型不知道哪些样本难分类一样？

**大师**：对！所以不同的损失函数就是不同的"评分规则"。普通交叉熵就跟"每道题错了扣一样的分"一样——欺诈交易和正常交易"错一题扣 1 分"。但如果有 60 个正常交易和 1 个欺诈交易，模型发现"全猜正常"只扣 1/61 的分，分数很高——这就是你 accuracy 98.4% 但欺诈召回 3% 的原因！

**技术映射**：交叉熵对每个样本的惩罚权重相等，在类别不均衡时模型偏向预测多数类。加权交叉熵、Focal Loss 等变体通过调整不同样本的 loss 权重来解决这个问题。

**小白**（快速记笔记）：那 Focal Loss 到底比加权交叉熵好在哪？不都是让模型多关注难分类的样本吗？

**大师**：区别在于"自动"还是"手动"。加权交叉熵是你手动说"欺诈样本 loss ×60"。但问题是——同一个欺诈样本，模型刚开始训练时完全认不出来（应该给高权重），但训练到后面已经能认出来了（应该降权重让模型去关注别的难样本）。Focal Loss 里的 `(1-pt)^γ` 因子自动实现了这个——预测概率越高（pt 越大），权重自动衰减。

```python
# 交叉熵: 只看"猜得对不对"
loss = -log(pt)

# 加权交叉熵: "欺诈猜错罚更重"
loss = -alpha * log(pt)

# Focal Loss: "错得越离谱罚得越重，猜对了就不用再罚"
loss = -alpha * (1 - pt)^gamma * log(pt)
#                     ^^^^^^^^^^^^^
#                     动态权重因子
```

**技术映射**：Focal Loss 通过 `(1-pt)^γ` 动态调节样本权重——易分类样本（pt 高）权重自动降低，难分类样本（pt 低）权重保持高位，实现"隐式样本挖掘"。

**小白**：那优化器呢？SGD、Adam、AdamW 这些有什么区别？我看大家都用 Adam，是不是无脑选 Adam 就行？

**大师**（笑着摇头）：SGD 就像开车时恒定踩油门——始终一个速度。如果路很平（损失曲面光滑），SGD 最稳，大概率找到最优解。Adam 就像自适应巡航——上坡自动加油，下坡自动收油，转弯减速。它在各种地形上都跑得快，但不一定找得到最精确的目的地。AdamW 是在 Adam 的基础上加了"限速牌"——防止开太快（权重衰减），近年来大模型训练几乎标配。

**小胖**：那我能不能这样理解——小模型用 Adam 快速收敛，大模型用 AdamW 防止过拟合，最终精调用 SGD 追求最优解？

**大师**：你总结得非常好！这恰恰是业界常用的"三阶段策略"。

**技术映射**：SGD（含 Momentum）适合光滑凸优化问题、小数据集精调；Adam 适合非平稳目标、稀疏梯度、高维参数空间；AdamW 解耦了权重衰减与自适应学习率，是大模型训练首选。

**小白**：最后问指标——Precision、Recall、AUC 到底分别衡量什么？为什么不均衡分类看 Accuracy 没用？

**大师**：来，用最简单的方式记住：

```
               预测=正常     预测=欺诈
实际=正常      TN (真阴性)    FP (假阳性, 误报)
实际=欺诈      FN (假阴性, 漏报) TP (真阳性, 命中)

Accuracy  = (TN+TP) / 全部        ← 不均衡时会被 TN 主导
Precision = TP / (TP+FP)          ← "我说是欺诈的里面，有多少真是欺诈？"
Recall    = TP / (TP+FN)          ← "真正的欺诈里面，我抓到了多少？"
AUC       = 正样本排在负样本前面的概率 ← 不受分类阈值影响，综合评估排序能力
```

**大师**：欺诈检测场景中，漏报一个欺诈交易可能损失几万元，误报一个正常交易只是多一次人工审核——所以 Recall 比 Precision 更重要。AUC 不依赖具体的分类阈值（0.5），是模型"排序能力"的综合体现。

**步骤一：自定义加权交叉熵 + 理解 Focal Loss 效果**

目标：在极不均衡数据集上对比标准交叉熵、加权交叉熵、Focal Loss 的效果。

```python
import tensorflow as tf
from tensorflow import keras
import numpy as np
from sklearn.metrics import classification_report, roc_auc_score

# 1. 生成极不均衡的二分类数据
np.random.seed(42)
n_pos, n_neg = 2000, 50000  # 正样本 2000, 负样本 50000
n_total = n_pos + n_neg

# 正样本：在特征空间偏右上
X_pos = np.random.randn(n_pos, 10) * 1.2 + np.array([0.8]*10)
# 负样本：在特征空间偏左下
X_neg = np.random.randn(n_neg, 10) * 1.2 - np.array([0.2]*10)

X = np.vstack([X_pos, X_neg]).astype(np.float32)
y = np.hstack([np.ones(n_pos), np.zeros(n_neg)]).reshape(-1, 1).astype(np.float32)

# shuffle
idx = np.random.permutation(n_total)
X, y = X[idx], y[idx]

print(f"总样本: {len(y)}, 正样本率: {y.mean():.2%}")

# 2. 定义三种损失函数
def weighted_bce(class_weight_ratio=25.0):
    """加权二元交叉熵"""
    def loss_fn(y_true, y_pred):
        bce = keras.losses.binary_crossentropy(y_true, y_pred)
        # 正样本权重 = class_weight_ratio, 负样本权重 = 1
        weights = y_true * (class_weight_ratio - 1.0) + 1.0
        return tf.reduce_mean(bce * weights)
    return loss_fn

def focal_loss(gamma=2.0, alpha=0.25):
    """Focal Loss for binary classification"""
    def loss_fn(y_true, y_pred):
        eps = 1e-7
        y_pred = tf.clip_by_value(y_pred, eps, 1 - eps)
        pt = tf.where(tf.equal(y_true, 1), y_pred, 1 - y_pred)
        focal_weight = tf.pow(1 - pt, gamma)
        ce = -tf.math.log(pt)
        loss = alpha * focal_weight * ce
        return tf.reduce_mean(loss)
    return loss_fn

# 3. 统一模型构建函数
def build_model():
    return keras.Sequential([
        keras.layers.Dense(64, activation="relu", input_shape=(10,)),
        keras.layers.Dropout(0.3),
        keras.layers.Dense(32, activation="relu"),
        keras.layers.Dropout(0.3),
        keras.layers.Dense(1, activation="sigmoid"),
    ])

# 4. 训练三种 loss 的模型
results = {}
for name, loss in [
    ("Standard BCE", "binary_crossentropy"),
    ("Weighted BCE (25x)", weighted_bce(25.0)),
    ("Focal Loss (γ=2)", focal_loss(gamma=2.0, alpha=0.25)),
]:
    tf.random.set_seed(42)
    model = build_model()
    model.compile(
        optimizer=keras.optimizers.Adam(1e-3),
        loss=loss,
        metrics=[keras.metrics.AUC(name="auc")],
    )
    model.fit(X, y, epochs=5, batch_size=512, validation_split=0.2, verbose=0)

    y_pred = model.predict(X, verbose=0).flatten()
    auc = roc_auc_score(y, y_pred)
    # 取最佳 F1 的阈值
    from sklearn.metrics import f1_score
    best_f1, best_thresh = 0, 0.5
    for t in np.arange(0.1, 0.9, 0.02):
        f1 = f1_score(y, (y_pred >= t).astype(int))
        if f1 > best_f1:
            best_f1, best_thresh = f1, t

    results[name] = {"auc": auc, "best_f1": best_f1, "best_thresh": best_thresh}

print("\n" + "="*60)
print(f"{'Loss Function':<22} {'AUC':>8} {'Best F1':>8} {'Best Thr':>8}")
print("="*60)
for name, r in results.items():
    print(f"{name:<22} {r['auc']:8.4f} {r['best_f1']:8.4f} {r['best_thresh']:8.2f}")
```

```
============================================================
```

**步骤三：自定义 AUC 指标与 sample_weight 处理不均衡**

目标：理解自定义指标写法，以及 sample_weight 在 fit 中的传递方式。

```python
import tensorflow as tf
from tensorflow import keras

class BinaryF1Score(keras.metrics.Metric):
    """自定义 F1-Score 指标（支持阈值调节）"""
    def __init__(self, threshold=0.5, name="f1_score", **kwargs):
        super().__init__(name=name, **kwargs)
        self.threshold = threshold
        self.tp = self.add_weight(name="tp", initializer="zeros")
        self.fp = self.add_weight(name="fp", initializer="zeros")
        self.fn = self.add_weight(name="fn", initializer="zeros")

    def update_state(self, y_true, y_pred, sample_weight=None):
        y_pred = tf.cast(y_pred >= self.threshold, tf.float32)
        y_true = tf.cast(y_true, tf.float32)

        tp = tf.reduce_sum(y_true * y_pred)
        fp = tf.reduce_sum((1 - y_true) * y_pred)
        fn = tf.reduce_sum(y_true * (1 - y_pred))

        self.tp.assign_add(tp)
        self.fp.assign_add(fp)
        self.fn.assign_add(fn)

    def result(self):
        precision = self.tp / (self.tp + self.fp + 1e-7)
        recall    = self.tp / (self.tp + self.fn + 1e-7)
        return 2 * precision * recall / (precision + recall + 1e-7)

    def reset_state(self):
        self.tp.assign(0.0)
        self.fp.assign(0.0)
        self.fn.assign(0.0)

# 使用 sample_weight 处理不均衡
# 假设 y_train 是 (0/1) 标签
# sample_weight = np.where(y_train == 1, 25.0, 1.0)

# model.fit(X_train, y_train, sample_weight=sample_weights, ...)
```

## 4. 项目总结

### 4.1 优点与缺点对比

| 方面 | 标准交叉熵 | 加权交叉熵 | Focal Loss |
|------|-----------|-----------|------------|
| 不均衡处理 | 差 | 好（需手动调权重） | 优秀（自适应） |
| 超参数数量 | 0 | 1（class_weight） | 2（α, γ） |
| 收敛稳定性 | 高 | 中（过大权重梯度震荡） | 中（γ 大时初期 loss 高） |
| 适用场景 | 均衡分类 | 固定不均衡比 | 动态不均衡、困难样本挖掘 |

| 优化器 | 适用场景 | 缺点 |
|--------|----------|------|
| **SGD** | 精调阶段、小数据集、凸优化 | 收敛慢，卡在鞍点 |
| **Adam** | 大部分场景的默认选择 | 泛化性可能不如 SGD（需验证） |
| **AdamW** | 大模型、Transformer、需要强正则化 | 比 Adam 多一个 weight_decay 超参数 |

### 4.2 适用场景

1. **Focal Loss** → 目标检测（RetinaNet）、极不均衡分类（欺诈/罕见病诊断）
2. **加权 BCE** → 已知道不均衡比例且比例不会剧烈变化（如固定比例的上采样/下采样后）
3. **AdamW** → Transformer、大规模预训练、需要权重衰减的任务
4. **AUC 指标** → 排序类任务（推荐、搜索、风控评分），不受阈值影响
5. **自定义指标** → 业务指标与机器学习指标不完全对齐时（如加权 F1、Profit-based metric）

### 4.3 注意事项

- **loss 与 metrics 的维度必须一致**：多输出模型每个输出配一个 loss 和一个 metric，Keras 按名称匹配
- **Focal Loss 的超参数 γ**：γ=0 退化为加权 BCE；γ 越大越关注难样本（通常取 2）；γ 过大会让模型只学习异常样本而忽略正常模式
- **AUC 不能直接作为 loss**：AUC 不可微，不能作为优化目标。但可以用可微的近似（如 pairwise ranking loss）提升 AUC
- **sample_weight 与 class_weight 不可混用**：`model.fit(class_weight={0:1, 1:25})` 和 `model.fit(sample_weight=...)` 效果类似但实现不同，后者更灵活（可以 sample-level 粒度）

### 4.4 常见踩坑经验

1. **坑**：使用 `class_weight` 后训练 loss 暴增 20 倍，以为模型崩了。
   **根因**：`class_weight` 会影响 `model.fit` 中显示的 `loss` 值（也是加权后的），不代表模型真的变差。
   **解决**：同时看 `val_accuracy`、`val_auc` 等不加权的指标来判断真实效果。

2. **坑**：Adam 训练到后期 loss 剧烈震荡不收敛。
   **根因**：Adam 的累积二阶动量在后期可能因梯度方向剧烈变化而"过冲"。
   **解决**：后期切换到 SGD（降低学习率），或使用余弦退火调度让 lr 逐步衰减到极小值。

3. **坑**：自定义 loss 函数中写了 `tf.reduce_sum` 而非 `tf.reduce_mean`，梯度 NaN。
   **根因**：`reduce_sum` 使得 loss 值随 batch_size 变化，大 batch 的 loss 巨大导致梯度爆炸。Keras 默认将每个样本的 loss 平均后传给优化器。
   **解决**：自定义 loss 函数签名是 `loss_fn(y_true, y_pred)`，Keras 对 batch 内每个样本调用后自动求均值。如果你的 loss 已包含 batch 维度的 reduce_mean，不要再用 reduce_sum。
