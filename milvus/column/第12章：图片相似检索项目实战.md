# 第12章：图片相似检索项目实战

> **定位**：从文本检索扩展到多模态检索。
> **版本**：Milvus 2.5.x
> **源码关联**：internal/proxy/search.go、pymilvus/orm/collection.py

---

## 1. 项目背景

某电商平台的商品库中有 300 万张商品图片。运营团队提出一个需求：用户上传一张"小红书爆款"的截图，系统能找到平台内最相似的商品。这就是"以图搜图"。

前端开发小陈接到任务后，第一反应是用图片的像素 RGB 值做比较——结果可想而知：同一件红色连衣裙，不同角度拍的 3 张照片，像素级相似度不到 10%。"红色"在 RGB 里是 (255, 0, 0)，但实际照片里裙子区域是 50 种不同的红色像素。

小陈找到算法同事求助。算法工程师推荐用 **CNN 预训练模型提取图片特征向量**——ResNet-50 或 CLIP 模型可以把图片编码成一个固定维度的向量，语义相似的图片在向量空间里距离很近。这个思路和文本 Embedding 完全一样，只是输入从文字变成了图片。

但小陈在实际操作中又踩了坑：

1. **图片预处理不一致**：训练时用 224×224 输入，推理时忘了 resize，导致向量不对。
2. **图片尺寸差异巨大**：商品图有 200×200 的缩略图和 4000×3000 的原图，批量处理时内存爆炸。
3. **冷启动问题**：新商品没有足够的 UGC 图片，只有商家上传的一张白底图，图像特征信息量不足。
4. **元数据设计缺失**：只存了图片向量，没存图片 URL、商品 ID、颜色、风格标签，结果搜到了相似商品但找不到商品详情。
5. **多模态不一致**：用户上传的是实拍场景图（有背景、有模特），但商品库里的图是白底棚拍图，特征向量有天然差异。

本章将实现一个完整的"上传图片找相似商品"小项目。

```python
pixel_diff = np.mean(np.abs(img1_pixels - img2_pixels))

vec2 = clip_model.encode_image(img2)  # 512维特征向量
```

**大师**："关键就是这个'特征向量'——它不是描述颜色，而是描述**语义特征**：领口形状、袖子的长度、裙摆的廓形、面料的纹理……CNN 或 ViT 模型把这些高层语义编码进了向量里。"

| 特征提取模型 | 输出维度 | 优势 | 适用场景 |
|-------------|---------|------|---------|
| ResNet-50 | 2048 | 经典、稳定、推理快 | 通用商品检索 |
| EfficientNet-B3 | 1536 | 精度/速度平衡好 | 需要较高精度的场景 |
| CLIP ViT-B/32 | 512 | 图文跨模态对齐（可用文字搜图） | 需要多模态检索 |
| DINOv2 | 768 | 自监督、细节丰富 | 精细款式区分 |

**小白**："那应该选哪个？"

**大师**："如果只做以图搜图，ResNet-50 就够用。如果还想支持'用文字描述搜商品图'（比如'红色碎花连衣裙'），必须用 CLIP 这类跨模态模型。

> **技术映射**：像素比较 = 对比两张照片每个像素点的颜色（受光照/角度影响大）；特征向量 = 提取形状/纹理/语义信息（鲁棒性强）；CLIP = 既能看图又能看文字的"双栖"模型。

#### 步骤 1：用 ResNet-50 提取图片特征向量

```python
# step1_image_embedding.py
"""用预训练 ResNet-50 提取图片特征向量"""
import torch
import torchvision.transforms as transforms
import torchvision.models as models
from PIL import Image
import numpy as np

class ImageEmbedder:
    """图片特征提取器"""
    
    def __init__(self, device: str = "cpu"):
        self.device = device
        # 加载预训练 ResNet-50（去掉最后的分类头）
        self.model = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V1)
        self.model.fc = torch.nn.Identity()  # 只取倒数第二层输出（2048维）
        self.model = self.model.to(device)
        self.model.eval()
        
        # 图片预处理（和 ImageNet 训练时一致）
        self.transform = transforms.Compose([
            transforms.Resize(256),
            transforms.CenterCrop(224),
            transforms.ToTensor(),
            transforms.Normalize(
                mean=[0.485, 0.456, 0.406],
                std=[0.229, 0.224, 0.225]
            ),
        ])
    
    def embed(self, image: Image.Image) -> np.ndarray:
        """单张图片提取向量"""
        img_tensor = self.transform(image).unsqueeze(0).to(self.device)
        with torch.no_grad():
            features = self.model(img_tensor)
        # L2 归一化（配合 COSINE 度量）
        features = features.cpu().numpy().flatten()
        features = features / np.linalg.norm(features)
        return features
    
    def embed_batch(self, images: list) -> np.ndarray:
        """批量提取向量"""
        tensors = torch.stack([
            self.transform(img) for img in images
        ]).to(self.device)
        with torch.no_grad():
            features = self.model(tensors)
        features = features.cpu().numpy()
        # L2 归一化
        features = features / np.linalg.norm(features, axis=1, keepdims=True)
        return features

# 使用示例
embedder = ImageEmbedder()
img = Image.open("test_dress.jpg").convert("RGB")
vec = embedder.embed(img)
print(f"向量维度: {vec.shape}")   # (2048,)
print(f"向量范数: {np.linalg.norm(vec):.4f}")  # 1.0000
```

#### 步骤 3：以图搜图服务

```python
# step3_image_search.py
"""以图搜图服务"""
import numpy as np
from pymilvus import Collection, connections
from collections import defaultdict

connections.connect(host="localhost", port="19530")
collection = Collection("product_image_search")

class ImageSearchService:
    """以图搜图服务"""
    
    def __init__(self, collection, embedder):
        self.collection = collection
        self.embedder = embedder
    
    def search(self, query_image, top_k: int = 10,
               category: str = None, min_score: float = 0.5):
        """搜索相似商品图片
        
        Args:
            query_image: PIL Image 对象
            top_k: 返回结果数
            category: 限定类目（可选）
            min_score: 最低相似度阈值
        
        Returns:
            [{"product_id": ..., "title": ..., "image_url": ..., "score": ...}, ...]
        """
        # 1. 提取查询图片特征
        query_vec = self.embedder.embed(query_image).tolist()
        
        # 2. 构造过滤表达式
        expr = f"category == '{category}'" if category else None
        
        # 3. 向量检索（多召回一些方便去重）
        results = self.collection.search(
            data=[query_vec],
            anns_field="image_vec",
            param={"metric_type": "COSINE", "params": {"ef": 128}},
            expr=expr,
            limit=top_k * 3,  # 多召回 3 倍做去重
            output_fields=["image_url", "product_title", "price", "category", "color", "style"]
        )
        
        # 4. 按 product_id 去重（同商品多张图只保留最高分）
        seen_products = {}
        for hit in results[0]:
            pid = hit.id
            if pid not in seen_products or hit.distance > seen_products[pid]["score"]:
                seen_products[pid] = {
                    "product_id": pid,
                    "title": hit.entity.get("product_title"),
                    "image_url": hit.entity.get("image_url"),
                    "price": hit.entity.get("price"),
                    "category": hit.entity.get("category"),
                    "color": hit.entity.get("color"),
                    "style": hit.entity.get("style"),
                    "score": round(hit.distance, 4),
                }
        
        # 5. 排序 + 过滤 + 截断
        items = sorted(seen_products.values(), key=lambda x: x["score"], reverse=True)
        items = [i for i in items if i["score"] >= min_score]
        return items[:top_k]
```

```python
# 文字"红色碎花连衣裙"的向量可以直接和图片向量做搜索

def text_search_image(collection, clip_model, text_query: str, top_k: int = 10):
    """用文字搜图片"""
    # CLIP 文本编码
    text_vec = clip_model.encode_text(text_query)

        data=[text_vec.tolist()],

    return [
        {
            "title": hit.entity.get("product_title"),
            "image_url": hit.entity.get("image_url"),
            "score": round(hit.distance, 4),

        for hit in results[0]
```

## 4. 项目总结

### 4.1 优缺点对比

| 维度 | 以图搜图（ResNet + Milvus） | 传统标签搜索 | 像素级比较 |
|------|--------------------------|------------|----------|
| 语义理解 | 强（能理解款式/风格） | 依赖人工标注质量 | 无 |
| 跨光照/角度 | 鲁棒 | 不适用 | 极差 |
| 标注成本 | 低（无需人工打标签） | 高（每张图需人工标注） | 无 |
| 新品类扩展 | 快（只需有新图片） | 慢（需更新标签体系） | N/A |

### 4.2 适用场景

- **电商以图搜图**：用户拍/截图找平台同款
- **版权检测**：检测图片是否为盗用
- **服装搭配推荐**：找与当前穿搭相似的搭配
- **工业质检**：产品外观一致性检测

**不适用场景**：需要精确文字 OCR 识别的场景（如搜身份证号）、极实时场景（单张推理 100ms+ 可能不够）。

### 4.3 注意事项

- **图片预处理必须和训练时一致**：ResNet 的 Normalize 参数必须是 mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]，错一个数字向量就废了。
- **向量归一化**：用 COSINE 度量时务必 L2 归一化向量，否则相似度计算不准确。
- **Batch 推理节省时间**：100 张图单独 inference 要 10s，batch=64 一起推理只要 2s。

### 4.4 常见踩坑经验

1. **模型在 GPU 上、输入图片在 CPU 上**：`RuntimeError: Expected all tensors to be on the same device`。解决：`img_tensor.to(device)`。
2. **图片通道数不是 3**：RGBA（4 通道）图片直接输入会报错。解决：`.convert("RGB")`。
3. **内存泄漏**：每次推理都 `torch.no_grad()` 但没清理中间变量。解决：定期 `torch.cuda.empty_cache()`。

### 4.5 思考题

1. 如果商品图片库有 300 万张图，ResNet-50 推理一张图 30ms，单机一天只能处理 288 万张。如何设计一个分布式批量推理方案来加速？
2. CLIP 模型的零样本能力（zero-shot）在以图搜图中有什么独特价值？比如"搜一件适合参加婚礼的连衣裙"这种带场景描述的搜索怎么办？
