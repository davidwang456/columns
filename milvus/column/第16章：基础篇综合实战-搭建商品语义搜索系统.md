## 1. 项目背景

某中型电商平台日均搜索请求 50 万次，现有搜索系统基于 Elasticsearch 的关键词匹配。用户体验团队调研发现：35% 的用户使用口语化描述搜索（"适合去海边穿的连衣裙"、"送男朋友的生日礼物"），但 ES 的关键词匹配只能理解"连衣裙"、"生日礼物"这些核心词，对"海边穿""送男朋友"这类语义描述完全无能为力。

业务方已导入 100 万件商品数据到 MySQL，技术团队在第1-15章中已经分别掌握了 Milvus 部署、Schema 设计、Embedding 写入、索引优化、标量过滤、SDK 封装、备份恢复和故障排查。现在到了"交卷"时刻——把所有知识串成一条完整的生产级搜索系统。

项目要求覆盖全生命周期：数据管道（MySQL → Milvus 同步）、在线搜索接口（语义搜索 + 类目/价格/库存过滤）、搜索结果评估、故障演练。验收标准：100 万商品内 Top10 搜索 P95 < 200ms，支持类目和价格过滤，具备备份恢复能力。

**第二幕：数据管道——MySQL 到 Milvus 的全量+增量同步**

**小白**："那 100 万商品数据怎么从 MySQL 进 Milvus？不会是全量导出再导入吧？"

**大师**："分两步——全量初始化 + 增量同步。全量用批处理，增量用 Binlog 监听。"

```
全量同步 (Day 0)                         增量同步 (Day 1+)
─────────────────────                    ────────────────────
MySQL → CSV 导出                          MySQL Binlog
    ↓                                       ↓
数据清洗（去重/截断/规范化）               Canal / Debezium
    ↓                                       ↓
Embedding 批量生成                        Kafka Topic
    ↓                                       ↓
Milvus Insert (500条/批)                  Embedding 单条/小批
    ↓                                       ↓
Flush + 索引构建                          Milvus Upsert
```

**大师**："全量同步脚本的核心逻辑——"

```python
def full_sync_mysql_to_milvus(mysql_conn, collection, embedder,
                              batch_size=500):
    """MySQL 全量同步到 Milvus"""
    cursor = mysql_conn.cursor()
    cursor.execute("SELECT id, title, price, category, in_stock FROM products")
    
    total = 0
    while True:
        rows = cursor.fetchmany(batch_size)
        if not rows:
            break
        
        ids = [r[0] for r in rows]
        titles = [r[1] for r in rows]
        embeddings = embedder.encode(titles).tolist()
        prices = [r[2] for r in rows]
        categories = [r[3] for r in rows]
        stocks = [r[4] for r in rows]
        
        collection.upsert([ids, titles, embeddings, prices, categories, stocks])
        total += len(rows)
        
        if total % 10000 == 0:
            print(f"同步进度: {total} 条")
    
    utility.flush([collection.name])
    print(f"全量同步完成: {total} 条")
```

**小白**："增量同步用 Upsert 对吧？那如果商品下架了（MySQL 里被删除），怎么同步到 Milvus？"

**大师**："好问题。MySQL 的 Delete 在 Binlog 里是 DELETE 事件。监听这个事件，在 Milvus 里执行 `delete(id in [...])`。如果你们用软删除（加一个 `is_deleted` 字段），那就变成 Upsert 更新该字段，搜索时用 `expr="is_deleted == false"` 过滤。"

> **技术映射**：全量同步 = 搬家（一次性把所有东西搬过去）；增量同步 = 快递补货（每天新增/变更的单独送）；Upsert = 覆盖式补货（新货到了旧的就替换）；Binlog = 快递单号（记录了什么数据发生了变化）。

```python
        },
        ...
    ],

```

**大师**："接口设计四个关键点——"

1. **用户看到的字段 vs Milvus 存的字段**：`product_id` 只用于内部检索，返回给用户的是 `title`、`price`、`image_url`。完整信息通过 MySQL 查。
2. **score 不等于 relevance**：0.93 的相似度不意味着"绝对相关"，可能是向量空间里的"巧合接近"。需要定期抽检 low-score 结果。
3. **query_rewritten**：如果系统做了查询改写（如"海边穿的"→"度假+海滩+夏季"），应该在响应里体现，方便排查和调试。
4. **search_time_ms**：每个响应都带搜索耗时，方便客户端做超时处理和用户体验优化。

```python
"""初始化商品搜索系统"""
from pymilvus import connections, Collection, utility
from pymilvus import CollectionSchema, FieldSchema, DataType
from sentence_transformers import SentenceTransformer

connections.connect(host="localhost", port="19530")

COLL_NAME = "product_search_prod"
DIM = 384

# 如果已存在，先备份再重建
if utility.has_collection(COLL_NAME):
    print(f"⚠ {COLL_NAME} 已存在，将重建")
    utility.drop_collection(COLL_NAME)

fields = [
    FieldSchema(name="product_id", dtype=DataType.INT64, is_primary=True),
    FieldSchema(name="title_vec", dtype=DataType.FLOAT_VECTOR, dim=DIM),
    FieldSchema(name="title", dtype=DataType.VARCHAR, max_length=512),
    FieldSchema(name="price", dtype=DataType.FLOAT),
    FieldSchema(name="category", dtype=DataType.VARCHAR, max_length=64),
    FieldSchema(name="brand", dtype=DataType.VARCHAR, max_length=64),
    FieldSchema(name="in_stock", dtype=DataType.BOOL),
    FieldSchema(name="rating", dtype=DataType.FLOAT),
    FieldSchema(name="sales_count", dtype=DataType.INT64),  # 销量（用于热销加权）
    FieldSchema(name="created_at", dtype=DataType.INT64),   # 上架时间戳
]

schema = CollectionSchema(fields, description="商品语义搜索系统 - 生产版")
collection = Collection(COLL_NAME, schema)

# 索引
collection.create_index("title_vec", {
    "index_type": "HNSW",
    "metric_type": "COSINE",
    "params": {"M": 16, "efConstruction": 200}
})
utility.wait_for_index_building_complete(COLL_NAME, timeout=300)
collection.load()

print(f"✓ 系统初始化完成: {COLL_NAME}, dim={DIM}")
print(f"  Schema 字段: {len(schema.fields)}")
print(f"  索引: HNSW + COSINE")
```

#### 步骤 2：批量导入 + 搜索服务

```python
# step2_search_service.py
"""生产级搜索服务"""
import json
import time
from typing import List, Dict, Optional
from pymilvus import Collection, connections
from sentence_transformers import SentenceTransformer

class ProductSearchSystem:
    """商品语义搜索系统（生产版）"""
    
    def __init__(self, host="localhost", port=19530,
                 collection_name="product_search_prod",
                 model_name="all-MiniLM-L6-v2"):
        connections.connect(host=host, port=port)
        self.collection = Collection(collection_name)
        self.model = SentenceTransformer(model_name)
    
    def bulk_import(self, products: List[Dict], batch_size=500):
        """批量导入商品数据"""
        total = len(products)
        imported = 0
        t_start = time.time()
        
        for i in range(0, total, batch_size):
            batch = products[i:i + batch_size]
            titles = [p["title"] for p in batch]
            embeddings = self.model.encode(titles, batch_size=64).tolist()
            
            self.collection.insert([
                [p["product_id"] for p in batch],
                [p["title"] for p in batch],
                embeddings,
                [p["price"] for p in batch],
                [p["category"] for p in batch],
                [p["brand"] for p in batch],
                [p["in_stock"] for p in batch],
                [p["rating"] for p in batch],
                [p.get("sales_count", 0) for p in batch],
                [p.get("created_at", int(time.time())) for p in batch],
            ])
            imported += len(batch)
            
            if imported % 10000 == 0:
                elapsed = time.time() - t_start
                print(f"  导入 {imported}/{total} ({imported*100//total}%), "
                      f"{imported/elapsed:.0f} 条/s")
        
        utility.flush([self.collection.name])
        elapsed = time.time() - t_start
        print(f"✓ 导入完成: {imported} 条, {imported/elapsed:.0f} 条/s")
    
    def search(self, query: str, top_k: int = 10,
               category: str = None, min_price: float = None,
               max_price: float = None, in_stock_only: bool = False,
               min_rating: float = None) -> Dict:
        """搜索商品"""
        t_start = time.time()
        
        # 1. 构造过滤表达式
        conditions = []
        if category:
            conditions.append(f"category == '{category}'")
        if min_price is not None:
            conditions.append(f"price >= {min_price}")
        if max_price is not None:
            conditions.append(f"price <= {max_price}")
        if in_stock_only:
            conditions.append("in_stock == true")
        if min_rating is not None:
            conditions.append(f"rating >= {min_rating}")
        expr = " and ".join(conditions) if conditions else None
        
        # 2. Embedding
        query_vec = self.model.encode([query]).tolist()
        
        # 3. 搜索
        results = self.collection.search(
            data=query_vec,
            anns_field="title_vec",
            param={"metric_type": "COSINE", "params": {"ef": 64}},
            expr=expr,
            limit=top_k,
            output_fields=["title", "price", "category", "brand",
                          "in_stock", "rating", "sales_count"]
        )
        
        # 4. 格式化响应
        items = []
        for hit in results[0]:
            items.append({
                "product_id": hit.id,
                "title": hit.entity.get("title"),
                "price": hit.entity.get("price"),
                "category": hit.entity.get("category"),
                "brand": hit.entity.get("brand"),
                "in_stock": hit.entity.get("in_stock"),
                "rating": hit.entity.get("rating"),
                "sales_count": hit.entity.get("sales_count"),
                "score": round(hit.distance, 4),
            })
        
        search_time_ms = (time.time() - t_start) * 1000
        return {
            "results": items,
            "total_hits": len(items),
            "search_time_ms": round(search_time_ms, 1),
            "expr_applied": expr,
        }
    
    def health_check(self) -> Dict:
        """健康检查"""
        return {
            "collection": self.collection.name,
            "num_entities": self.collection.num_entities,
            "load_state": utility.load_state(self.collection.name).name,
            "index_built": self.collection.has_index(),
        }

# 使用示例
system = ProductSearchSystem()

# 批量导入
sample_products = [
    {
        "product_id": i,
        "title": f"商品_{i}",
        "price": 99.9 + i * 10,
        "category": "测试",
        "brand": "测试品牌",
        "in_stock": True,
        "rating": 4.5,
        "sales_count": 100,
        "created_at": int(time.time()),
    }
    for i in range(1, 1001)
]
system.bulk_import(sample_products)

# 搜索
result = system.search("适合海边穿的连衣裙", top_k=5,
                       category="连衣裙", min_price=50, max_price=300)
print(json.dumps(result, ensure_ascii=False, indent=2))
```

```python
# step3_api.py
"""FastAPI 搜索接口"""
from fastapi import FastAPI, Query
from pydantic import BaseModel
from typing import Optional
from step2_search_service import ProductSearchSystem

app = FastAPI(title="商品语义搜索系统")
system = ProductSearchSystem()

class SearchRequest(BaseModel):
    query: str
    top_k: int = 10
    category: Optional[str] = None
    min_price: Optional[float] = None
    max_price: Optional[float] = None
    in_stock_only: bool = False
    min_rating: Optional[float] = None

@app.get("/api/v1/products/search")
async def search_products(
    q: str = Query(..., description="搜索词"),
    k: int = Query(10, ge=1, le=100),
    category: Optional[str] = None,
    min_price: Optional[float] = None,
    max_price: Optional[float] = None,
    in_stock_only: bool = False,
):
    result = system.search(
        query=q, top_k=k,
        category=category,
        min_price=min_price, max_price=max_price,
        in_stock_only=in_stock_only,
    )
    return result

@app.get("/api/v1/health")
async def health():
    return system.health_check()
```

```python
# step4_acceptance.py
"""验收测试：功能 + 性能 + 备份恢复"""
import time, json
from concurrent.futures import ThreadPoolExecutor, as_completed
import numpy as np

def acceptance_test(system, test_queries, concurrency=10, iterations=100):
    """验收测试"""
    print("=" * 60)
    print("商品语义搜索系统 - 验收报告")
    print("=" * 60)
    
    # 1. 功能验收
    print("\n--- 功能验收 ---")
    for q in test_queries[:3]:
        r = system.search(q, top_k=5, in_stock_only=True)
        print(f"  '{q}' → {r['total_hits']} 条, {r['search_time_ms']:.1f}ms")
    
    # 2. 性能验收
    print(f"\n--- 性能验收 (并发={concurrency}, 请求={iterations}) ---")
    latencies = []
    
    def do_search(q):
        t0 = time.time()
        system.search(q, top_k=10, in_stock_only=True)
        return (time.time() - t0) * 1000
    
    with ThreadPoolExecutor(max_workers=concurrency) as pool:
        futures = [pool.submit(do_search, test_queries[i % len(test_queries)])
                   for i in range(iterations)]
        for f in as_completed(futures):
            latencies.append(f.result())
    
    lat = np.array(latencies)
    print(f"  P50: {np.percentile(lat, 50):.1f}ms")
    print(f"  P95: {np.percentile(lat, 95):.1f}ms")
    print(f"  P99: {np.percentile(lat, 99):.1f}ms")
    print(f"  Avg: {np.mean(lat):.1f}ms")
    
    #判定
    p95_ok = np.percentile(lat, 95) < 200
    print(f"\n  验收标准: P95 < 200ms → {'✅ 通过' if p95_ok else '❌ 不通过'}")
    
    # 3. 健康检查
    print(f"\n--- 健康检查 ---")
    health = system.health_check()
    for k, v in health.items():
        print(f"  {k}: {v}")
    
    return {"p95_ms": np.percentile(lat, 95), "passed": p95_ok}

# 运行验收
test_queries = [
    "露营折叠椅", "蓝牙降噪耳机", "碎花连衣裙",
    "大容量登山背包", "静音机械键盘",
]
acceptance_test(system, test_queries, concurrency=10, iterations=100)
```

## 4. 项目总结

### 4.1 系统架构回顾

| 模块 | 技术选型 | 关键参数 |
|------|---------|---------|
| 向量数据库 | Milvus 2.5 Standalone | HNSW + COSINE, M=16, ef=64 |
| Embedding | SentenceTransformer all-MiniLM-L6-v2 | 384维 |
| API 层 | FastAPI | Port 8080 |
| 数据管道 | Python 批量同步 + 增量 Upsert | 500条/批 |
| 备份 | milvus-backup + JSON 导出 | 每日全量 |

### 4.2 优缺点

| 维度 | 优点 | 缺点 |
|------|------|------|
| 搜索质量 | 语义理解能力强，支持口语化查询 | 对专业术语（如"JEEP-212"）可能不如关键词 |
| 性能 | 100万条 P95 < 200ms | 数据量增长到千万级需扩容 |
| 运维 | 单机部署简单 | 无高可用，单点故障 |
| 扩展性 | 代码分层清晰，易扩展 | 需手动扩容 QueryNode |

### 4.3 注意事项

- **全量同步后务必验证数据量一致性**：`collection.num_entities` vs MySQL `COUNT(*)`。
- **搜索接口要加熔断**：如果 Milvus 响应超过 500ms，自动降级到 MySQL LIKE 搜索。
- **日志全链路追踪**：每个搜索请求记录 `trace_id`，关联 API → Embedding → Milvus 三段耗时。
