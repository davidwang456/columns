# 第33章：后端核心——Go服务架构剖析

## 1. 项目背景

"我想在Grafana中增加一个自定义的API端点，用来查询某个Dashboard在过去24小时被多少用户打开过。这需要修改Grafana后端代码——但我完全不知道API是怎么注册的、数据库是怎么操作的、中间件是怎么链式调用的。"

后端工程师老刘的需求很明确——需要一个"Dashboard使用统计"API。但Grafana的后端代码组织了50+个service、100+个API handler、复杂的Wire依赖注入。不搞清楚后端的"骨架"，写一个API都无从下手。

本章将深度剖析Grafana Go后端的五大核心机制：API路由注册、中间件链、服务层架构、数据库操作和插件系统。让你从"能看懂代码"进阶到"能写代码"。

## 2. 项目设计

**小胖**（看着pkg/api/目录下几十个文件）：大师，Grafana后端的Go代码，光pkg/api/目录就有几十个文件。我要加一个自定义API端点，从哪开始？

**大师**：加一个API端点的流程是这样的：

**第一步：定义路由**

在`pkg/api/api.go`的`registerRoutes()`方法中添加：
```go
func (hs *HTTPServer) registerRoutes() {
    // ... 其他路由 ...
    
    reqGrafanaAdmin := reqSignedIn && reqGrafanaAdmin
    
    hs.RouteRegister.Get("/api/custom/stats", 
        routing.Wrap(hs.GetDashboardStats), reqSignedIn)
}
```

**第二步：实现Handler**

在`pkg/api/dashboard.go`（或新建文件）中添加：
```go
func (hs *HTTPServer) GetDashboardStats(c *models.ReqContext) {
    stats, err := hs.DashboardService.GetStats(c.Req.Context(), c.OrgID)
    if err != nil {
        c.JsonApiErr(500, "Failed to get stats", err)
        return
    }
    c.JSON(200, stats)
}
```

**第三步：实现Service层**

在`pkg/services/dashboards/`中添加业务逻辑。这是Grafana最重要的分层约定——API层只做参数校验和响应，业务逻辑在Service层。

**小白**：那中间件链呢？比如认证、权限检查、审计日志这些？

**大师**：Grafana的中间件链在路由注册时指定。每个`reqXXX`变量都是一个中间件：

```go
reqSignedIn := middleware.ReqSignedIn        // 必须登录
reqGrafanaAdmin := middleware.ReqGrafanaAdmin // 必须是Admin
reqOrgAdmin := middleware.ReqOrgAdmin        // 必须是Org Admin
reqCanAccessDashboard := middleware.Authorize(accesscontrol.ActionDashboardsRead)
```

中间件按顺序执行：认证→授权→Handler。任何一个中间件返回错误，请求链就终止。

深入中间件实现：
```go
// pkg/middleware/auth.go
func ReqSignedIn(c *models.ReqContext) {
    if !c.IsSignedIn {
        c.JsonApiErr(401, "Unauthorized", nil)
        return
    }
}
```

**小胖**：那服务层之间怎么互相调用？Nginx通过Wire依赖注入？

**大师**：对。Grafana的Service依赖关系在`pkg/server/wire.go`中定义。比如`HTTPServer`依赖`DashboardService`，`DashboardService`又依赖`SQLStore`：

```go
var WireSet = wire.NewSet(
    ProvideHTTPServer,
    ProvideDashboardService,
    ProvideSQLStore,
    ...
)

func ProvideDashboardService(store *SQLStore) *DashboardService {
    return &DashboardService{store: store}
}
```

Wire编译时的魔力：你只写了"HTTPServer需要DashboardService，DashboardService需要SQLStore"，Wire自动生成代码按正确顺序创建它们。

**小白**：数据库操作呢？Grafana怎么操作PostgreSQL/MySQL？

**大师**：Grafana使用`xorm`作为ORM（对象关系映射）库。但Grafana在其上封装了一层`pkg/services/sqlstore/`。

典型的数据库操作：
```go
// 查询Dashboard
func (ss *SQLStore) GetDashboard(ctx context.Context, query *models.GetDashboardQuery) error {
    dashboard := &models.Dashboard{UID: query.UID}
    has, err := ss.engine.Where("uid = ?", query.UID).Get(dashboard)
    if err != nil {
        return err
    }
    if !has {
        return models.ErrDashboardNotFound
    }
    query.Result = dashboard
    return nil
}
```

数据库Migration：Grafana的所有表结构定义在`pkg/services/sqlstore/migrations/`。每次启动时自动检查数据库schema版本，执行未应用的migration。

**技术映射**：API Handler = 餐厅服务员（接收订单+上菜），Middleware = 安检通道（先过安检再进大厅），Service = 后厨（真正做菜的地方），Wire = 自动化装配线（零件自动组装成机器）。

## 3. 项目实战

**环境准备**：基于第32章的开发环境。

需求：创建一个API `/api/custom/hello`，返回一条欢迎消息。

func (hs *HTTPServer) HelloWorld(c *models.ReqContext) {

        "message": "Hello from Grafana!",

**3. 测试**：
```bash
curl -H "Authorization: Bearer admin-token" \
  http://localhost:3000/api/custom/hello
# 预期响应: {"message":"Hello from Grafana!","user":"admin"}
```

**步骤二：添加数据库查询**

需求：创建API返回Dashboard总数。

**1. 定义Model**（`pkg/models/dashboard_stats.go`）：
```go
package models

type DashboardStats struct {
    TotalCount  int64 `json:"total_count"`
    OwnedCount  int64 `json:"owned_count"`
}
```

**2. 添加Service方法**（`pkg/services/dashboards/dashboard.go`）：
```go
func (ds *DashboardService) GetStats(ctx context.Context, orgID int64) (*models.DashboardStats, error) {
    stats := &models.DashboardStats{}
    
    total, err := ds.store.GetDashboardCount(ctx, orgID)
    if err != nil {
        return nil, err
    }
    stats.TotalCount = total
    
    return stats, nil
}
```

**3. 添加数据库查询**（`pkg/services/sqlstore/dashboard.go`）：
```go
func (ss *SQLStore) GetDashboardCount(ctx context.Context, orgID int64) (int64, error) {
    count, err := ss.engine.
        Where("org_id = ?", orgID).
        Count(&models.Dashboard{})
    return count, err
}
```

**4. 注册Wire依赖**：修改wire.go并运行`make gen-go`。

**步骤三：深入理解Middlewaer链**

追踪一个完整的请求链路：

```
GET /api/dashboards/uid/abc
  ↓
Middleware 1: recovery.Recovery()      // 捕获panic
  ↓
Middleware 2: Logger(cfg)              // 记录请求日志
  ↓
Middleware 3: ReqContextMiddleware()   // 构建请求上下文(从Cookie提取用户信息)
  ↓
Middleware 4: ReqSignedIn              // 检查是否登录
  ↓
Middleware 5: orgRoleAuthorize()       // RBAC权限检查
  ↓
Handler: HTTPServer.GetDashboard()     // 业务处理
  ↓
Service: DashboardService.GetDashboard()
  ↓
SQLStore: engine.Where("uid=?", uid).Get()
  ↓
返回JSON响应
```

**步骤四：自定义中间件**

在`pkg/middleware/custom.go`：

```

```

**步骤五：数据库Migration**

创建新的表：
```go
// pkg/services/sqlstore/migrations/custom_mig.go
package migrations

import "xorm.io/xorm"

func AddDashboardAccessLogTable(mg *MigrationDriver) error {
    type DashboardAccessLog struct {
        ID          int64  `xorm:"pk autoincr 'id'"`
        OrgID       int64  `xorm:"org_id"`
        DashboardUID string `xorm:"dashboard_uid"`
        UserID      int64  `xorm:"user_id"`
        AccessedAt  int64  `xorm:"accessed_at"`
    }
    return mg.CreateTable(&DashboardAccessLog{})
}

// 在migrations.go中注册
func init() {
    AddMigration("Add dashboard_access_log table", AddDashboardAccessLogTable)
}
```

**常见坑点**
1. **Wire循环依赖**：如果Service A依赖Service B，Service B又依赖Service A → Wire编译报错。解决：提取出接口或公共依赖。
2. **数据库查询未传Context**：xorm的查询必须带`Context(ctx)`（带超时控制），直接用`engine.Where()`可能导致goroutine泄漏。
3. **JSON序列化循环引用**：Model中包含指针互相引用时，`json.Marshal`会无限递归。添加`json:"-"`标签跳过。

**步骤六：实战——实现Dashboard使用统计API**

完整实现一个带数据库持久化的自定义功能。

**1. Model定义**（`pkg/models/dashboard_stats.go`）：
```go
type DashboardAccessStats struct {
    DashboardUID string `json:"dashboard_uid"`
    Title        string `json:"title"`
    AccessCount  int64  `json:"access_count"`
    LastAccess   int64  `json:"last_access"`
}
```

**2. SQLStore实现**（`pkg/services/sqlstore/dashboard_stats.go`）：
```go
func (ss *SQLStore) GetMostAccessedDashboards(ctx context.Context, orgID int64, limit int) ([]*models.DashboardAccessStats, error) {
    results := make([]*models.DashboardAccessStats, 0)
    err := ss.engine.SQL(`
        SELECT d.uid AS dashboard_uid, d.title,
               COUNT(al.id) AS access_count,
               MAX(al.accessed_at) AS last_access
        FROM dashboard d
        LEFT JOIN dashboard_access_log al ON d.uid = al.dashboard_uid
        WHERE d.org_id = ?
        GROUP BY d.uid, d.title
        ORDER BY access_count DESC
        LIMIT ?
    `, orgID, limit).Find(&results)
    return results, err
}

func (ss *SQLStore) LogDashboardAccess(ctx context.Context, orgID int64, dashboardUID string, userID int64) error {
    log := &models.DashboardAccessLog{
        OrgID:        orgID,
        DashboardUID: dashboardUID,
        UserID:       userID,
        AccessedAt:   time.Now().Unix(),
    }
    _, err := ss.engine.Insert(log)
    return err
}
```

**3. Service层**（`pkg/services/dashboards/dashboard_stats.go`）：
```go
func (ds *DashboardService) GetMostAccessedDashboards(ctx context.Context, orgID int64, limit int) ([]*models.DashboardAccessStats, error) {
    if limit <= 0 || limit > 100 {
        limit = 20
    }
    return ds.store.GetMostAccessedDashboards(ctx, orgID, limit)
}
```

**4. API Handler**（`pkg/api/dashboard_stats.go`）：
```go
func (hs *HTTPServer) GetMostAccessedDashboards(c *models.ReqContext) {
    limit := c.QueryInt("limit")
    if limit <= 0 {
        limit = 20
    }

    stats, err := hs.DashboardService.GetMostAccessedDashboards(
        c.Req.Context(), c.OrgID, limit,
    )
    if err != nil {
        c.JsonApiErr(500, "Failed to get dashboard stats", err)
        return
    }
    c.JSON(200, stats)
}
```

**5. 注册路由**：
```go
// 在 pkg/api/api.go 的 registerRoutes() 中
hs.RouteRegister.Get("/api/dashboards/stats/most-accessed",
    routing.Wrap(hs.GetMostAccessedDashboards), reqSignedIn)
```

**6. 注入Dashboard访问日志中间件**：
```go
func (hs *HTTPServer) LogDashboardView(c *models.ReqContext) {
    uid := web.Params(c.Req)[":uid"]
    if uid != "" && c.IsSignedIn {
        _ = hs.SQLStore.LogDashboardAccess(
            c.Req.Context(), c.OrgID, uid, c.UserID,
        )
    }
}
```

**Grafana后端分层架构**

| 层 | 职责 | 示例文件 |
|----|------|---------|
| API Handler | HTTP请求/响应处理 | `pkg/api/dashboard.go` |
| Middleware | 认证/授权/日志/恢复 | `pkg/middleware/auth.go` |
| Service | 业务逻辑 | `pkg/services/dashboards/` |
| SQLStore | 数据库操作 | `pkg/services/sqlstore/dashboard.go` |
| Model | 数据结构定义 | `pkg/models/dashboard.go` |
| Migration | 数据库schema变更 | `pkg/services/sqlstore/migrations/` |

**最佳实践**
1. API层不写业务逻辑——只做参数校验、调用Service、构造响应
2. Service层不写SQL——通过SQLStore接口操作数据库
3. 新Model必须定义JSON Tag——用于API序列化
4. 数据库Migration不可修改已执行的migration——只能新增
