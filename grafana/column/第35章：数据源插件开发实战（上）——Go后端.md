# 第35章：数据源插件开发实战（上）——Go后端

## 1. 项目背景

"我们内部有一个自研的指标存储系统，用自定义协议通信。Prometheus采集不了，现有的Grafana数据源也无法直接对接。只能通过一个Python脚本把数据导出到Prometheus再展示——绕了一大圈，延迟至少5分钟。"

研发经理阿峰正在为团队的"数据孤岛"发愁。他们的自研存储系统使用gRPC通信、Protobuf序列化、私有认证机制——市面上没有任何现成的Grafana数据源能直接对接。唯一的选择是自己开发一个Grafana DataSource Plugin。

从Grafana 10开始，推荐的数据源插件架构是基于Grafana Plugin SDK for Go（即backend plugin）。Go后端负责与数据源通信（网络请求、认证、查询转发），React前端负责UI配置界面。两者通过gRPC通信。

本章将手把手教你开发一个完整的自定义数据源插件的Go后端——从项目脚手架到查询接口实现，再到数据帧构建和健康检查。

## 2. 项目设计

**小胖**：大师，我想给Grafana写一个数据源插件，让Grafana能直连我们公司的自研数据库。但这个"插件"到底怎么写？我看官方文档说要"backend plugin"、"frontend plugin"，还有"create-plugin"脚手架——头都大了。

**大师**：Grafana数据源插件实际上是一个独立的进程，通过gRPC与Grafana主进程通信。架构如下：

```
Grafana Server (主进程)
       |
     gRPC ←→ Backend Plugin (独立的Go进程)
                  ↓
              你的自研数据库
```

为什么是独立进程？安全隔离——插件崩溃不会拖垮Grafana主进程。协议是gRPC——跨语言、高性能、双向流式通信。

**小白**：具体需要实现哪些接口？

**大师**：Grafana Plugin SDK for Go定义了三个核心接口：

**`CheckHealth`**：健康检查。Grafana在数据源配置页点击"Save & test"时调用。

**`QueryData`**：执行查询。这是最核心的方法——Grafana每次刷新Dashboard面板时调用。

**`StreamHandler`**：实时数据流。用于WebSocket连接，持续推送数据到Grafana。

```go
type DatasourceInstance interface {
    CheckHealth(ctx context.Context, req *CheckHealthRequest) (*CheckHealthResult, error)
    QueryData(ctx context.Context, req *QueryDataRequest) (*QueryDataResponse, error)
}
```

**小胖**：那`QueryData`方法里，数据怎么返回给Grafana？

**大师**：通过DataFrame（数据帧）。一个QueryDataResponse包含多个DataFrame，每个DataFrame包含多个Field：

```go
response := &backend.QueryDataResponse{
    Responses: map[string]backend.DataResponse{
        "A": {
            Frames: data.Frames{
                data.NewFrame("response",
                    data.NewField("time", nil, []time.Time{...}),
                    data.NewField("value", nil, []float64{...}),
                ),
            },
        },
    },
}
```

Grafana拿到DataFrame后，交给前端React组件渲染成图表。数据源插件的工作就是把"你的数据格式"转成"Grafana的DataFrame格式"。

**技术映射**：Backend Plugin = 翻译官（你的数据格式 ↔ Grafana DataFrame），gRPC = 专用电话线（插件和Grafana之间的通信），CheckHealth = 体检（数据源能正常工作吗？），QueryData = 点菜（按查询条件返回数据）。

**步骤二：实现后端核心**

`pkg/plugin/datasource.go`：

```go
package plugin

import (
    "context"
    "encoding/json"
    "fmt"
    "time"

    "github.com/grafana/grafana-plugin-sdk-go/backend"
    "github.com/grafana/grafana-plugin-sdk-go/backend/instancemgmt"
    "github.com/grafana/grafana-plugin-sdk-go/backend/log"
    "github.com/grafana/grafana-plugin-sdk-go/data"
)

type CompanyDBDatasource struct {
    client *CompanyDBClient  // 你对自研数据库的客户端
}

func NewCompanyDBDatasource(settings backend.DataSourceInstanceSettings) (instancemgmt.Instance, error) {
    var config struct {
        ServerURL string `json:"serverURL"`
        AuthToken string `json:"authToken"`
    }
    if err := json.Unmarshal(settings.JSONData, &config); err != nil {
        return nil, err
    }
    
    token := settings.DecryptedSecureJSONData["authToken"]
    
    client, err := NewCompanyDBClient(config.ServerURL, token)
    if err != nil {
        return nil, fmt.Errorf("failed to create client: %w", err)
    }
    
    return &CompanyDBDatasource{client: client}, nil
}

// CheckHealth 健康检查
func (d *CompanyDBDatasource) CheckHealth(ctx context.Context, req *backend.CheckHealthRequest) (*backend.CheckHealthResult, error) {
    err := d.client.Ping(ctx)
    if err != nil {
        return &backend.CheckHealthResult{
            Status:  backend.HealthStatusError,
            Message: fmt.Sprintf("连接失败: %s", err.Error()),
        }, nil
    }
    
    return &backend.CheckHealthResult{
        Status:  backend.HealthStatusOk,
        Message: "连接成功",
    }, nil
}

// QueryData 执行查询
func (d *CompanyDBDatasource) QueryData(ctx context.Context, req *backend.QueryDataRequest) (*backend.QueryDataResponse, error) {
    response := backend.NewQueryDataResponse()
    
    for _, q := range req.Queries {
        res := d.query(ctx, req.PluginContext, q)
        response.Responses[q.RefID] = res
    }
    
    return response, nil
}

func (d *CompanyDBDatasource) query(ctx context.Context, pCtx backend.PluginContext, query backend.DataQuery) backend.DataResponse {
    // 1. 解析查询参数
    var qm QueryModel
    if err := json.Unmarshal(query.JSON, &qm); err != nil {
        return backend.ErrDataResponse(backend.StatusBadRequest, 
            fmt.Sprintf("json unmarshal: %v", err.Error()))
    }
    
    // 2. 调用你的数据库查询
    // 从query.TimeRange获取时间范围
    from := query.TimeRange.From
    to := query.TimeRange.To
    
    results, err := d.client.Query(ctx, qm.MetricName, qm.Filters, from, to)
    if err != nil {
        return backend.ErrDataResponse(backend.StatusInternal, err.Error())
    }
    
    // 3. 构建DataFrame返回
    frame := data.NewFrame("response",
        data.NewField("time", nil, results.Timestamps),
        data.NewField("value", nil, results.Values),
    )
    
    // 可选：添加Meta信息（单位、数据类型等）
    frame.Meta = &data.FrameMeta{
        Type: data.FrameTypeTimeSeriesWide,
    }
    
    return backend.DataResponse{
        Frames: data.Frames{frame},
    }
}

type QueryModel struct {
    MetricName string            `json:"metricName"`
    Filters    map[string]string `json:"filters"`
}
```

**步骤六：扩展功能——支持自定义查询参数**

在QueryModel中支持更多参数：
```go
type QueryModel struct {
    MetricName string            `json:"metricName"`
    Filters    map[string]string `json:"filters"`
    Aggregation string           `json:"aggregation"` // avg, max, min
    Interval    int              `json:"interval"`    // 秒
}
```

在`query`方法中根据这些参数调整查询行为。

**常见坑点**
1. **插件签名问题**：开发模式的插件未通过Grafana签名验证，必须在grafana.ini中配置`GF_PLUGINS_ALLOW_LOADING_UNSIGNED_PLUGINS`。
2. **DataFrame Field类型不匹配**：时间列必须是`time.Time`类型，数值列必须是`float64`（或nullable的`*float64`）。
3. **gRPC通信超时**：默认30s。长时间查询需要在QueryData中建立超时控制。

## 4. 项目总结

**数据源插件必须实现的接口**

| 接口 | 调用时机 | 必须实现 |
|------|---------|---------|
| CheckHealth | Save & test | ✅ |
| QueryData | 面板查询/Explore | ✅ |
| StreamHandler | WebSocket实时数据 | ❌ 可选 |
| CallResource | 自定义HTTP端点 | ❌ 可选 |

**插件开发流程**
1. `create-plugin` 脚手架创建项目
2. 实现Go后端（QueryData/CheckHealth）
3. 编译测试（mage build）
4. 开发React前端（QueryEditor/ConfigEditor）
5. 打包签名 + 发布
