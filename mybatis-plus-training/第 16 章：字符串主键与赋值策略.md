# 第 16 章：字符串主键与赋值策略

示例模块：`mybatis-plus-sample-id-string`。

## 1）项目背景

业务主键有时是 **UUID、单号、外部系统 ID**，需用字符串承载。MP 通过 `@TableId(type = IdType.AUTO)`（在部分示例中表示字符串序列策略，具体以模块配置为准）等与表侧序列或应用赋值配合。**批量插入**时仍需验证每条 id 是否符合预期。

**痛点放大**：字符串与数字列类型混用、大小写敏感、索引选择性差，都会导致**隐式转换**与**慢查询**。

**本章目标**：跑通 `IdStringTest`；理解字符串主键与 `Service.saveBatch` 组合；建立与第 15 章数字主键的对比心智。

## 2）项目设计：小胖、小白与大师的对话

**小胖**：字符串主键不是浪费空间吗？

**大师**：换的是**业务可读性**与**对接外部系统**的便利；要在存储与查询计划上付代价。

**技术映射**：**字符串 PK** ≈ 车牌号；**长整型雪花** ≈ 流水号。

---

**小白**：能跟雪花混用吗？

**大师**：同一库表应**统一策略**；混用是架构债务，除非分库分表按域隔离。

**本章金句**：选类型先看**查询形态与对接协议**，再看 ORM。

## 3）项目实战

**环境准备**：`mvn -pl mybatis-plus-sample-id-string -am test`。

**步骤 1：实体 `@TableId`**

```1:20:d:\software\workspace\mybatis-plus\mybatis-plus-samples\mybatis-plus-sample-id-string\src\main\java\com\baomidou\samples\entity\User.java
package com.baomidou.samples.entity;

import com.baomidou.mybatisplus.annotation.IdType;
import com.baomidou.mybatisplus.annotation.TableId;
import com.baomidou.mybatisplus.annotation.TableName;
import lombok.Data;

@Data
@TableName("sys_user")
public class User {
    /**
     * 字符串自增 ID
     */
    @TableId(type = IdType.AUTO)
    private String id;
    private String name;
    private Integer age;
    private String email;

}
```

## 4）项目总结

| 优点 | 缺点 / 边界 |
|------|-------------|
| 贴近业务单号 | 索引与存储成本 |
| 与外部系统对齐 | 类型混用风险高 |

**适用场景**：强业务含义主键、对接遗留编号体系。

**不适用场景**：超高并发窄表仅需要极简主键时优先考虑数字策略。

**注意事项**：列类型、字符集、排序规则。

**常见踩坑（案例化）**

1. **现象**：比较 id 不相等。**根因**：前后空格或大小写。**处理**：规范写入与 trim。
2. **现象**：ORM 映射 Long 与 varchar PK。**根因**：类型选错。**处理**：统一实体类型。
3. **现象**：批量赋值重复。**根因**：业务号生成非幂等。**处理**：幂等键与唯一约束。
