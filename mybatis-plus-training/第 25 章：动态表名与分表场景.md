# 第 25 章：动态表名与分表场景

示例模块：`mybatis-plus-sample-dynamic-tablename`。

## 1）项目背景

分表、按年归档、多租户分库等场景需要在运行时切换**逻辑表名**，同时复用同一套 `BaseMapper` 方法。MP 通过 **动态表名插件** + 解析器读取上下文（如 ThreadLocal）决定真实表名，**必须有白名单或规则约束**，避免任意字符串拼入标识位。

**痛点放大**：ThreadLocal 未清理会**串表**；多数据源下上下文与路由错乱会导致**写到错误分片**。

**本章目标**：跑通 `DynamicTableNameTest`；理解 `RequestDataHelper` 示例；建立与 SQL 注入防护的区分（标识符 vs 值）。

## 2）项目设计：小胖、小白与大师的对话

**小胖**：动态表名就是把表名拼进 SQL 吧？

**大师**：拼的是**标识符**，要有**规则**；和用户输入当「表名」是两回事。

**技术映射**：**动态表名** ≈ 可变门牌号；**参数绑定** ≈ 包裹里装啥货。

---

**小白**：和 ShardingSphere 有啥分工？

**大师**：MP 侧偏**轻量切换**；大规模分片路由、分布式事务往往要上专用中间件。

**本章金句**：动态表名管**访问哪张表**，参数绑定管**行级数据**。

## 4）项目总结

| 优点 | 缺点 / 边界 |
|------|-------------|
| 复用 Mapper | 调试路径复杂 |
| 适合规则分表 | 需严格清理上下文 |

**适用场景**：年月分表、可枚举逻辑表名。

**不适用场景**：超大规模分片（评估专用中间件）。

**注意事项**：`finally` 清理 ThreadLocal；与租户插件顺序。

**常见踩坑（案例化）**

1. **现象**：随机串表。**根因**：未 `removeRequestData`。**处理**：`try/finally`。
2. **现象**：SQL 表名非法。**根因**：规则外输入。**处理**：白名单校验。
3. **现象**：多数据源错库。**根因**：路由键未随表切换。**处理**：统一路由层。
