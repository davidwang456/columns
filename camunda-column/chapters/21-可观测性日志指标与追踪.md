## 1. 项目背景

用户投诉「流程卡住了」——运维需要**分钟级定位**：是作业积压、外部任务 Worker 挂了，还是 DB 锁？开发需要把日志里的 `processInstanceId`、`activityId` 与分布式追踪 **traceId** 对齐。本章要解决的**一条主线问题**是：建立 **日志—指标—追踪** 三板斧与流程引擎字段的**关联规范**，形成排障剧本。

---

## 2. 项目设计（三角色对话）

*Grafana 上 CPU 绿油油，客服电话却红彤彤。*
**小胖**：日志我全开 **DEBUG**，问题不就无所遁形？  
**小白**：磁盘和隐私先遁形了——生产要**结构化** + **关键字段** + **追踪采样**，别用 grep 洗澡。  
**大师**：MDC 钉 `processInstanceId`、`businessKey`（**脱敏**）；指标盯**作业时长、失败率、REST 慢查询、连接池、External Task backlog**；OpenTelemetry 从入口 filter 把 trace 透到 **Delegate/Worker**。脱敏规则**安全 + 合规**双签，别让开发拍脑袋。  
**小白**：CPU 很低为啥还慢？  
**大师**：流程系统常见脖子在 **DB** 和 **外部系统**——盯 **Golden Signals** 再加业务 KPI：待办堆积、超时实例。**CPU 躺平不代表你健康**，可能只是卡在锁和下游。

## 3. 项目实战

### 3.1 环境前提

- 可选 Prometheus、Grafana；日志聚合 ELK/OpenSearch。

### 3.2 步骤说明

1. 在启动实例入口写入 MDC。
2. 配置 Micrometer 指标导出（若 Spring Boot）。
3. 定义告警：作业队列深度超阈。
4. 演练：人为停 Worker，观察指标与日志能否定位。
5. 文档化「从用户 id 到 instanceId」查询路径（权限控制）。

### 3.3 源码与说明

**为什么关联 businessKey**：客服场景常用；注意隐私。

### 3.4 验证

- 故障注入演练在预定时间内定位根因；无敏感明文日志。

### 3.5 排障剧本（缩写）

1. 用户报障 → 记录 **businessKey** / 订单号。
2. Cockpit 查实例 **卡在哪一活动**。
3. 日志按 **instanceId** 拉全链路；看 **incident** 与 **异常栈**。
4. 若涉及外部任务 → 查 **Worker 指标** 与 **下游**。
5. 若涉及定时器 → 查 **作业延迟** 与 **时钟**。
6. 复盘写入 **第 24 章** 坑清单。

## 5. 附录：日志样例字段（推荐）

`timestamp`、`level`、`traceId`、`processInstanceId`、`activityId`、`businessKey`(脱敏)、`tenantId`、`userId`、`durationMs`、`errorCode`。字段稳定后，ELK 解析规则与仪表盘才可持续。

## 6. 课后作业（可选）

1. 在预发配置 **MDC**，打印一条带 `processInstanceId` 的样例日志。  
2. 与监控同事建三个面板：**REST P95**、**作业延迟**、**错误率**。  
3. 设计一次 **故障演练**：停 Worker，验证告警是否在 X 分钟内触发。  
4. 列出 **禁止入日志** 的字段清单（PII）。  
5. 采样率策略：**全量** vs **按租户** vs **按错误放大**。  

## 7. 章末提要（面向推广口播）

可观测性的目标不是「日志越多越好」，而是 **5 分钟内能回答**：**卡在哪、影响谁、是否扩大化**。给值班同事一张**排障决策树**胜过一百条 debug。记住把 **processInstanceId** 与 **traceId** 绑在一起，否则链路追踪和流程视图会对不上号。**凡是不能从日志还原的路径，就不该上生产。**

## 8. 深度追问（写给半年后的自己）

1. **日志保留天数**与合规是否一致？若缩短，谁签字？  
2. 采样率在 **大促**可否动态调低？开关谁掌握？  
3. 指标 **基数爆炸**（高基数 label）是否已治理？  
4. 链路追踪 **尾部采样**是否启用？成本如何？  
5. 若第三方 APM 与自建 ELK 并存，**权威**是谁？  
6. on-call 是否考核 **MTTR** 与 **误报率**？  

**补白**：可观测性是团队共同产品；只买工具不落地字段规范，必失败。
