## 1. 项目背景

「三天不审批自动升级」「两小时未支付关单」——这类需求本质是**时间驱动**的流程推进。BPMN 用**定时器事件**表达：可挂在流程上，也可作为**边界**挂在用户任务旁表示超时。本章要解决的**一条主线问题**是：理解 **ISO-8601 时长**、**cron** 与**作业（Job）** 的关系，并知道运维视角下**时钟、时区与作业积压**如何影响定时精度（作业细节见第 22 章）。

---

## 2. 项目设计（三角色对话）

### 2.1 小胖开球

小胖说：「我用 cron 每分钟扫表也能做超时，为啥要边界定时器？」

扫表能行，但**流程语义**散落在多处代码；边界定时器让「超时」在**同一张 BPMN** 上与主路径并列，测试与审计都简单。代价是引擎依赖**作业执行器**准时调度，运维要关注作业表与线程池。

### 2.2 小白追问

小白问：「第一，定时器是**一次性**还是周期？第二，夏令时切换怎么办？第三，CI 里测定时器怎么稳定？」

### 2.3 大师定调

大师归纳：

- **ISO-8601** 如 `PT48H` 表示 48 小时后触发；**cron** 适合周期性边界（按引擎支持情况）。
- **时区**：统一用 UTC 或业务时区配置，文档写明。
- **测试**：使用可控时钟、缩短时长、或在测试中触发作业执行（第 18 章）。

### 2.4 业务日历 vs 自然时间

小胖问：「三天算不算周末？」大师答：BPMN 定时器默认是自然时间；若需 **工作日 SLA**，要么在业务服务里算 deadline 写变量，要么引入**工作日历服务**；不要假装引擎懂「春节放假」。

## 3. 项目实战

### 3.1 环境前提

- 用户任务可停留；作业执行器开启。

### 3.2 步骤说明

1. 在用户任务旁加**非中断型边界定时器**（提醒）与**中断型**（超时转交）各一（按需求二选一亦可）。
2. 设置 `PT1M` 便于本地验证。
3. 发起流程后不完成任务，等待一分钟观察是否触发超时路径。
4. 记录 Cockpit 中 token 变化与作业日志。

### 3.3 源码与说明

BPMN 边界定时器（概念）：

```xml
<bpmn:boundaryEvent id="timeout" attachedToRef="approveTask">
  <bpmn:outgoing>toEscalate</bpmn:outgoing>
  <bpmn:timerEventDefinition>
    <bpmn:timeDuration>PT1M</bpmn:timeDuration>
  </bpmn:timerEventDefinition>
</bpmn:boundaryEvent>
```

**为什么区分中断/非中断**：前者取消原任务走向新路径；后者并行提醒。

运维检查点：作业是否 backlog、时钟是否 NTP 同步（第 21 章）。

### 3.4 验证

- 超时路径可重复触发；无「卡死」在用户任务。
- 与 SLA 报表字段对齐（进阶）。

### 3.5 SLA 报表对接提示

- 记录 **进入用户任务时间**、**超时时间**、**实际完成时间**（注意来源：引擎历史 vs 业务表）。  
- 与 BI 对齐 **时区** 与 **工作日定义**。

## 4. 项目总结

| 维度 | 内容 |
|------|------|
| 优点 | SLA 语义上图；与主流程并列易测试。 |
| 缺点 / 代价 | 依赖作业调度；长周期实例占用资源。 |
| 适用场景 | 超时升级、自动关闭、提醒。 |
| 不适用场景 | 超大规模秒级精度（需专项设计）。 |
| 注意事项 | 时区、作业积压、与业务日历（工作日）差异。 |
| 常见踩坑 | 以为定时器精确到毫秒；忘记作业执行器未跑。 |

**延伸阅读**：第 22 章作业；第 18 章测试时间。

## 5. 附录：定时器与「人工催办」

**非中断边界** 可触发通知（短信/邮件）而不取消原任务；**中断** 则进入升级路径。选型前先与业务确认「提醒是否算一次 SLA 违约」。

## 7. 章末提要（面向推广口播）

定时器章连接 **业务日历** 与 **工程时钟**：业务嘴里的「三个工作日」与 `PT72H` 不是同义词。推广时务必拉 HR/法务/客服一起对齐「**超时是否免责**」。运维则盯 **NTP 与作业积压**。测试用假时钟或短周期在 CI 复现，别用手「等三天」。**墙上可以贴：先对齐语义，再画边界事件。**

## 8. 深度追问（写给半年后的自己）

1. 多时区用户 **统一展示**规则是否存在？  
2. 夏令时切换的那一周 **回归集**是否覆盖？  
3. 若作业积压导致定时器 **整体漂移**，业务承诺如何调整？  
4. 「提醒」与「升级」是否 **同一 SLA 违约**？  
5. **节假日表**数据源是否权威？变更审批？  
6. 定时器相关 **审计字段**是否进入报表？  

**补白**：SLA 是合同；BPMN 只能表达技术语义，把合同附在需求后。
