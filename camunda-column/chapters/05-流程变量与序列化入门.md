## 1. 项目背景

网关条件读 `days`，服务任务读 `orderId`，历史审计想看到「当时客户快照」——这些都落在**流程变量**上。变量一乱，就会出现「本地能跑、上测试环境反序列化失败」「改个 DTO 类名历史实例全挂」。本章要解决的**一条主线问题**是：理解变量的**类型与存储形式**、**作用域（全局/局部）** 入门，以及 **JSON/XML/Java 对象** 在 Camunda 中的常见序列化风险与规避策略（自定义 SPI 见第 29 章）。

## 2. 项目设计（三角色对话）

### 2.1 小胖开球

小胖说：「变量不就是 Map 吗？我塞个 `User` 对象进去最省事。」

省事往往埋雷：**类结构变更**、**引擎与业务代码版本不一致**时，历史库里的字节流可能无法还原。

### 2.2 小白追问

小白问：「第一，**流程实例范围**和**执行范围**变量差别是什么？第二，大 JSON 字符串和 Java 序列化二进制哪个更可维护？第三，敏感信息（身份证）能放变量吗？」

### 2.3 大师定调

大师建议：

- **优先瘦数据**：变量里放 **id + 决策所需最小字段**；详情通过业务服务按 id 查询。
- **文本型 JSON**：可读、可版本迁移；配合 schema 或约定字段演进。
- **作用域**：全局变量在整个实例可见；局部变量绑定子执行路径，利于并行分支隔离（细节用时再查 API）。
- **合规**：敏感数据走脱敏或外部保险库；变量表备份与审计策略要前置（第 28 章）。

### 2.4 命名规范：别用拼音缩写满天飞

小胖问：「变量叫 `a`、`tmp` 行不行？」大师答：短期行，长期团队痛苦。建议：`camelCase` + **业务前缀**（`ord_`/`app_`）或统一词典；与网关、REST 字段对齐，减少映射层。

## 3. 项目实战

### 3.1 环境前提

- 可运行引擎；会调用 `RuntimeService#setVariable` 或通过启动参数传入变量。

### 3.2 步骤说明

1. 启动流程时传入 `days`（Integer）、`reason`（String）。
2. 在用户任务完成时写入 `approved`（Boolean）。
3. 在网关中用 `${days > 3 && approved}` 这类表达式验证类型是否匹配。
4. 故意制造一次**类型不匹配**（字符串 `"3"` 当数字用），观察异常信息并记录排障路径。
5. 将变量改为 **JSON 字符串** 存一个简单结构，完成同样网关判断（使用 Spin 或字符串解析，按项目规范）。

### 3.3 源码与说明

启动时变量（REST 示意）：

```json
{
  "variables": {
    "days": { "value": 5, "type": "Integer" },
    "reason": { "value": "年假", "type": "String" }
  }
}
```

**为什么显式 type**：减少引擎推断歧义；自动化测试时更稳定。

Java 写入：

```java
runtimeService.setVariable(executionId, "approved", Boolean.TRUE);
```

**为什么注意 executionId**：并行场景下变量写到错误的执行路径会导致「看不见变量」类问题。

### 3.4 验证

- Cockpit 中变量列表与预期一致；网关按预期分支。
- 单测断言变量类型与表达式求值（第 18 章）。

### 3.5 变量「体检」脚本思路

周期性抽样导出实例变量大小 Top N；超阈告警。可与 DBA 表空间增长关联分析（第 27 章）。

## 4. 项目总结

| 维度 | 内容 |
|------|------|
| 优点 | 变量是流程与业务数据桥梁；可读可测。 |
| 缺点 / 代价 | 滥用大对象拖慢 DB、升级困难。 |
| 适用场景 | 驱动网关、传递业务键、记录审批结果。 |
| 不适用场景 | 存完整业务聚合根长期真相源。 |
| 注意事项 | 命名规范、类型稳定、敏感信息治理。 |
| 常见踩坑 | 类重构导致反序列化失败；并行分支覆盖变量；变量过大。 |

**延伸阅读**：第 24 章踩坑合集；第 29 章自定义序列化。

## 5. 附录：敏感字段处理

身份证、手机号等：**展示脱敏**、**日志脱敏**、**权限最小化**；必要时仅存 **hash** 或外置令牌，流程里不落地明文。

## 7. 章末提要（面向推广口播）

变量是「**流程的短期记忆**」：记得清业务才顺，记得多团队才痛。牢记「瘦、显式、可测」：字段少、类型稳、单测覆盖表达式。**把变量当万能数据库的那一天，就是技术债爆表的前一天。**
