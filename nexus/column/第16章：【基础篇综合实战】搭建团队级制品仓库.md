# 第16章：【基础篇综合实战】搭建团队级制品仓库

## 1. 项目背景

云鲸科技新成立了一个 30 人的"天枢平台"研发团队，下设 Java 后端组（15 人）、前端组（8 人）、DevOps 组（5 人）和测试组（2 人）。团队负责人老周找到大师："我们这个新团队需要一套制品仓库方案——Java 要 Maven 私服、前端要 npm 私服、DevOps 要 Docker Registry 还要存安装包和脚本。但我只有一台 4C8G 的云主机，运维只有一个人，预算为零。能不能一周内把全套方案跑起来？"

过去 15 章零散地学了很多——Docker 部署、仓库创建、Maven/npm/Docker/Raw 配置、权限管理、API 脚本、搜索定位、清理策略、备份恢复、故障排查。每一章是一个独立的能力点，但还差最后一步：**把 15 个点串成一整套可交付的方案**。

本章将扮演"总工程师"的角色，从前 15 章的能力工具箱中挑选零件，拼装出一台可运转的团队级制品仓库：从需求拆解到仓库矩阵设计，从一键初始化脚本到客户端配置模板，从清理策略到备份方案，从排障手册到验收清单——输出一份完整交付物，让新人拿着这份文档就能独立维护。

## 2. 项目设计

大师召集了天枢平台的核心成员——老周、小胖、阿玲和浩子。

**大师**："天枢平台的需求是四个格式——Maven、npm、Docker、Raw，一个六人以上团队共享。我们先从仓库矩阵开始，然后建权限模型，再配清理策略和备份。"

**老周**："我不太确定仓库数量——是不是每种格式都要 hosted+proxy+group 三个？"

**大师**："标准答案是每格式一套 hosted+proxy+group，共 12 个仓库。但天枢平台有特殊性——你们是独立团队，不需要和其他团队共享仓库。所以可以精简：每种格式的 hosted 只建一个（不需要按团队拆分），proxy 每种格式一个，group 每种格式一个。"

**小胖**："那权限模型呢？又要分团队又要分格式，会不会很复杂？"

**大师**："用角色分层来简化。先建三个基础角色——`role-developer-read`（所有仓库只读）、`role-developer-write-java`（Java 仓库读写）、`role-developer-write-fe`（前端仓库读写）——再按实际岗位组合分配给每个人。运维单独 `role-ops-full`，测试单独 `role-tester-read`。CI 机器人每种格式一个专用账号。"

**浩子**："清理策略怎么定？我怕设太激进把有用版本删了。"

**大师**："分三层。第一层——SNAPSHOT 和临时构建：保留 14 天，再久没有价值。第二层——pre-release（alpha/beta/rc）：30 天未下载自动清理。第三层——正版 RELEASE：永远不清。Docker 再加一条——commit-hash 标签保留 7 天。这些策略先建在独立的测试仓库上演练，确认无误再绑到生产仓库。"

**阿玲**："备份呢？我们这台云主机自带云盘快照，还用做应用级备份吗？"

**大师**："快照是底层存储的 crash-consistent 还原——它不知道 Nexus 的数据库在快照时是否一致。所以流程是：先 Freeze Nexus（暂停写入），等 30 秒确保事务落盘，再触发云盘快照，最后 Unfreeze。这样就结合了应用级一致性保证和快照的恢复速度。"

> **技术映射**：综合方案 = 仓库矩阵（存储层）+ 权限模型（安全层）+ 客户端配置（接入层）+ 清理策略（治理层）+ 备份恢复（容灾层）。五层设计是最小可用生产方案。

## 3. 项目实战

### 3.1 环境准备

- 一台 Linux 云主机（CentOS 7+/Ubuntu 20.04+，4C8G，100GB 磁盘）
- Docker 20.10+ & Docker Compose v2+
- curl、jq
- 本章所有脚本文件存于 `~/nexus-platform/` 目录

### 3.2 分步实战

#### 步骤一：部署 Nexus 并初始化

**目标**：用 Docker Compose 部署 Nexus，预留所有需要的端口。

```bash
mkdir -p ~/nexus-platform && cd ~/nexus-platform

cat > docker-compose.yml << 'EOF'
version: "3.8"
services:
  nexus:
    image: sonatype/nexus3:3.70.0
    container_name: nexus
    restart: unless-stopped
    ports:
      - "8081:8081"   # Web UI & API
      - "5000:5000"   # Docker hosted
      - "5001:5001"   # Docker group
    volumes:
      - ./data:/nexus-data
    environment:
      - INSTALL4J_ADD_VM_PARAMS=-Xms1024m -Xmx2048m -XX:MaxDirectMemorySize=512m
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:8081/service/rest/v1/status || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 10
      start_period: 180s
    ulimits:
      nofile: {soft: 65536, hard: 65536}
EOF

sudo chown -R 200:200 ./data
docker compose up -d

# 等待就绪
sleep 120
# 获取初始密码
docker compose exec nexus cat /nexus-data/admin.password
```

#### 步骤二：一键初始化——仓库矩阵 + BlobStore + Realm

**目标**：创建 `init-platform.sh` 脚本，一键完成所有仓库和基础配置。

```bash
#!/bin/bash
# init-platform.sh：天枢平台 Nexus 一键初始化
NEXUS="http://localhost:8081"
AUTH="admin:admin123"
NEW_ADMIN_PASS="CloudWhale@2025"

echo "=== 天枢平台 Nexus 初始化 ==="

# 1. 修改 admin 密码
echo "[1/8] 修改 admin 密码..."
curl -u $AUTH -X PUT "$NEXUS/service/rest/v1/security/users/admin" \
  -H "Content-Type: application/json" \
  -d "{\"userId\":\"admin\",\"firstName\":\"Admin\",\"lastName\":\"User\",\"email\":\"admin@tianshu.com\",\"password\":\"${NEW_ADMIN_PASS}\",\"status\":\"active\",\"roles\":[\"nx-admin\"]}"
AUTH="admin:${NEW_ADMIN_PASS}"

# 2. 禁用匿名访问
echo "[2/8] 禁用匿名访问..."
curl -u $AUTH -X PUT "$NEXUS/service/rest/v1/security/anonymous" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'

# 3. 启用 Realms
echo "[3/8] 启用认证 Realm..."
curl -u $AUTH -X PUT "$NEXUS/service/rest/v1/security/realms/active" \
  -H "Content-Type: application/json" \
  -d '["NexusAuthenticatingRealm","NexusAuthorizingRealm","NpmToken","DockerToken"]'

# 4. 创建 BlobStore（容量隔离）
echo "[4/8] 创建 BlobStore..."
for BS in maven npm docker raw; do
  curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/blobstores/file" \
    -H "Content-Type: application/json" \
    -d "{\"name\":\"blob-${BS}\",\"path\":\"${BS}\"}"
done

# 5. 创建仓库（Maven + npm + Docker + Raw）
echo "[5/8] 创建仓库..."
# Maven
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/maven/hosted" -H "Content-Type: application/json" \
  -d '{"name":"maven-releases","online":true,"storage":{"blobStoreName":"blob-maven","writePolicy":"ALLOW_ONCE"},"maven":{"versionPolicy":"RELEASE","layoutPolicy":"STRICT"}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/maven/hosted" -H "Content-Type: application/json" \
  -d '{"name":"maven-snapshots","online":true,"storage":{"blobStoreName":"blob-maven","writePolicy":"ALLOW"},"maven":{"versionPolicy":"SNAPSHOT","layoutPolicy":"STRICT"}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/maven/proxy" -H "Content-Type: application/json" \
  -d '{"name":"maven-central","online":true,"storage":{"blobStoreName":"blob-maven"},"proxy":{"remoteUrl":"https://repo1.maven.org/maven2/","contentMaxAge":-1,"metadataMaxAge":1440},"maven":{"versionPolicy":"RELEASE","layoutPolicy":"PERMISSIVE"},"negativeCache":{"enabled":true,"timeToLive":1440},"httpClient":{"blocked":false,"autoBlock":true,"connection":{"retries":3,"timeout":60}}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/maven/group" -H "Content-Type: application/json" \
  -d '{"name":"maven-public","online":true,"storage":{"blobStoreName":"blob-maven"},"group":{"memberNames":["maven-releases","maven-snapshots","maven-central"]}}'

# npm
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/npm/hosted" -H "Content-Type: application/json" \
  -d '{"name":"npm-hosted","online":true,"storage":{"blobStoreName":"blob-npm","writePolicy":"ALLOW_ONCE"}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/npm/proxy" -H "Content-Type: application/json" \
  -d '{"name":"npm-proxy","online":true,"storage":{"blobStoreName":"blob-npm"},"proxy":{"remoteUrl":"https://registry.npmjs.org","contentMaxAge":-1,"metadataMaxAge":1440},"negativeCache":{"enabled":true,"timeToLive":1440},"httpClient":{"blocked":false,"autoBlock":true,"connection":{"retries":3,"timeout":60}}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/npm/group" -H "Content-Type: application/json" \
  -d '{"name":"npm-public","online":true,"storage":{"blobStoreName":"blob-npm"},"group":{"memberNames":["npm-hosted","npm-proxy"]}}'

# Docker
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/docker/hosted" -H "Content-Type: application/json" \
  -d '{"name":"docker-hosted","online":true,"storage":{"blobStoreName":"blob-docker","writePolicy":"ALLOW"},"docker":{"v1Enabled":false,"forceBasicAuth":true,"httpPort":5000}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/docker/proxy" -H "Content-Type: application/json" \
  -d '{"name":"docker-hub","online":true,"storage":{"blobStoreName":"blob-docker"},"proxy":{"remoteUrl":"https://registry-1.docker.io","contentMaxAge":1440,"metadataMaxAge":1440},"docker":{"v1Enabled":false,"forceBasicAuth":true},"httpClient":{"blocked":false,"autoBlock":true,"connection":{"retries":3,"timeout":120}},"negativeCache":{"enabled":true,"timeToLive":1440}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/docker/group" -H "Content-Type: application/json" \
  -d '{"name":"docker-public","online":true,"storage":{"blobStoreName":"blob-docker"},"group":{"memberNames":["docker-hosted","docker-hub"]},"docker":{"v1Enabled":false,"forceBasicAuth":true,"httpPort":5001}}'

# Raw
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/raw/hosted" -H "Content-Type: application/json" \
  -d '{"name":"raw-hosted","online":true,"storage":{"blobStoreName":"blob-raw","writePolicy":"ALLOW"}}'
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/repositories/raw/group" -H "Content-Type: application/json" \
  -d '{"name":"raw-public","online":true,"storage":{"blobStoreName":"blob-raw"},"group":{"memberNames":["raw-hosted"]}}'

# 6. 创建用户和角色
echo "[6/8] 创建用户和角色..."
# 简化版：为每个团队创建基本角色
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/security/roles" -H "Content-Type: application/json" \
  -d '{"id":"role-tianshu-dev","name":"平台开发者","privileges":["nx-repository-view-maven2-*-read","nx-repository-view-maven2-*-add","nx-repository-view-npm-*-read","nx-repository-view-npm-*-add","nx-repository-view-docker-*-read","nx-repository-view-raw-*-read"],"roles":[]}'

curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/security/roles" -H "Content-Type: application/json" \
  -d '{"id":"role-tianshu-readonly","name":"平台只读","privileges":["nx-repository-view-maven2-*-read","nx-repository-view-npm-*-read","nx-repository-view-docker-*-read","nx-repository-view-raw-*-read"],"roles":[]}'

# CI 机器人
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/security/users" -H "Content-Type: application/json" \
  -d '{"userId":"ci-bot","firstName":"CI","lastName":"Robot","email":"ci@tianshu.com","password":"C1B0tTianshu!","status":"active","roles":["role-tianshu-dev"]}'

# 7. 创建清理策略
echo "[7/8] 创建清理策略..."
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/cleanup-policies" -H "Content-Type: application/json" \
  -d '{"name":"cleanup-snapshots","format":"maven2","notes":"清除14天前的SNAPSHOT","criteriaLastBlobUpdated":14,"criteriaReleaseType":"SNAPSHOT"}'

# 8. 创建维护任务
echo "[8/8] 创建维护任务..."
curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/tasks" -H "Content-Type: application/json" \
  -d '{"action":"repository.cleanup","name":"每周清理过期制品","typeId":"repository.cleanup","schedule":"weekly","properties":{"repositoryName":"*"},"alertEmail":"ops@tianshu.com"}'

curl -s -u $AUTH -X POST "$NEXUS/service/rest/v1/tasks" -H "Content-Type: application/json" \
  -d '{"action":"blobstore.compact","name":"每周压缩BlobStore","typeId":"blobstore.compact","schedule":"weekly","properties":{"blobstoreName":"*"},"alertEmail":"ops@tianshu.com"}'

echo ""
echo "=== 初始化完成 ==="
echo "登录地址: http://$(hostname -I | awk '{print $1}'):8081"
echo "管理员: admin / $NEW_ADMIN_PASS"
```

#### 步骤三：客户端配置模板生成

**目标**：生成 Maven、npm、Docker 的客户端配置模板。

```bash
#!/bin/bash
# gen-client-configs.sh：生成客户端配置模板
NEXUS_HOST="192.168.1.100"  # 替换为实际 IP
NEXUS_USER="ci-bot"
NEXUS_PASS="C1B0tTianshu!"

mkdir -p ~/nexus-platform/client-configs
cd ~/nexus-platform/client-configs

# 1. Maven settings.xml 模板
cat > settings.xml << XEOF
<?xml version="1.0" encoding="UTF-8"?>
<settings>
  <servers>
    <server>
      <id>nexus-releases</id>
      <username>${NEXUS_USER}</username>
      <password>${NEXUS_PASS}</password>
    </server>
    <server>
      <id>nexus-snapshots</id>
      <username>${NEXUS_USER}</username>
      <password>${NEXUS_PASS}</password>
    </server>
  </servers>
  <mirrors>
    <mirror>
      <id>nexus-public</id>
      <mirrorOf>*</mirrorOf>
      <url>http://${NEXUS_HOST}:8081/repository/maven-public/</url>
    </mirror>
  </mirrors>
</settings>
XEOF

# 2. npm .npmrc 模板
AUTH_BASE64=$(echo -n "${NEXUS_USER}:${NEXUS_PASS}" | base64)

cat > .npmrc << NEOF
registry=http://${NEXUS_HOST}:8081/repository/npm-public/
//${NEXUS_HOST}:8081/repository/npm-public/:_auth=${AUTH_BASE64}
NEOF

# 3. Docker 配置
echo "Docker 客户端配置:"
echo "1. 编辑 /etc/docker/daemon.json:"
echo '{'
echo '  "insecure-registries": ["'${NEXUS_HOST}':5000","'${NEXUS_HOST}':5001"]'
echo '}'
echo ""
echo "2. 登录:"
echo "docker login ${NEXUS_HOST}:5000 -u ${NEXUS_USER} -p ${NEXUS_PASS}"

echo "配置文件已生成在: $(pwd)"
```

#### 步骤四：验收测试脚本

**目标**：编写自动化验收脚本，验证四类制品均可正常上传下载。

```bash
#!/bin/bash
# verify-platform.sh：验收天枢平台 Nexus 部署
NEXUS="http://192.168.1.100:8081"
AUTH="ci-bot:C1B0tTianshu!"
PASS=0; FAIL=0

check() {
    local desc="$1"; local expected="$2"; local actual="$3"
    if echo "$actual" | grep -q "$expected"; then
        echo "✅ $desc"
        ((PASS++))
    else
        echo "❌ $desc (期望包含: $expected)"
        echo "   实际: $actual"
        ((FAIL++))
    fi
}

echo "=== 天枢平台 Nexus 验收测试 ==="

# 1. 状态
STATUS=$(curl -sf -u $AUTH "$NEXUS/service/rest/v1/status" | jq -r '.status')
check "服务运行" "running" "$STATUS"

# 2. 仓库列表
REPOS=$(curl -sf -u $AUTH "$NEXUS/service/rest/v1/repositories" | jq -r '.[].name' | wc -l)
check "仓库数量>10" "" "$(test $REPOS -ge 10 && echo 'ok' || echo 'fail')"

# 3. Maven 上传下载
echo "test" > /tmp/verify-maven.txt
HTTP=$(curl -sf -u $AUTH -X PUT -w "%{http_code}" -o /dev/null \
  "$NEXUS/repository/maven-releases/com/test/verify/1.0/verify-1.0.jar" --data-binary @/tmp/verify-maven.txt)
check "Maven上传" "201" "$HTTP"

HTTP=$(curl -sf -u $AUTH -o /dev/null -w "%{http_code}" \
  "$NEXUS/repository/maven-public/com/test/verify/1.0/verify-1.0.jar")
check "Maven下载" "200" "$HTTP"

# 4. npm 上传（模拟）
echo '{"name":"@tianshu/verify","version":"1.0.0"}' > /tmp/package.json
HTTP=$(curl -sf -u $AUTH -X PUT -w "%{http_code}" -o /dev/null \
  "$NEXUS/repository/npm-hosted/@tianshu/verify" -H "Content-Type: application/json" -d @/tmp/package.json || echo "404_or_405")
check "npm仓库可访问" "" "$(test "$HTTP" != "000" && echo 'ok' || echo 'fail')"

# 5. Raw 上传下载
echo "raw-test" > /tmp/verify-raw.txt
HTTP=$(curl -sf -u $AUTH -X PUT -w "%{http_code}" -o /dev/null \
  "$NEXUS/repository/raw-hosted/test/verify.txt" --data-binary @/tmp/verify-raw.txt)
check "Raw上传" "201" "$HTTP"

# 6. 权限
HTTP=$(curl -s -o /dev/null -w "%{http_code}" "$NEXUS/repository/maven-public/")
check "匿名访问已禁用" "401" "$HTTP"

echo ""
echo "=== 验收完成: 通过 $PASS / 失败 $FAIL ==="
```

### 3.3 验收通过标准

| 验收项 | 通过条件 | 验证方法 |
|--------|---------|---------|
| Maven 上传 | `mvn deploy` 成功 | 执行 verify-platform.sh |
| Maven 下载 | `mvn compile` 通过 Proxy 缓存 | 清除 ~/.m2 后编译 |
| npm 发布 | `npm publish` 成功 | ci-bot 账号发布测试包 |
| Docker push/pull | 推送和拉取均成功 | docker login + push + pull |
| Raw 上传下载 | curl PUT/GET 返回 200 | verify-platform.sh |
| 权限隔离 | 匿名用户返回 401, 非授权用户 403 | curl 无认证访问 |
| 备份恢复 | 从备份恢复后仓库可用 | 恢复演练脚本 |

### 4.3 注意事项

- **密码不入库**：`init-platform.sh` 中的密码应替换为环境变量或 Vault 引用，不要提交到 Git
- **IP 地址替换**：模板中的 `192.168.1.100` 需替换为实际 Nexus 服务器地址
- **Docker insecure-registries**：生产环境应使用 HTTPS，不用 insecure 模式
- **首次部署等待**：Nexus 首次启动可能需要 3-5 分钟，`init-platform.sh` 需在 Nexus 完全就绪后执行

### 4.4 思考题

1. 天枢平台半年后扩张到 80 人，新增了 Python 组（需要 PyPI 私服）。在不重建已有仓库的前提下，如何在当前方案中扩展 PyPI 支持？需要新增哪些仓库、角色和清理策略？
2. 现有方案中所有仓库都绑定了独立的 BlobStore，但 Docker 的 BlobStore 在三个月后增长到 80GB，而 Maven 的只有 5GB。如何动态调整 BlobStore 的容量分配而不影响运行中的服务？

（第15章思考题答案：1. 可能的根因：① 客户端 settings.xml 的 mirror url 指向了 group 仓库但 group 中没有包含正确的 hosted 仓库，导致请求路由不到文件——验证：检查 group 成员列表；② Browse 页面的路径与 Maven 下载路径不一致（如 Browse 中文件在 maven-releases 仓库，但 maven-public URL 中 group 成员顺序将 maven-releases 排在了最后，先命中了 proxy 返回 404）——验证：直接访问 hosted 仓库的 URL 确认文件是否存在；③ Browse 页面显示了文件但实际 Asset Blob 已损坏或软删除——验证：检查 component 详情中 asset 是否可以下载。2. 自愈系统设计：① 定时任务（每分钟）检查 `/service/rest/v1/status` 的 `frozen` 字段；② 检测到 frozen 后，等待 2 分钟确认非临时冻结；③ 自动执行诊断脚本检查磁盘使用率；④ 如果磁盘 > 95%，触发 `docker compose exec nexus find /nexus-data/log -mtime +7 -delete` 清理旧日志 → 执行 Compact BlobStore → 检查磁盘是否恢复；⑤ 如果 frozen 超 10 分钟且磁盘正常，发 P0 告警（企业微信/钉钉/PagerDuty），附带诊断报告；⑥ 所有自动操作记录到操作日志，事后可审计。）
