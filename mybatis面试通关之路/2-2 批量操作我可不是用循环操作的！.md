```java
		( `first_name`, `last_name`, `age`) 
		VALUES
		(#{firstName},#{lastName},#{age})	
	</insert>
	
	<insert id="insertBatchStudentInfo" parameterType="com.davidwang456.mybatis.batch.StudentDTO">
		INSERT INTO `student` 
		( `first_name`, `last_name`, `age`) 
		VALUES
    <foreach collection ="dtos" item="dto" separator =",">
         (#{dto.firstName}, #{dto.lastName}, #{dto.age})
    </foreach >
</insert>
</mapper>
```

```java
package com.davidwang456.mybatis.batch;

import java.util.List;
```

## 总结

**1.方式不同，性能差异巨大**

注：环境不同可能会引起耗时存在差异。

| 插入方式                  | 耗时(毫秒) | 说明                             |
| ------------------------- | ---------- | -------------------------------- |
| 方式1：单条循环插入方式   | 146245     | 没有使用批量方式                 |
| 方式2：批量+循环方式      | 77334      | 使用批量方式                     |
| 方式3：foreach拼凑sql插入 | 失败       | 使用批量方式                     |
| 方式4：foreach拼凑sql改良 | 11958      | 批量方式，单次批量插入1000条记录 |

从数据来看，方式4效率最高，但要考虑mysql查询时max_allowed_packet的大小，批量插入的记录数收到不同数据表的限制。

**2.批量的使用方式不限于插入**

不仅仅时插入，查询，更新，删除都可以使用批量来提高效率！

**3.不同数据库不同量级的数据处理方式不同**

上面针对的是mysql的百万级别的数据，如果是小级别的如百千万，方式3也许会更好。故要因地制宜的使用不同的方式。

**4.超大量的数据可以解决大数据处理数据的思路：分而治之**

如上面描述的，数据量超过亿级别，使用mysql 本身提供的load命令也是一个可选项。

**5.大数据量数据时要考虑索引对数据的影响**

  如插入数据时，如果表中有索引，尽量先删除掉索引在插入数据，避免频繁的索引操作影响插入的性能。
