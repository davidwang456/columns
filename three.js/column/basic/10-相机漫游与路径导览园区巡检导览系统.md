# 第10章：相机漫游与路径导览园区巡检导览系统

## 元信息
- 学习目标：实现路径巡航、节点停靠、手动接管，构建可用于讲解和巡检的导览系统。
- 先修要求：掌握相机控制和事件联动。
- 预估时长：4-6 小时。
- 最终产出：可配置路径的自动导览模块。
- 适用角色：开发、测试、运维。

## 1. 项目背景（约500字）

园区大屏的一个高频场景是“值班交接讲解”：需要按固定顺序带新人看园区入口、重点设备区、告警热点区、应急通道。如果靠人工拖拽相机，讲解路径不稳定，重点区域容易漏掉；如果只靠固定截图，又失去三维场景价值。业务方希望有“自动讲解 + 人工接管”的巡检导览。

实现漫游看似简单，真正难点在体验一致性：速度不能忽快忽慢、转向不能突兀、停靠点要能稳定看清目标。若设计不当，很容易让用户晕眩。工程上还要解决“自动巡航与手动操作冲突”问题：用户一旦拖拽，系统应及时交出控制权；巡航恢复时要平滑衔接。

本章将建立路径导览的工程框架：路径节点定义、相机插值、目标点朝向、状态切换。做到既可用于业务演示，也可用于生产值班巡检。

## 2. 项目设计（剧本式交锋对话，约1200字）

### 第一轮：路径点怎么定义

小胖：  
“每次想讲哪里就手动飞过去，不更灵活吗？”

小白：  
“手动太依赖操作员。我们需要可复现路径，便于培训和回归测试。”

大师：  
“路径点配置化，讲解才能标准化。像地铁线路图，不是司机临场决定。”  

技术映射：  
“路径标准化” = `waypoint configuration`。

### 第二轮：转场如何平滑

小胖：  
“我直接把相机位置设到下一点，快准狠。”

小白：  
“硬切会跳变，用户会失去空间方向感。”

大师：  
“用插值和缓动，位置与朝向同步过渡。转场是叙事，不是瞬移。”  

技术映射：  
“平滑转场” = `lerp/slerp + easing`。

### 第三轮：人工接管怎么做

小胖：  
“用户一拖拽就打断巡航，之后咋办？”

小白：  
“是不是用状态机：auto、manual、resume？”

大师：  
“正解。明确状态才不会冲突，恢复巡航时从最近节点续航。”  

技术映射：  
“状态机” = `cameraTourStateMachine`。

## 3. 项目实战（约1500-2000字）

### 3.1 环境准备

```bash
npm install three
```

### 3.2 分步实现

#### 步骤1：定义巡航路径

```ts
type Waypoint = {
  position: THREE.Vector3;
  target: THREE.Vector3;
  stay: number;
};

const tourPath: Waypoint[] = [
  { position: new THREE.Vector3(18, 12, 18), target: new THREE.Vector3(0, 0, 0), stay: 1.2 },
  { position: new THREE.Vector3(8, 6, 10), target: new THREE.Vector3(2, 0, 0), stay: 1.5 },
  { position: new THREE.Vector3(-6, 5, 8), target: new THREE.Vector3(-3, 0, 1), stay: 1.2 }
];
```

结果：巡检线路可配置。  
坑：节点过密会造成频繁小幅抖动。

#### 步骤2：实现相机平滑过渡

```ts
function moveCamera(camera: THREE.PerspectiveCamera, from: Waypoint, to: Waypoint, t: number) {
  camera.position.lerpVectors(from.position, to.position, t);
  const look = new THREE.Vector3().lerpVectors(from.target, to.target, t);
  camera.lookAt(look);
}
```

结果：转场自然。  
坑：每帧创建 `Vector3` 可优化为复用变量。

#### 步骤3：接入状态机

```ts
type TourState = "auto" | "manual" | "resume";
let state: TourState = "auto";

controls.addEventListener("start", () => {
  state = "manual";
});

function resumeTour() {
  state = "resume";
  // 计算最近节点，平滑回到路径
}
```

结果：用户可随时接管并恢复巡航。  
坑：恢复时不做过渡会瞬间跳镜头。

## 4. 项目总结（约500-800字）

### 优点
1. 演示流程标准化，培训成本低。  
2. 讲解重点可复现，便于测试。  
3. 自动与手动兼容，用户体验友好。

### 缺点
1. 路径维护需要和业务场景同步更新。  
2. 转场参数调优有一定成本。  
3. 复杂地形仍需碰撞或避障策略。

### 常见故障案例
1. 相机穿模：路径点未避开建筑。  
2. 晕眩感强：速度和 FOV 变化过快。  
3. 恢复错位：最近节点计算不准确。

### 思考题
1. 如何为导览加入语音播报与字幕同步？  
2. 如何支持“按告警优先级动态重排巡航路径”？
