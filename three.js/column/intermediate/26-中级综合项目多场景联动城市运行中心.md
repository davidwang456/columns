# 第26章：中级综合项目多场景联动城市运行中心

## 元信息
- 学习目标：综合中级篇能力，交付多场景联动的“城市运行中心”准生产方案。
- 先修要求：完成中级篇 14-25 章。
- 预估时长：8-12 小时。
- 最终产出：含实时同步、权限、可观测、测试门禁的中级综合工程。
- 适用角色：开发、测试、运维。

## 1. 项目背景（约500字）

基础篇解决了“单场景可展示”，中级篇目标是“多场景可运营”。城市运行中心是典型需求：交通、能源、安防、环保四大场景要在同一平台联动，支持实时告警、跨场景跳转、角色权限控制和多端访问。系统不仅要看得见，还要“稳得住、查得到、控得住”。

从工程角度，这类项目挑战是组合复杂度：架构分层、性能优化、流式加载、实时同步、可观测、权限与测试都要同时生效。只要有一个环节薄弱，整体质量就会掉线。比如实时链路稳定但权限策略缺失，仍无法上线；性能优秀但无自动化回归，迭代后也会退化。

本章把中级篇所有能力收束成一个综合交付模板，作为高级篇前的“准生产基线”。目标不是追求所有功能最强，而是形成可复制的交付闭环。

## 2. 项目设计（剧本式交锋对话，约1200字）

### 第一轮：先做哪个场景

小胖：  
“四个场景全做太大了，要不先做一个最酷的？”

小白：  
“应该先做共性底座，再接入场景插件，不然每个场景都重复造轮子。”

大师：  
“中台先行，场景后挂。先把基础设施做稳，场景扩展才有速度。”  

技术映射：  
“中台先行” = `platform-first architecture`。

### 第二轮：联动如何避免串扰

小胖：  
“交通告警触发后直接跳安防页面，简单粗暴。”

小白：  
“跨场景事件需要规范，不然会互相污染状态。”

大师：  
“定义跨域事件协议，所有联动都走总线和编排层。”  

技术映射：  
“跨域事件协议” = `orchestrated event bus`。

### 第三轮：如何证明可交付

小胖：  
“能演示就算完成？”

小白：  
“要有质量门禁：性能、稳定性、安全、测试都要达标。”

大师：  
“准生产交付看的是体系，不是单点炫技。”  

技术映射：  
“体系达标” = `production readiness checklist`。

## 3. 项目实战（约1500-2000字）

### 3.1 环境准备

```bash
npm create vite@latest chapter-26-city-center -- --template vanilla-ts
cd chapter-26-city-center
npm install three
npm install -D playwright
```

### 3.2 分步实现

#### 步骤1：搭建中台骨架

```ts
// app/bootstrap.ts
initStore();
initRealtime();
initRenderer();
initSecurity();
initObservability();
registerScenes(["traffic", "energy", "security", "environment"]);
```

结果：能力模块按职责装配。  
坑：初始化顺序混乱导致依赖空指针。

#### 步骤2：实现跨场景联动编排

```ts
// orchestrator/linkage.ts
onEvent("alarm:critical", (e) => {
  switchScene(e.payload.sceneId);
  focusTarget(e.payload.targetId);
  recordOpsLog(e);
});
```

结果：告警驱动跨场景联动。  
坑：联动链路无幂等控制导致重复跳转。

#### 步骤3：接入质量门禁

```ts
runSmokeChecks({
  fpsMin: 30,
  reconnectSuccessRate: 0.99,
  criticalPathPass: true
});
```

结果：上线前可量化验收。  
坑：只测功能不测稳定性，线上风险高。

### 3.3 完整代码清单

- `src/platform/*`：中台基础设施
- `src/scenes/*`：四大场景插件
- `src/orchestrator/*`：联动编排
- `src/security/*`：权限与脱敏
- `src/observe/*`：可观测与告警
- `tests/*`：可视与交互回归

### 3.4 测试验证

```bash
npm run build
npm run test:e2e
```

验证：
1. 多场景切换流畅且状态一致；
2. 告警联动可跨场景定位；
3. 无权角色无法访问敏感对象；
4. 关键路径自动化测试通过。

## 4. 项目总结（约500-800字）

### 优点
1. 中级篇能力形成闭环交付。  
2. 支持多团队并行开发与运维协作。  
3. 为高级篇源码与极限优化打好基础。

### 缺点
1. 架构复杂度明显提升。  
2. 对工程规范与测试体系依赖更强。  
3. 初期建设成本较高。

### 常见故障案例
1. 跨场景状态错乱：事件协议不统一。  
2. 高峰期卡顿：实时消息与渲染更新未解耦。  
3. 权限绕过：资源层签名校验缺失。

### 思考题
1. 如何把本项目演进为“多租户城市平台”？  
2. 如何在不影响联动时延的前提下引入更细粒度审计？
