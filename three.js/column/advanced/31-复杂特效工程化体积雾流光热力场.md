# 第31章：复杂特效工程化——体积雾、流光、热力场

## 元信息
- 学习目标：将复杂视觉特效从“单点 demo”升级为“可配置、可维护、可降级”的工程能力。
- 先修要求：完成 Shader 与自定义 Pass 章节。
- 预估时长：6-8 小时。
- 最终产出：统一特效编排层与三类特效组件（体积雾、流光、热力场）。
- 适用角色：资深开发、架构师。

## 1. 项目背景（约500字）

高级项目中，视觉特效往往承载业务价值：体积雾用于表达污染扩散，流光用于展示管线方向，热力场用于突出风险聚集区。问题在于，这些效果常以“临时脚本”方式接入，参数散落、性能不可控、版本升级易崩。上线后就会出现“某台机器很好看，另一台直接掉帧”的尴尬。

复杂特效的挑战不在“能不能做出来”，而在“能不能长期维护”。如果没有统一管理层，特效会互相叠加污染：同一对象被多个效果重复改材质、更新循环互相抢时间片、开关状态不一致。最终业务团队不敢再开新效果。

本章目标是建立特效工程化框架：统一 effect 生命周期（mount/update/unmount）、统一参数协议、统一预算与降级策略，并把三类典型特效纳入同一编排系统，做到“可开可关、可观测、可回滚”。

## 2. 项目设计（剧本式交锋对话，约1200字）

### 第一轮：特效就是写 shader 吗

小胖：

技术映射：  
“效果编排” = `effect orchestration layer`。

### 第二轮：效果多了会打架

小胖：  
“体积雾和热力场叠一起，颜色全糊了。”

小白：  
“需要定义叠加顺序和混合策略。”

“先定语义优先级，再定渲染顺序，最后定混合公式。”  

技术映射：  
“语义优先级” = `effect priority + blend policy`。

### 第三轮：如何保证线上稳定

小胖：  
“效果开了确实好看，但偶发掉帧。”

小白：  
“应有性能预算和自动降级触发器。”

大师：  
“特效必须有安全阀，超过预算自动降档。”  

技术映射：  
“安全阀” = `adaptive fallback controller`。

## 3. 项目实战（约1500-2000字）

### 3.1 环境准备

```bash
npm install three
```

### 3.2 分步实现

#### 步骤1：定义统一特效接口

```ts
interface EffectModule {
  id: string;
  mount(scene: THREE.Scene): void;
  update(dt: number): void;
  setEnabled(v: boolean): void;
  dispose(): void;
}
```

结果：特效模块可被统一调度。  
坑：每个特效各写一套生命周期，难管理。

#### 步骤2：实现三类效果模块

```ts
class HeatFieldEffect implements EffectModule {
  // 维护热力材质、数据纹理、更新时间
}

class FlowLineEffect implements EffectModule {
  // 沿路径推进纹理偏移或粒子
}

class VolumetricFogEffect implements EffectModule {
  // 使用噪声或分层片元近似体积雾
}
```

结果：效果职责分离，便于独立调优。  
坑：共享 uniforms 导致不同效果串值。

#### 步骤3：加预算与降级控制

```ts
function applyBudget(fps: number) {
  if (fps < 24) disable("VolumetricFogEffect");
  else if (fps < 30) setQuality("FlowLineEffect", "medium");
}
```

结果：高负载时可自动保护可用性。  
坑：降级策略无回升机制，效果长期关闭。

### 3.4 测试验证

```bash
npm run build
```

验证：
1. 三种效果可独立开关；
2. 低帧率触发降级并可恢复；
3. 长时间运行无明显资源泄漏。

## 4. 项目总结（约500-800字）

### 优点
1. 复杂特效可控可维护。  
2. 视觉与性能可平衡。  
3. 后续新增效果接入成本低。

### 缺点
1. 架构复杂度提升。  
2. 参数调优工作量大。  
3. 多机型一致性验证成本高。

### 常见故障案例
1. 特效串扰：共享状态未隔离。  
2. 突发掉帧：无动态降级机制。  
3. 内存累积：效果模块未完整 dispose。

### 思考题
1. 如何把效果参数接入在线配置中心并支持灰度？  
2. 如何建立“视觉收益/性能成本”评估模型？
