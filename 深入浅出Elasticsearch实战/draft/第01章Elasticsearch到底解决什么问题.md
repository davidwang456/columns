# 背景

# 第01章 Elasticsearch 到底解决什么问题

很多团队在业务增长后，会同时遇到两个痛点：数据“能存但难搜”、日志“有很多但难查”。这时 Elasticsearch 往往被引入，但如果不了解它的边界，很容易出现“用对了工具却放错了位置”。本章先解决“为什么用、什么时候用、什么时候不用”。

## 本章目标

读完本章，你应该能回答 3 个问题：

- Elasticsearch（简称 ES）最擅长解决什么问题？
- 什么场景不适合用 ES？
- 在你们团队里，什么时候应该把 ES 作为核心组件引入？

---

## 1. ES 的核心价值：快、准、可扩展的检索与分析

很多团队第一次接触 ES，往往是因为“数据库查得太慢”或“日志太多查不动”。  
本质上，ES 的价值可以概括为三点：

### 1.1 全文检索能力强

ES 基于倒排索引，不是逐行扫描文本，而是先把“词”建立索引，再反查文档。  
这使它在“按关键词找内容”时非常高效，典型场景：

- 商品搜索（按标题、描述、品牌搜）
- 知识库搜索（按关键词定位文档）
- 日志检索（按错误码、关键词定位异常）

### 1.2 近实时分析能力好

ES 不只是“搜”，也可以做聚合分析，比如：

- 按时间统计错误数量趋势
- 按地区统计订单分布
- 按接口统计响应时间分位数

它适合“边写入边查询边统计”的业务，不需要先离线跑批才能看到结果。

### 1.3 分布式扩展自然

数据变大后，ES 可以通过分片和副本横向扩容，常见收益：

- 数据量增长时可加节点扩容
- 副本提高可用性并分担读请求
- 故障时可自动恢复部分能力

---

## 2. ES 不是什么：别把它当万能数据库

ES 很强，但不是“全能替代品”。下面这些场景通常不适合：

### 2.1 强事务场景

如果你需要严格的事务一致性（例如银行转账、强一致库存扣减），  
优先考虑关系型数据库，ES 更适合“检索和分析侧”。

### 2.2 复杂联表与强关系建模

ES 的关联能力有限，跨多表复杂 join 不如 MySQL/PostgreSQL 自然。  
在 ES 中通常会通过“反范式建模”来换取查询速度，但写入复杂度会提高。

### 2.3 超高频小更新且强实时一致

ES 支持更新，但本质是“重写文档”。  
如果是非常频繁、强一致的小字段更新，代价会比较高。

---

## 3. 什么时候该引入 ES：一个简单判断框架

可以用下面 4 个问题快速判断：

1. 你的核心诉求是不是“搜索体验”或“检索速度”？  
2. 是否需要对大量数据做实时过滤、排序、聚合？  
3. 数据量增长后是否需要横向扩展？  
4. 能否接受“近实时”（秒级可见）而不是“强一致立即可见”？

如果前 3 个“是”，第 4 个也可接受，ES 通常是高匹配方案。

---

## 4. 团队常见落地模式（推荐）

在工程实践里，ES 最常见的位置是：

- **主库在关系型数据库**：承担事务写入与核心一致性
- **ES 做检索与分析副本**：通过同步链路（CDC/消息队列/批处理）更新索引

这样分层的好处：

- 事务问题交给主库
- 搜索性能交给 ES
- 职责清晰，系统稳定性更高

---

## 5. 一个直观例子：电商搜索系统

假设你要做“商品搜索”：

- 用户输入“轻薄笔记本 16G”
- 系统要支持：关键词检索、品牌筛选、价格区间、销量排序、聚合统计

如果只用关系型数据库，随着数据增大和查询复杂化，性能会明显下降。  
而 ES 可以通过索引和查询 DSL 更自然地支持这类检索体验。

但订单创建、支付扣款、库存强一致扣减，仍应放在事务数据库中处理。  
这就是“ES 擅长什么，不擅长什么”的边界。

---

# 总结

- ES 是检索与分析引擎，核心优势是全文检索、近实时分析、分布式扩展。
- ES 不适合强事务和复杂联表主场景。
- 最稳妥的架构通常是“主库做事务，ES 做检索分析”。

一句话总结：**把 ES 放在它最擅长的位置，你会得到极高性价比；放错位置，维护成本会快速上升。**

## 实战（curl）

```bash
# 1) 集群可用性自检
curl -u "$ES_USER:$ES_PASS" "$ES_URL/"

# 2) 体验“检索引擎”能力：创建简单索引并搜索
curl -u "$ES_USER:$ES_PASS" -X PUT "$ES_URL/ch01_demo" \
  -H "Content-Type: application/json" \
  -d '{"mappings":{"properties":{"title":{"type":"text"}}}}'
curl -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/ch01_demo/_doc/1?refresh=wait_for" \
  -H "Content-Type: application/json" \
  -d '{"title":"Elasticsearch 入门指南"}'
curl -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/ch01_demo/_search?pretty" \
  -H "Content-Type: application/json" \
  -d '{"query":{"match":{"title":"入门"}}}'
```
