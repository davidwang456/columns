# 背景

# 第19章 可观测性体系：指标、告警与诊断地图

没有可观测性的集群，只能“出事后猜原因”。监控体系的目标不是看图，而是缩短从告警到定位、再到修复的时间。

## 本章目标
- 建立 ES 的最小可观测体系。
- 让问题可以“早发现、快定位”。

## 1. 必看指标
- 搜索与写入延迟
- 吞吐与拒绝数
- Heap/GC 与线程池队列
- 磁盘水位与段合并压力

## 2. 告警设计原则
- 告警要与业务 SLO 对齐。
- 既关注绝对阈值，也关注趋势变化。

## 3. 诊断地图
按“集群 -> 索引 -> 查询”三层排查：
1) 集群是否健康  
2) 哪个索引异常  
3) 哪类查询/写入触发问题

## 4. 团队协作建议
- 告警要有负责人和处置手册。
- 复盘中补齐缺失指标与阈值。

# 总结
- 监控不是看图，而是支撑决策。
- 指标、告警、手册三者缺一不可。

## 练习题
1. 设计你们团队 ES 的 5 个核心告警。  
2. 为每个告警写出“首查项”和“处置动作”。  
3. 复盘一次历史故障，补齐监控盲区。

