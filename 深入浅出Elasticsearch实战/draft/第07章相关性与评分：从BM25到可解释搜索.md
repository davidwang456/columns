# 背景

# 第07章 相关性与评分：从 BM25 到可解释搜索

“为什么这条排第一？”是搜索系统最常见问题。没有可解释的评分体系，调优就只能靠猜。BM25 与业务加权是把“经验”变成“可复用规则”的关键。

## 本章目标
- 理解搜索排序为何如此。
- 能做基础相关性调优并解释结果变化。

## 1. BM25 的直觉理解
BM25 综合考虑：
- 词在文档中的重要程度
- 词在全库中的稀有程度
- 文档长度对评分的影响

## 2. 影响排序的常见因素
- 字段权重（boost）
- 精确匹配优先（phrase）
- 多字段匹配（multi_match）

## 3. 可解释性工具
- `explain`：解释单条命中文档分数来源。
- 对比不同查询结构，验证排序是否符合业务目标。

## 4. 调优原则
- 先保证“结果正确”，再追求“结果更优”。
- 尽量减少黑箱规则，保留可解释路径。

# 总结
- 排序质量是“算法 + 业务规则”的结合。
- 可解释性是长期维护搜索质量的关键。

## 练习题
1. 为标题、品牌、类目设置不同 boost 并对比结果。  
2. 用 `explain` 分析某条“排序不合理”的结果。  
3. 对比 `match` 与 `match_phrase` 的行为差异。  

## 实战（curl）

```bash
curl -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/$IDX/_search?pretty" \
  -H "Content-Type: application/json" \
  -d '{
    "explain": true,
    "query":{
      "multi_match":{
        "query":"轻薄 笔记本",
        "fields":["name^3","category"]
      }
    }
  }'
```

## 实战（Java SDK）

```java
var resp = client.search(s -> s.index("products_v1")
    .explain(true)
    .query(q -> q.multiMatch(mm -> mm.query("轻薄 笔记本").fields("name^3","category"))),
    Map.class);
System.out.println(resp.hits().hits().get(0).score());
```
