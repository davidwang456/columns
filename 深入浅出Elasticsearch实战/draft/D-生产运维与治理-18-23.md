# 模块D：生产运维与治理（18-23）

## 18 部署拓扑与容量规划
### 本讲目标
给出可执行的容量估算与节点规划。
### 核心内容
- 先估算日增量、保留期、副本数。
- 再反推磁盘、CPU、内存与节点数。
- 分片大小控制在可恢复、可迁移范围。
### 新手必读
容量规划不是一次性动作，要按增长持续复盘。
### 进阶补充
按业务类型拆分集群或索引生命周期。
### 小练习
为“每天 500GB 日志”输出 90 天容量方案。

## 19 可观测性：指标、告警与诊断
### 本讲目标
建立从症状到根因的监控闭环。
### 核心内容
- 关键看板：延迟、吞吐、拒绝数、GC、磁盘水位。
- 告警要和 SLO 绑定，不要只看单点阈值。
- 指标需要串联节点、索引、查询三个层级。
### 新手必读
先把“慢查询 + 拒绝 + 磁盘水位”三类告警配齐。
### 进阶补充
用趋势告警识别性能退化而非仅识别故障。
### 小练习
定义你们团队的 5 条核心 ES 告警规则。

## 20 常见故障与恢复流程
### 本讲目标
掌握红黄集群的标准排障路径。
### 核心内容
- 优先看 unassigned shards 与分配原因。
- 常见根因：磁盘水位、节点离线、分片限制。
- 恢复后要做复盘，防止再次发生。
### 新手必读
排障先止血再优化：恢复可用性优先。
### 进阶补充
为不同故障准备自动化 Runbook。
### 小练习
演练一次“磁盘满导致黄/红”并完成恢复报告。

## 21 升级与兼容：滚动升级策略
### 本讲目标
以最小风险完成版本升级。
### 核心内容
- 升级前先读 breaking changes。
- 优先滚动升级，保留回滚窗口。
- 客户端版本与协议兼容要提前验证。
### 新手必读
升级流程必须有演练环境与回滚脚本。
### 进阶补充
跨大版本升级要提前评估 reindex 成本。
### 小练习
写一份“当前版本 -> 目标版本”的升级 checklist。

## 22 安全与权限：最小权限实践
### 本讲目标
在保证安全的前提下不影响研发效率。
### 核心内容
- 用户、角色、索引权限要分离设计。
- 应用推荐用 API Key，减少账号暴露。
- 生产环境默认启用 TLS 与认证。
### 新手必读
一切权限按“最小可用”原则发放。
### 进阶补充
多租户场景要定义清晰的租户边界和审计策略。
### 小练习
给开发、运维、分析三类角色各定义最小权限。

## 23 备份与灾备：Snapshot 与恢复验证
### 本讲目标
确保“真的能恢复”，而不是“只是有备份”。
### 核心内容
- Snapshot 是 ES 的标准备份机制。
- 备份策略要覆盖频率、保留与加密。
- 定期恢复演练是灾备有效性的唯一证明。
### 新手必读
至少每月做一次恢复演练并记录耗时。
### 进阶补充
按 RPO/RTO 指标反推备份频率与架构。
### 小练习
制定你们团队的备份与恢复验收标准。
