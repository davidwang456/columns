# 模块B：建模与查询进阶（07-12）

## 07 相关性与评分：从 BM25 到可解释结果
### 本讲目标
理解“为什么这条结果排在前面”。
### 核心内容
- BM25 关注词频、逆文档频率与字段长度。
- 通过 `boost` 调整字段重要性。
- 用 `explain` 验证评分是否符合业务预期。
### 新手必读
先做可解释，再做复杂排序。
### 进阶补充
可在 `function_score` 中叠加业务特征。
### 小练习
给“标题/品牌/类目”设置不同权重并对比排序变化。

## 08 聚合（Aggregations）：分析能力与代价
### 本讲目标
能稳定输出统计报表并避免内存风险。
### 核心内容
- bucket 聚合做分组，metric 聚合做指标。
- 高基数字段聚合容易慢且占内存。
- 大结果集分页用 `composite` 更稳妥。
### 新手必读
先从 `terms + avg/sum` 组合入手。
### 进阶补充
估算聚合内存开销，控制字段基数。
### 小练习
实现“按类目统计商品数与均价”。

## 09 排序与分页：深分页治理
### 本讲目标
避免 `from + size` 深分页导致的慢查询。
### 核心内容
- 深分页会让每个分片保留大量候选结果。
- 推荐 `search_after + PIT` 做稳定翻页。
- 需要稳定排序字段（建议包含唯一键）。
### 新手必读
浅分页可用 from/size，深分页改 search_after。
### 进阶补充
PIT 要控制生命周期，避免资源泄漏。
### 小练习
实现一个不会重复/漏数的翻页接口。

## 10 多字段与多语言建模
### 本讲目标
一个字段同时满足检索、过滤、排序、聚合。
### 核心内容
- `fields` 支持 `text` 与 `keyword` 共存。
- `normalizer` 用于 keyword 规范化。
- 多语言可用多字段或多索引策略。
### 新手必读
优先使用“一个主字段 + 一个 keyword 子字段”。
### 进阶补充
跨语言检索要考虑分词一致性与召回率。
### 小练习
设计 `name` 字段支持全文检索和排序聚合。

## 11 Nested/Object 与关联建模
### 本讲目标
避免对象数组查询出现“错配命中”。
### 核心内容
- `object` 扁平化后可能丢失对象边界。
- `nested` 保留边界，查询更准确但成本更高。
- parent-child 适合弱关联但实现更复杂。
### 新手必读
对象数组字段默认优先评估是否需要 nested。
### 进阶补充
复杂关系优先业务反范式，减少查询负担。
### 小练习
用 object 与 nested 各做一次订单明细检索对比。

## 12 查询性能工具箱：profile/慢日志/explain
### 本讲目标
掌握定位慢查询的标准流程。
### 核心内容
- 慢日志定位“哪条慢”。
- `profile` 定位“慢在何处”。
- `explain` 定位“排序为何如此”。
### 新手必读
问题定位顺序：慢日志 -> profile -> mapping/query 修正。
### 进阶补充
建立查询基线，持续回归压测。
### 小练习
分析两条慢查询并给出可执行优化方案。
