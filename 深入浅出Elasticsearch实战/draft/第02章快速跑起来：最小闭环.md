# 背景

# 第02章 快速跑起来：最小闭环

团队开始用 ES 时，最常见的问题不是“性能不够”，而是“大家本地验证步骤不一致”。因此需要一套标准化最小闭环：同样的命令、同样的预期结果、同样的排查方式。

## 本章目标
- 跑通 Elasticsearch 最小闭环：建索引、写文档、查文档、删索引。
- 用一套可复制脚本，帮助团队统一本地验证方式。
- 建立“先正确、再优化”的工程习惯。

---

## 1. 什么是最小闭环

最小闭环就是用最少步骤确认系统可用：

1. 创建索引（带基础 mapping）  
2. 写入样例数据（单条 + 批量）  
3. 执行查询（检索 + 过滤 + 排序）  
4. 清理索引（保证测试可重复）

它的目标不是“高性能”，而是“确定性可运行”。

---

## 2. 准备工作

本章默认 ES 已启动，且你有可用账号。

- 默认地址：`http://localhost:9200`
- 如启用安全：使用 `elastic-admin:elastic-password`

建议先设置环境变量，后续脚本更简洁：

```bash
export ES_URL="http://localhost:9200"
export ES_USER="elastic-admin"
export ES_PASS="elastic-password"
export IDX="products_v1"
```

Windows PowerShell 可用：

```powershell
$env:ES_URL="http://localhost:9200"
$env:ES_USER="elastic-admin"
$env:ES_PASS="elastic-password"
$env:IDX="products_v1"
```

## 实战（curl）

> 下面所有命令都可直接执行。若你本地未开启安全，去掉 `-u "$ES_USER:$ES_PASS"` 即可。

### 3.1 创建索引（显式 mapping）

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X PUT "$ES_URL/$IDX" \
  -H "Content-Type: application/json" \
  -d '{
    "settings": {
      "number_of_shards": 1,
      "number_of_replicas": 0
    },
    "mappings": {
      "properties": {
        "id":         { "type": "keyword" },
        "name":       {
          "type": "text",
          "fields": { "keyword": { "type": "keyword" } }
        },
        "category":   { "type": "keyword" },
        "price":      { "type": "double" },
        "in_stock":   { "type": "boolean" },
        "created_at": { "type": "date" }
      }
    }
  }'
```

验证索引是否创建成功：

```bash
curl -sS -u "$ES_USER:$ES_PASS" "$ES_URL/_cat/indices/$IDX?v"
```

### 3.2 写入样例数据（单条 + bulk）

单条写入：

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X PUT "$ES_URL/$IDX/_doc/p1?refresh=wait_for" \
  -H "Content-Type: application/json" \
  -d '{
    "id":"p1",
    "name":"轻薄笔记本 Pro 14",
    "category":"laptop",
    "price":6999,
    "in_stock":true,
    "created_at":"2026-03-01T10:00:00Z"
  }'
```

批量写入（NDJSON）：

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/$IDX/_bulk?refresh=wait_for" \
  -H "Content-Type: application/x-ndjson" \
  -d '
{ "index": { "_id": "p2" } }
{ "id":"p2","name":"游戏笔记本 Max","category":"laptop","price":8999,"in_stock":false,"created_at":"2026-03-02T08:00:00Z" }
{ "index": { "_id": "p3" } }
{ "id":"p3","name":"无线机械键盘","category":"keyboard","price":599,"in_stock":true,"created_at":"2026-03-03T12:30:00Z" }
'
```

快速检查 bulk 是否全部成功（应返回 `true`）：

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/$IDX/_bulk?refresh=wait_for" \
  -H "Content-Type: application/x-ndjson" \
  -d '{ "index": { "_id": "tmp" } }
{ "id":"tmp","name":"tmp","category":"tmp","price":1,"in_stock":true,"created_at":"2026-03-01T00:00:00Z" }
' | jq ".errors"
```

### 3.3 查询验证（检索 + 过滤 + 排序）

查询 1：全文检索 + 过滤 + 排序

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/$IDX/_search" \
  -H "Content-Type: application/json" \
  -d '{
    "query": {
      "bool": {
        "must": [
          { "match": { "name": "笔记本" } }
        ],
        "filter": [
          { "term":  { "category": "laptop" } },
          { "range": { "price": { "lte": 8000 } } }
        ]
      }
    },
    "sort": [
      { "price": "asc" }
    ],
    "size": 10
  }'
```

查询 2：聚合验证（按类目统计）

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/$IDX/_search" \
  -H "Content-Type: application/json" \
  -d '{
    "size": 0,
    "aggs": {
      "by_category": {
        "terms": { "field": "category" }
      }
    }
  }'
```

### 3.4 清理索引（可重复实验）

```bash
curl -sS -u "$ES_USER:$ES_PASS" -X DELETE "$ES_URL/$IDX"
```

## 实战（Java SDK）

如果你们团队是 Java 技术栈，建议把验证流程固化为一个冒烟测试。

### 4.1 Maven 依赖（示例）

```xml
<dependency>
  <groupId>co.elastic.clients</groupId>
  <artifactId>elasticsearch-java</artifactId>
</dependency>
```

### 4.2 最小 Java 示例

```java
import co.elastic.clients.elasticsearch.ElasticsearchClient;
import co.elastic.clients.elasticsearch._types.SortOrder;
import co.elastic.clients.elasticsearch.core.BulkRequest;
import co.elastic.clients.elasticsearch.core.SearchResponse;
import co.elastic.clients.elasticsearch.core.bulk.BulkOperation;
import co.elastic.clients.elasticsearch.indices.CreateIndexResponse;
import co.elastic.clients.json.jackson.JacksonJsonpMapper;
import co.elastic.clients.transport.ElasticsearchTransport;
import co.elastic.clients.transport.rest_client.RestClientTransport;
import org.apache.http.HttpHost;
import org.apache.http.auth.AuthScope;
import org.apache.http.auth.UsernamePasswordCredentials;
import org.apache.http.impl.client.BasicCredentialsProvider;
import org.elasticsearch.client.RestClient;

import java.util.List;
import java.util.Map;

public class EsChapter02Demo {
    public static void main(String[] args) throws Exception {
        String index = "products_v1";

        BasicCredentialsProvider cp = new BasicCredentialsProvider();
        cp.setCredentials(AuthScope.ANY, new UsernamePasswordCredentials("elastic-admin", "elastic-password"));

        RestClient lowClient = RestClient.builder(new HttpHost("localhost", 9200, "http"))
            .setHttpClientConfigCallback(hc -> hc.setDefaultCredentialsProvider(cp))
            .build();
        ElasticsearchTransport transport = new RestClientTransport(lowClient, new JacksonJsonpMapper());
        ElasticsearchClient client = new ElasticsearchClient(transport);

        // 1) create index
        CreateIndexResponse createResp = client.indices().create(c -> c
            .index(index)
            .settings(s -> s.numberOfShards("1").numberOfReplicas("0"))
            .mappings(m -> m.properties("name", p -> p.text(t -> t))
                .properties("category", p -> p.keyword(k -> k))
                .properties("price", p -> p.double_(d -> d))
                .properties("in_stock", p -> p.boolean_(b -> b))
            )
        );
        System.out.println("create acknowledged=" + createResp.acknowledged());

        // 2) bulk index
        List<BulkOperation> ops = List.of(
            BulkOperation.of(o -> o.index(i -> i.index(index).id("p1").document(
                Map.of("name", "轻薄笔记本 Pro 14", "category", "laptop", "price", 6999, "in_stock", true)))),
            BulkOperation.of(o -> o.index(i -> i.index(index).id("p2").document(
                Map.of("name", "游戏笔记本 Max", "category", "laptop", "price", 8999, "in_stock", false))))
        );
        client.bulk(BulkRequest.of(b -> b.index(index).operations(ops).refresh(r -> r.true_())));

        // 3) search
        SearchResponse<Map> searchResp = client.search(s -> s
                .index(index)
                .query(q -> q.bool(b -> b
                    .must(m -> m.match(mm -> mm.field("name").query("笔记本")))
                    .filter(f -> f.term(t -> t.field("category").value("laptop")))
                ))
                .sort(so -> so.field(f -> f.field("price").order(SortOrder.Asc)))
                .size(10),
            Map.class
        );
        System.out.println("hits=" + searchResp.hits().total().value());

        // 4) delete index
        client.indices().delete(d -> d.index(index));
        lowClient.close();
    }
}
```

## 5. 常见问题与排查

### 5.1 写入后查不到
- 没有 refresh，或者查询命中了别的索引。
- 排查：先查 `GET /{index}/_count`，再确认查询条件。

### 5.2 bulk 接口返回 200 但数据不全
- bulk 是“部分成功”语义，必须检查 `errors` 字段。
- 排查：遍历 `items`，记录失败原因并重试。

### 5.3 字段类型不对导致查询异常
- 动态映射把字段推断错了（如数字变 text）。
- 排查：`GET /{index}/_mapping`，必要时重建索引。

---

# 总结

- 最小闭环是 ES 项目启动阶段最重要的“共识资产”。
- 先用 curl 跑通，再用 SDK 固化成自动化测试。
- 每次新环境上线，先做这 4 步验证，可显著降低后续排障成本。
