# 背景

# 第10章 多字段与多语言：一套数据多种检索形态

同一个业务字段往往同时承担“搜索、过滤、排序、聚合”多种职责。若建模阶段没有一次性设计好，后续通常要用重建索引来补救，成本很高。

## 本章目标
- 用一套字段同时满足检索、过滤、排序、聚合。
- 掌握多语言场景下的基本建模方法。

## 1. 多字段（multi-fields）为什么重要
同一业务字段往往有多种用途：
- 全文搜索
- 精确筛选
- 排序与聚合

使用 `text + keyword` 组合可同时满足这些需求。

## 2. 常见建模方式
- 主字段：`text`（搜索）
- 子字段：`keyword`（过滤/聚合/排序）
- 需要规范化时配合 `normalizer`

## 3. 多语言处理思路
- 方案一：同一索引多字段（简单）
- 方案二：按语言拆索引（隔离性好）

选择取决于数据规模、查询复杂度和团队维护成本。

## 4. 实践建议
- 先统一字段命名与模板。
- 明确每个字段“谁来搜、怎么搜、如何排序”。

# 总结
- 多字段建模是“高可用搜索体验”的基础。
- 建模先于查询，查询依赖建模质量。

## 实战（curl）

```bash
curl -u "$ES_USER:$ES_PASS" -X PUT "$ES_URL/mf_demo" \
  -H "Content-Type: application/json" \
  -d '{
    "mappings":{"properties":{
      "name":{"type":"text","fields":{"keyword":{"type":"keyword"}}}
    }}
  }'

curl -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/mf_demo/_doc/1?refresh=wait_for" \
  -H "Content-Type: application/json" \
  -d '{"name":"Elastic Stack Guide"}'

curl -u "$ES_USER:$ES_PASS" -X POST "$ES_URL/mf_demo/_search?pretty" \
  -H "Content-Type: application/json" \
  -d '{"query":{"term":{"name.keyword":"Elastic Stack Guide"}}}'
```

## 实战（Java SDK）

```java
client.search(s -> s.index("mf_demo")
    .query(q -> q.term(t -> t.field("name.keyword").value("Elastic Stack Guide"))),
    Map.class);
```
