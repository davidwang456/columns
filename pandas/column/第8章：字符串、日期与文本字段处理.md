# 第8章：字符串、日期与文本字段处理

## 1. 项目背景

某在线教育平台的营销团队每月都要做一次"短信营销效果复盘"。数据源是短信平台导出的发送日志 CSV（sms_log.csv），包含 50 万条短信发送记录，每行有一个 `message` 字段存储原始短信内容，还有一个 `send_time` 字段存储发送时间。

运营经理老张需要从这些混在一起的短信内容中提取出结构化信息：活动名称、优惠券码、转化时间要求、目标用户群体描述，然后按"活动 × 渠道 × 日期"做效果分析。

**痛点一：短信内容是自由文本，信息提取全靠手工**。示例短信：

```
【XX教育】您的618年中特惠邀请码 YX618-USER-ABCD-1234 已激活，请在2025-06-30前使用，专享8折优惠！退订回T
```

老张需要从这一句话里提取：活动名称（618年中特惠）、邀请码格式（YX618-USER-XXXX-数字）、截止日期（2025-06-30）、优惠力度（8折）。50 万条短信，他用 Excel 的 MID/FIND/SEARCH 嵌套函数提取，公式复杂到不可维护，而且一个公式写错需要重算整个文件。

**痛点二：日期格式混乱导致分析失败**。短信平台记录的时间格式是 `2025-03-15 14:30:25+08:00`（带时区），但用户点击短信的时间从另一个系统导出，格式是 `20250315143025`。老张在 Excel 里手工转了两列日期格式，结果 VLOOKUP 时有一半匹配不上——因为一个有时区一个没有，肉眼看起来一样的日期实际差了 8 小时。

**痛点三：文本清洗中的空值和类型陷阱**。短信内容中有些是"发送失败"没有 content，有些是空白字符串 `''`，还有些是纯英文。老张用 Excel 筛选时，空值和空字符串看起来都是空的，但 ISBLANK 函数对空字符串返回 FALSE——筛选不出部分失败记录。

痛点流程：

```
短信日志 (50万条)
  ├── 短信内容 → Excel MID/FIND 嵌套 → 公式不可维护
  ├── 日期格式 → 手工转格式 → 时区导致匹配失败
  └── 空值陷阱 → ISBLANK/空字符串混淆 → 遗漏失败记录
```

本章将系统讲解 pandas 的 str 访问器和日期处理能力，用正则和向量化操作一次性解决文本字段的所有问题。

## 2. 项目设计：剧本式交锋对话

> 场景：老张把"日期始终匹配不上"的问题带到了周会上，小胖正在吃薯片。

**小胖**：（咔嚓咔嚓嚼薯片）"日期匹配不上？不就是 `2025-03-15` 和 `20250315` 的区别吗？手动去掉了 `-` 不就行了吗？"

**大师**："如果是 5 条数据，手动去 `-` 没问题。但如果 50 万条——还有不同时区、不同格式、不同分隔符呢？这就是为什么 pandas 有专门的 `pd.to_datetime()` 函数：它一次性处理整列的日期解析，支持 `format` 参数显式指定格式，性能远超手工处理。"

**小白**："`pd.to_datetime` 和 `read_csv` 里的 `parse_dates` 有什么区别？"

**大师**："关系是这样：
- `parse_dates`：在 **读取时** 就把字符串列解析为 datetime64，一步到位，IO 阶段完成；
- `pd.to_datetime()`：读取后 **单独转换**，更灵活——可以指定 `format`、处理 `errors`、处理混合格式；
- `df['dt_col'].dt`：转换完成后的 **日期访问器**，用于提取年、月、日、周、季等派生信息。

性能上，`parse_dates` 更快（C 级别读取+解析），但 `pd.to_datetime` 更灵活。生产环境建议在读取阶段用 `parse_dates` 处理已知格式的列，读取后再用 `pd.to_datetime` 处理格式不固定的列。"

**【技术映射：parse_dates = 在码头卸货时就分类打包，pd.to_datetime = 货物进仓库后再分拣——前者更快】**

**小胖**："那短信里提取邀请码和日期呢？总不能手工逐条看吧？"

**大师**："这就是正则表达式的用武之地。pandas 的 **str 访问器**（`df['col'].str`）提供了向量化的字符串操作：

```python
# 提取邀请码：YX618-USER-XXXX-数字
df['invite_code'] = df['message'].str.extract(r'(YX\d{3}-USER-[A-Z]+-\d+)')

# 判断短信内容是否含'退订'
df['has_unsubscribe'] = df['message'].str.contains('退订')

# 统计每个短信的字符长度
df['msg_length'] = df['message'].str.len()
```

`str.extract` 用正则的捕获组 `()` 提取子串，`str.contains` 做布尔匹配，`str.len` 统计字符数——全都是向量化的，比逐行 `apply` 快 10-50 倍。"

**【技术映射：str 访问器 = 流水线上的 OCR 扫描仪——每件产品经过时自动识别并提取信息】**

**小白**："那 `str.extract` 和 `str.extractall` 有什么区别？"

**大师**：

```python
# extract: 提取第一个匹配，返回单个结果
df['msg'].str.extract(r'(\d{4}-\d{2}-\d{2})')  # 第一条日期匹配

# extractall: 提取所有匹配，返回 MultiIndex DataFrame
df['msg'].str.extractall(r'(\d+)')   # 所有数字匹配（一个短信可能有多个数字）
```

**小白**："日期处理中时区是个大坑。`2025-03-15 14:30:25+08:00` 和 `2025-03-15 06:30:25` 实际上是同一时刻（前者是东八区当地时间，后者是 UTC 时间）。pandas 怎么处理？"

**大师**："pandas 的 `DatetimeIndex` 和 `Timestamp` 都支持时区：

```python
# 含时区字符串 → UTC
ts_utc = pd.to_datetime('2025-03-15 14:30:25+08:00', utc=True)
# → 2025-03-15 06:30:25+00:00

# 无时区字符串 → 本地化处理
ts_local = pd.to_datetime('2025-03-15 14:30:25').tz_localize('Asia/Shanghai')

# 统一同一时区再做比较
df['time_utc'] = pd.to_datetime(df['time_str'], utc=True)
```

关键原则：**所有时间数据在内部存储为 UTC，仅在展示时转为业务时区**。这能避免 90% 的时区 bug。"

**【技术映射：时区转换 = 不同国家的钟表——内部统一用格林尼治时间，各自的当地时间只是显示方式不同】**

**小胖**："那万一短信内容本身就有空值呢？`df['message'].str` 遇到空值会报错吗？"

**大师**："好问题。`str` 访问器遇到 NaN 时不会报错——它返回 NaN（而不是抛出异常）。这是 pandas 的一个重要设计决策：**向量化操作默认传递缺失值**。

但需要注意一个陷阱：空字符串 `''` 不是 NaN。`df['msg'] == ''` 是 False（对空字符串而言是不等于空字符串之外的任何东西），但 `pd.isna(df['msg'])` 对 `''` 也返回 False。如果你需要将空字符串也当作缺失处理：

```python
df['msg'] = df['msg'].replace('', pd.NA)
# 或
df['msg'] = df['msg'].where(df['msg'].str.len() > 0, other=pd.NA)
```"

**大师总结**："今天覆盖了 pandas 文本处理的四大核心能力：
1. `str.extract`：从半结构化文本中抠出结构化字段；
2. `str.contains`/`match`/`replace`：文本匹配和清洗；
3. `pd.to_datetime` + `dt` 访问器：日期解析和派生字段提取；
4. 时区和格式统一的规范化原则。

下面我们就从 50 万条短信日志里提取出活动、渠道、用户和转化指标。"

### 3.2 模拟短信日志数据

```python
# generate_sms_data.py
import pandas as pd
import numpy as np

np.random.seed(42)
N = 10000  # 模拟短信日志

activities = ['618年中特惠', '双11狂欢', '春季开学季', '老带新有礼', 'VIP专享折扣']
channels = ['短信', '小程序推送']
templates = [
    '【XX教育】您的{activity}邀请码 {code} 已激活，请在{deadline}前使用，专享{discount}优惠！回T退订',
    '【XX教育】{activity}活动火热进行中，您的专属优惠码{alternate_code}，有效期至{deadline}，{discount}等你来！',
    '【XX教育】{activity}：恭喜获得{discount}资格，验证码 {code}，截止{deadline}',
    '【XX教育】{activity}福利：老学员专属{discount}，码{alternate_code}，{deadline}前有效',
    None,  # 发送失败的场景
]

data = []
for i in range(N):
    template = np.random.choice(templates)
    activity = np.random.choice(activities)
    deadline = pd.Timestamp('2025-06-30') - pd.Timedelta(days=np.random.randint(0, 180))
    code = f'YX{np.random.randint(100,999)}-USER-{np.random.choice(["ABCD","EFGH","IJKL"])}-{np.random.randint(1000,9999)}'
    alternate_code = f'SP{np.random.randint(100,999)}{np.random.choice(["A","B","C"])}{np.random.randint(100,999)}'
    discount = np.random.choice(['8折', '7折', '5折', '满200减50', '第二件半价'])

    if template is None:
        message = np.nan
    else:
        message = template.format(
            activity=activity, code=code, deadline=deadline.strftime('%Y-%m-%d'),
            discount=discount, alternate_code=alternate_code
        )

    send_time = pd.Timestamp('2025-03-01') + pd.Timedelta(hours=np.random.randint(0, 720))
    data.append({
        'sms_id': f'SMS{1000000+i:07d}',
        'send_time': send_time.strftime('%Y-%m-%d %H:%M:%S+08:00'),
        'user_id': np.random.randint(10000, 50000),
        'phone': f'138{np.random.randint(10000000,99999999)}',
        'message': message,
        'channel': np.random.choice(channels),
        'status': np.random.choice(['发送成功','发送失败','用户退订'], p=[0.75, 0.15, 0.1])
    })

df = pd.DataFrame(data)
# 制造一些空字符串
df.loc[np.random.choice(df.index, 50, replace=False), 'message'] = ''
df.to_csv('sms_log.csv', index=False, encoding='utf-8-sig')
print(f"已生成 {N} 条短信日志")
```

### 3.3 分步实现

#### 步骤 1：文本提取——正则抠字段

**目标**：从短信文本中提取活动名称、优惠码、截止日期、优惠力度。

```python
# step1_extract.py
import pandas as pd
import numpy as np

df = pd.read_csv('sms_log.csv')

# 先统一处理：空字符串 → NaN
df['message'] = df['message'].replace('', np.nan)

# === 提取邀请码 (YX开头的格式) ===
df['invite_code_yx'] = df['message'].str.extract(r'(YX\d{3}-USER-[A-Z]{4}-\d{4})')
print(f"提取到 YX 邀请码: {df['invite_code_yx'].notna().sum()} 条")

# === 提取备用优惠码 (SP开头的格式) ===
df['invite_code_sp'] = df['message'].str.extract(r'(SP\d{3}[A-C]\d{3})')
print(f"提取到 SP 优惠码: {df['invite_code_sp'].notna().sum()} 条")

# 合并两种码
df['invite_code'] = df['invite_code_yx'].fillna(df['invite_code_sp'])

# === 提取截止日期 ===
df['deadline_str'] = df['message'].str.extract(r'(\d{4}-\d{2}-\d{2})前|有效期至(\d{4}-\d{2}-\d{2})|截止(\d{4}-\d{2}-\d{2})')
# 上述三个捕获组可能命中不同组，需要合并
# 简化方案：直接提取所有日期
df['deadline'] = pd.to_datetime(
    df['message'].str.extract(r'(\d{4}-\d{2}-\d{2})')[0],
    errors='coerce'
)
print(f"提取到截止日期: {df['deadline'].notna().sum()} 条")

# === 提取活动名称 ===
for act in ['618年中特惠', '双11狂欢', '春季开学季', '老带新有礼', 'VIP专享折扣']:
    mask = df['message'].str.contains(act, na=False)
    df.loc[mask, 'activity'] = act
print(f"提取到活动名称: {df['activity'].notna().sum()} 条")

# === 提取优惠力度 ===
df['discount'] = df['message'].str.extract(r'([578]折|满\d+减\d+|第二件半价)')[0]
print(f"提取到优惠力度: {df['discount'].notna().sum()} 条")
```

#### 步骤 2：日期解析与时区统一

**目标**：解析带时区的日期，统一转为 UTC 然后提取日期维度。

```python
# step2_datetime.py
import pandas as pd

df = pd.read_csv('sms_log.csv')

# === 解析带时区的发送时间 ===
# '2025-03-15 14:30:25+08:00' 这种格式 pandas 可以直接解析
df['send_time_dt'] = pd.to_datetime(df['send_time'], utc=True)
print(f"send_time 解析后: {df['send_time_dt'].dtype}")
print(f"时区: {df['send_time_dt'].dt.tz}")

# === 解析无时区的截止日期（从步骤1提取的deadline_str）===
df['deadline_str'] = df['message'].str.extract(r'(\d{4}-\d{2}-\d{2})')[0]
df['deadline_dt'] = pd.to_datetime(df['deadline_str'], format='%Y-%m-%d', errors='coerce')
# 假设截止日期为北京时间 23:59:59
df['deadline_dt'] = df['deadline_dt'].apply(
    lambda x: pd.Timestamp(x).tz_localize('Asia/Shanghai') if pd.notna(x) else pd.NaT
)

# === 提取日期维度 ===
df['send_date'] = df['send_time_dt'].dt.date
df['send_hour'] = df['send_time_dt'].dt.hour
df['send_weekday'] = df['send_time_dt'].dt.day_name()  # Monday, Tuesday...
df['send_month'] = df['send_time_dt'].dt.month
df['send_week'] = df['send_time_dt'].dt.isocalendar().week  # ISO 周数

print(f"\n日期维度提取结果:")
print(f"  按小时分布:\n{df['send_hour'].value_counts().sort_index().head(8).to_string()}")

# === 计算发送到截止的天数 ===
df['days_to_deadline'] = (df['deadline_dt'] - df['send_time_dt']).dt.days
print(f"\n距离截止日期的天数统计:")
print(df['days_to_deadline'].describe())
```

# step3_str_advanced.py
import pandas as pd
import numpy as np

df = pd.read_csv('sms_log.csv')
df['message'] = df['message'].replace('', np.nan)

# === 文本清洗 ===
# 统计短信长度分布
df['msg_length'] = df['message'].str.len()
print("短信长度统计:")
print(df['msg_length'].describe())

# 去除短信前后的【XX教育】 前缀
df['msg_clean'] = df['message'].str.replace(r'^【.*?】', '', regex=True)

# === 文本分类 ===
# 是否含退订提示
df['can_unsub'] = df['message'].str.contains('退订', na=False)
print(f"\n含退订提示: {df['can_unsub'].sum()} 条")

# 是否老学员短信
df['is_old_user'] = df['message'].str.contains('老学员|老带新', na=False)
print(f"老学员定向: {df['is_old_user'].sum()} 条")

# 短信复杂度（大写字母数/数字数）
df['uppercase_count'] = df['message'].str.count(r'[A-Z]')
df['digit_count'] = df['message'].str.count(r'\d')
print(f"\n平均大写字母数: {df['uppercase_count'].mean():.1f}")
print(f"平均数字数: {df['digit_count'].mean():.1f}")

# === 按关键词字段拆分 ===
# split 返回的是 list，需要 expand=True 拆成多列
df[['part1','part2']] = df['msg_clean'].str.split('，', n=1, expand=True)
print(f"\n按逗号拆分后 part1 示例:")
print(df['part1'].dropna().head(3).tolist())
```

#### 步骤 4：完整主流程

```python
# main_sms_analysis.py
"""短信营销效果分析主流程"""
import pandas as pd
import numpy as np

print("开始短信日志分析...")
df = pd.read_csv('sms_log.csv')
df['message'] = df['message'].replace('', np.nan)

# 1. 日期解析
df['send_time_dt'] = pd.to_datetime(df['send_time'], utc=True)
df['send_date'] = df['send_time_dt'].dt.date

# 2. 正则提取结构化字段
df['invite_code'] = df['message'].str.extract(r'(YX\d{3}-USER-[A-Z]{4}-\d{4})')[0].fillna(
    df['message'].str.extract(r'(SP\d{3}[A-C]\d{3})')[0]
)
df['deadline_dt'] = pd.to_datetime(df['message'].str.extract(r'(\d{4}-\d{2}-\d{2})')[0])

for act in ['618年中特惠','双11狂欢','春季开学季','老带新有礼','VIP专享折扣']:
    df.loc[df['message'].str.contains(act, na=False), 'activity'] = act

df['discount'] = df['message'].str.extract(r'([578]折|满\d+减\d+|第二件半价)')[0]

# 3. 分析维度
df['send_hour'] = df['send_time_dt'].dt.hour

# 4. 按活动×日期×小时汇总
summary = (
    df.groupby(['activity', 'send_date', 'send_hour'], dropna=False)
    .agg(
        发送数=('sms_id', 'count'),
        成功数=('status', lambda x: (x == '发送成功').sum()),
        退订数=('status', lambda x: (x == '用户退订').sum()),
        成功提取码=('invite_code', lambda x: x.notna().sum())
    )
    .reset_index()
)

summary['成功率'] = (summary['成功数'] / summary['发送数'] * 100).round(2)
print(f"\n===== 按活动汇总 =====")
act_summary = summary.groupby('activity').agg({'发送数':'sum','成功数':'sum','成功率':'mean'}).round(2)
print(act_summary.to_string())

# 5. 导出分析结果
summary.to_csv('sms_analysis_result.csv', index=False, encoding='utf-8-sig')
print(f"\n✓ 分析结果已导出至 sms_analysis_result.csv")
```

### 3.4 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `str.extract` 返回 NaN | 正则没匹配到 | 先用少量数据测试正则：`df.head().str.extract(...)` |
| `str.extract` 返回多列 | 正则中有多个捕获组 `()` | 每对 `()` 产生一列，注意引用方式 |
| `to_datetime` 大量 NaT | 日期格式与预期不符 | 显式指定 `format='%Y-%m-%d %H:%M:%S'` |
| 时区转换后时间偏差 | 无时区时间被当作 UTC | 先 `tz_localize` 本地化，再 `tz_convert` 转换 |
| `contains` 对 NaN 报错 | NaN 不能做字符串匹配 | 加 `na=False` 参数，NaN 返回 False |
| `extractall` 结果行数暴增 | 每条记录多次匹配都会产生一行 | 确认是否真的需要所有匹配，或改用 `extract` |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas str/dt 方案 | Excel 文本函数 | Python re 模块逐行 |
|------|-------------------|---------------|-------------------|
| 提取速度 | 向量化，C 级别 | 逐行公式计算 | Python 逐行，慢 |
| 正则功能 | 完整支持 PCRE 正则 | 有限（通配符为主） | 完整支持 |
| 日期处理 | 时区/格式/派生维度一站式 | 格式转换繁琐 | 需 datetime 模块配合 |
| 空值安全 | NaN 自动传递 | ISBLANK/空字符串陷阱 | 需手动判断 |
| 可读性 | 链式调用，紧凑 | 公式嵌套，难维护 | 代码可读但长 |
| 学习成本 | 中等 | 低 | 低 |

### 4.2 适用场景

- **适用场景**：
  1. 从半结构化文本（日志、短信、邮件、评论）中提取结构化字段
  2. 企业级报表中的日期解析和时区统一
  3. 批量文本清洗：格式标准化、敏感词过滤、关键词提取
  4. 数据质量检查：日期合法性、文本长度分布、正则格式校验
  5. 多数据源的日期字段对齐（不同的格式和时区统一）
- **不适用场景**：
  1. 自然语言处理（分词、情感分析等）——pandas 不是 NLP 工具
  2. 需要复杂文本解析（如 HTML/XML）——使用 BeautifulSoup 等专业库

### 4.3 注意事项

- **正则性能**：在百万级数据上用正则提取时，尽量用精确的模式而非 `.*` 通配——后者可能导致回溯爆炸；
- **日期 format 参数**：`pd.to_datetime` 加 `format` 参数可显著加速（避免了格式自动推断），建议生产环境显式指定；
- **str 访问器返回类型**：`str.extract` 返回 DataFrame（即使只有一个捕获组），`str.contains` 返回布尔 Series，`str.replace` 返回 Series；
- **空字符串 vs NaN**：清洗前统一做 `df['col'].replace('', pd.NA)` 消除歧义。

### 4.4 常见踩坑经验

1. **`str.extract` 用了 `expand=False` 导致返回 Series，但后续代码按 DataFrame 操作**：pandas 2.0 之后 `expand` 默认为 True，返回 DataFrame。团队中如果用不同版本可能导致行为不一致。
2. **日期时区混淆导致跨天统计**：一位同事用 `dt.date` 提取日期时，命令在 UTC 时间凌晨 3 点执行——但 `dt.date` 取的是 UTC 日期而非北京时间，报告里的日期差了一天。解决办法：`df['local_date'] = df['utc_time'].dt.tz_convert('Asia/Shanghai').dt.date`
3. **正则中 `\d{4}` 匹配到了不相关的数字**：短信内容中 `13812345678` 的手机号也被当作日期。解决：用更精确的边界匹配 `r'(\d{4}-\d{2}-\d{2})'` 要求有连字符分隔。
