# 第16章：【基础篇综合实战】从原始订单到经营分析报告

## 1. 项目背景

某连锁零售公司在全国 6 个城市拥有 24 家门店，每周产生约 5 万条交易记录。公司运营总监每周一需要一份"周度经营分析报告"，涵盖从原始订单清洗到最终 Excel 交付的完整流程。目前这份报告由数据分析师小何负责。

小何以前的做法是：每周一早上从 POS 系统导出 CSV（约 5 万行），在 Excel 里先后进行去重、补全缺失值、关联商品信息、按门店汇总、按品类透视、计算环比增长率，最后手动调格式交付。整套流程平均耗时 3 小时，其中 2 小时用于数据操作，1 小时用于格式美化。

**痛点一：重复劳动无法自动化**。5 万行 CSV 数据每刷新一次，所有清洗、聚合、透视操作都要重做一遍——步骤完全一致但必须手工执行。小何请病假时没有人能接手，曾经导致连续两周的周报延期。

**痛点二：数据链路不透明**。总监曾质疑"上周报告中 A 门店的销售额偏低"，小何无法追溯是在清洗还是聚合阶段出了问题，因为他没有保留中间结果。一次排查花了一整个下午，最后发现是去重时误删了 A 店的部分订单——但这个操作没有任何记录。

**痛点三：交付格式不稳定**。有时小何调整了列宽，下次导出时又忘了冻结窗格，区域经理打开报告后滚动到第 10 行就看不到表头了。还有一次他忘记给异常行标红，总监在会议上打开 Excel 后当场指出"这个负增长率为什么不标红？"

**痛点四：环比计算靠手动**。小何需要把本周数据和上周数据手工对齐——先在 Excel 里做本周透视表，再做上周透视表，手工复制到同一张表里再除。一旦周范围变了（比如总监要看 2 周的滚动趋势），所有步骤重新来过。

在基础篇的 15 章中，我们分别学习了：数据读取（第 4 章）、Index 操作与对齐（第 3 章）、缺失值与异常值清洗（第 6 章）、dtype 优化（第 7 章）、字符串与日期处理（第 8 章）、GroupBy 分组统计（第 9 章）、表连接（第 10 章）、透视表（第 11 章）、时间序列基础（第 12 章）、窗口函数（第 13 章）、数据导出与格式化（第 14 章）、测试验证（第 15 章）。本章将把所有这些能力串联起来，完成一个**端到端的 pandas 小项目**，并沉淀为可复用的项目模板。

## 2. 项目设计：剧本式交锋对话

> 场景：小何在周一的经营分析会上展示了他的"手工 3 小时"流程，大师看到了优化的巨大空间。小胖、小白和大师展开了多轮讨论。

**小胖**：（咬着面包）"何哥，你不是学过 pandas 吗？用 `read_csv` 读进来，`groupby` 分组，`to_excel` 导出——就三步，咋还要 3 小时？"

**小何**："三步是理想情况。实际上数据很脏——有重复订单、有空值、有异常金额。而且导出的 Excel 没有格式，区域经理打开全是白底黑字，还得我手动调。"

**大师**："小胖说的'三步'是对的，但每步都需要子步骤。这就是为什么我们需要的是**脚本化项目**而非临时代码。一个成熟的 pandas 项目结构应该是：

```
project/
├── data/              # 原始数据目录
├── output/            # 输出报告目录
├── src/
│   ├── load.py        # 数据读取

├── tests/             # 测试
├── config.yaml        # 配置文件
├── main.py            # 主入口
└── requirements.txt   # 依赖
```

这样每个文件职责单一，出问题能快速定位——比一个 500 行的 `analysis.py` 好维护得多。"

**【技术映射：模块化项目结构 = 餐厅后厨分工——洗菜/切菜/炒菜/装盘各司其职，而不是一个人全包】**

**小白**："那配置文件和硬编码有什么区别？我见过很多脚本直接在代码里写 `DATA_DIR = 'C:/Users/xiaohe/data/'`。这也能跑啊。"

**大师**："能跑但不可迁移。硬编码的路径在别人机器上跑不了，在服务器上也跑不了。配置化的方式用 YAML 文件或命令行参数。核心原则是**代码和配置分离**——代码描述"做什么"，配置描述"对什么数据做"。YAML 方式直观、可注释、支持层级结构。命令行方式 `python main.py --start 2025-03-17 --end 2025-03-23` 更适合定时任务。"

**【技术映射：配置分离 = 遥控器——同样的电视，不同人用不同的频道设置，但遥控器本身不变】**

**小胖**："那怎么保证每次跑的结果一样？我经常跑两次发现数字对不上！是不是数据有 bug？"

**大师**："可复现的三要素：
1. **固定随机种子**：只要涉及随机数（如模拟数据），就 `np.random.seed(42)`
2. **参数化日期**：不用 `pd.Timestamp('today')`，而是显式传入日期参数
3. **中间结果校验**：每一步后都输出行数和关键指标，便于对比

再加上测试——用 `assert` 验证每个清洗函数的输出符合预期。测试不仅可以验证正确性，更是一份'活文档'——看着测试就知道每个函数应该输出什么。"

**小白**："那清洗的边界条件怎么处理？比如某家门店一周完全没交易——它的数据会不存在于 CSV 中，但总监仍然希望在报告中看到这家店（销售额为 0）。"

**大师**："这是个典型的**缺失维度补全**问题。清洗完后，你需要用 `pd.MultiIndex.from_product` 生成所有 (城市, 门店, 品类) 的组合，然后用 `reindex` 补全缺失的组合，填充 0。这和我们在第 12 章学的时间序列缺失日期补齐是同一个思路——只是维度从时间换成了业务维度。

```python
all_stores = config['stores']  # 24 家门
all_cats = orders['category'].unique()  # 5 个品类
full_index = pd.MultiIndex.from_product([all_stores, all_cats], names=['store', 'category'])
# 按门店×品类聚合
store_cat = orders.groupby(['store','category'])['amount'].sum()
# 补全缺失组合
store_cat = store_cat.reindex(full_index, fill_value=0)
```"

**【技术映射：缺失维度补全 = 点名册——来了的同学画勾，没来的画叉（0），而不是把没来的人从名册上删掉】**

**小胖**：（挠头）"等一下——那怎么知道总销售额对不对？万一清洗中丢了一些数据怎么办？"

**大师**："用**一致性校验**。在分析模块的入口和出口各设一个检查点：`assert abs(分析结果的 total_sales - 原始数据的 amount.sum()) < 0.01`。如果对不上，说明中间步骤有问题。这在第 15 章讲过——三层断言（结构/业务/一致性）。"

**大师总结**："今天我们要完成的是一个综合实战项目。在动手之前，先记住四条工程原则：
1. 代码模块化——读/洗/算/导四步分离
2. 配置外置——YAML 或命令行
3. 可复现——固定种子 + 参数化 + 测试验证
4. 一致性校验——入口和出口的数据总量必须一致

基础篇的 15 个能力点——从 CSV 读取到 pytest 测试——今天一场实战全部串联起来。"

### 3.4 分步实现

#### 步骤 1：数据读取（src/load.py）

**目标**：从 CSV 读取原始数据，按配置的日期范围过滤，并做基础的完整性检查。

```python
# src/load.py
import pandas as pd
import yaml
import logging
from pathlib import Path

logger = logging.getLogger(__name__)

def load_config(path='config.yaml'):
    config_path = Path(__file__).parent.parent / path
    with open(config_path, 'r', encoding='utf-8') as f:
        return yaml.safe_load(f)

def load_data(config):
    """读取并解析原始数据，按日期范围过滤"""
    logger.info("[1/5] 加载数据...")
    orders = pd.read_csv(config['data']['orders'], parse_dates=['trade_time'])
    products = pd.read_csv(config['data']['products'])
    logger.info(f"  订单: {len(orders):,} 条, 商品: {len(products)} 个")

    start = pd.Timestamp(config['dates']['week_start'])
    end = pd.Timestamp(config['dates']['week_end']) + pd.Timedelta(days=1)
    orders = orders[(orders['trade_time'] >= start) & (orders['trade_time'] < end)]
    logger.info(f"  过滤后(本周): {len(orders):,} 条, 门店数: {orders['store'].nunique()}")

    # 入口校验：订单数不低于阈值
    min_orders = config.get('quality', {}).get('min_orders', 1000)
    if len(orders) < min_orders:
        logger.warning(f"  订单数 {len(orders)} < 最低阈值 {min_orders}，可能存在数据缺失")
    return orders, products
```

#### 步骤 2：数据清洗（src/clean.py）

**目标**：去重→补全缺失价格→修正异常金额→优化 dtype→补全缺失维度。

```python
# src/clean.py
import pandas as pd
import numpy as np
import logging

logger = logging.getLogger(__name__)

def clean_orders(orders, products, config):
    """清洗订单数据"""
    logger.info("[2/5] 清洗数据...")
    before = len(orders)

    # 去重（按 order_id，保留第一条）
    orders = orders.drop_duplicates(subset='order_id', keep='first')
    logger.info(f"  去重: {before:,} → {len(orders):,} (-{before-len(orders):,})")

    # 补全缺失价格（优先从商品主数据映射，其次用品类中位数）
    price_map = products.set_index('product')['cost_price'].to_dict()
    orders['unit_price'] = orders['unit_price'].fillna(orders['product'].map(price_map))
    still_missing = orders['unit_price'].isna().sum()
    if still_missing > 0:
        orders['unit_price'] = orders['unit_price'].fillna(
            orders.groupby('category')['unit_price'].transform('median')
        )
    logger.info(f"  补全价格: 缺失{still_missing} → 已填充")

    # 修正异常金额：负值归零，重新计算确保一致性
    orders['amount'] = orders['amount'].clip(lower=0)
    orders['amount'] = round(orders['qty'] * orders['unit_price'], 2)

    # dtype 优化（基础篇第 7 章）
    for col in ['category','product','store','city','status']:
        orders[col] = orders[col].astype('category')
    orders['qty'] = orders['qty'].astype('int8')
    mem = orders.memory_usage(deep=True).sum() / 1024**2
    logger.info(f"  dtype 优化后内存: {mem:.2f} MB")

    # 补全缺失维度：确保所有门店×品类组合都出现（缺失的填0）
    all_stores = config.get('stores', orders['store'].unique().tolist())
    all_cats = orders['category'].cat.categories.tolist()
    full_store_cat = pd.DataFrame(
        [(s, c) for s in all_stores for c in all_cats],
        columns=['store', 'category']
    )
    store_cat_sales = orders.groupby(['store','category'], observed=True)['amount'].sum().reset_index()
    store_cat_full = full_store_cat.merge(store_cat_sales, on=['store','category'], how='left').fillna({'amount': 0})
    store_cat_full['has_data'] = store_cat_full['amount'] > 0
    missing_combos = (store_cat_full['has_data'] == False).sum()
    logger.info(f"  缺失维度补全: {missing_combos} 个门店×品类组合无交易数据(已填0)")

    logger.info(f"  清洗完成: {len(orders):,} 条, {orders['store'].nunique()} 家门店")
    return orders, store_cat_full
```

#### 步骤 3：分析计算（src/analyze.py）

**目标**：门店KPI、品类透视表、环比增长、热销Top10、日均趋势。

```python
# src/analyze.py
import pandas as pd
import numpy as np
import logging

logger = logging.getLogger(__name__)

def compute_analysis(orders, products):
    """计算四张核心报表"""
    logger.info("[3/5] 分析计算...")

    # ===== 报表1：门店 KPI 看板 =====
    store_kpi = orders.groupby(['city','store'], as_index=False, observed=True).agg(
        销售额=('amount','sum'), 订单数=('order_id','nunique'),
        客单价=('amount','mean'), 销量=('qty','sum'), 毛利率=('unit_price',
            lambda x: ((x.sum() - products.set_index('product').loc[orders.loc[x.index,'product'],'cost_price'].sum()) / x.sum()).round(4)
            if len(x) > 0 else 0
        )
    ).round(2)
    # 简化毛利率计算
    store_kpi['销售额排名'] = store_kpi['销售额'].rank(ascending=False).astype(int)

    # ===== 报表2：城市×品类 透视表 =====
    cat_pivot = pd.pivot_table(
        orders, index='city', columns='category',
        values='amount', aggfunc='sum', fill_value=0,
        margins=True, margins_name='合计', observed=True
    ).round(0)

    # ===== 报表3：环比增长（模拟上周数据）=====
    np.random.seed(100)
    prev = store_kpi.copy()
    prev['销售额'] = prev['销售额'] * np.random.uniform(0.85, 1.15, len(prev))
    store_kpi['上周销售额'] = prev['销售额'].values.round(2)
    store_kpi['环比变化%'] = ((store_kpi['销售额'] - store_kpi['上周销售额'])
                           / store_kpi['上周销售额'] * 100).round(2)

    # ===== 报表4：热销 Top 10 =====
    top_products = orders.groupby('product', observed=True).agg(
        销售额=('amount','sum'), 销量=('qty','sum')
    ).nlargest(10, '销售额').reset_index()

    # ===== 报表5：每日销售趋势（时间序列）=====
    daily_trend = orders.groupby(orders['trade_time'].dt.date).agg(
        日销售额=('amount','sum'), 日订单=('order_id','nunique')
    ).reset_index()
    daily_trend['7日滚动均值'] = daily_trend['日销售额'].rolling(7, min_periods=1).mean().round(0)

    # ===== 一致性校验 =====
    total_from_kpi = store_kpi['销售额'].sum()
    total_from_raw = orders['amount'].sum()
    assert abs(total_from_kpi - total_from_raw) < 0.01, \
        f"数据不一致! KPI汇总={total_from_kpi:.2f}, 原始总和={total_from_raw:.2f}"

    logger.info(f"  门店: {store_kpi['store'].nunique()}, 品类: {orders['category'].nunique()}")
    logger.info(f"  总销售额: {store_kpi['销售额'].sum():,.2f}, 一致性校验通过")
    return store_kpi, cat_pivot, top_products, daily_trend
```

    wb = load_workbook(filename)
    hf = PatternFill(start_color='2F5496', end_color='2F5496', fill_type='solid')
    hfont = Font(bold=True, color='FFFFFF', size=11)
    red_fill = PatternFill(start_color='FFC7CE', end_color='FFC7CE', fill_type='solid')
    thin = Border(left=Side('thin'), right=Side('thin'), top=Side('thin'), bottom=Side('thin'))

    for ws in wb.worksheets:
        for cell in ws[1]: cell.fill, cell.font = hf, hfont
        ws.freeze_panes = 'A2'
        for col_idx in range(1, ws.max_column+1):
            ml = max(len(str(ws.cell(r=r, c=col_idx).value or '')) for r in range(1, ws.max_row+1))
            ws.column_dimensions[get_column_letter(col_idx)].width = min(ml+4, 35)
            for row in range(2, ws.max_row+1):
                ws.cell(row=row, column=col_idx).border = thin

    # 门店KPI：标红环比下降超 10%
    ws = wb['门店KPI']
    for row in range(2, ws.max_row+1):
        cv = ws.cell(row=row, column=ws.max_column).value
        if isinstance(cv, (int, float)) and cv < -10:
            for c in range(1, ws.max_column+1):
                ws.cell(row=row, column=c).fill = red_fill

    wb.save(filename)
    logger.info(f"  OK 报告已导出至 {filename}")
```

### 3.6 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| category 列 groupby 输出多余行 | `observed` 默认为 False | 全程设置 `observed=True` |
| `yaml.load` 不安全 | 使用了不安全的加载器 | 用 `yaml.safe_load` |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | 本章脚本化方案 | 手工 Excel 操作 | 商业 BI 工具 |
|------|-------------|---------------|-------------|
| 执行效率 | 全流程 < 5 秒 | 2-3 小时 | 分钟级 |
| 可复现 | 参数化日期 + 种子固定 | 人工操作，每次不同 | 仪表盘刷新 |
| 可维护 | 四模块分离，修改隔离 | 操作步骤靠记忆 | 配置化 |
| 可测试 | pytest 自动回归 | 无 | 有限 |
| 数据质量 | 清洗中校验一致性 | 靠肉眼 | 依赖数据源 |
| 部署成本 | pip install + cron | 无 | 平台订阅费 |

### 4.2 适用场景

- **适用场景**：
  1. 周度/月度经营分析报告的自动化生产
  2. 作为团队 pandas 项目的基础模板
  3. 数据链路需要可追溯、可审计的场景
  4. 多维度交叉报表（门店×品类×时间）的定时生成
  5. 需要将分析结果以 Excel 交付给非技术人员的场景
- **不适用场景**：
  1. 临时的一次性分析——不必建完整工程结构
  2. 数据源来自实时数据库（第 21 章扩展）

### 4.3 注意事项

- **模块导入路径**：项目根目录需在 PYTHONPATH 中，或用 `python -m src.load`；
- **category 与 observed**：这是基础篇最重要的好习惯——`observed=True` 和 `as_index=False`；
- **一致性校验**：分析模块出口必须校验总金额一致性——这是防止数据丢失的最后防线；
- **维度补全**：始终假设"有些门店可能没有交易"，用 reindex 补全而非依赖数据。

### 4.4 常见踩坑经验

1. **相对路径在 cron 定时任务中失效**：cron 的工作目录不同于交互 shell。用 `Path(__file__).resolve().parent` 获取脚本目录。
2. **openpyxl 修改后不保存**：在 with 块外操作 wb 后忘记 `wb.save()`—格式全丢失。
3. **category 列 observed=False 导致行数暴增**：24 个门店×5 个品类组合→120 行（30 行有数据，90 行全 0）。全程 `observed=True`。
