# 第17章：复杂清洗管道与链式调用

## 1. 项目背景

某出行平台的数据工程师小孙维护着一套"司机行为评分脚本"（driver_score.py），这套脚本从原始订单日志中计算每位司机的服务质量分数。脚本已经迭代了两年，由 3 位前同事陆续接手过，目前代码量约为 500 行——其结构是"逐行处理"模式，中间变量多达 25 个（df、df1、df2、df_clean、df_filtered、df_merged、df_result……）。

新需求来了：运营要求新增一条清洗规则——"取消率超过 30% 的司机标记为高风险"。小孙打开脚本后直接懵了：他找不到应该在哪一步插入这条规则——前同事的代码没有清洗"步骤"的概念，所有操作混在一起。

**痛点一：中间变量泛滥且命名混乱**。25 个中间变量，`df_clean` 在函数开头和结尾被重复使用（覆盖），`df_tmp` 出现在三个不同位置。小孙花了半天才搞清楚每条数据在什么阶段是什么状态。

**痛点二：清洗步骤无法独立测试**。因为所有逻辑混在一个函数里，无法单独测试"取消率计算"或"异常行程过滤"。想写个单元测试验证取消率的计算逻辑，得把整个函数跑一遍。

**痛点三：新增规则需要修改多处代码**。加一个"高风险标记"涉及：修改清洗函数（加规则）、修改中间变量名（确保不被覆盖）、修改输出格式——这三处可能散落在三个不同文件中。

痛点流程：

```
driver_score.py (500行, 25个中间变量)
  ├── 加一条规则 → 不知道插在哪
  ├── 想测试单步 → 必须跑全流程
  └── 修改输出 → 散落多处
```

本章将系统讲解如何用 pandas 的 assign、pipe、where、mask 以及函数式编程思想，将一盘散沙的清洗代码重构为可读、可测、可维护的清洗管道（pipeline）。

## 2. 项目设计：剧本式交锋对话

> 场景：小孙打开 500 行的 driver_score.py 愁眉苦脸，小胖凑过来看了一眼屏幕。

**小胖**：（边吃辣条边说）"这不就是 500 行嘛！一行行读呗！我经常写 500 行的代码！"

**大师**："行数不是问题，结构才是。500 行的一个函数和 500 行分 10 个函数——前者是'意大利面条'，后者是'流水线车间'。pandas 的 `pipe` 和 `assign` 就是帮你构建这种流水线的工具。"

**小白**："pipe 是什么？我从来没用过。"

**大师**："`pipe` 允许你把一个 DataFrame 作为第一个参数传给一个函数，然后链式调用下一个函数：

```python
# ❌ 传统写法——中间变量堆叠
df1 = load_data(path)
df2 = clean_duplicates(df1)
df3 = clean_null(df2)
df4 = add_features(df3)

# ✅ pipe 链式调用——无中间变量
result = (
    load_data(path)
    .pipe(clean_duplicates)
    .pipe(clean_null)
    .pipe(add_features)
)
```

`pipe` 的好处不仅仅是省了中间变量——它强迫你把每个处理步骤写成一个独立函数，把复杂的清洗流程拆解为一系列意图明确的小步骤。"

**【技术映射：pipe = 流水线上的工位——每个工位做一件事，工件从一端流到另一端】**

**小白**："那 assign 呢？和 pipe 有什么区别？"

**大师**："`assign` 专门用于**新增或修改列**。它的语法是 `df.assign(新列名=表达式)`：

```python
# ❌ 传统写法
df['cancel_rate'] = df['cancel_count'] / df['total_orders']
df['is_high_risk'] = df['cancel_rate'] > 0.3

# ✅ assign 链式
df = df.assign(
    cancel_rate = df['cancel_count'] / df['total_orders'],
    is_high_risk = lambda x: x['cancel_rate'] > 0.3  # lambda 接收当前 df
)
```

`assign` 可以在同一个调用中创建多个列，而且后面的表达式可以用 `lambda x` 引用前面刚创建的列——这避免了传统写法中必须分两行的尴尬。"

**小胖**："那 where 和 mask 呢？听起来像是筛选？"

**大师**："它们不是筛选（筛选用 `loc`），而是**条件替换**：

```python
# where: 条件为 True 保留原值，False 替换为 other
df['amount'].where(df['amount'] > 0, other=0)  # 负值 → 0

# mask: 和 where 逻辑相反——True 时替换
df['amount'].mask(df['amount'] < 0, other=0)   # 负值 → 0
```

`where` 和 `mask` 的区别只是条件逻辑的正反——`where` = 满足条件保留，`mask` = 满足条件替换。两者等价，看你用哪个更顺。"

**【技术映射：where = 红绿灯——绿灯（True）通过，红灯（False）拦截替换】**

**小白**："链式调用太长了会不会难以调试？出错了怎么排查？"

**大师**："这是个实际问题。链式调试三招：

1. **逐段执行**：把 `.pipe()` 的链拆开，一行一行跑，观察中间结果；
2. **插入 `.pipe(lambda x: (print(x.shape), x)[1])`**：打印当前 DataFrame 的行列数；
3. **写一个小辅助函数**：`def debug(df): print(df.shape); return df`，插入链中 `pipe(debug)`。

更重要的是——当你每个 pipe 步骤都是独立函数时，你可以单独测试每个函数，而不需要跑完整条链。这就是模块化带来的红利。"

**【技术映射：pipe 调试 = 工厂流水线的检测岗——每个工位之间放一个质检员，随时抽查半成品】**

**大师总结**："清洗管道的三件套：
1. `pipe`：串联处理步骤，每个步骤一个函数
2. `assign`：级联创建/修改列
3. `where` / `mask`：条件替换值

额外配合 `rename` 重命名、`drop` 删除列——形成函数式数据清洗范式。记住：**好的清洗代码读起来像说明书，而不是谜题**。"

np.random.seed(42)
N_DRIVERS = 500

drivers = pd.DataFrame({

    'completed_trips': np.random.randint(10, 500, N_DRIVERS),
    'cancelled_trips': np.random.randint(0, 50, N_DRIVERS),
    'total_earnings': np.round(np.random.uniform(500, 50000, N_DRIVERS), 2),
    'total_hours': np.round(np.random.uniform(10, 300, N_DRIVERS), 2),
    'rating': np.round(np.random.uniform(3.0, 5.0, N_DRIVERS), 2),
    'complaints': np.random.poisson(1, N_DRIVERS),

drivers['completed_trips'] = drivers[['completed_trips','total_trips']].min(axis=1)
drivers['cancelled_trips'] = np.minimum(drivers['cancelled_trips'], drivers['total_trips'] - drivers['completed_trips'])

# 制造脏数据
drivers.loc[drivers.sample(10).index, 'total_hours'] = np.nan
drivers.loc[drivers.sample(5).index, 'total_earnings'] = -1
drivers.loc[drivers.sample(8).index, 'rating'] = np.nan

### 3.3 分步实现

#### 步骤 1：用 pipe 构建清洗管道

**目标**：将清洗逻辑拆为 5 个独立函数，用 pipe 串联。

```python
# step1_pipe_pipeline.py
import pandas as pd
import numpy as np

def load_raw(path='drivers_raw.csv'):
    return pd.read_csv(path)

def fix_negative_values(df):
    """修正负值：earnings<0 → 0, trips<0 → NaN"""
    return df.assign(
        total_earnings = df['total_earnings'].mask(df['total_earnings'] < 0, 0),
    )

def fill_missing_values(df):
    """填充缺失：hours用城市中位数, rating用全局均值"""
    median_hours = df.groupby('city')['total_hours'].transform('median')
    return df.assign(
        total_hours = df['total_hours'].fillna(median_hours),
        rating = df['rating'].fillna(df['rating'].mean()),
    )

def add_derived_features(df):
    """添加派生指标"""
    return df.assign(
        cancel_rate = lambda x: (x['cancelled_trips'] / x['total_trips']).round(4),
        earnings_per_hour = lambda x: (x['total_earnings'] / x['total_hours']).round(2),
        complaint_rate = lambda x: (x['complaints'] / x['total_trips'] * 100).round(2),
        is_high_risk = lambda x: (
            (x['cancel_rate'] > 0.3) |
            (x['rating'] < 3.5) |
            (x['complaint_rate'] > 5.0)
        ),
    )

def optimize_dtypes(df):
    """优化 dtype"""
    return df.assign(
        city = df['city'].astype('category'),
        is_high_risk = df['is_high_risk'].astype('bool'),
    )

# === 管道 ===
df = (
    load_raw('drivers_raw.csv')
    .pipe(fix_negative_values)
    .pipe(fill_missing_values)
    .pipe(add_derived_features)
    .pipe(optimize_dtypes)
)

print(f"清洗完成: {len(df)} 名司机")
print(f"高风险司机: {df['is_high_risk'].sum()} 名")
```

#### 步骤 2：用 assign 级联创建列

**目标**：在一个 assign 调用中创建多个互相依赖的列。

```python
# step2_assign_chain.py
import pandas as pd

df = pd.read_csv('drivers_raw.csv')

# 一次 assign 创建 4 个列，后面的 lambda 可以引用前面的新列
df = df.assign(
    # 第一层：基础计算
    cancel_rate = df['cancelled_trips'] / df['total_trips'].replace(0, 1),
    # 第二层：引用 cancel_rate
    risk_level = lambda x: pd.cut(
        x['cancel_rate'], bins=[0, 0.1, 0.2, 0.5, 1.0],
        labels=['低','中','高','极高'], right=True
    ),
    # 第三层：复合条件
    earnings_per_hour = lambda x: x['total_earnings'] / x['total_hours'].replace(0, 1),
    quality_score = lambda x: (
        x['rating'] * 20 - x['cancel_rate'] * 100 - x['complaints'] * 5
    ).clip(0, 100).round(1),
)

print("assign 级联创建的结果:")
print(df[['driver_id','cancel_rate','risk_level','quality_score']].head(10).to_string(index=False))
```

#### 步骤 3：where / mask 条件替换实战

**目标**：用 where/mask 在不同列上执行条件替换。

```python
# step3_where_mask.py
import pandas as pd
import numpy as np

df = pd.read_csv('drivers_raw.csv')

# === where：条件满足保留，否则替换 ===
# earnings 为负 → 0
df['total_earnings'] = df['total_earnings'].where(df['total_earnings'] >= 0, other=0)

# rating 在合法范围外 → NaN
df['rating'] = df['rating'].where(df['rating'].between(1, 5), other=np.nan)

# === mask：条件满足替换，否则保留 ===
# 接单太少（<30）但投诉多（>3）→ 标记异常
df['is_suspicious'] = False
df['is_suspicious'] = df['is_suspicious'].mask(
    (df['total_trips'] < 30) & (df['complaints'] > 3),
    True
)

# === np.select：多条件分支替换 ===
conditions = [
    df['rating'] >= 4.8,
    df['rating'] >= 4.2,
    df['rating'] >= 3.5,
]
choices = ['金牌司机', '优质司机', '普通司机']
df['driver_level'] = np.select(conditions, choices, default='待观察')

print("where/mask/select 处理结果:")
print(df[['driver_id','rating','is_suspicious','driver_level']].head(10).to_string(index=False))
```

### 3.4 完整管道

```python
# main_pipeline.py
"""司机行为评分完整管道"""
import pandas as pd
import numpy as np

def load(path): return pd.read_csv(path)

def fix_data(df):
    return df.assign(
        total_earnings=df['total_earnings'].mask(df['total_earnings']<0, 0),
        total_trips=df['total_trips'].where(df['total_trips']>0, 1),
    )

def fill_na(df):
    return df.assign(
        total_hours=df['total_hours'].fillna(df.groupby('city')['total_hours'].transform('median')),
        rating=df['rating'].fillna(df['rating'].mean()),
    )

def add_scores(df):
    return df.assign(
        cancel_pct=lambda x: x['cancelled_trips']/x['total_trips'],
        earn_hr=lambda x: x['total_earnings']/x['total_hours'],
        score=lambda x: (x['rating']*20-x['cancelled_trips']/x['total_trips']*100-x.get('complaints',0)).clip(0,100),
        risk=lambda x: np.select(
            [x['score']>=85, x['score']>=65, x['score']>=45],
            ['优秀','良好','一般'], default='待改进'
        ),
    ).drop(columns=['cancelled_trips'])

def finalize(df):
    return df.rename(columns={c:'评分' if c=='rating' else c for c in df.columns})

# 管道
result = (load('drivers_raw.csv').pipe(fix_data).pipe(fill_na).pipe(add_scores).pipe(finalize))
print(f"管道输出: {len(result)} 行, {len(result.columns)} 列")
print(result['risk'].value_counts().to_string())
```

### 3.5 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| pipe 函数接收多个参数 | pipe 只传 df 给函数 | 用 `lambda df: func(df, arg1)` |
| assign 的 lambda 接收整个 df | 不接收单列 | `lambda x: x['col'] * 2` 而非 `lambda col: col*2` |
| where/mask 替换后 dtype 变 | 替换值改变了类型 | 替换后 `astype` 恢复 |
| 链太长难调试 | 中间状态不可见 | 插入 `.pipe(debug)` 函数打印 shape |
| drop 后列名不存在报错 | 列已被前一步删除 | `drop(columns=[...], errors='ignore')` |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pipe 管道 | 传统中间变量 | SQL CTE |
|------|---------|------------|---------|
| 可读性 | 步骤线性清晰 | 变量命名混乱 | WITH 子句清晰 |
| 可测试 | 每步可独立测试 | 需跑全流程 | 子查询可独立测试 |
| 可扩展 | 中间插入步骤 | 需改变量命名 | 中间插入 CTE |
| 调试 | 需 pipe(debug) | 直接 print 变量 | EXPLAIN 分析 |
| 学习成本 | 中等（函数式思维） | 低 | 低 |
| 性能 | 无额外开销 | 无额外开销 | 数据库优化 |

### 4.2 适用场景

- **适用场景**：
  1. 复杂数据清洗流程（5 个以上清洗步骤）的重构
  2. 需要频繁增删清洗规则的迭代场景
  3. 团队协作——每个人维护自己的 pipe 步骤函数
  4. 需要为每个清洗步骤编写独立单元测试
- **不适用场景**：
  1. 只有 2-3 步简单清洗——`pipe` 过度工程化
  2. 纯探索分析——临时代码不需要重构成管道

### 4.3 注意事项

- **pipe 函数签名**：第一个参数必须是 DataFrame，返回值也必须是 DataFrame；
- **assign 的 lambda**：`lambda x:` 中的 x 是整个 DataFrame，不是当前列。`x['col']` 是 Series；
- **where 和 mask 的 other**：如果不指定 other，默认填充 NaN——这可能导致后续计算静默跳过。

### 4.4 常见踩坑经验

1. **pipe 中修改了传入的 df（副作用）**：`def step(df): df['new'] = 1; return df` ——这个 df 是原 DataFrame 的引用，修改会反映到外部。解决：函数开头 `df = df.copy()`。
2. **assign 引用了被前一步删除的列**：`df.drop(columns=['A']).assign(B=lambda x: x['A']*2)` 报 KeyError——因为 drop 已经删了 A。调整步骤顺序。
3. **链式调用太长，报错时不知道哪一步出错**：Python traceback 会显示行号，但每一步内部逻辑复杂时仍难定位。建议在每个 pipe 函数内部加入 `assert` 做输入校验。
