# 第6章：缺失值与异常值清洗

## 1. 项目背景

某生鲜电商平台的商品运营团队每月需要做一次"全品类商品质量评估"，数据源为仓库管理系统导出的商品表（product_master.csv），包含商品编码、商品名称、品类、采购价、零售价、库存数量、入库日期、供应商、保质期天数等字段。

运营经理小王拿到这份数据后，按惯例先在 Excel 里做数据质量检查：

**痛点一：缺失值处理不一致**。部分商品的"采购价"为空，小王用"0"填充——结果月报中出现了"平均毛利率 -200%"的荒诞结论（零售价/0 = 无穷）。另有部分商品"保质期天数"缺失，小王手工逐个查纸质单据补全，2000 个 SKU 查了整整两天。

**痛点二：异常值凭经验判断漏检严重**。"零售价"列中有一个商品被误录为 99999 元（应该是 99.99 元），直接把平均客单价拉高了 40%。小王只在数据透视时"隐约感觉不对"，但没有系统化的异常检测方法，类似问题至少漏掉了 5 处。还有"库存数量"为 -15（退货流程中数据未更新），在求和统计时被当作了负库存。

**痛点三：清洗过程不可追溯**。小王在 Excel 里直接修改了原始单元格——改完后才发现有几处误操作，但已经找不到原始值了。当 CTO 在周会上质疑"这个毛利率为什么突然升了？"时，小王无法证明数据是在哪个环节被如何处理过。

痛点流程：

```
商品表 (2000行) ──┐
              缺失价格→填0→毛利率算错
              异常单价→未检出→均值失真
              负库存→参与了求和→库存对不上
              手工修改→不可追溯→无法复核
```

本章将系统讲解 pandas 中缺失值（NA/NaN/NaT/None）的处理策略和异常值检测方法（分位数/IQR/业务规则），并建立可追溯的清洗记录机制。

## 2. 项目设计：剧本式交锋对话

> 场景：小王拿着一份"月报数据异常复盘"走进技术团队，小胖正在啃鸡腿。

**小胖**：（嚼着鸡腿）"小王姐，你那个毛利率 -200% 是怎么算出来的？我数学不好也知道毛利率不可能为负啊。"

**小王**："别提了。有个商品的采购价是空的，我直接填了 0，毛利公式除以 0 就爆炸了。但问题是——我不填 0 填什么？"

**大师**："这个问题的根源在于：你没有区分'缺失值'和'零值'。在 pandas 中，这叫 **NA 语义**（缺失值语义）。表格里空着的意思是'我不知道'，不是'它是 0'。pandas 专门设计了 NaN（Not a Number）、NaT（Not a Time）、None 等特殊值来表示'缺失'。"

**【技术映射：NaN = 试卷上的空题——不填不等于得 0 分，而是'未作答'】**

**小白**："但我之前发现 `df['price'].mean()` 会自动跳过 NaN 吗？那填 0 和留 NaN 的区别是什么？"

**大师**："问得好。pandas 的大部分聚合函数（mean、sum、std 等）默认 `skipna=True`，会跳过 NaN 进行计算。所以留 NaN 求平均值，得到的是'已知价格的平均值'；而填 0 会让平均值被拉低——这两个结果完全不是一回事。"

**小胖**："那缺失值到底怎么处理？删掉还是填个值？"

**大师**："三种策略，各有适用场景：

- **删除（dropna）**：当缺失比例很低（<5%）且缺失不是系统性的，直接删除最简单；
- **填充（fillna）**：用均值、中位数、前向/后向值、或业务默认值填充。**关键是要记录你填了什么**；
- **插值（interpolate）**：对时间序列数据，用相邻点做线性或多项式插值。

选择哪种策略，取决于你对数据生成过程的理解——'为什么这个值是空的？'"

**【技术映射：缺失值处理 = 考试遇到不会的题——蒙一个（fillna）、空着跳过（dropna）、参考前后题的答案（interpolate）】**

**小白**："那异常值呢？99999 的零售价是怎么产生的？pandas 能自动检测吗？"

**大师**："pandas 不会自动检测——因为它不知道你的业务规则。但 pandas 提供了通用的统计工具来帮你发现异常值。最经典的是 **IQR 方法**（四分位距法）：

```
Q1 = 数据的 25% 分位数
Q3 = 数据的 75% 分位数
IQR = Q3 - Q1
异常值 = 值 < (Q1 - 1.5*IQR) 或 值 > (Q3 + 1.5*IQR)
```

此外还有 Z-score（标准差法）和业务阈值法（零售价不可能超过 10 万元）。IQR 方法不假设数据是正态分布，更鲁棒。"

**小胖**："那发现异常值之后呢？直接删了吗？"

**大师**："千万不要无声地删除。正确的做法是：
1. **标记**而非删除——新增一列 `is_outlier`；
2. **记录**检测规则和命中数量；
3. **追溯**到数据源头确认是录入错误还是真实异常；
4. **处理**分情况：录入错误修正、系统数据补回、或确认为真实异常保留。

我们会在实战中演示如何生成一份'清洗质量报告'，让每个清洗操作都可追溯。"

**【技术映射：异常值处理 = 体检发现血压偏高——先标记复测（标记），查生活习惯（追溯），再决定是否开药（处理），不能直接说'你没病'（无声删除）】**

**小白**："那 NaN、NaT、None 这些有啥区别？我以前在 Python 里只用过 None。"

**大师**："这就是 pandas 缺失值体系最微妙的地方：

| 值 | 类型 | 来源 | 场景 |
|----|------|------|------|
| `None` | Python NoneType | 用户输入、数据库 NULL | 通用，但性能差 |
| `np.nan` | NumPy float64 | 数学运算结果（如 0/0） | 数值列缺失标记 |
| `pd.NaT` | pandas NaTType | 日期运算结果 | 日期列缺失标记 |
| `pd.NA` | pandas NAType | pandas 1.0+ | 统一的 nullable 缺失标记 |

pandas 1.0 引入的 `pd.NA` 是'大一统'方案——它可以在数值、字符串、布尔、日期列中一致使用，不再需要根据 dtype 切换 NaN/NaT/None。配合 `dtype='Int64'`（注意大写的 I）或 `dtype='string'` 使用。"

**大师总结**："数据清洗是数据分析中占比最高（60-80%）也最被低估的环节。三条铁律：
1. 不要无声地改数据——清洗每一步都要有审计记录；
2. 缺失值 ≠ 零值 ≠ 异常值——三者的处理策略完全不同；
3. IQR 是异常值检测的第一道防线，业务规则是终极防线。

下面我们就把一份'脏'商品表洗成可信数据。"

```python
# generate_dirty_data.py
import pandas as pd
import numpy as np
from datetime import datetime, timedelta

np.random.seed(42)

n = 2000
categories = ['水果', '蔬菜', '肉类', '海鲜', '乳制品', '烘焙']
suppliers = ['供应商A', '供应商B', '供应商C', '供应商D', '供应商E']

# 基础数据
data = {
    '商品编码': [f'SKU{10000+i:05d}' for i in range(n)],
    '商品名称': [f'{np.random.choice(categories)}-{np.random.choice(["有机","优选","精选"])}-{i:04d}' for i in range(n)],
    '品类': np.random.choice(categories, n),
    '采购价': np.round(np.random.uniform(5, 200, n), 2),
    '零售价': np.round(np.random.uniform(10, 400, n), 2),
    '库存数量': np.random.randint(0, 500, n),
    '入库日期': [datetime(2025, 3, 1) + timedelta(days=np.random.randint(0, 90)) for _ in range(n)],
    '供应商': np.random.choice(suppliers, n),
    '保质期天数': np.random.choice([7, 15, 30, 60, 90, 180, 365], n),
}
df = pd.DataFrame(data)

# ---- 制造缺失值 ----
# 15% 的采购价缺失
mask_price = np.random.choice(df.index, int(n * 0.15), replace=False)
df.loc[mask_price, '采购价'] = np.nan

# 10% 的保质期缺失
mask_shelf = np.random.choice(df.index, int(n * 0.10), replace=False)
df.loc[mask_shelf, '保质期天数'] = np.nan

# 5% 的供应商缺失
mask_supplier = np.random.choice(df.index, int(n * 0.05), replace=False)
df.loc[mask_supplier, '供应商'] = None  # None 在 object 列中为缺失

# 3% 的入库日期缺失
mask_date = np.random.choice(df.index, int(n * 0.03), replace=False)
df.loc[mask_date, '入库日期'] = pd.NaT

# ---- 制造异常值 ----
# 5 个极端零售价（录入错误：小数点错位）
outlier_idx = np.random.choice(df.index, 5, replace=False)
df.loc[outlier_idx, '零售价'] = df.loc[outlier_idx, '零售价'] * 100

# 保留原始生成几个负库存（退货数据）
neg_stock_idx = np.random.choice(df.index, 8, replace=False)
df.loc[neg_stock_idx, '库存数量'] = np.random.randint(-50, -1, 8)

# 2 个异常采购价（0 或负数）
df.loc[np.random.choice(df.index, 2, replace=False), '采购价'] = 0

# 保持一份干净副本用于对比
df.to_csv('product_master_dirty.csv', index=False, encoding='utf-8-sig')
print(f"已生成脏数据: {len(df)} 条")
print(f"  缺失值统计:\n{df.isna().sum()}")
```

#### 步骤 2：缺失值处理策略

**目标**：针对不同列的缺失值，采用不同填充策略并记录清洗日志。

```python
# step2_missing_handle.py
import pandas as pd
import numpy as np

df = pd.read_csv('product_master_dirty.csv', parse_dates=['入库日期'])

# 初始化清洗日志
cleaning_log = []
original_null_counts = df.isna().sum().to_dict()

# === 策略 1：采购价 —— 用同品类中位数填充 ===
print("处理采购价缺失...")
median_by_cat = df.groupby('品类')['采购价'].transform('median')
fill_count_price = df['采购价'].isna().sum()
df['采购价'] = df['采购价'].fillna(median_by_cat)
cleaning_log.append(f"采购价: {fill_count_price} 条缺失 → 按品类中位数填充")

# === 策略 2：保质期天数 —— 按品类众数填充 ===
print("处理保质期缺失...")
mode_by_cat = df.groupby('品类')['保质期天数'].transform(lambda x: x.mode().iloc[0] if not x.mode().empty else np.nan)
fill_count_shelf = df['保质期天数'].isna().sum()
df['保质期天数'] = df['保质期天数'].fillna(mode_by_cat)
cleaning_log.append(f"保质期天数: {fill_count_shelf} 条缺失 → 按品类众数填充")

# === 策略 3：供应商 —— 缺失比例仅5%，填充"未知" ===
print("处理供应商缺失...")
fill_count_supplier = df['供应商'].isna().sum()
df['供应商'] = df['供应商'].fillna('未知')
cleaning_log.append(f"供应商: {fill_count_supplier} 条缺失 → 填充'未知'")

# === 策略 4：入库日期 —— 缺失仅3%，直接删除 ===
print("处理入库日期缺失...")
drop_count_date = df['入库日期'].isna().sum()
df = df.dropna(subset=['入库日期'])
cleaning_log.append(f"入库日期: {drop_count_date} 条缺失 → 删除行")

print(f"\n清洗后缺失值数量:\n{df.isna().sum()}")
print(f"\n===== 清洗日志 =====")
for log in cleaning_log:
    print(f"  • {log}")
```

#### 步骤 3：异常值检测（IQR + 业务规则）

**目标**：用 IQR 法检测数值异常，用业务规则检测逻辑异常。

```python
# step3_outlier_detect.py
import pandas as pd
import numpy as np

df = pd.read_csv('product_master_dirty.csv', parse_dates=['入库日期'])

# ===== 方法 1：IQR 法检测零售价异常 =====
Q1 = df['零售价'].quantile(0.25)
Q3 = df['零售价'].quantile(0.75)
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

print(f"零售价 IQR 检测:")
print(f"  Q1={Q1:.2f}  Q3={Q3:.2f}  IQR={IQR:.2f}")
print(f"  正常范围: [{lower_bound:.2f}, {upper_bound:.2f}]")

outlier_iqr = (df['零售价'] < lower_bound) | (df['零售价'] > upper_bound)
print(f"  IQR 检测到 {outlier_iqr.sum()} 个异常值")

# ===== 方法 2：业务规则检测 =====
# 零售价不能为负，不能超过 10000
outlier_biz_price = (df['零售价'] <= 0) | (df['零售价'] > 10000)
print(f"\n零售价业务规则检测: {outlier_biz_price.sum()} 个异常")

# 库存数量不能为负
outlier_biz_stock = df['库存数量'] < 0
print(f"库存为负: {outlier_biz_stock.sum()} 个异常")

# 采购价必须小于零售价
outlier_logic = df['采购价'] > df['零售价']
print(f"采购价 > 零售价: {outlier_logic.sum()} 个异常")

# ===== 合并所有异常标记 =====
df['异常标记'] = '正常'
df.loc[outlier_iqr & ~outlier_biz_price, '异常标记'] = 'IQR异常'
df.loc[outlier_biz_price, '异常标记'] = '价格异常'
df.loc[outlier_biz_stock, '异常标记'] = '库存为负'
df.loc[outlier_logic, '异常标记'] = '采购价高于零售价'
df.loc[outlier_biz_price & outlier_biz_stock, '异常标记'] = '多项异常'

print(f"\n===== 异常标记分布 =====")
print(df['异常标记'].value_counts().to_string())

# ===== 异常值处理 =====
# 零售价异常 → 用中位数替换
median_price = df.loc[df['异常标记'] == '正常', '零售价'].median()
df.loc[df['异常标记'].str.contains('价格异常'), '零售价'] = median_price
print(f"\n价格异常零售价已用中位数 {median_price:.2f} 替换")

# 采购价异常（0值）→ 用中位数替换
median_purchase = df.loc[df['采购价'] > 0, '采购价'].median()
df.loc[df['采购价'] <= 0, '采购价'] = median_purchase
print(f"0值采购价已用中位数 {median_purchase:.2f} 替换")

# 库存为负 → 标记待人工复核
print(f"\n负库存商品需人工复核:")
neg_stock = df[df['异常标记'].str.contains('库存为负')]
print(neg_stock[['商品编码', '商品名称', '库存数量', '异常标记']].to_string(index=False))
```

#### 步骤 4：生成清洗质量报告

**目标**：输出清洗前后的数据质量对比报告，做到可追溯。

```python
# step4_quality_report.py
import pandas as pd
import numpy as np
from datetime import datetime

df_raw = pd.read_csv('product_master_dirty.csv', parse_dates=['入库日期'])
# （此处假设已执行过步骤2和3的清洗）
# 实际项目中，以上步骤整合为一个函数

def generate_cleaning_report(df_before, df_after, report_path='cleaning_report.csv'):
    """生成清洗前后数据质量对比报告"""
    report_rows = []

    # 行数对比
    report_rows.append({
        '检查项': '总行数', '清洗前': len(df_before), '清洗后': len(df_after),
        '差异': len(df_before) - len(df_after), '说明': '删除行数'
    })

    # 各列缺失值对比
    for col in df_before.columns:
        null_before = df_before[col].isna().sum()
        null_after = df_after[col].isna().sum()
        if null_before > 0 or null_after > 0:
            report_rows.append({
                '检查项': f'{col} 缺失值',
                '清洗前': null_before,
                '清洗后': null_after,
                '差异': null_before - null_after,
                '说明': f'清洗了 {null_before - null_after} 个缺失值'
            })

    # 数值列统计对比（以零售价为例）
    for col in ['零售价', '采购价', '库存数量']:
        report_rows.append({
            '检查项': f'{col} 均值',
            '清洗前': round(df_before[col].mean(), 2),
            '清洗后': round(df_after[col].mean(), 2),
            '差异': round(abs(df_before[col].mean() - df_after[col].mean()), 2),
            '说明': '均值变化（应大幅修正异常值影响）'
        })
        report_rows.append({
            '检查项': f'{col} 标准差',
            '清洗前': round(df_before[col].std(), 2),
            '清洗后': round(df_after[col].std(), 2),
            '差异': round(abs(df_before[col].std() - df_after[col].std()), 2),
            '说明': '标准差变化（应显著减小）'
        })

    report_df = pd.DataFrame(report_rows)
    report_df.to_csv(report_path, index=False, encoding='utf-8-sig')
    print(f"清洗质量报告已导出至 {report_path}")
    return report_df

report = generate_cleaning_report(df_raw, df_raw)  # 示意
print(report.to_string(index=False))
```

### 3.4 完整主流程

```python
# main_cleaning.py
"""商品数据清洗主流程——可复现、可追溯"""
import pandas as pd
import numpy as np
from datetime import datetime

print(f"[{datetime.now():%H:%M:%S}] 开始数据清洗...")
df = pd.read_csv('product_master_dirty.csv', parse_dates=['入库日期'])
original_rows = len(df)

# 1. 缺失值处理
df['采购价'] = df['采购价'].fillna(df.groupby('品类')['采购价'].transform('median'))
df['保质期天数'] = df['保质期天数'].fillna(df.groupby('品类')['保质期天数'].transform(lambda x: x.mode().iloc[0]))
df['供应商'] = df['供应商'].fillna('未知')
df = df.dropna(subset=['入库日期'])

# 2. 异常值检测与处理
# 零售价 IQR
Q1, Q3 = df['零售价'].quantile(0.25), df['零售价'].quantile(0.75)
IQR = Q3 - Q1
price_outlier = (df['零售价'] < Q1 - 1.5 * IQR) | (df['零售价'] > Q3 + 1.5 * IQR)
df.loc[price_outlier, '零售价'] = df.loc[~price_outlier, '零售价'].median()

# 采购价 0 值修正
df.loc[df['采购价'] <= 0, '采购价'] = df.loc[df['采购价'] > 0, '采购价'].median()

# 3. 衍生列
df['毛利率'] = ((df['零售价'] - df['采购价']) / df['零售价'] * 100).round(2)
df['过期日期'] = df['入库日期'] + pd.to_timedelta(df['保质期天数'], unit='D')

# 4. 质量校验
assert df['零售价'].min() > 0, "存在非正零售价！"
assert df['采购价'].min() > 0, "存在非正采购价！"
assert df['毛利率'].max() < 100, "毛利率超过 100%！"

# 5. 导出
df.to_csv('product_master_clean.csv', index=False, encoding='utf-8-sig')
print(f"  清洗完成: {original_rows} → {len(df)} 行")
print(f"  ✓ 已导出 product_master_clean.csv")
```

### 3.5 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `fillna(0)` 导致均值严重偏差 | 0 是合法值不是缺失语义 | 用中位数/均值填充并记录 |
| `df[col].isna()` 对 None 不生效 | object 列中 None 未被识别为 NA | 读取时指定 `na_values=['None']` 或读取后 `df.replace('None', np.nan)` |
| `df.mean()` 静默跳过 NaN | `skipna=True` 是默认行为 | 先检查 `df.isna().sum()` 确认缺失情况 |
| IQR 边界把正常值当异常 | 数据本身是偏态分布 | 调整系数 1.5→3.0 或结合业务规则 |
| `interpolate` 填充不合理值 | 线性插值假设数据是单调的 | 对有季节性趋势的数据用 `method='polynomial'` |
| `dropna()` 删除了不该删的重要记录 | 未检查缺失列的重要性 | 对关键字段用 `dropna(subset=[...])` 精准删除 |

```python
import pandas as pd

def test_fillna_median():

def test_iqr_outliers():
    """验证 IQR 能检测异常值"""
    data = pd.Series([1, 2, 3, 4, 100])
    Q1, Q3 = data.quantile(0.25), data.quantile(0.75)
    IQR = Q3 - Q1
    lower, upper = Q1 - 1.5*IQR, Q3 + 1.5*IQR
    outliers = (data < lower) | (data > upper)
    assert outliers.sum() == 1  # 只有 100 是异常值
    assert outliers.iloc[4] is True
```

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 清洗方案 | Excel 手工处理 | SQL 清洗 |
|------|----------------|---------------|---------|
| 缺失值处理 | fillna/ interpolate/ dropna 无缝衔接 | 填充或删除需逐个操作 | IS NULL + COALESCE 直观 |
| 异常值检测 | IQR + 业务规则组合校验 | 只能靠肉眼和经验 | 需嵌套查询或窗口函数 |
| 可追溯性 | 日志 + 质量报告可自动生成 | 无，靠人工记录 | 可记录清洗 SQL |
| 批量处理 | 脚本一次性处理多列 | 逐列筛选修改 | CTE 链式处理 |
| 统计方法 | 内建分位数/标准差 | 有限 | 有限但可用 |
| 学习曲线 | 中等 | 低 | 低（SQL 用户）|

### 4.2 适用场景

- **适用场景**：
  1. 日常运营数据的定期清洗——每月/周的商品/订单/用户数据质量维护
  2. 机器学习模型的数据预处理——缺失值填充 + 异常值处理是标准步骤
  3. 数据仓库 ETL 的质量前置检查——在数据入仓前发现问题
  4. 多源异构数据的标准化清洗——不同系统导出的数据格式和规则不一致
  5. 需要审计追溯的数据合规场景——金融、医疗等行业
- **不适用场景**：
  1. 实时数据流清洗——pandas 批处理模式不适合流式
  2. 仅需简单过滤（如只保留某状态）的场景

### 4.3 注意事项

- **NA vs NaN vs NaT vs None**：pandas 1.0+ 的 `pd.NA` 是推荐的统一缺失标记，配合 nullable dtype（如 `Int64`、`string`）使用；
- **fillna 的 inplace**：`fillna(inplace=True)` 在新版 pandas 中已弃用，请用 `df['col'] = df['col'].fillna(value)`；
- **IQR 的边界值**：系数 1.5 是约定俗成，对极度偏态的数据可调整到 3.0；
- **异常值不一定是错误**：促销活动期间价格低于 IQR 下界是正常的——结合业务上下文判断。

### 4.4 常见踩坑经验

1. **用 `df[col].fillna(df[col].mean(), inplace=True)` 结果没生效**：`inplace=True` 只是语法糖，pandas 内部很多方法是先复制再操作。建议用 `df[col] = df[col].fillna(...)` 显式赋值。
2. **`df.isna()` 对空字符串 `''` 返回 False**：空字符串不是缺失值。如果业务上 `''` 等于缺失，需要 `df.replace('', np.nan)` 先做预处理。
3. **`groupby` 求中位数填充后出现 NaN**：这是因为某些分组的所有值都缺失，`median()` 返回 NaN。解决方案：加 `.fillna(全局中位数)` 做兜底，例如 `df['val'].fillna(df.groupby('cat')['val'].transform('median')).fillna(df['val'].median())`
