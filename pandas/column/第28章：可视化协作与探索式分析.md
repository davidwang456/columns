# 第28章：可视化协作与探索式分析

## 1. 项目背景

某电商平台的数据分析师小洪收到一个紧急需求：CEO 明天要在投资人会议上展示"平台用户增长诊断报告"，需要回答：用户增长是否健康？新增用户主要来自哪些渠道？老用户留存如何？用户活跃度的分布是怎样的？

数据已经在数据库里，小洪也熟悉 pandas。但他平时的工作流程是：用 pandas 做数据清洗和聚合 → 把结果导出 CSV → 用 Excel/Tableau 画图 → 截图放到 PPT 里。这个流程有两个致命问题：

**痛点一：分析→可视化的反馈循环太长**。pandas 算出"各渠道周新增用户趋势"需要 2 分钟，导出 CSV→打开 Tableau→拖拽字段→调整图表样式需要 20 分钟。如果 CEO 问"能不能分城市看一下"，整个流程需要重走一遍。

**痛点二：探索式分析效率低**。小洪在分析前不知道数据"长什么样"——有多少列？各列的分布如何？有无异常值？他用 `df.head()` 和 `df.describe()` 看，但一张数字表远不如一张图直观。他经常分析了一半才发现某列 50% 是空值。

**痛点三：Notebook 分析结果无法直接交付**。小洪在 Jupyter Notebook 里做了完整的分析（含图表），但 CEO 要的是 PPT。他只能在 Notebook 里右键保存图片，再一张张插入 PPT——10 张图就要操作 10 次。

痛点流程：

```
pandas 分析 (2分钟)
  └── 导出 CSV → Tableau/Excel 绘图 (20分钟) → PPT
        ├── 改需求 → 重新导出 → 重新绘图
        ├── 探索阶段看不到数据分布 → 分析了一半才发现问题
        └── Notebook 图表 → 手工截图 → PPT
```

本章将介绍 pandas 的原生可视化能力——plot API（线图/柱图/散点/直方图/箱线图/热力图）、describe 探索、以及从 Notebook 到交付的协作方式。

## 2. 项目设计：剧本式交锋对话

> 场景：小洪在加班插 PPT 图表，小胖路过看到他桌上一堆截图。

**小胖**：（吃着饼干）"洪哥，你这不是 pandas 里一行 `df.plot()` 就能画的图吗？为什么还要导出 CSV 再开 Tableau？"

**小洪**："pandas 画的图太丑了！默认样式没法拿去给 CEO 看。"

**大师**："这是个常见的误解。pandas 的 `plot` 是 matplotlib 的封装——你用 `df.plot()` 画出来的图和 matplotlib 完全一样。只要你懂几个样式设置，pandas 画出来的图可以直接用于汇报：

```python
import matplotlib.pyplot as plt

# 设置中文字体
plt.rcParams['font.sans-serif'] = ['SimHei']

# pandas 一行画图 + matplotlib 调样式
df.plot(x='date', y='new_users', kind='line', figsize=(10,5),
        title='周新增用户趋势', grid=True)
plt.ylabel('新增用户数')
plt.tight_layout()
plt.savefig('用户趋势.png', dpi=150)
```

3 行代码画图，2 行代码调样式——总共不超过 10 行。你的 Tableau 流程是 15 分钟。"

**【技术映射：pandas plot = 自带相机的手机——日常拍照足够了，不需要每次都去影楼（Tableau）】**

**小白**："pandas 的 plot 支持哪些图表类型？"

**大师**："`df.plot(kind='...')` 支持 13 种图表类型：

| kind | 图表类型 | 适用场景 |
|------|---------|---------|
| `'line'` | 折线图(默认) | 趋势分析 |
| `'bar'` / `'barh'` | 柱状图/水平柱状图 | 分类对比 |
| `'hist'` | 直方图 | 分布分析 |
| `'box'` | 箱线图 | 异常值/分布 |
| `'scatter'` | 散点图 | 相关性 |
| `'area'` | 面积图 | 累积/占比趋势 |
| `'pie'` | 饼图 | 比例分布 |
| `'density'` / `'kde'` | 密度曲线 | 平滑分布 |
| `'hexbin'` | 六边形箱图 | 大数据散点 |
| `'heatmap'` (需 seaborn) | 热力图 | 相关性矩阵 |

覆盖了 90% 的日常分析需求。"

**【技术映射：plot kind = 工具箱里的不同工具——锤子(bar)、尺子(line)、放大镜(hist)——选对工具才能解决问题】**

**小胖**："那探索式分析呢？就是数据拿到手先看看长啥样？"

**大师**："用 pandas 的三板斧：

```python
# 1. 看结构
df.info()          # 列名、dtype、非空行数
df.describe()      # 数值列的统计摘要
df.describe(include='object')  # 字符串列的统计

# 2. 看分布（一行画多张子图）
df.hist(figsize=(12,8), bins=50)  # 所有数值列的直方图
df['category'].value_counts().plot(kind='barh')  # 类别分布

# 3. 看关系
df.plot(kind='scatter', x='amount', y='frequency', alpha=0.3)
pd.plotting.scatter_matrix(df[['a','b','c']], figsize=(10,10))
```

先跑一遍三板斧——通常能发现 80% 的数据问题：异常值、分布偏态、类别不平衡——不需要看任何一行数据。"

**小白**："那分析结果怎么分享给团队？截图也不方便协作啊。"

**大师**："三个层次的分享方式：

1. **Jupyter Notebook**：数据分析师的标准工作台——代码+图表+注释在一起。导出为 HTML 分享。
2. **Streamlit / Dash**：把 pandas 分析构建成交互式 Web App——别人可以自己选参数看不同维度。
3. **自动报告**：把 matplotlib 的 `savefig` 结合 openpyxl 插入 Excel——生成带图表的自动化报告（第 14 章已讲）。

对 CEO 汇报这种场景：用 pandas + matplotlib 画好图，`savefig` 保存为 PNG，再插入 PPT 模板——写一个脚本批量生成，而不是手工截图。"

**大师总结**："pandas 可视化的三个核心能力：
1. `plot(kind=...)` —— 13 种图表一行画
2. `describe` + `hist` + `corr` —— 三板斧快速探索
3. `savefig` + Excel/HTML 导出 —— 无缝嵌入交付物

记住：**数据分析的最后一公里不是'算出数字'，而是'让别人看懂数字'。**"

### 3.2 模拟增长数据

```python
# generate_growth_data.py
import pandas as pd
import numpy as np

np.random.seed(42)
dates = pd.date_range('2024-07-01', '2025-03-31', freq='W-MON')
N = len(dates)

channels = ['抖音','微信','应用商店','地推','自然流量']
# 渠道占比随时间变化（抖音占比上升）
douyin_share = np.linspace(0.2, 0.45, N)

data = []
for i, d in enumerate(dates):
    total = int(np.random.normal(8000, 1500) + i * 50)  # 缓慢增长
    total = max(total, 5000)
    shares = [douyin_share[i]]
    remaining = 1 - douyin_share[i]
    for _ in range(3):
        s = remaining * np.random.uniform(0.2, 0.5)
        shares.append(s)
        remaining -= s
    shares.append(remaining)

    for ch, sh in zip(channels, shares):
        data.append({'week': d, 'channel': ch, 'new_users': int(total * sh)})

df = pd.DataFrame(data)

# 添加日期维度
df['month'] = df['week'].dt.month
df['quarter'] = df['week'].dt.quarter
df.to_csv('growth_data.csv', index=False)
print(f"已生成 {len(df)} 条增长数据")
```

**目标**：用热力图展示数值变量间的相关性。

```python
# step4_heatmap.py
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns  # 可选：pip install seaborn

plt.rcParams['font.sans-serif'] = ['SimHei']

df = pd.read_csv('growth_data.csv', parse_dates=['week'])

# 按周汇总，添加更多衍生指标
weekly_kpi = df.groupby('week').agg(总新增=('new_users','sum'),
                                    渠道数=('channel','nunique')).reset_index()
weekly_kpi['增长率'] = weekly_kpi['总新增'].pct_change() * 100

# 相关性矩阵
corr_matrix = weekly_kpi[['总新增','渠道数','增长率']].corr()

# 热力图
plt.figure(figsize=(8,6))
sns.heatmap(corr_matrix, annot=True, fmt='.2f', cmap='RdBu_r',
            center=0, square=True, linewidths=0.5)
plt.title('KPI 相关性矩阵')
plt.tight_layout()
plt.savefig('heatmap_corr.png', dpi=120)
print("OK 热力图已保存")
```

### 3.5 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 中文显示为方框 | 默认字体不含中文 | `plt.rcParams['font.sans-serif'] = ['SimHei']` |
| 保存的图空白一片 | `plt.savefig` 在 `plt.show()` 之后 | savefig 要在 show 之前调用 |
| 子图之间重叠 | 默认布局紧凑度不够 | `plt.tight_layout()` |
| plot 返回的对象类型不统一 | kind='line' 返回 Axes, kind='hist' 返回 array | 统一用 `plt.subplots()` 先创建画布 |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas plot | matplotlib 原生 | Tableau/BI |
|------|------------|---------------|-----------|
| 代码量 | 1-3 行 | 5-15 行 | 0 行（拖拽） |
| 灵活性 | 中 | 极高 | 受限于 UI |
| 与数据流集成 | 无缝（同一个脚本） | 需额外代码 | 需导出导入 |
| 交互性 | 无（静态图） | 无 | 高 |
| 学习成本 | 低 | 中 | 中 |
| 适用场景 | 快速探索/自动化报告 | 出版级图表 | 交互式仪表盘 |

### 4.2 适用场景

- **适用场景**：
  1. 数据探索阶段快速了解数据分布和趋势
  2. 自动化日报中的趋势图和对比图
  3. Jupyter Notebook 中的交互式分析
  4. 需要和 pandas 数据处理无缝衔接的可视化
- **不适用场景**：
  1. 需要高度自定义的出版级图表——用 matplotlib/seaborn 原生 API
  2. 需要交互式下钻的 BI 看板——用 Tableau/Streamlit

### 4.3 注意事项

- **中文字体**：Windows 用 `SimHei`，macOS 用 `Heiti SC`，Linux 可能需要安装中文字体包；
- **子图布局**：`df.plot(subplots=True)` 会把每列画一个子图——注意控制 `layout=(rows, cols)`；
- **大数据散点图**：`scatter` 对百万级点会很慢——用 `alpha` 调透明度或采样。

### 4.4 常见踩坑经验

1. **plot 后 plt.show() 清了画布**：在脚本中不需要 `plt.show()`，直接 `plt.savefig()`。`show()` 在某些后端会清空画布，导致 savefig 出空白图。使用非交互式后端：`import matplotlib; matplotlib.use('Agg')` 在脚本头部声明。
2. **多图保存时后一张覆盖前一张**：`plt.savefig()` 后没有 `plt.close()`——后续的图会继续画在同一个 Figure 上。每次 savefig 后调用 `plt.close('all')` 或使用 `plt.figure()` 创建新画布。最佳实践：使用 `with plt.style.context('seaborn-v0_8')` 或显式地 `fig, ax = plt.subplots()` 然后 `df.plot(ax=ax)`。
3. **时间轴刻度太密**：`df.plot()` 自动生成的日期刻度可能重叠。用 `plt.gca().xaxis.set_major_locator(mdates.MonthLocator())` 调整刻度密度。更完整的时间轴美化：设置 `plt.gcf().autofmt_xdate(rotation=45)` 自动旋转日期标签。
4. **`df.hist()` 默认为每列生成独立子图但布局不理想**：当 DataFrame 有 20+ 列时，`df.hist(figsize=(15,12), bins=30, layout=(5,4))` 显式指定行列布局，避免子图太密或太小。
5. **在 Jupyter Notebook 中 `df.plot()` 和 `plt.show()` 的行为不一致**：Notebook 中 `%matplotlib inline` 会自动显示图表，此时 `plt.show()` 是多余的。在脚本和 Notebook 之间迁移代码时，统一使用不含 `plt.show()` 的写法，仅在脚本末尾用 `plt.savefig()` 保存。

### 4.5 探索式分析的进阶技巧

除了基础的三板斧，以下技巧可以加速数据探索：

**快速筛选高价值变量**：用 `corr()` 矩阵快速找出相关性最高的指标对：
```python
corr = df.select_dtypes(include='number').corr()
high_corr = corr.where(np.triu(np.ones(corr.shape), k=1).astype(bool)).stack()
high_corr[high_corr.abs() > 0.7]  # 高度相关的变量对
```

**分组分布对比**：用 `df.groupby('category').describe()` 横向对比各组的统计分布，比逐组画图更快定位差异。

**时间序列快速诊断**：用 `df.set_index('date').plot(subplots=True, layout=(n,1), figsize=(10, 2*n))` 在同一个 Figure 中纵列排列所有时间序列指标，一眼看出趋势背离。

**Pandas Profiling**：`pip install ydata-profiling` 后一行 `df.profile_report()` 生成完整的数据探索 HTML 报告——包含缺失值、分布、相关性、警告等全部信息。适合在团队中分享数据的"第一印象"。
