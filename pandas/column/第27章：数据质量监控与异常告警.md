# 第27章：数据质量监控与异常告警

## 1. 项目背景

某在线教育平台的用户画像表（user_profile）每天凌晨由 ETL 管道从 5 个数据源抽取合并生成，约 300 万行。这张表被 20+ 下游任务引用——推荐系统、运营分析、风控模型、财务结算都依赖它。任何数据质量问题都会被放大并传导到下游。

数据运维工程师小江负责这张表的质量保障。上个月发生了三次严重的数据质量问题：

- **事件一**：上游数据源某天少传了一个省份的用户数据——用户画像表少了 60 万行，推荐系统的召回率下降了 15%，但直到 3 天后运营才发现推荐效果变差。
- **事件二**：ETL 中一个 JOIN 条件被人误改成 LEFT JOIN（应该是 INNER JOIN）——导致 user_level 字段出现大量 NULL，风控模型因为 `fillna(0)` 把所有新用户都标为了"零风险"。
- **事件三**：数据库迁移后 `created_at` 字段的时区从 UTC 变成了 Asia/Shanghai——所有时间戳偏移了 8 小时，日报中的"今日新增用户"总是对不上。

**痛点一：质量问题发现滞后**。三次问题中最早的一次在下游发现时已经延迟了 3 天——"没有数据不对"这个事实本身就是最大的问题。

**痛点二：检查逻辑散落各处**。小江之前写过一些质量检查脚本，但散落在 5 个不同文件里——有的用 print、有的用 assert、有的根本没被执行过。没有统一的执行框架和输出格式。

**痛点三：告警缺乏阈值和趋势判断**。"用户数比昨天少了"是正常波动还是异常？小江一开始设置了固定阈值"< 200 万就告警"，但节假日用户数少是正常的——告警整天响，大家已经麻木了。

痛点流程：

```
用户画像表 (300万行)
  ├── 上游少传数据 → 3天后才发现 → 推荐召回暴跌
  ├── JOIN 条件误改 → NULL 被当零 → 风控失效
  └── 时区偏移 8h → 日报对不上 → 无人告警
```

本章将用 pandas 构建一套数据质量监控系统——覆盖完整性、唯一性、合法性、及时性、一致性五个维度，配合统计阈值和趋势检测，让质量异常在 5 分钟内被发现。

## 2. 项目设计：剧本式交锋对话

> 场景：小江在复盘"推荐系统召回率暴跌 15%"的事故，小胖正在旁边吃鸡爪。

**小胖**：（舔着手指）"数据少了 60 万行——这还不好发现吗？select count 一下不就行了！"

**大师**："小胖，count 只能发现'少了'，但今天数据少 2% 是节假日效应还是上游挂了？昨天的数据少了 5% 是因为真的少了还是统计窗口变了？单看一个 count 数字不足以判断异常——你需要一个'基线'。"

**小白**："什么是基线？怎么建立？"

**大师**："基线就是**正常状态下的统计分布**。比如'每日用户数'——你可以统计过去 30 天的均值和标准差。如果今天的数据偏离均值超过 2 个标准差，就告警。这就是 3-sigma 规则。

pandas 实现起来非常简单：

```python
# 过去 30 天的每日用户数
daily_users = df.groupby('date')['user_id'].nunique()
mean = daily_users.mean()
std = daily_users.std()

# 今天的用户数
today_users = daily_users.iloc[-1]

# 判断是否异常
is_anomaly = abs(today_users - mean) > 2 * std
```

这就是'统计基线告警'——自动适应数据的自然波动，不会因为 2% 的正常波动就告警。"

**【技术映射：统计基线 = 人的体温——36.5℃ 是基线，37.5℃ 是感冒（>2σ），35℃ 也是异常（<2σ），而不是固定阈值"超过 38℃ 才算发烧"】**

**小白**："那除了行数，还需要监控什么？"

**大师**："数据质量的五个经典维度：

| 维度 | 检查内容 | pandas 实现 |
|------|---------|------------|
| **完整性** | 有没有缺行？关键列有没有 NULL？ | `df.shape[0]`, `df['col'].isna().sum()` |
| **唯一性** | 主键有没有重复？ | `df['id'].duplicated().sum()` |
| **合法性** | 值在合法范围内吗？ | `df['age'].between(0,120).all()` |
| **及时性** | 数据按时到达了吗？ | `df['date'].max()` |
| **一致性** | 汇总值等于明细和吗？ | `abs(df['amount'].sum() - expected) < 0.01` |

一个完善的质量检查应该覆盖这五个维度——就像体检不是只测体温，还要测血压、心率、血常规。"

**【技术映射：五维质量 = 体检报告——完整（器官不缺）、唯一（没有两个心脏）、合法（血糖不超限）、及时（数据新鲜）、一致（左右手血压不会差太多）】**

**小胖**："那指标漂移是什么意思？听起来很高级。"

**大师**："指标漂移就是**数据的统计分布发生了变化**。比如过去 30 天，用户年龄的均值一直是 28 岁，标准差 5 岁——突然今天均值变成 35 岁了。这种变化可能是数据问题（比如少拉了一个省的数据），也可能是业务变化（比如做了一次针对中年人的营销活动）。

检测方法：用 `describe()` 获取分布的均值、标准差、分位数，和过去 30 天的基线对比。

```python
today_stats = df['age'].describe()
baseline_stats = historical_df['age'].describe()
shift_pct = abs(today_stats['mean'] - baseline_stats['mean']) / baseline_stats['mean']
if shift_pct > 0.1:  # 均值偏移超过 10%
    alert("年龄分布疑似漂移")
```

**大师总结**："数据质量监控的三层架构：
1. **规则层**：完整性/唯一性/合法性 —— 硬约束（任何违反都告警）
2. **统计层**：均值和标准差偏离 —— 软约束（超过 N 个 sigma 告警）
3. **趋势层**：逐日变化幅度和方向 —— 预测性（趋势变坏提前告警）

下面我们就给用户画像表搭建这样的三层监控。"

### 3.3 分步实现

#### 步骤 1：五维质量检查

**目标**：对一张 DataFrame 执行完整性、唯一性、合法性、及时性、一致性检查。

```python
# step1_five_dimensions.py
import pandas as pd
import numpy as np

def check_completeness(df, key_cols):
    """完整性：行数和关键列的非空比例"""
    checks = {}
    checks['total_rows'] = len(df)
    for col in key_cols:
        null_count = df[col].isna().sum()
        checks[f'{col}_null_pct'] = round(null_count / len(df) * 100, 2)
    return checks

def check_uniqueness(df, key_col):
    """唯一性：主键重复数"""
    dup_count = df[key_col].duplicated().sum()
    return {'dup_count': dup_count, 'dup_pct': round(dup_count / len(df) * 100, 2)}

def check_validity(df, rules):
    """合法性：值是否在合理范围"""
    checks = {}
    for col, (lo, hi) in rules.items():
        if col in df.columns:
            invalid = (~df[col].between(lo, hi)).sum()
            checks[f'{col}_invalid'] = invalid
    return checks

def check_timeliness(df, date_col, max_delay_days=1):
    """及时性：最新数据日期距今多少天"""
    latest = df[date_col].max()
    delay = (pd.Timestamp.now() - latest).days
    return {'latest_date': str(latest.date()), 'delay_days': delay}

def check_consistency(df, total_col, detail_col, group_col):
    """一致性：汇总值是否等于明细和"""
    total = df[total_col].sum()
    detail_sum = df.groupby(group_col)[detail_col].sum().sum()
    diff = abs(total - detail_sum)
    return {'total': total, 'detail_sum': detail_sum, 'diff': diff, 'consistent': diff < 0.01}

# 示例使用
df = pd.DataFrame({
    'user_id': [1,2,3,4,4],
    'age': [25, 30, -5, 45, 150],
    'amount': [100, 200, 300, 400, 500],
    'date': pd.date_range('2025-03-31', periods=5),
})

print("完整性:", check_completeness(df, ['user_id','age']))
print("唯一性:", check_uniqueness(df, 'user_id'))
print("合法性:", check_validity(df, {'age': (0,120)}))
print("及时性:", check_timeliness(df, 'date'))
```

#### 步骤 2：统计基线检测——3-sigma 异常

**目标**：用滚动均值和标准差判断当前值是否异常。

```python
# step2_baseline_detection.py
import pandas as pd
import numpy as np

df = pd.read_csv('profile_quality.csv', parse_dates=['date'])

# 建立过去 30 天的基线（排除今天）
baseline = df.iloc[:-1]
today = df.iloc[-1]

# 各指标的均值和标准差
stats = {}
for col in ['user_count','avg_age','null_rate']:
    mean = baseline[col].mean()
    std = baseline[col].std()
    today_val = today[col]
    sigma = (today_val - mean) / std if std > 0 else 0

    stats[col] = {
        'mean': round(mean, 2), 'std': round(std, 2),
        'today': today_val, 'sigma': round(abs(sigma), 2),
        'status': 'ALERT' if abs(sigma) > 2 else 'OK'
    }

print("===== 3-Sigma 异常检测 =====")
for col, s in stats.items():
    status_icon = '!!' if s['status'] == 'ALERT' else 'OK'
    print(f"[{status_icon}] {col}: today={s['today']}, mean={s['mean']}, sigma={s['sigma']}")
```

#### 步骤 3：指标漂移检测

**目标**：检测分布的均值、分位数是否随时间漂移。

```python
# step3_drift_detection.py
import pandas as pd
import numpy as np

df = pd.read_csv('profile_quality.csv', parse_dates=['date'])

# 按 7 天窗口计算滚动均值
df['user_count_ma7'] = df['user_count'].rolling(7, min_periods=1).mean()
df['avg_age_ma7'] = df['avg_age'].rolling(7, min_periods=1).mean()
df['null_rate_ma7'] = df['null_rate'].rolling(7, min_periods=1).mean()

# 检测趋势：最近 3 天是否持续下降
recent = df.tail(3)
for col in ['user_count_ma7','avg_age_ma7']:
    if len(recent) >= 2:
        trend = '上升' if recent[col].iloc[-1] > recent[col].iloc[0] else '下降'
        change_pct = (recent[col].iloc[-1] - recent[col].iloc[0]) / recent[col].iloc[0] * 100
        if abs(change_pct) > 3:
            print(f"[ALERT] {col} 3日{trend} {change_pct:.1f}%")

print("\n最近 7 天指标趋势:")
print(df[['date','user_count_ma7','avg_age_ma7','null_rate_ma7']].tail(7).to_string(index=False))
```

#### 步骤 4：生成质量报告并告警

**目标**：将所有检查结果整合为一份质量报告。

```python
# step4_quality_report.py
import pandas as pd
import numpy as np
from datetime import datetime

df = pd.read_csv('profile_quality.csv', parse_dates=['date'])
baseline = df.iloc[:-1]
today = df.iloc[-1]

alerts = []

# 1. 完整性
user_count_today = today['user_count']

    alerts.append(f"[完整] 用户数 {user_count_today:,} < 历史最低90% ({threshold:,.0f})")

# 2. 唯一性
if today['dup_rate'] > 0.01:
    alerts.append(f"[唯一] 重复率 {today['dup_rate']*100:.1f}% > 1%")

# 3. 合法性
# （此处简化，实际应有更复杂的规则）

# 4. 统计异常
for col in ['user_count','avg_age','null_rate']:
    mean = baseline[col].mean()
    std = baseline[col].std()
    sigma = abs(today[col] - mean) / std if std > 0 else 0
    if sigma > 2:
        alerts.append(f"[统计] {col} 偏离 {sigma:.1f}σ (today={today[col]}, mean={mean:.0f})")

# 5. 趋势
if len(df) >= 3:
    recent = df.tail(3)['user_count']
    change = (recent.iloc[-1] - recent.iloc[0]) / recent.iloc[0] * 100

        alerts.append(f"[趋势] 用户数近3日变化 {change:+.1f}%")

# 输出报告
print(f"===== 数据质量报告 {datetime.now():%Y-%m-%d %H:%M} =====")
if alerts:
    print(f"异常数: {len(alerts)}")
    for a in alerts:
        print(f"  !! {a}")
```

### 3.4 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 基线数据包含异常日 | 异常日污染了基线统计 | 基线计算前用 IQR 剔除异常值 |
| std=0 导致 sigma 无限大 | 所有历史值相同 | 检查 std 是否为 0，为 0 则用固定阈值 |
| 节假日导致大量误报 | 节假日数据自然偏低 | 添加节假日标记，节假日用独立的基线 |
| 小样本波动大 | 前几天的数据太少 | min_periods 设置合理的阈值 |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 质量监控 | SQL 定时检查 | Great Expectations |
|------|---------------|-------------|---------------------|
| 灵活性 | 任意 Python 逻辑 | 受限于 SQL | 声明式 YAML 配置 |
| 部署 | 轻量，cron + 脚本 | 需数据库调度 | 需独立服务 |
| 统计基线 | 手写 rolling + sigma | 需存储过程 | 内置 |
| 告警 | 需自行接入（企业微信/邮件） | 同左 | 内置 |
| 学习成本 | 中 | 低 | 中高 |

### 4.2 适用场景

- **适用场景**：
  1. 每日定时运行的数据质量检查——ETL 的下游守门员
  2. 核心业务表的质量趋势监控（如用户表、订单表）
  3. 作为 CI/CD 的一部分，数据管道部署前的质检
  4. 小团队快速搭建的轻量监控方案
- **不适用场景**：
  1. 需要复杂数据契约管理的场景——用 Great Expectations/Pandera
  2. 需要分布式质量检查的 PB 级数据——用 Spark + Deequ

### 4.3 注意事项

- **告警疲劳**：阈值太敏感会导致频繁误报，团队逐渐忽视告警——阈值用 3σ（而非 2σ）减少误报；
- **基线更新**：长期趋势变化（如用户自然增长）会导致基线失效——每月重新计算基线；
- **节假日处理**：假期数据异常不等于数据质量问题——在质量报告中标记"非工作日"。

### 4.4 常见踩坑经验

1. **把所有检查放在一个函数里**：500 行的 `check_all()` 一旦某个检查抛异常就全部中断。改为每个检查独立函数，用 try/except 包裹，一个失败不影响其他。
2. **NaN 和 0 混淆**：`df['col'].mean()` 自动跳过 NaN——如果数据质量问题导致某列全是 NaN，mean() 返回 NaN 而不是 0，后续 sigma 计算全是 NaN。先判断 `df['col'].count() == 0`。
3. **百分比变化告警对小基数不适用**：某渠道昨天只有 10 个用户，今天变成 20 个——变化 +100%，但不代表数据质量问题。对用户数 < 100 的维度不计算百分比变化。
