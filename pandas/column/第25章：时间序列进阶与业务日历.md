# 第25章：时间序列进阶与业务日历

## 1. 项目背景

某券商研究所的量化分析师小程需要构建两个分析模型：A 股市场的个股收益率分析和节假日消费趋势预测。数据分别来自：

- **A 股行情数据**：每只股票的日线 OHLC（开盘/最高/最低/收盘价），覆盖 2023-2025 年
- **某电商平台节假日销售数据**：每天的分品类销售额和订单数

小程面临的挑战不是数据量大，而是**时间规则的复杂性**：

**痛点一：A 股交易日历不同于自然日历**。A 股一年约有 244 个交易日，而非 365 天。周六日休市、法定节假日休市、偶尔还有调休补班日开市。小程在计算"20 日滚动收益率"时，用 `rolling(20)` 得到的是"最近 20 个交易日"的滚动，而不是"最近 20 个自然日"——这恰好是他想要的，但参数设置错了的话就全错了。

**痛点二：节假日销量预测的日期对齐困难**。春节、国庆等长假期间的销量和平时完全不同，小程需要把"去年春节前 7 天"和"今年春节前 7 天"对齐比较——但春节是农历节日，每年公历日期不同。用自然日期根本无法对齐。

**痛点三：时区和夏令时导致跨国数据混乱**。公司在纽约和上海都有服务器，纽约服务器记录的订单时间是 EDT（东部夏令时），上海服务器记录的是 CST（中国标准时间）。一年中某些时段两者差 12 小时，某些时段差 13 小时。

痛点流程：

```
A股行情 + 电商销售
  ├── 交易日 vs 自然日 → rolling 参数选错
  ├── 农历节日对齐 → 自然日期无法对齐
  └── 时区/夏令时 → 跨时区数据偏差 1 小时
```

本章将深入 pandas 的时间序列高级特性——时区处理、DateOffset、CustomBusinessDay（自定义交易日历）、以及业务日历中的缺失日和补班日处理。

## 2. 项目设计：剧本式交锋对话

> 场景：小程在会上展示了他的 A 股分析，但"20 日收益率"的结果和 Wind 终端不一致。

**小胖**：（喝着旺仔牛奶）"20 日均值嘛！`df['close'].rolling(20).mean()` 一行搞定！结果不一样肯定是数据源的问题！"

**大师**："小胖，`rolling(20)` 是 20 个**数据点**，如果你拿到的数据是交易日数据（一年约 244 条），那这个 rolling 恰好是'20 个交易日'。但如果你拿的数据是自然日数据（一年 365 条，含周末的 NaN 行），rolling(20) 就变成'20 个自然日'——大概 14 个交易日。这就是为什么你的结果和 Wind 不一致。"

**小白**："那怎么让 rolling 按'交易日'而非'自然日'来算？"

**大师**："两个策略：

1. **只用交易日数据**：把周末和节假日的行全部删除，让 DataFrame 只有交易日，然后 `rolling(20)` 自然就是 20 个交易日。
2. **自定义交易日历**：用 `CustomBusinessDay` 定义哪些是交易日、哪些是节假日、哪些是补班日。

```python
from pandas.tseries.offsets import CustomBusinessDay
from pandas.tseries.holiday import AbstractHolidayCalendar

# 创建包含中国节假日的交易日历
class ChinaHolidayCalendar(AbstractHolidayCalendar):
    rules = [
        # 元旦、春节、清明、劳动节、端午、中秋、国庆
        Holiday('New Year', month=1, day=1),
        # ... 更多节假日规则
    ]

cbd = CustomBusinessDay(calendar=ChinaHolidayCalendar())
# 生成交易日序列
trading_days = pd.date_range('2025-01-01', '2025-12-31', freq=cbd)
```

有了交易日历，`date_range` 自动跳过非交易日，`rolling` 和 `shift` 都遵循这个日历。"

**【技术映射：CustomBusinessDay = 自定义日历——你自己圈出哪些天是工作日，pandas 只在这些天'上班'】**

**小白**："那日期偏移呢？比如'A 股 T+1 结算'——今天买入，明天（下一个交易日）才能卖出。"

**大师**："`DateOffset` 系列就是干这个的：

```python
# 普通天数偏移
pd.Timestamp('2025-03-28') + pd.DateOffset(days=1)  # 2025-03-29（自然日）

# 工作日偏移（跳过周末）
pd.Timestamp('2025-03-28') + pd.DateOffset(weekday=0)  # 下一个周一

# 自定义交易日偏移
pd.Timestamp('2025-03-28') + 1 * cbd  # 下一个交易日（跳过周末+节假日）
```

还有一个常用的 `BDay`（BusinessDay）——跳过周末但不跳过节假日。如果你的场景不需要处理节日，BDay 就够了。"

**【技术映射：DateOffset = 会看日历的秒表——知道什么时候该'跳'过周末和节日】**

**小胖**："那时区呢？上海和纽约差 12 小时——直接相减不行吗？"

**大师**："时区处理的三大原则：

1. **始终在读取时就附加时区**：`pd.to_datetime(ts_str, utc=True)` 或 `ts.tz_localize('Asia/Shanghai')`
2. **内部存储统一为 UTC**：所有时间戳在计算前转到 UTC，算完再转回本地时区展示
3. **用 `tz_convert` 而非手工加减小时**：`ts.tz_convert('US/Eastern')` 自动处理夏令时

```python
# 上海时间
shanghai = pd.Timestamp('2025-07-15 14:00:00', tz='Asia/Shanghai')
# 转为纽约时间（自动处理夏令时）
new_york = shanghai.tz_convert('America/New_York')
# 2025-07-15 02:00:00-04:00 (夏季差12小时，冬季差13小时)
```"

**大师总结**："本章是中级篇时间序列的进阶，核心掌握三件事：
1. 自定义交易日历（CustomBusinessDay + Holiday）
2. 时区的本地化和转换（tz_localize、tz_convert）
3. 日期频率和偏移（DateOffset、BDay、CDay）

一旦掌握了这三个，金融、物流、运营中几乎所有时间相关的问题都能迎刃而解。"

# generate_stock_data.py
import pandas as pd
import numpy as np

np.random.seed(42)

# 创建一个简单的交易日历（排除周末）
trading_days = pd.date_range('2023-01-01', '2025-03-31', freq='B')  # B = Business Day
# 简化：再排除几个模拟的节假日
holidays = ['2023-01-02','2023-01-23','2023-01-24','2023-01-25','2023-01-26','2023-01-27',
            '2023-04-05','2023-05-01','2023-10-02','2023-10-03','2023-10-04','2023-10-05','2023-10-06']
holiday_dates = pd.to_datetime(holidays)
trading_days = trading_days[~trading_days.isin(holiday_dates)]

# 模拟价格：随机游走
N = len(trading_days)
returns = np.random.normal(0.0005, 0.02, N)  # 日收益率
price = 100 * np.exp(np.cumsum(returns))       # 几何随机游走

df = pd.DataFrame({
    'date': trading_days,
    'open': np.round(price * np.random.uniform(0.99, 1.01, N), 2),
    'high': np.round(price * np.random.uniform(1.005, 1.03, N), 2),
    'low': np.round(price * np.random.uniform(0.97, 0.995, N), 2),
    'close': np.round(price, 2),
    'volume': np.random.randint(100000, 10000000, N),
})
df.to_csv('stock_daily.csv', index=False)
print(f"已生成 {len(df)} 个交易日数据 (2023-2025)")
```

### 3.3 分步实现

#### 步骤 1：CustomBusinessDay 自定义交易日历

**目标**：创建中国 A 股交易日历，生成正确的交易日序列。

```python
# step1_custom_bday.py
import pandas as pd
from pandas.tseries.holiday import AbstractHolidayCalendar, Holiday, nearest_workday
from pandas.tseries.offsets import CustomBusinessDay

# 定义中国 A 股节假日日历
class ChinaTradingCalendar(AbstractHolidayCalendar):
    rules = [
        Holiday('元旦', month=1, day=1),
        Holiday('春节-除夕', month=1, day=21, year=2023),
        Holiday('春节-初一', month=1, day=22, year=2023),
        Holiday('春节-初二', month=1, day=23, year=2023),
        Holiday('春节-初三', month=1, day=24, year=2023),
        Holiday('春节-初四', month=1, day=25, year=2023),
        Holiday('春节-初五', month=1, day=26, year=2023),
        Holiday('清明', month=4, day=5),
        Holiday('劳动节', month=5, day=1),
        Holiday('端午', month=6, day=22, year=2023),
        Holiday('中秋', month=9, day=29, year=2023),
        Holiday('国庆节', month=10, day=1),
        Holiday('国庆节2', month=10, day=2),
        Holiday('国庆节3', month=10, day=3),
    ]

# 创建交易日历
cbd = CustomBusinessDay(calendar=ChinaTradingCalendar())

# 生成 2023 年交易日
trading_days_2023 = pd.date_range('2023-01-01', '2023-12-31', freq=cbd)
print(f"2023 年交易日数: {len(trading_days_2023)}")
print(f"一月交易日: {trading_days_2023[trading_days_2023.month==1].tolist()}")
```

#### 步骤 2：DateOffset —— 交易日偏移

**目标**：用 BDay 和 CustomBusinessDay 做 T+N 日期计算。

```python
# step2_dateoffset.py
import pandas as pd
from pandas.tseries.offsets import BDay

df = pd.read_csv('stock_daily.csv', parse_dates=['date'])
df = df.set_index('date').sort_index()

# 自然日滚动均值（对比）
df['ma20_natural'] = df['close'].rolling(20).mean()

# 交易日滚动均值（数据本身就是交易日，rolling(20)=20个交易日）
df['ma20_trading'] = df['close'].rolling(20).mean()

print("自然日 vs 交易日 滚动均值（因数据只有交易日，两者相同）:")
print(df[['close','ma20_trading']].tail(5).round(2).to_string())

# === BDay 偏移 ===
today = pd.Timestamp('2025-03-28')  # 周五
tomorrow_bday = today + BDay(1)      # 下一个工作日 = 周一
print(f"\n今天: {today.date()} (周五)")
print(f"BDay(1): {tomorrow_bday.date()} (周一)")

# 10 个交易日后
t10 = today + BDay(10)
print(f"T+10: {t10.date()}")
```

#### 步骤 3：时区处理——tz_localize 和 tz_convert

**目标**：正确处理跨时区的时间戳。

```python
# step3_timezone.py
import pandas as pd

# === 无时区 → 本地化 ===
ts_naive = pd.Timestamp('2025-07-15 14:00:00')
ts_cst = ts_naive.tz_localize('Asia/Shanghai')  # 假设这是北京时间
ts_utc = ts_cst.tz_convert('UTC')
ts_ny = ts_cst.tz_convert('America/New_York')

print("时区转换:")
print(f"  原始(naive): {ts_naive}")
print(f"  北京(CST):    {ts_cst}")
print(f"  UTC:          {ts_utc}")
print(f"  纽约(EDT):    {ts_ny}")  # 夏令时差 12 小时

# === 读取时指定 UTC ===
ts_with_offset = pd.to_datetime('2025-07-15 14:00:00+08:00', utc=True)
print(f"\n带偏移量转 UTC: {ts_with_offset}")

# === 批量处理 ===
s = pd.Series([
    '2025-07-15 14:00:00+08:00',
    '2025-07-15 15:30:00+08:00',
    '2025-07-15 16:00:00+08:00',
])
dt = pd.to_datetime(s, utc=True)
print(f"\n批量转 UTC:\n{dt}")
```

#### 步骤 4：节假日销量对齐——用 offset 做同比

**目标**：用 DateOffset 实现"和去年同日比较"的排除节假日干扰。

```python
# step4_yoy_comparison.py
import pandas as pd
from pandas.tseries.offsets import BDay
import numpy as np

# 模拟每日销售数据
dates = pd.date_range('2024-01-01', '2025-03-31', freq='D')
np.random.seed(1)
sales = pd.DataFrame({
    'date': dates,
    'sales': np.random.randint(5000, 15000, len(dates)) + np.where(dates.weekday<5, 2000, -3000),
    'orders': np.random.randint(100, 400, len(dates)) + np.where(dates.weekday<5, 50, -80),
})

# 日环比：和前一天比
sales['prev_day'] = sales['sales'].shift(1)
sales['day_change_pct'] = ((sales['sales'] - sales['prev_day']) / sales['prev_day'] * 100).round(2)

# 周同比：和上周同一天比
sales['prev_week'] = sales['sales'].shift(7)
sales['week_change_pct'] = ((sales['sales'] - sales['prev_week']) / sales['prev_week'] * 100).round(2)

# 月同比：和 30 天前比
sales['prev_month'] = sales['sales'].shift(30)
sales['month_change_pct'] = ((sales['sales'] - sales['prev_month']) / sales['prev_month'] * 100).round(2)

print("===== 同比指标 (最近 7 天) =====")
print(sales[['date','sales','day_change_pct','week_change_pct']].tail(7).to_string(index=False))

# 注意：周同比 `shift(7)` 只适用于无缺失日的数据——如果有缺失日需先 reindex
```

### 3.4 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `tz_localize` 报 DST 歧义 | 夏令时回拨 1 小时导致重复时间 | 传 `ambiguous='infer'` 或 `ambiguous=[True,False]` |
| `CustomBusinessDay` 的节假日不生效 | 日历类的 rules 定义有误 | 检查 AbstractHolidayCalendar 的 observance 规则 |
| UTC 转换后小时数不对 | 原始时间被错误假设为 UTC | 先 `tz_localize` 到正确的本地时区，再 `tz_convert` |
| BDay 对 2025 年工作日计算不准 | 默认美国节假日 | 使用 CustomBusinessDay 加载自定义日历 |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 时间序列进阶 | Excel 日期函数 | Python datetime |
|------|-------------------|---------------|----------------|
| 交易日历 | CustomBusinessDay+Holiday 全自动 | 手工维护日期列表 | 需手写日历逻辑 |
| 时区处理 | tz_localize/convert 自动夏令时 | 几乎不支持 | pytz 手动处理 |
| 频率偏移 | DateOffset/BDay 语义清晰 | 手工公式 | timedelta 不支持工作日 |
| 节假日对齐 | offset + shift 结合 | 手工拉公式 | 手写比对逻辑 |
| 学习成本 | 高 | 低 | 中 |

### 4.2 适用场景

- **适用场景**：
  1. 金融时间序列分析（股票、期货、外汇）——交易日历是刚需
  2. 跨国业务数据分析——多时区时间戳的统一和转换
  3. 节假日营销效果评估——对齐去年同日（排除日历效应）
  4. 物流/供应链的时效计算（工作日 SLA）
- **不适用场景**：
  1. 不需要排除节假日的简单分析——`shift(1)` 就够了
  2. 农历/回历等非公历日历——pandas 不原生支持

### 4.3 注意事项

- **pandas 的 Holiday 规则**：`AbstractHolidayCalendar` 的 rules 是静态类属性——定义时就确定了，不能运行时动态修改；
- **BDay 不支持中国节假日**：默认的 `BDay` 只跳过周六日，需用 `CustomBusinessDay` 加载自定义日历；
- **时区本地化优于偏移量**：不要用 `+ pd.Timedelta(hours=8)` 手工调整时区——用 `tz_convert` 自动处理夏令时。

### 4.4 常见踩坑经验

1. **`pd.date_range('2025-01-01','2025-12-31', freq='B')` 生成的交易日和美国一样**：中国的节假日（春节、国庆）和美欧完全不同。必须加载自定义日历或用第三方库（如 `pandas_market_calendars`）。
2. **夏令时切换日 `tz_localize` 报 NonExistentTimeError**：夏令时开始那天（"spring-forward"）凌晨 2:00-3:00 不存在——`tz_localize(nonexistent='NaT')` 转为 NaT 或 `tz_localize(ambiguous='infer')` 自动推断。
3. **`shift(freq='B')` 和 `shift(1)` 的语义混淆**：`shift(freq='B')` 移动**索引的时间标签**（下个工作日），`shift(1)` 移动**数据值**（上一行）。两者完全不是一回事。
