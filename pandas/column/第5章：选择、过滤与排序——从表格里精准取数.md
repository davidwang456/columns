# 第5章：选择、过滤与排序——从表格里精准取数

## 1. 项目背景

某电商平台的风控团队每天需要从百万级的订单中筛选出"高价值异常订单"——定义是：订单金额超过 5000 元，且满足以下任一条件：同一用户 1 小时内下单超过 3 次、收货地址与 IP 归属地不一致、支付金额与订单金额不匹配。筛选结果需要按风险等级排序，并输出可复核的问题清单。

数据维度包括：订单号、用户 ID、下单时间、支付时间、商品金额、实付金额、优惠金额、收货省份、收货城市、IP 省份、IP 城市、订单状态。

风险分析师小李每天靠以下流程工作：

**痛点一：逐列筛选效率低下**。小李在 Excel 里先筛选"金额≥5000"，导出；再筛选"同一用户 1 小时下单≥3 次"，导出；再手工 VLOOKUP 比对地址不一致的订单。三个步骤分别在三个 Excel 文件里操作，一天要重复 10 次以上。

**痛点二：筛选逻辑变更时需要重做所有步骤**。上周风控规则新增了"订单实付金额与应付金额差异超过 20%"这一条，小李需要把之前的所有 Excel 筛选重新做一遍——因为不同条件是 AND 关系，但他之前是分步骤手工操作的，数据散落在不同文件。

**痛点三：SettingWithCopyWarning 引发的隐形 Bug**。小李尝试用 Python + pandas 写了个自动化脚本，但在链式索引操作后修改数据时，出现了奇怪的 SettingWithCopyWarning 警告。他忽略了这个警告继续用，结果筛选出的订单列表在后续修改中丢失了部分数据——因为他实际上是在一个临时拷贝上操作，原始 DataFrame 没有被更新。

此场景的核心挑战在于：**如何安全、高效、可维护地从大数据集中按复杂业务规则精准提取目标子集**。本章将系统讲解 pandas 的索引、选择和排序机制，从 `loc`/`iloc` 起步，到布尔索引和 `query` 方法，再到排序和链式操作的坑。

## 2. 项目设计：剧本式交锋对话

> 场景：小李把脚本发到团队 Review，代码里到处都是 `df[df['A']>0][df['B']<10]` 这样的链式索引。大师看到后皱起了眉头。

**小胖**：（指着屏幕）"大师，这代码有啥问题？我平时也这么写。先筛选 A 大于 0 的，再从结果里筛选 B 小于 10 的，逻辑很清楚啊。"

**大师**："逻辑没问题，但写法有隐患。你这一串 `df[...][...]` 叫**链式索引**（chained indexing）。第一个 `df[...]` 返回的是什么——是原 DataFrame 的一个视图（view）还是一个拷贝（copy）？答案是：**不确定**。取决于内部的内存布局，有时是视图有时是拷贝。"

**小白**："为什么不确定？这不是很可怕吗？"

**大师**："因为这取决于 pandas 内部的 BlockManager 如何存储数据。同一列类型相同的数据可能被合并存储，当你选择部分列时，可能直接返回原内存的视图，也可能触发拷贝。在 pandas 1.x 中这完全是未定义行为。好消息是 pandas 2.0+ 引入了 copy-on-write，默认行为更可预测——但这不意味着你可以随便写链式索引。"

**【技术映射：链式索引 = 给朋友指路说"往前走到红绿灯左转再右转"，中间可能走错——直接说地址（loc）最安全】**

**小胖**："那应该怎么写？"

**大师**："**用 `.loc` 一步到位**：

```python
# ❌ 链式索引——不确定是视图还是拷贝
df[df['amount'] > 5000]['status']

# ✅ 用 loc 一步完成
df.loc[df['amount'] > 5000, 'status']
```

而且如果要修改筛选出的数据，区别更大：

```python
# ❌ 链式赋值——可能出现 SettingWithCopyWarning
df[df['amount'] > 5000]['status'] = '高风险'

# ✅ 用 loc 一步完成
df.loc[df['amount'] > 5000, 'status'] = '高风险'
```

第二条 `loc` 写法保证修改的是原始 DataFrame，第一条可能改的是临时拷贝（改了等于没改）。"

**小白**："那 `loc` 和 `iloc` 能混合使用吗？比如我想选第 3-5 行，但要指定列名？"

**大师**："`loc` 和 `iloc` 的语义是独立的，一行代码中只能用一种。但你可以分步：

```python
# 先用 iloc 取行，再用 loc 取列
df.iloc[2:5].loc[:, 'amount':'category']
```

不过这样又退化成链式索引的风险——建议在数据分析中固定使用一种索引风格，要么全用 Index 标签（适合业务 ID），要么全用位置序号（适合自动化脚本）。"

**【技术映射：loc/iloc 选择 = 快递取件——要么用取件码（loc，按标签），要么报手机尾号（iloc，按位置），不能混着来】**

**小胖**："那复合条件怎么办？比如金额大于 5000 且状态是待支付的？"

**大师**："三种主流写法：

```python
# 写法一：布尔索引（最常用）
df.loc[(df['amount'] > 5000) & (df['status'] == '待支付')]

# 写法二：query 方法（字符串表达式，有时更可读）
df.query('amount > 5000 and status == "待支付"')

# 写法三：isin + between（多值匹配）
df.loc[df['status'].isin(['待支付', '已取消']) & df['amount'].between(1000, 5000)]
```

注意：布尔索引中多个条件用 `&`（与）、`|`（或）、`~`（非），**不是 Python 的 `and`/`or`/`not`**。每个条件必须包在括号里。"

**小胖**："凭什么不能用 `and`？"

**大师**："因为 `and` 是短路运算符，作用于布尔标量。而 `df['amount'] > 5000` 返回的是一个布尔 Series——你不能用 `and` 连接两个 Series。`&` 是元素级别的按位与操作，这才是正确的。"

**【技术映射：& vs and = 给一队人发指令——用"所有身高>180且体重<100的站出来"（&，逐个判断），而不是"身高>180？好的站出来了，等等你再问体重<100？"（and，只能判断一个人）】**

**小白**："那排序有没有性能差别？我看你用 `sort_values` 挺多的。"

**大师**："排序有两类：
- `sort_values(by='col')`：按列值排序，最常用；
- `sort_index()`：按 Index 排序，适合时间序列或多层索引；

还有一些便捷方法：
- `nlargest(10, 'amount')`：取前 10 大；
- `nsmallest(5, 'amount')`：取前 5 小；

`nlargest` 和 `nsmallest` 内部做了 Top-K 优化，比 `sort_values().head(10)` 快——因为不需要全量排序。"

**小白**："高效排序和索引之间有什么关联？"

**大师**："非常关键的关联：**排序后的 DataFrame 可以用 `searchsorted` 做二分查找**。如果你的数据按时间排序了，用 `searchsorted` 查找某个时间点的位置是 O(log n)，而用布尔索引扫描是 O(n)。在大数据量时这是天壤之别。"

**大师总结**："选择过滤排序是 pandas 日常使用频率最高的操作。记住三条原则：
1. 赋值用 `loc`，不要链式索引赋值；
2. 多条件用 `&` / `|`，按位而非逻辑运算符；
3. 预排序 + `searchsorted` 是处理有序数据的高性能路径。

下面我们就把风控规则翻成 pandas 代码，从百万订单里精准捞出异常单。"

```python
# generate_risk_data.py
import pandas as pd
import numpy as np
from datetime import datetime, timedelta

np.random.seed(42)

n = 100000  # 10 万条订单（模拟百万级场景）
base_time = datetime(2025, 3, 15, 0, 0, 0)

# IP 归属地可能和收货地不一致
cities = ['北京', '上海', '广州', '深圳', '杭州', '成都', '武汉', '南京']
city_pairs = [(rc, rc) for rc in cities]  # 正常：一致
cross_pairs = [(cities[i], cities[(i+1) % len(cities)]) for i in range(len(cities))]  # 异常：不一致

data = {
    'order_id': [f'ORD{60000+i:06d}' for i in range(n)],
    'user_id': np.random.randint(10000, 30000, n),
    'order_time': [base_time + timedelta(
        hours=np.random.randint(0, 24),
        minutes=np.random.randint(0, 60)
    ) for _ in range(n)],
    'pay_time': [None] * n,
    'goods_amount': np.round(np.abs(np.random.normal(2000, 1500, n)), 2),
    'paid_amount': np.round(np.abs(np.random.normal(2000, 1500, n)), 2),
    'coupon_amount': np.round(np.random.exponential(50, n), 2),
    'recv_province': '',
    'recv_city': '',
    'ip_province': '',
    'ip_city': '',
    'status': np.random.choice(
        ['已完成', '待支付', '已取消', '已退款'],
        n, p=[0.6, 0.15, 0.1, 0.15]
    )
}

# 90% 的订单 IP 和收货地一致，10% 不一致
for i in range(n):
    if np.random.random() < 0.9:
        rc, ipc = city_pairs[np.random.randint(0, len(city_pairs))]
    else:
        rc, ipc = cross_pairs[np.random.randint(0, len(cross_pairs))]
    data['recv_city'][i] = rc
    data['ip_city'][i] = ipc

# 支付时间：已完成/已退款的订单才有支付时间
for i in range(n):
    if data['status'][i] in ['已完成', '已退款']:
        ot = data['order_time'][i]
        # 支付通常在下单后 1-30 分钟内
        pay_delay = np.random.randint(1, 30)
        data['pay_time'][i] = ot + timedelta(minutes=pay_delay)

df = pd.DataFrame(data)

# 制造一些极端异常值
# 造 50 个大额订单
big_idx = np.random.choice(df.index, 50, replace=False)
df.loc[big_idx, 'goods_amount'] = np.random.uniform(5000, 50000, 50)

# 造 30 个金额不匹配的订单
mismatch_idx = np.random.choice(df.index, 30, replace=False)
df.loc[mismatch_idx, 'paid_amount'] = df.loc[mismatch_idx, 'goods_amount'] * np.random.uniform(0.3, 0.7, 30)

# 造 20 个高频用户（同一用户短时间内多个订单）
for uid in np.random.choice(df['user_id'].unique(), 20, replace=False):
    user_mask = df['user_id'] == uid
    n_orders = np.random.randint(5, 12)
    sample_times = [base_time + timedelta(minutes=m) for m in sorted(np.random.randint(0, 60, n_orders))]
    for j, idx in enumerate(df[user_mask].index[:n_orders]):
        df.loc[idx, 'order_time'] = sample_times[j]
        df.loc[idx, 'user_id'] = uid

df.to_csv('risk_orders.csv', index=False, encoding='utf-8-sig')
print(f"已生成 {n} 条风控订单数据")
print(f"  大额订单: {(df['goods_amount'] >= 5000).sum()}")
print(f"  地址不一致: {(df['recv_city'] != df['ip_city']).sum()}")
print(f"  金额不匹配(差异>20%): {(abs(df['goods_amount'] - df['paid_amount']) / df['goods_amount'] > 0.2).sum()}")
```

### 3.3 分步实现

#### 步骤 1：loc/iloc/at/iat 基础选择

**目标**：掌握四种索引器的用法和性能差异。

```python
# step1_indexing.py
import pandas as pd
import numpy as np

df = pd.read_csv('risk_orders.csv', parse_dates=['order_time', 'pay_time'])

# ---- loc：标签定位 ----
print("=== loc 标签定位 ===")
# 选择 order_id 为 ORD060042 的行，goods_amount 列
print(df.loc[42, 'goods_amount'])

# 使用布尔掩码 + 列标签
mask = df['goods_amount'] > 10000
print(f"金额>10000 的订单数: {df.loc[mask, ['order_id', 'goods_amount']].shape[0]}")

# ---- iloc：位置定位 ----
print("\n=== iloc 位置定位 ===")
# 第 100-104 行，第 0-2 列
print(df.iloc[100:105, 0:3])

# ---- at / iat：标量快速访问 ----
# at 和 iat 只能访问单个值，但比 loc/iloc 快
import time

# loc 批量访问
start = time.perf_counter()
for i in range(1000):
    _ = df.loc[i, 'goods_amount']
print(f"loc 1000次: {time.perf_counter() - start:.4f}s")

# at 批量访问
start = time.perf_counter()
for i in range(1000):
    _ = df.at[i, 'goods_amount']
print(f"at  1000次: {time.perf_counter() - start:.4f}s")
# at 通常比 loc 快 2-3 倍（单值场景）
```

#### 步骤 2：布尔索引——多条件组合筛选

**目标**：用布尔索引实现风控规则，筛选异常订单。

```python
# step2_boolean_filter.py
import pandas as pd

df = pd.read_csv('risk_orders.csv', parse_dates=['order_time', 'pay_time'])
df['pay_time'] = pd.to_datetime(df['pay_time'], errors='coerce')

# ===== 规则 1：高金额订单 =====
rule1 = df['goods_amount'] >= 5000

# ===== 规则 2：IP 与收货地不一致 =====
rule2 = df['recv_city'] != df['ip_city']

# ===== 规则 3：支付与订单金额差异超过 20% =====
# 注意：只对已支付订单检查
df['amount_diff_pct'] = (
    abs(df['goods_amount'] - df['paid_amount']) / df['goods_amount']
)
rule3 = (df['status'].isin(['已完成', '已退款'])) & (df['amount_diff_pct'] > 0.2)

# ===== 组合筛选：高金额 + (任一异常条件) =====
high_risk = df.loc[
    (df['goods_amount'] >= 5000) &
    (rule2 | rule3),
    ['order_id', 'user_id', 'goods_amount', 'paid_amount', 'amount_diff_pct',
     'recv_city', 'ip_city', 'status', 'order_time']
].copy()  # .copy() 防止后续操作触发 SettingWithCopyWarning

# 添加风险标签
high_risk['风险类型'] = '未知'
high_risk.loc[rule2.loc[high_risk.index], '风险类型'] = '地址不一致'
high_risk.loc[rule3.loc[high_risk.index], '风险类型'] = '金额不匹配'
high_risk.loc[
    (rule2.loc[high_risk.index]) & (rule3.loc[high_risk.index]),
    '风险类型'
] = '地址不一致+金额不匹配'

print(f"高风险订单总数: {len(high_risk)}")
print(f"\n风险类型分布:")
print(high_risk['风险类型'].value_counts())
```

#### 步骤 3：query 方法和优化选择

**目标**：对比 `query` 和布尔索引的可读性与性能。

```python
# step3_query_method.py
import pandas as pd
import time

df = pd.read_csv('risk_orders.csv', parse_dates=['order_time', 'pay_time'])

# 布尔索引方式
start = time.perf_counter()
r1 = df.loc[
    (df['goods_amount'] > 3000) &
    (df['status'].isin(['已完成', '已退款'])) &
    (df['recv_city'] != df['ip_city'])
]
t1 = time.perf_counter() - start
print(f"布尔索引: {len(r1)} 条, 耗时 {t1:.4f}s")

# query 方式（字符串表达式）
start = time.perf_counter()
r2 = df.query(
    'goods_amount > 3000 and '
    'status in ["已完成", "已退款"] and '
    'recv_city != ip_city'
)
t2 = time.perf_counter() - start
print(f"query 方式: {len(r2)} 条, 耗时 {t2:.4f}s")

# 对于大 DataFrame，query 有时更快（内部使用 numexpr）
# 对于中等数据，两者差异不大，query 的优势在于可读性
```

#### 步骤 4：排序与 Top-K

**目标**：按风险等级排序，输出 Top 问题清单。

```python
# step4_sorting.py
import pandas as pd
import numpy as np

df = pd.read_csv('risk_orders.csv', parse_dates=['order_time', 'pay_time'])

# ---- 基本排序 ----
print("=== 按金额降序排列 Top 10 ===")
top10_amount = df.nlargest(10, 'goods_amount')[['order_id', 'goods_amount', 'user_id']]
print(top10_amount.to_string(index=False))

# ---- 多列排序 ----
print("\n=== 按状态 + 金额降序排列 ===")
sorted_df = df.sort_values(
    by=['status', 'goods_amount'],
    ascending=[True, False]  # 状态升序，金额降序
)
print(sorted_df[['order_id', 'status', 'goods_amount']].head(10).to_string(index=False))

# ---- 排序后加速查找 ----
# 按 user_id 排序后做 searchsorted
df_sorted = df.sort_values('user_id')
df_sorted = df_sorted.reset_index(drop=True)  # 确保 Index 是连续的整数
target_uid = 15000

# searchsorted 二分查找，O(log n)
import time
start = time.perf_counter()
pos = df_sorted['user_id'].searchsorted(target_uid)
t_fast = time.perf_counter() - start

# 布尔索引扫描，O(n)
start = time.perf_counter()
_ = df[df['user_id'] == target_uid]
t_scan = time.perf_counter() - start

print(f"\nsearchsorted 查找耗时: {t_fast:.6f}s")
print(f"布尔索引查找耗时: {t_scan:.6f}s")
print(f"加速比: {t_scan / t_fast:.1f}x")
```

#### 步骤 5：完整风控筛选主流程

```python
# main_risk_filter.py
"""风控异常订单筛选主流程"""
import pandas as pd
import numpy as np
from datetime import datetime

print(f"[{datetime.now():%H:%M:%S}] 开始风控扫描...")

# ===== 1. 读取数据 =====
df = pd.read_csv('risk_orders.csv', parse_dates=['order_time', 'pay_time'])
df['pay_time'] = pd.to_datetime(df['pay_time'], errors='coerce')
df['amount_diff_pct'] = abs(df['goods_amount'] - df['paid_amount']) / df['goods_amount']

print(f"  读取 {len(df):,} 条订单")

# ===== 2. 筛选异常订单 =====
# 规则定义
high_amount = df['goods_amount'] >= 5000
address_mismatch = df['recv_city'] != df['ip_city']
amount_mismatch = (df['status'].isin(['已完成', '已退款'])) & (df['amount_diff_pct'] > 0.2)

# 综合异常条件
is_abnormal = high_amount & (address_mismatch | amount_mismatch)

abnormal_orders = df.loc[is_abnormal].copy()
print(f"  筛选出 {len(abnormal_orders)} 条异常订单")

# ===== 3. 高频用户检测（同一用户1小时内 > 3单）=====
abnormal_orders['hour_bucket'] = abnormal_orders['order_time'].dt.floor('h')

# 按 用户+小时 分组计数
freq_check = (
    abnormal_orders
    .groupby(['user_id', 'hour_bucket'], as_index=False)
    .size()
)
freq_users = freq_check[freq_check['size'] > 3]
print(f"  高频异常用户: {freq_users['user_id'].nunique()} 个")

# ===== 4. 标记风险等级 =====
abnormal_orders['风险等级'] = '低'
# 高金额 + 两个异常条件 → 高风险
abnormal_orders.loc[
    (abnormal_orders['goods_amount'] >= 10000) &
    (address_mismatch.loc[abnormal_orders.index]) &
    (amount_mismatch.loc[abnormal_orders.index]),
    '风险等级'
] = '高'
# 其余至少一个条件命中 → 中风险
abnormal_orders.loc[
    abnormal_orders['风险等级'] == '低',
    '风险等级'
] = '中'

# ===== 5. 按风险等级排序输出 =====
output = abnormal_orders.sort_values(
    by=['风险等级', 'goods_amount'],
    ascending=[False, False]  # 高→低风险，高→低金额
)

output_cols = ['order_id', 'user_id', 'goods_amount', 'paid_amount',
               'amount_diff_pct', 'recv_city', 'ip_city', 'status', '风险等级']
output[output_cols].to_csv('异常订单清单.csv', index=False, encoding='utf-8-sig')

print(f"\n===== 风险等级分布 =====")
print(output['风险等级'].value_counts().to_string())
print(f"\n===== Top 10 异常订单 =====")
print(output[output_cols].head(10).to_string(index=False))
print(f"\n[{datetime.now():%H:%M:%S}] 风控扫描完成，清单已导出至 异常订单清单.csv")
```

### 3.4 可能的坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| SettingWithCopyWarning | 链式索引 + 赋值操作 | 全部改用 `df.loc[row_mask, col] = value` |
| `&` 写成 `and` 报错 | `and` 是短路运算符，不支持 Series | 用 `&`（与）、`|`（或）、`~`（非） |
| 条件忘记加括号 | `&` 优先级高于 `>` | 每个条件加括号：`(df['A'] > 0) & (df['B'] < 10)` |
| filter 和 query 混用 | `df.filter` 是按列名/行标签过滤，不是按值 | 按值过滤用布尔索引或 `query` |
| query 引用中间变量 | `query` 作用域不包含外部变量 | 用 `@变量名` 语法：`df.query('amount > @threshold')` |
| sort_values 排序后 Index 混乱 | `sort_values` 默认不重置 Index | 加 `ignore_index=True` 或用 `reset_index(drop=True)` |
| isin + ~ 的坑 | `~df['col'].isin([a,b])` 对 NaN 也返回 True | NaN 不参与 isin 判断，需单独处理 |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 选择过滤 | SQL WHERE | Excel 筛选 |
|------|----------------|-----------|-----------|
| 复杂条件组合 | `&`/`|`/`~` 灵活组合，可调试 | SQL 子查询和 CASE WHEN 强大 | 逐列筛选，无法一次完成复杂组合 |
| 可复现性 | 代码化，可版本控制 | SQL 文件可版本控制 | 手工操作，不可复现 |
| 性能 | 中等，依赖 dtype 优化 | 依赖索引质量，高性能 | 行数多时极慢 |
| 筛选后操作 | 无缝衔接 groupby/sort 等分析 | 需子查询或 CTE | 需导出再导入 |
| 安全赋值 | `.loc` 保证原地修改 | UPDATE 语句 | 直接修改单元格，易出错 |
| 排序灵活性 | 多列、混合升降序、自定义 key | ORDER BY 灵活 | 有限多列排序 |

### 4.2 适用场景

- **适用场景**：
  1. 按业务规则从数据集中筛选目标子集，如风控异常订单、高价值用户
  2. 需要按多个维度进行交叉过滤（如时间+金额+状态）
  3. 筛选后需要立即进行统计分析或图表绘制
  4. 需要对大数据集做 Top-K 排名（nlargest/nsmallest 优势明显）
  5. 需要将筛选逻辑封装为可复用函数的场景
- **不适用场景**：
  1. 极其简单的筛选（如单条件等于某个值）——任何工具都能做
  2. 筛选逻辑极其复杂、需要正则或非结构化推理——建议先做数据加工

### 4.3 注意事项

- **SettingWithCopyWarning 不要忽略**：它不只是"警告"，而是"你现在的代码可能没生效"。pandas 2.0+ 的 copy-on-write 减少了这类场景，但仍建议全部改为 `.loc` 写法；
- **`inplace=True` 也需警惕**：很多 pandas 方法（如 `drop`、`sort_values`）的 `inplace=True` 实际上也是先复制再赋值，并不真正"原地"修改。推荐用 `df = df.sort_values(...)` 显式赋值；
- **query 中的变量引用**：要在 `query` 字符串里引用外部 Python 变量，必须加 `@` 前缀，如 `df.query('amount > @threshold')`；
- **大 DataFrame 排序**：`sort_values` 是全量排序，大数据集建议用 `nlargest`/`nsmallest` 做 Top-K。

### 4.4 常见踩坑经验

1. **`df['A'] == NaN` 总是 False**：NaN 不等于任何值（包括它自己）。判断缺失值必须用 `df['A'].isna()`，这是新手最容易犯的错误之一。SQL 的 `IS NULL` 语义同理。
2. **链式索引赋值静默失败**：一位同事写 `df[df['amount']>5000]['tag'] = 'risk'`，代码无报错、无警告（在某些版本下），但原始 DataFrame 的 `tag` 列没有被赋值——因为第二个 `[...]` 返回的是临时拷贝。花了三天排查出"为什么风控标签没有打上"。
3. **sort_values 后 Index 未重置导致 loc 异常**：`df.sort_values('amount')` 后 Index 还是原来的顺序。而后用 `df.loc[0]` 拿到的不是排序后的第一行而是原始第一行——因为 loc 是按 Index 标签查找的。排行榜的前三名用 `head(3)` 或 `iloc[:3]` 而非 `loc`.
