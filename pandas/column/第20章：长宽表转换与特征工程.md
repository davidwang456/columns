# 第20章：长宽表转换与特征工程

## 1. 项目背景

某在线教育平台的算法团队正在开发一个"用户流失预测模型"。数据科学家小陈需要从原始的用户行为日志中构建特征表（feature matrix），一行代表一个用户，一列代表一个特征，最终喂给机器学习模型。

原始数据有三张表：
- **user_actions.csv**：用户行为日志，包含 user_id、日期、行为类型（听课/做题/浏览/分享）、课程 ID，约 50 万行（长表格式——每个行为一行）；
- **user_profile.csv**：用户画像，包含 user_id、城市、年龄、注册渠道，约 2 万行；
- **user_subscriptions.csv**：用户订阅记录，包含 user_id、订阅开始时间、结束时间、套餐类型，约 3 万行（同一个用户可能有多条订阅记录）。

建模需要的一行一用户的特征表（宽表），小陈面临以下挑战：

**痛点一：长表转宽表——行为计数变特征列**。"过去 30 天用户各行为类型的次数"——这需要把行为日志的长表（每个行为一行）聚合成宽表（每种行为一列）。小陈在 Excel 里先做了透视表，但 50 万行的行为日志 Excel 根本打不开。

**痛点二：宽表转长表——用户套餐时间线展开**。"每个用户的每个订阅周期的时长"——订阅表是一行一个订阅周期，但小陈需要把它变成"用户 × 年月"的长格式面板数据（panel data），用于时序建模。

**痛点三：one-hot 编码后列数爆炸**。"城市"有 50 个值，"课程"有 200 个值——one-hot 编码后特征从 10 列膨胀到 260 列。小陈不知道如何优雅地做 one-hot 并控制列数。

痛点流程：

```
行为日志 (50万行, 长表)
  └── 特征工程
        ├── 长→宽：行为计数变特征列
        ├── 宽→长：订阅周期变时序面板
        └── one-hot：50城市→50列→维度爆炸
```

本章将系统讲解 pandas 的长宽表转换工具——melt、wide_to_long、pivot、get_dummies，以及如何与 category 类型配合控制特征维度。

## 2. 项目设计：剧本式交锋对话

> 场景：小陈在画特征工程流程图，纸上写满了"长→宽→长→哑变量"，小胖看得一头雾水。

**小胖**：（吃着巧克力棒）"长表宽表不就是行列转来转去吗？Excel 透视表拖一拖就搞定了。50 万行打不开？分 Sheet 啊！"

**大师**："小胖，这次你真的触及了 pandas 和 Excel 的根本分水岭——**数据形态的自动化转换**。长表（tidy data）和宽表（cross table）之间的转换不是手工拖拽能解决的——因为业务逻辑复杂，而且数据量巨大。

pandas 给了你四个核心工具：
- **`melt`**：宽表 → 长表（把多列"融化"为两列：变量名+值）
- **`pivot` / `pivot_table`**：长表 → 宽表（聚合后摊开）
- **`wide_to_long`**：更智能的宽表 → 长表（支持正则匹配列名）
- **`get_dummies`**：类别 → one-hot 编码（哑变量）"

**【技术映射：melt = 把横向排列的书一本本摞起来变纵向，pivot = 把纵向的书按类别摊成横向书架】**

**小白**："melt 和 wide_to_long 都是宽→长，有什么区别？"

**大师**："`melt` 是最通用的——指定 id_vars（保持不变的列）和 value_vars（要融化的列），全部熔化。适合列名没有规律的情况。

`wide_to_long` 更智能——它用正则匹配列名中的模式，自动识别"哪些列应该合并为一行"。典型场景：

```python
# 原始宽表：q1_result, q2_result, q3_result, q1_score, q2_score, q3_score
# 想变成长表：每行代表一个问题，列：question_no, result, score

pd.wide_to_long(df, stubnames=['q','score'], i='user_id', j='question_no', sep='_')
# 自动识别：q1_result→q, 1; score1→score, 1; ...
```

`stubnames` 是列名的"主干"部分，`sep` 是分隔符，`j` 是生成的标识列。"

**【技术映射：melt = 把所有东西倒进一个箱子（全量），wide_to_long = 按标签归类放进不同的抽屉（结构化）】**

**小胖**："那 one-hot 编码呢？不就是把'北京'变成[1,0,0,0,0]，'上海'变成[0,1,0,0,0]吗？"

**大师**："对，`pd.get_dummies(df['city'])` 一行搞定。但关键问题不是编码本身——而是**编码后列数爆炸**。

50 个城市 = 50 列。如果加上 200 个课程、20 个注册渠道——列数瞬间膨胀到 270 列，导致维度诅咒。

解决方案：
1. **低基数列用 one-hot**（10 个以下类别）
2. **高基数列用 target encoding**（用目标变量对类别做均值编码）
3. **极度高基数列（用户 ID）不做 one-hot**——改用行为计数特征

get_dummies 的 `prefix` 和 `drop_first` 参数也很有用——`drop_first=True` 会省略第一个类别（避免多重共线性）。"

**【技术映射：one-hot = 每个人举一个牌子——'我是北京人'是 True，其他城市是 False，50 个城市需要 49 个牌子（drop_first）】**

**小白**："那特征交叉和分桶怎么做？比如把 age 和 gender 交叉？"

**大师**："两个工具：

1. **pd.cut / pd.qcut**：连续值 → 分桶（离散化）
```python
pd.cut(df['age'], bins=[0,18,35,50,100], labels=['青少年','青年','中年','老年'])
pd.qcut(df['age'], q=5)  # 等频分 5 桶
```

2. **手工特征交叉**：直接拼接两列字符串，再 one-hot
```python
df['age_gender'] = df['age_bucket'] + '_' + df['gender']
pd.get_dummies(df['age_gender'])
```

但要注意——特征交叉会让列数进一步爆炸。先判断这个交叉是否有业务意义（age × gender 比 age × device_brand 更有解释力）。"

**大师总结**："特征工程的核心是'形态转换'：
1. 长→宽（pivot/pivot_table/groupby+unstack）：聚合为特征矩阵
2. 宽→长（melt/wide_to_long）：展开为时序面板
3. 类别→数值（get_dummies/cut/qcut）：喂给模型
4. 稀疏化（category dtype + 合理选择 one-hot 范围）：控制维度

下面我们就把行为日志、用户画像、订阅记录加工成可训练的特征表。"

import pandas as pd

np.random.seed(42)
N_USERS = 2000

# 用户画像

### 3.3 分步实现

#### 步骤 1：melt——宽表熔化为长表

**目标**：将多列"季度指标"熔化为两列（季度名+值）。

```python
# step1_melt.py
import pandas as pd

# 模拟季度指标宽表
wide = pd.DataFrame({
    'user_id': [1, 2],
    'Q1_revenue': [100, 200],
    'Q2_revenue': [150, 250],
    'Q3_revenue': [130, 230],
    'Q4_revenue': [180, 280],
})
print("原始宽表:")
print(wide.to_string(index=False))

# melt：id_vars 不变列，其余熔化
long = wide.melt(
    id_vars=['user_id'],
    var_name='quarter',
    value_name='revenue'
)
# 提取季度编号
long['quarter'] = long['quarter'].str.extract(r'Q(\d)')[0].astype(int)
print("\nmelt 后长表:")
print(long.to_string(index=False))
```

#### 步骤 2：wide_to_long——结构化宽转长

**目标**：将"q1_score, q2_score, q3_score + q1_time, q2_time, q3_time"转为长表。

```python
# step2_wide_to_long.py
import pandas as pd

# 模拟问卷结果宽表
survey = pd.DataFrame({
    'user_id': [1, 2],
    'q1_score': [5, 4], 'q2_score': [3, 5], 'q3_score': [4, 4],
    'q1_time': [120, 90], 'q2_time': [80, 110], 'q3_time': [95, 100],
})
print("原始宽表:")
print(survey.to_string(index=False))

# wide_to_long：自动识别 stubnames
long = pd.wide_to_long(
    survey,
    stubnames=['q'],  # 错误示范——应分开
    i='user_id',
    j='question_no',
    sep='_',
    suffix=r'\d+'
)
# 因为 score 和 time 是两个 stubname，需要分开处理
# 正确的做法：
survey_long = pd.wide_to_long(
    survey.rename(columns=lambda c: c.replace('_score','')+'_score' if '_score' in c else c),
    stubnames=['q'], i='user_id', j='question_no', sep='_', suffix=r'\d+_score'
)
print("wide_to_long 不适用此结构，改用 melt:")
# 实际更简单的方式：
long_score = survey.melt(id_vars='user_id', value_vars=['q1_score','q2_score','q3_score'],
                          var_name='question', value_name='score')
long_score['question_no'] = long_score['question'].str.extract(r'q(\d)')[0]
print(long_score.to_string(index=False))
```

#### 步骤 3：get_dummies——one-hot 编码

**目标**：对类别列做 one-hot 编码，并控制维度。

```python
# step3_get_dummies.py
import pandas as pd

profiles = pd.read_csv('user_profile_ch20.csv')

# === 基础 one-hot ===
city_dummies = pd.get_dummies(profiles['city'], prefix='city', drop_first=True)
print(f"city one-hot: {profiles['city'].nunique()} → {len(city_dummies.columns)} 列 (drop_first)")
print(f"列名: {city_dummies.columns.tolist()}")

# === 多个列同时 one-hot ===
dummy_cols = ['city', 'gender', 'reg_channel']
dummies = pd.get_dummies(profiles[dummy_cols], drop_first=True)
print(f"\n多列 one-hot: {len(dummy_cols)} 个原始列 → {len(dummies.columns)} 列")

# === 分桶后 one-hot ===
profiles['age_bucket'] = pd.cut(
    profiles['age'], bins=[0, 25, 35, 50, 100],
    labels=['18-25','26-35','36-50','50+']
)
age_dummies = pd.get_dummies(profiles['age_bucket'], prefix='age', drop_first=True)
print(f"\nage 分桶 + one-hot: {profiles['age'].nunique()} 个年龄 → 4 个桶 → {len(age_dummies.columns)} 列")
```

#### 步骤 4：完整特征工程流程

**目标**：从三张原始表构建一行一用户的特征矩阵。

```python
# step4_feature_pipeline.py
import pandas as pd
import numpy as np

profiles = pd.read_csv('user_profile_ch20.csv')
actions = pd.read_csv('user_actions.csv', parse_dates=['date'])
subs = pd.read_csv('user_subscriptions.csv', parse_dates=['sub_start','sub_end'])

ref_date = pd.Timestamp('2025-03-31')

# ===== 1. 长表→宽表：行为日志 → 每种行为的计数特征 =====
action_counts = (
    actions[actions['date'] >= ref_date - pd.Timedelta(days=30)]  # 近30天
    .pivot_table(index='user_id', columns='action_type', values='date',
                 aggfunc='count', fill_value=0)
    .add_prefix('cnt_')
    .reset_index()
)
print(f"行为计数: {len(action_counts)} 行, {len(action_counts.columns)-1} 种行为")

# ===== 2. 兴趣特征：听过的课程类别数 =====
course_diversity = (
    actions[actions['date'] >= ref_date - pd.Timedelta(days=30)]
    .groupby('user_id')['course_id'].nunique()
    .reset_index(name='unique_courses_30d')
)

# ===== 3. 订阅特征：宽表→长(面板)→特征 =====
# 计算每个用户的总订阅天数、当前是否在订阅期内
subs['sub_days'] = (subs['sub_end'] - subs['sub_start']).dt.days
sub_features = subs.groupby('user_id').agg(
    total_sub_days=('sub_days','sum'),
    max_sub_days=('sub_days','max'),
    current_active=('sub_end', lambda x: (x.max() >= ref_date)),  # 未过期
).reset_index()

# ===== 4. 画像特征 + one-hot =====
profile_features = profiles[['user_id','age']].copy()
profile_features['age_bucket'] = pd.cut(profiles['age'], bins=[0,25,35,50,100],
                                         labels=['18-25','26-35','36-50','50+'])
dummies = pd.get_dummies(profiles[['city','gender','reg_channel']], drop_first=True)
profile_features = pd.concat([profile_features, dummies], axis=1)

# ===== 5. 合并所有特征 =====
feature_matrix = (profile_features
    .merge(action_counts, on='user_id', how='left')
    .merge(course_diversity, on='user_id', how='left')
    .merge(sub_features, on='user_id', how='left')
    .fillna(0)
    .set_index('user_id')
)

print(f"\n特征矩阵: {len(feature_matrix)} 用户 × {len(feature_matrix.columns)} 特征")
print(f"前 5 列: {feature_matrix.columns[:5].tolist()}")
print(f"稀疏度: {(feature_matrix==0).sum().sum() / (feature_matrix.shape[0]*feature_matrix.shape[1])*100:.1f}%")
```

### 3.5 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| melt 后数据量爆炸 | 每条记录 × value_vars 列数 | 只 melt 必要的列 |
| wide_to_long stubnames 不匹配 | 列名格式不统一 | 先 rename 统一格式 |
| get_dummies 在测试集出现新类别 | 测试集有训练集没有的值 | 用 sklearn OneHotEncoder handle_unknown |
| one-hot 后内存不足 | 类别太多 | 预筛选低频类别为 'other' |
| pivot_table 对大数据量的性能 | 笛卡尔积计算开销 | 先 groupby 聚合再 pivot |

# test_ch20.py
import pandas as pd
import numpy as np

def test_melt():
    df = pd.DataFrame({'id':[1,2], 'A':[10,20], 'B':[30,40]})
    long = df.melt(id_vars='id', var_name='var', value_name='val')
    assert len(long) == 4
    assert long.loc[0,'val'] == 10

def test_get_dummies():
    df = pd.DataFrame({'city':['A','B','A','C']})
    dummies = pd.get_dummies(df['city'], drop_first=True, prefix='c')
    assert dummies.shape == (4, 2)
    assert 'c_B' in dummies.columns

def test_pivot_agg():
    df = pd.DataFrame({'u':[1,1,2], 'a':['X','Y','X'], 'v':[1,2,3]})
    pt = df.pivot_table(index='u', columns='a', values='v', aggfunc='sum', fill_value=0)
    assert pt.loc[1,'X'] == 1
    assert pt.loc[2,'X'] == 3

def test_cut():
    ages = pd.Series([15, 25, 40, 60])
    buckets = pd.cut(ages, bins=[0,18,35,50,100], labels=['未成年','青年','中年','老年'])
    assert buckets[0] == '未成年'
    assert buckets[3] == '老年'

if __name__ == '__main__':
    test_melt(); test_get_dummies(); test_pivot_agg(); test_cut()

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 特征工程 | sklearn Pipeline | 手工 Excel |
|------|---------------|-----------------|-----------|
| 长宽转换 | melt/pivot_table 灵活 | 无内置 | 手工透视表 |
| one-hot | get_dummies 一行 | OneHotEncoder 更规范 | 手工创建辅助列 |
| 分桶 | cut/qcut | KBinsDiscretizer | 手工 IF 公式 |
| 与模型集成 | 需手动衔接 | Pipeline 原生对接 | 不可能 |
| 灵活性 | 极高 | 受限于 API | 低 |

### 4.2 适用场景

- **适用场景**：
  1. 行为日志等长表数据 → 用户级特征矩阵
  2. 类别变量的 one-hot 编码
  3. 连续变量的分桶离散化
  4. 时序面板数据的构建（wide_to_long）
  5. 实验数据的清洗与聚合（melt + pivot）
- **不适用场景**：
  1. 已进入 sklearn Pipeline 的阶段——用 sklearn 的 Transformer
  2. 超大规模（亿级）稀疏矩阵——用 scipy.sparse 或 Spark MLlib

### 4.3 注意事项

- **get_dummies 的内存**：100 万行 × 200 个类别 = 2 亿个元素。用 `dtype='uint8'` 或 `sparse=True` 降内存；
- **drop_first 的解读**：去掉第一个类别后，回归模型的所有系数都是"相对于基准类"的变化——不是绝对值；
- **测试集一致性**：`get_dummies` 没有 `handle_unknown` 参数，生产环境建议用 sklearn 的 OneHotEncoder。

### 4.4 常见踩坑经验

1. **melt 后 value 列的 dtype 不一致**：如果 value_vars 包含 int 和 float 列，melt 后 value 列 dtype 会变成 object。解决：melt 前统一类型。
2. **wide_to_long 中 stubnames 的 regex 陷阱**：`suffix=r'\d+'` 会匹配任意数字——如果列名中还有别的数字（如 phone1、phone2），会错误匹配。用更精确的 regex，如 `suffix=r'_(Q\d+)'`。
3. **get_dummies 中类别过多导致列数爆炸**：某同事对 user_id 做了 one-hot，2 万列 × 50 万行直接 OOM。教训：高基数列用 target encoding 或直接删除。
