# 第13章：窗口函数与滚动指标

## 1. 项目背景

某第三方支付平台的风控团队每天监控"支付成功率"指标。支付成功率 = 成功支付笔数 / 总发起支付笔数。当成功率出现异常波动时，需要在 15 分钟内响应排查。

数据来自每分钟的支付日志（payment_log.csv），包含：时间戳（minute）、发起笔数（request_count）、成功笔数（success_count）、失败笔数（fail_count），覆盖最近 30 天共约 4.3 万条分钟级数据。

风控分析师小郑需要构建以下监控指标：
- **7 日滚动平均成功率**：平滑分钟级别的波动噪音，看趋势
- **7 日滚动标准差**：衡量成功率的波动幅度——标准差变大意味着系统不稳定
- **滚动最大值/最小值**：检测成功率是否有极端异常波动
- **累计成功率**（从平台成立至今）：和近期对比判断趋势偏离

小郑目前的做法是在 Excel 里手工算：先加一列辅助计算 "过去 7 天平均值"，公式为 `=AVERAGE(B2:B10080)`（7 天 = 10080 分钟）。然后下拉填充 4.3 万行——Excel 直接卡死。

**痛点一：Excel 不支持真正的"滚动窗口"**。Excel 的 AVERAGE 函数只能算固定范围，下拉填充要做 4.3 万次重新计算——每次计算都要扫描 10080 个单元格，总计算量是 4.3 万 × 1 万 = 4.3 亿次，机器根本扛不住。

**痛点二：不同窗口指标需要重复操作**。先算 7 日滚动均值，再算 7 日滚动标准差，再算 30 日滚动均值——每加一个指标就要重复一遍公式下拉的操作，而且公式里的范围引用（$A$1:$A$10080）一旦操作失误就错位。

**痛点三：窗口参数的细微差异难处理**。"最近 7 天"是从当天往前数 7 天（不含当天）还是含当天？Excel 里没有标准的滚动窗口语义，全靠手工定义范围——有时含头不含尾，有时含头含尾，沟通成本极高。

痛点流程：

```
支付日志 (43K行/分钟级)
  └── Excel 滚动指标
        ├── AVERAGE + 下拉 → 4.3亿次计算 → 卡死
        ├── 多指标需重复操作 → 手工易错
        └── 窗口语义不统一 → 沟通歧义
```

本章将系统讲解 pandas 的三种窗口函数——rolling（滚动窗口）、expanding（累计窗口）、ewm（指数加权窗口），以及它们在监控指标构建中的应用。

## 2. 项目设计：剧本式交锋对话

> 场景：小郑把 Excel 卡死的截图发到群里求助，小胖立刻蹦了出来。

**小胖**：（吃着薯片）"你要的是'过去 7 天的平均值'？这不就是 for 循环往前数 7 个格子求平均吗？一个 for 循环搞定！"

**大师**："小胖，4.3 万行 × 10080 列的滑动窗口，Python for 循环得跑到明天。pandas 的 `rolling` 是 C/Cython 实现的——一行代码 `df['success_rate'].rolling(7*24*60).mean()` 搞定，背后是高度优化的滑动窗口算法，和 for 循环比快 100 倍以上。"

**小白**："rolling、expanding、ewm 这三个我都见过，但它们的区别一直搞不清。"

**大师**："这是 pandas 窗口函数的三兄弟，本质区别在'窗口怎么定义'：

- **`rolling`**：**固定大小的滑动窗口**。比如 `rolling(7)`——始终看最近的 7 个数据点，窗口跟着时间往前走。适用于'近 7 天均值'、'近 30 天最大值'等指标。
- **`expanding`**：**从起始位置累积的窗口**。窗口从第 1 个数据点开始，越来越长。`expanding().sum()` 就是求累积和。适用于'自成立以来的总交易额'这类累计指标。
- **`ewm`**：**指数加权移动窗口**。越近的数据权重越大，越远的权重指数衰减。`ewm(span=7).mean()` 是近 7 天的指数加权平均——比简单移动平均对近期变化更敏感。

关键差异：rolling 是'矩形窗'（所有点等权重），ewm 是'指数窗'（近大远小权重不等）。"

**【技术映射：rolling = 近 7 天成绩单（每天等权重），expanding = 累计学分（从大一起），ewm = 最近作业更重要（有记忆衰减）】**

**小胖**："那 min_periods 和 center 是啥？"

**大师**："`min_periods`：窗口内最少需要多少个非空值才输出结果。如果设置 `min_periods=1`，即便窗口不满也会输出值（比如第 1 天的平均值就是它自己）。默认 `min_periods` 等于窗口大小——窗口不满时输出 NaN。

`center`：窗口怎么对齐。`center=False`（默认）窗口在数据点的'前面'——比如第 7 行的 rolling(7) 看的是第 1-7 行。`center=True`：窗口以当前点为中心——第 4 行的 rolling(7) 看的是第 1-7 行（前 3 + 自身 + 后 3）。

对于监控告警场景，通常 `center=False`——'用历史数据判断当前是否异常'。对于趋势报告场景，有时 `center=True` 更合理——'用前后数据平滑当前点的趋势'。"

**【技术映射：center=False = 用历史成绩评估当前，center=True = 用前后成绩平滑当前】**

**小白**："rolling 可以基于时间而不仅是行数吗？比如 '滚动 7 天' 而不是 '滚动 7 行'？"

**大师**："可以！当 Index 是 DatetimeIndex 时，rolling 支持基于时间的窗口：

```python
# 基于行数的窗口
df['success_rate'].rolling(7).mean()  # 7 行

# 基于时间的窗口（需要 DatetimeIndex）
df['success_rate'].rolling('7D').mean()  # 7 天
df['success_rate'].rolling('1H').mean()  # 1 小时
```

时间的 rolling 自动理解时间间隔——如果窗口内的数据点间隔不均匀（比如有的天没有记录），时间窗口会正确处理。`closed` 参数控制窗口是左闭右开还是左开右闭。"

**小白**："那 ewm 的 span 和 alpha 参数有什么区别？"

**大师**："ewm 有三种等价的参数化方式：

- `span`：跨度 N。衰减因子 α = 2/(N+1)。`span=7` 相当于'和近 7 天的简单平均衰减特性相当'。
- `alpha`：直接指定衰减因子。α=0.3 意味着当前值权重 30%，历史权重 70%。
- `com`：质心。α = 1/(1+com)。`com=3` 和 `span=7` 等价。

三种方式互相换算，选一个你最熟悉的用即可。监控场景通常用 `span`——'相当于近 N 天的影响'，业务方听得懂。"

**大师总结**："窗口函数的要素：
1. 窗口类型：rolling（固定窗）vs expanding（累计窗）vs ewm（指数衰减窗）
2. 窗口大小：行数（7）或时间（'7D'）
3. 聚合函数：mean/sum/std/min/max/quantile
4. 边界控制：min_periods、center、closed

下面我们用支付日志构建一套完整的成功率监控体系。"

### 3.2 模拟支付日志

```python
# generate_payment_log.py
import pandas as pd
import numpy as np

np.random.seed(42)

# 30 天的分钟级数据
dates = pd.date_range('2025-03-01', '2025-03-30 23:59', freq='1min')

base_success_rate = 0.98  # 基础成功率 98%
data = []
for ts in dates:
    # 模拟成功率波动：白天略低，凌晨高
    hour_effect = 0.005 * np.sin(np.pi * ts.hour / 12)
    # 随机波动
    noise = np.random.normal(0, 0.01)
    rate = np.clip(base_success_rate + hour_effect + noise, 0.85, 1.0)

    request = np.random.poisson(20)
    success = np.random.binomial(request, rate)
    fail = request - success

    data.append({
        'minute': ts.strftime('%Y-%m-%d %H:%M:%S'),
        'request_count': request,
        'success_count': success,
        'fail_count': fail,
    })

df = pd.DataFrame(data)

# 制造几段异常波动（成功率骤降）
for start in [5000, 15000, 25000]:
    for i in range(60):
        df.loc[df.index[start+i], 'success_count'] = max(
            0, int(df.loc[df.index[start+i], 'request_count'] * np.random.uniform(0.6, 0.8))
        )
        df.loc[df.index[start+i], 'fail_count'] = (
            df.loc[df.index[start+i], 'request_count'] - df.loc[df.index[start+i], 'success_count']
        )

df.to_csv('payment_log.csv', index=False, encoding='utf-8-sig')
print(f"已生成 {len(df)} 条分钟级支付日志 ({dates[0]} ~ {dates[-1]})")
```

### 3.3 分步实现

#### 步骤 1：rolling 固定窗口——滚动均值与标准差

**目标**：用 rolling 计算 7 日滚动均值和标准差，构建监控基线。

```python
# step1_rolling_basics.py
import pandas as pd

df = pd.read_csv('payment_log.csv', parse_dates=['minute'])
df = df.set_index('minute').sort_index()

# 计算每分钟的成功率
df['success_rate'] = df['success_count'] / df['request_count'].replace(0, 1)

# === 基于行数的 rolling ===
# 每小时约 60 行，7 天 = 7*24*60 = 10080 行
window_size = 7 * 24 * 60  # 10080

# 滚动均值
df['rolling_mean'] = df['success_rate'].rolling(window_size, min_periods=window_size//2).mean()
# 滚动标准差
df['rolling_std'] = df['success_rate'].rolling(window_size, min_periods=window_size//2).std()
# 滚动最大值/最小值
df['rolling_max'] = df['success_rate'].rolling(window_size).max()
df['rolling_min'] = df['success_rate'].rolling(window_size).min()
# 滚动分位数（上四分位和下四分位）
df['rolling_q25'] = df['success_rate'].rolling(window_size).quantile(0.25)
df['rolling_q75'] = df['success_rate'].rolling(window_size).quantile(0.75)

# === 基于时间的 rolling (需要 DatetimeIndex) ===
# 7 天滚动窗口（时间语义）
df['rolling_7d_mean'] = df['success_rate'].rolling('7D', min_periods=24*60).mean()

print("===== 滚动指标 (最后 5 行) =====")
cols = ['success_rate','rolling_mean','rolling_std','rolling_7d_mean']
print(df[cols].tail().round(4).to_string())

# 异常检测：当前值偏离滚动均值超过 2 个标准差
df['is_anomaly'] = abs(df['success_rate'] - df['rolling_mean']) > 2 * df['rolling_std']
print(f"\n2-sigma 异常点数: {df['is_anomaly'].sum()}")
```

#### 步骤 2：expanding 累计窗口

**目标**：用 expanding 计算累计成功率和累计总量指标。

```python
# step2_expanding.py
import pandas as pd

df = pd.read_csv('payment_log.csv', parse_dates=['minute'])

df['success_rate'] = df['success_count'] / df['request_count'].replace(0, 1)

# === expanding 累计计算 ===
# 累计总请求数
df['cum_requests'] = df['request_count'].expanding().sum()
# 累计成功数
df['cum_success'] = df['success_count'].expanding().sum()
# 累计成功率
df['cum_rate'] = df['cum_success'] / df['cum_requests']
# 累计最高/最低成功率
df['cum_max_rate'] = df['success_rate'].expanding().max()
df['cum_min_rate'] = df['success_rate'].expanding().min()

print("===== 累计指标 =====")
print(f"累计发起: {df['cum_requests'].iloc[-1]:,}")
print(f"累计成功: {df['cum_success'].iloc[-1]:,}")
print(f"累计成功率: {df['cum_rate'].iloc[-1]:.4f}")
print(f"历史最高单分钟成功率: {df['cum_max_rate'].iloc[-1]:.4f}")
print(f"历史最低单分钟成功率: {df['cum_min_rate'].iloc[-1]:.4f}")

# === 对比滚动均值和累计均值 ===
# 累计均值代表"历史平均水平"，滚动均值代表"近期水平"
# 两者偏离 = 近期系统表现偏离历史正常水平
df['deviation_from_history'] = df['success_rate'].rolling(10080, min_periods=10080).mean() - df['cum_rate']
print(f"\n近期(7日) vs 历史累计 偏离程度(最后): {df['deviation_from_history'].iloc[-1]:.6f}")
```

#### 步骤 3：ewm 指数加权窗口

**目标**：用 ewm 计算对近期波动更敏感的监控基线。

```python
# step3_ewm.py
import pandas as pd

df = pd.read_csv('payment_log.csv', parse_dates=['minute'])
df['success_rate'] = df['success_count'] / df['request_count'].replace(0, 1)

# === ewm 指数加权平均 ===
# span=7*24*60: 相当于简单平均 7 天的衰减特性
window_minutes = 7 * 24 * 60
df['ewm_mean'] = df['success_rate'].ewm(span=window_minutes, adjust=False).mean()

# === 对比 rolling vs ewm ===
# 取一个异常波动的样本区间
sample = df.iloc[5000:5200][['success_rate']]
sample['rolling_mean'] = sample['success_rate'].rolling(60, min_periods=1).mean()
sample['ewm_mean'] = sample['success_rate'].ewm(span=60, adjust=False).mean()

print("===== rolling vs ewm 对比（异常波动区间）=====")
print(sample.head(10).round(4).to_string())

# ewm 对近期变化反应更快——因为近期数据权重更大
# 异常突然发生时，ewm 能更早预警
print(f"\newm 的滞后性: ewm 在异常发生 {((sample['ewm_mean'] - sample['rolling_mean']).abs() > 0.005).idxmax()} 分钟时已明显偏离")
```

#### 步骤 4：构建多级异常预警系统

**目标**：用 rolling + expanding + ewm 组合构建三层预警。

```python
# step4_alert_system.py
import pandas as pd
import numpy as np

df = pd.read_csv('payment_log.csv', parse_dates=['minute'])
df = df.set_index('minute').sort_index()
df['success_rate'] = df['success_count'] / df['request_count'].replace(0, 1)

# 7 日滚动基线
w = 7*24*60
df['roll_mean'] = df['success_rate'].rolling(w, min_periods=w//2).mean()
df['roll_std'] = df['success_rate'].rolling(w, min_periods=w//2).std()

# 指数加权基线（对近期更敏感）
df['ewm_mean'] = df['success_rate'].ewm(span=w, adjust=False).mean()

# 累计基线
df['cum_rate'] = df['success_count'].expanding().sum() / df['request_count'].expanding().sum()

# ===== 三层预警 =====
# L1：轻微偏离（当前值 偏离 roll_mean 超过 2σ）
df['alert_l1'] = abs(df['success_rate'] - df['roll_mean']) > 2 * df['roll_std']

# L2：显著偏离（当前值 偏离 roll_mean 超过 3σ）
df['alert_l2'] = abs(df['success_rate'] - df['roll_mean']) > 3 * df['roll_std']

# L3：趋势偏离（ewm 和 cum_rate 偏离过大——近期趋势严重偏离历史）
df['alert_l3'] = abs(df['ewm_mean'] - df['cum_rate']) > 0.05  # 5pp 偏离

print(f"L1 预警（2σ）: {df['alert_l1'].sum()} 个时间点")
print(f"L2 预警（3σ）: {df['alert_l2'].sum()} 个时间点")
print(f"L3 预警（趋势）: {df['alert_l3'].sum()} 个时间点")

# 最近一次异常的时间
last_anomaly = df[df['alert_l2']].index.max() if df['alert_l2'].any() else None
print(f"\n最近一次 L2 异常: {last_anomaly}")
```

### 3.4 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| rolling 前 N-1 行全是 NaN | min_periods 默认 = window | 设置 `min_periods=1`（但要注意小窗口的统计不稳健） |
| 时间 rolling 报 TypeError | Index 不是 DatetimeIndex | `df.index = pd.to_datetime(df.index)` |
| rolling 后结果不对齐 | 默认 center=False | 检查是否需要 `center=True` |
| ewm 结果和预期不符 | `adjust=True`（默认）用递归公式 | 设为 `adjust=False` 得到简单指数加权 |
| expanding 窗口为空报错 | 某些操作不接受空窗口 | 确保数据至少有一行 |
| rolling.apply 自定义函数很慢 | 逐窗口 Python 调用 | 优先用内建聚合 (mean/std) |

### 3.5 测试验证

```python
# test_ch13.py
import pandas as pd
import numpy as np

def test_rolling_mean():
    s = pd.Series([1, 2, 3, 4, 5])
    rm = s.rolling(3, min_periods=1).mean()
    assert rm.iloc[0] == 1.0
    assert rm.iloc[2] == 2.0  # (1+2+3)/3
    assert rm.iloc[4] == 4.0  # (3+4+5)/3

def test_rolling_center():
    s = pd.Series([1, 2, 3, 4, 5])
    # center=True：窗口以当前点为中心
    rm_center = s.rolling(3, center=True, min_periods=1).mean()
    # 第1行：窗口为 [1,2] → (1+2)/2=1.5
    assert abs(rm_center.iloc[0] - 1.5) < 0.01

def test_expanding():
    s = pd.Series([1, 2, 3, 4, 5])
    ec = s.expanding().sum()
    assert ec.iloc[0] == 1
    assert ec.iloc[4] == 15  # 1+2+3+4+5

def test_ewm():
    s = pd.Series([1.0, 1.0, 1.0])
    ew = s.ewm(alpha=0.5, adjust=False).mean()
    # alpha=0.5: 当前权重 0.5, 历史权重 0.5
    assert ew.iloc[0] == 1.0
    assert ew.iloc[1] == 1.0

def test_time_rolling():
    idx = pd.date_range('2025-01-01', periods=5, freq='D')
    s = pd.Series([10, 20, 30, 40, 50], index=idx)
    rm = s.rolling('3D').mean()
    assert rm.iloc[0] == 10.0
    assert rm.iloc[2] == 20.0  # 3天窗口：第1-3天

if __name__ == '__main__':
    test_rolling_mean()
    test_rolling_center()
    test_expanding()
    test_ewm()
    test_time_rolling()
    print("OK 所有测试通过")
```

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 窗口函数 | Excel 公式 | Python for 循环 |
|------|----------------|-----------|----------------|
| 滚动均值 | rolling(n).mean() 一行 | AVERAGE + 下拉，慢 | 手写循环，极慢 |
| 指数加权 | ewm(span=n).mean() | 无内置支持 | 需手工实现衰减公式 |
| 累计指标 | expanding().sum() | SUM($A$1:A1) 下拉 | 手写累积循环 |
| 时间窗口 | rolling('7D') 原生支持 | 需辅助日期列 | 需手工计算窗口边界 |
| 多指标组合 | 链式调用 | 每个指标单独操作 | 每个指标单独循环 |
| 性能 | C/Cython 优化，毫秒级 | 秒到分钟级 | 秒到分钟级 |

### 4.2 适用场景

- **适用场景**：
  1. 监控指标的滚动均值/标准差——构建异常检测基线
  2. 金融时间序列的移动平均线（MA）、布林带等
  3. 累计指标（GMV、用户数）的实时追踪
  4. 对近期变化敏感的趋势监控（ewm 指数加权）
  5. 窗口函数可作为特征工程的输入（如滚动 7 日最大回撤）
- **不适用场景**：
  1. 窗口大小和数据集差不多大——直接全局聚合即可
  2. 需要复杂的自定义窗口逻辑（如窗口内过滤后再聚合）——可用 rolling.apply 但性能差

### 4.3 注意事项

- **rolling 的 min_periods 默认值**：等于窗口大小，意味着前 N-1 个结果全是 NaN。对于监控场景，建议设置一个较小的 min_periods；
- **ewm 的 adjust 参数**：`adjust=True`（默认）用加权递归公式（更精确），`adjust=False` 用简单递推公式（类 Excel）。多数场景 `adjust=False` 更直观；
- **时间 rolling 的 closed 参数**：`closed='right'`（默认）窗口包含右端点，`closed='left'` 不包含；
- **rolling 和 resample 的区别**：resample 输出结果行数 = 时间段数，rolling 输出结果行数 = 原始行数。

### 4.4 常见踩坑经验

1. **基于时间的 rolling 和基于行数的 rolling 混用**：一位同事对不连续的日期数据用了 `rolling('7D')`，窗口内实际包含的天数不一致，导致监控阈值不准确。教训：基于时间的 rolling 要确保 timestamp 的间隔是均匀的，不均匀的先用 `asfreq` 补齐。
2. **ewm 的 adjust=True 产生"首项偏倚"**：默认的 adjust 模式下前几个值的权重分配和简单指数平均不同——如果你从数学课本上学的公式是 α*x(t)+(1-α)*EMA(t-1)，那应该设 `adjust=False`。
3. **rolling.std() 在窗口为空时返回 NaN**：如果一整段数据都是 NaN，std 返回 NaN 而非 0——`fillna(0)` 后可能导致误判为"无波动"。教训：异常检测的阈值检查前确认 rolling_std 非零。
