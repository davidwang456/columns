# 第11章：透视表与交叉分析

## 1. 项目背景

某全国连锁零售集团的运营总监每周五下午要召开经营分析会，核心议程是对比各区域、各品类、各渠道的销售表现。数据分析师小赵每周四需要准备一份"销售经营看板"，包含至少四个维度：

- **区域 x 品类**：每个区域下各品类的销售额和毛利率
- **城市 x 月份**：各城市的月度销售趋势
- **渠道 x 品类 x 会员等级**：不同渠道不同品类的会员消费分布
- **门店 x 商品 Top 排名**：各门店内额 Top 5 商品及其占比

数据源是 week_sales.csv（约 20 万行），包含：订单号、日期、门店、城市、区域、品类、商品、数量、单价、销售额、毛利率、渠道、会员等级。

当前小赵的做法是 Excel 透视表——他把这 20 万行数据导入 Excel，然后用数据透视表功能，把"区域"拖到行、"品类"拖到列、"销售额"拖到值区域。看起来只需要几步操作，但实际操作中：

**痛点一：多维度切换极其卡顿**。20 万行数据在 Excel 透视表里，每拖动一个维度字段，Excel 就要重新计算 10-20 秒。当总监在会议上临时要求"把行从区域换成城市，列从品类换成渠道"时，小赵现场操作等了近半分钟，场面尴尬。

**痛点二：多维交叉无法一步完成**。一个透视表只能展示"行 x 列 x 值"三维数据。要同时看"区域-城市-品类-渠道-月份"五维数据，必须做多个透视表再手工拼接。小赵做了 6 个透视表才覆盖所有维度。

**痛点三：百分比和合计逻辑复杂**。透视表中"行汇总百分比"、"列汇总百分比"和"总计百分比"的语义容易混淆。小赵就曾经把"销售额占行总计的百分比"误当成"占总销售的百分比"，向总监汇报了错误数据。

痛点流程：

```
周销售 CSV (20万行)
  └── Excel 透视表
        ├── 拖动维度 → 20秒重算 → 会议现场卡顿
        ├── 五维交叉 → 需6个透视表 → 手工拼接
        └── 百分比混淆 → 行/列/总计搞错 → 错误汇报
```

本章将用 pandas 的 pivot、pivot_table 和 crosstab 三件套构建多维经营看板，一次代码输出所有维度交叉结果，0.1 秒完成重算。

## 2. 项目设计：剧本式交锋对话

> 场景：小赵在茶水间抱怨"Excel 透视表太慢了"，小胖端着咖啡凑过来。

**小胖**：（捧着咖啡杯）"不就是一个二维矩阵吗？行是品类，列是区域，交叉格子里填销售额——这我也会用 python 的 for 循环写啊。何必用什么透视表？"

**大师**："小胖，你行 x 列只有一个数字（销售额），但如果总监要同时看销售额、毛利率、订单数三个指标呢？如果你的行是三层（区域→城市→门店），列是两层（品类→渠道）呢？用 for 循环嵌套五层吗？pandas 的 `pivot_table` 就是干这件事的。"

**小白**："pivot、pivot_table、crosstab 这三个我经常搞混，有什么区别？"

**大师**："记住三个场景：

- **`pivot`**：最简单的行列重塑。要求行列组合唯一，不做聚合。像把一张工单表按"日期→行、员工→列、工时→值"摊开。如果同一个日期+员工有多条记录，会报错。
- **`pivot_table`**：带聚合的透视表。和 Excel 透视表完全对应——行、列、值、聚合函数（默认 mean）。同一个行列组合有多条记录时自动聚合。支持 margins（合计行列）和 fill_value。
- **`crosstab`**：交叉频次表。专门统计两个分类变量的交叉频次——有多少订单是"北京"的"已完成"状态？本质上是 `pivot_table` 的频次统计特化版。

简单记忆：pivot = 无聚合的 reshape，pivot_table = 有聚合的 Excel 透视表，crosstab = 计数的交叉表。"

**【技术映射：pivot = 翻牌（单纯改变排列方式），pivot_table = 计分板（同时统计得分），crosstab = 点名册（统计每类多少人）】**

**小胖**："那 pivot_table 里的 margins 是什么？我看到过这个参数但从没用过。"

**大师**："`margins=True` 会在透视表最右边加一列 'All'，最下边加一行 'All'——分别是行合计和列合计。你可以想象成 Excel 透视表里勾选了'显示行总计'和'显示列总计'。

配合 `margins_name='合计'` 可以把默认的 'All' 改成中文的 '合计'。注意：margins 的计算用的是和值区域相同的 aggfunc，如果 aggfunc 是 'mean'，margins 也是 mean——这可能不是你想要的'总计平均'。"

**【技术映射：margins = 考试分数表最右侧的"全班平均"和最下侧的"各科平均"】**

**小白**："那透视表可以加多个值字段吗？比如同时看销售额和毛利率？"

**大师**："当然可以。pivot_table 的 `values` 参数支持列表，`aggfunc` 也支持列表或字典：

```python
# 多个值 + 相同聚合函数
pd.pivot_table(df, index='区域', columns='品类',
               values=['销售额', '毛利率'], aggfunc='sum')

# 多个值 + 不同聚合函数
pd.pivot_table(df, index='区域', columns='品类',
               values={'销售额': 'sum', '毛利率': 'mean'})

# 同一个值 + 多个聚合函数
pd.pivot_table(df, index='区域', columns='品类',
               values='销售额', aggfunc=['sum', 'mean', 'count'])
```

最灵活的是字典方式——为每个值字段指定一个聚合函数。如果同一值字段要多种聚合，就用列表。"

**【技术映射：values + aggfunc = 餐馆菜单——每个菜（值字段）可以有不同的做法（聚合函数），全写在一张单子上】**

**小白**："crosstab 呢？它的 normalize 参数是什么意思？"

**大师**："`normalize` 控制百分比的计算基准：

```python
pd.crosstab(df['区域'], df['品类'], values=df['销售额'], aggfunc='sum', normalize='all')
# normalize='all'    → 每个格子 / 总合计
# normalize='index'  → 每个格子 / 该行合计（行内百分比）
# normalize='columns'→ 每个格子 / 该列合计（列内百分比）
```

这和 Excel 透视表里的'值显示方式 → 总计的百分比 / 行汇总的百分比 / 列汇总的百分比'完全对应。你之前搞混的'行汇总百分比' vs '总计百分比'，在 pandas 里就是 `normalize='index'` vs `normalize='all'`。"

**大师总结**："今天三个核心工具：
1. `pivot`：无聚合，行列组合必须唯一；
2. `pivot_table`：带聚合的全功能透视表，支持 margins 和多值字段；
3. `crosstab`：专门的交叉频次表，normalize 控制百分比比例。

记住：能用 pivot_table 时不要用 pivot（因为数据很少保证唯一性），能用 aggfunc 字典时不要用列表（因为返回 MultiIndex 列难受）。"

```python
# generate_sales_data.py
import pandas as pd
import numpy as np

np.random.seed(42)
N = 5000  # 模拟一周数据

regions = ['华东', '华南', '华北', '西南', '华中']
cities_in = {
    '华东': ['上海','杭州','南京','苏州'],
    '华南': ['广州','深圳','东莞','佛山'],
    '华北': ['北京','天津','石家庄','太原'],
    '西南': ['成都','重庆','昆明','贵阳'],
    '华中': ['武汉','长沙','郑州','南昌'],
}
categories = ['服装','电子产品','食品','家居','美妆']
channels = ['线上-App','线上-小程序','线下-门店','线下-专柜']
member_levels = ['普通','银卡','金卡','钻石']

data = []
for _ in range(N):
    region = np.random.choice(regions)
    city = np.random.choice(cities_in[region])
    cat = np.random.choice(categories, p=[0.3,0.2,0.2,0.15,0.15])
    product = f'{cat}-{np.random.choice(["A","B","C","D"])}'
    qty = np.random.randint(1, 5)
    price = np.random.uniform(30, 800)
    sales = round(qty * price, 2)
    margin = round(np.random.uniform(0.15, 0.55), 4)

    data.append({
        'date': pd.Timestamp('2025-03-17') + pd.Timedelta(days=np.random.randint(0,7)),
        'order_id': f'ORD{np.random.randint(80000,99999)}',
        'store': f'店-{region[:2]}-{np.random.randint(1,20):02d}',
        'city': city,
        'region': region,
        'category': cat,
        'product': product,
        'qty': qty,
        'unit_price': round(price, 2),
        'sales': sales,
        'margin_rate': margin,
        'channel': np.random.choice(channels),
        'member_level': np.random.choice(member_levels, p=[0.4,0.3,0.2,0.1]),
    })

df = pd.DataFrame(data)
df['margin_amount'] = round(df['sales'] * df['margin_rate'], 2)
df.to_csv('week_sales.csv', index=False, encoding='utf-8-sig')
print(f"已生成 {len(df)} 条销售记录，{df['region'].nunique()} 个区域，{df['category'].nunique()} 个品类")
```

### 3.3 分步实现

#### 步骤 1：pivot 与 pivot_table 基础

**目标**：理解 pivot 的限制和 pivot_table 的多参数使用。

```python
# step1_pivot_basics.py
import pandas as pd
import numpy as np

df = pd.read_csv('week_sales.csv', parse_dates=['date'])

# === pivot：无聚合的行列重整 ===
# 先确保城市-品类-日期组合唯一（汇总到天级别）
daily = df.groupby(['city','category','date'], as_index=False)['sales'].sum()
# 现在每个 (city, date) 只有一行 → pivot 安全
pivot_result = daily.pivot(index='date', columns='city', values='sales')
print("pivot 结果形状:", pivot_result.shape)
print(pivot_result.iloc[:3, :4].round(0).to_string())

# === pivot_table：带聚合的透视表 ===
print("\n===== 区域 x 品类 销售看板 =====")
pt = pd.pivot_table(
    df,
    index='region',           # 行维度
    columns='category',       # 列维度
    values='sales',           # 值字段
    aggfunc='sum',            # 聚合函数
    fill_value=0,             # 空值填 0
    margins=True,             # 显示合计
    margins_name='合计'
)
print(pt.round(0).to_string())

print("\n===== 区域 x 品类 毛利率（平均值）====")
pt_margin = pd.pivot_table(
    df, index='region', columns='category',
    values='margin_rate', aggfunc='mean', fill_value=0, margins=True, margins_name='合计'
)
print((pt_margin * 100).round(1).to_string())
```

#### 步骤 2：多值字段与多聚合函数

**目标**：用 pivot_table 输出多指标交叉看板。

```python
# step2_multi_metrics.py
import pandas as pd

df = pd.read_csv('week_sales.csv', parse_dates=['date'])

# === 单个值 + 多个聚合函数 ===
print("===== 区域销售：总额/均值/笔数 =====")
pt1 = pd.pivot_table(
    df, index='region', values='sales',
    aggfunc=['sum', 'mean', 'count'], fill_value=0
)
pt1.columns = ['总销售额', '平均客单', '订单数']
print(pt1.round(1).to_string())

# === 多个值 + 相同聚合 ===
print("\n===== 品类：销售额+毛利额 (sum) =====")
pt2 = pd.pivot_table(
    df, index='category',
    values=['sales', 'margin_amount'], aggfunc='sum', fill_value=0
)
print(pt2.round(0).to_string())

# === 多个值 + 不同聚合函数（字典方式） ===
print("\n===== 渠道 x 会员等级：多维看板 =====")
pt3 = pd.pivot_table(
    df,
    index='channel',
    columns='member_level',
    values={'sales': 'sum', 'margin_rate': 'mean', 'order_id': 'nunique'},
    aggfunc={'sales': 'sum', 'margin_rate': 'mean', 'order_id': 'nunique'},
    fill_value=0
)
print(pt3.round(2).to_string())
```

#### 步骤 3：crosstab 与百分比分析

**目标**：用 crosstab 做频次统计和百分比分析。

```python
# step3_crosstab.py
import pandas as pd

df = pd.read_csv('week_sales.csv', parse_dates=['date'])

# === 基础频次表：区域 x 品类的订单数 ===
print("===== 区域 x 品类 订单交叉频次 =====")
ct1 = pd.crosstab(df['region'], df['category'], margins=True, margins_name='合计')
print(ct1.to_string())

# === 带值的 crosstab：区域 x 品类的销售额 ===
print("\n===== 区域 x 品类 销售额交叉（normalize=all）=====")
ct2 = pd.crosstab(
    df['region'], df['category'],
    values=df['sales'], aggfunc='sum',
    normalize='all',   # 每个格子 / 总合计 = 占比
    margins=True, margins_name='合计'
)
print((ct2 * 100).round(2).to_string())

# === 行内百分比：每个区域的品类销售占比 ===
print("\n===== 各区域内品类销售占比 (normalize=index) =====")
ct3 = pd.crosstab(
    df['region'], df['category'],
    values=df['sales'], aggfunc='sum',
    normalize='index', margins=True, margins_name='合计'
)
print((ct3 * 100).round(1).to_string())

# === 列内百分比：每个品类的区域分布 ===
print("\n===== 各品类内区域分布 (normalize=columns) =====")
ct4 = pd.crosstab(
    df['region'], df['category'],
    values=df['sales'], aggfunc='sum',
    normalize='columns', margins=True, margins_name='合计'
)
print((ct4 * 100).round(1).to_string())
```

#### 步骤 4：构建多 Sheet 经营看板

**目标**：将多个透视表整合输出为 Excel 经营看板。

```python
# step4_dashboard.py
import pandas as pd
from openpyxl import load_workbook
from openpyxl.styles import Font, PatternFill, Alignment, numbers
from openpyxl.utils import get_column_letter

df = pd.read_csv('week_sales.csv', parse_dates=['date'])

# 四个看板 Sheet
dashboards = {
    '区域x品类销售额': pd.pivot_table(df, index='region', columns='category',
                        values='sales', aggfunc='sum', fill_value=0, margins=True, margins_name='合计'),
    '城市x渠道笔数': pd.pivot_table(df, index='city', columns='channel',
                        values='order_id', aggfunc='nunique', fill_value=0, margins=True, margins_name='合计'),
    '品类x会员客单价': pd.pivot_table(df, index='category', columns='member_level',
                        values='sales', aggfunc='mean', fill_value=0).round(2),
    'Top10门店销售': df.groupby('store')['sales'].sum().nlargest(10).reset_index(),
}

# 导出带格式
filename = '销售经营看板.xlsx'
with pd.ExcelWriter(filename, engine='openpyxl') as writer:
    for sheet_name, table in dashboards.items():
        table.to_excel(writer, sheet_name=sheet_name)

# openpyxl 添加样式
wb = load_workbook(filename)
header_fill = PatternFill(start_color='2F5496', end_color='2F5496', fill_type='solid')
header_font = Font(bold=True, color='FFFFFF', size=11)
number_fmt = '#,##0'

for ws in wb.worksheets:
    # 表头样式
    for cell in ws[1]:
        cell.fill = header_fill
        cell.font = header_font
        cell.alignment = Alignment(horizontal='center')
    # 数值列格式
    for col in range(2, ws.max_column + 1):
        for row in range(2, ws.max_row + 1):
            cell = ws.cell(row=row, column=col)
            if isinstance(cell.value, (int, float)):
                cell.number_format = number_fmt
    # 自适应列宽
    for col_idx in range(1, ws.max_column + 1):
        max_len = max(len(str(ws.cell(row=r, column=col_idx).value or '')) for r in range(1, ws.max_row + 1))
        ws.column_dimensions[get_column_letter(col_idx)].width = min(max_len + 5, 30)

wb.save(filename)
print(f"OK 经营看板已导出至 {filename}")
```

### 3.4 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `pivot` 报错 "duplicate entries" | 行列组合不唯一 | 先用 `groupby` 聚合，或用 `pivot_table` |
| pivot_table 结果有 MultiIndex 列 | 传入了 `aggfunc` 列表 | 取一层：`pt.columns = pt.columns.droplevel(0)` |
| margins 的 mean 不是"总平均" | aggfunc='mean' 时 margins 是各组均值的平均 | 表格外单独计算 `df['val'].mean()` |
| crosstab normalize 后各格子加起来≠ 100% | 是四舍五入 | 没问题，差值在万分之几 |
| pivot_table 数值列也成了 Index | 没指定 `values` 参数 | 指定 `values='sales'` 而非全部数值列 |
| aggfunc 字典 keys 报错 | 字典 key 必须是列名，value 必须是函数名 | 检查 `values={'sales':'sum'}` 中 'sum' 是否有效 |

### 3.5 测试验证

```python
# test_ch11.py
import pandas as pd
import numpy as np

def test_pivot_table_basics():
    df = pd.DataFrame({
        'r': ['A','A','B','B'], 'c': ['X','Y','X','Y'], 'v': [10,20,30,40]
    })
    pt = pd.pivot_table(df, index='r', columns='c', values='v', aggfunc='sum', fill_value=0)
    assert pt.loc['A','X'] == 10
    assert pt.loc['B','Y'] == 40

def test_pivot_table_margins():
    df = pd.DataFrame({
        'r': ['A','A','B'], 'c': ['X','Y','X'], 'v': [10,20,30]
    })
    pt = pd.pivot_table(df, index='r', columns='c', values='v',
                        aggfunc='sum', margins=True, margins_name='合计')
    assert pt.loc['合计','合计'] == 60

def test_crosstab_normalize():
    df = pd.DataFrame({
        'r': ['A','A','B'], 'c': ['X','Y','X'], 'v': [10,30,60]
    })
    ct = pd.crosstab(df['r'], df['c'], values=df['v'],
                     aggfunc='sum', normalize='all')
    # 总合计 100，各占比加起来 = 1
    assert abs(ct.sum().sum() - 1.0) < 0.001

def test_pivot_vs_pivot_table():
    df = pd.DataFrame({'d':['2025-01','2025-01','2025-02'],
                       'c':['BJ','BJ','SH'], 'v':[100,200,300]})
    # pivot 直接报错（重复），pivot_table 自动聚合
    try:
        df.pivot(index='d', columns='c', values='v')
        assert False, "应报错"
    except ValueError:
        pass
    pt = pd.pivot_table(df, index='d', columns='c', values='v', aggfunc='sum')
    assert pt.loc['2025-01','BJ'] == 300  # 100+200

if __name__ == '__main__':
    test_pivot_table_basics()
    test_pivot_table_margins()
    test_crosstab_normalize()
    test_pivot_vs_pivot_table()
    print("OK 所有测试通过")
```

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas pivot_table | Excel 透视表 | SQL GROUP BY + CASE WHEN |
|------|-------------------|-------------|-------------------------|
| 多维交叉 | 一行代码，任意行列组合 | 拖拽操作，多 Sheet 受限 | 需嵌套子查询或 PIVOT 语法 |
| 百分比计算 | normalize 三种模式语义明确 | 右键值字段设置，易混淆 | 需手工除子查询 |
| 多值聚合 | values+aggfunc 字典灵活 | 值区域拖多个字段 | 多个聚合函数分开写 |
| 输出集成 | 直接喂给下一段代码 | 需导出再导入 | 结果集可直接用 |
| 性能 | 百万行秒级 | 十万行开始卡顿 | 依赖索引 |
| 学习成本 | 中等 | 低 | 高（PIVOT 语法） |

### 4.2 适用场景

- **适用场景**：
  1. 多维度（3-5 维）的经营分析看板——区域 x 品类 x 渠道 x 时段
  2. 需要和 Excel 透视表结果交叉校验的报表
  3. 百分比分析（行内占比、列内占比、总占比）的快速计算
  4. 频次统计和交叉分布分析（crosstab）
  5. 作为数据管道的中间步骤——透视表结果直接传给 merge 或可视化
- **不适用场景**：
  1. 只需要一维 groupby 汇总——pivot_table 反而多设置一层列
  2. 行列组合极少（如 2x2）——直接用 groupby + unstack 更简洁

### 4.3 注意事项

- **pivot 比 pivot_table 快但不鲁棒**：确定数据唯一时才用 pivot，否则用 pivot_table（有自动聚合）；
- **pivot_table 默认 aggfunc 是 mean**：不指定时静默取均值，可能导致"销售额"变成平均值——始终显式指定；
- **MultiIndex 列的维护**：多 aggfunc 返回 MultiIndex 列，用 `df.columns = ['_'.join(c) for c in df.columns]` 拍平；
- **fill_value 的选择**：不能用 0 替代缺失值的场景（如毛利率）——用 NaN 更安全，保留空值语义。

### 4.4 常见踩坑经验

1. **pivot_table margins 返回的 mean 不是总平均**：一位同事用 `margins=True, aggfunc='mean'` 做毛利率看板，看到"合计"行显示 28%，就向总监报告"整体毛利率 28%"。但实际上 margins 的 mean 是各组均值的简单平均（而非加权），真实整体毛利率是 31%。教训：margins 的合计值用 `aggfunc='sum'` 最准确，平均类指标单独算。
2. **crosstab normalize='index' 时百分比基数被误解**：`normalize='index'` 是每行加总 = 1；但一位同事理解成"全球数据的百分比"，导致分析结论偏差。教训：使用 normalize 必在代码注释中写明基准。
3. **对 category 列做 pivot_table 结果包含未出现的类别**：如果某列有 category dtype 且 categories 包含"未出现"的值，pivot_table 的 columns 会包含这些未出现类别（值为 0）。用 `observed=True`（如果是 groupby 场景）或提前 `astype(str)` 解决。
