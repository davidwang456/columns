# 第12章：时间序列基础与重采样

## 1. 项目背景

某移动 App 的产品团队需要做一份"过去三个月的用户活跃度分析报告"，数据来自埋点系统导出的 daily_active.csv，包含两列：日期（date）和日活跃用户数（dau），覆盖 2025 年 1 月至 3 月共 90 天的数据。

产品经理小吴需要回答以下问题：
- "这三个月整体 DAU 趋势是上升还是下降？"
- "按周汇总的 DAU 是否有周期性规律？周末比工作日高还是低？"
- "以 7 天为窗口，DAU 的移动平均线是否出现了异常波动？"
- "和去年同期相比，DAU 增长了多少？"

小吴目前的做法是在 Excel 里操作：先用图表看趋势（粗略），然后在 Excel 里手工新增辅助列计算"周均值"。当她需要按周汇总时，要手工创建一个"周编号"列（WEEKNUM），再用透视表按周汇总。但她很快发现了问题：

**痛点一：缺失日期导致周均值失真**。2 月 1 日到 2 月 3 日之间数据缺失（服务器故障），Excel 透视表并不知道这几天"应该存在但不在了"，按周汇总时这周只有 4 天数据，但均值除的是 4 天——比实际低了 25%。

**痛点二：日期索引缺失导致环比计算复杂**。小吴想知道"这一天的 DAU 比前一天涨了多少"，她在 Excel 里手工新增一列，用"=B3-B2"公式——但表格一旦排序，公式就全部错乱，因为相对引用依赖行号。

**痛点三：时间维度转换靠手工**。当总监要求"把日数据改成周数据，再改成月数据"时，小吴必须重新做三次透视表。而且 Excel 的周编号和 ISO 周标准不一致，跨年周让人非常困惑。

痛点流程：

```
DAU 日志 (90天)
  └── Excel 时间分析
        ├── 缺失日期 → 周均值被低估 25%
        ├── 环比公式 → 排序后全错乱
        └── 日→周→月转换 → 手工重做三次
```

本章将系统讲解 pandas 的时间序列基础设施——DatetimeIndex、resample（重采样）、shift/diff（位移和差分）、以及缺失日期的补齐策略。

## 2. 项目设计：剧本式交锋对话

> 场景：小吴在汇报时发现"周均值明显偏低"，大师看了一眼他的 Excel 公式就发现了问题。

**小胖**：（咬着一个面包）"日期不就是一列字符串吗？用 groupby 按 'YYYY-WW' 分组不就行了？"

**大师**："字符串分组是一种办法——但如果你把日期当成普通字符串，你就会失去 pandas 最强大的时间序列功能。当日期被设为 DatetimeIndex 之后，一切都不一样了：

- `df.resample('W')`：一键按周汇总，自动识别缺失周
- `df.shift(1)`：取前一天的 DAU，完美处理间隔
- `df.loc['2025-01':'2025-03']`：按自然日期切片

这就是为什么 pandas 有专门的 DatetimeIndex——它不是普通的 Index，它是带时间语义的。"

**【技术映射：DatetimeIndex = 有内置日历的日程本——翻页就知道下周、上月、去年的对应日期，不需要手工查】**

**小白**："那 resample 是干什么的？和 groupby 有什么不同？"

**大师**："`resample` 本质上是时间专用的 groupby。差别在于：
- `groupby`：按任意键分组，不管顺序；
- `resample`：按时间频率分组，天然有序，且**能补全缺失的时间桶**。

比如你的数据只有 1 月 1 日和 1 月 3 日的记录——按天 resample 时会自动生成 1 月 2 日这一行（值为 NaN），而 groupby 不会。

```python
# DatetimeIndex 上的 resample
df.set_index('date').resample('D')['dau'].sum()

# 等价于 groupby + 日期派生列
df.groupby(pd.Grouper(key='date', freq='D'))['dau'].sum()
```

resample 的频率字符串：'D'=天, 'W'=周, 'M'=月, 'Q'=季, 'Y'=年, 'H'=小时, 'T'=分钟。还可以用 'W-MON' 表示以周一为起点的周，'QS' 表示季度开始。"

**【技术映射：resample = 自动分组的日历——不管某天有没有数据，日程本上该天必须存在】**

**小胖**："那 shift 和 diff 又是啥？我好像见过但从来没用过。"

**大师**："`shift` 和 `diff` 是时间序列的'平移'和'阶差'运算：

```python
# 今天的 DAU 和昨天对比
df['prev_day'] = df['dau'].shift(1)     # 上一行（前一天）的值
df['change'] = df['dau'].diff(1)         # 和上一行的差值
df['pct_change'] = df['dau'].pct_change() # 环比变化率

# shift 可以向前也可以向后
df['next_day'] = df['dau'].shift(-1)   # 下一行（后一天）的值
```

注意：`shift` 是按位置平移的——即使时间索引有间隔，`shift(1)` 也是取'上一行'而非'前一天'。如果数据不是严格等间隔的，需要先用 `asfreq` 补齐。"

**小白**："所以我需要先补齐缺失日期？"

**大师**："对，这是时间序列分析的标准前处理：

1. **创建完整日期范围**：`pd.date_range('2025-01-01', '2025-03-31', freq='D')`
2. **对齐（reindex）**：用完整日期范围做 reindex，缺失的自动填 NaN
3. **填充缺失**：对 NaN 做 `fillna` 或 `interpolate`

```python
full_dates = pd.date_range(start='2025-01-01', end='2025-03-31', freq='D')
df = df.set_index('date').reindex(full_dates)
df['dau'] = df['dau'].interpolate()  # 线性插值补缺失
```

这一步补齐之后，shift 和 diff 的结果才是真正'和前一天对比'——而不是'和上一行对比'。"

**【技术映射：reindex + date_range = 日历补页——把缺失的日期页都补上，该空白的地方先空白再填充】**

**小白**："那 resample 的聚合方法有哪些？除了 sum 和 mean 还有别的吗？"

**大师**："resample 对象支持几乎所有聚合操作：

```python
df['dau'].resample('W').agg(['sum', 'mean', 'max', 'min', 'std', 'first', 'last'])
```

还有两个 resample 特有的：
- `ohlc()`：开盘-最高-最低-收盘（金融 K 线），一次拿四个值
- `sem()`：标准误差

resample 还可以链式加方法：
```python
df['dau'].resample('W').mean().fillna(0)
```"

**大师总结**："时间序列分析的四个核心步骤：
1. 设置 DatetimeIndex → 解锁所有时间语义功能
2. `date_range` + `reindex` → 补齐缺失的时间点
3. `resample` → 任意时间粒度的升降采样
4. `shift` / `diff` / `pct_change` → 时间上的比较和趋势分析

下面我们把 90 天的 DAU 日志变成一份可交付的活跃度分析报告。"

### 3.2 模拟 DAU 数据

```python
# generate_dau_data.py
import pandas as pd
import numpy as np

np.random.seed(42)
dates = pd.date_range('2025-01-01', '2025-03-31', freq='D')

# 模拟有周期性的 DAU：工作日高、周末低 + 增长趋势 + 随机波动
base = 50000
trend = np.linspace(0, 8000, len(dates))
weekly = np.where(dates.weekday < 5, 1.0, 0.75)  # 周末偏低
noise = np.random.normal(0, 3000, len(dates))

dau = (base + trend) * weekly + noise
dau = np.maximum(dau, 0).astype(int)

df = pd.DataFrame({'date': dates, 'dau': dau})

# 故意删除几个日期的数据（模拟缺失）
missing_idx = [10, 35, 36, 60, 75]  # 1月11日, 2月4-5日, 3月1日, 3月16日
for idx in missing_idx:
    df.loc[df.index == idx, 'dau'] = np.nan

df = df.dropna(subset=['dau'])  # 删除缺失行（模拟数据丢失）
df.to_csv('daily_active.csv', index=False, encoding='utf-8-sig')
print(f"已生成 {len(df)} 天 DAU 数据（原始90天，现存{len(df)}天，{90-len(df)}天缺失）")
```

### 3.3 分步实现

#### 步骤 1：DatetimeIndex 与日期补充

**目标**：设置日期索引，补齐缺失日期，理解 DatetimeIndex 的时间语义。

```python
# step1_datetime_index.py
import pandas as pd
import numpy as np

df = pd.read_csv('daily_active.csv', parse_dates=['date'])
print(f"原始行数: {len(df)}")

# === 设为 DatetimeIndex ===
df = df.set_index('date').sort_index()
print(f"Index type: {type(df.index)}")
print(f"日期范围: {df.index.min().date()} ~ {df.index.max().date()}")

# === 补齐缺失日期 ===
full_range = pd.date_range(start=df.index.min(), end=df.index.max(), freq='D')
df = df.reindex(full_range)
print(f"补齐后行数: {len(df)}，缺失天数: {df['dau'].isna().sum()}")

# === 填充缺失值（线性插值）===
df['dau'] = df['dau'].interpolate(method='linear')
print(f"插值后缺失值: {df['dau'].isna().sum()}")

# === DatetimeIndex 带来的便利 ===
print(f"\n1月第一周数据:")
print(df.loc['2025-01-01':'2025-01-07'])
```

#### 步骤 2：resample 升降采样

**目标**：按周、月重采样，对比不同采样频率的聚合结果。

```python
# step2_resample.py
import pandas as pd

df = pd.read_csv('daily_active.csv', parse_dates=['date'])
df = df.set_index('date').sort_index()
full_range = pd.date_range(df.index.min(), df.index.max(), freq='D')
df = df.reindex(full_range)
df['dau'] = df['dau'].interpolate()

# === 按周重采样 ===
weekly = df['dau'].resample('W-MON').agg(['sum', 'mean', 'max', 'min', 'std'])
weekly.columns = ['周总和', '周均值', '周最高', '周最低', '周标准差']
print("===== 按周汇总 (周一为起点) =====")
print(weekly.round(0).to_string())

# === 按月重采样 ===
monthly = df['dau'].resample('ME').agg(['mean', 'sum'])
monthly.columns = ['月均DAU', '月总活跃']
print("\n===== 按月汇总 =====")
print(monthly.round(0).to_string())

# === 按周且区分工作日/周末 ===
df['weekday'] = df.index.weekday
df['is_weekend'] = df['weekday'] >= 5
weekend_comparison = df.groupby('is_weekend')['dau'].mean()
print(f"\n工作日均DAU: {weekend_comparison[False]:.0f}")
print(f"周末均DAU:   {weekend_comparison[True]:.0f}")
```

#### 步骤 3：shift、diff 与环比分析

**目标**：用 shift 和 diff 计算日环比变化。

```python
# step3_shift_diff.py
import pandas as pd

df = pd.read_csv('daily_active.csv', parse_dates=['date'])
df = df.set_index('date').sort_index()
full_range = pd.date_range(df.index.min(), df.index.max(), freq='D')
df = df.reindex(full_range)
df['dau'] = df['dau'].interpolate()

# === shift：获取前一天/后一天的值 ===
df['prev_day'] = df['dau'].shift(1)
df['next_day'] = df['dau'].shift(-1)

# 验证：shift(1) 就是上一行
print("shift 验证（前5行）:")
print(df[['dau', 'prev_day']].head().to_string())

# === diff：计算日环比变化 ===
df['day_change'] = df['dau'].diff(1)       # 差值
df['day_pct'] = df['dau'].pct_change() * 100  # 变化率(%)

# 找出变化最大的 3 天
largest_changes = df.nlargest(3, 'day_change')
largest_drops = df.nsmallest(3, 'day_change')

print(f"\n===== DAU 日变化 Top 3 (大涨) =====")
print(largest_changes[['dau', 'day_change', 'day_pct']].round(1).to_string())

print(f"\n===== DAU 日变化 Top 3 (大跌) =====")
print(largest_drops[['dau', 'day_change', 'day_pct']].round(1).to_string())

# === 7 日移动平均（下周详讲，本章先感受）===
df['ma7'] = df['dau'].rolling(7, min_periods=1).mean()
print(f"\n最后 7 天的 7 日移动平均:")
print(df[['dau', 'ma7']].tail(7).round(0).to_string())
```

#### 步骤 4：PeriodIndex 与 asfreq

**目标**：理解 PeriodIndex 及其与 DatetimeIndex 的转换。

```python
# step4_period.py
import pandas as pd

df = pd.read_csv('daily_active.csv', parse_dates=['date'])
df = df.set_index('date').sort_index()
full_range = pd.date_range(df.index.min(), df.index.max(), freq='D')
df = df.reindex(full_range)
df['dau'] = df['dau'].interpolate()

# === DatetimeIndex → PeriodIndex ===
# PeriodIndex: 时间"段"而非时间"点"
monthly_period = df['dau'].resample('ME').sum()
monthly_period.index = monthly_period.index.to_period('M')
print("PeriodIndex (月份):")
print(monthly_period.to_string())

# === asfreq：改变频率但不聚合 ===
# 从日数据"降采样"到周数据（取每周最后一天）
weekly_last = df['dau'].asfreq('W-FRI')
print(f"\nasfreq 到每周五: {len(weekly_last.dropna())} 个有效值")

# === date_range 生成时间序列 ===
# 生成未来 30 天的日期范围
future_dates = pd.date_range('2025-04-01', periods=30, freq='D')
print(f"\npd.date_range 生成: {len(future_dates)} 天, {future_dates[0].date()} ~ {future_dates[-1].date()}")
```

### 3.4 完整主流程

```python
# main_dau_report.py
"""App 用户活跃度分析报告"""
import pandas as pd
import numpy as np

df = pd.read_csv('daily_active.csv', parse_dates=['date'])
df = df.set_index('date').sort_index()

# 1. 补齐缺失日期并插值
full_range = pd.date_range(df.index.min(), df.index.max(), freq='D')
df = df.reindex(full_range)
df['dau'] = df['dau'].interpolate()

# 2. 日环比
df['day_change'] = df['dau'].diff(1)
df['day_pct'] = (df['dau'].pct_change() * 100).round(2)

# 3. 7日移动平均
df['ma7'] = df['dau'].rolling(7, min_periods=1).mean().round(0)

# 4. 按周汇总
weekly = df['dau'].resample('W-MON').agg(
    日均DAU='mean', 周最高='max', 周最低='min', 周环比='last'
).round(0)
weekly['周环比'] = weekly['日均DAU'].pct_change() * 100

# 5. 按月汇总
monthly = df['dau'].resample('ME').agg(月均DAU='mean', 月最高='max').round(0)

# 6. 汇总报告
print("===== App 活跃度趋势报告 (2025 Q1) =====")
print(f"季度日均 DAU: {df['dau'].mean():.0f}")
print(f"季度最高 DAU: {df['dau'].max()}")
print(f"季度最低 DAU: {df['dau'].min()}")
print(f"趋势: {'上升' if df['dau'].iloc[-1] > df['dau'].iloc[0] else '下降'}")
print(f"周末效应: 工作日均值 {df[df.index.weekday<5]['dau'].mean():.0f} vs 周末均值 {df[df.index.weekday>=5]['dau'].mean():.0f}")
print(f"\n月度趋势:")
print(monthly.to_string())

# 7. 导出
df.reset_index().rename(columns={'index':'date'}).to_csv('dau_analysis.csv', index=False)
print(f"\nOK 分析结果已导出至 dau_analysis.csv")
```

### 3.5 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| resample 后日期变成 PeriodIndex | 某些操作自动转换了类型 | 用 `.to_timestamp()` 转回 DatetimeIndex |
| `shift(1)` 在缺失日期上不是"前一天" | shift 按位置而非时间 | 先 `reindex` 补齐完整时间范围 |
| resample('W') 默认周日为每周第一天 | pandas 遵循 ISO 惯例可配 | 改用 `'W-MON'` 或以周一为起点的频率 |
| 跨年周编号混乱 | ISO 周和自然年不同 | 用 `df.index.isocalendar().week` 取 ISO 周 |
| `asfreq` 下采样返回 NaN | 聚合方法不对 | 降采样用 resample，asfreq 是纯对齐 |
| date_range 包含时区信息 | 默认无时区，手动指定 | `pd.date_range(..., tz='Asia/Shanghai')` |

import pandas as pd
import numpy as np

def test_datetime_index_slice():
    idx = pd.date_range('2025-01-01', periods=10, freq='D')
    s = pd.Series(range(10), index=idx)
    subset = s.loc['2025-01-03':'2025-01-05']
    assert len(subset) == 3
    assert subset.iloc[0] == 2

def test_resample_sum():
    idx = pd.date_range('2025-01-01', periods=10, freq='D')
    s = pd.Series(range(10), index=idx)
    result = s.resample('W').sum()
    assert result.sum() == 45

def test_shift():
    s = pd.Series([1, 2, 3, 4], index=pd.date_range('2025-01-01', periods=4))
    s_shifted = s.shift(1)
    assert pd.isna(s_shifted.iloc[0])
    assert s_shifted.iloc[1] == 1
    assert s_shifted.iloc[3] == 3

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 时间序列 | Excel 时间函数 | SQL 时间函数 |
|------|---------------|---------------|-------------|
| 缺失补全 | reindex + interpolate 自动补齐 | 需手工加行 | 需生成日期序列表 |
| 频率转换 | resample 一行完成 | 透视表+手工分组 | DATE_TRUNC + GROUP BY |
| 位移运算 | shift/diff/pct_change 向量化 | 公式依赖行号 | LAG/LEAD 窗口函数 |
| 日期范围 | date_range 灵活生成 | 手工拖拽填充 | GENERATE_SERIES |
| 多粒度分析 | 日/周/月/季/年随意切换 | 需多次操作 | 多次查询 |
| 时区支持 | 内置 tz 参数 | 几乎不支持 | 依赖数据库配置 |

### 4.2 适用场景

- **适用场景**：
  1. 日活/月活/流水等时间序列指标的升降采样分析
  2. 环比/同比变化率的快速计算
  3. 缺失日期补齐和趋势插值
  4. 时间维度上的滚动聚合（移动平均等，第 13 章深入）
  5. 财务报表、金融行情等时间密集型数据的处理
- **不适用场景**：
  1. 不需要时间语义的纯分类数据——DatetimeIndex 反而增加开销
  2. 毫秒级高频交易数据——pandas 可能不够快

### 4.3 注意事项

- **resample 的 label 和 closed 参数**：`label='right'` vs `'left'` 决定时间桶用哪个端点标记，`closed='right'` vs `'left'` 决定桶的闭合边界。默认 `label='left', closed='right'`——即[左开右闭]，标签用左端点；
- **shift vs tshift**：`shift` 移动数据（行内容），`tshift` 移动索引（时间标签），pandas 2.0+ 中 `tshift` 已弃用，用 `shift(freq='D')` 替代；
- **pct_change 的除零问题**：当上期值为 0 时 `pct_change` 返回 inf，用 `replace([np.inf, -np.inf], np.nan)` 处理。

### 4.4 常见踩坑经验

1. **resample('M') 的月末边界问题**：`resample('M')` 默认 `closed='right'`，1 月 31 日的数据会归入 1 月桶（标签为 1 月最后一天）。但如果你期望 1 月 31 日归入 2 月桶——设置 `closed='left'`。
2. **reindex 后行数暴增**：一位同事对跨 3 年的时间范围做 `reindex(freq='D')`，生成了 1000+ 天的空白行——但数据实际只有 200 天。教训：reindex 前确认原始数据的实际时间范围，避免用全局 min/max 做边界。
3. **shift 依赖行序而非日期**：数据按日期排序但日期不连续（1 月 2 日之后是 1 月 5 日），`shift(1)` 返回的是 1 月 5 日的值而非 1 月 3 日的值。务必先 `reindex` 补齐完整日期再做 shift。
