# 第14章：数据导出与报告交付

## 1. 项目背景

某连锁零售品牌的区域经理每周需要向总部提交一份"经营周报"，包含以下内容：

- 各门店的销售额、客单价、订单数排名
- 各品类的销售占比饼图数据
- 高库存预警商品清单（库存周转 > 90 天）
- 异常指标标红（销售额环比下降 > 20% 标红）

数据来源于第 9 章的 POS 交易数据和第 6 章的库存数据，分析师小李已经用 pandas 完成了所有分析和计算，剩最后一步：**把结果变成别人能看、能用的交付物**。

小李目前的交付方式是：在 pandas 里用 `to_csv` 导出 CSV，再用 Excel 打开 CSV，手工调格式（加表头颜色、调整列宽、冻结窗格、标红异常值），然后另存为 XLSX。听起来很合理——但这都是纯手工操作。

**痛点一：CSV 缺乏格式信息**。CSV 只能存数据，不能存颜色、列宽、字体、冻结窗格等格式。小李每次都要花 20-30 分钟在 Excel 里手动美化。更烦的是，某次他调整了数据再重新导出，所有的格式调优都要重做。

**痛点二：多 Sheet 和格式化需要手工重复**。周报包含 6 个 Sheet：门店 KPI、品类趋势、库存预警、周环比、Top 商品、异常汇总。每个 Sheet 都要手动设置表头颜色、数字格式、列宽——6 x 5 分钟 = 30 分钟。

**痛点三：异常标红无法自动化**。销售额下降 20% 的门店需要标红，但 CSV 不支持条件格式。小李只能打开 CSV 肉眼找异常行，再手工标红——一旦看漏就漏报了。上周就有两个异常门店被他漏标，区域经理在会议上被总部点名批评。

痛点流程：

```
分析好的 DataFrame
  └── to_csv → CSV 文件
        └── Excel 打开 → 手工调格式(30min)
              ├── 表头颜色 → 手工设置
              ├── 多 Sheet → 手工拆分
              └── 异常标红 → 肉眼查找 → 可能漏报
```

本章将系统讲解 pandas 的数据导出能力——to_csv/to_excel/to_parquet、ExcelWriter 的多 Sheet 格式化、Styler 的条件格式、以及导出前数据校验清单。

## 2. 项目设计：剧本式交锋对话

> 场景：小李在周报中被点名批评"漏标了两个异常门店"，小胖帮忙分析原因。

**小胖**：（吃着棒棒糖）"把 DataFrame 导出成 Excel 谁不会啊？`df.to_excel('report.xlsx')` 一行搞定！"

**大师**："一行搞定的是**裸数据**。如果你不设置格式，导出的 XLSX 就是白底黑字的——没有表头样式、没有异常高亮、没有数字千分位、没有列宽自适应。老板打开后第一反应：'你就给我看这个？'

pandas 的 `ExcelWriter` 配合 `openpyxl` 可以做：
- 表头颜色和加粗
- 数字格式（千分位、百分比、金额）
- 自适应列宽
- 冻结首行（表头固定）
- 条件格式（标红异常）

这才是'交付物'和'裸数据'的区别。"

**【技术映射：to_excel 裸导 = 草稿纸交给老板，ExcelWriter 格式化 = 装订成册的正式报告】**

**小白**："那多个 Sheet 怎么一次导出？我记得 to_excel 写两次会覆盖。"

**大师**："用 `pd.ExcelWriter` 上下文管理器：

```python
with pd.ExcelWriter('report.xlsx', engine='openpyxl') as writer:
    df1.to_excel(writer, sheet_name='门店KPI', index=False)
    df2.to_excel(writer, sheet_name='品类趋势', index=False)
    df3.to_excel(writer, sheet_name='库存预警', index=False)
```

三行代码导出三个 Sheet。然后用 openpyxl 给每个 Sheet 加样式。"

**【技术映射：ExcelWriter = 文件夹封面——一个文件放多个 Sheet，一次装订】**

**小胖**："那如果我想要异常值自动标红呢？Excel 的条件格式 pandas 能做吗？"

**大师**："两种方式：

1. **Styler（pandas 原生）**：在 Jupyter/HTML 中做条件样式，导出为 HTML。适合 Jupyter Notebook 展示。
```python
df.style.applymap(lambda x: 'color: red' if x < 0 else '')
```

2. **openpyxl 条件格式**：导出 XLSX 后用 openpyxl 的 `conditional_formatting`。适合 Excel 交付。
```python
from openpyxl.formatting.rule import CellIsRule
ws.conditional_formatting.add('B2:B100', CellIsRule(operator='lessThan', formula=['1000'], fill=red_fill))
```

Styler 适合在 Python 生态内展示（Notebook、HTML 报告），openpyxl 适合 Excel 交付给非技术人员。"

**【技术映射：Styler = PPT 演讲幻灯片（展示用），openpyxl = Word 正式公文（交付用）】**

**小白**："那 to_parquet 和 to_csv 怎么选？之前说 Parquet 性能好，什么时候用 Parquet 导出？"

**大师**："一句话：**如果下游是另一个程序或数据分析师**，用 Parquet；**如果下游是人、要在 Excel 里打开**，用 CSV/XLSX。

| 格式 | 适用场景 | 类型保留 | 人类可读 | 压缩率 |
|------|---------|---------|---------|-------|
| CSV | 人看、简单交换 | 差（全是字符串） | 是 | 低 |
| XLSX | 老板看 | 好 | 是（需 Excel） | 中 |
| Parquet | 程序读 | 完美（dtype 保留） | 否 | 极高 |
| JSON | API 交换 | 中 | 是 | 低 |

Parquet 的核心优势是**保留 dtype**——不会像 CSV 那样'年龄从 Int8 变成 object，日期从 datetime64 变成字符串'。如果你的数据管道是多步骤的（A 脚本导出 → B 脚本读取），用 Parquet 作为中间格式是最佳选择。"

**【技术映射：CSV = 记事本便条（人都能看），Parquet = 压缩包（只有程序能高效解压），XLSX = 正式文件（老板要的格式）】**

**小白**："导出前有没有什么数据校验是必须做的？"

**大师**："导出前必须过一个'交付清单'：

1. **行数校验**：导出行数 = 预期行数（例如门店数 = 12 家）
2. **列名校验**：列名不含特殊字符、不重复
3. **类型校验**：数值列确实是 int/float 而非 object
4. **空值校验**：关键列（门店名、销售额）不含 NaN
5. **数据范围校验**：金额 > 0、百分比在 0-100
6. **合计校验**：汇总值 = 明细总和（误差 < 0.01）

这六条校验在 `to_excel` 之前执行，可以避免把'脏数据'以'漂亮格式'发给老板——格式越漂亮，错误越隐蔽。"

**大师总结**："数据导出的三个阶段：
1. **导出前**：六条校验清单确保数据正确性
2. **导出中**：ExcelWriter + openpyxl 格式化，Styler 做条件样式
3. **导出后**：选择合适格式——给人看用 XLSX，给程序读用 Parquet

记住：**格式是最后一公里的事——先把数据做对，再把格式做美。**"

### 3.2 模拟经营数据

```python
# generate_report_data.py
import pandas as pd
import numpy as np

np.random.seed(42)
stores = [f'店-{i:02d}' for i in range(1,21)]
categories = ['服装','电子','食品','家居']
products = [f'{c}-{chr(65+i)}' for c in categories for i in range(5)]

# 周销售数据
N = 3000
sales = pd.DataFrame({
    'store': np.random.choice(stores, N),
    'category': np.random.choice(categories, N),
    'product': np.random.choice(products, N),
    'sales': np.round(np.random.uniform(500, 5000, N), 2),
    'order_count': np.random.randint(1, 10, N),
    'margin_rate': np.round(np.random.uniform(0.1, 0.5, N), 4),
})
sales['margin'] = round(sales['sales'] * sales['margin_rate'], 2)

# 门店周报 KPI
store_kpi = sales.groupby('store').agg(
    销售额=('sales','sum'), 订单数=('order_count','sum'),
    客单价=('sales','mean'), 毛利率=('margin_rate','mean')
).round(2)
store_kpi = store_kpi.sort_values('销售额', ascending=False).reset_index()

# 品类汇总
cat_summary = sales.groupby('category').agg(
    销售额=('sales','sum'), 占比=('sales', lambda x: x.sum()/sales['sales'].sum())
).round({'销售额':0, '占比':4}).reset_index()
cat_summary['占比'] = (cat_summary['占比'] * 100).round(2)

# 高库存预警（模拟）
inventory = pd.DataFrame({
    '商品': [f'{c}-{chr(65+i)}' for c in categories for i in range(5)],
    '库存量': np.random.randint(10, 500, 20),
    '日均销量': np.random.randint(1, 30, 20),
})
inventory['周转天数'] = (inventory['库存量'] / inventory['日均销量']).round(1)
inventory['预警'] = inventory['周转天数'] > 90

# 周环比数据
prev_week = store_kpi.copy()
prev_week['销售额'] = prev_week['销售额'] * np.random.uniform(0.8, 1.3, len(prev_week))
store_kpi['上周销售额'] = prev_week['销售额'].round(2)
store_kpi['环比变化%'] = ((store_kpi['销售额'] - store_kpi['上周销售额']) / store_kpi['上周销售额'] * 100).round(2)

store_kpi.to_csv('store_kpi.csv', index=False)
cat_summary.to_csv('cat_summary.csv', index=False)
inventory.to_csv('inventory.csv', index=False)
print(f"门店数: {len(store_kpi)}, 品类数: {len(cat_summary)}, 商品数: {len(inventory)}")
```

### 3.3 分步实现

#### 步骤 1：导出前数据校验

**目标**：在导出前执行六条数据质量检查。

```python
# step1_precheck.py
import pandas as pd

store_kpi = pd.read_csv('store_kpi.csv')
cat_summary = pd.read_csv('cat_summary.csv')
inventory = pd.read_csv('inventory.csv')

def validate_before_export(df, name, required_cols=None, gt_zero_cols=None):
    """导出前数据校验"""
    checks = []
    # 1. 必要列存在
    if required_cols:
        missing = set(required_cols) - set(df.columns)
        checks.append(('必要列', len(missing) == 0, f"缺失: {missing}" if missing else 'OK'))

    # 2. 无完全空行
    all_null = df.isna().all(axis=1).sum()
    checks.append(('空行', all_null == 0, f"{all_null} 行全空" if all_null else 'OK'))

    # 3. 关键列无 NaN
    if required_cols:
        nulls = {c: df[c].isna().sum() for c in required_cols if df[c].isna().any()}
        checks.append(('空值检查', len(nulls) == 0, str(nulls) if nulls else 'OK'))

    # 4. 数值范围
    if gt_zero_cols:
        issues = {c: (df[c] <= 0).sum() for c in gt_zero_cols if (df[c] <= 0).any()}
        checks.append(('数值>0', len(issues) == 0, str(issues) if issues else 'OK'))

    print(f"\n===== {name} 导出前校验 =====")
    all_pass = True
    for check_name, passed, detail in checks:
        status = 'OK' if passed else 'FAIL'
        print(f"  [{status}] {check_name}: {detail}")
        if not passed:
            all_pass = False
    return all_pass

validate_before_export(store_kpi, '门店KPI', required_cols=['store','销售额'], gt_zero_cols=['销售额','订单数'])
validate_before_export(cat_summary, '品类汇总', required_cols=['category','销售额'], gt_zero_cols=['销售额'])
validate_before_export(inventory, '库存预警', required_cols=['商品','库存量'], gt_zero_cols=['库存量','日均销量'])
```

#### 步骤 2：ExcelWriter 多 Sheet 导出 + 格式化

**目标**：用 ExcelWriter + openpyxl 导出带格式的多 Sheet 报告。

```python
# step2_formatted_export.py
import pandas as pd
from openpyxl import load_workbook
from openpyxl.styles import Font, PatternFill, Alignment, Border, Side, numbers
from openpyxl.utils import get_column_letter

store_kpi = pd.read_csv('store_kpi.csv')
cat_summary = pd.read_csv('cat_summary.csv')
inventory = pd.read_csv('inventory.csv')

filename = '经营周报.xlsx'
with pd.ExcelWriter(filename, engine='openpyxl') as writer:
    store_kpi.to_excel(writer, sheet_name='门店KPI', index=False)
    cat_summary.to_excel(writer, sheet_name='品类汇总', index=False)
    inventory.to_excel(writer, sheet_name='库存预警', index=False)

# openpyxl 格式化
wb = load_workbook(filename)

HEADER_FILL = PatternFill(start_color='2F5496', end_color='2F5496', fill_type='solid')
HEADER_FONT = Font(bold=True, color='FFFFFF', size=11)
RED_FILL = PatternFill(start_color='FFC7CE', end_color='FFC7CE', fill_type='solid')
RED_FONT = Font(color='9C0006')
THIN_BORDER = Border(
    left=Side(style='thin'), right=Side(style='thin'),
    top=Side(style='thin'), bottom=Side(style='thin')
)

for ws in wb.worksheets:
    # 表头样式
    for cell in ws[1]:
        cell.fill = HEADER_FILL
        cell.font = HEADER_FONT
        cell.alignment = Alignment(horizontal='center')

    # 冻结首行
    ws.freeze_panes = 'A2'

    # 自适应列宽 + 边框
    for col_idx in range(1, ws.max_column + 1):
        max_len = max(len(str(ws.cell(row=r, column=col_idx).value or '')) for r in range(1, ws.max_row + 1))
        ws.column_dimensions[get_column_letter(col_idx)].width = min(max_len + 4, 40)
        for row in range(2, ws.max_row + 1):
            ws.cell(row=row, column=col_idx).border = THIN_BORDER

# 门店KPI：标红环比下降 > 20%
ws = wb['门店KPI']
for row in range(2, ws.max_row + 1):
    change_val = ws.cell(row=row, column=ws.max_column).value
    if isinstance(change_val, (int, float)) and change_val < -20:
        for col in range(1, ws.max_column + 1):
            ws.cell(row=row, column=col).fill = RED_FILL
            ws.cell(row=row, column=col).font = RED_FONT

# 库存预警：标红预警商品
ws_inv = wb['库存预警']
for row in range(2, ws_inv.max_row + 1):
    if ws_inv.cell(row=row, column=ws_inv.max_column).value is True:
        for col in range(1, ws_inv.max_column + 1):
            ws_inv.cell(row=row, column=col).fill = RED_FILL

wb.save(filename)
print(f"OK 格式化周报已导出至 {filename}")
```

# step3_styler.py
import pandas as pd

store_kpi = pd.read_csv('store_kpi.csv')

# === Styler 链式调用 ===
styled = (
    store_kpi.style
    # 标红负数
    .applymap(lambda x: 'color: red' if isinstance(x, (int,float)) and x < 0 else '', subset=['环比变化%'])
    # 销售额渐变条
    .bar(subset=['销售额'], color='#4472C4', width=80)
    # 百分比格式
    .format({'客单价': '{:.0f}', '毛利率': '{:.2%}', '环比变化%': '{:+.1f}%'})
    # 标题
    .set_caption('门店经营KPI周报')
    # 表头样式
    .set_table_styles([{
        'selector': 'thead th',
        'props': [('background-color', '#2F5496'), ('color', 'white'), ('font-weight', 'bold')]
    }])
)

# 导出为 HTML
html = styled.to_html()
with open('report.html', 'w', encoding='utf-8') as f:
    f.write('<html><head><meta charset="utf-8"><title>经营周报</title></head><body>')
    f.write(html)
    f.write('</body></html>')
print("OK HTML 报告已导出至 report.html")

import pandas as pd
import os

store_kpi = pd.read_csv('store_kpi.csv')

for name in ['store_kpi', 'cat_summary']:
    csv_size = os.path.getsize(f'{name}.csv')
    parquet_size = os.path.getsize(f'{name}.parquet')
    print(f"{name}: CSV {csv_size}B vs Parquet {parquet_size}B ({csv_size/parquet_size:.1f}x)")

# 读回验证 dtype 保留
df_back = pd.read_parquet('store_kpi.parquet')
print(f"\n读回后 store dtype: {df_back['store'].dtype}")  # category 被保留

### 3.4 常见坑及解决方法

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| to_excel 覆盖已有 Sheet | 默认 mode='w' | 用 `ExcelWriter(..., mode='a')` 追加，或一次写所有 Sheet |
| openpyxl 修改后保存不生效 | 没调用 `wb.save()` | 格式化后必须调用 `save()` |
| Styler 导出 HTML 中文乱码 | to_html 默认不设编码 | 写文件时加 `encoding='utf-8'` |
| Parquet 写入报错 "No module named 'pyarrow'" | 未安装 pyarrow | `pip install pyarrow` |
| to_csv 用 GBK 打开乱码 | 默认 encoding='utf-8' | 加 `encoding='utf-8-sig'` 兼容 Excel |
| float 在 Excel 中变科学计数 | 数字太大或太小 | 设置 `number_format='#,##0.00'` |

## 4. 项目总结

### 4.1 优点 & 缺点

| 维度 | pandas 导出方案 | 纯 Excel 手工 | BI 工具(Tableau等) |
|------|---------------|-------------|-------------------|
| 自动化 | 脚本一次配置，重复运行 | 每次手工操作 | 仪表盘自动刷新 |
| 格式灵活度 | openpyxl 精细控制 | Excel 手动操作 | 受限于 BI 模板 |
| 条件格式 | openpyxl/Styler 编程控制 | 手动条件格式 | 内置条件格式 |
| 多格式支持 | CSV/XLSX/Parquet/JSON 统一导出 | Excel 另存为 | 导出受限 |
| 数据安全 | 代码可审计 | 手工无审计 | 平台审计 |
| 学习成本 | 需了解 openpyxl API | 低 | 中 |

### 4.2 适用场景

- **适用场景**：
  1. 定期（日/周/月）自动化报告生成——格式固定、数据更新
  2. 需要同时交付 Excel（给人）和 Parquet（给系统）的场景
  3. 多 Sheet 多格式的复杂报表
  4. 需要条件格式（异常标红、渐变条）的监控看板
  5. 数据管道中间格式——Parquet 保留 dtype 和压缩
- **不适用场景**：
  1. 纯交互式数据探索——用 Jupyter Notebook 而非导出
  2. 需要复杂图表（如地图、桑基图）的报告——配合 matplotlib/plotly 或 BI 工具

### 4.3 注意事项

- **to_excel 的 engine 选择**：`.xlsx` 用 `openpyxl`，`.xls` 用 `xlwt`（已弃用）。新项目一律用 `.xlsx + openpyxl`；
- **to_csv 的编码**：给 Windows Excel 用户建议用 `encoding='utf-8-sig'`，避免 BOM 导致的中文乱码；
- **Styler 的性能**：Styler 是逐单元格渲染的，大数据量下很慢——建议先采样或聚合再 Styler；
- **Parquet 的压缩**：默认 snappy 压缩，可选 gzip/brotli/lz4。写入时 `compression='gzip'` 可进一步压缩但读取变慢。

### 4.4 常见踩坑经验

1. **to_excel 后打开报"文件已损坏"**：通常因为 openpyxl 版本过旧或 Excel 版本兼容问题。`pip install --upgrade openpyxl` 解决。另一个原因是在 openpyxl 操作中重复使用了已关闭的 workbook 对象。
2. **openpyxl 样式设置后数字格式丢失**：openpyxl 修改单元格样式时会覆盖 pandas 写入的默认格式——包括数字格式。解决办法：设置 fill 和 font 的 `copy=True`，或者先用 pandas 写入数值后再用 openpyxl 设置 `number_format`。
3. **Styler 在 Jupyter 中正常但导出 HTML 后样式丢失**：Styler 的 `to_html()` 默认不包含 CSS 中的某些高级样式（如 `bar` 的渐变条）。解决办法：在 HTML 中引入 pandas Styler 的默认 CSS 或手动添加。
